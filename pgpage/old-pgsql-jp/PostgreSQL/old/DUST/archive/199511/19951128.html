postgres95のテーブルをW3から検索するシステムを試してみよう！ 

このシステムは、SGI IRIX 5.3 上の postgres95-1.0 patch level 14 + jp patch の
データベースを、postgres95 に附属の tcl/tk インターフェイスでアクセスする
デモです（tcl/tk インターフェイスは、Tcl7.4p2/Tk4.0p2 + jp patchから作りまし
た）

データベースとテーブル名を選んでアトリビュートの一覧を表示させた後、ア
トリビュートに検索キーを入力して該当するレコードを表示することができま
す。検索キーとして日本語の正規表現が使えますし、int4 や oid などの数値タ
イプの場合は、範囲指定(n-m, <n, >n)も可能です。また、各フィールド内にお
いて、複数の検索キーをスペースで区切って列挙した場合は、それぞれの値の
or 検索となります。この辺の処理に興味ある方は、 ソースを御覧下さい。 

なお、postgres95 では、スキーマ定義などの情報も、 system catalog という一群
のテーブルになっており、自由に検索できます。このデモで最初のフォームの
「テーブル名」のところにデフォルト値として入っている "pg_class" も実は
system catalog の一つで、各々のテーブル（postgres95 では「クラス」と称す
る）の情報が格納されています。このデモを使って system catalog をブラウズ
してみれば、postgres95 の構造を理解するのに役立つと思います。是非御活用
下さい。
