高橋です。

動作中の postmaster のバージョン番号を知る方法はありませんか。

現在 PostgreSQL-6.5.3 を動作させている計算機（HP-UX 10.20）に
新たに 7.0.3 を入れてみました。
もしものために、6.5.3 の環境を残しておきたいと考え、
/usr/local/pgsql にインストールしていた 6.5.3 を
/usr/local/pgsql-6.5.3 に move して、
7.0.3 を /usr/local/pgsql-7.0.3 にインストールしました。

上記のインストールディレクトリ以外にファイルはコピーされる
のでしょうか。もちろんインストールの設定次第でしょうが、
prefix を /usr/local/pgsql-7.0.3 に設定した場合です。

次に、regression test での Fail がたくさんあったことは
気にせず、pg_ctl -w start によって、7.0.3 を起動したつもりになりました。
プロンプトには # がつき、日付形式は ISO 形式になりましたが、
石井さんのペジによるとなくなったという
 datetime 型や timespan 型が使えます。

将来はなくすが互換性の観点からしばらく残しておくということ
なんでしょうか？

インストール時の regression test で Fail が多く出力されたので、
正しくインストールできたのか不安になっています。


＃ regression test の select は Fail したのに、自分でＤＢを作成して、
　select したときには正常な答えが返ってくるのはどういうことなんでしょう？
