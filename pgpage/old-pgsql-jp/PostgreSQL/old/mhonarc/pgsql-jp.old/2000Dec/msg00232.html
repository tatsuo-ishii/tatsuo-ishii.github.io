鈴木と申します。

>> \copy table_name to file_name
>> \copy table_name from file_name
>> を使いデータのエクスポート,インポートができるとはおもいますが、
>> もっと汎用的にCSVファイルのインポート等はできるのでしょうか。
>> できればデリミタをフレキシブルにして様々なＤＢとの連携を考え
>> たいと考えているのですが、なにか良い方法は無いでしょうか。
>
>CVS形式のインポート、エクスポートは可能です。
>(以下は、インポートの例です。)
>
>$ cat tmp.data
>1,aaa
>2,bbb
>3,ccc
>4,ddd
>$ psql
>kaori=# create table t1 (i int,t text);
>CREATE
>kaori=# \copy t1 from 'tmp.data' using delimiters ','

私の環境
・Kondara Linux
・PostgreSQL 6.5.2
では、「\copy t1 from 'tmp.data' using delimiters ','」
を実行すると、「You haveextra tokens after the filename」とのエラーが発生します。
「copy t1 from 'tmp.data' using delimiters ','」
でしたら問題無く動作しているのですが。

PostgreSQLの「\copy」コマンドは、「copy」と違い、一般ユーザにも使用が許可されているもので、
「using 〜」以下のオプション指定はスーパユーザ権限で「copy」コマンドを
使用しなければ実行できない（指定できない）と思っていたのですが…

7.0.xでは、「\copy」コマンドは拡張されているのでしょうか。
それとも、私のサーバ設定が間違っているのでしょうか。

ご指導よろしくお願いいたします。
