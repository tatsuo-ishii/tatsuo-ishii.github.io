古賀と申します。

今回、以下の環境で、PostgreSQL7.0.3のインストールを
行っています。　その際、initdb時で、encodingエラーが生じました。

OS:RedHat6.2
CPU：Celeron500
HDD:20G
--------------------------

1 root にて、
  mkdir   /usr/local/pgsql
  mkdir  /usr/local/src/postgresql-7.0.3

 chown postgres:postgres /usr/local/pgsql
 chown postgres:postgres /usr/local/src/postgresql-7.0.3

2 postgresユーザにて
 cd /usr/local/src/postgresql-7.0.3
 tar xzf postgresql-7.0.3-patched-20001204.tar.gz 
（usr/local/src/postgresql-7.0.3にソースが展開される）

 cd usr/local/src/postgresql-7.0.3/src/

 ./configure --enable-multibyte=EUC_JP
make all
make install



ここまでは、http://www.sra.co.jp/people/t-ishii/PostgreSQL/7.0/install-basic.html
にならって行っているのですが、次の
$initdb
時に、
initdb: pg_encoding failed

Perhaps you did not configure PostgreSQL for multibyte support or
the program was not successfully installed.

と表示され、pg_encodingエラーとなってしまいます。
コンパイル時に、エラーメッセージを確認しましたが、よくわかりませんでした。

ちなみに、postgresql-7.0.3-patched-20001117.tar.gzでも同様です。

ちなみに、このマシンには、デフォルトで、ｒｐｍのpostgresgが
インストールされていましたが、 rpm -e にて削除しました。


初歩的な質問で恐縮ですが、宜しくお願いいたします。。
