たとえば、"abc" を大文字／小文字、全角／半角を無視して検索するための正
規表現は、

[aAａＡ][bBｂＢ][cCｃＣ]

となります。まだ PostgreSQL が ~* をサポートしていないとき、この方法で
大文字／小文字を無視する検索を実現していました。

おみつさんには言わずもがなですが:-)、

>あとはめんどくさいですしスピードもかかりますが、
>全角 -> 半角の変換関数でも作ってやる方法でしょうか。
>という関数を使って
[略]
>keyword ~* zen2han('英全角文字')
>
>などとしてやれば一応できます ;p

create index zenhanindex on test_tbl using btree (zen2han(keyword) text_ops);

で functional index をあらかじめ作成しておけばスピードが上がります。
