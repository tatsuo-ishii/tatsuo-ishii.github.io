SQL 文で、 テーブル定義を変更するときは、
alter table 文を使います。
構文は create table と似ています。

-- tbl_wk を作成する。
create table tbl_wk 
(
	id_tbl_wk int4,
	val_tbl_wk varchar(8)
) ;

-- tbl_wk の特定フィールドの使用桁数を増やす。
alter table tbl_wk 
(
	val_tbl_wk varchar(80)
) ;

ただ、これで川俣さんののぞみの通りに動くかどうかは、
わかりません。
boolean -> int4 の型変換のルールってあるのでしょうか。


地道な方法をとった方がいいのではないですか。

データを加工する
1.	CSV に落す。
2.	ブール値を整数値に変換する。
3.	インサート文に直す。

テーブルを作り直す。
1.	drop table
2.	create table
              flags int4
                  ~
データのリストア	
1.	新しいテーブルにインサートを実行。
