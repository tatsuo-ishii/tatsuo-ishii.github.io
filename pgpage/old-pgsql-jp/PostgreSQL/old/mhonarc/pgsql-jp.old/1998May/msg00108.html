すみません。
私、依然は Oracle をよく使っていたのですが、

Oracle では、NUMBER型という型があって、固定小数点数と
浮動小数点数を格納できるデータ型がありました。

ORACLE のマニュアルでは

ANSI SQLのデータ型と Oracle のデータ型の対応表では

  NUMERIC(p,s), DECIMAL(p,s), DEC(p,s)    = NUMBER(p,s)

で、 DB2データ型 と ORACLE データ型との対応表では

  DECIMAL(p, s) = NUMBER(p, s)

となっていたので、そのままの考え方で使い始めて、
驚いたわけです。

で、私の理解では、例えば NUMERIC(10,3) とは、

マイナスの値の場合はちょっと違うのですが、

整数部 7 桁、 小数点以下 ３桁の精度を
保った形で、データを格納できることを意味していた
と思います。
