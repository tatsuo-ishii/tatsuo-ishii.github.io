桑村です。

  Plagiaware 版のパッケージを作る際に contrib ディレクトリのモジュールを
例題としてパッケージにまとめるのに、それぞれ、ばらばらで不便を感じたので
Makefile を直しました。
  とりあえず、*.so ファイルのインストール先は $PBLIB/modules ディレクトリ
に統一してます。
  今回は、 refint.so のコンパイル時にフラッグを立てなくては failed が出る
ことを反映して contrib ディレクトリそのものに Makefile を作り対応しました。
また、README, README.jp ファイルも付けました。以下、README.jp です。

----------------------------------------------------------------------

  このディレクトリには PostgreSQL-6.3.2 に寄せられた関数やツールが
含まれます。

警告：
  以下のインストールを行なって関数を使う際は、ソースコードをご確認
になってから、ご自分の責任のもとでお使い下さい。

----------------------------------------------------------------------
内容：

 - contrib -\	コントリビューション・プログラム(ソース附属)
            |
            |- apache_logging -\	Apache WEB サーバのロギング
            |
            |- array -\		配列反復関数
            |
            |- datetime -\	日付関数
            |
            |- int8 -\		64ビット整数演算
            |
            |- ip_and_mac -\	IP と MAC アドレスの拡張
            |
            |- linux -\		バックエンド起動シェル
            |
            |- miscutil -\	アサートチェックその他
            |
            |- pginterface -\	C/4GL
            |
            |- sequence -\	通番設定
            |
            |- soundex -\	soundex 関数
            |
            |- spi -\		トリガー関数他
            |
            |- string -\	文字列変換関数
            |
            |- unixdate -\	整数から日付への変換
            |
            |- userlock -\	ユーザロック

-----------------------------------------------------------------------
インストールの仕方：

  このディレクトリにパッチを当てた後、
  
        % zcat 6.3.2_contrib_Makefiles.patch.gz | patch -p1
  
  次のコマンドの実行で C 関数のモジュールは $(LIBDIR)/modules/ にインス
トールされます。

	% make clean
	% make
	% make install


  モジュールのインストール後、実際にその関数を使いたい場合は、手動で SQL
コマンドを実行し関数の登録をする必要があります。

	% cd [module source dir.]
        % psql -e [database name] < [file name].sql
