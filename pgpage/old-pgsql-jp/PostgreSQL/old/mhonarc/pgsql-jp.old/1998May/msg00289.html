At Wed, 27 May 1998 10:38:31 +0900
t-ishii@sra.co.jp wrote:

t-ishii> o limit coredumpsize が 0 になっている(まさか)

これは、現在はまだ /etc/rc.local から httpd を起動するようにしていないため、
毎回 root アカウントで httpd を起動しています。
このアカウント (roo) は coredumpsize は 0 ではありません。

t-ishii> o 問題のプロセスが nobody かなんかで走っているため、ServerRoot に書き
t-ishii>   込みができない

ServerRoot のパーミッションは nobody.nogroup で httpd で書き込み可能に
しているつもりです。実際、PHP のアクセス記録は書き込まれています。

t-ishii> gdb を複数上げて、今いるすべての httpd に attach しておくという手もあ
t-ishii> ります:-)

やはりこの方法になるのですね...

t-ishii> EXTRA_CFLAGS= -I./jp.regex
t-ishii> を付け忘れている、っていうのがありますが、大丈夫ですか？

わたしのところでは、これは大丈夫です。

おみつさんに頂いた、漢字ファイル名のアップロードに関する変更で SEGV
が出るのを追おうとして、apache の core が出ずに試行錯誤をしております。

とりあえず、gdb 複数 attach と /tmp に cd してから core する方法、
両方試して見ます。

＃ でも、絶対どこかの設定が間違っていると思うんですよね。
＃ それがわからない...
