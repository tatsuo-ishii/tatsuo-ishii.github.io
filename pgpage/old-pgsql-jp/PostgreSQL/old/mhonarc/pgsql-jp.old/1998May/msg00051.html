僕はてっきり、 Linux の X Window System の上で動作する
単独のアプリケーションで Access のような操作間隔を持つものかと
思いましたので感激したのですが、どうやら

WWW から cgi を通して PostgreSQL にアクセスするようなものですね。
まず、それを明確にした方がよいです。

開発環境の前に、どんなものをつくりたいか明確にして
それがある程度イメージできたら、世の中に類似したものがないか調べます。
普通は、まったく世の中に似たようなものがないということはありません。

もし、存在しないならできないと思った方がよいです。

もちろん一部の天才といわれる人は別ですが。
 
そしてそれを徹底的に調べる、極端にいえば言葉は悪いがまねするのです。
そうすれば、何を勉強すべきかわかるでしょう。

誰かが Apache+PHP/FI で作成していたならば、それをそのまままねる。
これなら、Apache と PHP/FI を勉強すればよいとわかりますね。

そして分からなくなったら、その誰かにメールを出したら、もしその人が親切な
方なら教えてくれるかもしれません。

とにかく最初は人のものをまねるです。

でもまねるにも、とっかかりもわからないような一切わからないものは駄目です。
どっかで少しでも聞いたことがあるようなもので出来ているというものがよいです。

目標を高く持つのは必要なことですが、実際作業するときは
中身をばらして、細かい小さな到達可能な目標にすべきです。

ケネディ大統領も月に人間を立たせる目標を立てましたが
実際に作業をする人が、こんな目標をたてたらパニックになります。

実際に作業するには、ロケットの推力を出すにはどうするか
ロケットの重さを軽くするにはどうするか、無重力の人への影響は
など、細かく目標をわけて達成したのです。

Access95 のようなものを作成する目標は立派ですが
いきなり、そんなものを目標にすると何を勉強したらよいか
わからなくなります。

データを入れるにはどうするの、こうやれば文字がでるなど
細かい作業をしていけば、そのうちに何を勉強すればよいか
わかってきます。
そのとき、細かい作業ばかりだと、本来の目標を忘れます。
そのためにも、人のまねをできるところはまねしながら
作業していけば勉強にもなるし最終とはいえなくてもとりあえずの目的は
達成できるかもしれません。
