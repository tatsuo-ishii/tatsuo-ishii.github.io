kgotoh> > . => 不可視属性とでも言うのですか？
kgotoh> > 気が付かずに ls -l /tmpでfileがないものと勘違いしていました。
kgotoh> 
kgotoh> 「不可視属性」という表現はこの場合は適当ではありません。
kgotoh> MS-DOS や Windows などで「不可視属性」がついているファイルは
kgotoh> 明らかにファイルのハンドリングが他の物と異なります。このように
kgotoh> ファイルの「属性」となっているものは「不可視属性」と呼んでも
kgotoh> いいでしょう。
kgotoh> 
kgotoh> しかし、.cshrc などは UNIX では 'dot file' という様な呼ばれ方をします。
kgotoh> フィル名が . (dot) で始まるファイルは ls では -a オプション無しでは
kgotoh> 表示できないとか、シェル上でワイルドカード展開する際に特に指定され
kgotoh> なければ( .[a-z]* とか…)対象にならないという「プログラム仕様」の
kgotoh> 世界の話しです。
kgotoh> 
kgotoh> # dot で始まるものと言っても、. と .. は特別扱いですが
kgotoh> 
kgotoh> 他のいかなるコマンド(プログラムインタフェース上)でも dot file は
kgotoh> 他のファイルと同様に扱えるのですから、「不可視属性」とは異なるものです。
