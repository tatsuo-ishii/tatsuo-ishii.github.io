座光寺といいます。

地図情報に利用としているのですがインデックスとして
RTreeを用いるにはどのようにしたら良いのでしょうか？

今以下の様なテーブルをこしらえて
Table = vvpoint
+--------------------+-----------------------+------+
|      Field         |         Type          |Length|
+--------------------+-----------------------+------+
| id                 | int4                  |    4 |
| layer              | int2                  |    2 |
| point              | point                 |   16 |
+--------------------+-----------------------+------+
これにデータを６万件弱ほど
copy vvpoint from '/tmp/point1.txt';
で投入したまでは良かったのですが、、、

select '' as one, p.* from vvpoint p where p.point ~= '(10.5,11.5)';
とかしますと５秒程かかって結果が表示されます。

インデックスを生成してないから仕方ないのかと思い
それじゃインデックスを作ろうとしたのですが、
create index vvpoint_index on vvpoint using rtree (point);
とすると
ERROR: Can't find a default operator class for type 600.
と蹴られてしまいます。
point型に対するインデックス生成の為の比較条件式みたいなものが必要な
のかと思って調べているのですが答えが見つからずに居ます。

ここに書いてあるよとか助言頂けると幸いです。
