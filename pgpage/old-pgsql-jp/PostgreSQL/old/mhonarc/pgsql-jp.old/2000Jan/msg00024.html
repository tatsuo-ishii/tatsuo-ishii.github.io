postmasterは

 123 ?        S      0:00
/usr/local/pgsql/bin/postmaster -D/usr/local/pgsql/data -S -i

と起動している状態です。  postmaster起動時のスクリプトでは正常に起動しなかっ
たので

#!/bin/sh

if [  -x /usr/local/pgsql/bin/postmaster -a -d /usr/local/pgsql/data ]; then
     rm -f /tmp/.s.PGSQL.5432
     echo "Starting PostgreSQL postmaster..."
     su postgres -c
"/usr/local/pgsql/bin/postmaster -D/usr/local/pgsql/data -S -i"
fi

に変更しております。 この状態でregression test を行っても同じ状態です。

/etc/service で 5432 を記述:する必要があるのでしょうか？
