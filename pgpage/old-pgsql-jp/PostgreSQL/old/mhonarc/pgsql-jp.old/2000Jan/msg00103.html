ということはPostgreSQL的にはやっぱり無視でいいんでしょうか?

> また、webアプリケーションの場合は何回かコード変換することも
> 有るので、非対称な変換はちょっと気持ち悪いです。

非対称な変換と言えば、IBM拡張文字に2つだけ非対称変換にならざるを得ない
文字があります。SJIS で言うと 0xfa54 と 0xfa5b ですが、これらは EUC
code set 1 (JIS X0208) に変換されますが、逆に EUC -> SJIS の変換をして
も IBM拡張文字には戻りません。

> > ところで、NEC選定IBM拡張文字って実際に使われているのですか?M$でさえも
> > 「使わないように指導している．」とのことですが:-)

> 私は「指導された」 記憶が無いので良く分かりませんが :)

> 
>  ・同じく "IMEパッド" から "メモ帳" だと ED40h をカット&ペースト
