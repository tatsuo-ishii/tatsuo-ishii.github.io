石井です。

> 現在 Solaris2.6 上で PostgreSQL6.5.2 を運用して開発を行っている
> のですが、同時アクセスユーザがデフォルトの値だとすぐにパンクしてしまい、
> 困っております．セットアップをした際に /etc/system を編集して共有メモリ
> を増やしてようやく(?)バックエンドが立ち上がるようになったのですが、
> これ以上増やしてもシステム的に問題はないのでしょうか？

単に増やす分にはあまり問題ないと思いますが...

> 理想としては２５０人程度が同時にアクセスできるようにしたいと考えている
> のですが、このためには具体的に何をすればよろしいのでしょうか？
> 「シーラカンス本」を見てやろうとしたのですが、LinuxとFreeBSDの例しか
> なく、自分のシステムではどのようにすればよいのかがわかりません．

あれをそのまま Solaris に当てはめれば良いのですが。Solairs に精通した
人に聞いてみた方が良いかも。

> なお、マシンの物理メモリは１２８MBとなっております．

この程度のメモリでは 250人のサポートはちょっと無理かと思います。せめて 
512MB、できれば 1GB 以上にしたいところですね。もちろん swap が十分あれ
ば 128MB でも動くことは動くでしょうが...

それはともかく、250人をサポートする設定ですが、最低限の値としては、

set shmsys:shminfo_shmmax=268435456
set semsys:seminfo_semmns=256
set semsys:seminfo_semmni=16
set semsys:seminfo_semmap=18

という感じでしょうか。shmmax の方は概算です。実際に postmaster を起動
してエラーになるようでしたらもっと増やしてください。
