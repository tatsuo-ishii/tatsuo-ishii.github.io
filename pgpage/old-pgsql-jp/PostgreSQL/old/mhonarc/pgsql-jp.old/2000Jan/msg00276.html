岡さんの御書きのように database の情報は pg_database あたりに書かれて
いるはずですが、この file には他の情報も記述されているはずです。ですか
ら、おそらく移動したい database の data file だけを持っていっても期待
の動作はしないはずです。

ですから、期待の動作をさせるためには、所定の命令を使う方が better だと
思います。

その方法としては、たとえば
1./usr/local/pgsql 以下を全部移動する
→computerB に今迄あった database がなくなってしまう危険性がとても大き
いです。

2.pg_dump あたりを使って一回 computerA の database をとりだす。それを 
computerB 上で読み込ませる。

などが考えられます。

個人的には、下手に pg_database などをいじって対処するよりは、2 のよう
な手段を使った方が無難で良いのでは？、と思います。
