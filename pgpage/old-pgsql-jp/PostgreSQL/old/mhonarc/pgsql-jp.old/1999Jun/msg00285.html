[pgsql-jp 8851] is /usr/local/pgsql/ necessary?

  • From: Shoichi Kosuge <s-kosuge@str.hitachi.co.jp>
  • Date: Tue, 15 Jun 1999 21:30:05 +0900
こすげです。

　前から思っていたのですが、
　インストールする時に、/usr/local/pgsql/ ディレクトリがないと
次のようなエラーが出ます。

postgres@bash-2.02$ gmake install
gmake lexverify
gmake[1]: Entering directory `/home/postgres/postgresql-6.5/src'
gmake -C lextest all
gmake[2]: Entering directory `/home/postgres/postgresql-6.5/src/lextest'
gmake[2]: Nothing to be done for `all'.
gmake[2]: Leaving directory `/home/postgres/postgresql-6.5/src/lextest'
gmake[1]: Leaving directory `/home/postgres/postgresql-6.5/src'
gmake -C utils install
gmake[1]: Entering directory `/home/postgres/postgresql-6.5/src/utils'
gmake[1]: Nothing to be done for `install'.
gmake[1]: Leaving directory `/home/postgres/postgresql-6.5/src/utils'
gmake -C backend install
gmake[1]: Entering directory `/home/postgres/postgresql-6.5/src/backend'
mkdir /usr/local/pgsql/lib
mkdir: cannot make directory `/usr/local/pgsql/lib': ファイルもディレクトリもあ
りません。
gmake[1]: *** [/usr/local/pgsql/lib] Error 1
gmake[1]: Leaving directory `/home/postgres/postgresql-6.5/src/backend'
gmake: *** [install] Error 2
postgres@bash-2.02$

　原因は、/usr/local/ の下に、2階層下の/usr/local/pgsql/lib/
を作成しようといているところです。
　で、ディレクトリを作成して、もう一度gmake install すると
ちゃんとうまくいくのですが、いまいちスマートでないな、という
気がします。みなさんどのように解決されてますでしょうか。

・環境
  Solaris 2.6
  Sun Ultra 5 (270MHz UltraSPARC-IIi)
  128MB Memory
  gcc 2.7.2.1
  bison  1.25
  make 3.76.1
  flex 2.5.4
・configure オプション
  ./configure --with-mb=EUC_JP
