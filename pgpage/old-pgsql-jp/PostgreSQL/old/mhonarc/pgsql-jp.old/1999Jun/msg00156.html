先日、自己紹介を済ませたばかりの杉原と申します。
初めて質問させていただきます。

シーラカンス本と、PostgreSQL詳解の２冊の本を
頼りに、いろいろと試しています。
Apache1.3.4+PHP3.0.6+PostgreSQL6.4.2は、
パッチ済みのものを石井さんのところから持ってきて
インストールしました。
PHPによる接続もPerl+CGIによる接続も、すべて
うまくいっているのですが、気になることがあります。
PSQLで接続して、「\d + テーブル名」でテーブルの定義を
表示させようとすると、

ERROR:  typeidTypeRelid: Invalid type - oid = 0

というエラーが出るのです。
selectとかupdateとか、いろんな操作は支障なく
できるのでほったらかしてあったのですが。。。
pgsql_perl5のインストールの時、テストスィートを
実行した時に、設定を間違えてしまい途中で落ちたので
もう一度実行すると、「pgperltestデータベースは既に
存在します」というメッセージが出て、何度やっても同じでした。
それで、「データベースを消せばいいのか」と思い、
destroydb pgperltestとやったところ、
これも失敗して、

ERROR:  typeidTypeRelid: Invalid type - oid = 0
destroydb: database destroy failed on pgperltest.

のメッセージがここでも表示されました。
にもかかわらず、気にせずそのままmake installしたの
ですが、先に書いた通り、Perl+CGIによる接続は
支障なく動作しているように見えます。
一体何が間違っているのでしょうか？

初心者ですので、いろいろと質問が的外れかも知れませんが、
このエラーが出なくなって、データベースを削除できるように
するにはどうしたらいいのか、原因究明のための調査方法など
何でも結構ですのでお教え下さい。
もしかして、つまらないＦＡＱだったらお許し下さい。
過去ログからはこの問題は探し出せませんでした。
よろしくお願いいたします。
