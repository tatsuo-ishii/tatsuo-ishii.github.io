　片岡 様、藤沼 様

　MDACをダウンロードしました。

> "Do not install Driver Manager"を選択した場合、条件（インストール済みのド
> ライバマネージャのバージョン）によってはPostgreSQL ODBC Driver自身のインス
> トールが却下されます。

　もう一台、Windows95機を繋いで、PostgreSQL ODBC DriverをDo not install Driver
Managerでインストールしようとしたら、却下されてしまいました。
　英文なので、正解がどうか判りませんが、「ODBCがないから出来ない」ようなメッセー
ジでした。

　仕方がないので、Install Driver Manager で実行したら、やはり32ビットODBCは英語
になってしまいました。

　MDACで日本語に戻そうとしたら、DCOM95がないから、インストールできない旨の表示が
出て慌てましたが、MDACのページから辿っていくと、英語ながらDCOM95のダウンロードに
辿り着きましたので、これも拾って来ました。

　結果の判らないものをインストールするのは少し怖かったのですが、まずDCOM95をイン
ストールし、続いてMDACをインストールしたところ、無事に32ビットＯＤＢＣが日本語に
戻りました。

　もう一台も同様の方法で32ビットODBCを修正して、PostgreSQLのクライアントは2台と
なりました。ただ、両機とも同じpentium133MHz機なのに、クエリーの実行速度が違った
りして、ちょっと首を捻ってます。メモリ容量まで同じにしてあるんですけどね。不思議
‥‥
　いずれ、実行速度の謎は解いてやろうと思っています。

　お二方のご指導に感謝いたします。
