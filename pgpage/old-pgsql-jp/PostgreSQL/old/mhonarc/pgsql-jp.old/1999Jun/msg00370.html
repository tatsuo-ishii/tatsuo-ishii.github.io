私のPowerBook 2400c も昨晩 Kernel 2.2.1 にあげてみました。
PowerBook 2400c/180 (メモリ80MB、内蔵ディスク4GBに換装済)
LinuxPPC R4
GNU Make 3.76.1
gcc egcs-2.90.25
GNU Bison 1.25
flex 2.5.4

御覧のように、egcs のバージョンが違うだけで、基本的に杉原さんとほとん
ど同じ構成です。これで

PostgreSQL6.5 + Apache 1.3.6 + PHP3.0.9

を動かしてみましたが、Kernel 2.1.115 などと違って全くカーネルがクラッ
シュすることもなくなり、非常に快適です。杉原さんの2.2.1のお話をお聞き
しなければ多分 2.1.24 のままでいったことでしょう。お礼を申し上げます
(2.1.24 では PCMCIA カードが使えないのが致命的！)。

ところで、私の環境では杉原さんのところで出ている -O2 の問題は発生しな
いので、カーネルと言うよりも、コンパイラのバージョンで現象が表面化した
りしなかったりするようです。この問題はもともとは PostgreSQL のコーディ
ングのポータビリティのなさによるもので、現在はTODOに追加されており(6.5
ではなくてcurrent--開発中バージョン)、6.6 で修正されるはずです。

ところで、-O0 による解決ですが、-O2 との性能差が相当あり、あくまで一時
しのぎであると御理解下さい。
