取りあえず、Solaris2.6( x86 )版のパッケージを作りました。
会社のサーバは私用では使えないので、プロバイダにアップしようかと思ったら
何と9,859,597バイトもありました。
LinuxやFreeBSDでもこんなもんですか？

ちなみに、作成条件は
cd postgresql-6.5/src
./configure --with-template=solaris_i386_gcc --with-mb=EUC_JP
--with-perl --with-tcl --with-odbc
gmake all
gmake install
cd postgresql-6.5/doc
gmake install
cp postgresql-6.5/COPYRIGHT /usr/local/pgsql/
cp postgresql-6.5/HISTORY /usr/local/pgsql/
cp postgresql-6.5/INSTALL /usr/local/pgsql/
cp postgresql-6.5/README /usr/local/pgsql/
cp postgresql-6.5/register.txt /usr/local/pgsql/
です。

インストールは、
tar -xzf pgsql.6.5.i86pc.Solaris.2.6.pkg.tgz
su
pkgadd -d . pgsql
です。

アンインストールは
su
pkgrm pgsql
です。

というわけで本題ですが、うちのプロバイダのディスクスペースには入り切らなかったので
ftpで送ろうと思ったのですが、/incoming/jukディレクトリには書き込み権限が無い様です。
どうしましょうか？
