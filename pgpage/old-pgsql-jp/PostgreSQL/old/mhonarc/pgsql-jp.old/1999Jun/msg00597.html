サーバークライアント形式のDBをいじってみたくて、というか、知識が必要にな
り参加しました。
確か、Linux 1.2.13の上だったと思いますがPostgre95(SQLの前進？）をインス
トールに成功した記憶があります（残念ながらバリバリ使った記憶ではありませ
ん）。調子にのってSolaris でもインストールしようとしましたが１週間格闘し
できなかったのでめげてしまいました。

その後、IIS+ACCESSや、DBと連携しない単なるcgiで満足する生活を送っていた
のですが、それではすまない状況になってきました。


さて、いきなり質問で申し訳ないのですが、Initdbの段階で
3行ほど処理したようなメッセージが出て、....pgsql/pg_hba.conf.sampleを使
った云々と言ったあと、

 Valid username not given. You must specify the username for the
Postgres superuser for the database system you are initializing,......

というメッセージが出て先に進ませてくれません。

$initdb --username=postgres （以下略）としても環境変数USERにpostgresとセ
ットしても同じです。

ただし、本当にユーザー名が認識されていないのかと思い、initdb(シェルスク
リプト）にecho文を挿入して $POSTGRES_SUPERUSRNAME がどう認識されているか
チェックしてみるときちんとpostgresと出ます。

一方、スクリプト中ほどの記述にヒントを得て、コマンドラインから
pg_id postgres と入力してみたところ、「NOUSER」と返ってきます。

これは何を意味しているのか、どなたかご存知ないでしょうか？
installの不成功でしょうか？

昨日このＭＬアーカイブをダウンしてみましたが、これとまったく同じ症状の方
はいらっしゃらなかったと思います（余談ですが、私のマシンでは約7000枚の
JISファイルをnkfのバッチ処理でsjisにコンバートするのに３時間かかりまし
た。普段マシンパワーをろくに使っていない（入力待ちがほとんどの）マシン
が、こき使われているのを見てなんとも言えぬ快感を覚えました。こんな感覚は
私だけでしょうか？）。

私の環境は日本語Linux5.2（RedHat）で、PostgreSQLに関してはデフォルトの設
定だと思います。PG_VERSIONは6.3です。

「だと思います」というのは、ある本の勧めで、動作確認(initdb)もしないで、
6.4へrpm形式でアップグレードしましたが、冒頭の症状が出るので、一旦すべて
削除し、ソースから6.4をインストールしなおしてみました。しかしやはり同じ
症状なので、また6.3のrpmをインストールし直して、もとの状態に戻ってみた
という次第です。いずれにせよ私の環境ではいつも同じところでとまってしまっ
ているということです。

ひょっとしてこれはPostgreSQL以外の部分の問題なのか....？

どなたか何かヒントになるようなことをご存知でしたらご教示ください。よろし
くお願いします。
