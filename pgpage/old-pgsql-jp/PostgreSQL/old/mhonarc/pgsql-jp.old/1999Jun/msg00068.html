小菅です。

At 18:49 +0900 99.6.3, Takeshi Sugai wrote:
> 菅井と申します。
> Solaris 2.6 で  postgresql-6.4.2 をインストールしようとしたところ、
> 以下のようなエラーが生じました。
> 
> 基本的には、
> http://www.sra.co.jp/people/t-ishii/PostgreSQL/6.4/install-basic.html
> のページにある通り、下記を実行して、makeしました。
> だれかわかる方、アドバイスお願いします。

　私もSolaris 2.6 ですが、6.4.2 をmake できます。
いちおう、参考までに。

・環境
  Solaris 2.6
  Sun Ultra 5 (270MHz UltraSPARC-IIi)
  128MB Memory
  gcc 2.7.2.1
  bison  1.25
  make 3.76.1
  flex 2.5.4
・configure オプション
  ./configure --with-mb=EUC_JP


> make[1]: Entering directory `/opt/postgres/postgresql-6.4.2/src/utils'
> gcc -I../include -I../backend     -Wall -Wmissing-prototypes   -c version.c 
> -o version.o
> version.c: In function `ValidatePgVersion':
> version.c:79: warning: subscript has type `char'

　私のところでは、postgresql-6.4.2/src/utils/ の下で以下を
実行すると、正常終了します。

postgres@bash-2.02$ gcc -I../include -I../backend     -Wall -Wmissing-prototypes
   -c version.c -o version.o
postgres@bash-2.02$

　原因は、、、わかりません。実行されるgcc のバージョンとかは確認
しましたでしょうか。新しいものを入れたつもりが古いバージョンの
ものを動かしていたとか。つまり、/opt に最新版を入れたのに、
/usr/local/bin/ のものが動いていた、というようなことはないで
しょうか。
