　時間がかかる理由は LN( n ), LOG( n )において、n の有効桁数
が数百桁になっているためですが、このテストを regression test
に組込む必要はどこにあるのでしょうね？

  開発時の単体テストとしては、数百桁の限界値テストも必要かと
思いますが、 regression testのような総合テストでは、このよう
な非現実的なテストは不要だと思います。

  私は、このテストが単に時間を無駄にする、ユーザにとってテス
トする価値のない無意味なテストだと思ってます。（有効桁数が30
桁ぐらいでしたら有効なテストだと思います。)

　この regression (big)test のために、PostgreSQLユーザに対し
て「PostgreSQLの不安定さを印象づけてしまう」有害とも言えるテ
ストパターンになっていると思っているのですが..

　いかがでしょうか？
