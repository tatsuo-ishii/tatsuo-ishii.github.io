はじめまして， 中村繁雄と申します。
いろいろ戸惑いながらPostgreSQLを勉強中の初心者です。
よろしくお願いします。

さて，インストール時に以下のような不具合が発生しました。
対処方法を教えていただけるとありがたいのですが。

PC=自作 MPU=Pentium133MHz  Mem=80MB
   Disk 2台接続 Primay は OS=Window95
　　　　　　　　Secondary は OS=Turbo Linux 3.0

PostgreSQL 6.4.2 インストールの手引(基礎編)に従って，インストール
しています。　使用したソースはPostgresql-6.4.2-patched.tar.gz(99.4.14)です。

当初メモリーは64MBで，99年1月に入手したソースPostgresql-6.4.2-patched.tar.gz
では，問題なくインストールできていました。

最近，99.4.14日版と入れ替えようとして，トラブルが発生したものです。
PCのメモリーを80MBに増設した状態でインストールすると，
./configure --with-xyz(いくつかのオプションを設定）
この後の　make all で，しばらくログを吐き出した後，compile Error の
メッセージが出てプロンプトに戻ります。
（正常インストールと比較すると，プロンプトに戻る時間は数分の１と短い）
Liloの入力でmem=80Mと設定しても，しなくても結果は同じです。

このため，以前正常にインストールできた状態（PCのメモリーを64MBに戻す）にして
インストールしたところ，上と同じオプションの設定で正常にインストールできました。

メモリー容量に関する注意やトラブルに関する報告は今まで見たことが無いので，
この現象は私のPCに特有のことなのでしょうか。
