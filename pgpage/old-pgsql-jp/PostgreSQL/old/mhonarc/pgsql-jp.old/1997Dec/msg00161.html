The easiest remedy I found (makes gcc work on all Solaris releases :-)
is keeping only the following files in gcc-dir/include:

ls -l /opt/arch/gnu/lib/gcc-lib/sparc-sun-solaris2/2.7.2/include
total 70
-rw-r--r--   2 root     root        3717 Jan 13  1994 float.h
-rw-r--r--   2 root     root        3049 Dec  5  1995 limits.h
-rw-r--r--   2 root     root         137 Jun 15  1993 proto.h
-r--r--r--   2 root     root        5111 Dec  5  1995 stdarg.h
-r--r--r--   2 root     root        8868 Dec  5  1995 stddef.h
lrwxrwxrwx   1 root     root          21 Aug  4 12:36 syslimits.h -> /usr/include/limits.h
-r--r--r--   2 root     root        6826 Dec  5  1995 va-sparc.h
-r--r--r--   2 root     root        5111 Dec  5  1995 varargs.h

And then edit the <gcc-dir>/specs file, append to theline under *cpp: the
following:  %{!ansi:%{!traditional:-D__STDC__=0}}

This copies how Sun's include files expects C compilers to behave.
〜略〜

 自宅の 2.6 Intel は、これに準じた方法で使っております。
