桑村です。

  Plagiaware という日本語 Linux パッケージのβ版が公開されました。

	ftp://ftp.jaist.ac.jp/pub/os/linux/Plagiaware

  Plagiaware は Slackware をベースにした日本語 Linux パッケージです。実は、
この Plagiaware に日本語 PostgreSQL と日本語 PHP/FI モジュール組み込みの
Apache を提供しようとしています。これら PostgreSQL 関連のアーカイブは、現
在、JAIST のPlagiaware のFTP ディレクトリ下の plagia/contrib/Undocumented
にありますが、以下はその最新版です。

  以下、12/15 版 Apache, PHP/FI, PostgreSQL バイナリアーカイブの説明です。

	http://www.rccm.co.jp/~juk/outgoing/pgsql/pgsql62b.tgz
	http://www.rccm.co.jp/~juk/outgoing/pgsql/regex_jp.tgz
	http://www.rccm.co.jp/~juk/outgoing/pgsql/pgaccess.tgz
	http://www.rccm.co.jp/~juk/outgoing/www/apache1b.tgz
	http://www.rccm.co.jp/~juk/outgoing/www/phpfi2b.tgz

  また、http://www.rccm.co.jp/~juk/outgoing/original/ の下のサブディレクトリ
にオリジナルのソースとパッチのアーカイブが置いてあります。


	o pgbrowse は現在の管理者がわからないのではずし、そのかわりに
	 pgaccessを入れてあります。

	o GD ライブラリは UNISYS の LZW 圧縮アルゴリズムを含むフリーソフト
	 配布へのライセンス形態が有償に変更されたらしいのでやめました。

	o PHP/FI のライブラリは Apache とは分けました。PHP/FI の日本語
	 ドキュメント(html) を htdocs/php/doc/doc-j.html に置きました。

	o PostgreSQL のドキュメントを /usr/doc/pgsql/ の下に移し、日本語
	 FAQ を入れました。

	o regex_jp ライブラリを PostgreSQL と分けました。
	（regex_jp の共有ライブラリ化は PostgreSQL 日本語パッチの作者
	  (石井達夫さん)が素早く対応して下さいました。感謝します。）

--
apache1b: PHP/FI 対応 apache httpd 1.2.4

 PHP/FI(+LIBPQ)をモジュール組み込みした Apache サーバです。 PHP/FI 経由で
PostgreSQLへのインターフェースライブラリ LIBPQ を呼出し可能にしています。
PHP/FIの共有ライブラリと PostgreSQL がインストールされている必要がありま
す。PHP/FIの記述を含むページの拡張子は、 ".phtml" とすれば認識可能です。
 また、 URL でユーザホームを '~' をユーザ名の前に付けて表すのを '~' がな
くても認識できるようにするためのパッチも当てています。この機能を作動させ
るために srm.confの中で "UserDirWOTilde on" を設定してます。


phpfi2b:	PHP/FI 2.0b12 + kanji2.0b12.patch-fix2

  PHP/FI は Perl に似た記述によるHTMLファイル埋め込み型の簡単なプログラム
言語で、HTMLファイルに直接簡単なスクリプトを置くこのが可能です。本パッケ
ージでは Apache ウェブサーバのモジュールとして組み込むための共有ライブラ
リとして提供しています。また、PostgreSQLのPostgreSQLへのインターフェース
ライブラリ LIBPQ を呼出し可能にしているため、 PostgreSQL がインストールさ
れている必要があります。
  

pgsql62b:	PostgreSQL-6.2.1 + 6.2.1jpPL2.patch

  POSTGRES はカリフォルニア大学バークレー校(UCB)で開発されたオブジェクト
リレーショナル・データベースマネージメントシステム(ORDBMS)です。サーバ／
クライアント・モデルとなってますのでネットワーク経由で利用できます。UCBで
の開発は終了しましたが、現在はその SQL 版である PostgreSQL の開発がネット
上で行なわれています。日本語(EUC)化により日本語の正規表現による検索も可能
となっています。EUCの正規表現のプログラムを使うために regex_jp ライブラリ
がインストールされている必要があります。
  文字端末用クライアントプログラムの psql は readline を必要とします。


regex_jp:	日本語 PostgreSQL 用正規表現ライブラリ

  日本語 GNU awk用に作成された日本語正規表現ライブラリを
PostgreSQL-6.2.1-jpPL2 用にした物です。 PostgreSQL 本体とはライセンス形態
が異なるため別のアーカイブとしてまとめています。


pgaccess:	pgaccess-0.62 + jp.patch

  Tcl/Tk で書かれた PostgreSQL 用インターフェース・プログラムです。
wish4.2jpで日本語の入力も可能であることが確認されています。表の定義やデー
タの入力の他、スプレッドシート上に検索したデータ(200件まで)を表示し、その
データを直接修正したりもできます。
