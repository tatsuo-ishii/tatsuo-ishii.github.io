あの，こけているのはもしかして，src/bin/pg_passwd の下で
pg_passwd を作成するところでしょうか?  もしもそうで，
pg_passwd.c の 2 行目に Y. Ichikawa とあれば私が
以前に作成したもので，確かに index 関数を使っております．
ですが，元々 Solaris2.5.1_sparc で開発したものですので，
OS の library 構成の問題では無いと思います．
不安な場合は以下のテストをしてみて下さい:

	% /usr/ccs/bin/nm /usr/lib/libc.a | grep index

結果に index 関数の登録が含まれていれば OK のはずです:

	/usr/lib/libc.a[index.o]:
	[6]     |         0|      12|FUNC |GLOB |0    |2      |index


それから，PosgreSQL とは関係の無いところの，例えば gcc/gmake
あたりの configuration に問題があるのかもしれませんので，
とりあえず -v 付の make してみることをお勧めします:

	% cd src/bin/pg_passwd
	% make clean
	% make CC="gcc -v" all

これの結果を添付して頂けると，問題解決が早いのでは無いかと思います．
