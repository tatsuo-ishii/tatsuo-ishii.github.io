>ちょっとですが解決の糸口が見えたような気がします。
>
>さて、今回は素朴な質問です。過去のメールアーカイブ、マニュアルを探したのですが
>うまく発見できませんでしたので、よろしくご教示下さい。
>
>１）TEXTの大きさは何bytesなのでしょうか？
>　　large objectが8KB以上なので、それより小さいことは推測できるのですが
>　　可変長とはいっても制限があると思われますので。

制限は「1 レコードの大きさが 8KB を超えては行けない」ということだけで
す。したがって、もし TEXT タイプのカラムが１個だけのテーブルの場合、
8KB 近くの TEXT を作れます。

>２）配列の個数は明示的に与えるのが普通だと思うのですが、たとえば
>　　create tabel table1 { text[]};
>    という配列個数を宣言しないテーブルを作成できるのでしょうか。

はい、できます。

>
>３）たとえば配列text[]を宣言し、n個データが入っているとしてk番目（n>k)の
>　　データを削除したとき、そのあいた部分を詰めるのは自動的に行われるのでしょうか
>　　それともvacuumや何らかの操作で明示的に詰めるのでしょうか？

うーん。そういえば、そもそも配列の特定の要素だけを削除する方法ってある
のかな？

>４）Postgresqlの特徴として継承があげられますが、どの位の階層？継承できるのでしょ
>　　うか？
>　　create table table1 {}
>    create table table2 {}(inherit table1)
>    create table table3 {}(inherit table2)
>　　というのがどの位まで可能なのでしょうか。できればここまでやったことがある
>　　という例をお教え願えれば幸いです。

やったことがないのでパス。

>５）さらに、たとえば上の例ではtable3はtable1,table2の”構造を継承”するのであっ
>　　て、table1に入力したデータはtable3から検索できるわけではないというふうに
>　　理解していますが、これはまちがいでしょうか？データまで継承できるのでしょうか

「構造を継承する」であってます。

>６）複数のテーブルを１つのテーブルに継承できるのでしょうか？たとえば
>    create table table3 {}(inherit tabel1,table2)とできるのでしょうか？

できません。
