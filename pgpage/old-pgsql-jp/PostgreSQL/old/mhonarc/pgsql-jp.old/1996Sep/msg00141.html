1.2.  A Short History of the POSTGRES Project

1.2. POSTGRES プロジェクトのショートストーリー

Implementation of the POSTGRES DBMS began in 1986.  The initial
concepts for the system were presented in [STON86] and the definition
of the initial data model appeared in [ROWE87].  The design of the
rule system at that time was described in [STON87a]. The rationale and
architecture of the storage manager were detailed in [STON87b].

POSTGRES DBMS の開発は1986年に始まりました。システムの初期の概念は[STON86]
で発表されました。そして、初期のデータモデルの定義は[ROWE97]で公開されました。
当時のルールシステムの雛型は[STON87a]で説明されました。リレーショナルと
ストレージマネージャーの原理とアーキテクチャは[STON87b]で詳しく説明されました。

コメント：
    rationale: 関係 -> 原理
    storage:   収容 -> ストレージ(そのままの方が通りが良いのでは？)

POSTGRES has undergone several major releases since then. The first
"demoware" sys tem became operational in 1987 and was shown at the
1988 ACM-SIGMOD Conference. We released Version 1, described in
[STON90a], to a few external users in June 1989. In response to a
critique of the first rule system [STON89], the rule system was
redesigned [STON90b] and Version 2 was released in June 1990 with the
new rule system. Version 3 appeared in 1991 and added support for
multiple storage managers, an improved query executor, and a rewritten
rewrite rule system. For the most part, releases since then have
focused on portability and reliability.

POSTGRES はその後数種類のメジャーリリースが行われました。最初のデモシステムは
1987年に動かせるようになり、1988年にACM-SIGMOD 会議で示されました。
私達は[STON90a]に記述された Version 1 を1989年の 6 月に２、３の外部の
ユーザーにリリースしました。最初のルールシステムである[STON89]の批評に
答えて、ルールシステムは再設計され[STON90b]新しいルールシステムとともに
1990年 6 月に Version 2 がリリースされました。Version 3 は 1991 年に公開され
マルチプルストレージマネージャー、改良されたキュアリーエグゼキューター、
新しく書き直されたリライトルールシステムのサポートを加えました。
その後のリリースの多くの部分は移植性と信頼性に中心がおかれました。

コメント：

    query executer: 専門的な用語もしくはPOSTGERS固有の用語と思われる。
    rewite rule system: 専門的な用語と思われる。


POSTGRES has been used to implement many different research and
production applica tions.  These include: a financial data analysis
system, a jet engine performance monitor ing package, an asteroid
tracking database, a medical information database, and several
geographic information systems. POSTGRES has also been used as an
educational tool at several universities.  Finally, Illustra
Information Technologies picked up the code and commercialized it.

POSTGRES は多くの異なる研究や生産アプリケーションを実装するのに使われて
きました。これらは財務分析、ジェットエンジンパフォーマンスモニタのための
パッケージ、小惑星追跡データベース、医療情報データベース、多くの地理的情報
システムなどを含みます。POSTGRES はまた教育のためのツールとして多くの大学
で使われています。最終的に Illustra Information Technologies がコードを
ピックアップして商業化しました。

コメント：
    適用例を増やしました。

POSTGRES became the primary data manager for the Sequoia 2000
scientific computing project in late 1992. Furthermore, the size of
the external user community nearly doubled during 1993. It became
increasingly obvious that maintenance of the prototype code and
support was taking up large amounts of time that should have been
devoted to database research.  In an effort to reduce this support
burden, the project officially ended with Ver sion 4.2.

POSTGERS は 1992年の後半に、Sequoia 2000 scientific computing roject の主要
なデータマネージャとなりました。そのうえ、外部利用者共同体の数は 1993年を
通じておよそ倍になりました。これは段々明らかになったことなのですが、
プロットタイプコードの維持やサポートは本来データベース研究に捧げるべき時間の
大部分を必要としていました。このサポートの負担を減らす努力により、
プロジェクトは Version 4.2 において公式に終了しました。

1.3.  What is POSTGRES95?

  POSTGRES95とは何か？

POSTGRES95 is a derivative of the last official release of POSTGRES
(version 4.2). The code is now completely ANSI C and the code size has
been trimmed by 25%. There are a lot of internal changes that improve
performance and code maintainability. POSTGRES95 runs about 30-50%
faster on the Wisconsin Benchmark compared to v4.2.  Apart from bug
fixes, these are the major enhancements:

POSTGRES95 は、POSTGERSの最後の公式リリースである Version 4.2 から
派生したものです．そのコードは完全にANSI Cで書かれ，コードのサイ
ズは25 %にまで小さくなりました．多くのパフォーマンスとコードの維持性を
改善した実質的な改良は多く存在します．POSTGRES95はVersion 4.2と比べて，
Wisconsin Benchmarkでは30 %ないし50 %程早く走ります．バグフィックスは
別として，多くの改良が存在します．

コメント：
    derivative: 派生

* The query language POSTQUEL has been replaced with SQL (implemented
in the server). We do not support subqueries (which can be imitated
with user defined SQL functions) at the moment. Aggregates have been
re-implemented. We also added support for GROUP BY. The libpq
interface is still available for C programs.

問い合わせ言語、POSTQUEL は(サーバで実行される)SQLに取って代わられました。
私達は(ユーザ定義のSQL関数で同様な事ができますから)副問い合わせを
サポートしません。アグレゲートは再び実装されました。私達はまた、GROUP BY
のサポートを加えました。libpg インターフェースは C プログラムのために
現在でも利用できます。

コメント：

removed.)

ラージオブジェクトインターフェースは綿密に検討されました。私達はインバーション
ラージオブジェクトインターフェースをラージオブジェクトを記憶するための
メカニズムとして残しました。(これは削除されたインバーションファイルシステム
