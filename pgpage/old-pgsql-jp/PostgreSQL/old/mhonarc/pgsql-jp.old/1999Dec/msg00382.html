  • From: Toshimi Aoki <toshi@saluda.kinotrope.co.jp>

青木です。

Solaris 7/SPARC で regression test を行うと polygon, geometry で
pqReadData() -- backend closed the channel unexpectedly.
が出てしまう状況です。

postmaster -d 3 の出力を見ると 
/usr/local/pgsql/bin/postmaster: CleanupProc: pid 1054 exited with status 138

のように 138 で exit しているようです。


環境:
  Solaris 7 SPARC (Maintenance Update 4 applied)
  gcc 2.95.2
  bison 1.28
  flex 2.5.4

gcc -O2 でコンパイルしていたので、試しに -O0 としたところ、polygon で
は死なないようになりました。

Solaris 7 Intel では問題ないので、SPARC 固有もしくは gcc  のバグという
ことも考えられるので、Solaris SPARC をお使いのみなさんに状況をお聞きし
たいです。

よろしくお願いします。
