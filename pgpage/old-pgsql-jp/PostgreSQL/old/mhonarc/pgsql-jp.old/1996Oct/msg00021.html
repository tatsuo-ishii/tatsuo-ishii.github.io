桑村です。

 そろそろ、会社の Postgres95（1.02 からパッチで1.06にした）をそっくり
アップデートしようと思い、念のためホームページを確認したら、1.08 になっ
ていたのでそちらをインストールしました。もちろん、石井さんのパッチもあ
ててです。


  1.08自身にまだ直ってないところがあって、所定の修正以外に３箇所ほど
修正が必要でした。

  Makefile.global でコンパイルフラッグの -I の後ろにスペースが入っている
	CFLAGS += -I../../backend/port/$(PORTNAME)
のをつめたのと、
	READLINE_INC
が psql の Makefile 中では READLINE_INCDIR になっていること。
  それと、pg_dump.c で、
	void setMaxOid(FILE *fout);
のプロトタイプ宣言が必要なことです。


  もう一つ、それとは別に不思議なことが（以前から）あったのですが、
	make; make install
が成功して、いざ、 initdb をしようとするとエラーになるのです。原因は
不明ですが、一度 make clean をしてから、再度、make; make install を
した後はうまく行きました。


  というわけで、(もっとも 1.08 はバグフィクスなので当然ですが）

ersion		プラットフォーム		動かしている方
==================================================================
1.08+jp1.06	DEC3000-6000/DEC-OSF1-V3.0J	juk@rccm.co.jp
