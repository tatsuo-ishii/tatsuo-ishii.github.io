はじめまして、小野＠adunitと申します。
近く、地元のお店情報サイトを公開すべく、
初のデータベース運用で奮闘しております。

このサイトでは、お店の情報をオーナーさん
が自由に書き込めるものにし、更新なども
リアルタイムに行えるようにします。

そこで、運用方法として、ブラウザーから
入力された書きこみをPHP、PostgreSQLを用いて登録、
当方でのチェック・編集はPostgreSQL ODBCを介して
Access97にて行う予定です。

実際、postgresでデータベースを作成し、
Access側からデータを閲覧・書き込みするところまで、
なんとかなりました。
# いや、ここまでも、充分大変だったのですが(^^;

前置きが長くなってすみません、ここから質問です。
データの不整合を防ぐため、Access側で編集中の
レコードをロックしたいのですが、Accessのヘルプを見ると、

> ODBC (Open Database Connectivity) を使って、リンクした
> SQL データベースのテーブルのデータを編集するとき、Access は
> レコードをロックしません。その代わり、SQL データベースの規則でロックが管
> 理されます。この場合には、データベースに対するレコードのロックの設定内容
> にかかわらず、 Access で [ロックしない] が設定されているように見えます。

とのことで「ロックは管理される」とありますが、
これは何の設定もなしに、ロックは保証されるという
ことでしょうか? それとも、別途設定が必要に
なるのでしょうか?

以上、つたない文章で申し訳ございませんが、
どうぞご教授のほういただければ幸いです。

付記：ブラウザ側で書き込みがされる場合は、
　　　begin〜endで囲んでおります。

環境：Server: FreeBSD 2.2.8R/ Apache 1.3.6/
　　　        PHP Version 3.0.6 (kanji-patch)/
　　　　　　　PostgreSQL 6.4.2
　　　Client: Windows 98/ PostgreSQL ODBC(19991216)/
　　　　　　　Access 97
