C コンパイラについてもなんか本を呼んだほうがいいかも。

y73s> $ cc -I /usrsys/pgsql/include -L /usrsys/pgsql/lib/ test.c

y73s> 上記のエラーが出てしまいます。
y73s> libpq.a libpq.so の存在と、
…
y73s> ちなみに -lpq では下記のエラーがでます。
y73s> $ cc -I /usrsys/pgsql/include -lpq test.c 
y73s> /usr/bin/ld: cannot open -lpq: No such file or directory
y73s> collect2: ld returned 1 exit status

cc -I /usrsys/pgsql/include -L /usrsys/pgsql/lib/ test.c -lpq

-I, -L はそれぞれインクルードファイルとライブラリのパスを、
-l はライブラリファイル名の lib を抜けた名前を指定するのに使います。
＃と、ライブラリパスの中を cc が探しに行く

直接リンクしたければ、

cc -I /usrsys/pgsql/include test.c /usrsys/pgsql/lib/libpq.a

かな。
