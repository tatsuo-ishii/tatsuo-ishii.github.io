[pgsql-jp 13033] Qtを使ったクライアント

  • From: Toshihiro Inoue <tino@pa2.so-net.ne.jp>
  • Date: Thu, 03 Feb 2000 01:53:56 +0900
井上俊博です。
ツールキットは Qt を使用しております。

PostgreSQL では C++ はマイナーなようで
libpq++ の話題などもあまり見掛けないようですが、
Qt で簡単なクライアントのサンプルを作りましたので
参考までに添付いたします。

コンパイルには Qt-2.0 以上が必要です。
FreeBSD-3.3R, PostgreSQL-6.5.3,
qt-2.1.0-snapshot-19991215 で動作確認しています。

configure はありませんので、
アーカイブの中の MakeInclude を適宜書き換えてください。
大抵はパスの部分の修正だけで何とかなると思います。
また、-lxpg4 というのは FreeBSD 以外では不要です。

コンパイル後に test というデータベースを作成し、
（簡単のため test という名前はハードコーディングしています）
ソースと一緒に入っている test.sql を実行してください。
$ createdb test
$ psql -f test.sql test

そうすればサンプルは簡単な辞書として機能します。
（あまりにも簡単過ぎてまったく役には立ちませんが）

邪道かもしれませんがハッシュ関数感覚で扱いたかったため、
EPgTable という libpq++ のラッパーを定義しています。
（E は Eazy です）

PostgreSQL を利用してフリーの翻訳ソフトを作ろうかと思っていますので、
そちらの方も進展がありましたらご報告させていただきたいと思います。
今後とも宜しくお願いいたします。
