利光です。
　こんにちは

>>> Access 上の内部形式に変換した上で処理する場合がある...
>
>>> Outer Join は 6.5.3 でもサポートされていません
>
>ということは、MS-Access上でODBC接続したPostgreSQLのDBに
>ついて外部結合できているように見えて、それはMS-Access上
>で実現されている？ということになるのでしょうか？？

　MS-AccessがクライアントでODBCというサーバに
  接続にいっているイメージだと思います。
  ようは、ODBCドライバ側で変換がかかって
　MS-AccessのSQL　=> ODBC => PostgreSQLのSQL
　になる感じです。（ExcelとかもあるのでSQL変換というのは
 正しくないですが）
  ですので、ODBC部分がいまいちだと
　いまいちな性能しかでないようです。
　あと、MS-AccessのSQLビューにPostgreSQLのSQL文を
 書いても大丈夫だったと思います。

　コントロールパネルにODBCというのがあるので
　そこでトレースをONにするとどのようなODBC関数が
　呼ばれているかわかります。

　簡単なSELECT文であれば殆どそのままですが、
  外部結合等あるといろいろ変換がかかるのがわかります。
