Taro Yamamoto wrote:
> 

> datetime もしくは他の日付データ型において
> INSERT の時刻を自動的に millisecond まで入力したいのですが、
> INSERT 時に now 定数を使うと millisecond は求められないようなのです。
> 何かしら millisecond まで自動的に入力する方法はないでしょうか？
> 
> PostgreSQL 6.5.3
> FreeBSD 3.4-R
> Apache 1.3.9
> 
> の環境で Perl から操作しています。

datetime 型の列に、ミリ秒を直接入力してやれば、ちゃんと
date_part で、表示されるのですが、  default  では、でき
ないのでしょうか？。
私は、PHP でプログラムしているので、プログラムで、ミリ秒
まで直接 INSERT/UPDATE するようにコーディングしました。

もっとうまい方法があったら教えてください。

CREATE test TABLE ( dd datetime default CURRENT_TIMESTAMP );
insert into test(dd)values('1987-12-31 23:58:59.123456');
select dd, 'SEC', date_part('second', dd)     from test;

（実行結果）
dd                       |?column?|date_part                           
-------------------------+--------+---------
1987-12-31 23:58:59.12+09|SEC     |59.123456  

同じく PostgreSQL 6.5.3 です。
