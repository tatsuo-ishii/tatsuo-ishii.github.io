>> like '%心%' のような条件検索をした場合、例えばEUC「心」0xBFB4 の
>> 1バイト目と2バイト目が他の文字の2バイト目と次の文字の1バイト目に
>> マッチしてしまうことがあります。
>> 例が悪いですが、「心」で検索すると、疑眼 B5BFB4E3 の中央の2バイト
>> が見つかってしまう現象です。
>> 
>2バイトを意識して処理しているはずですが、そうなってなかったらバグです。
>ところで、PostgreSQLをコンパイルするときに
>
>configure --with-mb=EUC_JP

確認してみました。結論から言うと、
・RedHat6.1日本語版に英語版のPostgreSQL6.5.3をインストールした。
・その上に日本語版のPostgreSQL6.5.3をインストールし直した。
この状態では、2バイト処理がうまく行っていませんでした。

RedHat日本語をインストール直後に、最初から日本語版のPostgreSQLを
インストールしたところ、うまく行きました。

どうもお騒がせして申し訳ありませんでした。(_ _)
