はじめまして、fukuyama@tacktech.co.jp と申します。
シーラカンス本第1版を片手に、門戸を開きました。

早速で恐縮なのですが、御教授いただきたい事があります。
次のようなテーブルを作成しました。

create table TEST
(
logdate date,
TestValue1 numeric(100,30),
TestValue2 numeric(100,30),
TestValue3 numeric(100,30),
TestValue4 numeric(100,30),
TestValue5 numeric(100,30),
TestValue6 text
);

次に、insertをしようとしました。
insert into TEST values
(
'2000-02-21 13:44:58.0',
-0.9945218953682373,
0.10452846326765373,
0.10452846326765373,
0.10452846326765373,
0.10452846326765373,
'hogehoge'
);

すると、

ERROR:  Unable to convert left operator '-' from type 'unknown'

と、怒られてしまいます。

ところが、

insert into TEST values
(
'2000-02-21 13:44:58.0',
-0.994521895368237,	←末尾の'3'を削除
0.10452846326765373,
0.10452846326765373,
0.10452846326765373,
0.10452846326765373,
'hogehoge'
);

とすると、正しくinsertできます。
一体何故なのでしょうか? どこかに入力桁数の設定があるのでしょうか。
あるいは、そういうもの (^_^;; なのでしょうか?

環境は、
RH Linux5.2
カーネルバージョンは、 
Linux ---- 2.0.36 #1 Tue Dec 29 13:11:13 EST 1998 i586 unknown
PostgreSQLは、
Ver. 6.5.2 
を使用しています。

初歩的なことで申し訳ありませんがよろしくお願いします
