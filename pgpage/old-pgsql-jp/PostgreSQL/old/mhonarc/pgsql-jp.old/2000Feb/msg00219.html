[pgsql-jp 13179] LANG の値による不具合

  • From: "N.Ogiso" <ogiso@use-net.co.jp>
  • Date: Mon, 21 Feb 2000 12:39:28 +0900
こんにちは。

バックエンドの postmaster は PostgreSQL の super user である
postgres によって起動されますが、ユーザ postgres の環境変数
LANG に ja_JP.ujis なとの値がセットされていると、データベースの
unique index の値に日本語を入れた場合、文字列が違っていても
一意制約違反で insert できなくなることがあります。
export LANG=
などとしておくとこの問題は起こりません。このようなことは既に
報告されていますでしょうか。それとも自分のところだけで起きて
いる事象でしょうか。

もし他でも起きているとした場合、これは glibc の問題なのでしょうか。
環境は
PostgreSQL 6.5.3
glibc-2.1.2
です。
