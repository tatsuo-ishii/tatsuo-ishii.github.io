> 質問なのですが（素朴な疑問かも知れませんが）
> numeric型のデータ型の項目に空欄を入れる（データが無い時）
>
> update member set empnum='' where memberno = '0000001';
> 又は
> update member set empnum='null' where memberno = '0000001';
> とするともちろんエラー（ERROR:  Bad numeric input format ''）になりますが

この2つは通常、文字列と判断されてしまうのだと思います。''に囲まれた部分は数字
でなくてはならないと思います。'null' は、「null」 と言う文字列として扱われます。

> update member set empnum=null where memberno = '0000001';
> とするとアップデートできます、ここではempnumがnumeric型です

NULL値を入力したいのであれば、この方法が良いでしょう。
あと内容にもよりますが、オール 0 ('0000000')を何も無い値として扱うのも手です。
この場合はコードとして扱っているようなので問題ないと思いますが、金額などてして
numeric型を使う場合は計算式でnull値があると解がnullになってしまったりして、正し
く計算されなくなります。
