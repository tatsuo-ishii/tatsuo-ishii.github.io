[pgsql-jp 15476] ORDER BYしたSELECT同士のUNION

  • From: Kazuki Takeuchi <k-take@ncjoho.co.jp>
  • Date: Tue, 20 Jun 2000 22:03:34 +0900
竹内＠札幌ともうします。

RedHatLinux6.1J+PostgreSQL6.5.3で使用しています。

下記の様なSQL文を作成したとします。

SELECT a1,a2,a3 FROM t1 ORDER BY a3
UNION ALL
SELECT b1,b2,b3 FROM t2 ORDER BY b3

ここでt1、t2のテーブル構造、フィールド名は
全く同じとします。

上記のSQL文をpsqlで実行すると

ERROR:  parser: parse error at or near "union"

と表示されます。

双方のOREDER BY句を消すと結果が表示されますが
ソート順はマチマチです。

どうにかソートしたSELECT同士のUNIONを行う方法は
無いでしょうか？
