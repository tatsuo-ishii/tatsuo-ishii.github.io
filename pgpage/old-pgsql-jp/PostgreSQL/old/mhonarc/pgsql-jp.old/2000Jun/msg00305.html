持田さんのパッチでも同様の事ができるはずですが、これらのパッチを
使わないとオリジナルの JDBC ドライバで日本語が扱えないわけでは
必ずしもありません。

これらのパッチの説明にもありますが、Java プログラムを走らせる環境の
エンコーディングと DB のエンコーディングが同じであれば、オリジナルの
ままでも特に支障はありません。
少なくとも、私のところではまったく支障は生じていません。

DB が EUC で、Windows 環境の Java Application や Applet から使用する
場合にはこれらのパッチのお世話になった方がよいでしょうが、デフォルト
エンコーディングが同じ場合や、Servlet などの場合はあまり気にする必要は
ないです。Servlet などの場合は、そっちの方で DB のエンコーディングに
合わせればいいだけですので。
