[質問編]
はじめまして。
初めて、PostgreSQLをインストールしてみました。
initdb　の段階で、下記のようなエラーが出ます。
--------------------------------------------------------------
initdb: pg_encoding failed

Perhaps you did not configure PostgreSQL for multibyte support or
the program was not successfully installed.
--------------------------------------------------------------
[pgsql-jp 14750] の方と全く同じ現象です。
※参加したばかりなので、webの本文検索で過去発言を検索しました。

OSは TurboLinux Server 6.0J で、ユーザは postgres
PostgreSQL7.0.2　です。

./configure --enable-multibyte=EUC_JP --with-perl --with-odbc
としています。
make後、....ready to Install
のメッセージも出て、
make install　後、...Thank you....
のメッセージもでます。

環境変数は、
POSTGRES_HOME=/usr/local/pgsql
LD_LIBLARY_PATH=:/usr/local/pgsql/lib
PGLIB=/usr/local/pgsql/lib
PGDATA=/usr/local/pgsql/data

[解決編]
投稿する前に、もう一度、INSTALLを見ておこうと思い、たどっていると、
On some systems the following is the preferred method, but you must
have root access. Edit file /etc/ld.so.conf to add a line
/usr/local/pgsql/lib
Then run command /sbin/ldconfig.

とあって、もしや、と思ってこの通りすると、見事、通りました。
環境変数だけではダメだと言うことでしょうか？
まぁ、やっと次に進めます。
