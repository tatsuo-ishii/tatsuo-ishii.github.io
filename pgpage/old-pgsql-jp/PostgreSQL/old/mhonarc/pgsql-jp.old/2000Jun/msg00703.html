儘田　雄一郎 wrote:

> > # cd /etc/rc3.d/
> > # ln -s ../init.d/postgres S98postgres

蛇足ですが、 S (start) を作ったら、必ず K (stop) も
作っておきましょう。 
Kxxx がないと、shutdown や、UPS 停電時に、DB のシャット
ダウンができません。

Linux 系なら、run level 6 が、shutdown reboot だから、
rc.0, 1, 2, 6 に  Kxxx  rc.3,4,5 に Sxxx を シンボリック
リンクすればいいでしょう。K S の次の番号は、番号順に起動
されるので、他の処理との関係で決めてください。

。。。ばっさり省略。。。。

> OSはTurboLinuxServerでX起動なので、起動時のエラーメッセージが読めません。

X Window でいきなり起動する場合は、run level が 3 じゃないですね。

Linux なら、起動時の画面に表示される内容は、
    /var/log/dmesg
に入ってます。コンソ−ルに表示されるような情報は、
   /var/log/messages
に入ります。これらのログは、必ず確認しましょう。
通常、/etc/rc.d/ 以下のスクリプトには、起動時に画面と、
/var/log/messages に情報を書き込んでいるはずです。
