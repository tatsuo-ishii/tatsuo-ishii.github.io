SQL でエラ−になったときは、

１。エラ−となった SQL 文を、PHP ではなく、DB で単体実行してみること。

２。SQL の参考書と使用DBのマニュアルをみながら、簡単な例を実行
    して、DB そのものが動いていることを確認すること。

３。SQL 文を徐々に追加しながら、どこまで追加したらエラ−になる
    か確かめていけば、たいていの問題は、自力で解決できる筈です。

４。どうしてもわからないときは、SQL 文(PHP プログラムでなく)
    と、生のエラ−メッセ−ジを付けて、投稿してみてください。

SQL 文単体だと動くのに、PHP からだとエラ−になる場合は、PHP の
DB 呼び出しの引数を全部プリントして表示してみることです。

引数に値が入っていなかったり、引数の順番を間違えていたり、タイプ
ミスなどの場合が多いと思います。

あとは、前の方でエラ−になっているのに、エラ−チェックをさぼった
ために、後の方のステ−トメントで、エラ−表示されるような場合です。

# PHP では、次のような簡単なエラ−チェックもできるので、テスト
# など簡単な場合でも必ずエラ−チェックしましょうl。
#  $dbcon = pg_connect("dbname=hogge") or die("pg connect error");

以上のことを、ご自分で実行されてから、再度投稿することを、おすす
めします。

DB 名と version, PHP の Version と OS 名は必ず書きましょう。
