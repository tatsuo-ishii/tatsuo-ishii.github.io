# 本日無事にPostgreSQLユーザ会総会を終えました。

ところで、本家のMLで、開発中の fmgr(function manager) の改良版のテスター
を募集しています。function manager は、PostgreSQL のすべての関数呼出を
管理します。たとえば、いろいろなデータ型の入出力も fmgr の管理下にある
関数が行います。従来の fmgr では、ある種のプラットフォームにおいて、オ
プティマイザを有効にしてコンパイルしたPostgreSQLが正常に動作しない、と
いう問題がありました。「ある種のプラットフォーム」には、

Alpha
PowerPC
Sparc

などが含まれます。Alpha の方は本家のML参加者にテスターがいるのですが、
PowerPC のテスターがいません。私の LinuxPPC はバージョンが古いため、
fmgr の問題が起きず、現象を確認できません。どなたか、LinuxPPC R4 より
も新しいものでテストを行っていただけないでしょうか。

また、NetBSD などで上記CPUで動くものでも、テストを行った方が良いかも知
れません。

テストの方法ですが、開発中のスナップショット 
(ftp://ftp.postgresql.org/pub/dev/postgresql.snapshot.tar.gz)をインス
トールし、src/Makefile.global の CFLAGS 中のコンパイルフラグに -O2 の
オプションを有効にして PostgreSQL を作って regression test するだけで
す。

なお、開発中のバージョンでは configure は src の下ではなく、その上で行
うことに気を付けて下さい。
