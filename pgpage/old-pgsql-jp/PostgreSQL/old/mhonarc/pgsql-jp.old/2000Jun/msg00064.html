> こんにちは、タケウチです。
>
> 作業の関係で、テーブルを変更する必要が出てきました。
> 石井さんの改訂本では、「alter table」というものがあるそうですが
> バグで使用しない方が良いとのこと。
>
> では、他にテーブルのカラムの追加や削除するSQLやコマンドは
> 何かありますか？
>
> ちなみに、バージョンはPostgresql 6.5.2（改訂本 添付ＣＤ使用）です。

カラムの削除は ALTER TABLE では対応していませんので、まずテンポラリのテーブルに
データを退避させます。
つぎに、新しく修正したテーブルを定義させ、退避させたテーブルをそこに展開します。
面倒でもこれが一番確実ですよ。(^^)b
User's Manual の ALTER TABLE の項に載っています。

PostgreSQL 7.0以降になると、ALTER TABLE を使わないと外部キーの定義がすごく面倒に
なりますね。
しかし未だに、PostgreSQL の ALTER TABLE は削除機能が弱いですよね。

OUTER JOIN と ALTER TABLE の完璧な実装を心待ちにしているのですが・・・。(^^;)
