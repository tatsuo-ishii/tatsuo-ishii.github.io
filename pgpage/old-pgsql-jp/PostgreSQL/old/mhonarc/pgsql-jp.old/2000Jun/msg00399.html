以下のようなテーブル(address)を生成して、レコードが更新される
直前に、そのレコードの更新日付項目(update)に現在日時を(自動
的に)設定する処理をトリガー(とＳＱＬ関数)等を用いて行いたいの
ですが、うまく使えません。
どなたかアドバイスを頂けないでしょうか？
＃別の方法があれば、特にトリガー、ファンクションには拘りません。

======================================================
-- テストテーブルの生成
CREATE TABLE address (
    id            INT4,
    update     INT2,
    address    TEXT,
    name       TEXT
);
-- 更新日時項目に現在日時を設定するSQL関数の生成
CREATE FUNCTION update_func( int4 ) RETURNS ??? AS
'UPDATE address SET update = \'NOW\' WHERE id = $1'
LANGUAGE 'sql';
-- トリガーの生成及び呼び出し関数の登録
CREATE TRIGGER update_trg
BEFORE UPDATE
ON address FOR EACH ROW
EXECUTE PROCEDURE update_func( ???.id );
======================================================

CREATE FUNCTIONで戻り値に値を返さない(VOID?)のような
設定は可能でしょうか？　それとも同じ値を常時返すように設
定(があれば)すれば良いのでしょうか？
CREATE TRIGGER で更新されたレコード番号をupdate_func
のパラメタとして渡したいのですが可能でしょうか？(擬似オブ
ジェクトというのでしょうか？)
