> そうです、ユーザデータ対するＤＢＭＳの管理表領域という意味でシステム
> という言葉を用いました。

「システム」という言葉はかなり範囲が広いものです。
自分では自明だと思っていても、何通りも取りようがあるために、相手に
とっては意図をくみかねる事もままあることですので、できるだけ詳細に
書くようにした方が結局はお互いにとって無駄が減る事になります。

> なるほど。再インストールは最終手段なのですね。確かにウィンドウズは
> ＯＳだろうと、アプリだろうとよく再インストールを行いますね。実際それ以外
> の手段がないケースが多いと思います。ＮＴ上で動くＳＱＬサーバなどの
> ケースは知りませんけど。

DBMS の管理表領域の中身が不整合になるというケースでは、それが DBMS
自身の不良によって発生したのでなけれれば、DBMS プログラム自身の入れ換え
というのは何も意味を持ちません。

したがって、同じバージョンのプログラムを再インストールするというのは、
まったく無駄な行為です。

ひょっとして小林さんは、プログラムと管理表領域などが不可分であると
認識してらっしゃるのでしょうか。
で、管理表領域を作り直すには再インストールしかないと思ってらしたとか？
だとしたらそれは誤解です。

DBMS プログラムをインストールする作業と、DB 領域などを作成する作業は
それぞれ独立した作業です。
作業手順もしくはインストーラの動作モードによっては、このふたつが一緒に
なっている事もあるでしょうが、それはあくまでも便宜上そうなっているに
過ぎません。

これは別に、Oracle や PostgreSQL に限った話ではありません。
その他の DBMS でも同じ事です。

MS SQL Server でも、この点に関しては同じはずだと思いましたが…
なにせ、実体としては Sybase ですからね。:-)
