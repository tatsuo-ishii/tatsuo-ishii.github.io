> 再現方法は以下です。

 [pgsql-jp 15401] の、

| Connecting to Database URL = jdbc:postgresql//<HostName>:5432/postgres
| Exception caught.
| java.sql.SQLException: No suitable driver

の件ですよね、これ、URL の記述ミスでしょう。"//" の前に ':' がないです。
「JDBC ガイド」

  http://java.sun.com/products/jdk/1.2/ja/docs/ja/guide/jdbc/getstart/connection.doc.html

を見ると、

| JDBC URL の標準構文は以下のとおりです。その構文には 3 つの部分があり、
| コロンによって区切られています。
| 
|        jdbc:<subprotocol>:<subname>

てことで、上だと "postgresql//<HostName>" がサブプロトコルになります。
PostgreSQL の JDBC ドライバのサブプロトコルは "postgresql" ですから、
一致しません。従って、「適当なドライバがないよ」ってことになります。

 JDBC ではサブプロトコルの異なる JDBC ドライバを
複数組み込んで使うことが想定されているということだと思います。
きっと、それでちゃんと動くんでしょう。
