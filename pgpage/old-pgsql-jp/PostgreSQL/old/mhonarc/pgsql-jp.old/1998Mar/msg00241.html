ブリヂストンの藤井です。

[pgsql-jp 3299]で市川さんが指摘されていた以下の件は、6.3 で直っていまし
た。

>可変長データのフィールド長 (char (6) なども内部では長さフィールドを
>持つようです) のデータがおかしくなっているようです．
>
>以下のように定義したとします:
>
>       CREATE TABLE book4 (
>        a1 char (4),
>        a2 char (4),
>        a3 char (6),
>        a4 char (6),
>        a5 text)
>
>これにタプルを一つだけ挿入し，インスタンスを表示させます:
>
>       insert into book4 values ('1', '2', '3', '4');
>       select * from book4;
>
>既にご指摘の通り core をはきますので，直接の原因になっている
---

PostgreSQL 6.3 を使用。
6.3 用のMB(multi-byte)パッチは当てていません。

-------------------------------------------
スクリプト
-------------------------------------------
drop table book;
CREATE TABLE book(
a1 char(4),
a2 char(4),
a3 char(6),
a4 char(20),
a5 char(4),
a6 char(10),
a7 char(24),
a8 char(24),
a9 char(32));
insert into book
VALUES ('1','2','3','4','5','6','7','8','9');
select a1 from book;
select * from book;

-------------------------------------------
スクリプト
-------------------------------------------
drop table book;
CREATE TABLE book(
a1 char(4),
a2 char(4),
a3 char(6),
a4 char(6),
a5 text);
insert into book
VALUES ('1','2','3','4');
select * from book;
