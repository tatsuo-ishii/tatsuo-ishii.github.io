  この土日で、 jade をインストールして、手始めに PostgerSQL 附属の英語版
のsgmlのドキュメントをコンパイルしてみようと思ったのですが、うまく行きま
せんでした。(といよりjadeがうまくインストールできてないのだと思います。
)Makefileを参考に jade を実行してみるとコアダンプしてしまします。インスト
ールしたのは、

	o Davenport DTD DocBoock V3.0
	o Norman Walsh Dockbook Stylesheet
	o James Clark Jade 1.0.93

です。

	% jade -t html -d $DOCKBOOK_PATH/dockbook.dsl tutorial.sgml

とかすると、
	dockbk30/catalog:30:0:W DDTDECL catalog entries are not supported
とか、
	stylesheet.dtd:25:0:E: character "  " invalid: only minimum data characters allowed 
とかと言われ Segmentation Fault になります。

# SGML や DSSSL を理解しないで、やろうというのが無茶だとはわかっていますが、
# なんとかできないものですかね。

  ちなみに、マシンは i86-Linux-2.0.33 で jade のコンパイルは Makefile の
インストール先だけを変えて、コンパイルオプションとかは配布のままです。
