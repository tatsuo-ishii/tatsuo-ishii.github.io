>始めまして植田@物性研です。

>折角 PostgreSQL 側でユーザー毎にアクセス権限の制御が出来るのだから、
>
>1.nobody には select だけを許可。その上で operator といったユーザーを
>追加し、そのユーザーに insert や update などを許可するようにする。
>2.このユーザーの認証に mod_auth_pg95 あたりを使用したい
>
>などが出来ないかな？と考えています。一応 .htaccess の Auth_PGpwd_table 
>のところに pg_user を書いてみるなんて方法で通らないかな？なんて思い、
>試してみたのですが、ものの見事に蹴られました ;-)

perl にこだわらないのであれば、PHP/FI で、PostgreSQL の認証を
可能にするパッチがありますので、それを使えば解決すると思います。
この場合、認証ウィンドウ(.htacess で認証をかけたときに使われるのと
同じウィンドウ)からPostgreSQLのユーザ名とパスワードを入力
させることになります。

あるいは、update を行うページに .htaccess で認証をかけ、パスワードは
決め打ちで PostgreSQL に送出する。

perl でも同様のことは容易にできると思いますが、私は
あまり perl が得意でないので、調べていません。
