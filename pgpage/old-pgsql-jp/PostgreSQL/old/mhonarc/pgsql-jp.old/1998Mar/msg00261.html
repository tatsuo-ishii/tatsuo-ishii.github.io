桑村です。

  Plagiaware 用の PHP/FI Apache モジュールに GD 1.3 を組み込みました。今回
は、浦栃さんの新しい日本語パッチを使いました。しいて変更点といえば、
kanjiconv.c の output_code=JIS にしたくらいです。GDはおみつさんによる VFlib
拡張のパッチがあたっています。いつものように、

	http://www.rccm.co.jp/~juk/outgoing/Database/

-rwxr-xr-x   1 juk      root         3980 Mar 29 23:39 diskdb1.pgsql
-rwxr-xr-x   1 juk      root       188688 Mar 29 23:39 gd13.tgz
-rwxr-xr-x   1 juk      root       265073 Mar 29 23:39 phpfi201.tgz
-rwxr-xr-x   1 juk      root       727515 Mar 29 23:39 apach125.tgz

に最新のを置きました。今度は、gd13.tgz も必要になります。以下の注意を
お読み下さい。

--
apach125: Apache-1.2.5 + PHP/FI module
apach125: 
apach125: PHP/FI(+GD+LIBPQ)モジュールを組み込んだ Apache HTTP サーバです。
apach125: PHP/FI 経由で PostgreSQL へのアクセスが可能です。PHP/FI の記述を
apach125: 含むページの拡張子は、".phtml" となってます。また、URL でユーザ
apach125: ホームを表す '~' を省くことができるパッチを当てています。この機
apach125: 能を使うには srm.confの中で "UserDirWOTilde on" に設定します。
apach125:  
apach125: (注意: PHP/FI, PostgreSQL, GD, VFlib がインストールされている必
apach125:  要があります。)
apach125:  

phpfi201: PHP/FI-2.0.1 + jp_urat-5.3
phpfi201: 
phpfi201: PHP/FI は Perl に似た記述による HTML ファイル埋め込み型の簡単な
phpfi201: プログラム言語です。当モジュール組み込み Apache WEB サーバを利用
phpfi201: することによって、HTML ファイル内に直接プログラムを書くことが可
phpfi201: 能です。このパッケージでは PostgreSQL にアクセスすることを可能に
phpfi201: してます。また、GD による GIF イメージデータのハンドリングもでき
phpfi201: るようにしてます。
phpfi201: 
phpfi201: (注意: PostgreSQL と GD および VFlib がインストールされている必
phpfi201:  要があります。)
phpfi201: 

gd13:     GD 1.3 + jp.patch
gd13:     
gd13:     GIF イメージデータを扱うためのライブラリです。VFlib を使って日本
gd13:     語文字を書き込めるようなパッチを当ててあります。webgif は附属の
gd13:     サンプルプログラムで、GIF イメージを透明化したりインターレース化
gd13:     したりできます。
gd13:     
gd13:     (注意: VFlib がインストールされている必要があります。)
gd13:
