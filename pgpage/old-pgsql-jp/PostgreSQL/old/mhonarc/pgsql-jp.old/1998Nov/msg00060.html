はじめまして、武田と申します。

社内のあるシステムに、PostgreSQLを導入しようと検討中で、少し前
よりＲＯＭしていました。現在、次の環境でテストを行っています。

構成
[HW]
Machine :DEC VENTURIS 575
CPU     :Pentium 575
MEM     :40MB
HDD     :1G

[SW]
OS      :RedHat4.2
DB      :PostgreSQL6.4


過去のMLのアーカイブやホームページなどを見せていただいたのです
が、PostgreSQLの制限・仕様を見つけることができなかったので、こ
ちらに質問させていただきます。

１．PostgreSQLの制限・仕様はどこをみればわかるのでしょうか？
    テーブル数の制限、物理ファイルの容量制限、テーブルの列数の
    制限、WHERE句の条件数の制限、SQL文の長さ制限など

２．SELECT文のWHERE句で、テーブルを結合し表示するために複数の
    条件を指定したところ、次のエラーになります。

    pqReadData() -- backend closed the channel unexpectedly.
        This probably means the backend terminated abnormally
        before or while processing the request.
    We have lost the connection to the backend, so further processing
    is impossible.  Terminating.


どんなことでも結構なので、調べる手がかりを教えていただけないで
しょうか？ よろしく、お願いいたします。
