Mikio Nakajima <minakaji@osaka.email.ne.jp> wrote:

>   動作を見る限り、そんな感じですよね。SELECT のカラムに指定していない
> カラムは DISTINCT のフィルタリングの対象外にした方が便利だと思いますが、
> 規格上はどうなのかはまた別問題ですね。
> 
> > これが果して(SQL92的に)正しい動作なのかどうか分かりませんが、
> > 本家に聞いてみます。

　ORDER BY はあくまで SELECT で抽出された結果をソートする指定であり、ソー
ト対象のカラム名は SELECT で指定されたカラム名でなければならない…という
のがSQLの考え方だと思います。（例えば「SQL92完全ガイド」（トッパン）
p.271 参照。）

　ですから、厳密に言えば SELECT で指定されないカラム名を ORDER BY で指定
するとエラーにするべきものでしょうが、私の経験した範囲の処理系ではたいて
い通ったように思います。（詳しく覚えてなくてすみません。）しかし、その場
合の解釈は今回のような問題を引き起こす曖昧さをはらんでしまいますね。

　規格そのものにあたってのことではないので、誤りがあるかも…。
