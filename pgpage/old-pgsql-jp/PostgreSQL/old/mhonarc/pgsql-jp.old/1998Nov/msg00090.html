[pgsql-jp 5871] array of char

  • From: Masahiro Akagi <makagi@jaist.ac.jp>
  • Date: Fri, 13 Nov 1998 01:14:56 JST
こんばんは、赤木＠JAISTです。

PostgreSQL 6.4 について、
型 char の配列型のことで質問があります。

例えば、
	create table tmp (
	         array   int4[]
	);
や
	create table tmp (
	         array   text[]
	);

のように定義することはできますが、

	create table tmp (
	         array	char[]
	);
や
	create table tmp (
	         array	varchar[]
	);

では、エラーがでます。
ERROR:  parser: parse error at or near "["

以前は、
	create table tmp (
	         array	char16[]
	);

のように出来ていたはずですが、
これは仕様が変わったのでしょうか？
それとも、char型の変更にともなうバクでしょうか？
はたまた、私の勘違いでしょうか？

いずれにせよ、Postgres のポリシーとして、すべての型について、
配列で属性を作ることができる。というのがあるはずですから、
char型の配列はできるはずではないか、と考えています。
