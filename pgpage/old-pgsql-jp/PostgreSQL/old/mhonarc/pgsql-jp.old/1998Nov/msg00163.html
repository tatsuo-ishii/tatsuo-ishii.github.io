--------------------------------------------------------------------------
postgreSQL 6.4 パッチについて           石井達夫

このディレクトリには PostgreSQL 6.4 (http://www.postgresql.org/) のパッ
チが格納されています。インターネット上では、
ftp://ftp.sra.co.jp/pub/cmd/postgres/6.4/patches-6.4 から入手できます。

自分で必要なパッチを当てるのが「通」なのですが:-)、全部のパッチを当て
るための shell script "PATCH" が用意されていますので、とにかく全部パッ
チを当てたい、という方はご利用下さい。

postgresql-v6.4 のソースを展開し、そこ(postgresql-6.4 ディレクトリの一
つ上、すなわち postgresql-v6.4 が見えている状態)で、以下を実行します。

        % sh /foo/bar/PATCH

もちろん /foo/bar は PATCH のあるフルパスに置き換えて下さい。

＊注意＊
1. PATCH スクリプトは、PATCH とパッチファイルが同じディレクトリ内にあ
ることを前提としています。

2. Solaris 付属の patch コマンドは使えません。GNU patch を入手して下さい。
------------------------------------------------------------------------
改定履歴：

1998/11/21 mb_fix.patch と large_obj.patch pgaccess.patch を追加
1998/11/21 本ファイル新規作成
------------------------------------------------------------------------
1. mb_fix.patch:
   multi-byte 拡張が有効のときに BINARY CURSOR が正しく動かない問題と、
   pg_dumpall が正しく create database 文を出力しない問題を解決するパッチ。

2. large_obj.patch:
   large object を create しようとするとバックエンドが落ちるバグを修正。

3. pgaccess.patch: PostgreSQL 6.4 付属の pgaccess で、データ入力のた
   めに kinput2 を利用可能にするパッチ。
------------------------------------------------------------------------
