lpk_kanto_yoshikawa@mba.nifty.ne.jp wrote...
>　しばらくぶりで、質問にやってきました。
>　しばらく順調に動いていたのですが、新しいホストをたてようと、別の
>マシンに6.5.2をインストールしたところ、コンパイルやインストールは
>上手くいったのですが、なぜかinitdbで初期化が出来ず、起動できないで
>おります。
>
>> initdb
>/usr/libexec/ld-elf.so.1: Shared object "libpq.so.2" not found
>
>/usr/libexec/ld-elf.so.1: Shared object "libpq.so.2" not found
>Unable to determine a valid username.  If you are running
>initdb without an explicit username specified, then there
>may be a problem with finding the Postgres shared library
>and/or the pg_id utility.
>>
>
>
>"libpq.so.2"が「見つからない」と言っている様で、対処の
>/usr/libexec/を探すと確かにありませんが、これはコンパイルかなにか
>に失敗しているのでしょうか？

普通は /usr/local/pgsql/libの中にlibpq.so.2があると思います。
環境変数LD_LIBRARY_PATHに/usr/local/pgsql/libが入っていないのではないでしょうか。
または環境変数PGLIBに/usr/local/pgsql/libが入っていないかだと思います。
