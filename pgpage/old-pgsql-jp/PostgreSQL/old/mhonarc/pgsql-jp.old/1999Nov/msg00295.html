[pgsql-jp 11723] ODBCでvarcharに配列を使ったレコードの更新

  • From: Keisuke Minami <keisuke@rccn.com>
  • Date: Thu, 11 Nov 1999 23:17:18 +0900
こんばんは。ハイファイネットの三並です。

WindowsでODBCを使ってPostgreSQLにアクセスしているのですが
問題が発生したので質問させていただきます。

WindowsでAccess97からODBC経由でリンクを張ってPostgreSQLにアクセスをしています。
ここでvarcharに配列を使ったデータを含んだレコードは更新をしようとすると失敗するのです。

具体的には
psqlで
create table test(a int2 primary key, b varchar(8)[]);
insert into test(a,b) values(1,'{1,3}');
insert into test(a,b) values(2,'{ab,6}');
を入れておき、
Accessからこのテーブルにアクセスしてレコードを変更しようとすると
「ODBC--リンクテーブル'test'で更新に失敗しました。」
OKとすると
「Error while executing the query;
ERROR: Unable to identify an operator '=' for types '_varchar' and 'unknown'
 You will have to retype this query using an explicit cast (#1)」
とエラーが出てしまいます。

何かわかる方がいらっしゃいましたらよろしくお願いします。

環境
PostgreSQL側
Pentium75 Mem 48MB
FreeBSD2.2.8
PostgreSQL6.5.3

Win側
Pentium133 Mem 64MB
WinNT4 SP5
PostgreSQL ODBC Ver 6.40.00.06
日本語版は1999/08/26です。
