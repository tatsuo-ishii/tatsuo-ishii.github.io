[pgsql-jp 11932] CHAR型フィールドで数値演算ができる？

  • From: "Atsushi MORIMURA" <morimura@i-buddy.net>
  • Date: Sun, 28 Nov 1999 15:56:07 +0900
  • Importance: Normal

こんにちは．守村＠横浜メディアサービスと申します．
現在PostgreSQL6.5.2をSolaris2.6上で運用し、libpqを使用した
CGIを使用してシステム構築をしております．

現在以下のようなテーブルがあります：

Id            |Name  |Birthday
000000000001  |太郎  |19991128
000000000002  |次郎  |19991125
000000000003  |三郎  |19890124
000000000004  |四郎  |00000000
000000000005  |吾郎  |19540530

Id →　char(12)
Name → varchar(16)
Birthday →　char(20)

このテーブルを使用して誕生日からユーザーを検索する機能が急に必要と
なったのですが、ここで問題が発生しました．

Birthdayフィールドは固定長２０文字で取得してしまっているので、
SELECT * FROM tblData WHERE CAST(Birthday As int2) = 19991128;
というクエリーはエラーを起こしてしまいます．

テーブルのスキーマは簡単に変更することができないのでなんとかならないものかと
思っていて、ためしに
SELECT * FROM tblData WHERE Birthday < 19991130 AND Birthday > 19991120;
というクエリーを投げたところ、思い通りの結果が返ってきました．

Char型のフィールドを強引に数値型として扱っているのですが、これは内部的にどのよ
うな
動きをしているのでしょうか？また、このやり方で問題はないのでしょうか？

何かご存知の方がおられたらご教授願います．

以上です．よろしくお願いいたします．
