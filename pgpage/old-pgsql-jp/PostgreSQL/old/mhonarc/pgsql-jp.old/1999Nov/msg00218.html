  この問題は、PostgreSQLの問題でもなく、FreeBSDの問題でもなく、単に
  csh の環境設定の仕方が間違っているというだけの話です。

  まず、csh をログインシェルに使用した場合の初期化ファイルの読み込み順位
  を説明します。
  
  1) /etc/csh.cshrc
  2) /etc/csh.login
  3) ~/.cshrc
  4) ~/.login

  2)と4)はcsh をログインシェルとして使用した場合のみ読み込まれます。
  この起動順位を考慮すると、
  シェルの起動時に毎回必ず設定しなければならないシェル変数の設定は ~/.cshrc に
  子プロセスで使用する環境変数は ~/.login に記述するのが一般的な使い分けです。

  あとは ~/.cshrc の記述方法で不適切なところもあるようですが、それはご自分で
  勉強されてください。
  例えば .cshrc では Command Search Path の設定は setenv PATH とは書きません。

  最近は PC-UNIX関係の良い書籍が沢山出ていますから、それを利用なさっても
  よいし、手っ取り早く man csh で調べるのも良いです。
