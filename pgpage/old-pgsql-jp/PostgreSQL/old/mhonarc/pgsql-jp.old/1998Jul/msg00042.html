  • From: "Masaya SEKI" <seki@ooi.dai-ichi-life.co.jp>
  • Date: Fri, 3 Jul 1998 13:37:40 +0900
いつもお世話になっています。初心者の関です。

WIN95からPostgreSQLのテーブルをアクセスしようとしています。
ODBCの呼び出しのやり方とユーザ登録について
教えて下さい。


１．ODBCの呼び出し
(1)問題
CLIENT(WIN95)にPostODBCドライバ（CDrom）をインストールし、
ACCESSのインポートをSERVER(Linux)にかけましたが、
リンクテーブルが表示されず、
「ODBC--呼び出しは失敗しました。
Could not establish a communications link to the server (#2)
Could not connect the server(#4236092)」
のエラーメッセージが出ました。

(2)経過
TurboLinuxで RPMパッケージ（バイナリ）のPostgreSQLを
インストールしました。
postmasterの起動コマンドを確認したところ
[postgres@server1 postgres]$ ps X  でみると
pid   tty   stat   time   command
328  ?      S      0:00    postmaster  -S -O  -F
でした。
また、client(win95)からサーバ名のPINGと、postgresへの
telnetも通ります。

(3)解決に向けやっていること

postmaster -D/usr/local/pgsql/data -S -i
に変更したいと思っています。
RPMのPostgreSQLインストールでシステム起動時にpostmasterの起動する
設定がされていますので、この設定ファイルを見つけているんですがわかりません
。
/etc/rc.d/rc.localには該当コマンドが在りません。

初心者なもんで赤面ですが、カーネルの設定ファイル（カーネルの起動スクリプト
）
ってどこに在るんでしょうか。？Apacheには...confみたいなものがあったよう
なんですが。

他のホストからの要求に答えられるようにするため、
/usr/local/pgsql/data/pg_hba.confファイルに

host    all    0,0,0,0       0,0,0,0     trusu を最後に追加しました。

この場合、「指定した表に対して権限を与える」ために
grant文は必要ないのでしょうか。
 


２.postgresの登録ユーザIDとホストユーザID

postgresでlog-inしユーザの登録したいのですが、
通常のhost accountと同じIDになりません。

Postgresでlog-inし
$  createuser
enter name of user to add で    toshiba  と入力し
enter user's postgres id  となり 新規のIDの要求があり
通常のhost accountと同じIDになりません。

ハチャメチャな質問ですみません。

宜しくお願いします。
