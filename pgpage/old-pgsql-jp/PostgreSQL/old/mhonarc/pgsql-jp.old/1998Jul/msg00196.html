PostODBCについて質問です。

Microsoft ActiveX Data Object を VB5.0上で使って、サーバ上のDBにアクセスしようとしています。
ドライバとしてPostODBCを使用しています。

クライアント上でODBCデータソースの設定を以下のように行いました。
データソース名：test
サーバ名：192.168.1.1（サーバのIPアドレス）
ポート番号：5432
データベース：sample（もちろん実際にあるDBです。）
ユーザ名：postgres
認証タイプ：ホスト認証

サーバ側の設定としてはpg_hba.confに次の行を追加しました。

host    all             0.0.0.0         0.0.0.0         trust

またVBのプログラムは次のようなものです。

Private Sub Command2_Click()
	Dim newque As Object

	Set newque = CreateObject("ADODB.Connection")
	newque.Open ("test")
	newque.execute ("select * from bunsyo")
End Sub

このプログラムを実行すると newque.Open ("test") のところまではいくのですが、次の
	newque.execute ("select * from bunsyo")
のところで
	”SELECT”
とだけエラーメッセージが出てとまってしまいました。
MS Acessを使ってこのテーブルにリンクし、表示することはできています。（データの追加や削除はできません）
初心者なものですから、サーバ側でのDBの設定がわるいのか、ActiveXDataObjectが悪いのかPostODBCがわるいのか、わかりません。

どなたか、同じようなことをされている方はいらっしゃらないでしょうか。どこを直したらよいのか詳しく教えていただければうれしく思います。
よろしくお願いします。
