>>>>> On Mon, 20 Jul 1998 14:50:00 +0900, Kaihei Koyama <koyama@rios.co.jp> said:

>> 本当にクリティカルな業務でハードウェアの信頼性などが要求されるところ
>> 以外ではハードウェアが PC でいいというのは嬉しい人も多いでしょう。

> PC系のServer専用機のDiskArrayを無しにして、外付けのハードRAIDにすれば
> それなりのハードの信頼性はあると思います。

工夫次第という面があるとは思いますが、例えば銀行オンラインやそれに
(業務的に)類した極めてクリティカルなものを除けば、ハードウェア的な
信頼性は PC Server 機を使えば保てるとも思います。
でも、そういうマシンでメーカが動作保証している OS は(なぜか) NT だけ
だったりする事も多い様です。あとは NetWare くらい。

そういうマシンは数百万(安いモデルでも 100数十万 ?)以上するし、その
マシン上で FreeBSD/Linux も動くだろうけど、ハードメーカが保証してない
OS だとサポートや保証や障害時の問題切り分けなどの問題がややこしそう。
で… あれば、Sun のマシン買って Solaris 動かす方が運用上の面倒が
少ない。って事になりませんか？

FreeBSD Ready や Linux Ready の PC Server 機が出てくれば、こういう
点はクリアできるんでしょうけど。

> InterbaseはFreeBSD用にポーティングします。元ネタをSunOS版にしようかと
> 話をしています。本の発表ぐらいでMLも開設します。セマフォをどうしようか
> 少し悩んでますけど．．．

Oracle でも Sybase でも Solaris などで使う時にカーネルはリコンフィグ
しないといけない事が多いですから、FreeBSD でも GENERIC カーネルで
Interbase が動かないといけないという事はないでしょう。
PostgreSQL も皆さんご存知のように、FreeBSD 2.2.X の GENERIC カーネル
では動きません。;-p

私は DBMS 運用を行うには、カーネルリコンフィグくらいを行う技術力は
あって当り前だと思います。そうじゃないと実質的な運用はできないでしょう。
# 技術力の中には、「マニュアルを事前にきちんと読む」も含みます。

ソフトメーカとしては、無用の質問(マニュアルをまったく読まない人の
何と多い事か！)を避けるために、「pkg_add して終り」にしたいのかも
知れませんが…

ところで、FreeBSD/Linux 版の Interbase は価格的にどのくらいの値付け
になるのでしょうか？ 差し支えなければ教えて下さい。_o_
