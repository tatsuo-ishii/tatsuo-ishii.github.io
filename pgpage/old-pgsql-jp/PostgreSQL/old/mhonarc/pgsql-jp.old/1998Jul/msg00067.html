PostgreSQL 日本語版の要求する正規表現ライブラリっていうのは PostgreSQL 
バックエンドの話で、PHP/FI では PostgreSQL フロントエンド(libpq)しか使
わないので無関係です。じゃあなぜ PHP/FI 日本語パッチに日本語対応正規表
現ライブラリが付属しているかと言うと、PHP/FI の regex 関係の関数で日本
語も扱えるようにしたいからです。

>> MySQL を make した事がないんで、MySQL が OS の libregex.a ではなく、
>> 独自のものを要求するかどうかがちょっとわかないのですけど。^^;
>
>MySQL は独自の libregex.a を使用しています。どういう由来のものかは
>わかりませんが、ドキュメント中にこういう記述がありました。
>
>HISTORY
>     Written  by  Henry  Spencer  at   University   of   Toronto,
>     henry@zoo.toronto.edu.
>
>
>> MySQL の日本語パッチが独自の正規表現ライブラリを要求するのであれば
>> 出たとこ勝負。そうでなければ大体大丈夫じゃないかと思います。
>
>日本語化は正規表現までは対応していません。

MySQL の構造を理解していないので外しているかも知れませんが、MySQL も 
PostgreSQL 同様クライアント/サーバ構成になっているとすると、PHP/FI に
リンクされるのは MySQL のクライアントライブラリなので、MySQL 本体(サー
バ)がどんな正規表現ライブラリを使っていようと、PHP/FI(Apache) にリンク
される時点では関係ないのではないかと想像します。

というわけで、EXTRA_LIBS の設定は

EXTRA_LIBS=-L. -lphp -lgdbm -L/usr/local/pgsql/lib -lpq -L/usr/local/mysql/lib -lmysqlclient -L./jp.regex -lregex -lm

でいけると思いますが。
