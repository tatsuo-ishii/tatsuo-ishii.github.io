はじめまして、藤本＠東工大と申します。

諸事情で:-) ODBC経由で使えるデータベースを立ち上げることに
なりまして、PostgreSQLを選択致しました。


さて、のっけから質問で申し訳ないのですが、ODBCの動作試験として
手元にあるロータス アプローチ97 （だってMS高いんだもん）から
PostgreSQLのデータベースをアクセスできません。

試した事は、以下の通りです。

1. Solaris 2.5.1 for x86 上で PostgreSQL-6.3.2 + MB patch β3
  （＋ tcl, ecpg, pgaccess ）を作成。

2. make all runtest して、いくつか fail するものの大方OKに
   なることを確認。

2. fujimoto が試験的に作成したデータベース "address" 
   （中にaddress_tab というテーブルを作成）に対して、
   psql を用いてローカルおよびリモートの Solaris for SPARC
   マシンから（trust および password で）アクセス可能である
   ことを確認。

3. PostgreSQL ODBC Driver 6/10 を用いた場合
  3-1. アプローチから〔開く〕や〔データのインポート〕を用い、
       ファイルの種類として「ODBCデータソース」を選択すると、
       address というフォルダが表示される。これを開くと
       fujimoto@address が表示される。さらにこれを開くと
       address_tab が表示されるが、開こうとすると
	「"address_tab" が見つかりません」
       と言われる。
  3-2. ファイルの種類として「PostgreSQL」を選択した場合は、
       fujimoto@<サーバ名> が表示され、これを開くと
       address_tab が表示されるが、選択しても何も起きない。

4. PostODBC 6/10 を用いた場合
   fujimoto@address を開こうとすると、アプローチがハング
   アップしてしまう。

どうも気分的にはアプローチと Post[greSQL]ODBCドライバとの
相性っぽい気がしますが（特に4.）、もしその他の原因で
思い当たる方がいましたら、ぜひご教授ください。

＃しかし、過去のMLアーカイブを見ると、今どきアプローチなぞ
＃使っている人はほとんどいないようですね、、、
