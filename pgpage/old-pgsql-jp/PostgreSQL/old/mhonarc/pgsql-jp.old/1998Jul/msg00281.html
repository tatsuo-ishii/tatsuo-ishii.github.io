釈迦に説法かもしれないんですが、ご参考までに。
私も詳しいわけではないので、詳細は雑誌などを参考にされた方が
よいと思います。

Web Server
  クライアントから受け取った URL をもとにファイル名を確定し、
  そのファイルの中身をクライアント側に返す。ただそれだけ。
  やりとりには HTTP というプロトコルを使用する。
  実装系の１つとして Apache とか MS IIS(Internet Information
  Server) というものがある。

Database Server
  クライアントから受け取った要求を元に、データベースに関する
  いろいろな操作をする。やりとりに使用するプロトコルは、たぶん
  実装依存（自信無し）。実装系として PostgreSQL とか MS SQL 
  Server とかがある。

Database Client
  Database Server にいろいろな要求を出して、データを更新したり、
  結果を表示したりする。以下のように、いろいろな実装形態がある。

  ODBC
    クライアントからアクセスする際の規格の１つ。アプリケーション
    からは、すべての ODBC ドライバが同じインターフェースで使用
    できる。Database Server ごとにアクセスするプロトコルが異なる
    ため、サーバごとに ODBC ドライバが必要。要するに、Database
    Server ごとのプロトコルの違いを吸収するためのしくみ。
  JDBC
    Java クライアントからアクセスする際の規格の１つ。以下 ODBC
    と同様。
  Server Side Script
    HTML 文書の中に独自の言語を埋め込むことができる。この言語の
    文法の中でサポートすることにより、Web サーバを Database Client
    として使用することができる。この独自言語から ODBC を経由して
    動かしてもよいし、直接固有プロトコルで Database Server と
    接続してもよい。実装系として、以下のようなものがある。

    Active Server Pages
        Microsoft が IIS で使用するためにつくったしくみ。ODBC
        経由で PostgreSQL にも接続できる（と、今月号の SD 誌
        に紹介があった）。
    PHP
        直接に接続できる Database Server が複数サポートされており、
        その中に PostgreSQL も含まれる。特徴として、Apache の
        モジュール（サブルーチン）として組み込むことができることが
        あげられる。この場合は、Apache のインストール（バイナリ
        構築）に先立って、ソースやオブジェクトレベルで PHP 
        モジュールが組み込まれている必要があるので、当然 PHP を
        先にインストールしておかなければならない。

ということで、Apache のコンパイルと PostgreSQL のコンパイルには
直接の関係はありません。
