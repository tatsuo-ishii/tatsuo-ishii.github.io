石井＠日商エレクトロニクスです。

[pgsql-jp 2975]で、質問させていただいた
SunOS 4.1.4-JLE1.1.4 で PostgreSQL-6.2.1 の
regression test が All Failed になってしまった者です。

JP Patch + Official Patch [1-6] で試しました。

原因としては、parser がうまく動いてくれない
ようでしたが結局解決しないまま、別のマシンで
make したものをもってきてそれを使って凌いで
います。

[All-Failな環境-1]
SunOS 4.1.4-JLE1.1.4
gcc version 2.7.2.2
GNU Bison version 1.25
flex version 2.5.4

[All-Failな環境-2]
SunOS 4.1.4-JLE1.1.4
gcc version 2.7.2.3
GNU Bison version 1.25
flex version 2.5.4

[OKな環境]
SunOS 4.1.3-JLE1.1.3_U1
gcc version 2.7.2.1
GNU Bison version 1.25
flex version 2.5.4

[All-Failな環境-1,2]は同一マシンで
gcc のバージョンをアップして試しました。

システム環境の違い（4.1.4-JLE1.1.4 と 4.1.3-JLE1.1.3_U1）に
因るものなのか、コンパイル環境の違い（gcc-2.7.2.1 と 2.7.2.[2-3]）に
因るものなのかはっきりしません。

SunOS 4.1.3-JLE1.1.3_U1 の gcc のバージョンをアップして試してみれば
すこしは切り分けができるんでしょうが．．．

SunOS 4.1.[3-4] をお使いの方々、こんな現象に遭遇したことありませんか？
解決方法または解決のためのアドバイスありましたら、よろしくお願いします。
