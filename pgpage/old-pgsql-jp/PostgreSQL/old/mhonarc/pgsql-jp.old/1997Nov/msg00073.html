At  6:53 PM 97.11.11 +0900, Motonobu Takahashi wrote:
>  たかはしもとのぶ@NTTDATA です。
>
>|endian の問題ではなかったような気がするのですが、手元に Solaris
>|2.5.1x86 がないので確認できません。int 系のテストの expected と 
>|results の diff を見せて頂けませんか？
>
>以下の通りです。
>
>%diff results/int2.out expected/int2.out
>10c10
>< WARN:pg_atoi: error reading "100000": Result too large
>---
>> WARN:pg_atoi: error reading "100000": Math result not representable
>
>
>%diff results/int4.out expected/int4.out
>10c10
>< WARN:pg_atoi: error reading "1000000000000": Result too large
>---
>> WARN:pg_atoi: error reading "1000000000000": Math result not representable

これでしたら心配ありません。単に、エラーメッセージの文字列が
違うだけです。regression test は、単純にあらかじめ用意してあるテスト結果
(expected)と文字列が一致しているかどうかで、success/fail の
判定をしています(おそらく reference は Linux/x86)。

なお、int2.sql などにはわざとエラーを出すような SQL 文を入れてあるので、
エラーになっていること自体も特に問題ではありません。
