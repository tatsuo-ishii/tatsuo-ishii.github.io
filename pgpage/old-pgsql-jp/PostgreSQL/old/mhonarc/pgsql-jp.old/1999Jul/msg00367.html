Debian potatoで6.5を使っています。

日本語を使いたかったので、rulesの中でconfigureを指定している
所で、--with-mb=EUC_JPとして、buildしなおしました。

field type   length 
--------------------
name  text   var
tel   text   var

のテーブルを作り、正規表現を使って検索を行ったところ
エラーがでてしまいます。

1)select * from tel where tel.name ~ '木';
ERROR regcomp failed with error parentheses not balanced

日本語コード以外で例えば、'abfc' 'a'  '12'は検索できます。
2)select * from tel where tel.name ~ 'a';

また 
3)select * from tel where tel.name='鈴木';
は検索できます。

どこが原因で1)がうまくいかないか、分かりません。
