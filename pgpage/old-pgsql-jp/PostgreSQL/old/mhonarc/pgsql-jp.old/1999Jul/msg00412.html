藤沼＠イイガです。

<19990723103036I.rok@itc.keio.ac.jp> の、
   "[pgsql-jp 9608] when is currval('sequence') available." において、
   "CHOI Kyong-Rok <rok@itc.keio.ac.jp>"さんは書きました：

> create sequence hoge; で作った sequence に対して, psql で
> 
> select currval('hoge');
> 
> した時に, ちゃんと値が得られる場合と得られない場合があります. 
> 
> psql 起動後, 一度でも select nextval('hoge'); を実行していると 
> currval('hoge') の値を得ることができますが, 一度も 
> nextval('hoge') を参照していない場合には
> 
> ERROR:  hoge.currval is not yet defined in this session
> 
> となってしまいます.
> 
> こういうものですか？

こういうもののはずです。Oracle でもそうでした。

currval は最終値を得る手段としては使えません。
他のユーザー・プロセスからも currval される可能性、したがって
複数のユーザーないしプロセスが同じ sequence number を使う可能性を
考えると、この動作は納得が行くのではないでしょうか。

日本語に直せば、
nextval = 「あなたに割り当てられた番号は」
currval = 「あなたに割り当てられている番号は」
という解釈になるのではないでしょうか。

間違っていたらバシバシ指摘してください。(^^;
