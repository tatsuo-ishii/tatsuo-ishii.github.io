集約関数を使ってselectするとエラーになってしまいます。

Linux上に postgresql-6.4.2 をインストールして使用しているのですが、
集約関数（例えばmax）を用いたselectを行うと、
「No such function 'max' with the specified attributes」
というメッセージでエラーになってしまいます。

SQL文は
test=> select max(col_name) from table_name;
とか
test=> select avg(col_name) from table_name;
（※ col_nameのタイプはoidです。）

を実行しているだけなのですが、No such function となってしまいます。
何が悪いのでしょうか?
