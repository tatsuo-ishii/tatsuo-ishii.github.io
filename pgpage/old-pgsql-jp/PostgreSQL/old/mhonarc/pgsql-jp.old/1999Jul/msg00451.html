> # 仕様決めの手伝いと技術支援はできますけど、

　これからも、pgbashをずっとお願いしますよ(^^)。

　早速ですが、fetch in cur into :ROW のシェル変数への代入
に関して、次の問題点があります。

 (1) $ROWが既存のシェル変数の名前とぶつかる可能性がある。
 (2) シェル変数のデータ型は文字型と整数型しかない。

 (1)については、ユーザ側で":_ROW"のようにアンダーバーをつ
けるなどの名前重複回避策を講じなければならないと思うのです
が、何かいいアイデアありますか？

 (2)については、DB側の全てのデータ型に対して、$ROWには文字
型にして代入するしかないと思いますが、bashコマンドにおいて
実数演算（整数型と同様の四則演算程度で良いのですが）に関し
て何かいい方法がありますか？

# 例えば、次のような比較／数値演算を考えています。

  if [ numcomp $ROW '-' $ROW2 '>' '1.0E-4' ]; then
     numexpr $ROW '-' $ROW2 '+' '3.1'
     if [ pgcomp $PGEXPR '<' '1.0E-6' ]; then
        ...
     fi
  fi

  numcomp : 比較演算を行う。結果は true or false。
  numexpr : 数値演算を行う。結果は $PGEXPRに文字列として返す。


# (2)の問題は、ややこしいので次回のバージョンで実現ということ
　にしますが、この機能が無いと、pgbash のプログラミング能力が
　実用レベルにならない可能性がありますので、検討だけはしておき
　たいと思います。
