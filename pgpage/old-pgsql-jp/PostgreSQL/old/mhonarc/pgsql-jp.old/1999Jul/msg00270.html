# KAKASIやChasenは使ったことがないので、全くはずしているかも
　しれません。そのときは、ご容赦を。

　このシステムは、Webを使用するとしまして、操作を次のようにし
ます。

  1.第１ページは Namazu+KAKASI を使用した全文検索にする。
　2.１で得られた結果(ユニークID)をホームページに表示する。
　3.表示された"ユニークID"をクリックすると、PostgreSQLのDB
    検索を行いその結果をホームページに表示する。

　ここで問題となるのは、１の全文検索の対象となるファイルの構
成およびその内容です。それは次のようにします。

　1.ファイル名を"ユニークID"とする。<===(ユニークIDと単語を結ぶ)
　2.１つのディレクトリにいれるとファイル数が多くなりすぎるの
　　ディレクトリを並列化＋階層化してファイルを分けて格納する。
　　(通常、"ユニークID"が(並列)階層構造になっているはずですから、
    その構造をそのまま利用する方がファイル管理上好ましいです)
  3.ファイルの内容は"書名"もしくは検索キーワードとする。

  このようにした、ファイル群に対して、Namazuのインデックス作成
を行います。あとは、ホームページをうまく作成すれば、さばけるの
ではないかと思います。


>＃　Cとかでプログラムを組めば出来ない事はないと思いますが、それだけの
>＃　スキルがないので、勉強中です。

  C を使用しなくても、上記の各ステップの途中で、シェルスクリプト
もしくは Perl, awk などを使ったプログラミングをせざるを得ない可
能性があります。

　特に、Namazuの検索結果をそのまま表示してもPostgreSQLのキー値に
ならない場合は、Namazuの検索結果を一度フィルタープログラムにとお
して加工する必要がありますので。
