On Fri, 16 Jul 1999 17:01:02 +0900
清水　昭博 <gen@hek.co.jp> wrote:

> > SQL92		PostgreSQL	Oracle for Linux　Sybase for Linux	Informix
> > ---------------------------------------------------------------------
> > decimal	◯          ○->△            ○                ○

decimal 型ですが、１０進固定小数点演算をサポートし
ているか、精度は何桁かが問題になると思います。
また、C をホスト言語とする場合は、C の１０進演算
ライブラリが提供されているかも重要です。
積算見積りとか、特に官公庁の規定にあるような、
少数第何位で切上げ／切捨て云々して積算という
ような処理をする場合とか、COBOL の汎用機の計算
結果と一致させなければいけないような時に重要。

Informix では、C の１０進演算ライブラリーまで
添付されています。


細かいとこですが、Informix について訂正を。

> DBMS固有組み込みデータ型
>   データ型	PostgreSQL	Oracle for Linux        Informix
>   ---------------------------------------------------------------------
>   money     ×          ×                      (decimal)

Informix の money は、内部では decimal と同じ扱い
ですが、ツール類で、貨幣記号や３桁毎のカンマ編集
をするなどで money と decimal を区別しています。

>   rowid     (oid?)      レコードの物理位置を    ×→○
>                         表す16進文字列
Informix は、 rowid を持っています。
勿論、Informix-SE for Linux にもあります。

PostgreSQL の oid と違って、レコードの論理的位置
を直接示すので、最も早い行アクセスができます。
内部的には Integer なので、C 言語だと、配列に放り
込んでいろいろと処理するのに便利。

＃今回は、データー型の比較ですが、OUTER JOIN とか
＃実務でよく使う機能の有無の比較もしたいですね。
