  パラパラと細切れで済みませんが、geometric の geometric_intersect と 
geometric_same について質問です。まだ、他にもあるのですが、、、

  動作環境は以下の通りです。

    FreeBSD3.2 (PentiumII 400、メモリ 160M、UW SCSI 4G)
    PostgreSQL 6.5.1 (--with-mb=EUC_JP)
    ftp://ftp.jp.postgresql.org/pub/pgcd/examples/geometric-0.1.tar.gz
    pgaccess.patch

    Vine Linix1.1 (K6-2/400、メモリ 256M、UW SCSI 4G)
    PostgreSQL 6.5.1 (--with-mb=EUC_JP)
    ftp://ftp.jp.postgresql.org/pub/pgcd/examples/geometric-0.1.tar.gz

  まず、geometric_intersect です。実際には、ある 3 次元領域にひっかかるものを
調べようとしたのですが、以下のようにしても落ちてしまいました。

    sugita=> select geometric_intersect('(100,200)','(150,300)');
    pqReadData() -- backend closed the channel unexpectedly.
	    This probably means the backend terminated abnormally
	    before or while processing the request.
    We have lost the connection to the backend, so further processing is impossible.  Terminating.
    $ 

  次に、geometric_same です。

    ====  README.euc_jp  ====
    ・geometric_same(geometric, geometric) returns bool
      same(geometric, geometric) returns bool

      2つのオブジェクトが合同かテストします。

「合同」とのことなので、大きさがそのままで位置をずらした場合には t になると思
うのですが f になります。

    sugita=> select geometric_same('(10,10)[](20,20)','(10,10)[](20,20)');
    geometric_same
    --------------
    t             
    (1 row)

    sugita=> select geometric_same('(10,10)[](20,20)','(11,11)[](21,21)');
    geometric_same
    --------------
    f             
    (1 row)

    sugita=>
