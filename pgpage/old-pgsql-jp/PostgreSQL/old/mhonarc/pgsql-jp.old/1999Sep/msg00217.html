From: Hiroshi Ozeki <ozeki@ecotec.co.jp>
> さて、ついでに質問ですが、
> OpenZOLARというフリーソフトですが、FreeBSD、
> PostgreSQL6.5.1環境で動かした方おられますでしょうか？
> PostgreSQL6.4 対応となっているので無理なのかなとは
> 思ったのですが、一応試しています。でも駄目でした。
> 現象として internal server error となります。

OZEKIさんはじめまして。私は、PostgreSQL対応のWWW-DB連携
ツールであるOpenZOLARの開発をしています。

但野さんからも紹介いただきましたが、本日中に、OpenZOLAR
の最新版をリリースする予定です。これまでのリリースは、
FreeBSD 2.2.8のみの対応でしたが、今回のリリースから、
FreeBSD版は、2.2.8, 3.2の双方のバージョンに対応しており、
ビルドはPostgreSQL6.5.1でおこないました。

ただ、この'internal server error'ですが、OpenZOLARの対応
しているPostgreSQLのバージョンによるものではなく、CGIの
実行時にPostgreSQLのライブラリを発見できないという問題
ではないでしょうか。2.x系と3.xでは、ライブラリの指定が異
なりますが、ldconfigにPostgreSQLのライブラリのパスである
/usr/local/pgsql/lib(標準で)が渡されているか確認してみて
下さい。
