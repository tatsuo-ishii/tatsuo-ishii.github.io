[pgsql-jp 10787] 自己紹介＋改行を含む文字列の処理について

  • From: Tomohiko Saito <tomosan@cc.okayama-u.ac.jp>
  • Date: Sun, 19 Sep 1999 12:00:24 +0900
はじめまして、斎藤と申します。

今まで、主にWindows畑で、Visual Basic や MS-ACCESSを使用して
Jetデータベースや、Microsoft SQL Serverを利用していました。

費用と安定性の点で、フリーのUNIXのデータベースソフトを使用
することになり、今回、FreeBSDとPostgreSQLでの開発をすることに
なりました。クライアントはWindowsマシンでVisual Basicを中心に
作成することになると思います。少しでも皆様方のお知恵をお借り
したくメーリングリストに参加させていただくことになりました。
よろしくお願いします。

さて、さっそく質問なのですが、
今までMS-SQL Server 6.5で、varchar型のデータを多く扱ってきた
のですが、PostgreSQLでは、text型が比較的短い文書を扱うのに
便利なのでtext型に変更しようと考えています。
しかし、この文字列には改行が含まれており、このままではテキスト
ファイルにした場合改行コードが悪さをしてしまって、copyコマンド
でもうまくインポートできません。

区切り文字をタブやコンマに変更するには、using delimiters を
使用すればいいのですが、文字列全体を囲むキャラクタを指定する
オプションとかはあるのでしょうか？

また、改行を含む形で保存できたとして、pg_dumpコマンド等で
バックアップをファイルに作成した場合、うまくリストアできる
のでしょうか。

今後のことを考えて、文字列中の改行コードを保存前に、いったん
\nという形にでも置換して保存し、読み出して表示する前に逆に
改行コードに置換してから表示するといった手順を踏んだほうが
よいのでしょうか。

以上、よろしくお願いします。

なお、開発環境は FreeBSD 3.1　PostgreSQL 6.4.2です。
クライアントは Windows NT4.0 WS(sp5)、VisualBasic6.0(sp3)
です。
