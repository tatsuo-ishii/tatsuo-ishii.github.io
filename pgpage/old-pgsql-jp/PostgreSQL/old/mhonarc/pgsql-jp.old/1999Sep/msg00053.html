> はじめて投稿させていただきます。立命館大学の平林と申します。
[略]
> システムの実装はC言語によるCGIを中心に行っています。簡単のために、
> できればPHPを使いたいのですが、日本語の「分かち書き」処理をする方法
> がPHPでは思いつかないからです。
> 
> PHPで「分かち書き」をうまくやる方法はないでしょうか？

kakasi とか chasen というプログラムを使えば「分かち書き」になると思い
ます。たとえば kakasi では上の文章は

|システム の 実装 は C 言語 による CGI を 中心 に 行って います 。 簡単 
|のために 、 できれば PHP を 使い たいのですが 、 日本語 の 「 分か ち 
|書き 」 処理 を する 方法 が PHP では 思い つかないからです 。
|
|PHP で 「 分か ち 書き 」 を うまくやる 方法 はないでしょうか ？

のようになります。kakasi や chasen については WWW で検索すればたくさん
情報が得られますし、書籍なら「日本語全文検索システムの構築と活用」(馬
場肇著、ソフトバンク)がおすすめです。

PHP から kakasi を呼び出すには、kakasi をプロセスとして起動するか、
kakasi をライブラリ化し、PHP に組み込むかですね。この辺になると単純な
プログラミングの問題です。

> また、PostgreSQLとはあまり関係ない質問で恐縮なのですが、IE4.0以上が
> 送信してくるUnicodeをCGIでうまくEUCに変換する方法をご存知の方がいら
> っしゃれば、教えていただきたいのです。

あらかじめ Unicode が日本語のみを扱っていることが分かれば、変換テーブ
ルによって Unicode -> 日本語 EUC への変換は可能です。変換テーブルは 
WWW 上から手に入ります(URL失念)。その上で PHP を拡張します。後は、lv 
というツールがこの機能を持っているのでそれを利用する。

そういえば、βテスト中の PHP の日本語版は Unicode を扱えるらしいので、
もしかしたらすでにそういう機能があるのかもしれません。
