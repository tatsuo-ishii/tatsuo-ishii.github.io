吉川さんありがとうございます。

過去のアーカイブを見たのですが
> 　＞　set client_encoding to 'SJIS';
はデータベースのエンコーディングがEUC_JPで
クライアントがSJISエンコーディングのときにデータベース側で
SJIS<->EUC_JP変換を行う指示をするもののようです。

ぼくが想定したのはデータベースのエンコーディングがSJISのとき
(createdbコマンドで-E SJISオプションを付ける)だったんですが
どうもSQLパーザーにバグがあるような気がしています。
というのもSJIS文字コードにしたkterm上からpsqlで次のようにすると

sjis=> set client_encoding to 'SJIS';
SET VARIABLE
sjis=> create table book (本 text, author text) ;
ERROR:  parser: parse error at or near "{"

という結果が出ます。
(JDBCからUNICODEやMULE_INTERNALでおなじことをした場合はうまくいきます)

# ということでいつか直ると思ってJDBCドライバは
# SJISエンコーディングで流す実装にします。
