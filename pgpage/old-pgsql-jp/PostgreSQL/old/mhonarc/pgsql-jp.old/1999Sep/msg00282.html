　おまけの情報です。

At 0:58 PM 99.9.18 +0900, Shoichi Kosuge wrote:
>　自分の環境でも再現しました(Solaris 2.6 + PostgreSQL 6.5.1)。
>更に、ダンプをとって見てみると、NULL と'' は異なるようです。
>NULL は\N
>'' は何もなし、というか、長さ0 の文字列というか。

　Informix では、'' と NULL は別物でした。更に数値フィールドにおいての 0 と 
NULL も別物でした。うっかり insertすると NULL が入り、後々苦労することが何度
も起きたので、フィールドは全て NOT NULL にしてました。使うSQLツールによって、
'' がNULLだったり空白だったりしたのも混乱に拍車を掛けました。
　そりは仕様だと思います。NULLを探すには、NULLというキーワードで検索が出来た
と思いました。空白を入れたい場合は、' ' なんてしてました。
　NULL は '' や 0 ではなく、虚数(?)である、と思った方が良いと思います。
