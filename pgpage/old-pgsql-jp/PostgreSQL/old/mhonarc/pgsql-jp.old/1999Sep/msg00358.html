こんばんは。三並です。

WindowsでODBCを使ってPostgreSQLにアクセスしているのですが
日本語の部分で問題が発生したので質問させていただきます。

WindowsでOffice2000のMS QUERY 32からODBC経由でPostgreSQLにアクセスをします。
レコードの中で編集の許可にチェックを付けます。
レコード内にまったく日本語が使われていないレコードに日本語の内容を入れようとすると
「レコードにアクセスできません。列を増やしてレコードを指定するか、
または他のユーザーがレコードを変更しています。」
とエラーが出て更新が行えません。
エラーメッセージに出ているようなことはありませんし、
列を増やしてもエラーは出なくなりません。

日本語が含まれるレコードの日本語が入っていない列に
日本語を入れても同様のエラーは出ませんでした。
他の部分ではまったく問題なくPostgreSQLもODBCも動作しています。
PostgreSQL ODBC DRIVERの問題なのか、MS QUERYの問題なのか
はたまた私の操作ミスなのかがわからずに困っています。

何かわかる方がいらっしゃいましたらよろしくお願いします。

環境
PostgreSQL側
Pentium75 Mem 48MB
FreeBSD2.2.8
PostgreSQL6.5.2

Win側
Pentium133 Mem 64MB
WinNT4 SP5
PostgreSQL ODBC Ver 6.40.00.06
日本語版は1999/08/26です。

PostgreSQLの文字コードはEUCです。
