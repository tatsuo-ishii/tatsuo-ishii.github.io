　php3, pgsql_perl5 ではアプリケーションを作らなければ無理
ですね。psql -E でバックエンドに送信している SQL を調べて、
それからデータベースからの読込値をテーブル形式で出力するプ
ログラムを作成することになります。

　..が、今、流行のpgbash(^^)ならば超簡単です。次のようにし
ます。

----
#!/usr/local/bin/bash
echo "Content-type: text/plain"
echo ""
enable -f /usr/local/pgsql/lib/libpgbash.so exec_sql
#
exec_sql "connect to dbname  as db1" ... DB1への接続
exec_sql "connect to dbname2 as db2" ... DB2への接続
exec_sql -l databases                ... データベース一覧
exec_sql -d db1 -l tables            ... DB1のテーブル一覧
exec_sql -d db2 -l tables -l indeces ... DB2のテーブルと
　　　　　　　　　　　　　　　　　　　　  インデックス一覧
----

  -l, -p, -r などのデータベース情報表示オプションや pgbash 
のインストールにつきましては次を参照してください。
  http://www.psn.co.jp/PostgreSQL/pgbash/

　但し、現在のバージョンでは、PostgreSQLがパスワードを要求す
る場合は処理できません。また、ブラウザから表示条件を指定して
表示することが簡単ではありません。さらに、データベース情報の
表示は Content-typeが "text/plain"でなければなりません。
(pgbashの次のバージョンでは以上の問題点は全て改良されてます)
