ですね。
PHP では、変数や、こうした内部情報を Hash Table で抱えているんですが、
代表的な Table の Root には、persistent なものと、session 限りのものと
2 つあるようです。

で、Apache の組み込み module として動作するときだけ、この persistent
な Hash Table が意味があって、Apache の KeepAlive だけ情報が保持できる
と。
ですので、pconnect を使って、postmaster に接続すると、persistent な
Hash Table に接続情報の構造体を格納しているみたいです。

この辺は、詳細に見なかったので、上っ面な情報ですみませんが...
