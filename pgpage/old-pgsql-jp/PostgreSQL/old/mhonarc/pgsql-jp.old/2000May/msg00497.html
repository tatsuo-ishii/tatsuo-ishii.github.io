石井です。

> create table sample(
>   fld1 int .....,
>   fld2 varchar(10) .....,
>     :
>   fldN int .....,
>   primary key(fld1,fld2,...fld9)
> );
> <createは成功>
> と primary key を9つ使用した場合に
> select * from sample;
> を実行すると
> ERROR: index_info: no amop 403 11 1
> とテーブルにアクセスできません。なお、primary key
> が8つの場合はOKでした。
> 
> これは何か、primary key に制限があるのでしょうか?

6.5.3 or それ以前のバージョンの制限です。7.0 では解消されています。
