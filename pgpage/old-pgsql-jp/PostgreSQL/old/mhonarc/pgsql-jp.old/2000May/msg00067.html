始めまして
佐々木＠三軒茶屋です。

わたしは現在、Linux周りの環境設定について勉強している最中なのですが、
JDBC接続がだけがどーも！うまくいきません！

ちなみに環境は以下の通りです

========環境========

TurboLinux4.1
JDK1.2(JAVA2)
postgreSQL-6.5.3(パッチずみ)


色々な方にお世話になりながら牛歩のペースではありますが、
ちょっとづつ進展しています。

そこで今回JAVAプログラムを実行したところ
以下のエラーが出ました

========実行エラー========

[ysasaki@Linux java]$ java syain2
 --- SQL Exception ---
Connection refused. Check that the hostname and port is correct, and that the postmaster is running with the -i flag, 
which enables TCP/IP networking.
        at postgresql.Connection.openConnection(Compiled Code)
        at postgresql.Driver.connect(Compiled Code)
        at java.sql.DriverManager.getConnection(Compiled Code)
        at java.sql.DriverManager.getConnection(Compiled Code)
        at syain2.main(Compiled Code)
ErrorCode :0

エラーメッセージを読む限りではプログラムとしては以下の箇所が間違っているようで、
色々手直しをしてみたのですがどうもうまくいきません！

========プログラムソース========

  try {
       Connection db = DriverManager.getConnection("jdbc:postgresql://203.183.119.79:5432/ysasaki","ysasaki","sasaki");


後、考えられるのはpostmasterの起動方法なのですが起動時にオプション「-i」を付けても付けなくても
エラーメッセージは変わりません。
