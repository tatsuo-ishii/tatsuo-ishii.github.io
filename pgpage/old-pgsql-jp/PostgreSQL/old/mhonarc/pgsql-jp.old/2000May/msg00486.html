> > 文字とを比較」します。
> ．．．
> >  「SQL92完全ガイド」を見るとSETには１文字の例しか載っていま
> > せんので、(1),(2)のいずれが正しいのかわかりませんでした。
> > 
> >   oracleはどうなんでしょう？　＞　よしおかさん

> > # trimに関してはOracleの仕様と合わせているようなので。

> OracleだとSET文字は一文字だけですね．
> 
> SQL92のスペックはどうなんでしょう．ちゃんと調べていないので
> よく知らないのですが，SQL99だと
> <character value expression>
> みたいです．

SQL92完全ガイドのおまけによれば、SQL92でも <character value
expression> となっています。で、<character value expression> は、
<value expression primary> なので、とくに1文字でなければならない、とい
う制限はないように見えます。
