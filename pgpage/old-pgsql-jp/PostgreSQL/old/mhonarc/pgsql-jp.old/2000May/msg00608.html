>$seq = nextval('yoyaku_seq')　とすると

この文をそのままSQLに食わせるとは思いませんが...。
nextval()は、PostgreSQLのCREATE SEQUENCEを実行したときに作られる
PostgreSQLの関数なんで、SQL文に入れれば使えると思います。
# serial型は、CREATE SEQUENCEを "こっそり(?)" 使用しています。

test4pg=# \d
       List of relations
    Name    |   Type   | Owner
------------+----------+--------
 samp       | table    | kamiya
 samp_i_seq | sequence | kamiya
(2 rows)                

test4pg=# \d samp
                            Table "samp"
 Attribute |  Type   |                   Modifier
-----------+---------+----------------------------------------------
 i         | integer | not null default nextval('samp_i_seq'::text)
 f         | text    |                             
Index: samp_i_key   

test4pg=# select nextval ('samp_i_seq');
 nextval
---------
       3
(1 row)  

INSERT 文の中で、このシーケンスを使う方法です:
INSERT INTO samp VALUES (NEXTVAL('samp_i_seq'),'foo');

シリアル型が指定してあるなら、無理にINSERTしなければ自動的に数値
が入ったと思いますが。
