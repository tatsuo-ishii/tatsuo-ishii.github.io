PostgreSQL 6.5.3 と PHP 3.12beta4 で、漢字の表名と列名を使って
いますが、いまのところ、文法的な問題には出会っていません。
# 漢字のＤＢ列名で問題があれば、私も知りたい。

pg_exec で、SELECT したあと、pg_fetch_object のクラス変数が
$rec->住所  というように漢字の変数名になりますが、これでも
動いています。
# PHP の文法上、漢字の変数名が許されているのか調べていない
# ので、ご存知の方は、ご教授願います。

あとは、ツールとか、言語や、開発方針なんかの問題ですね。
# db_ml でも話題になったことがあったと思うけど、賛否両論ですね。

下記のデモは、殆んどが漢字の列名・表名を使ってます。漢字だと
列名をそのまま見出しに使っても十分わかりやすいという利点が
あります。

DB検索デモ   
        http://www.softkoubou.co.jp/cslweb/index.php3

ただし、全角記号や全角英数字は避けた方がいいかもしれません。
# 本来、アスキーコードに対応する文字があるものを、全角半角
# で別の文字として扱うこと自体が好ましくないし。
# Informix も、何年か前に、全角記号が列名表名に使えなくなり
# 全部直したことがあります。 

列名に漢字を使うことの短所は、C 言語等漢字の変数を使えない
言語やツールを使用するときにめんどくさいということです。
あと、日本以外では使えないので、海外進出で外国に端末を置く
時に時に問題ですね。
