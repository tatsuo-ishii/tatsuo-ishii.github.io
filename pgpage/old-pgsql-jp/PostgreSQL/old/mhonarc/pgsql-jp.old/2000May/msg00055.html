update sample set data=translate(data,'b','z') where data ~ '^a-b';

ではいかがでしょうか？

但し、 6.5.3以前のバージョンではtranslate(a,b,c)でbとcは１バイト文字
しか指定できません。
7.0からは複数バイトの指定ができるという報告があったと思いますので
update sample set data=translate(data,'a-b','a-z') where data ~ '^a-b';
でもｏｋだと思いますが・・・

あとは、ユーザ関数を作るとか、ＰＨＰなどで処理するとかかな？

> 今、データの更新で悩んでおります。
> 下記のようにテーブル名sampleにcode,dataがあったとします。
> dateの「b」の部分を「z」に置換したいと思っております。
> 
> sample
> -------------
> code | data
> -------------
> 1    | a-b
> 2    | a-b-c
> 3    | a-b-d
> 4    | a-e
> 5    | a-b-d-f
> 6    | a-e-g
> 
> この場合は次のような手順でやらなければいけないのでしょうか？
> sql> select code from sample where data ~ '^a-b';
> sql> update sample set data = 'a-z'   where code = '1';
> sql> update sample set data = 'a-z-c' where code = '2';
> sql> update sample set data = 'a-z-d' where code = '3';
> sql> update sample set data = 'a-z-d-f' where code = '5';
> 
> 出力されたcodeを1つずつupdateをする方法しかないのでしょうか。
> 下記のような置換はないのでしょうか。
> sql> update sample set data = /^a-b/a-z/ where data ~ '^a-b';
