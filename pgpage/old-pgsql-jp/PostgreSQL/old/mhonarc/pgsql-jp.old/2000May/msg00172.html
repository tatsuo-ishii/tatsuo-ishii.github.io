PostgreSQLをWindows98で使用可能かということですが・・・。
DBを使ったシステムは、基本的にバックエンド（サーバー）とフロントエンド
（クライアント）に別れます。まぁ、中間に位置するものが他にあってミドルウェ
アなどと呼ばれてたりします。バックエンドにはDBエンジンと呼ばれるデータベー
スを管理するアプリケーションがあります。
PostgreSQLではpostmasterがそれです。
MSで言うところのMS-SQL Server ServiceやAccess2000に付属のMSDE、JETデータ
ベースエンジンにあたります。
MS-Accessはデータベースエンジン（バックエンド）とフロントエンドが一体にな
ったアプリケーションと考えれば良いと思います。（実際にはJETはモジュール化
されていますので、 Excel、VBなどからもDB本体(MDBファイル)にアクセス可能で
す。VBでAccessライクな物を作ることも可能です）。

postmasterはWindows上では動かない（と思います）。
ので、PostgreSQLをWindowsシリーズにインストールする事は無理です。
ただ、UNIX系サーバーにインストールされたpostmasterに対して、Windowsからア
クセスしてデータのやり取りをする事は可能です。

代表的なのはODBC(Open DataBase Connectivityだっけ？）を利用する方法です。
ODBCは、規格の異なる各種DBに対して統一した窓口を用意してくれます。
で、PostgreSQL用のODBCドライバってのがありまして、コレをインストールする
とWindowsからPostgreSQLデータベースを利用可能になります。
ODBCドライバは本家も用意してくれていますが、元のバグを修正して日本語に対
応させたODBCドライバを片岡さんが提供して下さってますので、（感謝しながら
：笑）それを使うと良いでしょう。これがあれば、VBやらExcelやらAccessなどか
ら PostgreSQLを利用する事ができます。

他には、Webブラウザを使ってPostgreSQLとデータをやり取りできます。
正確にはWebブラウザで直接やりとりする訳じゃなく、データの入力やら閲覧やら
が出来るって事です。
MSのASP(Active Server Pagesでしたかね？）と似たものが、UNIX系にも数種あり
まして、それらがPostgreSQLに対応しているので、Webサーバーを介して、クライ
アントのWebブラウザからデータのやり取りが可能・・・って意味です。
WebサーバーってのはMSではIIS(Internet Information System)やPWS(Pear Web
Serverまたは Personal Web Server)にあたります。
誤解無きようお願いしたいのは、ASPはODBCによるDBへのアクセスが可能なので、
ASPからPostgreSQLとデータのやり取りをしても問題ない（と思う）って事です。
WebサーバーにIIS、ミドルウェアにASPを・・・とMSで固めたから、DBはMS-SQLや
MSDEじゃなきゃダメって事はないです。

古いPCで構わないので、1台に Linux入れてPostgreSQLを入れちまえば、後はいろ
んな方法でPostgreSQLを利用する事ができるので、他が全部MSでも、問題ありま
せん。

・・・・と丁寧に説明してPostgreSQLユーザーを逃がさない・・と。
