VMware は一口で言えば、Linux や NT 上で仮想マシン(仮想的な PC ハードウェア
システム)環境を提供するものです。

VMware 自身を走らせる OS をホスト OS、仮想マシン上で動かす OS をゲスト
OS と呼びます。
ゲスト OS は、仮想マシン上でブートしてゆくわけです。確か、VMware の
仮想マシン上では仮想 BIOS みたいな環境も提供されていたはず。

ちなみに、この "ホスト OS"、"ゲスト OS" という呼び名は VMware 用語では
なくて、仮想マシン環境を提供する VM 系 OS (汎用機の世界では昔から
こういうものがあります)では一般的な呼び名です。

Linux 版 VMware でも、ゲスト OS が Windows 系だと決まっているわけでは
ありません。FreeBSD とか、Linux などをゲスト OS 上で動かす事もできます。

# 自分で VMware を試した事はないので、正確には「できるそうです」けど…

原理的にはゲスト OS は何でもよい事になってるのですが… まぁ、中には
動かないものもあるかも知れません。
動かない OS にとっては、仮想マシン環境が不十分であるという事ですね。
完全な仮想マシン環境であれば、何でも動くはずです。

Linux 版の他に、NT/Windows2000 版 VMware もあります。
こっちの方では、NT 上の VMware の仮想マシン上で、Windows98 を動かしたり、
Linux や FreeBSD とか、NT を動かす事もできるでしょう。
