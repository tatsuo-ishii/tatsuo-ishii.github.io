早速試してみました。（コンパイルからtest実行までですが…）

- UnixWare 7.1.0, gcc-2.95.1, postgres6.5.1, bash-2.0.3
- Solaris 2.6 x86, gcc-2.8.2, PostgreSQL-6.5, bash-2.0
の2つです。
両者とも、
gcc -I../../include -I../../backend    -Wall -Wmissing-prototypes -I../../interfaces/libpq -fPIC   -c exec_sql.c -o exec_sql.o
In file included from exec_sql.c:42:
exec_sql.h:103: parse error before `__P'
exec_sql.h:104: parse error before `__P'
exec_sql.h:105: parse error before `__P'
となりましたので、

stones.my.domain% diff ~/pgbash-1.2.1/exec_sql.h exec_sql.h
37a38,44
> #if !defined (__P)
> #  ifdef __STDC__
> #    define __P(x) x
> #  else
> #    define __P(x) ()
> #  endif /* __STDC__ */
> #endif
と変更を加えています。

testはOKでした。

上の変更は、私のgcc環境のせいかも知れません。（sys/cdefs.hが存在しない
ので、__Pマクロが定義されていない様子でした。）

以上です。
