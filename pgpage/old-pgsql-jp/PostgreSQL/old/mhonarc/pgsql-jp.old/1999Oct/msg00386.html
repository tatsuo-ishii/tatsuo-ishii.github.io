[pgsql-jp 11362] 複数テーブルからデーターをセレクトする方法

  • From: "tyuuniti" <tyuuniti@cablenet.ne.jp>
  • Date: Wed, 27 Oct 1999 23:31:06 +0900
  • Importance: Normal
こんにちは、范＠初心者です。

postgresql 及び php3を勉強中です。ホームページで商品情報を検索できるように
下記ようなdatabaseのテーブルを作成：

create table goods_tab
(
    code      char(10),
    henmei     char(20),
    price      int,
    kubun    int2,
    .....
);


create table kubun_tab (
kubun int2,
kubun_name varchar(50)
);

そこで、code, hinmei, price, kubun_name をホームページに表示するため
php3のＳＱＬ文は書きように書きました：


SELECT goods_tab.*, kubun_tab.*, goods_tab.kubun
FROM goods_tab INNER JOIN kubun_tab ON goods_tab.kubun = kubun_tab.kubun
WHERE (((goods_tab.kubun)=$kubun));

エラーなりました。（access97では、正しく表示できます）

それで下記ように書き直した：

SELECT goods_tab.*, kubun_tab.* FROM goods_tab WHERE goods_tab.kubun =
$kubun;

$kubunは別のphp3より渡された変数です

データーは表示にされましたが、でも　kubun_tab の件数分を
同じデーターは繰り替え表示され、複数テーブルからデーターを
正しく抽出する方法を教えていただけますか？

よろしく御願い致します。
