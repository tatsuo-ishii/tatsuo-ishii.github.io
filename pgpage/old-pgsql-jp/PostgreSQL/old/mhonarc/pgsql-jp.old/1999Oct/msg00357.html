>   わたしは知りません。少なくとも誰がロックしているかを知る正しい方法というの
> は無いと思います。

ようするに、元記事の方はSELECTから返って来ないのだが、もしかしたらロッ
クされているかもしれないと言っているだけですので、実際にロックされてい
るのかどうか全く不明なわけです。まず最初にすべきことは、postmaster を
起動する際に -d 3 オプションを付けて実際に何が起きているのか調べること
でしょう。すべてはそれからだと思います。

それと、vacuum を cron から起動させているのなら、そのスクリプトの中で 
vacuum が正常に終了したかどうかわかるようにするのは簡単なことです。

echo vacuudm begin
vacuumdb hoge
echo vacuumdb end

とか。とりあえず情報収集に全力を尽くすべきだと思います。

なお、本当にselectがvacuumの完了待ちになっている場合は、Linux なら ps
auxw|grep post などでそのことを知ることができます。

postgres  6778  0.5  3.5  8608  2816  ?  S    08:34   0:00 /usr/local/pgsql/bin/postgres localhost t-ishii test SELECT waiting
