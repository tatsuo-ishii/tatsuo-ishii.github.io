>杉原＠大阪府立中央図書館です。
>
>> そこで今回、早速なのですが、大量データをCOPYした
>> ときのCOPY時間に関して教えていただきたいことがあります。
>>
>> 列数30（1レコードがテキスト形式で160byte）の表に、
>> 10万レコード分のデータが入ったテキストファイルを
>> 10個用意し1ファイルずつCOPY-FROMして、最終的に
>> 100万レコードを格納しました。
>> COPY-FROMの際に各ファイルごとに処理時間をtime
>> コマンドで記録していったのが下のデータです。
>> その結果、1ファイル（10万レコード）COPYするたびに
>> 時間が15分程度ずつ増してしまいます。
>>
>> これでは、大量なデータを格納するようなデータマートなど
>> のアプリケーションでは時間がかかりすぎて使えません。
>>
>> なぜ、このようにデータを追加するたびにCOPY時間が増して
>> いくのか、また、それを回避する方法をご存知の方いらっしゃっ
>> たら教えていただけないでしょうか。
>
>この現象は、postmaster の起動オプションで
>postmaster -o　'-F'
>を指定することにより回避できます。
>私も初心者なので、詳しいことは分かりませんが、
>おそらく、このオプションが無い場合はテーブルの整合性を保ち
>ながらcopyしているような気がします。
>（このオプションを付けてうまくcopyできなかったことはありませんが）


アドバイスに従ってpostmasterを
　postmaster -i -o '-F' -S
とオプションをつけて、前回と同じCOPY-FROMのテストを
行いました。

結果は、1ファイル（10万レコード）のCOPY-FROMが約27秒で、
１０個のファイルすべてほぼ一定した時間で完了しました。
前回のテストでは、10個目のファイルをCOPY-FROMするのに
2:11:04もかかっていたのに！！！！

大量データを追加する場合には、このオプションを付けたほうが
良いですね。
ただ、このように起動した場合には何に気を付けないといけないか
を知っておかないと危険そうですね。
