At 14:51 +0900 99.10.04, Hirofumi Terada wrote:
> ld: cannot open @FDFTK_LIBS@: No such file or directory
> apxs:Break: Command failed with rc=65536
> make: ***[libphp3.so] Error 1
> ●質問
> ○どうしたら正常にインストール出来るのでしょうか？

apxs が「@FDFTK_LIBS@」と　言ってくる場合は　apxs が　正常にインス
トールされていない or apache のソースフォルダにある apxs を使おう
としているのではないかとおもいます.

apache のインストールされているディレクトリを 指定して

--with-apxs=/usr/local/apache/bin/apxs 

を　configure 時に　つけてみたらどうなりますか？
もしくは　Makefile のしかるべき行を編集するとか
＃/usr/local/apache は Apache のインストール先に　てきとうに変えて
＃ください．

＃apxs は perl のスクリプトなので　エディタで　みてみるとか... :-p
