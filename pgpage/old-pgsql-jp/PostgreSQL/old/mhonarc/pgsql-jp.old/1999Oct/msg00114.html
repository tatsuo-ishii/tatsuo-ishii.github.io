渋谷です。

私も Cobalt RaQ2J(64MB, 4.3GB)でベンチしてみましたので、メールします。

RaQ2J の CPU は MIPS 250MHz で、RedHat 5.1 がベースになっています。

RaQ2J では -B 512 は core dumpし、-N 64 は起動できませんでした。
PostgreSQL は 6.5.1(本家 SRPMS を Cobalt 用に書きなおしたもの)です。

postmaster のオプションは "-i -S -D/var/lib/pgsql" に表のオプションをつけて
実行しています。
pgbench のオプションは "-t 100" です。また、Makefile.in を RedHat 用のパスに
変更しています。

# 開発用の PII300, 128MB, Vine-1.1 で X 起動中(メモリフリーはほぼ0)より少し
# 早い結果でした。

結果としては、メモリを積めばそれなりに使えそうだという感触でした。
