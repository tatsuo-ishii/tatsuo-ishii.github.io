はじめまして、菅野岳郎＠HTSと申します。よろしくおねがいします。

　私は現在仕事でLinux、PostgreSQL、PHP3を使ってWebからデータベースを検索
するアプリケーションを作成しようとしているのですが、PostgreSQLに問題が発生
していて大変困っております。PostgreSQLおよびLinuxははじめてなのでご迷惑を
おかけしますが、ぜひMLの皆さんのご協力をお願いします。なお開発は以下のような
環境で行なっております。
	OS:Turbo Linux 3.0 日本語版
	DB:PostgreSQL6.5.2

　さて、どのような問題か具体的に言うと、PostgreSQLをインストールする際に
環境変数の設定として、.bash_profileの行末に以下のような6行のスクリプトを
追加しました。

export PATH="$PATH":/usr/local/pgsql/bin
export POSTGRES_HOME=/usr/local/pgsql
export PGLIB=$POSTGRES_HOME/lib
export PGDATA=$POSTGRES_HOME/data
export MANPATH="$MANPATH":$POSTGRES_HOME/man
export LD_LIBRARY_PATH="$LD_LIBRARY_PATH":"$PGLIB"

このように設定して、

# source .bash_profile

を実行しました。すると実行直後は

$ psql

を受け付けてくれて、PostgreSQLを操作できるのですが、
いったんPostgreSQLのスーパーユーザーでなくなる、もしくはそのまま
ログアウトして、もう一度PostgreSQLのスーパーユーザーになって

$ psql

とコマンドラインから入力しても
.bash_profileの設定が無効になってしまうのか、わからないのですが

bash: psql: command not found

と返事が返ってくるだけで、psqlが使えません。仕方がないので、もう一度

# source .bash_profile

と入力してやっとpsqlが使えるようになります。つまり使う前にいちいち
sourceコマンドを実行しないとPostgreSQLが利用できないのです。
Postmasterの自動起動は正常に行なわれています。
PostgreSQLのインストールには

　　『Linux/FreeBSD　SQLデータベース構築入門』
　　（トップマネジメントサービス著）

という本を参考にしました。
　
　この件に関してどなたか詳しい方がいらっしゃいましたら、
フォローをよろしくお願いします。
