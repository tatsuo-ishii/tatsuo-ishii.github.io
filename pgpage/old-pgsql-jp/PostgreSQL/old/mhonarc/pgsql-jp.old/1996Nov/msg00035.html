Tatsuo Ishii <t-ishii@sra.co.jp> さんは書きました：

>o PHP/FI を make するときに apache のモジュールを作ることにすると、
>php.cgi ができません。こういうもんなんですか？
そうだと思います。ただし、プロバイダ等のサーバでは、rootにはなれないので、
私は、php.cgiだけ別のマシンでコンパイルしてホームディレクトリのpublic_htmlの
下にphp.cgiをコピーし、.htaccessを作成して、phpが動くようにしました。^_^

>o PHP/FI を作りなおす時に、毎回 install の沢山の質問に応えるのが苦痛な
>のですが、何か良い方法はないでしょうか？

コンパイル用のMakefileだけ別に持ってればよかったんじゃなかったかな？
ただし、同一バージョンが前提ですが。
おおウソだったらご容赦ください。

>frametest の下に .htaccess と .htpasswd がありましたが、サンプル用でしょ
>うか。
.htaccessの中身は以下のようになっているので、自分用に直さないと、
自分でアクセス出来ません。私はこれでハマってました。^_^;;
AuthName "Frametest Area"
AuthUserFile /home/juk/public_html/frametest/.htpasswd
<LIMIT GET>
        require valid-user
        deny from all
        allow from .rccm.co.jp
</LIMIT>

追伸
このデモプログラムは素晴らしいです。ただ、現在のphpのバージョンの関係で、
一部がうまく動きませんでした。 関数StripSlashesがうまく機能しなかったのです。
