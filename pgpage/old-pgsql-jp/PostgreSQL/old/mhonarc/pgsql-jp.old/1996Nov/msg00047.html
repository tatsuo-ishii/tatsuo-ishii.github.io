始めまして、なんですが、実は今まで Nifty から参加させて頂いていたんです。
ようやくメールの環境が揃ったので、今後はこちらから参加させて頂きます。
しばらくネットワークの設定に夢中で、やっと postgres95 を再インストール
し始めました。が、新しい環境になると postgres が core dumpe してしまい
ます。なぜなんだろう？
今の環境は FreeBSD 2.2-961014-SNAP と postgres95-1.09 + jp.patch
です。
/usr/local/postgres にソースを展開し、Makefile.global の PORTNAME=
を BSD44_derived に変更、src/mk/port/postgres.mk.BSD44_derived
の LDADD+= -L/usr/local/lib -lfl の部分を LDADD+= -L/usr/lib -lfl
に変更して、/usr/src/usr.bin/lex が flex なので、ここに README.flex の
patch を手であてて、make;make install のあと postgres95 を gmake して
gmake install それから initdb しました。
あ、ここまでの作業は、以前からあった postgres のアカウントで作業してい
ます。ここでもちろん postgres のアカウントで postmaster -S とやると

Bad system call (core dumped)

になってしまいます。前は問題なかったのに。flex の関係でしょうか?
もし、同じようなトラブルで解決された方がおられましたら教えてください。

挨拶か質問か分からなくなってしまいまして、失礼しました。
では今後とも宜しくお願いします。
