>openfilesを20ぐらいにするとさらに安定するのでしょうか。

ユーザ数によります。pstat -T で、

255/680 files
51M/127M swap space

のような表示が得られると思いますが、255/680 の 680 が file table の数
です。

最大ユーザ数(正確にはバックエンドの数)を n とし、n * 20 を計算してみて
これに対してfile table の数にものすごく余裕があれば(10倍以上)
openfilesを20にする必要はあまりないでしょう。

もちろん openfiles を設定すれば更に安心ですが。
