"Tomoaki Kohzu." <kohzu@helios.sumitomocorp.co.jp> さん:
>テーブル定義も自動的に行うという事ですか？

テーブルは create してから実行するようです。
最初から作るときは、一度テーブルをdropします。
ORACLEでは下記のようなSQL風な指定をSQLloaderに対して行います。

LOAD DATA
INFILE '入力ファイルのパス名.dat'
INTO TABLE テーブル名
APPEND
FIELDS TERMINATED BY '	'
TRAILING NULLCOLS
(
  colomn1,
  colomn2,
  colomn3,
	...
)

この時、下記のようなインデックス用テーブルが定義されていれば、
インデックスも生成してくれるようです。
CREATE UNIQUE INDEX インデックステーブル名
   ON テーブル名
(
        column1
);

インデックスは、最初作らずにデータだけロードしてしまい、ロードが終った時点で
上記のインデックステーブルを生成するSQLを実行してもいいようです。

>テーブルがすでに用意されていることを前提にすると、下記の sql文 があります。
>copy 投入先テーブル名 from 'パス/テキストファイル名.csv';
>ただし、あくまでも追加データの投入という事になりますので、
>既存データを上書きする事はできないですね。

copy コマンドは、このような操作と同等なことができるのでしょうか。
copy は 1レコードづつ insertするよりは高速処理が期待できるのですね。
インデックス情報も上記のように定義されていれば、作られるのでしょうか。
psqlレベルで、全て解決できるとしたら、嬉しいですね。
