島＠札幌在住のものです。

こんにちは。

RedHatLinux 4.2(2.0.36) + 6.4.2 + patch(2/20のときの) で
本に書いてある、郵便番号の検索DBを構築しているのですが、
amdでmountしている先ハードディスク上のpgpost/data以下に移動して、

psql -f create.sql postal
とやると、
copy postal from '/mnt/xxxx/pgpost/data/postal.data';
ERROR:  COPY command, running in backend with effective uid 104, could not pen
file '/mnt/xxxx/pgpost/data/postal.data' for reading.  Errno = No such file 
or directory (2).

とエラーがでてるのですが、
mount先で作業するのではなく、
ログインマシンのハードディスクで作業をすると
きちんと読み込まれます。

作業をしているマシンは、postgreSQLのクライアントで、
PGHOST,PGPORTなど、設定して、
postgreSQLのサーバに psql -lで通信はできているのです。

ちなみに、マウント先のcreate.sqlの /mnt/xxxx/pgpost/data/postal.data
を/tmp/postal.dataに変えて、同ファイルを/tmp以下にコピーしても同じ結果となり
ました。

マウント先で作業をすると×なんてことはあるのでしょうか？
