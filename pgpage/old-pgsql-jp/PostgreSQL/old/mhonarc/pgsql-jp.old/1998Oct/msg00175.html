川野＠ステーションガイアです。

Postgresの勉強中ですが、「like」と「~」の違いが、よく分かりません。
教えていただけますでしょうか。

雑誌のＣＤＲＯＭに付いていた、「郵便番号一覧」(?)データを元に
次のようなテストしました。

０．市区名(kanj_city)でインデックスを作成
	> create index kanj_city_key on zip_table using btree (kanj_city);

１．「~」でselectする。
	> select kanj_city from zip_table where kanj_city ~ '^川崎市*';

２．「like」でselectする。
	> select kanj_city from zip_table where kanj_city like '川崎市%';

perlから１と２のテストを１０回／セットで行うと
	１の場合　平均　０．３３秒／１０回
	２の場合　平均　０．４３秒／１０回
となりました。

検索結果は、同じようになりましたが、スピードの違いがでております。
今まで、書式の違いだけで内部的には同じことを実行していると思って
いたのですが、どのような違いがあるのでしょうか。また、どのように
使い分けているのでしょうか。


また、下記の環境でテストしました。

CPU: Pentium II 400Mhz
MEM: 128MB

Postgres 6.3.2 + jumbo_patch
