NEC情報システムズの杉村と申します。はじめまして。

NECのあるところで200台くらいのWSとそれらで
構成されたネットワークの管理をしています。
データべースについては、最近SoftwareDesignや
UNIX MAGAZINEの連載記事を読んだ程度の初心者です。
(まだcreate indexがいまいちわかってない)
よろしくお願いします。

いまはNEC/EWS4800 R12.1 と SUN/SS20 Solaris2.5 で
postgreSQL-6.3.2を動かしてます。

ここに現在WSに登録されているアカウント名と同じ
名前を create user しようと思ったのですがつまづきました。


(1) アカウントsugimura とアカウントsugi

hoge=> drop user sugimura;
ERROR:  removeUser: user "sugimura" does not exist
hoge=> drop user sugi;
ERROR:  removeUser: user "sugi" does not exist

という状態で create user すると

hoge=> create user sugimura;
CREATE USER
hoge=> create user sugi;
ERROR:  defineUser: user "sugi" has already been created

アカウントsugiが作成できません。
しかし作成する順番を変更すると作成できます。

hoge=> create user sugi;
CREATE USER
hoge=> create user sugimura;
CREATE USER

すでに存在しているユーザ名との比較方法に問題がある
ような気がしますが、どうなのでしょう。

(2) アカウントm-sugi

hoge=> create user m-sugi;
ERROR:  parser: parse error at or near "-"

と、parseエラーになります。

hoge=> create user "m-sugi";
CREATE USER

と" "で囲んでやると作れましたが、
(postgres% createuser m-sugi でも作れました)

hoge=> grant select on table to m-sugi;
ERROR:  aclparse: non-existent user "m"
hoge=> grant select on table to "m-sugi";
ERROR:  aclparse: mode flags must use "arwR"

grant で parseエラーになってしまいます。
回避する方法はあるのでしょうか？


以上、つまらない質問かもしれませんがよろしくお願いします。
