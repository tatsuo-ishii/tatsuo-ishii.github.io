う〜ん、ORDER BY のこの制限がなくなると

skk=> SELECT kanji FROM minakaji_private_jisyo WHERE yomi = 'かんじ' AND okuriari = 0 ORDER BY date DESC; 
kanji
-----
漢字 
完治 
寛治 
幹事 
監事 
(5 rows)

こういう風に出力できて、Emacs の中では

(setq test "漢字 
完治 
寛治 
幹事 
監事 ")

  --> "漢字 
完治 
寛治 
幹事 
監事 "

(split-string test "\n")
  --> ("漢字 " "完治 " "寛治 " "幹事 " "監事 ")

;; このように Lisp のリストに候補を新しいもの順に入れられたら良いので
;; す。
;; ちょっと不要な空白が入っているけど、これを取り除くのは簡単。

のように加工作業がかなり簡素化できそうです。これは本当に嬉しい p(^_^)q。
