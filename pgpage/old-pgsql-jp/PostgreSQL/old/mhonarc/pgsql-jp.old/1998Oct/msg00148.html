いままで、データーベースというものは、ずっと昔にボーランドの パラドッ
クス for win というのをいじって自分のもっているレコードの検索なんかが
出来るようにトライしたことがあるのですが、以来数年御無沙汰しているよう
な状況です。

この度 PostgreSQL を linux にインストールして、なんとか create table
hoge ... とか insert into hoge values (...); とかして select * from
hoge; なんてやってみて「なるほど〜」と感心しているようなレベルです。
ＳＱＬというのもそういう言葉がある、くらいしか知らないので大変です。^^;

で、さっそくぶち当たっているのですが、昔昔のパラドックスの経験でいえば、
テーブルにはキーというものがあって、特定の項目にキーを設定すると、キー
が重複しないように出来る、とおもうのですが、PostgreSQL でそういうこと
をするにはどうしたら良いのでしょうか？つまり

hoge| poo
----+----
foo |baa

というようにデータが１個だけあるテーブルに

hoge| poo
----+----
foo |baa
foo |boo

というような行が足せてしまうのですが、そうでなくて「キーである項目 
hoge にはすでに foo という値があるので、もう一個 hoge = foo な行を足す
ことは出来ません」としたいのですが・・・・

こういうのは PostgresSQL にデーターを渡す以前にやらないと行けないので
しょうか？たぶんそうじゃないと思うのですが・・・・

とても間抜けなＱだとおもうのですが、どなたかどうか教えて下さい。 m(_._)m

そのレベルであれば◯◯が参考になるはず、なんていう情報も大変嬉しいです。
