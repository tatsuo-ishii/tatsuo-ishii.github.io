話は MUA に限ったことではなく、いわゆる「パソコン通信」の site に *無
手順* 接続している場合なども同じ現象が発生する可能性がありますね。

＃ System が標準で取り扱う種類以外の header を host に渡すことができな
＃ い。

これには Nifty ROAD2〜7（HyperROAD は除く）や ASAHI-Net、PC-VAN などが
含まれます。いずれの場合も操作 menu（若しくは command）で「返信」の操
作をすれば大丈夫なのかもしれませんが、Nifty4U を用いて Nifty につない
でいる場合は間違いなく駄目です。

＃ Nifty4U+ は使っていないのでわかりません（事情はあまり変わらないよう
＃ な気がしますが）。また、「返信」の操作をした場合 header 情報がどう
＃ なるかも調べていないのでわかりません。Nifty-Manager から HyperLOAD 
＃ につないだ場合（おそらく host-user 間には SMTP は使われない）はどう
＃ なるんだろうか？

まあ、これらの site で mail の送受信を行なう場合でも、PPP で接続して 
SMTP で送りつける分には問題ないはずなんですが‥‥。Nifty もようやく 
SMTP を support したことだし。

＃ まさか、gateway でどっかの header を意図的に reject するなんてこと
＃ は‥‥
