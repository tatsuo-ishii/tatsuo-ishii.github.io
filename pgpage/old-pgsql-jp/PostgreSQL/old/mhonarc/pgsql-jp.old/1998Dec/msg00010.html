石井です。

# 操作ミスで空メールを出してしまいました。どうも済みません。

ところで、

> KIUCHI@kobeです。
> 初歩的なご質問ですが宜しくお願いします。
> こちらの環境は
> linux 2.0.33 + postgresql 6.3.2です。
> 
> 以下のようなテーブルを作成し
> AA,CC,DDにデータが入っている状態でBBにはデータが
> 入っていない場合、BBにデータが入っていないレコードが２個以上
> ある場合、SQL のupdate Commandを利用し、データを更新しよう
> とすると、duplicateできないというメッセージが出て、updateが
> 完結しないのですがなにか良い方法があるでしょうか？
> 宜しくお願いいたします。_(_ _)_
> 
> create table A(
> AA   int4   primary key,
> BB   varchar(20)   UNIQUE,
> CC   varchar(20) ,
> DD   varchar(20)
> );

「データが入っていない」というのは SQL の NULL のことですね。SQLでは、
UNIQUE なカラムでは、同じ値が2個以上あってはいけないし、NULL が2個以上
あってもいけないとしています。

したがって、質問の状態は、正しい(SQLで定義された)動作なので、やむを得
ないでしょう。
