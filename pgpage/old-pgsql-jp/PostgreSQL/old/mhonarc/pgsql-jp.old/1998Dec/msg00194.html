はじめまして、手嶋と申します。
postgresql とＰＨＰを使ってお客さんと店舗のマッチングの実験を
やっています。

今までEUC_JPで動的にページを作成していて問題なく動いていた
のですが、ＳＪＩＳに変更しようと思い、
   initdb -e "SJIS" 
   createdb -E "SJIS dbname
   PGCLIENTENCODING=SJIS
と設定し、以前のEUCのデーターをnkfで変換して、
copy コマンドで新しいデーターベースに挿入しました。

ここで、
dbname=>\z
dbname=>\dt
などのコマンドを実行すると
pqReadData() -- backend closed the channel unexpectedly.
This probably means the backend terminated abnormally before
 or while processing the request.
となります。
またLIKE 演算子を使った以下の様な問い合わせを実行すると、
select * from tablename where colname LIKE 'te%';
上と同じエラーメッセージが表示され、バックエンドが止まってしま
います。
select * from tablename where colname='tejima';
の様な問い合わせは問題ありません。

何が原因なのでしょうか？

環境は
Red Hat Linux release 4.2   Kernel 2.0.35 on an i686
PostgreSQL 6.4 + jumbo_981104.patch 
  ./configure --with-mb=EUC_JP  --with-perl
PHP-3.0
です。

よろしくおねがいします。
