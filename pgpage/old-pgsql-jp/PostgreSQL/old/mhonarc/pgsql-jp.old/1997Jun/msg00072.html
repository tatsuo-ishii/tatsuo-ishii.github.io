  話しが脱線してしまいますが、6.1 はリリースになったのかな。
  6.1 への移行をしようとして、 pg_dump で flex-2.5.3 のバグによるエラーに
逢ったので報告しておきます。バージョンの移行の参考になるかと思いますので、
pgsql-jpMLの方にも投稿します。


石井さん:
> P.S. 本日会社の MkLinux に apache-2.1+PHP/FI-2.0b11+おみつさんの漢字パッ

  ぼくもアップデートしてみようと思い、本家サイトにアクセスすると
postgresql-v6.1.tar.gzというファイルがダウンロードできてしまいました。石
井さんの 6.0 のときの jp.patch をあてて一部 Makefile を直したのですが、
実行モジュールまではできました。

postgresql-v6.1/src/backend% diff -u Makefile.orig  Makefile
--- Makefile.orig       Thu May 22 13:00:15 1997
+++ Makefile    Sat Jun 14 16:25:50 1997
@@ -41,8 +41,14 @@
 SRCDIR = ..
 include ../Makefile.global
 
+ifdef JP
+REGEX= regex.jp
+else
+REGEX= regex
+endif
+
 DIRS = access bootstrap catalog commands executor lib libpq \
-       main nodes optimizer parser port postmaster regex rewrite \
+       main nodes optimizer parser port postmaster $(REGEX) rewrite \
        storage tcop utils
 
 ifdef TIOGA


インストールをする前にデータのダンプをするのを忘れてたので、pg_dump
を実行すると flex-2.5.3 のバグによるエラー

	isViewRule(): SELECT failed

が出てしまい、 flex-2.5.4 をインストールした後にもう一度 6.0 をインストー
ルし直して、やっと、postgresql-v6.1に附属の全ダンプ用のシェルでダンプする
ところにまで至りました。

	% postgresql-v6.1/src/bin/pg_dump/pg_dumpall > dump.all
