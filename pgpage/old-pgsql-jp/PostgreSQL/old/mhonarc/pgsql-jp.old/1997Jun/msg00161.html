[pgsql-jp 2039] Cannot test Wisconsin Benchmark

  • From: tsone@ibm.net
  • Date: Tue, 24 Jun 1997 07:04:02 +0900
  曽根俊彦 (tsone@ibm.net) です。

  FreeBSD 2.2.1-RELEASE 上で postgreSQL v6.1 をインストールして見まし
た。インストールには成功したようなのですが、regression test を実施後、
Wisconsin Benchmark のテストをしようとしたところ、

% gmake bench.out.perquery

/bin/sh ./create.sh && \
/bin/sh ./runwisc.sh >bench.out 2>&1
=============== destroying old bench database... =================
FATAL 1:Database system does not exist.  PGDATA directory 'template1'
not found.  Normally, you create a database system by running initdb.

=============== creating new bench database... =================
FATAL 1:Database system does not exist.  PGDATA directory 'template1'
not found.  Normally, you create a database system by running initdb.

FATAL 1:Database system does not exist.  PGDATA directory '-Q' not
found.  Normally, you create a database system by running initdb.

RESULTS OF BENCHMARK ARE SAVED IN FILE bench.out
/bin/sh ./perquery <bench.out 2>&1 > bench.out.perquery
BREAKDOWN OF BENCHMARK IS SAVED IN FILE bench.out.perquery

といったような状態で、終了してしまいます。

  当然 bench.out.perquery は、

query  0:   0.000 real   0.000 user   0.000 sys

という結果に終っています。

  インストールについては、postgreSQL(J) メーリングリストのページの「イ
ンストールの手引」にほぼ従いました。

１ 一般ユーザー tsone でファイルの展開、パッチ当て、コンパイル、インス
   トールを行う。

２ JP=1 の1行だけの Makefile.custom を作成。

３ ユーザー postgres を作成。path に /usr/local/pgsql/bin
   LD_LIBRARY_PATH に /usr/local/pgsql/lib を追加。/usr/local/pgsql の
   オーナーを postgres に変更。

４ ユーザー postgres で regression test を実施。

５ src/test/bench に移動後、環境変数 PGDATA に /usr/local/pgsql/data 
   をセットしています。

  ベンチマークテストの結果を投稿している方の中には、FreeBSD ユーザーも
いらっしゃるようなので、私の何らかのミスであると思いますが、何が原因な
のか、見当が付きません。

  解決法を教えて頂ければ幸いです。よろしくお願いします。

           曽根俊彦
        (tsone@ibm.net)
