>とうとう、Apache 1.2が安定バージョンとしてリリースされましたね。
>Apache + Php/fi + PostgreSQLで、社内の図書の検索システムを作成中ですが、
>PostgreSQL 6.1が出れば、併せて全てを一新しようと思います。
>ついでに、ＯＳも、Linux1.2.13→2.0.29にします。

昨日さっそく Apache 1.2 + PHP/FI 2.0b11 + おみつさんの漢字パッチ +
PostgreSQL 6.1 beta + jp.patch + MkLinux.patch を手元の MkLinux
DR2.1+shared lib に入れてみました。PHP/FI以外は標準モジュールのみです
が、お蔭様で順調です。PHP/FI の regression test も pg95 用のテスト
(PostgreSQL でも動く)も含めてほぼ通りました。

通らなかったのはフィールドタイプを取って来るテストでした。PostgreSQL 
では float = float4 なのですが、想定されている結果は float = float8 で
した。多分 postgres95 の時に作られたテストスィートだからでしょう。

ところで、PostgreSQL 6.1 ではかなり機能追加もあり、なかなか楽しみです
が、データベースの中身は 6.0 と互換性がありません。そこだけちょっと注
意が必要です。(6.1 のインストールマニュアルに 6.0 からの移行方法が詳し
く書かれているので心配はありません)

後、6.1 では新しく Sparc/Linux もサポートされてますね(いしかわさん、ご
苦労さまです)。ただ、6.0 の時と同様 SunOS は難ありです。一応パッチも本
家に進呈したのですが、不採用でした:-<
