石井です。

>> 「書き込む」っていうのはどこに何を書き込んでいたのでしょう。正確にどの
>> ような手順でコンパイルを行なったのか示して下さい。
>
>　前述のとおり、記憶が錯綜してますが、教本「SQLデータベース構築入
>門(ROCUS社)」にはMakefile.customというファイルに、MB=EUC_JPと一行
>書き込んでコンパイルすると、日本語が使えるようになると書かれていま
>す。
>　後はmake、make installの順番になっています。

この手順は PostgreSQL 6.3.2 以前にしか適用できません。6.4 以降では、

./configure --with-mb=EUC_JP

を実行後、make; make install の手順です。

>　――と、いうことは、今使用しているPostgreSQLは、EUC以外のコード
>を受け付けない設定になっているという事の様ですね。
>　perlでの直接書き込みも含め、これは真剣にインストールのやり直しを
>検討しないといけないのかもしれません。

そうです。間違った手順でインストールされたので、現状英語版になっている
はずです。
