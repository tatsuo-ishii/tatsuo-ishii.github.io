lpk_kanto_yoshikawa@mba.nifty.ne.jpさんの<37BA3408.7A0BF98B@mba.nifty.ne.jp>から
>　ただ、6.4.2が既にある環境では、どうやらmakeのやり直しは出来ない
>ようで、エラーになりますので、思い切って6.5.1をダウンロードして、
>こちらにアップグレードする事にしました。

makeするだけならversionは関係ないと思いますが、ご使用のOSを
触ったことがないのでなんとも言えませんけど。

>　日本語対応のEUC_JPとperlのインターフェース導入を両方指定する方法
>を探しています。せっかくURLも教えていただいたので、そちらもあたっ
>てはみたのですが、探し方が悪いのか、./configure-のオプションの付け
>方がみつからずにいます。
>　素直に考えると、
>　../configure --with-mb=EUC_JP-perl

 % ./configure --with-mb=EUC_JP --with-perl

とすれば良いです。
 % ./configure --help
としてどんなオプションが使えるか見てみましょう。

その後、
 % gmake
 % gmake install
という通常の流れですが

Pg.pm(Perlインタフェース)のInstallは、rootの権限がいるので

 % gmake
 % su
   Password:*****
 $ gmake install

とするか、Pg.pmだけ、後からInstallします。
この場合、
 % ./configure --with-mb=EUC_JP

--with-perl は不要です。

 PostgreSQLのソースディレクトリ下の
 例)/usr/local/pgsq/下にある場合、
 % cd /usr/local/pgsql/postgresql-6.5.1/src/interfaces/perl5
 % perl Makefile.PL
 % gmake
 % gmake test
 % su
   Password:****
 $ gmake install

という手順になります。
