ここまでしなくてもと思いますが。

システムカタログに出力形式に関するデーターを持てる
といいんですが。ユーザー定義の表だと、rename などに
対応できなくて。

可能なら、カンマ編集、前ゼロ表示、非表示、画面表示幅
(INTEGER, FLOAT のみかな)などを、持てるといいのですが。
システムカタログを拡張するとか、rename などでトリガ
ーを起動するとかうまい方法はありませんか。

以前、ユーザー定義の別表に、表名+列名を Primary Key
として、にシステムカタログにない情報を追加して、
一見ものすごく便利になったのですが、度重なるＤＢの
変更(ALTER TABLE)に連動して、手作業で保守するのが大
変で、結局捨てました。

UNIFY 4.0 という昔のDBは、出力幅とか編集形式につい
て、列定義のときに一緒にデーターを入れるようになっ
てました。これは結構便利だった。(ANSI SQL の出来る
前の話です)。

Informix ONLINE version 4 では、money 型は、カンマ
編集で通貨記号付。その他は、すべて前ゼロ非表示カン
マなしです。(select * from で表示のとき)。

私の CSL WEB FORM では、NUMECIR,DECIMAL,FLOAT は、
全て3桁毎のカンマ付というおおざっぱな処理にしてます
が、これは PHP なので使う人がソースを直せばいいので。
