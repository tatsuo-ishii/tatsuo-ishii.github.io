>自己レスです。
>> データ型で"serial"を指定した場合で、insert 時のそのシリアルを取得したい
>> と思います。
>正しいかどうかわかりませんが、
>select nextval('hoge_seq');
>でシリアルを取得して、保存しておき
>insert 文で取得したシリアルを指定して行うようにするとできるみたいです。
>
>ただ、この方法だと、シリアルを取得後、処理が途中で異常終了すると、
>シリアルが飛び飛びになってしまいます。

シーラカンス本(たしか)では、
insert into t1 values (nextval('hoge_seq'), カラム2, カラム3・・・)
という風に、インサート文の中にnextvalを入れていたと思います。
これなら、処理が途中で・・・とういこともないですね。

(最初の質問が、インサートした後のシリアル値が欲しいのか、インサートする
時のシリアル値が欲しいのか解らなかったので、眺めてました)

インサートされたシリアル値を求めるのは、
select currval('hoge_seq');
です。これこそ、自分がインサートしてから誰かがインサートしてたら、値が
変わってしまうかもしれない値です(たぶん)。
(curval('hoge_seq')だったかもしれません。マニュアル見てください)
