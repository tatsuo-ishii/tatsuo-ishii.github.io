こんにちは、タケウチです。

環境：apache + perl5 + postgresql6.5.2

以前、山本様に回答いただいて仮想JOIN時のorder byの使用について
同じ記述を行うと以下のようなメッセージがでました。

> ---------------->error list
> pqReadData() -- backend closed the channel unexpectedly.
>         This probably means the backend terminated abnormally
>         before or while processing the request.
> We have lost the connection to the backend, so further processing is impossible.
>   Terminating.
> 			<------------------------

過去のＭＬを参考にさせていただきましたが、原因がよく分かりませんでした。
JOINにorder byをつけた時に、上記のエラーが出るのですが他の時（select や
insertなど）は表示されません。

参考まで、私が行いたいjoinのSQLを以下のように挙げます。

--------------------> sample sql 
select co.co_01,mco.mco_03,mco.mco_06,mco.mco_09,co.co_03,co.co_06,co.co_13 
>from co,mco where co.co_01 = mco.mco_02 union all 
select co.co_01,null as null_03,null as null_06,null as null_09,co.co_04,
co.co_06,co.co_13 from co where not exists 
(select * from mco where co.co_01 = mco.mco_02) ;
			<----------------------------------------
