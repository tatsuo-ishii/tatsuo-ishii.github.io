[pgsql-jp 15919] ふりがなから○行だけ取り出す方法

  • From: "Y. Fujimoto" <fujimoto@mxc.mesh.ne.jp>
  • Date: Fri, 07 Jul 2000 11:23:16 +0900
PostgreSQL-ML各位

こんにちは。藤本です。

LASER5 Linux 6.0 + PostgreSQL 6.5.3 + PHP 3.0.13 にてブラウ
ザから利用するＤＢシステムを作成しております。

テーブル内の“ふりがな”が入力されたフィールドから、五十音の
○行だけを取り出す良い方法を探しています。

例えば、

hoge0001=> \d ctm_tb
Table    = ctm_tb
+------------------------+--------------------------+-------+
|         Field          |          Type            | Length|
+------------------------+--------------------------+-------+
| ctm_code               | char() not null          |     8 |
| ctmcls_code            | char()                   |     1 |
| ctm_name               | varchar()                |    32 |
| ctm_name_k             | varchar()                |    40 |
| ctmgrp_code            | char()                   |     8 |
| ctm_zip                | char()                   |     7 |
| ctm_add1               | varchar()                |     8 |
| ctm_add2               | varchar()                |    60 |
.....
+------------------------+--------------------------+-------+

という顧客情報テーブルの「ctm_name_k」フィールドに、顧客名称
の“ふりがな”が平仮名で格納されていた場合、テーブルから、

　・か行の顧客を取り出したい

時は、

hoge0001=> select * from ctm_tb where ctm_name_k like 'か%'
hoge0001-> or ctm_name_k like 'き%'
hoge0001-> or ctm_name_k like 'く%'
hoge0001-> or ctm_name_k like 'け%'
hoge0001-> or ctm_name_k like 'こ%'
hoge0001-> or ctm_name_k like 'が%'
hoge0001-> or ctm_name_k like 'ぎ%'
.....

とするしか手が無いのでしょうか？。

それとも、○行検索用のフィールドを新たに設けて、そこに「あか
さたな」を１文字ずつ入れてインデックスを付けて検索する等の方
法の方が良いのでしょうか？。

どなたか良案をお持ちの方、ご教授願いいたします。
