>>>>> On Wed, 8 Apr 1998 14:46:27 +0900 (JST), Masaru Nishiyama <nishi@ns.wellcomm.co.jp> said:

>> gcc -I/usr/local/pgsql/include/ -lpq -o *****.cgi *****.c
>> と入力すると、キチンとコンパイルも終了し、*****.cgiという名前の実行ファイル
>> が作成されます。

>> しかし、Postgres v6.0(Linux 2.0.30)で同じように、

>> と入力すると、

> /usr/i486-linux/bin/ld: cannot open -lpq: No such file or directory
> のようなエラーメッセージが出てきてしまいます。
> このメッセージの解決策がわからないところです。

メッセージの通りでして、libpq.a が見つからないのです。

これはリンカの標準検索パスに libpq.a が含まれるディレクトリが
ないからです。
私は Linux は使った事がないので、Linux でリンカの標準検索パスに
追加する方法はわかりませんが、リンケージ時の指定で対応する事も
できます。libpq.a が /usr/local/pgsql/lib にあると仮定した場合、

 gcc -I/usr/local/pgsql/include/ --L/usr/local/pgsql/lib -lpq -o ***.cgi ***.c

とすればいいです。
