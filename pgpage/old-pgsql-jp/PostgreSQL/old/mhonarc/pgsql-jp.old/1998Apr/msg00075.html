>いま、以下のようなテーブルがあるとします。
>
>テーブル名: sukikirai
>  tabemono | konomi
>-----------+-----------
>  mikan    | suki
>  sukiyaki | kirai
>  cheeze   | suki
>
>ここに、'karubi','suki' をＣ言語でLIBPQを使って追加しようとした
>のです。
>マニュアルにあるサンプルプログラムを参考にしながら、
>
>res = PQexec(conn, "DECLARE myportal CURSOR FOR insert into \
>             sukikirai values ('karubi','suki')");
>
>と書いたのですが、その後 PQresultStatus(res)としてみると、以下のような
>エラーが出てしまいます。
>
>    PQresultStatus()--pointer to PQresult is null
>
>さらに、PQerrorMessage(conn)でエラーメッセージを見ると、
>
>    ERROR: parse error at or near "insert"
>
>と書いてありました。
>しかし、CURSOR を使用せずに
>
>res = PQexec(conn, "insert into sukikirai values ('karubi','suki')");
>
>としたら、あっさり追加されました。
>selectなどは、DECLARE CURSOR FORを使ってもうまく動いたのに、insert だけ
>そうもいかないのはなぜでしょうか。
>また、この２つの間には、どういった違いがあるのでしょうか。

PostgreSQL の declare 文は select にしか使わないのでは？psql の help 
で見るとこんな感じですので。

test=> \h declare
Command: declare
Description: set up a cursor
Syntax:
declare <cursorname> [binary] cursor for
	select [distinct]
	<expr1> [as <attr1>],...<exprN> [as <attrN>]
	[from <from_list>]
	[where <qual>]
	[group by <group_list>]
	[order by <attr1> [using <op1>],... <attrN> [using <opN>]]
	[union [all] select ...];
