	石井様、Gotoh Kazumasa 様。私が、Postgres をけなすつもりで、元記事
を書いたわけではないことはお分かり頂けたと思います。電子出席簿を PostgreSQL
で実現するには、心配なことがたくさんあって二の足を踏んでしまうという不安感
から記事を書いてしまったわけでした。どうもすいません。
	その他に、セキュリティも気になっています。6.2.1 までだと、IP アドレ
スでアクセス制御をしていたようにおもいますが、学生さんも同じネットワークを
使っていて、IP アドレスの捏造があるかもしれない。DB-username と password 認証
でデータベースアクセス制限をかけて、grant or revoke でテーブルアクセス制限をか
けなきゃいけないのかなとか考えています。
	気長にやってみます。私の目論見としては、PostgreSQL + ODBC + Access で
プロトタイプを作って、評判を聞いて、学校が必要とするならお金がつくでしょう。
そしたら、Oracle 8 か Informix Universal Server で業者さんに作ってもらおうか
なとか考えています。
	でも、システム作るとメンテナンスが回ってくるからな。ここは、一つぐっと
こらえて、システム開発せずに自分のやりたいことに没頭するか。悩ましいところです。
とにかく、ODBC + Access の勉強は、この年齢では辛い:-)。
