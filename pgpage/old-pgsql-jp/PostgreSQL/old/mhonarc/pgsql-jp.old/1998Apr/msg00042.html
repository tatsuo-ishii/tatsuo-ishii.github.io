何か目茶苦茶なバージョンナンバーを書かれていますが、^^;
PostgreSQL 6.1.1 から 6.2.1 に移行したい… という事でしょうか？

今から移行するのであれば、6.3.1 への方が良いのではないかとも
思いますが、まあこれは人それぞれの考え方による事ですね。

PostgreSQL は 6.1 系と 6.2系、そして 6.2系と 6.3系の間では
物理的なデータベースの互換性が無いとドキュメントに書いてあります。

このような場合、どのような DBMS を使っていても(ORACLE でも Sybase
でも Informix でも UNIFY2000 でも HiRDB でも…)やる事は同じでして、
移行プログラムを作成するという方法がひとつ。そしてもうひとつは、
一般ファイルとの export/import ユティリティを使う方法です。

元のデータベースからテーブル単位に、一般ファイル(普通のテキスト
ファイル)へデータを export します。次に新しい DMBS でそのデータを
import すればおしまい。
もちろん、新しい DBMS で create table などは行った後の話しですし、
場合によってはテキストファイルなどのフォーマット変換が必要な場合も
あります。バイナリデータなどはまた別の方法を使う事になるでしょう。

> ＤＢやユーザも新たに設定ＯＫです。又稼動確認のあと
> V6.1.1をクリアする場合は/usr/local/pgsql/postgresql-v6.1.1
> をクリアすれば良いのでしょうか。

必要なデータの待避を行った後は、旧システムはすべて削除して
よいという事になるでしょう。
しかし、「それがどこか」という話しは、旧システムのインストールを
行った笹尾さんにしかわからないので、ここで聞いても無駄です。
PostgreSQL がインストール時にどのようなディレクトリ構造を取るかの
一般的な話しはインストールドキュメントに書いてあります。
それ以上の事は、各システム管理者の掌握事項なので、他の人はわかり
ません。
