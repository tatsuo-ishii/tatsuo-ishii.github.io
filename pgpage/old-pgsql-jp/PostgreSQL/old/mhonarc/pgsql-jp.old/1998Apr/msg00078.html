テーブルを作成するSQL文の書式ですが、主キーとなる項目が、レコードの登録時に自動的に数をインクリメントするような指定をするにはどのようにすればよろしいでしょうか。
SQL Server では、IDENTITYというキーワードを使って、次のように書いておりました。

　　create table <tablename> (ID int IDENTITY PRIMARY KEY, ・・・ )

この指定により、ID という項目に値を指定しないで行を追加すると、自動的に前の行より１つ大きい数字が登録されます。
PostgreSQLでも primary key は指定できるようなのですが、自動的に前の行より大きい数が登録されるような指定はどのようにすればよろしいでしょうか。あるいは何を参照すれば書いてありますでしょうか。
