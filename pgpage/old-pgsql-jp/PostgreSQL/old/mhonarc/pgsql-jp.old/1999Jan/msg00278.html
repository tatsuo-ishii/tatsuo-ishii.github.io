> 今、調べているのは select * fromに対してのwhereの使用方法で、
> $query = "select * from $table_ where and number like '%$rtext%'";
> $rtn = pg_exec($con, $query);
> 
> とすると、
> Bad response to postgres95 query (ERROR: parser: parse error at or near
> "and" ) on line 35
> $rtn = pg_exec($con, $query);
> とか出てどうするのか理解していないために対処のしようが無く困っています。

$query = ... というのは確かに PHP/FI の命令ではあるのですが、"" の
中身に関しては、PHP/FI は $xxxx をその時の値に置き換えた後そのまま
PostgreSQL サーバに渡すだけで、何も関知しません。この置き換えられた
文字列部分は、他の方もコメントされているように、標準的なSQLという文
法です。

エラーエッセージの中に 'postgreXXX' という文字列が見えるので、これ
は PostgreSQL サーバが文句を言っている（SQLの文法エラー）であること
がわかります。

Web サーバ経由でデバッグするといろいろ大変です。psqlを使って直接
PostgreSQL に接続し、手で SQL 文を発行してやれば、少なくとも文法エ
ラーかどうかのチェックはできます。出力も確認できるので、これが通っ
てからそれを PHP/FI のスクリプト側に copy&paste して、定数の部分を
PHP/FI の変数などに置き換えてやればいいでしょう。

psqlでも必要最小限の SQL 文法のヘルプを表示できますし、拙作のpsqlj
を使えば日本語ヘルプを見ながら仕事ができるようになりますよ。psqlj
は以下のURLから取得できますが、サーバの再コンパイルが必要となります。
