[On Jan 19, "Yuki" <yukihiro@osaka.ilinx.co.jp> writes:]
>
>$query = "select * from $table_ where number like '%$rtext%'";
>とすると
>ERROR:  There is no operator '~~' for types 'int2' and 'unknown'
>        You will either have to retype this query using an explicit cast,
>        or you will have to define the operator using CREATE OPERATOR

おそらく「number」はint2型で定義されているのでしょう。
で、PostgreSQLが
	int2型とよう分からん型('%$rtext%')とで使う~~(like)演算子は知らん
と言ってる訳です。

>僕の使っているPostgreSQL6.3.1ではlikeは使えないのでしょうか？

psqlで「\do」してみれば分かるように、likeは
	左辺       右辺
	----------+----
	bpchar    |text
	name      |text
	text      |text
	varchar   |text
という組み合わせでしか使えません。
