石井です。

> 辻と申します。
> PostgreSQL mailing list in Japan
> PostgreSQL 6.4.2+Apache 1.3.3+PHP3.0.6 のインストール方法
> (DSOversion)
> に従ってバージョンアップを試みていますが、成功しておりません。
> 　
> Cannot load /usr/local/apache/libexec/libph3.so into server :File not found
> となります。
> （libph3.soは存在しますので、中身に不具合があるのかなともおもいます。）
> 
> ＰＨＰのＭａｋｅｆｉｌｅの修正は
> APXS_LDFLAGS = -L/usr/local/pgsql/lib -lpq -L/usr/local/lib -lgdbm -lm
					     ~~~~~~~~~~~~~~~~~~~~~~~こ
れは大丈夫なんでしょうか？ Linux だと、libgdm は /usr/lib にあるのだと
思うのですが。

APXS_LDFLAGS = -L/usr/local/pgsql/lib -lpq -L/usr/lib -lgdbm -lm

としたらどうでしょう？また、まさか libpq.so が /usr/local/pgsql/lib に
ないってことはないですよね？

これで改善されない場合は、問題を切り分けるために、apachectl を実行後、

ls -lu /usr/local/apache/libexec/libph3.so

としてアクセス日付をチェックし、本当にこの libph3.so を見に行っている
か調べてみましょう。

ちなみに、Apache の DSO はパス指定で直接 shared library を読み込むので、
LD_LIBRARY_PATH とか、ldconfig は関係ないと思います。
