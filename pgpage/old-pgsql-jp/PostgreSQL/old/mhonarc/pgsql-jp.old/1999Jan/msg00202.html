> ところで、この "Mincho:Helvetica-12" と言うのはどういう意味でしょうか？
詳しくは、Incompat80jp を見て頂くとして、簡単に説明します。

tk8.0 からは、フォントの指定として named font を使用するように
なっています。これにより、関連するウィジェットのフォント属性を
一斉に変えたりできます。

日本語パッチを当てた tk8.0jp は、latin フォントと漢字フォントを
組み合わせた named font (コンパウンドフォント)でフォント指定が
できます。

Mincho:Helvetica-12 は、tk8.0jp でデフォルトで定義されている
コンパウンドフォントの１つでウィジェットへのフォント指定を
省略した場合はこれが用いられます。これが実際にどのフォントを
使っているかは、font config コマンドで確認できます。

% font config Mincho:Helvetica-12
-foundry adobe:misc -family helvetica:fixed -size 10 -weight normal 
-slant roman -underline 0 -overstrike 0 -charset iso8859:jisx0208.1983 
-pointadjust 0.900000 -compound 
{{-adobe-helvetica-medium-r-normal--12-*-*-*-p-*-iso8859-1} 
{-misc-fixed-medium-r-normal--14-*-*-*-c-*-jisx0208.1983-0}}
% 

# 適当に改行してあります。

また、-font 指定で latin フォントが指定された場合はフェイル
セーフ機能として k14 フォントで 日本語が表示されます。

# 欧米で作成された Tcl/Tk スクリプトがそのまま使えるように
# しています。

ただし、ポストスクリプトを出力する場合は、フェイルセーフ
機能が無いので、コンパウンドフォントを指定するか、何も
指定しない必要があるようです。

# 今回の件も -font 指定を削除する対応がありましたね。:-)
