> なんとか、使えるようになっていますが、ＴＥＬＮＥＴ上でも日本語がつかえず、
> また、コンソール上ででも日本語の表示が出来ません。

 TELNET とおっしゃるのが、漢字表示機能をもっているリモートの
ターミナル（例えば MS-Win の telnet.exe や X 上の kterm）上で、
コンソールというのが FreeBSD の VGA のシステムコンソールであってますか?

 psql は、GNU readline というライブラリを組み込むようにコンパイルすると、
行編集ができるようになります。このライブラリを日本語化するパッチが
ありますので、これを使うと漢字を扱うことができるようになります。
readline の日本語化パッチの入手については、

 http://bw.com-net.co.jp/~mldb/pgsql_jp/frame.html

で検索してみたところ、[postgres95 1632] にありました。

 日本語パッチのあたった readline を適当なところへインストールします。
PostgreSQL をコンパイルするときに、configure が勝手に readline を探して
くれます。

  :
checking for main in -lreadline... yes
checking for write_history in -lreadline... yes
  :
checking for readline/readline.h... yes
checking for readline/history.h... yes
  :

と表示されれば OK ですが、no になったら --with-includes= と 
--with-libs= でヘッダとライブラリの位置を指定します。

 psql だけしか関係ないので、コンパイルしたときのディレクトリが保存して
あれば、bin/psql/psql を作りなおすだけで OK だと思います。

 私が使っている日本語化 readline は大昔の 1.1 のものなので、
他にも留意点があるかも知れません。

> 「/stand/sysinstall」コマンドは日本語の表示はしてますから、フォントはある
> はずですなのですが...。

 FreeBSD は詳しくないですけど、これって必要なフォントだけ中にかかえてた
ような気がします。システムのコンソールで漢字を入れるのはそんなに
簡単じゃないと思います。(kon + uum?)
