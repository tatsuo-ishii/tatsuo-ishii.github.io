> 現在、タプル長が最大 16KB ほどで、
> 15 万行程度のデータベースを作っています。

SQL で書くと以下のようになっています。
xmcdf の部分に 10KB 以上のラージオブジェクトが入ります。
これではファイル数が十数万個になり、実用に耐えません。

create table Disks (
       id char(8) primary key,		-- Disk ID, 64bit unique key
       xmcdf oid,			-- xmcd info file oid
       );

データはすべてテキストなので次のようにそのまま放り込むと、
8KB のタプル長制限にひっかかります。

create table Disks (
       id char(8) primary key,		-- Disk ID, 64bit unique key
       xmcdf text,			-- xmcd info text
       );

このような場合、ソースをいじってタプル長を拡張する以外に、
何か方法はないでしょうか。

今のところ、PostgreSQL-6.5.3 を使っています。

シーラカンス本にはそのうちにタプル長制限がなくなると書いてありますが、
PostgreSQL-7.0 では未実装のようですし…
