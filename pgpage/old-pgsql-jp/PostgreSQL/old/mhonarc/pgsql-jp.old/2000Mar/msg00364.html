> データベース作成許可のユーザ登録されたユーザが゛"createdb a" を実行すると
> 
> psql:error in loading shared libraries:libpq.so.2.0:cannot open shared object
>  file:No such file or directory
> createdb:database creation failed on a.
> 
> というメッセージが表示されます。

メッセージで示されている通りなんですが… libpq.so.2.0 が見つからない
のです。

libpq.so.2.0 が存在するディレクトリに LD_LIBRARY_PATH を通しましょう。
すなわち、そのユーザが PostgreSQL のプログラムを実行するための環境が
整備されていないのです。

他にも必要なものが未設定になっていないかどうか、チェックした方がいいです。
