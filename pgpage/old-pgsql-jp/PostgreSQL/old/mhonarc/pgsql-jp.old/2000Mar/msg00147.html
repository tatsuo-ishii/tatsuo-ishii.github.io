# 超亀レスです。

> 川嶋と申します。
>
> ACCESS2000でMoveNextを使用したら、”ODBC--呼び出しが失敗しました。”
> とエラーが帰ってきました。
> データ５１件目のフェッチで毎回このエラーが発生致します。
>
> 又、ODBCのエラーログにはこのようなメッセージが吐れていました。
>
> MSACCESS        ffc317c1:ffc4d70d
> EXIT  SQLFetch  with return code -1 (SQL_ERROR)
> 		HSTMT               0x0ace149c
>
> DIAG [S1010] [Microsoft][ODBC Driver Manager]
> 	関数シーケンス エラーです。(0)

  この直後のログが欲しかったです。というのも、PostgreSQLサーバやODBCドライバ
からのエラーメッセージは、この後に読み出されるからです。

> ODBC接続のタイムアウトでエラーになっているのでしょうか？
> 違うでしょうね、きっと。
> また、PostgreSQLのODBCプロパティにはタイムアウトの設定が
> あるのでしょうか？

  ODBC APIによるタイムアウトの設定は可能ですが、実際には無視されます。つま
り、機能的には「ない」ということです。具体的にはタイムアウトしません。
