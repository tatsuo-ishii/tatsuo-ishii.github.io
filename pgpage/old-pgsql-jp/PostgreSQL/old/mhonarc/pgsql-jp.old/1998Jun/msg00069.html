nobsunです。

> 多分 「su postgres」などとして行ったのではないかと思います。

まさに、そのとおりでした。

> initdb のデフォルトでは環境変数 USER を参照してます。 
> su で USER は変わらないようなので、

これを知りませんでした。

> --username=postgres などのオプションをつけるか、
> postgres でログインしなおす(;P) などすれば大丈夫だと思います。

アドバイスをいただいてから、はじめて、man initdb で
マニュアルを読んで(オイオイ>>私^^;;)、
環境変数 PGUSER をセットすればよいことを知り、

$ cd
$ vi .bashrc
$ cat .bashrc
PATH=$PATH:/usr/local/pgsql/bin
MANPATH=$MANPATH:/usr/local/pgsql/man
PGLIB=/usr/local/pgsql/lib
PGDATA=/usr/local/pgsql/data
PGUSER=postgres
export PATH MANPATH PGLIB PGDATA PGUSER
$ source .bashrc

としてから、initdb をすると、ちゃんと /usr/local/pgsql/data や
/usr/local/pgsql/data/base などができました。

後は、INSTALL文書の15から20までを無事終了することができました。
(21以降は明晩試す予定)

結局、私は、
「initdb のデフォルトでは環境変数 USER を参照している」
「su で USER は変わらない」
という基本の２点を知らず、su postgres として initdb を実行したので、
postgresでログインしなおすことを思いあたらず、ひたすら
？？？？？という状態でした。

アドバイスありがとうございました。
