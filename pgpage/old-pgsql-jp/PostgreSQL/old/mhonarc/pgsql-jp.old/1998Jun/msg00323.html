> 　関連してお尋ねします。片岡さんの「PostgreSQL ODBC Driver 日本語版」のＨＰでは、
> 注意として次のことが書かれています。
> 
> 　「インストール先のシステムにODBCドライバマネージャがない場合、オリジナルの
> 　　セットアッププログラムはPostgreSQL ODBC Driverのインストールと同時に
> 　　英語版のODBCドライバマネージャもインストールします。ご注意ください。」
> 
> 　ODBCドライバマネージャがあるかどうかは、どうすればわかるのでしょうか？
> すでに（例えばSQL Server用の）ODBCドライバを設定したことがあれば、
> 「ドライバマネージャ」があるということでよろしいのでしょうか。

  コントロールパネルに「32ビットODBC」というアプレットがあれば、ODBCド
ライバマネージャがあるということです。ちなみに「32bit ODBC」となってい
ると、これは英語版のODBCドライバマネージャということになります。

> 　あるいは、オリジナルのPostgreSQL ODBC Driver をインストールする際、
> 
>   1  Do not install Driver Manager
>   2  Install Driver Manager
>   3  Install Driver Manager(with version checking)
> 
> で“3”を選択すれば、懸念される問題は回避できると考えてよろしいのでしょうか？

  PostgreSQL ODBC Driverのインストール時にこの選択画面が現れたというこ
とは、すでにODBCドライバマネージャは存在します。とりあえず"1"を選択す
るのが安心だとおもいます。

# ODBCドライバマネージャのバージョンがあまり古いと、上記選択画面があら
# わせることなく最新の英語版がインストールされるようです。一度これに会
# い、日本語版に戻すのに苦労しました。
