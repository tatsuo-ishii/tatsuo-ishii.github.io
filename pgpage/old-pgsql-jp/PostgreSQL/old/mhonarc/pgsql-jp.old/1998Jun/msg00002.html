> 
> 私の所は Windows 系 しか使えない職員が多いのでできれば Windows 系
> のクライアントを使いたいと思うのですが、Access が使えないとなると、
> 何か Windows 系のもので PostgreSQL が使えるものをご存知ないでしょうか。

Access が使えないのではなく ODBC ドライバの完成度が
まだ低いレベルにあるものと思われ、そのために動作が不安定
であったり機能が限定されているのではと考えています。
したがって、Windows で PostgreSQL に接続するには ODBC
ドライバを経由するしかないのですから、何を使おうと同じこと
でしょう。

# Visual C++ でも簡単なプログラムを作ってみましたが制限が
# かなりきつくて結構苦労しました。

問題解決になるかは分かりませんが、Apache + PHP/FI など
を利用してプログラムを書き、Windows ではブラウザを使って
操作するという形態が一番手っ取り早いと思います。
