ざっと見た感じでは、シーラカンス本にはインデックス作成については
触れられていないようです。

ただ、シーラカンス本自身が PostgreSQL の全機能に触れているわけでは
なく、他の DBMS と比較して PostgreSQL の特徴的な部分の記述を中心に
書いているようにも見受けられますので、それはそれでもよいと私は思います。

> 　Accessからのクセというか、何となく使うソフトの名前を冠していない
> と不安ですが、SQL用の書籍を探せば良いのでしょうか？
> 　PostgreSQLとあう本があればいいのですが‥‥

SQL 自身は標準化された技術ですから、プロダクトの名前がついてないと
いけないという事は原則的にはありません。

後は、PostgreSQL が標準 SQL に対して、どのような変更や機能追加を
行っているのか、または何が実装されていないかを押さえておけばよいと
いうのが基本的な考え方だと思います。
このような点に関する説明は、PostgreSQL のマニュアルにも書いてあり
ますし、シーラカンス本でも触れられています。

また、参考書籍については、石井さんの Web ページの以下の URL にもいくつ
かがあげられています。

 http://www.sra.co.jp/people/t-ishii/PostgreSQL/doc-jp/index.html

このリストの中にもありますが、

  「SQL92 完全ガイド」 トッパン ISBN4-8101-8922-8
  「プログラマのための SQL」 トッパン ISBN4-8101-8949-X

は色々と参考になる本だと思います。
必ずしも、これらの本にある SQL 文がそのまま PostgreSQL で使えるわけ
ではありませんが、それは先に書いたように読み替えてゆけばよいのです。

# まぁ、中には PostgreSQL では実行不能なものもありますけど。(^^;

色々な本がありますから、ご自分で書店でパラパラめくってみて、自分に
合いそうなものを買ってくればよいと思います。

場合によっては、Oracle の本でもよいと思います。Oracle の本には
Oracle を使ってシステムを開発する場合のことなどに触れられている
ものも多いので色々な例があげらています。

あとは、「Oracle ではこのような事ができるみたいだが、PostgreSQL では
どのようにするのがろうか？」という考え方で調べてゆくという方法も
あるでしょう。

それからくどいですが、PostgreSQL 自身にもマニュアルはあるのですから、
疑問に感じることがあれば、まずはマニュアルを読んでみる… という事から
はじめてゆくのが適当であろうとも思います。
