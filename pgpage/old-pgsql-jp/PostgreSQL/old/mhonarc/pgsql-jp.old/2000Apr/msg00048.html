[pgsql-jp 13751] beta3が遅い order byとlimitの組み合わせ

  • From: 今川　晃<aimagawa@okayama-coop.or.jp>
  • Date: Wed, 5 Apr 2000 10:31:43 +0900
今までは6.5.3でした。そのときはレスポンスは良かったのですが
7.0beta3では極端に遅くなります。6.5.3は消したので比較できま
せんが、明らかに7.0beta3では遅くなってます。

データ　３５万件
フィールド name  varchar(16)　index張ってます

select * from t_hoge
        where name = 'ほげ　ほげお'
↓↓
7.0beta3  一瞬で検索します

select * from t_hoge
        where name = 'ほげ　ほげお'
        order by hoge_cd
↓↓
7.0beta3  一瞬で検索します

select * from t_hoge
        where name = 'ほげ　ほげお'
        limit 10
↓↓
7.0beta3  一瞬で検索します

select * from t_hoge
        where name = 'ほげ　ほげお'
        order by hoge_cd
        limit 10
↓↓
7.0beta3  遅すぎる（１０秒以上待ち）

"order by" と "limit"の組み合わせではindexを使用していない
のでは？と思うくらい遅いです。この遅さ何とかならないでしょ
うか？

6.5.3ではこんな事は無かったです。十分速かったです。
