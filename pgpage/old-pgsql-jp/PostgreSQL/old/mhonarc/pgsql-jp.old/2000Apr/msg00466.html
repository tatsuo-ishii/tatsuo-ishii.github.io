> どう「にっちもさっちもいかない」のかを説明しないと、話を聞いてる方に
> とっても「何が何だかわからない」としかならないですけど…

私も何が問題だか良く分からないのですが:-)

> > テーブルは
> > CREATE TABLE test(
> >    name            text,
> >    sankou          oid
> > );
> > 上記のような形で作成しています
> > oid型のところにIDが入るのは分かるのですが、本のとうりに考えれば
> > $lobj_fd = $conn->lo_open($lobjid, $mode)で開くのですよね？
> 
> それでよいように思いますが… テーブルから oid (ラージオブジェクト ID)
> を取得して、それを使って lo_open() の後に lo_read() すれば、ラージ
> オブジェクトに格納したデータが取得できるようです。

lo_open()するまえにトランザクションを開始していないのではないかと推測
します。

$conn->exec("begin");

とやってからlo_open()して下さい。で、lo_close()した後に

$conn->exec("end");

ですね。
