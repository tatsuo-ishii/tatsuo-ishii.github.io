こんにちは。

> > Redhat Linux + Postgresql 6.5.3 + Apache + Jserv 1.1の環境でPostgresへの
> > データ登録を行ってみたのですが、日本語が文字化けしてしまいます。
> > 
> > psqlからデータを登録したときは問題ありません。
> > 
> > おそらく、JavaはUnicodeでデータベースがEUCであることが原因だと思いますが、
> > どう対応したらよいのかわからず困っています。
> > 
> > どなたか、対応方法をお教えいただけませんか？
> 
> jserv.properties のどこかに、wrapper.bin.parameters　ってのがあって、そこで
> java コマンドの起動時の引数を指定できたと思います。
> 
> 
> そこで -Dfile.encoding=EUC-JP とすると、文字化けしません。

EUC-JP, EUC_JPとも試してみましたが、変化ありませんでした。

javaソースでパラメータと取得するときに、

 String aa = new String(req.getParameter("param1").getBytes("8859_1"),
"JISAutoDetect");

として、コード変換をかけて取得すると、postgresデータベースへは正しく保存
出来るようになりましたが、レスポンスの出力で、 

out.println(aa);

が今度は文字化けしてしまうようになりました。

resp.setContentType("text/html;charset=EUC_JP");
という指定はしています。

JDKは、Blackdown 1.1.7Bです。

なにかいい方法を知っている方がいたら教えてください。
