>>  上記の環境ならマルチバイト対応のはずですが、先のエラーメッセージからはその
>>ように取れません。動作しているPostgreSQLサーバを再度ご確認ください。

sea=> \l
datname   |datdba|encoding|datpath
----------+------+--------+----------
template1 |  1002|       1|template1
sea       |  1002|       1|sea         <<---※
regression|  1002|       1|regression
(3 rows)
　とやってみて、"1" となっているので一瞬大丈夫なのかなと思っ
たのですが、

　サーバのpsqlプロンプトから 以下のようにしても
    sea=> set CLIENT_ENCODING TO 'SJIS' ;
    NOTICE:  Unrecognized variable client_encoding
    SET VARIABLE

　と怒られてしまいます。ご指摘の通りサーバがマルチバイトになっ
ていなかったみたいです。


　どうすえば直るか分からなかったので、結局PostgreSQLの再イン
ストールをすることにしました。

　前回はやっていませんでしたが、今回は、regression test をし
て、
    euc_jp .. ok

となったのを確認しました。再度Windowsクライアントからlibpq経
由でアクセスしてみたところ、ちゃんと文字化けなくフィールドの
漢字が表示されました。


　クライアントから見ていて動作が違ったので、てっきりクライア
ント側の環境かと思ったのですが、サーバだけだと一見正常に動い
ているように思いました。こんなことあるのですね。

　どうもありがとうございました。
