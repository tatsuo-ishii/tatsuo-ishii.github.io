石井です。

> mydb=> SELECT COUNT( DISTINCT movie_title ) FROM movies_stars;
> ERROR:  parser: parse error at or near "distinct"
> 
> とエラーが出てしまいます。
> 書籍では、重複した行を削除した結果が表示されると説明しています。
> これは、書籍で説明しているSQL92とPostgerSQL6.4.2のSQLとの違いに
> よるものでしょうか。
> もし、違いによるものであれば、重複した部分を取り除いた行数をしり
> たい場合はどうすれば良いのでしょうか。

COUNT(DISTINCT ...)は PostgreSQL ではまだサポートされていません。今の
ところ、一時テーブルを使うしかないのではないでしょうか。

insert into temp_table select distinct movie_title from movie_stars;
select count(*) from temp_table;
drop table temp_table;

とか。
