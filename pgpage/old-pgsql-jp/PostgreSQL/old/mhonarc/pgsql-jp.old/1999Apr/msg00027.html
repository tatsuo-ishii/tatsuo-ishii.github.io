藤沼＠イイガです。

<19990401154335Q.matsu@nakamoto-sogo.co.jp> の、
   "[pgsql-jp 7823] "INSERT" from PHP" において、
   "midbook@hiroshima-cdas.or.jp (Tsukasa Matsubara)"さんは書きました：

> WEB Browser経由で<FORM>を利用してデータを入力し、PHPのスクリプトで
> PostgreSQLにデータを Insert します。
> 登録完了などのメッセージを表示します。
> 
> WEB Browser(私の場合は Netscape)のBackボタンで再度登録フォーム
> に戻ります。
> 
> そしてそのフォームを 再度submit します。
> これでは何回も同一フォームのデータが登録されることになるので
> これを回避するナイスな方法はないでしょうか。

自分は、
・データを格納するテーブルに ID を格納する列を定義し、Primary Key か
　Unique 制約を有効にしておく
・フォームをロードするときにユニークな ID を取得し（sequence などを
　使って）、Hidden でフォームに埋め込んでおく
という方法を使っています。

こうすれば、
・その ID で登録データがない（すなわち、最初の submit である）場合には
　INSERT は成功する
・その ID で登録データがある場合には、INSERT は失敗する
というようになり、お望みの結果が得られるのではないかと思います。
　
この方法の欠点は Sequence の値を使い果たしたらおしまいというところに
ありますが、通常の Sequence は 2^32 の値をとれるので、それはあるとしても
遠い将来の話だと思います。
# とか言っていると、2000問題の二の舞いになるかも・・・。
