> PostgreSQLの機能で、データベース本体をWWWサーバの動いている

はい、その通りですね。
# PostgreSQL の機能、というよりは、ネットワークを介したクライアント/サー
# バーなシステムであればなんでも一緒ですよね。

> にあったので、PerlのDBD::Pg,DBIを別のホストFreeBSDにおこうと
> しましたが、DBIはＯＫですが、DBDの方は、インストールが
> 分かりません。とにかく、別のホストから、PostgreSQLを
> アクセスしたいのですが、駄目です。

篠原さんがどういうことを行なって駄目だったのか全くわからないのでアドバ
イスのしようもないのですが、適当に想像して書きます。私が間違って理解し
てたら申しわけありません。

計算機 A: PostgreSQL が動いているデータベースサーバー
計算機 B: Web サーバー

という構成の時に、B から A へアクセスしたい、ということですよね? つま
り、PostgreSQL な見方では、A がサーバーで B がクライアント。

この場合、A には普通に PostgreSQL を入れて postmaster を動かしてあげ
ればいいですね。B のほうへは PostgreSQL のプロトコルを喋れるものをなに
か入れてあげないといけませんが、篠原さんの場合はそれが DBD::Pg だとい
うことですね? DBD::Pg は libpq が必要ですので、libpq を入れてあげましょ
う。面倒なら B でも普通に PostgreSQL を入れてあげればいいですけど、
PostgreSQL のソースを展開した後、configure をして、src/bin/psql で
make をすれば psql と libpq ができちゃいます。あとは DBD::Pg を入れる
だけ、ですね。

では、がんばってください。
