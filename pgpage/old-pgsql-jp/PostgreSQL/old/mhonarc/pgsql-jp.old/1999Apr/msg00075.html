i386-linux で 6.5snapshotをテストしました。

環境は
	AMD K6
	IBM 互換機
	Raw Hide 1.3.2
	Linux Kernel 2.2.4
	glibc-2.1.1
	gcc version egcs-2.91.66 19990314/Linux (egcs-1.1.2 release)
	flex version 2.5.4
	GNU Bison version 1.27
です。（あまり参考にならない環境かも・・・）

テストしたのは04/05版です。
	5922610 Apr  5 16:04 postgresql.snapshot.tar.gz


make は成功しました。

regression testでは
	int2, int4, float8, geometry, datetime, triggers, misc
がfailします。


int2
int4
	エラーメッセージの違いのようです。
		expected: Math result not representable
		results:  Numerical result out of range

float8
	expected ではエラーなのに results では計算できるようです。

geometry
	ほとんど丸め誤差のようですが、全然違うところもあるようです。
		expected: (-1.83690953073357e-16,3)
		results:  (1.53102359078377e-11,3)

datetime
triggers
	なんだかずいぶん結果が違うような・・・

misc
	Non-functional update, only first update is performed
	と出るようです。
