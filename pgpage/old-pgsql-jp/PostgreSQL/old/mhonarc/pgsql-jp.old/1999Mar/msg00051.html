まつむらです。

今回は、桐というマイナーなデータベースとODBCドライバの組合わせなので、
回答が寄せられるか心配なのですが、もし原因がお分かりの方がおられましたら
ご教示願います。

実行環境は次の通りです。

サーバー：
 PostgreSQL ver 6.4.2 with MB patch
 OS: FreeBSD 2.2.6-Rel

クライアント：
 
 OS:Windows95
 ODBC driver: psqlodbc-025.tar.gz + PSQLODBC_jp_bin_19981207.lzh
 Appli: 桐 ver.7 （管理工学研究所）

障害の状況は次の通りです。

桐で、ローカルの表（元々はPostgresサーバーからダウンロードしたもの）を
「書出し」「外部データベース」で書き込もうとすると、

	KD1672: ODBCエラー
	'test'データソースの最大項目数(0)の制限を越えました(10)

というエラーになってしまいます。(10)というのは、書込をしようとした
表の項目数です。どうも、最大項目数というのがうまく設定されていないよう
なのですが、ODBCドライバの設定項目には、最大項目数を設定するオプションなんて
無いんですよね。Driver Optionと Data Source optionで、ReadOnlyのチェックは
外しています。また、読み込みについては問題なくいけてます。
Accessでは問題なく書き込みも出来ているので、サーバー側のユーザー権限の
問題でも無さそうです。

参考までに、DriverとData Sourceのアドバンスオプションは、
Driver:
	KSQO,Use declare/Fetch, Unknown Size=maximum,
	Data Type Options=Text as LogVarChar,Bools as char
	Cache Size=100,Max VarChar=254,Max Long VarChar=4096
	SysTablePrefix="dd_;"
Data Source:
	Protocol=6.4
になっています。
