内山です。

Tatsuo Ishii さんは書きました:
>>仕事は主にperlでやって気が付いたら数万行にもおよんだんですが、
>># といっても見てくれの部分と横着で行数が増えてます
>>同じ事をOracleでやったら、開発にはもっと時間がかかったように思えます。
>
>ここは、DBMS として PostgreSQL を使うほうが、Oracle を使うよりも生産性
>が高いということでしょうか？
># だとしたら嬉しい:-)

	仕方がないので　禁句を発言しますか。

	1.PostgreSQL は　perl（ただ）へのインターフェースがあり　CGI との連携では
		確かに生産性が高い。
	2.しかし　ログファイルがないので　バックアップが完全ではない。
	3.JAVA JDBC 等のユーザインタフェースを含む言語のサポートは
		Oracle PostgreSQL で同等。
	4.すなわち　PostgreSQL は　入門・中級者が　web と　データベースを
		連携するのには最も良い。
	5.業務に使う。すなわち　メディアクラッシュ・システムダウン等
		まで考慮した場合、Oracle に軍配が上がる。
	6.web との連携を考えた場合でも　お金をかければ　Oracle は開発環境を
		提供している。

	結果として　お金があれば　Oracle を使うのが良い。同等の RDBMS 機能だけを
	勉強するには　PostgreSQL が多分一番良い。安価である。
