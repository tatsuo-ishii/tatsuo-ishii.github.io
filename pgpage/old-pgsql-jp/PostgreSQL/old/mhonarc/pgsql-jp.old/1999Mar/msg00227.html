[pgsql-jp 7474] psqlで日本語のSQLを読込む方法

  • From: 吉澤　潤 <yoshi@ic.daito.ac.jp>
  • Date: Fri, 12 Mar 1999 12:57:28 +0900

　初めまして、吉澤と申します。

　オープンカレッジの会員・講座・講師のデータを管理するために
Microsoft Access97だけを使っていたのですが、複数の端末からデ
ータの追加や修正を行うことになり、データベースサーバーとして
PostgreSQLを使わせていただくことになりました。

　サーバー：    FreeBSD 2.2.7R
                PostgreSQL 6.4.2 (patched-990308)
  クライアント：Windows95
                Microsoft Access97 
                　　　　　+ PostgreSQL ODBC Driver(6.40.00001)
  
  石井氏の「PostgreSQL完全攻略ガイド」を参考に構築している最
中です。
             
  メーリングリストへの投稿は初めてなので、ヘッダ、文言等々
おかしなところがありましたら、ご指摘ください。

　最初から質問で申し訳ないのですが、以下のようなSQL文が書かれ
たファイルを作成し、（日本語はEUCコード、インストール時の
--with-mbもEUC_JPです。）

---------  aaa.sql --------
CREATE TABLE T_DBEXT01 (
	あああ	int2,
	いいい	char(10)
);
---------------------------

　% psql exten < aaa.sql を実行したところ T_DBEXT01というテーブ
ルは作成されませんでした。% psql -n extenでコマンドラインから上
記のSQL文を入力するとテーブルが作成されました。また、フィールド
名の日本語をやめて、半角英文字にすると % psql exten < aaa.sql
でテーブルが作成されました。

　同じようにして psqlのプロンプトから \i aaa.sqlで読む込んでも
日本語のフィールド名ではテーブルは作成されませんでした。

　フィールド名が日本語のテーブルをファイルから読む込むことによ
って作成したいのですが、何か良い方法はありますでしょうか？フィ
ールド名を日本語にしないというのが一番良い解決方法なのですが、
すでにあるMicrosoft Access97のプログラムの中身(VBA)を有効流用し
たいと考えています。

　すでにMLに投稿されていた内容やマニュアルに書いてある内容でした
ら、申し訳ありません。記事番号や検索キーワード、マニュアルのポイ
ンタを教えていただけたらと思います。
