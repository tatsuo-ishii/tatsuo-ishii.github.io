とのことでしたので、
http://www.interwiz.koganei.tokyo.jp/software/PsqlODBC/index.html
からたどって、PostgreSQL ODBC Driver を導入しました。

で、やっぱり日本語の文字列が化けます。
テストの方法が悪いのかもしれませんので現象が出る手順を
以下に示します。

postgresql の日本語パッチの当たったファイル
(postgresql-6.4.2-patched-990217.tar.gz)を解凍して、
src ディレクトリにて ./configure --with-mb=EUC_JP で
コンフィグを実行します。
次に、make; make install でインストールまでやっちゃい
ます。
pg_hba.confを編集して、LAN内のマシンからアクセスできる
ようにします。
initdb して、
postmaster -S -i -D /usr/local/pgsql/data
でバックエンドを起動します。
適当なテーブルを作って、そこにEUCにてデータを登録します。
NT4.0SP4の環境に PostgreODBC 6.40.0001を導入します。
次に、
http://www.interwiz.koganei.tokyo.jp/software/PsqlODBC/index.html
の情報に従って、日本語版のODBCドライバを導入します。
NT側で、Microsoft Access 7.0 を使用して、このPostgreSQL
のテーブルをアタッチし、内容を確認します。
