ネットワークを DB を利用して管理する際に、MIB / SNMT という言葉が
必ず出てきますが、これは DB と通信プロトコルそのものなので...
これに PostgresSQL が使えないかと考えています。

IP Address 型がなぜほしいか？

私は、ネットワークルータの自動設定＆監視サーバを作ったりしているのですが、
ルータの IP を文字列で管理するとソートやグループ化に支障が出ます。
たとえば、192.168.1.1 のアドレスがどのマスクで切られているかによって、
192.168.0.0 のネットワークに含まれるのか、192.168.1.0 に含まれるのか
不明な事があります。
また、IPは単純な10進でないので、新しいルータのアドレスを決定する際に
安直にインクリメントできません。254 を超えてはいけないので...
192.168.254.254 の次は、192.169.1.1 になる、しかもマスクによって
許される場合とそうでない場合がある。
また、IP とマスク情報からネットワークアドレスやブロードキャストアドレス
を計算するのがメンドクサイ。

要は IP 型とそれに関連するファンクションが標準であるとかなり楽になるなぁ
と思っています。
