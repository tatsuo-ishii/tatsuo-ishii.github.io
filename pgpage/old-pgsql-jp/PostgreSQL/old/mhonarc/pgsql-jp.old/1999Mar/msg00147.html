Linux だと ipcs -m かな。手元の MkLinux では、postmaster が正常に動い
ているときはこんな感じです。

$ ipcs -m

------ Shared Memory Segments --------
key        shmid     owner     perms     bytes     nattch    status      
0x0052e2ca 0         t-ishii   700       49164     1                     
0x0052e2c1 1         t-ishii   600       831176    1                     
0x0052e2c7 2         t-ishii   600       96804     1        

このように、3 つ shared memory segment が取られています。この数は正常
なら変わらないはずです。ちなみに、nattch は、アクティブなバックエンド
の数 +1 になるはずです。
