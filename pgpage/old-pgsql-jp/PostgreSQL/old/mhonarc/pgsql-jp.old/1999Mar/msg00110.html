    はじめまして．みかげです．
    
    ウェブリンクのシステムを作っているのですが，リンク情報などの
    データを PostgreSQL で管理しています．
    
    輪を作るタイプのリンクシステムで，次のリンク，前のリンクへ
    進む機能はできたのですが，ランダムに適当な場所へジャンプ
    させる良い方法がわかりません．
    
    PostgreSQL 6.4.2 で，Pg.pm を使用して制作しています．
    
    リンクデータのテーブルは，
    
    ホームページの制作者の ID 番号（int，sequence だけど穴あり），
    輪の中での位置（int，重複しないランダムな値），
    リンクが有効かどうかのチェック（bool），
    ＵＲＬ（text），
    バナー画像ファイル名（text），
    
    というような項目からなっていて，有効なもののうち，
    どれか１つをランダムに取り出したいのです．
    
    SELECT * FROM link; してから，perl 側で１つ選べば
    実現はできるのですが，この方法だとデータが多くなるほど
    処理が重くなってしまいます．
    
    SQL のレベルで，ランダムに１つデータを取り出す方法は
    ないものでしょうか？
    良い方法をご存じの方いましたら，教えてください．
    
    
    それでは，よろしくお願いします．
