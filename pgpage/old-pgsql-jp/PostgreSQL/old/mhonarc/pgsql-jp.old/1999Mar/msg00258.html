[pgsql-jp 7505] 新旧郵便番号検索画面から　select.php3 が動きません。

  • Date: Mon, 15 Mar 1999 09:32:49 +0900
シーラカンス本を読みながら、PostgreSQL の勉強をしています。
ご多分に漏れず「新旧郵便番号検索システム」をインストールして
検索画面を表示するところまで行ったのですが ( p.194 図 4.3.6 )
検索ボタンを押すと

Method Not Allowed
The requested method POST is 
not allowed for the URL /postal/select.php3.

Apache/1.3.2 Server at host2.my_domain Port 80

と叱られてしまいました。どのような部分に問題があるのでしょうか。

１）PostgreSQL6.3.2/Plamo1.3 へ IE4/Win98 でアクセスしています。
    URL は http://host1.my_domain/ で検索画面が表示されます。
２）Win98 のユーザーは postgres ( host2.may_domain ) で、
    EnablePlainTextPassword=0x01 にしてあります（関係ない？）。
    取り敢えず、Samba は問題ありません。
３）馬鹿なことを、と思いつつ、
    grant select on postal     to postgres;
    grant select on prefecture to postgres;
    もやっておきました。もちろん変化無し。

なお Apache のインストールで、教本と違っていたのは
p.189 図 4.3.3 のエラーメッセージです。私の環境では
このようなエラーメッセージは出力されず、すんなりと
インストールできました。たぶん、Apache は
ちゃんと動いているんだと思います。

それから、もう一つの疑問です。
http://host1.my_domain/ で検索画面は表示されますが
php3 のサンプルスクリプトを見に行くと、不思議な
ことが起ります。
http://host1.my_domain/php-3.0.5/examples/data.php3
と URL を打ち込んでも、結果は File does not exist です。
ところが、そのファイルを表示する方法があります。

まず http://host1.my_domain/php-3.0.5/ のページ
を表示し、そこから、リンクをたどることによって
http://host1.my_domain/php-3.0.5/examples/data.php3
にたどり着くことができ、目標の画面を表示することができます。
私にはちょっと理解できません。どなたか教えて下さい。

もしかして IE 4.01 が シンボリックリンクをサポートして
いないということかな？ まさか???　それにしても
The requested method POST is 
not allowed for the URL /postal/select.php3.
となるのは理解に苦しみます。Help!
