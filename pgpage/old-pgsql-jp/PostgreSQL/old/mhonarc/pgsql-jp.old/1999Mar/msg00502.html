> Plamo版のcsvtopgをくわむらさんのとおりトライしてみたのですが、
> 自分の経験不足のため
> 以下の結果になってしまいました。
> 
> [503]postgres@server1: root> cd
> [504]postgres@server1: ~> ls
> COPYRIGHT     apps          data          include       register.txt
> HISTORY       bin           doc           kojin.csv
> README        contrib       examples      lib
> 
> [506]postgres@server1: ~> csvtopg kojin.csv | psql -c "copy boshunin from
> stdin"
> 
> ERROR:  COPY command failed.  Class boshunin does not exist

  デフォルトのデータベース(UNIXユーザと同じ名前のデータベース)に
boshuninというクラス(テーブルのこと)がないということでは。

> コマンドを分解して
> [507]postgres@server1: ~> csvtopg kojin.csv > kojinn.csv
> meibo=> copy boshunin from '/usr/local/pgsql/kojinn.csv' using delimiters
> ',';
> COPY
> meibo=> select*from boshunin;
> h_no|j_no|k_no|k_name|k_address|k_toroku|k_haigyo|k_ssk|k_nsan
> ----+----+----+------+---------+--------+--------+-----+------
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
>     |    |    |      |         |        |        |     |
> (10 rows)
> うまくコピーされていません。

  csvtopg で区切り(delimiters)はタブとなります。すなわち、"using
..." は不要となります。
