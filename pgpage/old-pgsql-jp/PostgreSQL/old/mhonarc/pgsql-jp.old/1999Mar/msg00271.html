　FreeBSD 2.2.7, PostgreSQL-6.4.2 を使用して、psql, ecpg の 
float8 のテストを行ってみました。

  Create Table, Insert, Select, Update の比較的単純な SQL文
でテストしましたが、OKのようです。

　どのような SQL文が通らないのでしょうか？


# 既知のことなのかもしれませんが...
  プリコンパイラー(ecpg)の倍精度定数の取り扱い方がおかしいよ
  うです。

　ホスト変数(begin - end decalre section間の変数)だけではなく、
　普通の変数に倍精度定数を代入するときまでチェックして parse
  errorとなります。

　double  dval;
  dval = 1.1234567890123456; <-- parse error at end of input

  但し、Insertに直接、倍精度定数を記述すると OK です。
　exe sql insert into t1 values( 1.1234567890123456 );
