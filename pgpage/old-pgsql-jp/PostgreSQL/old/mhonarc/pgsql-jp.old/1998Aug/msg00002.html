田中久太郎＠三鷹市です。
＃ひさしぶりの投稿です。

RDB でツリー構造を表現するにはどのようなテーブル
構成が、扱いやすいでしょうか。

具体的には会社の部や課ををテーブルで表したいのです。
すでに社員のテーブルはあって社員番号、名前、メール
アドレスなどの列があります。

これに、部の名前や課の名前やその親子関係を表すモノと
各部課に所属する社員の情報をリレーションしたいのです。

何かオススメのテーブル構成などありましたら、御教授くだ
さい。PostgreSQL でしか実現できないような ORDB なら
ではの構成でもいいです。

（例）
　　　　　　　　　　　　　　　　　　　　　　　┌─ヒラ中曽根
　　　　　　　　┌──Ａ部─────┬─ａ課───┤
　　　　　　　　│　　鈴木部長　　　│　竹下課長　├─ヒラ小渕
　本社──┬──┤　　佐藤副部長　　│　　　　　　│
　菅代表　│　　│　　(ｃ課課長兼務)│　　　　　　└─ヒラ橋本
　　　　　│　　│　　　　　　　　　├─ｂ課
　　　　　│　　│　　　　　　　　　│　田中課長
　　　　　│　　│　　　　　　　　　│
　　　　　│　　│　　　　　　　　　└─ｃ課
　　　　　│　　│　　　　　　　　　　　佐藤課長
　　　　　│　　│
　　　　　│　　└──Ｂ部──・・・
　　　　　│　　　　　福田部長
　　　　　│
　　　　　├─Ｋ事業部──┬─Ｃ部・・・
　　　　　│　岸事業部長　│
　　　　　│　　　　　　　│
　　　　　│　　　　　　　└─Ｄ部・・・
　　　　　│
　　　　　├─Ｊ事業部・・・
　　　　　│
