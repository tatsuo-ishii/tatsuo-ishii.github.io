ラージオブジェクトを使うと Postgres で管理できますが...

CREATE TABLE グラフィック (
	name	text,
	rem	text,
	data	oid   -- <- ここがグラフィックやサウンドのデータ
);

などとして、psql などから

INSERT INTO グラフィック
	VALUES ( '絵1', '1番目', lo_import('ファイルへのパス') );

とすればファイルのデータをラージオブジェクトとして
Postgres に保存できます。
# その他のラージオブジェクトについての情報は
# ドキュメントやソースを見ましょう?? ;p

こういったデータを編集したりすることがあるなら、
どこかにファイルとして保存しておいて
そのパスだけを管理した方が簡単かとも思います。
