石井です。

MLのメンバーの方に教えられて UNIX USER 9月号の「CGI で拓く Webの新世界〜
郵便番号データベースを作ろう」という記事を読んでみました。
が...これはちょっとひどすぎる:-<

一応郵政省の郵便番号データベースをPostgreSQLで扱う、というストーリーで、
今回はPostgreSQLのインストールと、psqlでの検索を扱っているのですが、
ちょっと目についただけでも：

o PostgreSQL 6.3.2 ではパッチを当てなくても日本語の正規表現検索が
  可能なのにそのことを知らない

o 「日本語正規表現を可能にするために」といいながら、MB=EUC_JP の
  指定や、configure --with-mb=EUC_JP をしていない

o initdb のステップが抜けている

o postmaster に database ディレクトリを与えないで起動している。
  かと言って環境変数の指定を行っているわけでも無い

o psql では日本語入力ができないと思っている

o index を付けずに検索を行い、「検索に数分もかかる」と文句を言っている

特に最後の点は、読者に PostgreSQL のパフォーマンスに対して著しい
誤解を招くもので、許しがたいものです。

記事を読むと、一応自分で動かしてはいるようですが、initdb のような重要な
ステップを抜かしているところを見ると、この人メモも取らないで記事を書いて
としか思えませんね。
