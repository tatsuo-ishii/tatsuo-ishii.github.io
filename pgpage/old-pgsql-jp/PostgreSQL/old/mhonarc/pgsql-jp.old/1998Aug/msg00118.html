On Wed, 12 Aug 1998 11:18:33 +0900, t-ishii@sra.co.jp wrote:
> >  Plamo Linux 用の PostgreSQL パッケージの最近のアップデート状況です。石井
> >さんによる bigtext を contrib に加え、お茶の水大学の市川さんによる ecpg
> >(PL/SQL のようなプリプロセッサ)の日本語パッチを当てました。
> 
> ちょっと質問ですが、パッチを当てた後の ecpg は SJIS 専用になるのですか？

いえ．ecpg と ecpgsj の2種類が作成されます．前者は SJIS 以外を考えて
作成されています．後者は sjis 専用です．

  o 内部で特殊文字の処理をする必要もあるのですが，そこは MB patch に
    入っていたルーチンを使っているので，一本化しています．内部処理は
    環境変数 PGCLIENTENCODING の影響を受けます．
  o lexer 部分だけは sjis 対応に一本化することがどうしても難しかったの
    で，分離しました．

結局2番目の理由で SJIS 用を分離して作成するようになっているのです．
普通に make すると，ecpg と ecpgsj の2種類が作成されます．
なお，コードを単純にしたかったので，ecpgsj での 環境変数
PGCLIENTENCODING を見て内部でのエスケープ文字処理が行われて
います．コンパイル時にもこの設定をしなくてはならないというのが
難点ですが，あまり複雑にしたくなかったので．
