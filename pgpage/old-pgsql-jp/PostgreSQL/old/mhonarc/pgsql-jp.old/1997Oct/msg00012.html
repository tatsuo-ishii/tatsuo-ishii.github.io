いつも、質問するばかりで恐縮なのですが、もう一つ質問させてください。

Postgresql には、継承という機能がありますが、それを使って
データベースを作っています。このとき、「ダウンキャスト」
のようなことが出来る方法はないでしょうか？

entry というテーブルがあって、そこから article と book という
テーブルを継承してつくりました。
entry には id, title という属性があります。
article にはそれに加えて、journal という属性があります。

今ここで、article に次のようなタプルがあるとします。
(id, title, journal) = (5, 'life cycle assessment', 'cleaner production')

book や entry にもタプルはあります。

このとき、次のようにすれば、entry を含めて、派生した全てのテーブル
を対象にして、特定の語句を含むタイトルを持ったタプルの属性 id を
検索可能です。

select id from entry* where title ~* 'assessment';

このとき、この id を持つタプルが実際はどの表(entry, article, book)
に属しているかを調べたりするような方法はありませんでしょうか？
もし、ご存知の方がいらしゃったら教えていただけませんか。

# 用いられているのかも、ちょっと興味があります。
