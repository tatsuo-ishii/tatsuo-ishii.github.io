>嘉山と申します

>matubara takahiro <matubara@green.ee.ous.ac.jp> さんは書きました：
>>松原です。
>>初歩的なことですが、データを一括してPostgreSQLに入れたいのですが、
>>copyコマンドの使い方がよく分かりません。データは、１行でそれぞれの
>>項目は半角スペース2個で区切られてます。
>
>デフォルトですと 半角スペース２個じゃなくて TAB １個で区切られている
>テキストファイルだとそのままcopy コマンドで使用できると思います。
>その他の区切り文字の場合は多分区切り文字を指定しなければならない
>でしょう。

いまやって見たら、「半角スペース２個」というのは区切り文字に指定できな
いようです。したがって、事前に TAB 1個かなんかに置き換えないといけない
ですね。後、ご存知とは思いますが、日本語がデータに混じっている場合は 
EUC にしておく必要がありますし、改行コードも Unix (\n)でないといけませ
ん。

>>copyコマンドはスーパーユーザーしか使えないのですか。
>
>そんなことはないです。

たぶんここで言っている「スーパーユーザ」というのは、PostgreSQL の「スー
パーユーザ」のことだと思うのですが、確かに直接ファイルからの copy は 
PostgreSQL のスーパーユーザしかできません。普通のユーザの場合は、標準
入力からの copy が使えます。

たとえば、copy.data というファイルに、SQL 文とタブ区切りのデータを入れ
ておき、こんな感じで実行します。

% cat /tmp/copy.data
copy table1 from stdin;
aaa	bbb
ccc	ddd
% psql -e test < /tmp/copy.data
QUERY: copy a from stdin;
