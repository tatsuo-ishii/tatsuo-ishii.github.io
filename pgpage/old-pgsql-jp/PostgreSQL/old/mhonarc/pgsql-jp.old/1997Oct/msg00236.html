Minohata> 今回は、SQLコマンドのFETCHとMOVEについて少々質問をさせていた
Minohata> だきます。
Minohata> $query = $conn->execute( $query_string ) || die
Minohata> "$Postgres::error\n"; while( @values = $query->fetchrow() )
Minohata> { print @values; print "\n"; }

これは、多分FETCHとMOVEの問題ではなくて、$conn->execute( $query_string
)の呼びだし方の問題のような気がいたします。

# $conn->executeは複文を受けつけるんですか？ > 詳しい方

一度、単文ごとに$conn->executeを複数回送出させてみて、反応を見るという
のはいかがでしょうか？

# psqlを見る限り、psql側で';'をセパレータに複文を単文に切って、そのつ
# どPQexec()を呼びだしているように見えます。

ところで、この問題の本質だと思うのですが、postgresql-6.2についてくる
ver1.4のPg.pmでは、$value = $result->getvalue($tup_num, $field_num)と
いうような感じで、ダイレクトにタップルとフィールドの位置を選択して結果
を得られるようになっています。

結果のN行目からM行を取り出したいというような時には、こちらの方がより楽
だと思います。
