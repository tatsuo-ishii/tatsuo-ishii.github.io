>元記事と全然違う話で、かつPostgreSQL の話でもないのですが、
>教えてください。
>
>Apache の1.2.4 とPHP/FI の組合わせで、Apache 側でユーザ認証をかけた場
>合、access_log に出力されるREMOTE_USER のフィールドがおかしくなる現象
>がありませんでしょうか？

REMOTE_USER というと、access_log で、hostname の次が - で、その次の
認証したユーザ名ですよね？それならこちらの環境では正しく記録されてます。

.htaccess は以下のようなもので、
-------------------------------------------
AuthName TEST
AuthType Basic
AuthUserFile /usr/mgr/t-ishii/public_html/postal/TESTPWD
<Limit GET PUT>
order deny,allow
allow from all
require user postal
</Limit>
-------------------------------------------
これで、"postal" と記録されます。TESTPWD は、普通に htpasswd 
で作ったものです。

>PHP/FI は2.0.b12、jp patch ありでもなしでも同じでした。

こちらのPHP/FI 2.0b12、jp patch ありです。うーん、違いは FreeBSD の
バージョンだけですかね？
