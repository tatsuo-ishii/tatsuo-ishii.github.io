->基本的にbbbにあってaaaにないデータはないという条件です。
->現在以下の様な方法でやっているのですが、他にないでしょうか？

	これは、aaa と bbb は和両立なリレーションなので、
集合の差を求める問題です。通常、リレーショナル完備なら、

	select * from aaa where not exists ( select * from bbb );

で、出来るはずです。この他に、リレーショナル完備である条件は

和集合演算ができる、
	select * from aaa union select * from bbb;

直積演算ができる、
	select aaa.*, bbb.* from aaa, bbb;

射影演算ができる、
	select adat from aaa;

選択演算ができる
	select * from aaa where adat > '0002';

です。これが一般的なリレーショナル完備の条件ですが、
この他に、共通集合演算、結合演算、商演算などもあり、全部で八つ
の演算のうち、独立な五つの演算が出来れば良いことになっています。
ちなみに上に上げた五つの演算のうち、postgres で出来ないのは、
和集合演算と差集合演算です。
	リレーショナル完備は、コッド博士の提唱したリレーショナル
データベースモデルの一要素ですが、postgres はオブジェクトリレーショ
ナルデータベースなので、別にリレーショナル完備にこだわることは
ないでしょう。
	私の考えでは、ORDBMS の出現によりデータベースの設計方法論が、
RDBMSの時と変わってくるのは当然とも言えます。そんな中で、リレーショ
ナル完備性にこだわることも無いでしょう。
