記憶をたよりにかいてるのでうろ覚えですが、たとえば C言語の場合は
PQexec(conn,"BEGIN"); と PQexec(conn,"END");の間で複数回の INSERT 文の実
行を行えばよいのではないかと思います。Perl でも同じことが恐らくできるで
しょう。
通常 RDBMS は一つのトランザクション内でのすべての変更が保証されていま
す。言いかえると取り消した場合は全ての更新が取り消されることになります。
乱暴な言い方をすると変更分をメモリー内に記録しておき END または COMMIT
で一気に write() する感じでしょうか。

奥村さんが"[pgsql-jp 3369] Re: input a lot of data"書かれている様に、ど
れだけがんばれるかは環境によります。
