市川氏に教えていただきました方法で　initdb  の Command  は見つかりまし
た。
initdb のあるフォルダーで実行して見ても、どうやっても
initdb:Command not found
になってしまいます。

configure　の設定がまずいのではとのご指摘でした。次のように設定していま
す。

% ./configure --with-mb=EUC_JP

また、コマンドサーチパスに追加した内容は次のとおりです。

setenv PATH "$PATH":/usr/local/pgsql/bin
setenv POSTGRES_HOME /usr/local/pgsql
setenv PGLIB $POSTGRES_HOME/lib
setenv PGDATA $POSTGRES_HOME/data
setenv MANPATH $POSTGRES_HOME/man
setenv LD_LIBRARY_PATH $PGLIB

csh　を使っていますので

source ~/.cshrc  を実行しています。

gmake gcc flex  はすべて最新のバージョンを使っています。
initdb  が実行できない以外はエラーメッセージらしきものは一切ありません。

FreeBSD-3.1-Release
Postgresql-6.4.2-patched-990507.tar.gz

に問題があるのではなくやはり私のやり方が間違っているのでしょうか、よろし
くお願いします。
