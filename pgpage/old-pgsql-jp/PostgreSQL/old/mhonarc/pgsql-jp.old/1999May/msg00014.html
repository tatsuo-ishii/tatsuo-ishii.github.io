石井です。

> [質問]
> データの追加を行うのに、numelic type serial を用いた table に対して
> \copy コマンドの使い方がよく分かりません。具体的には、
> database pen が、
> |     1 | papa        | little     |
> |     2 | mama        | big        |
> |     3 | uncle       | middle     |
> への追加を、データファイルから、copy する際、番号の部分を省略したいのです。
> file のデータが以下のような場合ではコピー出来ない、でいます。
> 
>  papa2(TAB)daddy
>  mama3(TAB)mammy
> 
> [希望]
>    今後も何回か、不定期に追加するので、定型フォーマットファイルから
>    番号抜きでコピーを行いたい。例えば 'NULL'(TAB)data(TAB)data など。

copy は NULL は \N です。つまり、

\NULL(TAB)data(TAB)data

となります。オンラインマニュアルに書いてあります。

ただご質問のケースは

create table pen ( num serial, name char(8), penname char(8) );

というテーブルなので、実際問題として num に copy で NULL を入れようと
するとエラーになってしまいますが、insert文なら

>  2. insert into pen (name,penname) values ('wate','daddy');
>   は結果として  4(or any num),wate,daddy を追加した事になる（ようだ？）
>   #番号を省略出来ている、ことになる?

ということで目的を達成できます。
