> PostgreSQL6.4.2のinstallで
> # gmake allとコンパイルを実行したところ
> ELF interpreter /usr/libexec/ld-elf.so.1 not found
> Abort trap
> というメッセージが表示されて、コンパイルが進みません。

 FreeBSD が 2.2 系列から 3.x 系列に移る時に binary が a.out という形式
から ELF に変更されました。その時によく「こんな error が発生する」と話
題になりました。

この問題は、3.x 系列の packages とかを 2.2.x 系列で使う時に良く発生し
ます。これは, 2.2.x 系列では ELF の binary を実行するのに必要な 
library と ld-elf.so.1 という命令が存在しないためです。

おそらく、
> freeBSD2.8.8
(これはたぶん FreeBSD-2.2.8 だと思うのですが)をお使いになられているの
に、

> gcc 2.7.2.1
> GNU make 3.77
> flex2.5.4
> Bison 1.25
> gdbm 1.7.3

の内のいずれかを、3.x 系列の packages で install されてしまったためで
しょう。

どこで error が起きているのかを調べてその binary を置き換えるのが一番
楽でしょう。

> と設定し、いよいよコンパイル
> % gmake all
> を実行すると、
> ELF interpreter /usr/libexec/ld-elf.so.1 not found
> Abort trap

ここで始めて error が出ているのでしたら、多分 gmake が 3.x 系列のもの
になっているのでしょう。これを置き換えましょう。
