こすげです。

At 20:26 +0900 99.5.6, Ken Kawahata wrote:
> 川畑と申します。
> 
> 一般アカウントでperlからpostgreSQLにアクセスする際、
> use Pg;
> で以下のようなエラーがでます。
> 
> Can't load '/usr/lib/perl5/site_perl/i586-linux/auto/Pg/Pg.so' for
> module Pg: File not found at
> /usr/lib/perl5/i586-linux/5.00404/DynaLoader.pm line 166.
> 
> pgsql_perl5は
> # make install
> まで行い、特に異常メッセージは出ておりません。
> postgresアカウントで同一スクリプトを実行した場合、上記エラーは出ずに、正
> 常に動作します。
> 該当ユーザーアカウントのpostgreSQLへの登録は行っており、psqlからデータベ
> ースにアクセスすることはできます。

　とりあえず、私のメモをばひとつ。たびたび出てくる質問と回答を
自分なりにまとめているだけで、検証したわけでもなんでもないので、
そこは御了解を。

■ Pg.so がない。
[症状]
Can't load '/usr/lib/perl5/site_perl/i586-linux/auto/Pg/Pg.so' for module Pg: File not found at /usr/lib/perl5/i586-linux/5.00404/DynaLoader.pm line 166.

[対策]
(1) /etc/ld.so.confに /usr/local/pgsql/libを追加
(2) apache の場合、httpd.conf に
SetEnv LD_LIBRARY_PATH /usr/local/pgsql/lib
(3) その他
