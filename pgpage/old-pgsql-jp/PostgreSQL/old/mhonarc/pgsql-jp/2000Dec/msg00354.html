[pgsql-jp 19232] ユーザーの認証が上手くいきません

  • From: Takamasa Suzuki <i992009d@mbox.media.nagoya-u.ac.jp>
  • Date: Mon, 25 Dec 2000 20:42:28 +0900
はじめまして、鈴木＠名大です。

Linux 2.2.6-15apmac
PostgreSQL 7.0.2
PHP 3.0.15-i18n-ja
Apache 1.3.14を使用しています。

以上のような環境でデータベースを構築して運用しています。
このデータベースは一般には公開できないのでパスワードを用いて
ユーザーの管理をしたいと考えています。

PHPのスクリプト中で
Header("WWW-authenticate: basic realm=\"database $dbname\"");
としてユーザー名とパスワードを取得し
pg_connect("dbname=$dbname user=$PHP_AUTH_USER password=$PHP_AUTH_PW");
としてデータベースへ接続しています。

またデータベースへのユーザー登録は
create user foo with password "foo";
として作成し
grant all on mytable to foo;
として許可をあたえました。

問題はここからなのですが、ブラウザー(IE)を立ち上げデータベースへの接続を
試みると認証用のダイアログボックスは出てくるのですが、正しく動作していま
せん。

具体的にはユーザー名にfooを入力しパスワードを空欄にしておくと、認証を通
過してしまいます(データが見られます)。間違ったパスワードや、ユーザー名を
変えると拒否されるので全く動作していないわけではないと思います。

このままでは困るので/usr/local/pgsql/data/pg_hba.confを書き換え
local	all					crypt
host	all	127.0.0.1	255.255.255.255	crypt
としました。こうすると正しいユーザー名とパスワードでないとwebから接続できな
くなりました。

でもこうするとコンソールからpsqlとしたときにもパスワードを
求められます。これを改良し、
1.Web経由のときはユーザー名とパスワードを正しく認証し、
2.コンソールからのpsqlの起動のときにはパスワードを聞いてこない、
ようにするにはどうしたらよいでしょうか。

皆さんのお力をお貸しください。わかりにくい点があればどうぞ指摘してください。
