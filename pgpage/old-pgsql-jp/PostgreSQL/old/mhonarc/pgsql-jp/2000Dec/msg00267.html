  はじめまして、小林と申します。

  現在PostgreSQLにて、プログラミングを行っております。そこで、
PostgreSQLの設定で、ruleがあるのですが、うまく機能してくれません。
  (私があまり理解していないというのもあるのですが…。)

  先に使っているOS・バージョンなどをお知らせ致します。
           OS:    FreeBSD 3.2-RELEASE(ちょっと古いですが)
   PostgreSQL:    PostgreSQL 6.5.3
   
  
  「困っていること」
     現在VIEW表を用いてA(ここではAと言う)という仮想表を作成し、検索・
   更新・削除を行いたいと思っています。
     PostgreSQLの場合、VIEW表で更新・削除を行う場合RULE設定が必要
   ということで次の指定を行う。(aは項目名)

        create rule A_upd as on update to A    \
                    do instead update A set a = new.a where a = current.a;
    
    ところが、ERROR発生！
    
        ERROR:  current: Table does not exist.
                     ^^^^^^^^^^^^^^^^^^^^^

    テーブルが無いよという意味でしょう。次にwhere句をとってテストしてみる。

        create rule A_upd as on update to A    \
                    do instead update A set a = new.a;     

    CREATEと表示され、どうやら出来たようです。

    次に変更がきくかテストしてみる。

        update A set a = 'test';

    またまた、ERROE発生！
       
        ERROR:  query rewritten 10 times, may contain cycles

    テストとしてやってみたのですが、どうもうまくいきません。どうしても
    VIEW表の変更・削除をしたいと思っているのですが、(今後の為にも)どの
    ように設定すればよろしいのでしょうか？
    大変素人的問題なのかもしれませんが、皆様からのご教授を承りたく思っ
    ております。

    よろしくお願い致します。
