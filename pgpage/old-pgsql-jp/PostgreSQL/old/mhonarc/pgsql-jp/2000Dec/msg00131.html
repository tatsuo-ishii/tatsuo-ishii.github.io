dbbash のメーリングリストでも流したのですが、bash 組み込みコマンドの
abbrev コマンドを作成しました。
abbrev コマンドを使うと、pgbash 上で SQL のキーワードを簡単に入力でき
るようになります。

ファイルの場所は、
   http://terry.nikkeibp.co.jp/~kubo/tmp/abbrev-bashbuiltin-0.0.1.tar.gz
です。
tar + gzip で展開して、

   cd abbrev-bashbuiltin
# system の bash のヘッダファイルがある場合
   ./configure --with-bash=/usr/include/bash
# または、
   ./configure --with-bash="bashのソースファイルのある場所"

   make

とすると、abbrev という共有オブジェクトができるので、
   enable -f "abbrevのある場所(必ずパス指定込みで)" abbrev unabbrev
とすれば abbrev コマンドが使えるようになります。

あとは、
   abbrev S='Select'
とすると、S + スペース が 'Select ' へと展開されます。

サンプルの設定を abbrev-bashbuiltin/sample の下に入れておきました。
abbrev-crazy の設定を適用して、
  "s * f a_table w a_column i 1,2,3)"
をタイプすると、
  "Select * from a_table where a_column in (1,2,3)"
へと展開されます。
# abbrev-crazy はやりすぎなんで、abbrev-normal の設定ぐらいが妥当かな？

abbrev-crary は pgbash の pgbashrc の設定とかぶっているので、使いたい
人は各自設定を修正してください。
修正した人は修正内容を ML に流してくれるとうれしい。
私は pgbash を使ってないので....。
