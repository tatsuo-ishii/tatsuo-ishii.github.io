Key＠京都市左京区です。

Postgres で少々大きめのシステムにおいて下記のようなエラーに遭遇しました。
現在はリトライでしのいでいますが、根本的な対策を行いたいところです。

気軽にパッチやバージョンの上げ下げができないので、
調査もろくにできず、にっちもさっちもいかない状況です。
yahoo, gooやgoogleではかからないし、こちらの ML のいわきりさんのアーカイブでも
"pg_noname" で検索してもかかってこないのでまれな状況なのかもしれませんが
もし情報をお持ちでしたらご教授いただければ幸いです。


= 環境
* PostgreSQL: ほぼ 7.0.3-patched-20001117
* OS: Linux 2.2.16 (smp,i686,MiracleのデフォルトSMPカーネル) + glibc 2.1.3
       (/proc/sys/{kernel,fs}の数値を少し大きくしたものがあります)
      FreeBSD 3.4-RELEASE (共有メモリの量を増やすなどしています)

= 現象
* cannot create pg_noname.32556.0: File exists
  などと怒られている
* 発生原因はいずれも select 文、 query には依存していないみたい

= 追跡
pg_noname をソース中で探すと postgresql-7.0.3/src/backend/catalog/heap.c に
使用されていましたが、まだちゃんと追跡することはできていません。
コメント中に

 * if heap_create is called with "" as the name, then heap_create will create
 * a temporary name "pg_noname.$PID.$SEQUENCE" for the relation

などとかかれているので、衝突しなさそうなものが衝突している
ということはアルゴリズムのバグなのか などと考えていますが…
