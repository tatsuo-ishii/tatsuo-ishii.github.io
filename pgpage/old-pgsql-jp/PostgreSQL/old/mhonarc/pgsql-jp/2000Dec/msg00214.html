日本語対応に修正したJDBCドライバを使っています．
Postgresqlのバージョンは７．０．２です
クライアントはWindows９５です
データベースのエンコーディング設定はEUC-JPです
表示にSwingを使ってます．

結果に日本語を返すCASE文が期待した結果を返しません．
select name,
 case substr(phone_number,1,2)
  when '01' then '北海道'
  when '03' then '東京'
  when '04' then '関東' end
 from address_book;
電話番号の部分だけ文字化けが起きたように表示されます。
# 特定の記号が表示されるので文字化けではないようですが・・・
# このときnameは日本語がちゃんと表示されています。

insert文は期待したとおりEUC-JPで記録されます．

select * from address_book where name ~ "^村上";
のようなSQL文に日本語を含むものも動いています．

もちろんpsqlでは問題なく表示できるのですが、
上に書いた私の環境では日本語が表示できないのです．

ResultSet rs = st.executeQuery(sql);
rs.getString(1);
や
ResultSet rs = st.executeQuery(sql);
new String(rs.getBytes(1), "EUC-JP");
を試しましたが改善されませんでした．

問題はどこにあると考えられるでしょうか
ご教示いただければ幸いです。
