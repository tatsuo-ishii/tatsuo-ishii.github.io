# 英語耳をもっと鍛えなきゃ、と痛感。通訳なしだと30%ほどしかわからぬ。

　閑話休題。先日の席上で聞きそびれてしまった質問なんですが、どなたか情報
をご存知でしたらお聞かせ願えませんか?

　Bruceさんが、TODO/Performanceのときに、「とある業者のベンチマークテス
トで、他DBMSに比べて処理性能は落ちるだろうと思ってたらびっくり、とても優
秀じゃないか。(うろ覚え)」のような話をされていましたが・・・

・このテスト結果は、どこかで参照できますか?
・各Productsのプラットフォームは共通だったんでしょうか? (・・・そんなわ
けあるまい、MSSQL はWindows NT専用のハズだし)
・そうでなかったとしたら、PostgreSQLはどのプラットフォームを用いたので
しょうか?

# たぶん Linux のような気はする・・・


　今回、業務でPostgreSQL-7.0.2をBSDI 4.1上にて利用しました。このとき、
チューニングのためのカーネルパラメータ調整にとても苦労してしまいました。

　とりあえず、SHMMAXPGSを、8192を超えて構成したら、カーネルが刺さるとこ
ろまでは確認しました。FAQ-BSDIの著者さんに報告しなくちゃ。でも、
SHMMAXPGS 8192においても、-N 512まで行けることは確認しました。


　余談。で、「Shared memoryなんて嫌いだっ、なんでmmap(2)じゃないんだろ
う、実験コードとか改造の痕跡とか残ってないかな?」とか独り言吐きながら
ソース覗いて遊んでたんですが、TODOに、「Sysv shared memoryではなくmmapを
使うようにする」って書いてありますね。まだ誰も手をつけてないのでしょう
か、また、mmapではない歴史的な理由があるのでしょうか?
