高橋です。

>> このときに検索結果を一つづつ表示させたいと思っています。
>> 具体的には、xxxx を表示して、「次へ」ボタンを押すと「ｙｙｙｙ」
>> を表示したいのです
>> # goo の [次の10件] のように。

私も WEB 上でこのような検索をしたいと考えています。

> 　ふつうのCGIだと、PostgreSQLのセッション情報は失われてしまいますよね?
> CGIでなければ、カーソルで取り出していけばいいような気はしますが・・・
> 
> ・データの入れ替わりはほとんどない
> ・DBサーバはそれほど忙しくない
> 
> のであれば、OFFSET, LIMITを用いてもバチはあたらないとおもいます。

現在は OFFSET, LIMIT を用いているのですが、検索結果が数百件から数千件も
ある場合にはとても時間がかかる結果になっています。
OFFSET, LIMIT を用いて検索しても、用いない場合の大量の検索結果を
バッファに入れた後で、ソートしてから OFFSET, LIMIT 条件にあう分だけを
検索結果として返しているようなので、１０件しか必要でないのに、
全件検索と同じだけの時間がかかってしまいます。

> 　あとは、結果表示CGIにて、sidおよびどこまで表示したかを持ち回れば、望み
> の動作をさせられるんじゃないでしょうか。

そうですね。
解決策は OFFSET, LIMIT を使わない時点での検索結果を減らすことなので、
sid を検索条件に組み入れれば効果的ですね。
＃１ページ目の検索は全件検索と同じになってしまいますが・・・

それに、検索結果を sid の順でソートする場合には有効だけど、
任意のキーでソートしたい場合にはどうすればいいのだろうか？
何か効率的な検索方法があれば教えてください。
