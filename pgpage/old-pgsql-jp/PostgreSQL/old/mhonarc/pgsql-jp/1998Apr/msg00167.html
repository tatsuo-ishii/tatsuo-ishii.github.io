>> >  さて、さっそくパッチを当てたのですが、Linux-ELF で libpgtcl.so を作
>> >るためのパッチ linux_elf.patch-980421 を当てていたので、src/configure 
>> >へのパッチは当たっていません。regression テストをすると:
>> >
>> >! ERROR:  function octet_length(text) does not exist
>> >
>> >となるのですが？
>> 
>> initdb はし直しましたか？
>
>  それはしてないのです。ということは、DB をダンプ／リストアする必要がある
>のですか？

基本的にはそうです。octet_length() の追加により、システムカタログの 
pg_proc あたりが変更されてます。

ただ、単に regression を確認するだけで良いなら、initdb -t で template1 
データベースのみ更新してくれますので、これでOKです(ただし、template1 
に登録したユーザ定義関数などはなくなってしまうので、ご注意！)。
regression 中に、regression データベースが drop->新規createされ、この
時に更新されたシステムカタログがコピーされるからです。

既存のデータベースでも octet_length()を使いたい、ということになるとダ
ンプ／リストアしかないですね。

もちろん、octet_length()を使わないのであれば、何もする必要無いですが。
