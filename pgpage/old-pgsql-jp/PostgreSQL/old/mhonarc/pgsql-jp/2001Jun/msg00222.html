飯田と申します。ODBCについて教えて頂きたい事があります。

PsqlODBCドライバをWindows98にインストールしMS-AccessからPostgreSQLに接続して
テーブルの参照や更新はできたのですが、PostgreSQLから他のデータベースにODBC経由
で接続するのはどうしたら良いのでしょうか？
PostgreSQL7.1のプログラマーズガイドによると、ODBCドライバを使用するシステムには、
ドライバマネージャが存在しなければならない。との事だったのですが、Linuxのドライバ
マネージャとはiODBCがこれに当たるのでしょうか？さらにODBCドライバはpsqlODBCの様に
接続先データベースによって各種リリースされているのでしょうか？
この場合の接続先の情報(DSN)はodbc.iniというファイルに記述するのでしょうか？
odbcinst.iniというファイルはすでに私のシステムに存在しているのですが、これは何の
役目を果たすのでしょうか？

また、iODBCのソースをダウンロードしconfigureしようとしたところうまくいきません。
以下のようなメッセージが出ます。
checking for GTK - version >= 1.2.3...no
* * * The test program faild to compile or link. See the file config.log for the
* * * exact error that occured. This usually means GTK was incorrectly installed
* * * or that you have moved GTK since it was installed. In the latter case, you
* * * may want to edit the gtk-script: /usr/bin/gtk-config

PowerBookG3にLinuxPPC2000という環境にPostgresql-7.1.1をインストール。
configureのオプションには--enable-multibyte=EUC_JP --enable-odbcを指定し、
その後 make, make installしています。

以上、よろしくお願い致します。
