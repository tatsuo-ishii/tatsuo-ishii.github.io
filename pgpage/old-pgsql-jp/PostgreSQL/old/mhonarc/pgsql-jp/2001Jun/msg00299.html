この件は度々でていますね。対処方法も簡単で、PostgreSQLをDBに使う場合
PHP3の国際化版ではコード変換を行わず、
PostgreSQLのクライアントエンコーディングを好みで選択してください。
G3対応のEUC_JPではJISX0212の文字も安心して利用できます。

PHP3国際化版のコード変換はG3未対応なのに対して、PostgreSQLのコード
変換はG3対応対応という仕様の違いが原因です。
双方のマニュアルやコメントをよくお読みください。
仕様の事はPostgreSQL Adminitorator Guide(邦訳 PostgreSQL 管理者用 ガイド)
Multibyte の項か 邦文 README.mb.jp をお読みください。
クライアントエンコーディングの設定はPostgreSQL Reference Manual の
SET CLIENT_ENCODING の項をお読みください。
