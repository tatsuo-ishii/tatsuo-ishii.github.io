[pgsql-jp 21814] delete from dbname でエラーになるサンプル

  • From: yaemon@alles.or.jp ( T.P.S. 中川)
  • Date: Thu, 14 Jun 2001 11:51:17 +0900 (JST)
  パイプ喫いの中川です。

  まず、はじめに断っておきます。
この件に関して私は困っていません。運用で逃げることができます。

  ただ再現率100% のエラーが出るサンプルコードがあって、
再現条件をもっと絞り込んでバグ報告すれば PostgreSQL に
貢献できそうだと思っているのですが、諸般の事情でそこまで手が
回らないで、そのサンプルコードを試してみたい方
( 再現条件を絞り込んでみようという方、あるいは私と違う環境で
  追試してみたい方 ) がいらっしゃればコードを直接にお送りします、
という報告です。



  まず、当方の環境は FreeBSD 4.2-RELEASE  上の PostgreSQL 7.1.1 に 
pl_exec.patch.gz (5/15にgetしたもの) を当てた状態のデータベースで、
Perl 5.005_03 + DBI-1.18 + DBD-Pg-1.00 を使用しています。


Perl Script から insert したデータを psql から delete しようと
すると、下記のようなエラーが出ます。

-----
yaemon=> delete from jobMaster;
ERROR:  Unable to identify an operator '=' for types 'int4' and 'text'
        You will have to retype this query using an explicit cast
-----

このテーブル、jobMaster の primary key を text に定義してあると
再現率 100% でエラーが出て、当方の仕事の運用としては、現在は
char(8) で充分なので データベース定義、Perl Script の双方を
char(8) に変更することでしのいでいます。

  再現してみたい方にお送りするのは、テーブル定義ファイル、(24行)
Perl スクリプトファイル(コメント込み391行)、あと 10行のサンプル
データということになります。


  もしも上記のエラーが既知のものでしたら、どうもすみません。
