"大森" == Yasuhide OMORI / 大森保英 <omori@m-t.com> wrote:

大森> FSSTND/FHS の枠組に基づいたファイル配置と、PostgreSQL の参考書籍に書か
大森> れているファイル配置がズレているだけの気がします。

PostgreSQL の解説はPostgreSQL のデフォルトのファイル配置を前提として書
かれていてしかるべきで、それを「FSSTND や FHS に合致していない」と責め
るのはどうかと思います。

# それが不満だという人は、自分でFSSTND/FHS に沿った本を執筆なされば良
# いでしょう。ここのML を見る限り、それなりに需要はあると思いますが。

問題は「FSSTND/FHS とファイル配置がずれている」という点ではなく、「たっ
たそれだけ(単にpathを読み替えるだけ)の問題」を解決できない人に、
PostgreSQL のようなサーバーアプリケーションを「簡単に利用できる」と錯
覚させてしまった点にこそある、と私は最初から主張しているつもりです。

大森> rpm -ql postgresql
大森> などで見えるので、「隠蔽」と言われる意味が分かりません。解説よろしくお
大森> 願いします。

rpm が隠蔽するものは「パッケージの中身」ではなく、「インストールという
作業そのもの」と私は前発言で言っています。

「見ることができる」と「見える」は違うものです。ましてや「判る」「理解
する」は、全く別のものでしょう。

rpm -ivh hoge
rpm -Uvh hoge

# 実は私もこれくらいしかrpm のコマンドは覚えてませんが(笑)

というプロセスで、実際に何が行われているかきちんと理解できている人は、
そもそもインストールやセットアップなどでトラブらないと思います。

大森> PostgreSQL の rpm package が入っているか否か調べずにオロオロするのは、
大森> そもそも PostgreSQL 以前の問題でしょうし…。

全くその通り。しかしそういう問題でつまづく人を増やした土壌は、結果的に
は rpm というシステムにある、と私は言いたい訳なんですが。

#「Linux が普及したから」という見方もあるでしょうけど、その一因はやは
# り rpm というパッケージ管理システムにあると思うので。

「rpm で簡単インストール」って、どのRedHat 系ディストリビューションの
解説本見ても書いてありますけど、いざバージョン違いやパッケージングの違
い、ましてや「自分で /usr/local 以下にインストールした」ことが原因でコ
ンフリクトが起こった場合のトラブルシュートについてきちんと解説してある
物は少ないと思います。

"rpm xxx hoge" で解決できることは、「ディストリビューションが提供する
物、あるいはそのディストリビューションに対応してrpm でパッケージングさ
れた物だけで構成される範囲内」だけであって、万能ではない。だから、OS 
やコマンド、それにアプリケーションに対する理解を深めるよう精進しましょ
うね、というごくごく当たり前のことを私は主張したいだけなんですけど。

「なんかよく判んないけど、書いてある通りに動かないのでOS から全部再イ
ンストールした」という、どこかのOS のような間違った手法が定着するのだ
けは勘弁してほしいですね。
