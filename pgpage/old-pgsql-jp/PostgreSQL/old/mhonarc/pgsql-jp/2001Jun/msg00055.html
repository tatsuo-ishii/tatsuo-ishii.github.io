[pgsql-jp 21676] 異なるversion のPostgreSQL の使用

  • From: Naofumi Kondoh <nkon@shonan.ne.jp>

ソフト工房の近藤です。

異なる version の PostgreSQL を使用する方法について、
皆様の行なっている方法をお教え頂きたく。

私が考えているのは、次のような方法で、どれにしようか
迷っているところです。

(A) 同時使用しない場合。(同一 port 番号使用)。

    指定 version 起動時に、$PATH と $LD_LIBRARY_PATH
    の指す実体をシンボリックリンクで使用したいversion
    のパスに張替える。
    例えば、 
        rm /usr/local/pg
        ln -s /usr/local/pgsql.7.1    /usr/local/pg
    $PATH は、 $PATH:/usr/local/pg/bin  のままにしておく。

    この方式の利点は、ポート番号指定のないアプリケーション
    を変更せずにそのまま動かせることです。
   
    １人専用の開発機の場合は、この方が便利かと。


(B) 異なる version を同時使用する場合。
    version ごとに異なる port 番号で postmaster を起動。

(B.a) source コマンドで、$PATH,$LD_LIBRARY_PATH などを
      使用したい version 用に切り替る。

(B.b) 上の方法は、自分がどのversionを使っているのか混乱
      しそうなので、version ごとに異なる user を作って、
      su - pg653  などとして、切り替る。

一長一短で迷ってます。経験者のノウハウや失敗談などお聞か
せいただけないでしょうか。
