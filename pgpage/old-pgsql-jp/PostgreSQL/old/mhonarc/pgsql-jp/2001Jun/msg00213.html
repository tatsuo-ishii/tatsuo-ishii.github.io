山口＠初心者.埼玉です

サーバー：OS/LINUX(SlackWare)
　　　　　　＋Apache＋PostgreSQL7.02＋PHP4
ローカルPC：OS/windows98＋Office2000
（IBM Think Pad i-1400）

AccessからPostgreSQLへODBCでデータを移行しました。
テーブルは全部で10個あり、
テーブルごとに内容を参照できるwebページを作りたいのです。
試しに市販の参考書にあるスクリプトを実行してみようと思い、
-----------------------------------------------
<?php
$conn = pg_pconnect ("dbname=databook");
if (!$conn) {
    echo "接続できませんでした\n";
  exit;
}

$result = pg_exec ($conn, "select * from data1");
if (!$result) {
    echo "pg_exec()エラーが発生しました\n";
    exit;
}
$arr = pg_fetch_array ($result, 0);
echo $arr[0] . "<- array\n";
$arr = pg_fetch_array ($result, 1);
echo $arr["perm"] . "<- array\n";
?>
-----------------------------------------------
として、webからアクセスしてみたところ
Warning: PostgreSQL query failed: ERROR: data1: Permission denied. 
というエラーメッセージが出ました。
Permission denied　は認証を拒否されたという意味だと思うのですが、
足りないプロセスがあるということなのでしょうか？
教えていただけると助かります。
よろしくお願いします。
