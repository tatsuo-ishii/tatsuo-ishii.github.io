。。。略。。。。

> # insert into test_tbl (sid, name) values ( 104, 'ddd' );
> ERROR:  Cannot insert a duplicate key into unique index なんちゃら
> 
> このinsert文を、
> 「もしもunique違反な値をinsertしそうになったら
>   何もしない（エラーを出さない）」
> という風になるように書きたいのですが、

私ならプログラム言語を使って、同一トランザクション
内で、SELECT して見つかったら、エラーにする。
見つからなければ INSERT するというふうに書きます。

こういうことをする目的は、ユーザーにわかりやすい
エラー表示をするためでしょう。例えば、このメール
アドレスは登録済ですとか。

SQL だけで、エラーメッセージを出さないようにした
いという目的はなんでしょうか。テストなど特殊な
処理でエラーメッセージが山のようにでるので、重要
なエラーを見逃さないようにしたいというなら、ファイル
に落しながら、ERROR:  Cannot insert a duplicate key
の文字列を grep -v ではじいてしまうとか。
# 外したかな。

。。。略。。。
