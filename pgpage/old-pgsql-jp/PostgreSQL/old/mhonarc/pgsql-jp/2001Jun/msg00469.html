>insert into test (testdata)
>  values ('abcde\nabcde');
>
>を実行しますと、select した際に PostgreSQL 7.1.2 では
>
> testdata
>---------
> abcde
>abcde
>
>という結果になります。
>
>しかし、Oracle 8.1.6 では
>
>TEST
>------------------------------------------------------------------
>abcde\nabcde
>
>となります。
>
>PostgreSQL でも Oracle と同じ挙動をするようにすることは
>できるでしょうか?
>SQL92 の仕様も探していますが、
>本来正しい動作としてはどちらになるのでしょうか。

付属ドキュメントsql-syntax.htmlの"1.1.2.1. String Constants"を見ると，
Cのように'\'をエスケープキャラクタとして用いるのは，PostgreSQLの
独自仕様のようです．

もし，"\n"を'\'と'n'のままでINSERTさせるということなら，アプリ側で

insert into test (testdata)
  values ('abcde\\nabcde');

のように'\'を二重化させるということになるのだと思います．
