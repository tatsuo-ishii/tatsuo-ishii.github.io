PostgreSQLをSSLをサポートした状態で起動させたいのですが、起動できません。

[当方の環境]
RedHat Linux 6.2J
PostgreSQL Version 7.1.1-patched-20010508
OpenSSL 0.9.6a


上記環境で、添付ドキュメントのとおりに "--with-openssl=/usr/local/ssl"オプション
を指定してコンパイル、無事インストールしました。エラー等は出ませんでした。

OpenSSLを使って server.key及びserver.csrを作成、$PGDATA/に両ファイルを配置。
その後initdbを実行、pg_hba.confを編集、postmasterの起動を試みたのですが、
マニュアルにあるとおりに -l オプションを与えても、postmasterが起動しません。

> Try 'postmaster --help' for more information.

のように出てしまいます。

-l オプションを指定しない場合は、正常にpostmasterが起動します。
postmasterを起動させるユーザアカウント権限等、思いつくことはチェックして見たので
すが、起動させることができないままになっています。


どのような箇所を見直せばよいのか、ご存知の方はおられますでしょうか。
よろしくご教示くださるよう、お願いします。
