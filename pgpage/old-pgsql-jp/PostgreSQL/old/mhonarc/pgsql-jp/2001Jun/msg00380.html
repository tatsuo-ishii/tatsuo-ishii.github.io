渡辺と申します。

postgre7.0.3 + PHP4.0.5 という環境でアプリを作っております。

unique値を設定したフィールドがあるテーブルに
なんらかの値をinsertするとき、その値がunique違反ならば
当然エラーを出しますよね。

例えば、

# select * from test_tbl;

sid | name
----+-------
101 | aaa
102 | bbb
103 | ccc
(sidもnameもunique制約あり）

# insert into test_tbl (sid, name) values ( 104, 'ddd' );
ERROR:  Cannot insert a duplicate key into unique index なんちゃら

このinsert文を、
「もしもunique違反な値をinsertしそうになったら
  何もしない（エラーを出さない）」
という風になるように書きたいのですが、
どんな風に書けばよいでしょうか？
どなたか少し御知恵いただけませんか？

postgreな話題というより一般SQLな話題なうえに、
そのへんの資料にありそうな簡単な話なのですが意外と見当たらなくて
いきづまってます。
（ああ、なんか簡単な書き方で解決できそうなのに・・・）

＃ＳＱＬの書き方に関するＭＬなんてご存知の方いらっしゃったら
＃情報モトムです。
