> 境です。
> 可変長項目の条件式について教えて下さい。
>
> 環境は、Postgresql V7.0.3 ｏｓ：Solaris2.6を使用しています。
>
> create table でvarchar(10)で指定した項目に対して次の様なデータが登録されて
いたとします。
>
> １レコード目：1234567890（１０バイト）
> ２レコード目：ABC（３バイト）
>
> そして検索時に
>
> select * from テーブル名 where 項目名　='1234567890'
>
> を検索すると１行目が抽出されますが、
>
> select * from テーブル名 where 項目名　='ABC'　　　　−−（１）
>
> とすると２行目が検索出来ません。
> （select * from テーブル名 where 項目名　='ABC       '
> 　　と7バイトの空白を入れると抽出されます）

これで抽出されるということは、そのように登録しているからですね。
varcharといえども'ABC      'で入れたら空白を除くことはしませんのでそのまま入
ります。
私のところでは'ABC'といれたらきちんとselectされますよ。
