杉村です。

RedHat 6.2J + PostgreSQL 7.1.2 という環境で、
SQL文での改行文字の扱いに悩んでいます。

create table test (
  testdata varchar(256)
);

とテーブルを作り、ここに文字列データを insert するものとします。

insert into test (testdata)
  values ('abcde\nabcde');

を実行しますと、select した際に PostgreSQL 7.1.2 では

 testdata
---------
 abcde
abcde

という結果になります。

しかし、Oracle 8.1.6 では

TEST
------------------------------------------------------------------
abcde\nabcde

となります。

PostgreSQL でも Oracle と同じ挙動をするようにすることは
できるでしょうか?
SQL92 の仕様も探していますが、
本来正しい動作としてはどちらになるのでしょうか。
