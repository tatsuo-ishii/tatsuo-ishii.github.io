山下 です 
初めての投稿です. よろしくお願いします.

検索で検索できない状態があります. 
LIKE の検索文字列とそのときのヒット数が下記のようになります 
%コイル%   0 件 
%コイ%     313 件 
%ル%       0 件 
%コ%       2283 件 
%イ%       624 件 
%ル        0 件 

同じテーブルで正規表現で検索してみる 
'.*ル'    525 件 
'.*ル.*'  0 件 
'.*コ.*'  2283 件 

です. 「ル」が探せない?

テストに数件入れて結果表示しても件数がある物は、
表示できるが0件の物は、表示しない.
またその時に '.*ト.*' で正規表現検索したら
  コイル
  ダイヤモンド
の項目がヒットしました. 「ト」の文字が無いのに
上記以外にもトの文字がる物は、当然表示してます


コマンドは、 
SELECT COUNT(*) FROM shinamono WHERE 品名 LIKE '%ル%'; 
又は、 
SELECT * FROM shinamono WHERE 品名 ~ '.*ト.*'; 
のようにしてます. 

使っているバージョンは、 
PostgreSQL 7.1.2 
Vine 2.0 PPC (FTP版) 
Power Mac G3 (Blue G3) 
シーラカンスの本を参考にしながらコンパイルとインストール 
しました. 
データベースのエンコーディングは、UNICODE で
psql からは、EUC_JP で実行してます.
