> 最近では「インストールタイプ」を選択させて，あとは自動ってのが主流じゃ
> ないですかね．もちろん一つ一つパッケージを選択するのもアリですが，選択
> 対象になるパッケージがやたらめったら多いので，途中でめげます．(^_^;)

そうそう、あげくに依存関係がどうしたこうしたとか言われてしまう。

> となると，そのタイプに応じたディストリビュータの指定したパッケージがど
> んどん入っていくわけですが，確かに個人的にはちょっと気持ち悪いです．こ
> の辺りは BSD系なんかではどうでしたっけ?

*BSD でも、ものによるんじゃないかと思いますが、FreeBSD や BSD/OS は
伝統的な方法に近いと感じます。
つまり、4.1.4 までの SunOS みたいな感じ。でも Solaris も似たようなもんか。

「何々向け」というようなくくりでインストールする事もできますが、
もう少し細かく指定しようとしても、よくある Linux ディストリビューションの
ようにプログラムをひとつひとつ指定する必要はありません。

細かく指定すると言っても、「マニュアル」とか、「ゲーム」「ドキュメント」
「基本コマンド」みたいな感じ。
記憶があやふやだけど、(^^; まぁ、イメージ的にはそういう感じです。

「開発」といっても、C コンパイラとヘッダファイルと make,yacc,lex,ld 
みたいなコマンドが入るかどうかというだけ。
ごそごそと他の雑多多様なものが入る事はないし、そういう余分なものを
省こうとするとひとつひとつ指定する事を強要される事もない。

つまり、「極めてフツーの UNIX」であるだけです。良くも悪くも
フツーの UNIX なのです。

Linux は基本部とオプション部という区分けがないのですが、FreeBSD などでは
「FreeBSD という固有のもの」があって、それ以外のものが ports/packages で
提供されているのです。
この部分に関する考え方はまるで違うといってもよいでしょう。

Linux は… 「Linux は Linux なのであって UNIX ではない」と私は思います。
Linux しか知らない人は、「これが UNIX というものか」と思うのは色々な
意味で大間違いであるという事を知っていて欲しいですね。

> # それ以前の OS 自体に関する知識が少ない方の質問も良く見るような…．

それ以前に、コンピュータそのものに関する知識も足りないような… ;-p

> 結局この辺りはどの OS / Distribution を選ぶか? ではなくて，管理者の素
> 質の話になるのでしょうね．

素質以前に、「コンピュータ(システム)は管理が必要である」という事を
理解している/理解する気があるかどうかなのではないでしょうか。

# 頭では理解していても、素質がなくてできない人はやはりいますしね

ちょっとイヤらしい言い方をすれば、所詮 MS-DOS/Windows あがりや
Macintosh あがりの人間はねぇ… とでもなるのかな。;-p
