> 環境移行で　Solaris８を用いることになりました
> その際　PostgresSQLのコンパイルが必要になると思いますが
> フォルテを用いているのですが　
> コンパイルする際に　ｇｃｃがなくて　できない状態です。

どういうエラーメッセージが出るのですか？

gcc でなくとも ANSI C 準拠 C コンパイラがあれば、コンパイルは
できます。Forte も C サポート版はちゃんと C コンパイラがついて
くるはずですが、それを使っているということなんですかね？
いずれにしろ、回答に必要な情報が無いですね。

ただ、PostgreSQL はドキュメントに書いてあるとおり GNU make などが
必要になるので、GNU make などのインストールは必要になります。

> ｇｃｃをインストールしてなくても　コンパイルはできるのでしょうか？

ですから、ちゃんとした C コンパイラがあればコンパイルはできます。

> できないのであれば
> ｇccのインストール方法を教えてください。

何か C コンパイラがあるという前提の場合には、gcc のソースパッケージを
持ってくれば、そこにインストールの仕方は書いてあります。
それを読んでください。

C コンパイラが無いので gcc のバイナリパッケージを持って来たい場合には、
SunSite などからダウンロードする事になるでしょう。
こっちは、SunSite やダウンロードしたパッケージに説明があったはずです。

いずれにしろ、「gcc のインストールの仕方」などは pgsql-jp で聞くべき
話ではないですな。
