堀田％相変わらず質問のレベルが低い＠諫早市です。

＞su postgres -c "/usr/local/pgsql/bin/postmaster -B 256 \
＞ -D/home/postgres/data > /var/log/pgsql.log 2>&1 &"

でバックエンドを動かして、

＞server1:~$ ls /home/postgres/data/base/
＞dnsmaint/   template1/  zipdb/

zipdb というデータベースがあり、

＞server1:~$ psql zipdb
＞POSTGRESQL の会話的 SQL モニターへようこそ。
＞POSTGRESQL の著作権、については COPYRIGHT ファイルをご覧ください。
＞
＞   \? で \ コマンドのヘルプを表示します。
＞   \q で終了します。
＞   \g または ';' で終わらせると、クエリーを実行します。
＞ 現在データベース 'zipdb' に接続されています。

と、ちゃんと接続できるんですけど、

server1:~$ destroydb zipdb
ERROR:  destroydb: database zipdb does not exist.
destroydb: database destroy failed on zipdb.

になってしまいます。なぜでしょう？
