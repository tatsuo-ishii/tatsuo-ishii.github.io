＃よって大嘘こいている可能性大。

> There is no operator '<' for types '_int2' and '_int2'
というのは、「int2[4] の順序関係が判りませんぜ」という
ことでしょうから、IPアドレスを順序関係のハッキリする数値
［たとえば、((a*256)+b)*256)+c)*256+d］に置き換える
関数 ip2int とかを作ってあげて、

SELECT hostname, ip2int(ip_addr) ipnum FROM node
   WHERE domainname='hogehoge'  ORDER BY ipnum;

とかするといいかもしれませんね。

えっ ip2int(int2[4]) はどうやって作るかって？
すいません。そこまではわかりません。
