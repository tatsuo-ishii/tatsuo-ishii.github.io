> > 
> > 　Access のクエリーの昇順・降順での並び替え、SQL文でのORDER 
> > BYでのエラーが出ていたので、困っていたのですが、ご本家英語の
> > PostgreSQL ODBCのページに対応策として
> > 
> > 　1) PostgreSQL server へのパッチ
> > 　2) Jet database engine のアップデートの方法
> > 
> > が掲載されていましたが、これは日本語版でも使用が可能でしょう

> 
>   分かりません。しかしここでおっしゃっている日本語版というのは何をさし
> ているのでしょうか。PostgreSQL MBパッチ、PostgreSQL ODBC 日本語版、
> Access 日本語版。
> 
>   1) PostgreSQL server へのパッチ
> 
>   これは恐らく問題無いでしょう。このパッチで、SELECT文のターゲットリス
> トに含まれていないカラムをORDER BY句に指定できるようになるようです。
> 
>   でもPostODBCもPostgreSQL ODBC Driverも、ODBCのドライバ情報取得APIで
> 「ORDER BY句に指定したカラムはSELECT文のターゲットリストに指定しなけれ
> ばならない」とちゃんと提示していますので、この問題はMS Jet database
> engineのバグともいえるのですが。
> 
>   2) Jet database engine のアップデートの方法
> 
>   これを日本語版Windowsにインストールしても大丈夫でしょうか。良いよう
> な気もしますが。でもJETのエラーメッセージが英語になったりして。
> 
>   とりあえずこれをインストールしなくても動くようなので、PostgreSQL
> server へのパッチだけにしておいた方が（今のところ）安全な気がします。

　PostgreSQL MB patch + PostgreSQL ODBC 日本語版 + Ａｃｃｅｓｓ日本語版
で使ってます。

　ご助言をうかがって、早速試してみました。両方のパッチを当てたところで、
うまく、ＯＲＤＥＲ　ＢＹやＡｃｃｅｓｓのテーブル表示での昇順・降順の並び
替
えがうまく行きました。ありがとうございました。
