私は半分遊びで(www-admin の特権を乱用して?), ある特定の ftp
server 内のファイルを検索する Web ページをつくっています. 
( http://www.mk.ecei.tohoku.ac.jp/~uesaka/atchie/ )
このページは検索キーワード(gcc とか)によっては数百件のデータ
が結果として得られますので, 表示数を10とか50とかに限定して表
示できるようにしています. 

前置きが長くなってしまいましたが, 私の場合は, 何も考えずに, 
すべての結果を取り出してきて, ある範囲だけを切り出して, 表示
するようにしています. 最初は無駄かなと思ったのですが, 実際作っ
てみるとそんなにストレスは感じませんでした. http server や 
PostgreSQL は Sun SPARCStation ELC というかなり古いマシン上
で動いています. それでも問題はないようです. まぁ, そんなに頻
繁にアクセスがあるわけではないからでしょうけど. 

私はPHP/FIを使っていますので, 次のように書いています. 
現在何番目からを表示しているかとか, いくつ表示するかといった
情報は hidden で渡しています. 

/* 現在何番目からを表示しているかは beginidx として渡します */
echo "<input type=\"hidden\" name=\"beginidx\" value=$endidx>\n";
/* 検索するファイル名 */
echo "<input type=\"hidden\" name=\"filename\" value=$filename>\n";
/* 全部でいくつのファイルを表示するのか */
echo "<input type=\"hidden\" name=\"items\" value=$items>\n";
/* submit のボタンです */
echo "<input type=\"submit\" value=\"See more $items items.\"></form>\n";

ご希望でしたら, 全ソースファイルをお送りします.
