インデックスがbtreeでないというエラーに対する対処について

----- Original Message ----- 
From: "ネットマンの北川" <kitagawa@netman.co.jp>
Sent: Monday, July 16, 2001 10:47 PM
Subject: [pgsql-jp 22410] btree エラー


> 　FATAL 1:  Index pg_trigger_tgrelid_index is not a btree

同様の現象を経験しております。
約3000件程度のデータをデータベースに追加する際、処理時間を
短縮するため、いったんインデックスをドロップしてから追加処理
をおこない、その後インデックスを再作成するという処理をおこなって
おりますが、インデックス再作成中になんらかの影響で停止して
しまった場合に、データ抽出のためのインデックスを使ったselectを
実行すると上記エラーが表示されたことがあります。

対処方法としては
・drop index でインデックスを削除
・create index でインデックスを再作成
　create index pg_trigger_tgrelid_index on テーブル名
　using btree(インデックス名);

もしcreate indexをおこなってもまだ同様のエラーが出る場合には
インデックスファイル自体が壊れている可能性があります。
その場合にはインデックスファイルの容量を確認して、0でなければ
強制的に中身を空っぽにするなどの対処をしてからインデックスの
再作成をする必要があります。
