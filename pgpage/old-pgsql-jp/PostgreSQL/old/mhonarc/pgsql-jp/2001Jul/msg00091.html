渡邉と申します。
現在、PostgreSQL 6.5.2から7.1.2へ移行中です。
(さらにPHP3から4へ移行中)

これまで、
SELECT TABLE1.A, TABLE1.B FROM TABLE1,TABLE2 WHERE ...
UNION
SELECT TABLE1.A, TABLE1.B FROM TABLE1 WHERE ...
ORDER BY TABLE1.A, TABLE1.B

というSQL文を使用していたのですが、7.1.2にすると

Warning: PostgreSQL query failed: ERROR: ORDER BY on a
UNION/INTERSECT/EXCEPT result must be on one of the result columns
in test.php on line 19

というエラーが出始めました。ORDER句を
ORDER BY A, B
と書き直すとうまく動くので、PostgreSQLのバージョンによる差異のようです。

リリースノートを一通り眺めましたが、このような記述はないように
思えました。このような仕様変更があったのかどうか、それとも設定
ミス・移行ミスなのかどうか、ご存知の方いませんか？
