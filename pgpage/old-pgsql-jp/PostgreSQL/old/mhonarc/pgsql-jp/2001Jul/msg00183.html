こんにちは、百海と申します。

PostgreSQL 6.5.2 を使用しており、DB名 sample、テーブル名 member の
レコード数は 40328、data/base/ にあるファイル member のファイルサイズは
約13MB、daba/base/ にあるファイル数は 15450 です。


psql 上で
psql > SELECT * FROM member;
のようなSQLを実行すると以下のようなエラーが発生します。

Backend message type 0x44 arrived while idle
Backend message type 0x44 arrived while idle
We have lost the connection to the backend, so further processing is impossible.  Terminating.


\copy コマンドもエラーになります。
psql=> \copy member to /usr/local/pgsql/member.copy
pqWait() -- connection not open
PQendcopy: resetting connection
Copy failed.


レコード数をカウントすることはできます。
psql > SELECT COUNT(*) FROM member;
count
-----
40328
(1 row)


また pg_dump を実行しようとすると

pqWait() -- connection not open
PQendcopy: resetting connection
SQL query to dump the contents of Table 'member' did not execute
correctly. After we read all the table contents from the backend,
PQendcopy() failed.  Explanation from backend: ''.
The query was: 'COPY "member" TO stdout;
'.

のようなエラーが表示されます。

このような状況なのですが何とかして member テーブルのデータを読み出す
ことは出来ないでしょうか？
