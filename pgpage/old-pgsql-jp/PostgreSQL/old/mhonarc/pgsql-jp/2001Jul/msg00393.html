山口＠埼玉です。

--------------------------------
OS：LINUX(SlackWare)
　　＋Apache＋PostgreSQL7.02＋PHP4
--------------------------------

Ms-AccessのデータをODBCでPostgreSQLへ
エクスポートしました。
Accessではオートナンバーで入力順に番号を
振り当ててこれを主キーにしていました。

PostgreSQLでは、シーケンスを使って
オートナンバーに代わる作業ができるということですが、
過去ログなどを見る限り、既に相当数の情報が入力さ
れたDBに対応させるのは難しいように思いましたので、
web入力時にDBの番号列（カラム名 = aute_no）から最大値を選ばせ、
新規登録データに”最大値 + 1”を割り当てたいと考えました。

前置きが長くなりましたが、
先ず、最大値がきちんと選ばれるかテストするため
---------------------------------
<?php
$dbcon = pg_connect ( "dbname=xxxxx user=xxxxx");
if (!$dbcon) { die( "<hr>$dbcon失敗<hr>"); }
$max1 = "select max(aute_no) from userdb1";
if (!$max1) { die( "<hr>$max1失敗<hr>"); }
echo pg_exec ($dbcon, $max1);
?>
---------------------------------
というphpファイルをアップして、webから見てみると
画面上に　Resource id #2　と出ます。
これが何を意味するのか、現在の私にはわかりません。

検出されるべき数字は 2001 で、
telnetで　psql　=>select max(aute_no) from userdb1";
を実行した場合は検出されます。

「Resource id #2」の意味、
その他、以上から問題点をお気付きの方がいらっしゃいましたら
何卒アドバイスお願い致します。
