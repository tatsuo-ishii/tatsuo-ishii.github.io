  • From: "吉岡　誠" <yoshioka@d-gate.co.jp>
  • Date: Tue, 10 Jul 2001 17:12:30 +0900
はじめまして　よしおかといいます。

PostgresSQLをマシンを起動したときに
自動起動しようと思い、rc.localファイルに

[　rc.local　ファイルにて　]

POSTGRESDIR=/usr/local/pgsql
if [ -x $POSTGRESDIR/bin/postmaster -a -d $POSTGRESDIR/data ] ; then
    rm -f /tmp/.s.PGSQL.5432
    su - postgres -c "$POSTGRESDIR/bin/postmaster -S -i"
    echo -n 'postmaster '
fi

と記述しています。もちろん .bashrcファイルではパスを記述しています。

[ .bashrcファイルにて　]

export PGHOME=/usr/local/pgsql
export PATH=$PATH:$PGHOME/bin
export PGDATA=$PGHOME/data
export PGLIB=$PGHOME/lib
export MANPATH=$PGHOME/man
export LD_LIBRARY_PATH=$PGLIB

と記述しております。これで再起動して netstat -autコマンドで
起動中のサービスを参照してもpostgresqlは動いていません。
ちなみに手動でpostmaster -S -i　では正常に起動します。
どなたかうまく自動起動する方法をご存知でしたらご教授ください。
よろしくお願い致します。

※以下は動作環境です

OS              RedHat6.2
WebServer   Apache1.3.14
                  JDK1.3
                  tomcat3.2.1
DB              PostgreSQL7.0.3
