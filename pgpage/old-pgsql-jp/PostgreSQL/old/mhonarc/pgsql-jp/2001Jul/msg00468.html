青木です。
お世話になっております。

江本さんが教えてくれた通り
copy テーブル名 from ファイル名 using delimiters '|' with null as '';
とすることで空のデータでもエラーなく読み込めるようになりました。
ありがとうございました。

EMOTO Masahiko さんは書きました:
>At Wed, 25 Jul 2001 16:19:48 +0900,
>Takashi AOKI wrote:
>> ---テーブル---
>> create table aaa
>> (
>>   bbb text,
>>   ccc date
>> );
>> 
>> ---データ---
>> aaa|2000-10-10|
>> bbb||
>> ccc|2001-01-01|
>> 
>> 上記のようなテーブルとデータがあったとして
>> copyでデータを読み込むと
>> ERROR: copy: line 2, Bad date external representetaion ''
>> となりdate型に空の項目があるとエラーとなってしまいます。
>> 過去ログを読んだところ
>> [pgsql-jp 10546] でテキストに \N を入力すれば
>> ＯＫになるとありましたが、テキストデータに手を加えることなく
>
>
>> db1=# \h copy
>> Command:     COPY
>> Description: Copies data between files and tables
>> Syntax:
>> COPY [ BINARY ] table [ WITH OIDS ]
>>     FROM { 'filename' | stdin }
>>     [ [USING] DELIMITERS 'delimiter' ]
>>     [ WITH NULL AS 'null string' ]
>> COPY [ BINARY ] table [ WITH OIDS ]
>>     TO { 'filename' | stdout }
>>     [ [USING] DELIMITERS 'delimiter' ]
>>     [ WITH NULL AS 'null string' ]
>>
