MS SQL Server 6.5 相手で似た問題が発生した場合は、
timestamp列を追加して、ACCESSに全項目指定を止めさせる
という方法がありますが、PostgreSQLにも使えるかどうか。

http://support.microsoft.com/support/kb/articles/Q152/0/21.ASP

http://support.microsoft.com/support/kb/articles/Q208/8/58.ASP?LN=JA&SD=SO&F
R=0


いずれにせよ、ACCESSのリンクテーブルを直接開くとネットワーク帯域
を浪費するので、面倒でも画面を作り、スナップショット型で限定した
データを取り出して、SQL文で更新するのが安全だと思います。
