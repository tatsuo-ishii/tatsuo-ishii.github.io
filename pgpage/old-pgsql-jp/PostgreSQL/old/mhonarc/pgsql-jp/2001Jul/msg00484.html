先にリリースさせていただいた pgload(仮) を使って、
  pgload query -c 1 -n 10 -q "..."
という形（1同時接続、10回クエリ発行）で実験してみました。

PGLoad(仮)と、下記の実験のもう少し詳細はことは
  http://www.itboost.co.jp/inst/inst_33.php
にあります。

それぞれ
       lesson : 7891 行
  classlesson : 9874 行
というテーブルで、1 同時接続、10 回のクエリ発行、それぞれの
クエリでは 221 件の結果が返ってくる、という条件です。

IN を使った場合、

> Average time (par one query)        =     2.31 (sec.)
> # of request which can be processed =    26.00 (req./min.)

で、EXISTS を使った場合、

> Average time (par one query)        =     0.21 (sec.)
> # of request which can be processed =   287.43 (req./min.)

と、このぐらい（約10倍!）の差がでました。

PostgreSQL 7.1.1 on i686-pc-linux-gnu, compiled by GCC egcs-2.91.66 
です。
