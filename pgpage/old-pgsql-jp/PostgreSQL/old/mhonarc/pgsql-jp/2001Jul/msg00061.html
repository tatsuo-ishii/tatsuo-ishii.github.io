> [環境]

SQL Server 7 SP-3
PostgreSQL 7.0.3
PostgreSQL ODBC Driver日本語版 7.1.0.5
という環境で試してみましたが、ウチでも分散クエリは失敗
しました。OPENQUERYは成功。

>  >select * from PGLINK...test
>  >サーバー : メッセージ 7313、レベル 16、状態 1、行 1
>  >プロバイダ  'MSDASQL' で指定したスキーマまたはカタログが無効です。

SQL ServerのBooks Onlineによると
「注意 SQL Server は、現在のログインに基づいてリモート テーブル
の所有者を推定することはありません。分散クエリを正しく実行する
には、完全修飾名を使用してください。」
となっていますので、分散クエリにはカタログとスキーマ部分が必要
ですが、
sp_tables_exでPostgreSQLのテーブルを表示しても、表示される
テーブルのカタログ列、スキーマ列とも空なので指定のしようが
ないと思います。

POSTGRES.NULL.NULL.TableName も
POSTGRES.''.''.TablrName もダメでした。
(POSTGRESはテストに使ったリモートサーバー名)
