ソフト工房の近藤です。

PostgreSQL のスーパーユーザー権限は、OS の ROOT 
権限に例えられるので、共同使用のレンタルサーバー
の場合は、ROOT/SUPER USER 権限を利用者に与えない
のは当然ではないでしょうか。

契約時に、いくつかの PostgreSQL user を割当てて
もらっておけば、createuser できなくてもそれほど
不都合はないでしょう。

また、Web User (nobody など)に、SELECT, UPDATE/DELETE
権限を与えていると、他の人の DB の読み書きもでき
てしまいます。こちらの方が問題では。

いずれにせよ、同一のポート番号で動く PostgreSQL 
をレンタルで共用するのは、善意の利用者を想定しない
と難しいのではないでしょうか。

例えば、SORT WORK は、多分同一パーティションのはず
ですから、だれかが、巨大なソートや、マージジョイン
をおこなって、DISK FULL にしてしまう危険性もあるで
しょう。

このあたりは、割切って利用するか、ハウジングなどで
自社専用サーバーにするかしかないのでは。
