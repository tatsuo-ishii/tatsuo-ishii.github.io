ＮＴＴの金升です。

postgres95用のODBCドライバーを試してみました。
といっても、付属のサンプルアプリが動くかやってみただけですが。

サーバはSun SS20 SunOS4.1.3_U1 + JLE + postgres95 1.01 +jp 
クライアントはWindows95です。

バイナリーアーカイブを展開して POSTODBC.DLL を windows\system 
にコピーします。

windows\odbc.ini (無い場合は新規に作成)を以下の様に設定しました。

----------------- WINDOWS\ODBC.INI ---------------------------
[PostgresData1]
Driver=C:\WINDOWS\SYSTEM\POSTODBC.DLL
; Make this entry point to the location where POSTODBC
; is located on your maching
Description=A Test Data Source Powered By Postgres95
Servername=gohan.osaka.ish.cae.ntt.jp
Port=5432 ; this is the default port for Postgres95
Database=test
; name of the database as it is known by Postgres95
---------------------------------------------------------

postmaster が動いているマシンgohan.osaka.ish.cae.ntt.jp
では HBA を有効にしてmake したので、postgres95/data/pg_hba
がデフォルトのままだと接続できませんでしたので、クライアントの
アドレスが接続できる様に修正しました。
ひょんなことからＨＢＡの確認ができました。(^^;)

バイナリアーカイブにはコンパイル済みの sample.exeが入っていました。
起動すると、データソース名の入力要求がきますので、ODBC.INIに設定
した PostgresData1 を指定します。

そしてユーザ名を聞いてきますので、データベースにアクセス可能な
ユーザ名を入力します。

あとは、psql みたいに SQL を入力すると問い合わせ結果が表示されます。
ただし日本語は表示できません。

以上です。
