井上俊博です。

以前 Mac OS X Server でのコンパイルのお話が出ていましたが、
この度 iBook を入手したので Darwin でのコンパイルに挑戦してみました。
結果から言いますと失敗しました。

まず configure は次のようにして通しました。（かなり無茶ですが）
./configure --host=ppc-freebsd --with-template=generic

make するとしばらく進むのですが、link.h がないと言って止まります。
無視しても差し支えなさそうだったのでソースから #include を消しました。
すると別の所で sem.h がないと言って止まりました。

こればかりはどうしようもなかったため諦めました。
この辺りを Mach の枠組みで実装し直すしかなさそうですね。

仕方ないので当面は Darwin では Berkeley DB を使うことにしました。
こちらは小細工をしなくても configure で Darwin を認識して、
コンパイルもまったく問題なく終了しました。
