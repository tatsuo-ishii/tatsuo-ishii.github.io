業務って言っても色々あると思うんですが、、在庫管理とか経理系とかを
想定してるんでしょうか。それとも何でもいいのかな？

私自身は客先のシステムで PostgreSQL を使った事はまだありません。
これは単にそういう機会に恵まれていないだけですが…

自社の社内掲示版の記事管理には PostgreSQL を使っていますが、こういう
のはあまり「業務プログラム」とも言えないかな。

この春に運用を開始した自作のショボイやつで、Apache 1.3.9 + JServ 1.1
上での Servlet で実装してみました。
PostgreSQL 自身はまだ 6.5.3 だったりします。

# Tomcat 3.1/3.2beta + PostgreSQL 7.0.2 などの組合せに移行したいとこ
# なのですが、何となくのびのびになってるなぁ。(^^;

この掲示版システムで使っているテーブルは 10数種類ほどですが、社員
テーブルなどは実は Oracle だったりします。:-)
これは他のシステムとの絡みからこうなっていて、PostgreSQL は記事管理
関係の 8種類のテーブルを使っています。

そういうわけで、プログラムは JDBC で Oracle にアクセスしたり PostgreSQL
にアクセスしたりしてて、ちょっと変だと言えば変かな。(^^;
全部 Oracle にしなかったのは、単に PostgreSQL でやりたかったからです。

5月からの運用で投稿記事はまだ 800通ほどなので、性能的にも運用的にも何も
問題ないですねぇ… 規模が小さすぎ。(^^;

話は違いますが、どこかに Servlet で書いたフリーなスケジュール管理ソフト
ってないですかねぇ。
現在社内ではサイボウズのを使ってるんですが、ユーザ管理やユーザ認証が
バラバラになるので運用上やりにくいんです。
で、自作しようかと考えてるんですが、月刊スケジュール表示などの HTML
表示部が面倒臭そうなので着手がずるずると延びてます。(^^;
一部流用しながら使えるのがあればなぁ… なんてのはムシがよすぎか。(^^;
色んなもののユーザインタフェース部を参考にして書くしかないかな。
