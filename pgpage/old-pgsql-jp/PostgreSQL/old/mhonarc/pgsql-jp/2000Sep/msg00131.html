From: 加藤さん
> そこで、質問なのですが、PostgreSQLの排他制御はどこまで可能
> なのでしょうか？
> データベースのロックは可能か？
> 排他制御を行うにあたり考慮しておく点がありましたら、ご教授
> ください。

  RDBMSとして一般的な排他制御はPostgreSQLでも可能です。
  以下のSQL文について調べてみてください（何となくトランザクションで囲んでみ
ました）。これらはPostgreSQLで排他制御（や同時実行性能）に関連しそうな物で
す。

  BEGIN
  SET TRANSACTION ISOLATION LEVEL 〜
  SELECT 〜 FOR UPDATE
  LOCK TABLE 〜
  COMMIT

> あわせて、今回ApacheJservでServletを開発しておりますが、
> このApacheJservの機能でも排他制御に関してなにか可能なので
> しょうか。

  私はJDBC使ってないので良く知りませんが、上記のSQL文を発行することに違いは
ないので、特に注意点はないと思います（トランザクションの制御方法は違うかもし
れない…）。
