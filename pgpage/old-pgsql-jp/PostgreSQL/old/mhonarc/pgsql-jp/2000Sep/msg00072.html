佐伯様、かさはら様、清島様、石井様ありがとうございます。

>
>> |SELECT DECODE(COLOR, 1, '赤', 2, '青', '緑') FROM ...
>> は以下のように書けそうです。
>>
>> SELECT
>>     (CASE
>>         WHEN COLOR = 1 THEN '赤'
>>         WHEN COLOR = 2 THEN '青'
>>         ELSE '緑'
>>     END)
>> FROM
>>     ...
>
>はい，これでOKです．といっても，decodeなるものが何をするのか理解してい
>ないので、元記事の方の意図通りなのかはわかりませんが...  ところで仮に
>意図通りとすると，caseがあればdecodeはいらないように見えます．わざわざ
>decodeなるものを用意しているところをみると，Oracleにはcaseがないのでしょ
>うか?
>
私の乏しい経験の中ではOracleでcaseを使ったことはありません。それが、Oracle
に実装されていないと断言はできませんが。

Oracleからの移行でSQLを触りたくない、という意識が強すぎdecodeという方言
に縛られてしまいました。（caseの場合少々構文が長くなりますが）
SQL標準のcaseで十分というより、標準を使うべきでしょう。
