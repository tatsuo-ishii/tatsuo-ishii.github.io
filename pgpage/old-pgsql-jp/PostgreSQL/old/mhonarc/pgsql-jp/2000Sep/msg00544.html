Conyです。

少しみなさまのお知恵を拝借したく思います。
内容ですが、
   create table a1 (
     code int4,
     statename varchar(50),  // 都道府県名
     cityname   varchar(50)  // 市町村名
   );
といったテーブルがありデータは
    1001 大阪府 大阪市
    1002  大阪府 寝屋川市
     ・・・・
    2001  兵庫県 神戸市
    2002  兵庫県 姫路市
のようになっています。
(codeの４桁目以上が都道府県をあらわし、３桁目以下が市町村をあらわす）
これをある事情で下記のVIEWにしたい
    create view city as select
      citycd as code%1000
      name  as cityname
      from a1;
    create view state as select
      stateid as code/1000
      name as statename
    from a1;
こうした時にcitycd as code%1000とstateid as code/1000でステートメントエラー
が
発生します。これを何とか回避したいのですが、なにかいい方法があればご助力
ください。

環境は
   OS ： Turbo Linux 6.0
   PostgreSQL ６．５．３
です。
よろしくお願いいたします。
