Tadao Nakamura <tadaon@ffs.fujifilm.co.jp> さんは、
「[pgsql-jp 17953] Solaris7 でバックエンド数を増やせない 」について書きました:

> set shmsys:shminfo_shmmax = 0x7fffffff
> 
> を宣言しており、FAQにはこれ以外に共有メモリに関する記述は
> ないようですが、他に変更すべき部分がありますか？

 たとえば semsys:semmni と semsys:semmns とか。
 Oracle や SYBASE を入れるときには変更しないといけなかったと思い
ます。

 # たとえば semsys:semmns=200 と semsys:semmni=70 という設定が
 # された機械が手元にあります。

 semmni はセマフォ識別子の最大数を設定し、標準値は 10、値の範囲は
1〜65535 です。

 semmns はシステムにおける System V セマフォの最大戸数を設定する
もので、標準値が 60、値の範囲は 1〜符号付整数の最大値です。


 各パラメータについて詳しいことは、最近になって用意された
  Solaris Tunable Parameters Reference Manual
    http://docs.sun.com:80/ab2/coll.707.1/SOLTUNEPARAMREF/@Ab2TocView?Ab2Lang=ja&Ab2Enc=euc-jp
から
  Solaris Tunable Parameters Reference Manual: System V Semaphores
    http://docs.sun.com:80/ab2/coll.707.1/SOLTUNEPARAMREF/@Ab2PageView/idmatch(CHAPTER2-63)#CHAPTER2-63?Ab2Lang=ja&Ab2Enc=euc-jp
をご覧下さい。
