この可能性が非常に高いように思います。上の error を良くみてみると、そ
れぞれ path  が異なっていることが分かります。

したがって考えられる状態としては、

1. 2 つの perl が存在している
2.apache の make は、/usr/local/lib/perl5 以下に file を置いている 
perl で make されている。
3.ところが、command path の perl は、/usr/bin なりにあるものである。

というようなものが考えられます。

対策ですが、
(1)とりあえず .cshrc なりの path を設定しているところを書換えて、不具
合が起きないようにする
#かなり場当たり的です
(2)2 つある perl 、一方だけを使用するようにする
#消すのが面倒なら、実行 permission だけを落しておくだけでも良いかもし
#れません。

などというようなものが考えられると思います。
