これは PostgreSQL ではなく、PHP への質問ですねぇ。
PHP の ML で質問した方が適当なのではないかとも思いますが…

私自身は Windows で PHP を使おうとした事もないので、特に調べては
いませんが、これは NT 環境で PHP4 を PostgreSQL アクセスが可能な
ように make しようとすると ODBC 経由以外に方法が無いのでどうしたら
よいか？  という質問でしょうか？

そんな事はないんじゃないかと思いますけど…
そうなんでしたっけ？ > Windows で PHP4 を使ってる人

ソースから make する場合、Windows 側に PostgreSQL のライブラリと
ヘッダファイルさえあれば、ネイティブなアクセスを行なう PHP4 を
作る事ができると思いますが。

もしも本当に NT ではこのような make ができないというのであれば、
ソースを自分で Hack するか、誰かが Hack してくれるのを待つしか
ないなという事になるでしょう。

私の想像ですが、そちらではバイナリの PHP4 for NT をどこからか
持って来てそのまま使おうとしているのでしょうか？
で、その PHP4 のバイナリが特定の DBMS アクセス用に make されて
なくて ODBC 対応になってて、何かの DB にアクセスする時には
DB 側の ODBC ドライバを用意して ODBC リソースを定義せよ。
と、なってるだけとか？

何を使おうとしているのかを明記した方が、そちらの事情が他人にに
わかりやすいと思います。というか、そうしないと伝わらないですね。

> 出来れば、NTは、日本語環境で、
> 構築したいのですが・・・

PHP4 の国際化パッチって出てたかな？ それがあればおそらくは問題無く
make できる… または make できるようにする事が可能であろうと思い
ますけど、何か問題があるとか？

PHP4 国際化パッチがなくとも、日本語がまるで使えないという事は
ないと聞いたようにも思いますが、PHP3 i18n 版のようにはいかない
でしょうね。
