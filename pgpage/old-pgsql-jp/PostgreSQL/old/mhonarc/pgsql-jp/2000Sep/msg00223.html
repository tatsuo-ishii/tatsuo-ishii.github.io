[pgsql]のメーリングリストの皆様

はじめまして蓮沼＠学芸大です。
postgreSQLの超初心者なのですが、アドバイスを
頂きたいです。

環境：vinelinux2.0CR(kernel 2.2.16-0vl1)

バージョンを6.5.3-6から7.0-2vl1にアップグレードしました。
すべてソースからrebuildしました（以下参照）。
＝＝＝＝＝＝＝＝＝
postgresql-6.5.3-6.i386.rpm           
postgresql-devel-6.5.3-6.i386.rpm
postgresql-jdbc-6.5.3-6.i386.rpm
postgresql-odbc-6.5.3-6.i386.rpm
postgresql-perl-6.5.3-6.i386.rpm
postgresql-python-6.5.3-6.i386.rpm
postgresql-server-6.5.3-6.i386.rpm
postgresql-tcl-6.5.3-6.i386.rpm
postgresql-test-6.5.3-6.i386.rpm
から
postgresql-7.0-2vl1.i386.rpm
postgresql-devel-7.0-2vl1.i386.rpm
postgresql-jdbc-7.0-2vl1.i386.rpm
postgresql-odbc-7.0-2vl1.i386.rpm
postgresql-perl-7.0-2vl1.i386.rpm
postgresql-python-7.0-2vl1.i386.rpm
postgresql-server-7.0-2vl1.i386.rpm
postgresql-tcl-7.0-2vl1.i386.rpm
postgresql-test-7.0-2vl1.i386.rpm
postgresql-tk-7.0-2vl1.i386.rpm
＝＝＝＝＝＝＝＝＝
そのまま
#rpm -Uvh /usr/src/redhat/RPMS/i386/hogehoge.i386.rpm
でうまくいかなかったので、はじめに古いpostgresqlを
#rpn -e hogehoge
でアンインストールし（もちろん、データベースが空で
ないという警告メッセージがでました）、改めてインス
トールしました。どうやら正常にインストールできたよ
うなのですが、(etc/rc.d/init.d)内のデーモンが起動し
ません。
#./postgresql start
Checking postgresql installation: old version. Need to Upgrade.
See /usr/doc/postgresql-7.0/README.rpm for more information.
となってしまいます。

README.rpmを参照すると記述があり、UPGRADINGに際しては
データベースのバックアップを取るということらしいのは、
分かったのですが、記述通りのpostgresql-dumpがうまくい
きません。
＝＝＝＝＝
bash$ postgresql-dump -t /var/lib/pgsql/backups/seisdata.bak -p /var/lib/pgsql/base/seisdata -d
/usr/bin/postgresql-dump: [: /var/lib/pgsql/backups/seisdata.bak: unary operator expected
/usr/bin/postgresql-dump: [: /var/lib/pgsql/backups: unary operator expected
/usr/bin/postgresql-dump: [: /var/lib/pgsql: unary operator expected
/usr/bin/postgresql-dump: [: /var/lib: unary operator expected
/usr/bin/postgresql-dump: [: /var: unary operator expected
The PostgreSQL data directory, /var/lib/pgsql/data, and the dump file are missing.
＝＝＝＝＝
もちろんUPGRADINGに際しては、シーラカンス本を参考に
%pg_aumpall -o >db.out
というコマンドでバックアップは作ってあるので、データベース
そのものは消してしまっても問題なのですが...。

申し訳ありませんが、どなたかトラブル対処法をお教え下さい。
よろしくお願い致します(^_^;)。
