シーラカンス本やら、Web やらは見てみたのですが、そもそも RDBMS に
おける排他制御の取り扱いがよくわかりません。

例えば

  接続
  UPDATE 〜
  切断

とやるのは、C言語でいう

  fp = fopen(fname, 〜);
  fwrite(fp, 〜);
  fclose(fp);

みたいなもので、

  接続
  BEGIN
  UPDATE 〜
  COMMIT
  切断

とやるのは同様に

  fp = fopen(fname, 〜);
  flock(fp, 〜);	/* 排他 */
  fwrite(fp, 〜);
  fclose(fp);

みたいなものだという理解でよろしいのでしょうか?
それともトランザクションを使わなくとも、基本的には
ある程度は自動的に排他制御してくれたりするものなのでしょうか?
# そうすると今度はトランザクションを使う/使わないの判断は
# どこで行うべきかという疑問が...

どなたか教えていただけませんでしょうか?
