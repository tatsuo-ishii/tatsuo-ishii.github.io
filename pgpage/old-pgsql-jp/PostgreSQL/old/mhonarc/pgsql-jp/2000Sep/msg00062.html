At 20:56 +0900 2000.09.05, HIROSHI OOTA wrote:
>7.0.2のエンコーディングの指定方法に関して教えてください。
>
>1. 7.0.2でconfigureのオプションに--enable-multibyte=EUC_JPを
>   指定して作成した(はず)にも関わらず。データベースのエンコー
>   ディングがデフォルトではEUC_JPになりません。
>   (データベースを作成してpsqlの\lで確認したところSQL_ASCII)
>2. configureのオプションで
>   --enable-multibyte=MULE_INTERNAL
>   としてもsrc/include/config.hは
>/* Set to 1 if you want to use multibyte characters */
>#define MULTIBYTE 1
>   となり--enable-multibyte=EUC_JPの時と変わりません。
>
>これはインストールに失敗したのでしょうか?
>それとも、1が正しい動作だとすると、「7.0.2でSQL_ASCII以外を使用
>したい場合は必ずcreatedbでエンコーディングを指定する」は正しいで
>しょうか?

環境(OS と、そのバージョン) がわからないのでなんとも言えないですが、
ありがちなやつは確認されましたでしょうか。
今回のインストールの前にすでにインストールされている古いバイナリが
動いているというやつですが。
あと、postmaster を立ち上げる時は何のユーザでやっているとか
root からsu して入って失敗したので直接postgres で入ったとか
いうことがあれば、それも書いてくれるといいかもしれません。
過去記事の中にもいろいろありますが、参照されましたでしょうか。

--enable-multibyte=なんちゃら　の指定については、ソースについて
くるdoc/README.mb.jp の中に書いてありますが、ちょっと引用します。

>   なお、configure で選択したエンコーディングは、あくまで initdb のための
>   デフォルト値程度の意味しかありません(initdb では引数でエンコーディングが
>   指定できます)。したがって、異なるエンコーディングを使用するために
>   わざわざ PostgreSQL をリコンパイルする必要ありません。
>
>   initdb は shell script なので、デフォルトのエンコーディングは script を
>   適当なエディタで編集することにより簡単に変更できます。initdb の 42行目
>   付近に、
>
>   MULTIBYTE=EUC_JP
>
>   のような行があるので、= 以降を希望するエンコーディングに変えるだけです。
