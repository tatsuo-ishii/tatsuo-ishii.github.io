----------------------------------------------
以下の環境で PostgreSQLを使用しています。
----------------------------------------------
PC: DELL OptiPlex  Intel Pentium II

OS : Debian GNU/Linux 2.2 (poteto)
       kernel version. 2.2.16

RDBMS: PostgreSQL 7.02

----------------------------------------------
psql で DBに接続できなくなった状況
----------------------------------------------
Debianパッケージのセキュリティーやバグの修正版を
ネットワークから定期的に自動更新するようにしていました。
アップグレード対象を安定版(stable)にリンクしていたため
安定版パッケージが 2.1(slink)から 2.2(poteto)に移行され
パッケージ全体が自動的に入れ換えられてししまいました。
その直後、postmaster が起動できなくなり。
PostgreSQLのソースを再コンパイルしなおしましたら、
postmaster は起動できるようになりました。

Debian GNU/Linux 2.1 (slink)上では
PostgreSQL 6.4 -> 6.5 ->7.0 とアップグレードして
何も問題なく psql でＤＢに接続できてました。

----------------------------------------------
postmaster は起動されているようです。
----------------------------------------------
$ ps  axu
USER       PID %CPU %MEM   VSZ  RSS TTY      STAT START   TIME COMMAND
postgres   126  0.0  0.5  4244  928 ?        S    04:25   0:00
              /usr/local/pgsql/bin/postmaster -D
/usr/local/pgsql/pgdata -S -i

----------------------------------------------
症状：
psql を実行すると「セグメンテーション違反」のメッセージが
出力されＤＢにアクセスできません。
----------------------------------------------
$ createdb   test_db
CREATE DATABASE
$ psql test_db

Type:  \copyright for distribution terms
       \h for help with SQL commands

       \q to quit
