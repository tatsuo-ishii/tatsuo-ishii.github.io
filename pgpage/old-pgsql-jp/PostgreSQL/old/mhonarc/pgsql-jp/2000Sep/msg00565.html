> とりあえず libpq の API を VBA から扱う方法はありませんでしょうか？

私自身は VB はまったく使わないし、自分で VB のコードを書く事はまず
ないんですが、C ライブラリを VBA で呼び出す事ができないという事は
普通にできないんでしたっけ？

直接にはできないというのであれば、C で VBA のための wrapper 関数を
書けばいいんじゃないでしょうか。
libpq の API を VBA から使えるように wrap するわけです。

VBA などに限らずとも、直接使用できないインタフェースである場合は、
wrapper を用意するのが技法としては一般的でしょう。

何カ月も悩んでいるのでしたら、こうした方が早いのではないかと…
