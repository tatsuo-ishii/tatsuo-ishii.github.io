みなさん、こんにちは。島村です。

「改訂版 PostgreSQL 完全攻略ガイド」に従って構築した環境があります。この
中の apache と PHP を使ったサンプル、pgimage-1.2 も順調に動いていまし
た。

この環境で <form> タグを使ったページを作っているのですが、日本語のデータ
を次のページにうまく渡す事ができません。デコードに失敗しているようにみえ
ます。英数字は記号文字なども含めてうまく渡っています。

ここで、/usr/local/lib/php3.ini を調べてみたのですが、シーラカンス本に従
ってファイルの末尾に

i18n.http_output = PASS
i18n.internal_encoding = EUC
i18n.script_encoding = AUTO
i18n.http_input = AUTO

と記述している分をコメントアウトして、ファイル中ほどに書いてある初期値の
ままの

i18n.http_output = SJIS
i18n.internal_encoding = EUC-JP
i18n.script_encoding = AUTO
i18n.http_input = AUTO
i18n.http_input_default = SJIS

にすると、日本語データは文字化けせずに、次のページに渡ったのですが、今度
は pgimage-1.2 で登録していた画像が表示されなくなりました。

両方の機能を一台のマシンの中で使いたいのですが、どのようにしたらいいでし
ょうか？

Plamo Linux 1.4.5
PostgreSQL 6.5.2
PHP3 3.0.12
Netscape 4.0.8
