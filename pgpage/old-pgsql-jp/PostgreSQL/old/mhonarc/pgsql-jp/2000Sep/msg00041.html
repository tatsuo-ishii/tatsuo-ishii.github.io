　馬場さん、こんにちは。初めまして。

> 使用しているメーラーは、「電信八号 Ver 321.2b6-stable」です。
> 少しまえから、ある特定の ML からのメールで、Subject が文字化け
> するようになってしまいました。

　原因の解説の前に、解決法を述べますが、最も簡単な解決法
は、電信八号を Version 32.1.3.0 020 にアップデートする事
です。

> > メーラーの作者に連絡してあげてみてください。
> と、いうことで、「電八倶楽部」さんにご報告したしだいです。
> これは、「電信八号 Ver 321.2b6-stable」のバグなのでしょうか？

　相手のメーラーまたは ML サーバの RFC1468 違反です。

> > おそらく、お使いのメーラーが Subject:を折り返し、それを元に戻す時の
> > 処理が適切ではないのだと思います。

　Subject を折り返した時の処理が不適切なのは相手です。

> > > の立ち上げ<ここに改行>

　ISO-2022-JP なら、GyRC = ESC $ B で JIS X 0208-1983 に
designate したら、改行の前に GyhC = ESC ( B で US-ASCII に
designate し直さなければなりません。

> > MIME関係の処理のバグの可能性もあります。

　MIME 関係の処理のバグも相手にあります。

> > > について

　GyRAGyRC は、ESC $ @ ESC $ B で、これは ISO-2022 では
JIS X 0208-1990 への designate ですが、ISO-2022-JP では
未定義です。ESC $ @ か ESC $ B か、どちらか単独でなけれ
ばなりません。

　というわけで、電信八号は、RFC にあまりに忠実であるが故
に、いいかげんな MIME 実装をしている人々に「そっちのバグ
なんじゃないか」という言いがかりをつけられる受難の電生を
送ってきたわけです。

　電信八号も世間にもまれて理想主義者から、少し現実主義者
になって Version 32.1.3 系で丸くなりました。(^^;

　では、Enjoy!
