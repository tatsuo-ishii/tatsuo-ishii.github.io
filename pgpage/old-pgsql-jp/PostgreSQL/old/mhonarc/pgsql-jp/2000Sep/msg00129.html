上記環境を前提としてお聞きしたいのですが、

以下の様な２種のプログラムを実行した場合の
解釈はそれぞれ正しい解釈なのでしょうか？

------ 例１ -----------------------
for ($i=0; $i<10; $i++) {
  $sql = "update aaa set bbb='aaa' where id = $i ";
  $result = pg_exec($connect_id, $sql);
}

------ 例２ -----------------------
pg_exec($connect_id, "begin");
for ($i=0; $i<10; $i++) {
  $sql = "update aaa set bbb='aaa' where id = $i ";
  $result = pg_exec($connect_id, $sql);
}
pg_exec($connect_id, "end");
pg_exec($connect_id, "commit");


例１の解釈
loop １回毎にcommitされる。（pg_execと同時にcommitされる）

例２の解釈
beginからendまでをトランザクションと見なしloopの間はcommit
されない。
