いつもお世話になっております。

表題の通りなんですが、PostgreSQL において、あるテーブル内の
レコード数に限度を設けたい場合にはどのような方法が考えられる
でしょうか? なお、限度となる数は動的に変更できる方が望ましいです。
また、PostgreSQL のバージョンはとりあえず 6.5.3 を想定しています。

ちなみにチケット予約システムのようなものを作ろうとしています。
発行できるチケットには限度がありますので、レコード数についても
同じく限度を設けたいというわけです。

普通に考えると、

  SELECT count(*) FROM 予約情報

してからもし、count(*) が MAX(チケット数) でなければ

  INSERT INTO 予約情報 VALUES(予約データ)

する、というような形になると思います。

もう少しお手軽(?)な方法として、sequence を利用する、ということも
考えたのですが、これは

  CREATE SEQENCE 予約数 MAXVALUE チケット数

のように属性として最大値を指定できるのはいいのですが、
動的に最大値を変更することができないようですし、チケットの
種類ごとに sequence を作成しなければならないので、あまり
現実的ではなさそうです。

また、同様にテーブルの"制約"についても

  CREATE TABLE 予約情報 (
    :
    , CHECK ( count(*) < チケット数)
  )

のようなことができませんので、これも使えそうにありません。

とりあえずは最初の方法を考えているのですが、
もし他にお勧めな方法がありましたら教えてください。
# ちなみに現状特に最初の方法で困っている、というわけではありません。
# 勉強させていただくつもりでお伺いしています。
## このような質問でも別に構いませんよね?
