岩永といいます。
初めまして。

ソフトウェアの検査手順書のデータベースを作成しているのですが
以下のようなエラーが起きて困っています。

最初に、データベース sample 中のテーブル test_procedure の
update をしようとしたところ、

ERROR: test_procedure: cannot extend. Check free disk space.

と言うエラーメッセージが出ました。
ここで、テーブル test_procedure に対して vacuum を行いました。

[postgre 53] psql -c 'vacuum test_procedure' sample
VACUUM
[postgre 54]

このあとデータベース sample に接続しようとしたところ、

[postgre 55] psql sample
Connection to database 'sample' failed.
FATAL 1:  cannot write init file -- descriptor length

というエラーメッセージが出て、接続ができない状態になりました。
postmaster 自体は起動している状態なのですが、これは何が原因なのでしょうか？
vacuum がまだ終了していないのでしょうか？

ためしに関係ないデータベース test に接続しようとしてみました。

[postgre 87] psql test

   type \? for help on slash commands
   type \q to quit
   type \g or terminate with semicolon to execute query
 You are currently connected to the database: test

他のデータベースには正しく接続できます。

postgreSQL のバージョンは6.5.3、OSは HP-UX 11.00です。
