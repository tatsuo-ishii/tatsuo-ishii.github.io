postmaster -B 128 -N 64 -i -o -F -S
で同時接続数を 64 本にセットして、1つの perl script 内から同一の
PostgreSQL に DBI/DBD::Pg で connection を張ってみたところ、
64本までは  ok、65 本目で
DBI->connect(dbname=basic) failed: Sorry, too many clients already at ./test.pl 
でエラーとなりました。

おそらく PostgreSQL の同時接続数設定を決めれば、あとはマシンのリソース
リミットまでいけるのではないでしょうか？
