sshはWin98からTeraTermProでやっています。
Clientからpsqlから上記を実行した結果です。

Testdb=> show client_encoding;
NOTICE:  Current client encoding is EUC_JP
SHOW VARIABLE
Testdb=> show server_encoding;
NOTICE:  Current server encoding is EUC_JP
SHOW VARIABLE

で、両方ともエンドーディングはEUC_JPです。
ちなみに　export PGCLIENTENCODING=SJIS をたたいてからpsqlを実行すると、日本
語が正常に表示されました。
それと、今度はサーバで、sshで接続した場合、

Testdb=> show client_encoding;
NOTICE:  Current client encoding is EUC_JP
SHOW VARIABLE
Testdb=> show server_encoding;
NOTICE:  Current server encoding is EUC_JP
SHOW VARIABLE

でした。そして export PGCLIENTENCODING=SJIS をたたいてからpsqlを実行すると、
今度は日本語が表示されませんでした。EUC_JPの環境でSJISで表示しようとしても当
然ですよね。

結果、サーバのpostgresqlにはEUC_JPでデータが作成されている。
Win98のODBCドライバで、EUC_JP --> SJIS に変換されるので、Access97では、正常
に表示される。
Win98のTeraTerm上では、export PGCLIENTENCODING=SJIS で EUC_JP --> SJIS に変
換されるので正常に表示される。
サーバ上では、EUC_JP --> EUC_JP だが、日本語の環境設定がまずいため？
という認識をいたしました。正しいでしょうか？
