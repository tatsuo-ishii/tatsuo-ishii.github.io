内田です。

時間まちなら、

 struct timeval sleeptime;
 sleeptime.tv_sec = XXX;
 sleeptime.tv_usec = YYY;
 (void)select(0, 0, 0, 0, &sleeptime);

とか nanosleep(2) なんかもありますね。

KUBO Takehiro さんは書きました:
># 頭の良いコンパイラなら、無意味な処理を削除したコードを掃くこともある
># かも。

さすがに時間待ちの空 Loop を置き換えることは不可能じゃないですか (^^;;
Loop の時間は実行環境依存ですから、完全に等価に他の命令で置き換えるのは
できないと思いますが。

それに、あえて無駄を承知で空ループを回すことも無いとはいえないでしょう。
使い道があるかどうか知りませんが、実行環境の処理速度に依存した待ち時間を
作りたい場合とか。
