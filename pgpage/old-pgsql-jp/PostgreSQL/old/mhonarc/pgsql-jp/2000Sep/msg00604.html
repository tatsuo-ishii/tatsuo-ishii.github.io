[pgsql-jp 17936] postgreSQL-7.0.2 の実行について

  • From: Kazuki Takeuchi <joker@pluto.dti.ne.jp>
  • Date: Fri, 29 Sep 2000 01:22:26 +0900
札幌の竹内と申します。

Kondara MNU/Linux 1.2（FTP版）を使用しております。

今までpostgreSQL-6.5.3を使用していたのですが
postgreSQL-7.0.2を使ってみようと思い、

http://www.sra.co.jp/people/t-ishii/PostgreSQL/7.0/install-basic.html
を参考にインストールを行いました。

tar玉を展開後、
./configure --enable-multibyte=EUC_JP
make all
make install

までは特にエラーも出ずに終了しました。

パスと環境変数の設定を行い、
initdbすると次のようなエラーが出てDBが初期化されません。

-----------------------------------------------------------
initdb: pg_encoding failed

Perhaps you did not configure PostgreSQL for multibyte support or
the program was not successfully installed.
-----------------------------------------------------------

ちなみに.bashrcの追加部分は
-----------------------------------------------------------
PATH="$PATH":/usr/local/pgsql/bin
export POSTGRES_HOME=/usr/local/pgsql
export PGLIB=$POSTGRES_HOME/lib
export PGDATA=$POSTGRES_HOME/data
export MANPATH="$MANPATH":$POSTGRES_HOME/man
export LD_LIBRARY_PATH="$LD_LIBRARY_PATH":"PGLIB"
-----------------------------------------------------------
のようになっております。

configureの際には--enable-multibyteしてるにも関わらず
上記のようなエラーが出て困っております。

どなたかご助言いただけると幸いです。
