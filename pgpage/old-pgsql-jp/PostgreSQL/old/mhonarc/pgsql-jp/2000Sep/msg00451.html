Masashi Ohba <ohba@intelight.co.jp> wrote:
;; どう直すのか分からないし、正常に動いた結果がどうなるのか
;; 分からない以上、java関係全然知らない私には、この説明では
;; 何のことやらって感じ。
;; 
;; 1.jserv.propertiesってどこにあるの？
;; 2.classpathの本当の設定って何?
httpd.conf(apach.con)より
  Include /usr/local/etc/apache/jserv/jserv.conf
jserv.confより
  ApJServProperties /usr/local/etc/apache/jserv/jserv.properties
jserv.propertiesより
  # CLASSPATH environment value passed to the JVM
  # Syntax: wrapper.classpath=[path] (String)
  # Default: NONE (Sun's JDK/JRE already have a default classpath)
  # Note: if more than one line is supplied these will be concatenated using
  #       ":" or ";" (depending wether Unix or Win32) characters. JVM must be
  #       able to find JSDK and JServ classes and any utility classes used by
  #       your servlets.
  # Note: the classes you want to be automatically reloaded upon modification
  #       MUST NOT be in this classpath or the classpath of the shell
  #       you start the Apache from.


;; 3.それってOSによって違ってたりしないの？
