	当方でも、フロントエンドを私のマシンにして、バックエンド
を学生のマシンにして、クライアントサーバー型のアーキテクチャで
PostgreSQL v6.1.1 が動きました。意外と簡単で、クライアントで
psql を起動する前に、PGHOST 環境変数を設定するだけで良いんですね。
でも、実は私のマシンに v6.1.1 をインストールするのにずいぶんと
手間取ってしまって、無理矢理ソースを変えて通してしまったんですが。
	これから、java で PostgreSQL を動かすことに学生と一緒に
トライしてみます。とにかく、来年、まともに java でデータベース
インターフェースを作るための予備段階として、ノウハウをためる
ことが目的です。何かと、分からないことがあって質問するかと思いま
すが、よろしくお願いいたします。
	それと、今のところ、PostgreSQL はクライアントも Unix 系の
マシンに限られているように思うんですけれど。psql に似たクライアント
アプリケーションを java で作ったら、windows や Mac からも PostgreSQL
にアクセスできるようになるでしょうか? psql のソースがあれば、
なんとかできるかな。psql って、まず、最初に自分のマシンの postmaster 
を見に行って、それから、バックエンドに通信をはじめるのでしょうか?
それとも、psql は、最初からサーバーマシンの postmaster と通信するん
でしょうか?
	野望だけでかくて、実技がついていかないような気もしますが、
石井さん、皆さん、これからもなにとぞよろしくお願いいたします。
