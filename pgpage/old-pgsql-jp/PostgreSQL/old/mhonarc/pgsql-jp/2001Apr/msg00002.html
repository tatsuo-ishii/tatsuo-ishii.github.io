事を試みています。
サーバ：
    VineLinux2.1+Apache+Postgresql7.0.2
    IP：172.16.1.1
クライアント：
    Windows98
    IP：172.16.1.2
です。
HTMLとJavaのソースは以下の通りです。
---HTML（一部）---
<body>
<applet
 code=jdbc.class
 width=400
 height=300>
 <param name=archive value="postgresql.jar">
</applet>
</body>

---Java（一部）---
　private void init(){
　　Choice chc.setBounds(new Rectangle(59, 22, 209, 18));
　　try{
　　　Class.forName("org.postgresql.Driver");    //※1
　　　Connection conn = DriverManager.getConnection("jdbc:postgresql://172.16.1.1/testdb", "miura", "masamasa");
//※2
　　　Statement st = conn.createStatement();
　　　String sql = "select * from login";
　　　ResultSet rs = st.executeQuery(sql);
　　　while(rs.next()){
　　　　chc.addItem(rs.getString(1));
　　　}
　　}
　　catch(Exception ec){
　　　chc.addItem("Error");
　　}
　　this.add(chc, null);
　}
です。
正常にDBに接続してSQLが実行されればコンボボックスに一覧表示される
のですが、異常の場合は”Error”が表示されるアプレットです。
このｈｔｍｌをクライアントからアクセスすると”Error”が表示されます。
URL:http://172.16.1.1/~miura/
ｔｒｙの一部分をコメントアウトして実行してみると、どうやら※2の所で
catchに飛んでいるようです。（※1は正常）
”miura”はLinuxのユーザでPostgresqlのユーザでもあり、testdbのオーナーです。
パスワードの部分の”masamasa”は”miura”がLinuxにログインするときの
パスワードでいいのでしょうか？
サーバで同じようにアクセス（自分自身）しても”Error”が表示されます。
ちなみにサーバで接続文字列を”jdbc:postgresql://localhost/testdb”にすると
一覧表示がされて、うまく行きます。
クライアントからpingをすると応答は正常です。

どのようにしたらクライアントからサーバのDBにアクセスできるのでしょうか？
