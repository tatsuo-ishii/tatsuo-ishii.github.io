PostgreSQL7.1に付属のInstallってファイルの

If You Are Upgradingって個所に

旧バージョンのデータのバックアップ
旧バージョンを停止
旧バージョンのディレクトリをリネーム
新バージョンをインストール
新バージョンのinitdbで初期化
新バージョンを起動
新バージョンへデータを書き戻す

って手順が書いてます。

新しいのをインストールしてって所ですることが
次のInstallation Procedureに書いてます。

基本的にやることは
インストール元とインストール先のディレクトリの
所有者やグループはちゃんと設定した上で
postgresユーザーが

./configure
gmake check
gmake install
gmake install-all-headers

したうえで
上記のinitdbをする部分へ戻ればいいです。

INSTALLってファイル名だから、当然インストール作業が
書いてるわけで、サッとでも目を通しましょうね

旧バージョンでrpmを使ってるとインストール先とかが
違いそうだけど、ソースのドキュメントがディストリビューション
固有のことまで言及してることはそう無いし
新バージョンをrpmで済ます方法は知りません。
Linux使ったこと無いし。
