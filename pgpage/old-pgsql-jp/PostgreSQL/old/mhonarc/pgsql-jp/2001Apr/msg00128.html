> ところで石井さんのメールの末尾の

> > TIP 4: Don't 'kill -9' the postmaster
> 
> ってなんですか？

man kill してみてくださいまし。


これは、プロセスを無差別に殺すなっていうことでしょう。

kill -9 すると、問答無用でプロセスを殺しにかかるので、
プロセス(この場合 postmaster)が、
メモリー上や一時 file に抱えてる data を本来あるべき場所に
格納する暇を与えられずに、強制終了させられてしまうということです。

DB でこれやるのは、考えるだけで冷や汗もんですね。

当然、正常終了前に OS を落とすとか、電源を落とすとかもだめです。

もし不幸にしてやってしまったら、柏手を打つとか、十字を切るとか…。
