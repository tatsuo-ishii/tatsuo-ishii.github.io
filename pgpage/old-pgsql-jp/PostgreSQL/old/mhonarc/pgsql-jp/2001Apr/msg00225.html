お二人のおっしゃっていることは，私もおおむね賛成です．

ただ，configure/makeまでは一般ユーザ権限で行って，make installの
時だけrootになるというのは，PostgreSQLに限らず他のプログラムの
インストールでも結構一般的なことではないかと思うのですが．
# もちろん，そうしない人もいるでしょうが．

7.1に付いているドキュメントのinstall-procedure.htmlでも
----------------------------------------------------------------------
4.Installing The Files
 :
Make sure that you have appropriate permissions to write
into that area. Normally you need to do this step as root.
Alternatively, you could create the target directories in advance
and arrange for appropriate permissions to be granted.
----------------------------------------------------------------------
となっているようなので，make installをrootでやること自体は
好ましくはない，ということでもないと思います．

一方，postmaster実行時の権限は，もちろんpostgres等のユーザを作って
その権限で行うべきですし，そもそもpostmasterはrootで実行しようと
しても動かないようになっているようですし．
