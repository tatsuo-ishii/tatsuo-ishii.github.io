マシンのファイルシステムのブロックサイズではなく、PostgreSQLのデータベー
スのブロックサイズです（とか言い切ってますけど、100%確実な情報ではありま
せん）。Oracleでも指定するdb_block_sizeと同等のものだと推測します。

何かのコマンドか設定ファイルでブロックサイズの値を知る方法があるかもし
れませんが、私は知りません。

 *

マニュアルを検索しても、BLCKSZの説明がありませんでしたので、ソースを検索
すると、src/include/config.hの123行目あたりに

#define BLCKSZ  8192

とありました。多分これだと思います。
また、123行目あたりの前にBLCKSZの説明がありますのでご覧下さい。

You can set it bigger if you need bigger tuples

と書かれているので、変更可能な値ですね。initdbの際にそのデータベースのブ
ロックサイズが決まるようです。
