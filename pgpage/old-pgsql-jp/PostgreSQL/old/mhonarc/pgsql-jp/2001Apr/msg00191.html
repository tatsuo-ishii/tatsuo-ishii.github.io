松嶋です。

PostgreSQL 7.1 に WAL が実装されたので、その速度
面での効果を確かめるべく、簡単な、ほんとに簡単な
測定を行ってみました。

http://www.itboost.co.jp/inst/inst_16.php

体感速度に毛の生えたぐらいのテストですが、傾向ぐ
らいはわかるのではと思い、実験し、まとめてみまし
た。ご覧いただき、ご意見・ご指摘などいただけたら、
と思います。

傾向としては、7.1 の方が、7.0.2 より
 ・ 同時に走っているトランザクション数が多いほど、速い
 ・ データベースファイルのサイズが大きいほど速い
という感じでした。
# もしかして、当たり前の結果でした？

ついでに MySQL との比較もやっているのですが、あ
んまり比較になっていないのかも、です。結果を見る
と MySQL と PostgreSQLが等価なことをやってるよう
にはみえませんでした。数字が違いすぎます。そのあ
たりについても、知っている方がおられましたらよろ
しくお願いします。


測定に使ったプログラムですが、Thread を使いたか
ったということもあり、C で書きました。が、libpq 
とか libmysql* を勉強している暇はありませんでし
た。

ということで、psql とか mysql に大量のSQLを書い
たファイルを食わせるコマンドを systemで発行する
というダサい測定方法になってしまいました。一応、
system によるプロセス発行のオーバーヘッドも測定
して引いてはいますが…。

速さの傾向を見る程度、という感じで捕らえていただ
ければ、と思います。^^;
