はじめまして、ヤナリと申します。

  Redhat Linux 7.0.1J　+ Apache 1.3.17 + PHP 3.0.18 + PostgreSQL 7.0.3

ってな、環境のシステムのプログラムを引き継ぐ事になってプログラムの改良を
行っているのですが、設計段階では予想もつかない長いSQL文(textファイルで80KB)が
生成されてpg_execの処理で約40分くらい掛かってしまいます。
SQL生成部分のアルゴリズムは見なおして、処理時間がかからないようにするつもり
なのです。

ボタン
をクリックされると、SQL実行状態のプロセスが何個も残っていきます。
小さいプロセスも積もればエライ事になりかねない・・・php3の処理が終わっているなら
postgresの処理も止めていたいってのが心情です。
それで、pg_execの処理を中断できれば非常に安心なシステムになりそうなのですが、
中断する方法ってないでしょうか？

set_time_limit(),ignore_user_abort(),register_shutdown_function()とかを使って 
なんとか
できないかと思ったのですが・・・pg_exec実行中はタイムアウトとかも発生しないよ

無理なら無理だと、顧客に納得してもらえば良いのですが・・・何か良い方法があれば
教えてください。
