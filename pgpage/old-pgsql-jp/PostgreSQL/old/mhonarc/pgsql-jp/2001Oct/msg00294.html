小泉＠コイズコムデジタルワークスと申します。

PostgreSQLを6.5.3から7.1.xにアップデートしたところ、次のような現象に
悩まされております。

【背景】
create table prefecture2 (
  code int2,
  name text,
  ship int2
);

というテーブルに、

1	北海道	11
2	青森県	12
3	岩手県	12
・
・
・

というように、47都道府県のデータが入っています。
これを、

select * from prefecture where name = '北海道';

とすると、どういうわけか47都道府県分の全レコードが結果として返ってき
てしまいます。


select * from prefecture where code = 1;
select * from prefecture where ship = 11;

これらでは問題ありません。

他のテーブルも調べてみましたが、where句でtext属性のフィールドを使うと、
同じ現象が起こります。私がアクセスできるサーバ4台すべてで発生していま
す。

4台の環境は、

RedHat6.2J（カーネル2.2.19-6.2.1）
PostgreSQL 7.1.3 および 7.1.2
PHP 4.0.6
Apache 1.3.17

……といったところです。

解決策をご教授いただければ幸いです。よろしくお願いします。
