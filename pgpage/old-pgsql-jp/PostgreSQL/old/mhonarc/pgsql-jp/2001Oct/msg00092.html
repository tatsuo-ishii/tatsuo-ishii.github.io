はじめまして。hirota と申します。

今年の１月にこのメーリングリストに登録して以来、いつも大変参考
にさせていただいております。
おかげさまで Apache 1.3.20 + PHP 4.0.6 + PostgreSQL 7.1.3 によ
るWebシステムを業務用としてなんとか構築することができました。

さて、さっそくですが標記の件についてご質問させて頂きます。

データベースをWindows側からクリックひとつでメンテナンスできれば
業務担当者の作業を軽減できるかなと思い、libpqを使用した簡単なメ
ンテナンス用アプリをVC++で組み始めました。作業は順調に進んでいた
のですが、途中で日本語対応の必要に迫られ、谷山様がポストされてい
た[pgsql-jp 19539]やインターウィズ様
(http://www.interwiz.koganei.tokyo.jp/index.html) が提供されてい
るpostgresql-6.5-win32jp.patchなどを参考にして、マルチバイトオプ
ションを付けてlibpqをコンパイルしようと、ここ１週間ほど試行錯誤
を続けてきましたが、どうしても通すことができません。

SOGA Telecom 様 (http://hp.vector.co.jp/authors/VA023283/) が提
供されているPostgreSQL-7_1_3_win32_bin_rv08.zip内のlibpq.dll、
libpqdll.libを使用すれば問題なく日本語を扱うことができるのですが、
業務用のシステムはできるだけソースからビルドしたものを使いたいと
いう、スキルの伴わないこだわりもあり、なかなかあきらめることがで
きせん。

どなたかマルチバイトオプション付でコンパイルに成功された方はいら
っしゃらないでしょうか？何かヒントでもいただければ幸いです。よろ
しくお願い致します。


使用環境：
Windows2000 SP2
Visual C++ 6.0 SP5
PostgreSQL 7.1.3
