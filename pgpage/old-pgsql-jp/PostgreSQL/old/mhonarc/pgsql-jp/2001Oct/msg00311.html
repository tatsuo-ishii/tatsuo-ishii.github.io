[pgsql-jp 23581] ODBC Driver 07.01.0006 日本語版の使用例について

  • From: zero4649@mx1.freecom.ne.jp
  • Date: Wed, 31 Oct 2001 22:31:41 +0900
お世話になっております。

VC++6.0で作成したアプリケーションからODBC経由でPostgreSQLに
アクセスしたいのですが、適当なサンプルもしくは解説ページが
ありましたら教えて頂けないでしょうか。

ODBCの方はhttp://www.interwiz.koganei.tokyo.jp/software/PsqlODBC/から
07.01.0006をインストールし、Accessでリンクしてみて動作確認できたのですが、
VC++で作成したアプリケーションからのアクセスがうまくいきません。

具体的には

CString ConnectStr = "ODBC;DSN=postgresql;UID=user1;PWD=xxxx;";
if (!m_db.OpenEx(ConnectStr, 0)){  <-- ODBCのダイアログ出現
  TRACE("OpenDB() Error !\n");
  return -1;
}
TRACE("OpenDB() OK !\n");

のOpenExで、ODBCマネージャーのダイアログが出てしまうのと、

m_rs = new CRecordset( &m_db );
CString strSQL = _T("SELECT * FROM code1");
if (!m_rs->Open(CRecordset::snapshot, strSQL, CRecordset::none)){
  TRACE("OpenRS() Error !\n");
  return -1;
}
m_rs->MoveFirst();
CString strValue;
while (!m_rs->IsEOF()){
  m_rs->GetFieldValue("Code1", strValue);  <-- エラー
  TRACE("Code1 = %s\n", strValue);
  m_rs->MoveNext();
}

で、m_rs->Openは成功しているようなのですが、一回目のGetFieldValueで
エラーとなってしまいます。


※ODBC Driver 07.01.0006 日本語版ですが、Win2000では１秒もかからずに
リンクファイルを開けるのに、Win98からだと、10秒ぐらいかかってしまいます。
ハード構成が異なるので単純には比較できませんが、Win2000ではほとんど
気にならないDISKのアクセスが、Win98だとものすごくIOが発生しているようです。
（Win2000:CPU=PIII 700,MEM=128, Win98:CPU=K6-2 400, MEM=392)

過去ログに同様な話題がありましたが、結局どのように対処すればよろしいの
でしょうか？
