はじめまして
藤本＠JDCです。

現在、PostgreSQL7.1.1を使用しているのですが、Perl用インターフェースであるPg
に
ついて分からない点があります。(RedHat7.0Jにて使用）

下記のような２つのテーブルから select句を使ってデータを取り出そうとしている
のですが
うまくいきません。

○テーブル１
col1(varchar) | col2(varchar) | col3(varchar)
---------------------------------------------------------------
  0000001             01               1231234

○テーブル２
col4(varchar) | col4(varchar)
------------------------
  1231234           12345

○SQL文（col3とcol4がテーブル１とテーブル２を結ぶキーになっています）
select col1,col2  from テーブル１,テーブル２  where  col3 = col4  and col4
like '12%'

現象としては、上記のSQL文をPerlにてPgを使用して実行すると、戻り値が
"PGRES_TUPLES_OK"
以外になり、うまくいきません。
同様のSQL文をPHP4もしくは、psqlにて実行するとうまく検索できます。

過去の記事などを検索しましたが、解決策をうまく見つけることが出来ませんでした
ので、
できればご教授ください。

○Perlソース
$sql = "select col1,col2  from テーブル１,テーブル２  where  col3 = col4
and col4 like '12%'";
$res = $conn->exec($sql);
if (PGRES_TUPLES_OK  == $res->resultStatus) {
    print "SELECT正常終了\n";
} else {
    print "SELECT異常終了\n";
}
