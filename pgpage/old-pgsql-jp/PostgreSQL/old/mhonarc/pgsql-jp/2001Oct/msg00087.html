初めまして、永安と申します。

複数の列を合わせて主キーとして利用する方法が分かりません。

具体的には、データを登録される人間にユニークなIDなどは振っておらず、
フルネーム、国籍および性別という属性をまとめて主キーに設定したい
といったような状況です。

先日、RDBMSの本を見ていたところ、「複合キー」という概念を知り、
シーラカンス本にも

「主キーはひとつまたは複数の列から成り、それらを連結した値は
そのテーブルの中で唯一であるという制約を受けます」

という記述が p.130 にあります。

ので、可能だと思うのですが、具体的に発行すべきSQLが分かりません。

サーチエンジンで調べてみたところ、Oracle には

alter table TABLENAME add primary key (
	FIELD_1,
	FIELD_2,
	...
);

という方法があることだけは分かりましたが、同じようなSQLを発行してみても
PostgreSQLでは、

snaga=# create table test ( id_1 int primary key, id_2 int not null );
NOTICE:  CREATE TABLE/PRIMARY KEY will create implicit index 'test_pkey' for table 'test'
CREATE
snaga=# ALTER TABLE test add primary key ( id_1, id_2 );
ERROR:  ALTER TABLE / ADD CONSTRAINT is not implemented for that constraint type.

となってしまいます。

PostgreSQLのマニュアルも読んでみたのですが、イマイチ方法が分かりません。

あるいは、「create index unique」を使うのでしょうか？

どなたかご存知の方がおられましたらお教えいただけると助かります。
よろしくお願いします。
