>
> 自分は7.1.2しか持っていないのですが、
> $ ./configure --help | grep connection
>   --with-maxbackends=N    set default maximum number of connections [32]
> ということで、./configure実行時に渡すのかなと。
> 7.1.3のconfigureコマンドのヘルプを確認されてみてはどうでしょうか？

ご指摘の通り、--with-maxbackends=N オプションをつけて、configure を実行し、
インストールを行うと、MAX_CONNECTIONSの上限を変更できました。

postgres.conf には、反映されませんが、--with-maxbackends=N の Nの値を
大きくすると、postmaster起動時、共有メモリや、セマフォに関するエラーが出るの
で、
MAX_CONNECTIONSが増えているのだと思います。
パラメータを変えていろいろ試してみます。
