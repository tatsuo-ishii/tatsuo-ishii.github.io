石井です。

Zend エンジン組み込みの PHP 4.0b1 を試してみました。例によって 
PowerBook 2400c+LinuxPPC R4 です。

o インストール

PHP4 は PHP3 と共存できないので、まず PHP3 を無効にします。
すでに稼働中の apache-1.3.6(DSO)+PHP-3.0.9+PostgreSQL 6.5を停止、

	LoadModule php3_module        libexec/libphp3.so

をコメントにし、

	AddType application/x-httpd-php3 .php3
	AddType application/x-httpd-php3-source .phps
を
	AddType application/x-httpd-php .php3
	AddType application/x-httpd-php-source .phps

に変更する。

PHP4 のインストール自体はPHP3と変わりません。

./configure  --with-pgsql --with-apache=../apache_1.3.6 --enable-track-vars --with-apxs=/usr/local/apache/bin/apxs

を実行後、make, make install するだけです。
/usr/local/apache/libexec/libphp4.so がインストールされるほか、
httpd.conf に自動修正が入ります。

o テスト

phpinfo(); を実行してみました。表示内容はPHP3とあまり変わりません。
Zend のロゴが誇らしげです。

o PHP3との互換性

詳しいことはドキュメントを見て頂くとして、PostgreSQL インターフェイス
が気になるので pgimage を試してみましたが、とりあえずそのまま動きまし
た(ソースを見ると、PostgreSQLインターフェイスは PHP3 のものをほとんど
そのまま使っているように見えます)。EUCコードなら日本語も通るようです。
また、気のせいか、PHP3 より高速になっているような気がします(Zend のデ
バッグフラグはonのまま)。

-------------------------

PHP4 は Zend の採用もさることながら、セッション管理のサポートが大きい
と思います。今後が楽しみです。
