必要なのでしょうか。

具体的には
 $ psql -h localhost testdb
と入力するとセッションを開始しますが
 $ psql -h myHostname testdb
では  User authentication failed と表示されて
セッションを開始できません。

myHostname は localhost と同じマシンであり
アドレス 192.168.1.1 を割り当てています。
もちろん ping myHostname での疎通は OK です。

[作業した事]
*Debian2.1(slink) に入っている Postgresql 6.3.2-15 を導入しました。
*ユーザpostgresで createuser し、そのユーザでデータベース testdb を
 作成しました。
*ネットワークから利用したいために -i オプション付きで起動するよう、
 設定ファイルを書き換えました。(ps の出力で確認したところ -i 付きで
 起動しています。)
* pg_hba.conf には
    host  all  192.168.1.0  255.255.255.0   trust
  を書き加えてあります。

  同じように User authentication failed になった。
