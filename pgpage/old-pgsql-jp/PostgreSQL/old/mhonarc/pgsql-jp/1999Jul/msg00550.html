  標準です。
  PostgreSQLが*正式に*定義しなおしているsprintf代用品はないので、とりあえず
標準品を使いました。

> sprintf()は、FreeBSDでは
>
>      These functions return the number of characters printed (not
> including
>      the trailing `\0' used to end output to strings).
>
> となってます。
> また、NEXTSTEP3.3Jという4.3BSDのちょっと古いものをベースにしていたOSでは
>
> 	RETURN VALUE
> 	     Printf, fprintf, and sprintf return a negative value for un-
> 	     successful calls and stores an error code in errno, other-
> 	     wise  returns zero.
>
> となってます(NEXTSTEPは、古いんで、さすがに無視してもらってもいいの
> ですが...)。

  貴重な情報ありがとうございます。

> よってポータビリティを上げるためには、sprintf()を直につかうのではなく、
> もうちょっと移植性のあがる方法で使った方が良さそうです。

  sprintfにいくつかのバージョンがあることは知っていたので、明らかに手抜きで
した。PostgreSQLにはSPRINTF_CHARマクロがあるのでこれで判断してもいいのです
が、ここで1回使っているだけなので2行に分けることにしました。

  どうもありがとうございます。
