始めまして。川口と申します。
PostgreSQLというフリーのデータベースがLinux上で
使えるということで、自作マシンにインストールして試してます。

環境は
Vine Linux 1.1CR
PostgreSQL 6.4.2
を
CPU:K6 233
MEM:192M　　の様なマシンで動かしています。

PostODBCについて質問します。
このマシンをサーバーとしてNT4.０のクライアントに
PostODBC V06.40.0004をインストールしてAccessから
テーブルリンクして使用するつもりだったのですが、
リンクしたテーブルに２３００件位のデータを作成した所、
Accessからそのテーブルを開くのに１分近く掛かってしまいます。
立て続けに同じテーブルを開くと３分位かかります。

ちなみにテーブルレイアウトは
create table 製品情報 (
    製品番号    varchar(20),
    版数          float8,
    製品名称    varchar(80),　
    SBU          varchar(10)
    );
データには半角カナ文字も含んでいます。
３行だけのデータでも１０秒くらい掛かります。
　　　　　　　　　　　　　　　　　
PostODBCはこんなにも遅いものなのでしょうか？
どなたか教えて下さい。

テーブル名・カラム名を英字にしたりもしたのですが
変化無でした。
PostODBC V06.40.0004はパッチ済みのものです。
