> PostgreSQLとApacheを別々のサーバに入れて連携を取りながら
> 運用していくことはできるのでしょうか？
> それともやはり二つのサービスを一つのサーバでやる必要がある
> のでしょうか？

原則的には、別個の machine で運用されていても問題ないはずです。だから、
極論を言えば、同一 network に無くても大丈夫ということになります。
#さすがに TCP/IP network 以外は難しいと思いますが :-)

ただ、これはあくまでも原則論です。実際に検討する場合は PostgreSQL と 
apache の間を、とりもつ software について、個別に実装を検討する必要が
あります。

今迄、私が使ったものは、Pg.pm と PHP/FI ですが、それらについての私見は、
次のようになります。

-- ここから私見
・PHP/FI(多分 PHP も可):pg_connect という関数で、hostname と database 
が指定出来るので、別個の machine でも問題は無いはず。
#勿論、PostgreSQL の access 権限とかは順当に設定されている場合を仮定し
#ています

・Pg.pm:(少なくとも私が使っている 6.4.2 附属のものは) hostname が指定
できないので、別個の機械で動かす場合は、手を加える必要がありそう。

・(使ったことはないですが) libpq などを使った C interface:多分問題無い
でしょう。
-- ここまで私見
御参考になれば幸いです。

ただ、あくまでも私見です。大きな勘違い等があるかもしれません。詳細につ
いては、石井さんの web page(http://www.sra.co.jp/people/t-ishii/) とか、
PHP の日本語 site(http://www.cityfujisawa.ne.jp/%7Elouis/apps/phpfi/)
とかを、あたられるのが宜しいかと思います。
