山崎です。

今まで自分のサーバでPostgresを使っていましたが、マシンが一台
余りましたので、別サーバからアクセスさせてみようと挑戦中です。
ちなみに、6.4.2を使っています。

構成としては

   DB    -------   CGI
といった形でです(192.168.25.2(DB) -- 192.168.25.1(CGI))。
DBマシン側にはsampleというDBをpostgresユーザで作成しました。


DBマシンの方の/usr/local/pgsql/data/pg_hba.confに
host         client      192.168.25.1 255.255.255.0    trust
を付け加えたのですが、CGIマシン側で、postgresユーザにsuし、
%psql -h 192.168.25.2 -d sample
とやっても
Connection to database 'sample' failed.
Missing or mis-configured pg_hba.conf file

とメッセージが出てしまいます。
pg_hba.conf以外で何か作業をする必要があるのでしょうか？
