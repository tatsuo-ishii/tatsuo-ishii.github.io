宮本@HDEです。

Postgres(SQL)の型と、HTMLのフォームの種類とを
うまく連係する方法をどなたかやっておられませんか？

例えば、

miyamoto=> \d test

Table    = test
+----------------------+-------------------------+-------+
|        Field         |           Type          | Length|
+----------------------+-------------------------+-------+
| a                    | varchar()               |   255 |
| b                    | int                     |     4 |
+----------------------+-------------------------+-------+

というテーブルを、

miyamoto=> \d test

Table    = test
+------------------+---------------------+-------+---------+----------+
|      Field       |         Type        | Length| formtype|formLength|
+------------------+---------------------+-------+---------+----------+
| a                | varchar()           |   255 |     text|        30|
| b                | int                 |     4 |     text|        10|
+------------------+---------------------+-------+---------+----------+

みたいにして、
<INPUT TYPE=text NAME=a SIZE=30>
<INPUT TYPE=text NAME=b SIZE=10>
を生成する時に
ひっぱって来れると非常に便利（美しいとはいいませんが）
なのですが。

今は、「HTMLフォーム定義用テーブル」を別途作って
リレーションさせて対応しているのですが、
それだと入り組んで分けわかない状態です。

確かテーブルの横にこういうのを追加するのってできたような気がするのですが、
こういうのをやっておられる方っていないんでしょうか？

それとも、mysqlみたいにDreamweaverで、postgres用configとかを作ったほうが
実用的なのかしらん。（こちらも誰か作っていませんか？）
