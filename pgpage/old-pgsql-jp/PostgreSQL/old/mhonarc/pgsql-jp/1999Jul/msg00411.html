[pgsql-jp 9608] when is currval('sequence') available.

  • From: CHOI Kyong-Rok <rok@itc.keio.ac.jp>
  • Date: Fri, 23 Jul 1999 10:30:36 +0900
こんにちわ.

create sequence hoge; で作った sequence に対して, psql で

select currval('hoge');

した時に, ちゃんと値が得られる場合と得られない場合があります. 

psql 起動後, 一度でも select nextval('hoge'); を実行していると 
currval('hoge') の値を得ることができますが, 一度も 
nextval('hoge') を参照していない場合には

ERROR:  hoge.currval is not yet defined in this session

となってしまいます.

こういうものですか？

PostgreSQL は, 6.5 を Digital UNIX 4.0e で

./configure  --with-CC=cc --without-CXX \
--with-template=alpha_cc --with-mb=EUC_JP --with-perl

として作ったものを使っています. 

-----[ サンプルのセッション始まり ]-----
rok@ginger:626 %psql
Welcome to the POSTGRESQL interactive sql monitor:
  Please read the file COPYRIGHT for copyright terms of
POSTGRESQL
[PostgreSQL 6.5.0 on alphaev56-dec-osf4.0e, compiled by cc ]

   type \? for help on slash commands
   type \q to quit
   type \g or terminate with semicolon to execute query
 You are currently connected to the database: rok

rok=> select currval('hoge');
ERROR:  hoge.currval is not yet defined in this session
rok=> select nextval('hoge');
nextval
-------
      2
(1 row)

rok=> select currval('hoge');
currval
-------
      2
(1 row)
