こんにちわ.

REALbasic は PostgreSQLのフロントエンドに使えるかという話です. 

日本語版の REALbasic が発売されて, 早速買ってきました(Standard版). 

これまでの広告とか箱に書いてある文句を信用すると, Database にア
クセスできるのは Professional 版だけだと思っていましたが,
Standard 版でも雰囲気くらいは楽しめるようです.

＃マニュアルに「Postgreデータベース(Linux)を開きます」と書いてあ
＃りますが, もちろん, Linux でなくても大丈夫でした. 

Standard 版の制限としては, 使えるデータの件数が 50 件までという
のと, アプリケーションがビルドできない(統合環境からは実行できま
す)というもので, チョロっといじった限りではわりと簡単に使えそう
です. 

私は, PostgreSQL 6.3.2 と組み合わせて試してみましたが, ちゃんと 
select も update もできますし, なかなかいい具合です. 

データベースに, 使える型は smallint, integer, varchar, float,
double, boolean, date, time なんですが, PostgreSQL の text がそ
のままでは, varchar として認識されないので, NativeValue を使って, 
コード変換する必要がありました(コード変換はTextEncodingConverter
で簡単にできます).

ただ, それだけのために Standard版(19500円)と Professional版(約5
万円)の差額を出す気になるかというと, うーん, 個人の趣味でやるに
は高いかなぁという感じでしょうか. 

rok
