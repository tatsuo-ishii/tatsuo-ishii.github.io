> 北村＠zoozee です。

> 移植性を考えると、date + start のが良いかな？と思ったのですが、
> date型 + time型 :: timestamp型 って、方法として間違ってるのでしょうか？
> (結果的には OK なのですが、ちょっと疑問に思ったので)

user manual には、+ の右辺にはintervalをおく３種類しか書いてませんが、
pg_operator を見ると、datetime_pl・timedate_plということでoperatorが
定義されてるので、dateとtimeを+しても問題ないと思います。
（7.1.2で既にありました）
ちなみにdate+time=timestampなので、キャスト不要のはずです。

user-7.2-A4.pdfを見てますが、このTable 4-16.、 time - time が抜けてる
一方、interval - time つう意味不明な演算が入ってますね。
