いつもお世話になっております。
ひらのと申します。

現在、PostgreSQL7.1.3を利用しているのですが
最新版の7.2への移行のために、データのバックアップ、
リストアを行なっております。

そのために以下の作業を行ないました。

１．7.1.3の元データをバックアップ
  $ pg_dumpall -o > backup.txt

２．7.2のインストール
  $ configure --enable-multibyte=EUC_JP
  $ gmake
  $ gmake install
  $ initdb -E EUC_JP

３．バックアップデータのリストア
　$ psql -e template1 < backup.txt

ここで「３」を実行すると、以下のメッセージが出力され
バックアップデータが反映されません。

COPY "test_table1" WITH OIDS  FROM stdin;
ERROR:  copy: line 47, Invalid EUC_JP character sequence found 
(0x8032)
lost synchronization with server, resetting connection
COPY "test_table2" WITH OIDS  FROM stdin;
ERROR:  copy: line 3816, Invalid EUC_JP character sequence found 
(0xd200)
lost synchronization with server, resetting connection


インストール時のオプションの指定が間違っているか、
もしくは足らないのか、途方に暮れてます。
申し訳ありませんが、助言のほど宜しくお願い致します。
