USE の あさの と申します。

ほとんど、FAQ 的な質問で申し訳ありません。
PostgreSQL は初心者です。

PostgreSQL 6.3.2 を使える環境にして、いざテーブルを
定義しようとしたところで、驚くようなメッセージが
表示されたので、そういうものなのかどうかを教えて下さい。

やったことは、

mydb=> create table foo (a numeric(10,3));
ERROR:  NUMERIC precision 10 must be 9

おや？

mydb=> create table foo (a numeric(9,3));
ERROR:  NUMERIC scale 3 must be zero

おや？

で、

PostgreSQLのマニュアルを読んでみると、
数値のようなデータ型は INT とか FLOAT とか
のようなマシン依存的な型のサポートがるの
のみのようなので、それらに置き換えられる
のは、百歩譲って理解できるのすが、どうして、
FLOAT 型系に置き換えるのではなくて、
INT型系に置き換えるのでしょうか？

あるいは、この現象はなんかの間違いで、
制御可能なんでしょうか？

正直にいえば、最初にやりたかった、

create table foo (a numeric(10,3));

を自然に処理できるようにしたいのですが、
データ型の再定義みたいな方法はないの
でしょうか？
