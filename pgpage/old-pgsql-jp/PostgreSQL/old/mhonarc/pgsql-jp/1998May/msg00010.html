> Visual C++ で作成したプログラムで ODBC を使って PostgreSQL
> に接続しています。しかし Recordset を snapshot で Open
> しようとすると
> 「ODBCスタティックカーソルにはスナップショットのサポートが必要です。」
> というメッセージボックスが出力されてテーブルのエレメントを
> 取得することができません。
> また dynaset で Open すると X 印だけのメッセージボックスが
> 表示され同じくエレメントを取得することができません。
> 
> Recordset を forwardOnly にするとテーブルのエレメントを取得
> することができるのですが、これは PostgreSQL の或いは使って
> いる ODBC ドライバの仕様なのでしょうか。
> それとも PostgreSQL をコンフィグレーションすると snapshot や
> dynaset でも利用できるようになるのでしょうか。

  PostgreSQL ODBC も PostODBC も（以後 PostODBC）、この辺の実装は完全
ではないです。ODBC API では SQLSetScrollOptions などが対応するのですが、
PostODBC ではこの API は実装していません。ちなみに PostODBC のカーソル
のデフォルト動作は（と言うか唯一可能は動作は）、「スナップショットでカー
ソル移動は自由」です。
  この辺は DAO（を使ってるのですよね？）の内部仕様にも関係してくるので、
なぜスナップショットではなくてダイナセットのフォワードオンリーとなるの
かは解りませんが、何となく PostODBC の問題の様な気もします。

  ちょっと調べてみますけど、期待はしないでください。
