はじめまして、おいかわ＠いーえー といいます。

今まで NT SQL サーバ上に貯まっていただけの、20 万件の
データを活用しはじめようと思ったのですが、NT SQL だと
非常に処理が重たいというところに気が付きました。
私は SQL に関係するのがはじめてなので、パフォーマンス
の判断のしようがないのですが、「UNIX だったら、きっと」
という期待で、SQL サーバを探していたところ、この ML を
見つけました。

しかし、PostgreSQL-6.3.2 のインストールでつまずいてい
ます。

 FreeBSD-2.2.6
 PostgreSQL-6.3.2
 apache-1.2.6
 PHP/FI 2.0.1

PostgreSQL のインストールは完了し、
 パスを通し
 setenv LD_LIBRARY_PATH /usr/local/pgsql/lib
 initdb --pgdata=/usr/local/pgsql/data --pglib=/usr/local/pgsql/lib
 postmaster -D/usr/local/pgsql/data -S -i
とし、postmaster も起動しているのですが、
 gmake all runtest を
行うと

=============== destroying old regression database... =================
Connection to database 'template1' failed.
PQexec() -- There is no connection to the backend.
destroydb: database destroy failed on regression.
=============== creating new regression database...   =================
Connection to database 'template1' failed.
PQexec() -- There is no connection to the backend.
createdb: database creation failed on regression.
createdb failed
ACTUAL RESULTS OF REGRESSION TEST ARE NOW IN FILE regress.out

となりデータベースにアクセスできてないようです。
postmaster も落ちてしまいます。

いきなりの質問で恐縮なのですが、
アドバイス、よろしくお願いします。
