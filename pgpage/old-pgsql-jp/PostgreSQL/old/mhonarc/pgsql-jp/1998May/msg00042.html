> . => 不可視属性とでも言うのですか？
> 気が付かずに ls -l /tmpでfileがないものと勘違いしていました。

「不可視属性」という表現はこの場合は適当ではありません。
MS-DOS や Windows などで「不可視属性」がついているファイルは
明らかにファイルのハンドリングが他の物と異なります。このように
ファイルの「属性」となっているものは「不可視属性」と呼んでも
いいでしょう。

しかし、.cshrc などは UNIX では 'dot file' という様な呼ばれ方をします。
フィル名が . (dot) で始まるファイルは ls では -a オプション無しでは
表示できないとか、シェル上でワイルドカード展開する際に特に指定され
なければ( .[a-z]* とか…)対象にならないという「プログラム仕様」の
世界の話しです。

# dot で始まるものと言っても、. と .. は特別扱いですが

他のいかなるコマンド(プログラムインタフェース上)でも dot file は
他のファイルと同様に扱えるのですから、「不可視属性」とは異なるものです。
