On Thu, 14 May 1998 16:48:47 +0900,
"Tomoyoshi ASANO" <asa@use-net.co.jp> wrote:
> 
> ORACLE のマニュアルでは
> 
> ANSI SQLのデータ型と Oracle のデータ型の対応表では
> 
>   NUMERIC(p,s), DECIMAL(p,s), DEC(p,s)    = NUMBER(p,s)
> 
> で、 DB2データ型 と ORACLE データ型との対応表では
> 
>   DECIMAL(p, s) = NUMBER(p, s)
> 
> となっていたので、そのままの考え方で使い始めて、
> 驚いたわけです。
> 
> で、私の理解では、例えば NUMERIC(10,3) とは、
> 
> マイナスの値の場合はちょっと違うのですが、
> 
> 整数部 7 桁、 小数点以下 ３桁の精度を
> 保った形で、データを格納できることを意味していた
> と思います。

Oracle の NUMBER() 型は直観的にわかりやすいかもしれませんね :-)
# 一度 Oracle の勉強をしてみたいと思いながらも手がでない...

現在のところ PostgreSQL では精度の管理まではやっていないみたいです。
# 多分...

それで、NUMERIC ですが、
PostgreSQL ユーザーズマニュアルとパーサのソースから
	NUMERIC = int4
と読めました。

試しに

	CREATE TABLE a1 ( num numeric );
	CREATE TABLE a2 ( num numeric(9,0) );
	CREATE TABLE a3 ( num int4 );

とやってみたのですが、
どれも int4型のフィールドを持つテーブルになってしまいました :-)

あくまでキーワードのインプリメント程度な気がします。
