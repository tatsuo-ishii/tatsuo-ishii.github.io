市川＠お茶大です．

> ”file table を増やす、postgres の max user を減らす” のにはどこをいじれ
> ばよいのでしょうか (postmaster起動時？ or システムカタログのどこか？)。

file table 数にはシステムの上限とプロセスの上限があるのですが，
こちらでは手っ取り早く ulimit コマンドを起動スクリプトに入れて
プロセス上限の方を引き上げるようにしています:

	ulimit -n 300
	... postmaster の起動 ...

勿論 300 個の FD が本当に取れるかどうかはカーネル設定によると
思います．

ちなみに，これを使っているのは Solaris2.5.1 client の default 設定
で，他には何もしていませんのですが，300 FD まできっちり取って
走ります．(動的に変えているのかな ...)

OS によっては 100 位で引っ掛かるのでその場合はカーネル設定の
変更が必要ですが，まずは limit/ulimit あたりがきくかどうか
試してみると良いのではないでしょうか．
