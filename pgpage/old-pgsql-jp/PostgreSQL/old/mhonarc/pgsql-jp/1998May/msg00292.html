直接の mail で連絡を下さった方もいらっしゃるのですが、FreeBSD 2.2.X
では、daemon に core を吐かせないようになっているようです。
/etc/login.conf での設定があるのですが、これを書き換えても私の
ところではうまく core を吐きません。

FreeBSD ではセキュリティのために daemon プロセスは core を吐かせない
ような仕掛けにカーネルがなっているとかいないとかいう話しもある
みたいでして、現在 FreeBSD-users-jp ML でも質問中です。^^;

> o 問題のプロセスが nobody かなんかで走っているため、ServerRoot に書き
>   込みができない

もちろん、ServerRoot 以下の所有者は wwwadm です。

> gdb を複数上げて、今いるすべての httpd に attach しておくという手もあ
> ります:-)

問題の他に、K6 200MHz マシンではうまく動くものが、Pentium Pro 200MHz
マシンでは httpd が落ちると言う現象が起きて、頭を抱えています。^^;

両方とも同じように make してるんだけど… OS はともに FreeBSD 2.2.6
