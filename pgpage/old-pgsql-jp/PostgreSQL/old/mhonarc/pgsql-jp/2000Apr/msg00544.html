なんか、PostgreSQLのMLなのに・・・。
PHPの話題ですみません。

PostgreSQLには、
isfinite(datetime)
という妥当性をチェックする関数があります。
でも、これって使い方わかりません。

例えば、
select isfinite('now'::datetime);
とすると、t(TRUE)が返ってきますが、
select isfinite('200/02/30'::datetime);
とすると、datetimeのキャスト部でエラーになっちゃいます。

select isfinite('200/02/30');
では、なんとt(TRUE)が返ってきます。

 select isfinite(datetime('2000/02/30')) ;
とすると、datetime関数がエラーになります。
当然、::dateなんてつけると、キャストでエラーです。

どう使うんでしょうか・・・・。
マニュアルには、例として
isfinite('now'::datetime)
なんてありますが、「そりゃ当然有効だろ！」って
ツッコミ入れたくなります。

では。
