そうですか。機会があれば、gdb を使って、どこで落ちているのか見ていただ
ければと思います。

> 　ところで、二点ほど気になることがあります。今日一日エンド
> ユーザーに使ってもらったのですが、

> verify_password: user 'Admin' not found in password file.
> Password authentication failed for user 'Admin'
> 
> 　のログは出力されませんでした。このログだけで解釈すると
> Adminユーザーが接続しようとして、パスワードが登録されて
> いないという意味だと思うのですが、クライアントのODBCから
> は、Adminというユーザーは使っていません。Apache+PHPも
> 使っているのですが、同じくAdminユーザーは使っていません。
> ところがpostmasterが落ちた日のログでは、上記のメッセージが
> ところどころに表示されていました。（学内LANでしかも外部と
> 隔離されているLANなのに、どこからかクラッキングされていた
> りして．．．）

そのようですね。postmaster に -i を付けて起動している場合、IP address 
や network address でブロックしていない限り、他のホストから psql など
で接続しようと思えばできますからね。
