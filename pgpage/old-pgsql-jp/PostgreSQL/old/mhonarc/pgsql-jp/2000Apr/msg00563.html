ちょっと、何をなさりたいのかがよく見えないのですが、ある IP アドレスの
ホストからの Java プログラムなり、psql なりの接続は reject したくて、
local からは許したいのですか？

「telnet からだと reject してても入れる」の意味がよくわからないの
ですが、アクセスを禁止しているマシンから telnet で入ってしまうと
psql が使えてしまう。という事ですか？

でしたら、それは当たり前です。どこから telnet して来ようが、その
ホストに login してしまったら、PostgreSQL にとっては local からの
アクセスになります。

他のホストから telnet で login された場合を何とかしたいのであれば、
PostgreSQL 側では無理なので、TCPwrapper などで、telnet そのものを
禁止してしまうか、または PostgreSQL 側で、各 PostgreSQL ユーザに
パスワードをつけて、pg_hba.conf では、

   local   all              password passwd

か、または

   local   all                      crypt

のいずれかの指定をする事になるでしょう。

そういう意味とは違ってたりしますか？ (^^;
