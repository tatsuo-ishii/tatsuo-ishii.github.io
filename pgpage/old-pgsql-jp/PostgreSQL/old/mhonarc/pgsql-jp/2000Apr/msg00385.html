On Thu, 20 Apr 2000 23:17:20 +0900
Tatsuo Ishii <t-ishii@sra.co.jp> wrote:

> ところで、ちょっと気になったのですが、本題の postmaster の障害が起きた
> とき、実際に postmaster はコアを吐くのですか?

　はい。coreを吐いているようです。エンドユーザーから使えなく
なったという連絡が入って、ps で調べるとpostmasterがなくなっていて、
postmasterを起動したディレクトリに core ファイルが作成されていた
ので、そう理解しています。

　ところで、二点ほど気になることがあります。今日一日エンド
ユーザーに使ってもらったのですが、

verify_password: user 'Admin' not found in password file.
Password authentication failed for user 'Admin'

　のログは出力されませんでした。このログだけで解釈すると
Adminユーザーが接続しようとして、パスワードが登録されて
いないという意味だと思うのですが、クライアントのODBCから
は、Adminというユーザーは使っていません。Apache+PHPも
使っているのですが、同じくAdminユーザーは使っていません。
ところがpostmasterが落ちた日のログでは、上記のメッセージが
ところどころに表示されていました。（学内LANでしかも外部と
隔離されているLANなのに、どこからかクラッキングされていた
りして．．．）

　また、お恥ずかしい話なのですが /usr/local/pgsql/data 
ディレクトリのオーナーはpostgresになっていたのですが、それ
以外のファイルオーナーが root になっていたのを、今日気
がつきました。（6.5.0 -> 6.5.3にした時の失敗 ^_^;）

　どちらにしろ、しばらく様子を見てみようと思っています。
