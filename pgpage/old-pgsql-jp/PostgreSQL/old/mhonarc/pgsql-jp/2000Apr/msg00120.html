文が分かりにくかったようですみませんでした。
通常のコンソール上やktermでは過去のコマンドがカーソルキーで
参照できて、psqlの状態でだけ参照できないので、てっきり
postgreSQLの設定ミスなりインストールミスだろうと思ってこちら
に投稿してしまいました。
postgreSQLとは全く関係ないのであれば、この質問は終わらせたい
と思います。
＃ただ、インストールミスなどで、他の機能にも影響する恐れがあ
るのが怖かったもので…。

TERM環境変数というのをどこで調べたらいいのか、恥ずかしいです
が分かりませんでした。
名前に共通点がある/etc/termcapを見てみましたが、さっぱりです…。

> > また、カーソルキーが使えなくとも、Ctrl-P, Ctrl-N (それぞれ ↑ と ↓
> > に対応)とか、Ctrl-F, Ctrl-B (→ と ← に対応)などは使えるのではないかと
> > 思いますが。
> > 
> > 本来はカーソルキーではなく、これらのキーで操作を行うのが本道です。;-)
> 
> この話は同じです。やはり、
> 
> > man readline してみましょう。(^_^)
> 
> という事で…

まず、Ctrl-P,Ctrl-Nなどでもpsqlの状態からは機能しませんでし
た。
man readlineが出来なかったため、
/usr/doc/readline-2.2.1/readline.htmを見ました。
で、/etc/inputrcを見てみましたが、psqlで過去コマンドを参照で
きるマシンのものと見比べましたが、同じものでした。
----/etc/inputrc----
set input-meta on
set convert-meta off
set output-meta on
--------------------

この機能は、通常のコンソールなりで過去のコマンドを参照するに
しろ、psqlの状態で参照するにしろ、同じreadlineという機能(?)
と言う事でいいのでしょうか…。

本当に素人丸出しで申し訳ありません…。
