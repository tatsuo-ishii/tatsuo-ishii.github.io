安藤@富士通です。

PostgreSQLをインストール使用と以下の環境でインストールしようとしたところ、
いずれのコンパイルでも、gmake installで以下のメッセージが出て途中で
止まります。
# コンパイル時には特にエラーメッセージは出ていません。

gmake[1]: *** [postgres11] Error 2
gmake[1]: Leaving directory `/export/home/andou/build/postgresql-6.5.2/src/
backend'

実際に/usr/local/pgsqlには、bin/, lib/, include/が作成されますが、
中身は空です。

他に修正/設定する場所がありましたら教えて下さい。

--
Solaris 2.6 (SPARC)
 - postgresql-6.5.3-patched-20000215.tar.gz
 - flex version 2.5.4
 - ./configure --with-mb=EUC_JP --without-CXX
   または、
   ./configure --with-mb=EUC_JP --without-CXX --with-template=solaris_sparc_gcc
 - gmake または、gmake COPT="-O0"

Solaris 7 (SPARC)
 - postgresql-6.5.3-patched-20000215.tar.gz
 - gcc 2.95.2 (C++あり)
 - flex version 2.5.4
 - pl_comp.cにextern修正
 - ./configure --with-mb=EUC_JP
 - gmake または、gmake COPT="-O0"
