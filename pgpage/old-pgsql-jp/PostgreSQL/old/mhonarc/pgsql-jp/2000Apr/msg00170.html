> というエラーがでます。PKを複数設定するに当たって、どのような処置を
> すればよいのでしょうか？

世の中に存在するどのような DBMS でもひとつのテーブルで指定できる
Primary Key はひとつのみです。

テーブルの複数タプルの中で、あるタプルを一意的に識別したい時に用いる
ものが Primary Key なのですから、ひとつしか指定できないのは自明でしょう。

何をしたいのかよくわかりませんが、検索効率を上げるためにインデックスを
設けたいというのであれば、Primary Key 以外のものは、当該カラムに対して
インデックスを作成すればよいでしょう。

複数のカラムをまとめて Primary Key として扱いたいというのであれば、
あなたが書いた例を元にすると、

  create table instant_msg (
    usr_id varchar(16),
    msg_id varchar(5),
    tranusr_id varchar(16),
    contents text,
    midoku_flg varchar(2),
    del_flg varchar(2),
    author varchar(20),
    modify_dt datetime,

    primary key(usr_id, msg_id)
 );

と、なると思います。テストしてないけど。(^^;

一般的な SQL の本とか、PostgreSQL のマニュアルを見ればこのあたりの
事はわかるかと思いますので、マニュアル類は読むようにした方がよい
でしょう。

> また、テーブルを作成する際に、.sqlファイルの実行ではなくもっと効率の
> よい方法はないでしょうか？

psql で対話的に定義を行うより、ずっと効率がよいと思いますけど？
どのあたりが効率が悪いと感じてらっしゃるのでしょうか？
