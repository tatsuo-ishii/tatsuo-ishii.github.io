こじまと申します。

主にＪＡＶＡとＤＢを使った仕事をしております。

接続がうまくいったときのソース、コンパイル用のバッチファイル、および実行用の
バッチファイルを以下に示します。
（WindowsNT上で開発したときのものです。）

■ソース

 Connection connection = null;
 String id = "hoge";
 String password = "hoge";
 String jdbcDriver = "postgresql.Driver";
 String jdbcUrl = "jdbc:postgresql://HOSTNAME/test1";

 try{

  Class.forName(jdbcDriver);
  connection = DriverManager.getConnection(jdbcUrl, id, password);

  System.out.println("接続しました。");

 }catch(SQLException sqle){

  System.out.println("接続できませんでした。(" + sqle.getMessage() + ")");

  connection = null;

 }catch(Exception e){

  System.out.println(e.getMessage());

  connection = null;
 }


■コンパイル用バッチファイル

set
CLASSPATH=.;C:\jdk1.2.2\jre\lib\i18n.jar;C:\jdk1.2.2\jre\lib\tools.jar;C:\jd
k1.2.2\jre\lib\dt.jar;
C:\jdk1.2.2\bin\javac -deprecation test.java

* jdbcで定義されているのはインターフェースなので、コンパイル時にはPostgreSQL
のjdbcドライバのパスはCLASSPATHに含まれていなくても構いません。

■実行用バッチファイル

set
CLASSPATH=.;c:\jdk1.2.2\jre\lib\i18n.jar;C:\jdk1.2.2\jre\lib\tools.jar;C:\jd
k1.2.2\jre\lib\dt.jar;.\jdbc\jdbc6_5-1_2.jar;
C:\jdk1.2.2\bin\java test

* 実行時にはjdbcドライバがロードされるので、PostgreSQLのjdbcドライバのパスは
CLASSPATHに含まれている必要があります。
