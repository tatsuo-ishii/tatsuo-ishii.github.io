使用方法は間違っていないと思います。

indexが使用されない原因としては
1. vacuum analyzeをしていない。
2. 件数が少ない。
などが考えられます。
試しに
vacuum analyze product;
してみてください。
また、タプル数は何件ぐらいでしょうか？

>   下のようなテーブルを操作する時 explain で見ると、２番目の場合には
> index を使っていないのですが、index を使わせる何か方法があるので
> しょうか？
>   FAQ だったら、ごめんなさい。(該当記事の番号を教えて頂ければ幸いです。)
>
>   create table product(
>       id_product varchar(16),
>       id_category char(3),
>       prod_name   text
>          :       :
>   );
>   create unique index product_pkey on product(id_product);
>   create index prod_category_key on product(id_category);
>
> １)Index が使われている場合
> # explain
> # select * from product
> # where id_product='D10278259';
> NOTICE:  QUERY PLAN:
>
> Index Scan using product_pkey on product  (cost=0.00..2.01 rows=1
width=97)
>
> EXPLAIN
>
> ２)index を作っていても利用されていない場合
> # explain
> # select * from product
> # where id_category='004';
> NOTICE:  QUERY PLAN:
>
> Seq Scan on product  (cost=0.00..7.65 rows=136 width=97)
>
> EXPLAIN
