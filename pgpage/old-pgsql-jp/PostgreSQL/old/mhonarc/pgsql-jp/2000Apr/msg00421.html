　こんにちは、海野といいます。

　現在、勉強中です。今後ともよろしくお願いします。


　さて、いきなり質問で申し訳ありませんが、Windowsクライアン
ト環境からPostgreSQLサーバへアクセスするときの漢字コードの取
り扱いについて教えて下さい。


＜要件＞
    サーバでは「EUC」を使いたい。
    クライアントからは libpq.dll / ODBC経由 両方使いたい


    日本語Windows版 libpq / 日本語ODBC ドライバは片岡さんの
    ページからいただきました。
    http://www.interwiz.koganei.tokyo.jp/software/PostgreSQL/index.html


　ODBCドライバ経由との比較で、試してみたところでは、
        サーバ              クライアント
    ----------------------------------------------------
        SJIS                ODBC                NG
                            libpq.dll           OK

        EUC                 ODBC                OK
                            libpq.dll           NG

        JIS                 ODBC                NG
                            libpq.dll           NG

　となり、ODBC経由で入力したデータとlibpq.dll経由で入力した
データの文字コードが異なってしまっているようです。


片岡さんのページの 日本語libpq.dll 説明には、

「環境変数 PG_CLIENTENCODING が設定されていない場合に自動的
　にシフトJISが選択されるように、仕様を変更してあります。」

とありますが、クライアントのWindowsマシンでは何も設定しなく
て(デフォルトのまま)でいいのでしょうか。

　よろしくお願いします。


＜環境＞
サーバ
    FreeBSD 3.3R
    PostgreSQL  6.5.3   ./configure --with-mb=EUC_JP
                            でコンパイル

クライアント
    Windows 98

    以下は、片岡さんのページからいただきました。ありがとうございます。
    日本語ODBC ドライバ
    http://www.interwiz.koganei.tokyo.jp/software/PsqlODBC/index.html
    日本語Windows版 libpq 
    http://www.interwiz.koganei.tokyo.jp/software/PostgreSQL/index.html
