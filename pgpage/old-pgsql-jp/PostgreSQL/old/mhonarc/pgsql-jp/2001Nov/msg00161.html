[pgsql-jp 23745] 特定のデータでinsert等が出来ない

  • Date: Thu, 15 Nov 2001 12:15:55 +0900
  • References: <OF3F5D4EFE.9BB4B643-ON49256B03.00353DD8@ryoin.co.jp> <20011115114337.83E9.YUUMEN@oumiya.ne.jp>

こんにちは、蕪木＠ＢＩＧです。

ちょっとトラブルが発生しているのですが、適切な解決方法が見つから
ないので、お知恵を拝借出来ればと思っております。

データベースにある特定の文字列のデータを insert しようとすると、
psql がだんまり状態になってしまい、しばらく待っても止まったままに
なってしまいます。データはインデックスの対象（重複チェックしてます）
になっています。

仮に aaaa というデータとすると、insert する前ならば psql で

select * from hoge_tbl where data ~ '^abcd';
select * from hoge_tbl where data = 'abcd';

等としても該当するデータが出力されたり、「(0 rows)」だったりする
のですが、いざ

inser into hoge_tbl (data) values ('abcd');

しようとすると、ずっとだんまりになってしまいます。

ちなみに

inser into hoge_tbl (data) values ('abcdabcd');

は問題ございません。

だんまりになると、マシンを再起動しない限りは次の insert もなにも
できなくなってしまいます。

これはデータベースファイル（またはインデックスファイル）の一部が
壊れているからでしょうか？　どのようにすれば確実に「壊れている」
と判りますでしょうか？

また、もし壊れているとするならば修復しないといけないと思うのです
が、pg_dumpall をしても途中でエラーとなりデータベースの吸い出しが
出来ないのです。こんな状態なので、vacuum するのもためらわれますし
第一同様にエラーが出て出来ないような気がしています。

どのように解決したらよいでしょうか？

どうかお知恵を御貸し下さい。
