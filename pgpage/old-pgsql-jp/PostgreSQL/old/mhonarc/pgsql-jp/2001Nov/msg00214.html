はじめまして。中村と申します。

早速ですが、お教え願いたい事があってMLに投稿させてもらいました。

Postgres7.1.3+RedHat7.1で、ecpgを使用してDB用のバッチを作成したく
思っております。

Cのソースコードを作成した後、
ecpg -I /usr/local/pgsql/include ./pg_name.pgc -o ./pg_name.c

でプリプロセスを行い、その後、

LIB=/usr/local/pgsql/lib
INCLUDE=/usr/local/pgsql/include

pg:
        gcc -g -I$(INCLUDE) pg.c -o pg -L$(LIB) -lecpg -lpq

などでmakeしてロードモジュールを作成いたしました。

ecpgでもgccでも、No Error/Warningで処理できたのですが、いざ
実行してみますと、
./pg: error while loading shared libraries: libecpg.so.3: cannot load
shared object file: No such file or directory
なるメッセージが出力され、実行できない状態になります。

libecpg.so.3なるライブラリは、/usr/local/pgsql/libで、
libecpg.so.3 -> libecpg.so.3.2.0
となっており、libecpg.so.3.2.0にリンクされています。

念のため、
LD_LIBRARY_PATH=/usr/local/pgsql/lib
などと指定もしてみましたが結果は変わりません。

もちろん、psqlやその他コマンドは正常に稼働しております。

どなたか、この現象に対する対処をご存知の方がおられましたらお教え
ねがえないでしょうか？
