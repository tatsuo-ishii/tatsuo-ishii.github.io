さて。
http://www.sra.co.jp/people/t-ishii/PostgreSQL/
の「PostgreSQL 7.0 から 7.1 への変更点」を参照したのですが分から
なかったので教えて下さい。

RedHat7.1+PostgreSQL7.0.2で今まで環境を構築していました。
今度PostgreSQL7.1.3に変更したのですが、今まＤＢのエンコードは
SJISで行ってました。
createdb -E SJIS hogehoge

ところが、7.1.3で同じ事を行うと
$ createdb -E SJIS
createdb: SJIS cannot be used as a database encoding
と怒られていまいます。

PostgreSQL7.1.3ではSJISでDBを構築できないのでしょうか。
エンコードリストには入ってないのでダメそうですが・・・。

困っている点は
１．既存のデータが全てSJISの為EUC-JPで作るとデータを移行できない。
２．EUC-JPでデータベースを作成するとクライアント側から丸付数字や
　　ギリシャ数字でデータベースへ登録できない。文字化け必至。
　　試しに
　　$ configure
　　　--prefix=/usr/local/pgsql --enable-multibyte=EUC_JP 
　　　--with-tcl --with-java --enable-syslog 
　　　--enable-locale --enable-odbc
    $ make ; make install
    $ initdb
    $ postmaster -S -i > /usr/local/pgsql/log/logfile
　　$ createdb -E EUC-JP hogehoge
[postgres@pioneer postgres]$ psql -l
        List of databases
 Database  |  Owner   | Encoding
-----------+----------+----------
 hogehoge  | postgres | EUC_JP
 template0 | postgres | EUC_JP
 template1 | postgres | EUC_JP

    でデータベースを作成し、日本語ODBCドライバをPCへインストール
　　Accessで漢字を登録すると「Unterminated quoted string(#7)」
　　と怒られます。

宜しくお願い申し上げます。
