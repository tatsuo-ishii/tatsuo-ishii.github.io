hirokazu_inoue> 　JDBCドライバを使いPostgresSQLにアクセスしたいのですがうまくいきません。
hirokazu_inoue> 誰か分かる方がいたら教えて下さい。

　JDBC ドライバの設定とは関係なさそうですね。


hirokazu_inoue> 環境変数「.bashrc」には、
hirokazu_inoue> export CLASSPATH=/usr/local/pgsql/share/java/postgresql.jar:/
hirokazu_inoue> usr/local/pgsql/share/java/postgresql-examples.jar
hirokazu_inoue> を追加してます。
hirokazu_inoue> 
hirokazu_inoue> [postgres@itsrv postgres]$ set | grep PATH
hirokazu_inoue> でCLASSPATHが設定されたのを確認してます。
hirokazu_inoue> 
hirokazu_inoue> そして、下のコマンドでサンプルプログラムを動かしてみるとエラーになります。
hirokazu_inoue> [postgres@itsrv postgres]$ java example.basic jdbc:postgresql:test t-ishii
hirokazu_inoue> ""
hirokazu_inoue> Exception in thread "main" java.lang.NoClassDefFoundError: example/basic

　example/basic が見つからないということですね。原因はわかりません。

#直接関係ありませんが、t-ishii というユーザで、test というＤＢを作ってい
　るのですか？


hirokazu_inoue> その上、上記のソース（p15_1.java）をコンパイル・実行をしてみると、
hirokazu_inoue> [postgres@itsrv postgres]$ javac p15_1.java
hirokazu_inoue> [postgres@itsrv postgres]$ java p15_1
hirokazu_inoue> Exception in thread "main" java.lang.NoClassDefFoundError: p15_1
hirokazu_inoue> とコンパイルは成功するのですが、実行はエラーになります。
hirokazu_inoue> 
hirokazu_inoue> CLASSPATHを通した途端、javaコマンド自体うまく動作していない様なのですが、
hirokazu_inoue> 分かる方がいらっしゃったらよろしくお願いします。

　カレントディレクトリが CLASSPATH に設定されていないから、p15_1.class 
を見つけられないでいます。

　とりあえず、カレントディレクトリを CLASSPATH に追加すれば、動くように
なります。

export CLASSPATH=/usr/local/pgsql/share/java/postgresql.jar:/usr/local/pgsql/share/java/postgresql-examples.jar:.
