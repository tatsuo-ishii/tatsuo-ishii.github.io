Oracle の場合は Oracle8i 8.1.6 か 8.1.7 からそのような機能が実装されて
いたかと思いますが… XDK を使う事になるのかな？ よく知らん。 (^^;

いずれにしろ PostgreSQL には現状ではそのような機能は実装されていない
はずです。
7.2 でもそのような機能はないですよね…？ > 識者の方

したがって、XML 文書全体をそのまま text 属性のカラムに格納するとか、
または XML のタグと対応するカラムを用意するようなテーブル定義を行い、
ユーザが作成するデータ登録アプリケーション自身で XML のパースを行って
エレメントに分解し、対応するカラムにデータを登録する事になります。

いずれの方式をとるかは、そのテーブルをどのように利用しようとして
いるかによると思います。
つまり、設計方針次第だということです。
