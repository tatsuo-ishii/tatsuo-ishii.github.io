はじめまして、
なかはら＠電通大ともうします。


今年の8月頃からバイト先で必要にかられて postgreSQL を使いはじめました。
それから、ここのところ SQL づいて、
個人的に日記をデータベース化してみたり、
自分の勉強をかねてサークルむけの入門記事を書いてたり、
久しぶりに計算機的に楽しいネタになってます。


で、本題ですが、
不定長のリストをどう扱うべきかで困っています。

実際には www の bookmarks を postgreSQL で処理させようとしていて、
各行には URL が１つと、ジャンルと優先順位の組が複数あります。


 SQL 的に不特定多数のデータを扱う方法を知らないので、
はっと考えたのは、単純に

create table bookmarks
(
        category        varchar(256),   -- 分類
        url             varchar(256),   -- URL
);

というようなスキーマで、 category には

  "comp",1,"lang",3,"palmtop",4

というような感じで ずらずら並べて、
あとは perl 等の ハッシュとして取り込んで処理させる
というのを考えたのですが、
せっかく SQL を使ってデータを抽象的に
扱えるようにしようとしているのに、
それに逆行しているような気がしてしまいます。


うまく SQL だけて処理できないものでしょうか。
ご教示いただけるとありがたいです。
