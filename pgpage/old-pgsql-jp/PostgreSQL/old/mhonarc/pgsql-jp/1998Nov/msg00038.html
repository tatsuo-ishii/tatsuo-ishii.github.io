On Thu, 5 Nov 1998 atsunabe@si.hokuriku-tc.ac.jp wrote:

> 適切なパスをどこに書けばいいのでしょうか？
> ある人は”.profiles”という人もあれば、”.cshrc”に書けば良いという人もいます。

  sh 系(bashなど)は .profile で csh系(tcshなど)は .cshrc ほかにも初期化
ファイルはありますので "man bash" や "man tcsh" を実行して確認してください。

> それとパスの記述式がよくわかりません。

 sh 系は
  PATH=/bin:/usr/bin; export PATH
のように、csh 系は
  set path = ( /bin /usr/bin )
のように初期化ファイルに入れておきます。
# UNIX の入門書とかにあるとおもいますので、ご一読下さい。

> $PGDATA/filesの下にファイルがあるかどうかということですけど、そのパスの
> いきかたが分かりません。すみませんが、教えていただけないでしょうか。

  ちょっと意味不明です。おそらく、勘違いされています。
  $PGDATA は PGDATA という環境変数をシェルスクリプトの中とかで参照すると
きに使う記述です。話題により、省略してそう書くことがよくあります。PGDATA
には "/usr/local/pgsql/data" というのが設定されていることが多いのですが、
このディレクトリが存在していると initdb はできません。
