後、PerlのCGIの動作確認に入った途端に、先に進まなくなってしまいました。

環境
・ Solaris x86 2.6
・ Perl 5.005_01

Perl CGIは、[pgsql-jp 5228]の望月様のCGIを用いて確認させていただく事にしまし  
た。

#!/usr/local/bin/perl
use Pg;
$conn = Pg::setdb('localhost', "", "", "", 'test');
print "Content-type: text/html\n\n";
print "<HTML><TITLE>TEST</TITLE><BODY>\n";
$result = $conn->exec("SELECT * FROM homepage");
for ($tate = 0 ;$tate < $result->ntuples ; $tate++){
	for ($yoko = 0; $yoko < $result->nfields; $yoko++) {
		$fname = $result->fname($yoko);
		$value = $result->getvalue($tate, $yoko);
		print "$fname $value <br>\n";
	}
}
print "</BODY></HTML>\n";
exit;

これをコマンドラインから実行すると、下記のメッセージが出ます。

Can't locate loadable object for module Pg in @INC (@INC contains: 
/usr/local/lib/perl5/5.00501/i86pc-solaris /usr/local/lib/perl5/5.00501 
/usr/local/lib/perl5/site_perl/5.005/i86pc-solaris 
/usr/local/lib/perl5/site_perl/5.005 .) at ./test.cgi line 2
BEGIN failed--compilation aborted at ./test.cgi line 2.

これはPg.pmが、見つからないと言っているのでしょうか。
ちなみにPg.pmは、"/usr/local/bin/postgresql-6.3.2/src/interfaces/perl5/"にあ  
ります。
