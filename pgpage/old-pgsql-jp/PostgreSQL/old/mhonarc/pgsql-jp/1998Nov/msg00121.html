Apache の拡張モジュールで mod_auth_pgsql という postgres を使う
認証モジュールがありますが、そのための .htaccess に追加される
ディレクティブの中に

  ● Auth_PGpwd_whereclause
  ● Auth_PGgrp_whereclause

というのがあります。

私にはこの使い方がさっぱり分かりません。

ご存知の方がおられましたら、これはどのように使用するのか、
他のディレクティブとの関係、また、.htaccess のサンプルなど、
ご教授していただきますようお願いします。

以下は README にある上記ディレクティブの説明です。

● Auth_PGpwd_whereclause

  Syntax: Auth_PGpwd_whereclause SQL fragment 
  Context: directory, .htaccess 
  Override: AuthConfig 
  Status: Extension 

  This option allows you to exercise greater control over the 
  SQL code used to retrieve the user name and password from 
  the database. You can use this to search for the username 
  using more attributes in the table than the pwd_field. 

  The basic SQL statement used to retrieve a user's password 
  for checking looks like this: 

    select <pwd_field> from <pwd_table> where <uid_field>='<remote_user>'

  The pwd_whereclause will be added to the end of this statement 
  and must fit logically. 

● Auth_PGgrp_whereclause

  Syntax: Auth_PGgrp_whereclause SQL fragment 
  Context: directory, .htaccess 
  Override: AuthConfig 
  Status: Extension 

  This option allows you to exercise greater control over the SQL 
  code used to retrieve the group name and corresponding user
  from the database. You can use this to search for the group name 
  using more attributes in the table than the gid_field. 

  The basic SQL statement used to retrieve a group name and user 
  name for checking looks like this: 

     select <uid_field> from <grp_table> where <gid_field>='<required group>'

  The gid_whereclause will be added to the end of this statement 
  and must fit logically.
