佐藤＠キヤノンです。

> そこで皆さんにお伺いしたいのですがexit、これ以外のプラットフォームで 
> BIG/LITTLE_ENDIAN の定義がどうなっているか情報を頂けないでしょうか？特
> に、FreeBSD に関して知りたいです。情報には、OS のバージョン、コンパイ
> ラのバージョン、また BIG/LITTLE_ENDIAN をどう定義しているか(値をセット
> しているか、単に @define だけなのか)を明記して下さると有難いです。

PostgreSQL の テストプラットフォームに載っていたようですので、
参考までに DEC MIPS on Ultrix 4.3 の情報です。

「ULTRIX 4.3 RISC では、cc/gcc-2.6.3 ともに類似の定義はない」です。

ただし、[pgsql-jp 2131] で小島さんのご指摘のように

bind を入れ換えてあるマシンでは
<arpa/nameser.h> で

#define LITTLE_ENDIAN   1234    /* least-significant byte first (vax, pc) */
#define BIG_ENDIAN      4321    /* most-significant byte first (IBM, net) */
#define PDP_ENDIAN      3412    /* LSB first in word, MSW first in long (pdp) */

が定義され、BYTE_ORDER もそのマシンに即した値に設定されます。


あと、石井さんが心配されていました FreeBSD に関しては、既に何人かの
方から報告のあったとおり、

FreeBSD 2.2.1-RELEASE では
<machine/endian.h> で

#define LITTLE_ENDIAN   1234    /* LSB first: i386, vax */
#define BIG_ENDIAN      4321    /* MSB first: 68000, ibm, net */
#define PDP_ENDIAN      3412    /* LSB first in word, MSW first in long */

#define BYTE_ORDER      LITTLE_ENDIAN

と定義されております。
