  • From: Toshiharu Takaba <tosh-t@ascii.co.jp>
  • Date: Wed, 14 Oct 1998 00:05:52 +0900 (JST)
たかばです。

タイトルどおり、PostgreSQL の話題ではなく、
インフォミックス社のデータベースサーバの簡単な試用レポートです。


インフォミックス社のデータベースサーバが、

	http://www.informix.com/idn

より、登録を行って、無償でダウンロードできるようになっていましたので、
試してみました。ファイルサイズは以下のように、
トライできなくもないサイズです。

	Informix SE & ESQL/C Bundle for OpenLinux 1.2 & SuSE Linux 5.2
		 9,585,881 bytes
	Informix Japanese Language Supplement V2.00.UC1
		12,428,196 bytes
	Informix JDBC Driver V1.22.JC1
		 1,140,469 bytes

	SE: ファイルシステムにデータファイルを作成する DB サーバ
	ESQL/C: 埋め込み SQL プリプロセッサ
	Japanese Language Supplement:
		日本語環境、MacOS でいうJapanese Language Kit みたいなもの

実行は、FreeBSD (2.2.5) の Linux エミュレーション環境で行ってみました。
インストールすると、SE + ESQL/C + JLS で、約 26 MByte くらいになりました。

	# du -s -k /usr/informix/
	25384   /usr/informix/

サーバでは、ほぼ問題なく日本語を扱えるようでした。
また、ほとんどのメッセージも日本語です。
( PostgreSQL の psql にあたるクライアント dbaccess でテスト )

JDBC ドライバのリリースノートを見ると、
まだ Global Language Support (GLS: Informix 用語 ?) 機能を
サポートしていないそうで、
テストしてみましたがやはり日本語を扱うことはできませんでした。

また、PostgreSQL の postgres にあたる、sqlexec と言うプロセスが、
クライアントプロセスの終了後、ゾンビとなって残ってしまいます。
postmaster にあたる sqlexecd と言うプロセスを終了させると、
成仏するようでした。

ESQL/C は、Linux のコンパイル環境までそろえなければならないので
試用しませんでした。

以上、参考まで。
