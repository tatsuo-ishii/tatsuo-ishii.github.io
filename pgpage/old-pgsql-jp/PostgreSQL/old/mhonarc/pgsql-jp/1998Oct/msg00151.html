# グランパワーの方でしたっけ？

> で、さっそくぶち当たっているのですが、昔昔のパラドックスの経験でいえば、
> テーブルにはキーというものがあって、特定の項目にキーを設定すると、キー
> が重複しないように出来る、とおもうのですが、PostgreSQL でそういうこと
> をするにはどうしたら良いのでしょうか？つまり
> 
> hoge| poo
> ----+----
> foo |baa
> 
> というようにデータが１個だけあるテーブルに
> 
> hoge| poo
> ----+----
> foo |baa
> foo |boo
> 
> というような行が足せてしまうのですが、そうでなくて「キーである項目 
> hoge にはすでに foo という値があるので、もう一個 hoge = foo な行を足す
> ことは出来ません」としたいのですが・・・・

お察しの通り、当然キーはあります(^^;。

CREATE TABLE hoge(
    foo TEXT   PRIMARY KEY
);             -----------

とやるか、またはテーブルがすでに存在するのであれば、

CREATE UNIQUE INDEX hoge_index ON hoge(foo);
       ------

上記の下線の部分がユニークにするためのキーワードです。
ただし 'PRIMARY KEY' の場合は単に「主キー」にするよ、という
指定ですが、これにすると暗黙に UNIQUE 指定をしたようになると
思っています。間違っていたらどなたか突っ込んでやってください。

http://www.remus.dti.ne.jp/~sim/postgres/man/manl/create_table.l.html

には primary key 句の記述がないですが、これは man が追いついて
いないんだと思います。
