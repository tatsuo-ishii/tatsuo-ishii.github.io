> 宮本＠Tickleと申します。
> 
> 勉強の為にpostgreSQL-6.5.2をダウンロードし
> #echo "MB=EUC_JP" > Makefile.custom
> #make install
> を実行すると
> You must use GNU make to use Postgres.  It may be installed
> on your system with the name 'gmake'.
> 
> NOTE:  If you are sure that you are using GNU make and you are
>        still getting this message, you may simply need to run
>        the configure program.
> と表示されました。
> 直訳すると
> Postgresを利用するためにはGNU 作成 (make)を使わなければなりません。
> ’gmake'名でシステムにインストールできるはずです。
> 
> 注意：GNU makeを確実に利用しているのに、このメッセージが出た場合は
> 設定（Configuration)プログラムを稼動させる必要があります。
> となると思うのですが・・・。
> どうすれば回避できるのでしょうか？。

  書いてある通りです。
  GNU make (gmake) を用いてコンパイルしましょう。

  使用しているマシンに gmake が無い場合は portsなり、package なり
  を用いてインストールすると良いでしょう。
