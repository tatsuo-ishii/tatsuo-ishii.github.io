最初の文の2フレーズ目は
「あなたのシステムに'gmake'の名前でおそらくインストール済です」
という感じでしょうか。
「make」は「make」「configure」は「configure」。
それぞれプログラム名だから。

＃なんでもかんでも日本語にすればいいものではない。


>どうすれば回避できるのでしょうか？。

プログラムをインストールできる立場ということなので
おそらく、rootに慣れるんですよね？
which gmakeの結果は？

FreeBSDのmakeはgmakeではありませんので
インストールされてないならば
あらかじめgmakeをインストールしてください。
portsなりpackagesを使えば簡単にインストールできるでしょう。

あとはPostgraSQLの管理者になって
./configure　設定いろいろ(マニュアルか参考書を参照)
gmake all
gmake install

＃シーラカンス本に出てますね

2.2.8のportsのPostgreSQLのバージョンが分からないけど
最新バージョンでなくてもいいならportsでインストールするコマンド
make
make install
make clean
gmakeが未インストールの場合、gmakeもソースをftpで持ってきて
勝手にインストールしてくれます。
