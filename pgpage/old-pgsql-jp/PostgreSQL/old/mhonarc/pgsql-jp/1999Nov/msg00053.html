> OS FreeBSD 2.2.8です。

FreeBSD に附属の make は UCB make と呼ばれるものです。GNU の make とは
別物です。ということで、考えられる対処法をいくつか列挙させていただきま
す。 

1)もしかしたら GNU の make が gmake という command で install されてい
るかもしれない。だから gmake として make をしてみる。
#一時期 gmake が、FreeBSD に標準で install されていたような記憶がある
#のですが、何時だったかは良く覚えていません ^^;

2)ports/developers/make とかいう感じで GNU make の ports があったよう
な気がします。これを make して GNU make を install する、or packages 
を使って GNU make install する。その後、gmake として PostgreSQL を 
gmake として make する

3)GNU の site から make の source を拾ってきて install する。その後、
その make を使って PostgreSQL を make する。

4)PostgreSQL6.5.2 の ports が出来るのを待つ。出来たら、それを install 
する

#もし、GNU make が install されている場合は、command 検索 path を確認
#してみた上で、which make とか which gmake としてみて、企図通りの make 
#が実行されているかを check してみるのが良いかもしれません
