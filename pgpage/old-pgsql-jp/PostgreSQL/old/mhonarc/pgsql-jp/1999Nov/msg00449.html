    クリスマスイベントのサイトの制作に関わったのですが，
    ここで PostgreSQL を使用していますので，紹介します．(^^;
    
    http://www.xmas.ne.jp/
    
    PostgreSQL に，FastCGI と perl(DBI) で構築しています．
    （チャット本体は C ですが，DB との連携は無しです）
    
    マシンは３台で，１台を DB サーバーとほぼ全てのコンテンツに，
    残り２台に，チャットと MyWeb をそれぞれ割り当てています．
    
    VACUUM は，１日〜数日に１回，手作業で httpd を
    止めてから行っています．
    
    立ち上げ直後 SQL がまずくて重くなったり，
    動作中に vacuum をかけたところ固まり，kill したら
    pg_statics を壊してしまって非常に焦ったり(^^;;しましたが，
    現在は快適に動作しています．
    
    
    perl + DBI の場合，mod_perl か FastCGI のどちらかを
    使わないと重すぎると思うのですが，ここでは FastCGI を
    使用しました．
    （mod_perl は以前使ったときにどうもイマイチでしたので…）
    
    
    といった感じです．ご参考になれば(^^;
