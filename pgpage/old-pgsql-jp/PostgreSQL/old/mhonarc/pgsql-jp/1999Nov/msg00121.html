ウ〜ン… 私はいつも FreeBSD でやってるんですが、RedHat 5.2J で試して
みました。
結論から言うと何も問題ありません。

> $ ./configure --with-mb=EUC_JP
> $ gmake all
> 
> You must use GNU make to use Postgres.  It may be installed
> on your system with the name 'gmake'.
> 
> NOTE:  If you are sure that you are using GNU make and you are
>        still getting this message, you may simply need to run
>        the configure program.

これは、GNU make でない make を使うと、GNUmakefile を見ずに
Makefile を見に行くので起こる現象です。

GNU make では、Makefile があっても GNUmakefile があればそちらを
実行します。

と、すると GNU make をお使いでないという事になりますが、RedHat 5.2
の make は GNU make なのでそれはおかしいですねぇ…

何かよぶんな設定を行なうと、Makefile を先に見に行くのでしょうか…

# ちなみに RedHat 5.2 の /usr/bin/gmake は /usr/bin/make へのシンボリック
# リンクです。

/usr/bin/gmake または /usr/bin/make を実行する前に PATH の通っている
どこかで GNU make でない make が gmake として実行されているという
事はありませんか？

gmake -v とやって、

  GNU Make version 3.76.1, by Richard Stallman and Roland McGrath.

          Free Software Foundation, Inc.
  This is free software; see the source for copying conditions.

  PARTICULAR PURPOSE.

  Report bugs to <bug-gnu-utils@prep.ai.mit.edu>.

というように出てくるでしょうか？ 上記のものは RedHat 5.2J で実行した
結果です。
