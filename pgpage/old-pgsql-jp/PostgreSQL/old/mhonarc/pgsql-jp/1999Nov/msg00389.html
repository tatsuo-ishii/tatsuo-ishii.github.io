こんにちは。　宮坂です。　
メールありがとうございます。


> 
> １．pg_hbaでTCP/IP接続を許可していること。

　pg_hbaの中で、
	host  all  0.0.0.0   0.0.0.0  trust
	local all 		      trust

　としてます。　これで許可してるといえるでしょうか。


> ２．postmaster を -i で起動していること。

　最初の質問に書いたのですが、postmasterは
	postmaster -S -i
	postmaster -d 2 -i (デバック)
　
　どちらもあげて見ました。


> ３．hosts.allowで、そのホストにアクセス権があること。

　これは特に手をふれてないのですが、Linux,php,postgres.apacheがインスト
ールされている
　マシンで、アクセスしてるのですがそれでもphpからpostgresに接続する場合
アクセス権が必要なのでしょうか。
　ちなみに、デバック上でlocalhostが接続に行ってます。　


> ４．unix user と postgres user の対比に間違えがないこと。

　これはちょっとわからないのですが、ユーザーはnobodyで接続に行ってます。
　もちろんpostgreSQLにはnobodyはユーザー登録されてます。

それでも繋がらないのです。
psqlなら、postgreSQLに繋がります。
何か環境が、いけないのでしょうか。

宜しくお願いします。
