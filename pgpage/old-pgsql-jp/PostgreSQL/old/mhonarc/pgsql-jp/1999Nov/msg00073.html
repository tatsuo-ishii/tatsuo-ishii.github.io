さて、本題なのですが…
> その後gmake installを実行すると下記の様なエラーが出ました。
> gmake lexverify
> gmake[1]: Entering directory `/usr/local/src/postgresql-6.5.2/src'
> gmake -C lextest all
-- snip --
> gmake[1]: Leaving directory `/usr/local/src/postgresql-6.5.2/src/backend'

以上の messages をざっと眺めたのですが、いわゆる install に失敗したと
いう意味を表わす error messages は無いのではないのでしょうか？

#標準 error 出力に出される出力は error message だけではありません。気
#が付いて欲しい事なんかも、ずらずらと出力される場合があります。
#また GNU make は UCB make よりも messages を冗長に出力するきらいがあ
#ります。

たとえば、
> gmake[1]: Nothing to be done for `install'.
これは、単に "この directory では gmake install と言われてもする作業が
無いよ" という意味ですし、

さらに
> mkdir /usr/local/pgsql/lib
ここで /usr/local/pgsql/lib という directory が make されています。

これらを考えると、問題なく PostgreSQL が install されているように思え
ます。
