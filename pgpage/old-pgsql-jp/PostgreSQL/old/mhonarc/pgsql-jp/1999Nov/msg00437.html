家入＠ｓｄｋｔと申します。

PostgreSQLにODBC経由Access97で業務システムを作っています。

joinの書き方について質問させてください。

例えば、３つのテーブルが親、子、孫の関係であればSQL89形式
のjoinで実現できることは確認しました。

<例1>
create table T01(t1 int4, t11 text); --親
create table T02(t1 int4, t2 int4, t22 text); --子
create table T03(t1 int4, t2 int4, t3,int4, t33 text); --孫
というテーブルがあった時、
select T01.t11, T02.t22, T03.t33 from T03,T02,T01
 where T03.t2 = T02.t2 and T02.t1 = T01.t1;
というSQLで、孫からLOOKUPした子と親の情報を取る事が出来ました。

これが、子、父、母の関係の時、joinできるのでしょうか。

<例2>
create table T01(t1 int4, m1 int4, m2 int4); --子
create table M01(m1 int4, m11 text); --父
create table M02(m2 int4, m22 text); --母
というテーブルがあった時に、
「T01.m1 = M01.m1のjoin」と「T01.m2 = M02.m2のjoin」の
join表が欲しいのです。
つまり、子からLOOKUPした父と母の情報を取りたいのです。

どなたか方法をご存知でしたら教えてください。
