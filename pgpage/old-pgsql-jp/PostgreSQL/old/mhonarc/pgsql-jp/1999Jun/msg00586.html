> ご助力をお願いします
> 
> やったこと：
> 6.4.2のpostmasterと別に6.5用のpostmaster（名前はpgmaster)をadduser
> 6.5をコンパイル＆インストール（6.5のpostmaster）
> initdbを実行（データディレクトリは出来ました）
> postmasterを実行 （以前石井さんが投稿なさったように別ポートで
> postmasterを立ち上げてます）
> createdb でerror
> 内容は以下の通りです
> --------------------------------------
> % createdb lib3
> Connection to database 'template1' failed.
> FATAL 1:  SetUserId: user 'pgmaster' is not in 'pg_shadow'
> 
> createdb: database creation failed on lib3
> ----------------------------------------

こちらでもやってみました。

sim@server1:~$ createdb hoge
データベース 'template1' に接続できません。
FATAL 1:  SetUserId: user 'sim' is not in 'pg_shadow'

createdb: database creation failed on hoge.

sim@server1:~$ su - postgres
postgres@server1:~$ createuser sim
Enter user's postgres ID or RETURN to use unix user ID: 10008 ->
Is user "sim" allowed to create databases (y/n) y
Is user "sim" a superuser? (y/n) n
createuser: sim was successfully added
postgres@server1:~$ exit
sim@server1:~$ createdb hoge
(OK)

ということで、単に 6.5 の方で
createuser pgmaster していないからではないかと思います。
