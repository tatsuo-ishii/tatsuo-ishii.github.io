さっそくですが、ちょっと詰まっています、、、

主、従のテーブルがあり、主のテーブルの項目から
従のテーブルの項目を参照したいのですが、

select s.data1 j.data2 from syu s,juu j
where s.cd = j.cd;

とすると、主のレコードが存在していても、
従のレコードが存在しないと表示されません。

オラクル（PL/SQL)では、

select s.data1 j.data2 from syu s,juu j
where s.cd = j.cd(+);

としておくと、主のレコードは必ず表示することができます。

このような検索をPostgresで実現するにはどうしたらよいのでしょうか？
