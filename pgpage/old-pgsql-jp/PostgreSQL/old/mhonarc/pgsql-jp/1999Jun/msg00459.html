書いたものと同じ理由のような気がします。
(エラーメッセージの違い)

[pgsql-jp 8884] PostgreSQL 6.5 test on Slackware
> regressionでは、
> pg_atoi: error reading "100000": Numerical result out of range
> というエラーメッセージを期待していたようですが、私の環境では
> pg_atoi: error reading "100000": Math result not representable
> というエラーが返されたようです。

postgresql-6.4.2/src/test/regress/expected/int2.out
(regression testが期待した結果)
と
postgresql-6.4.2/src/test/regress/result/int2.out
(実際に吐き出された結果)
を比較してみて、エラーメッセージ以外に違いが認められなければ
このfailは無視してかまいません。

regression testでは、SQLの実行結果が期待通りであるかを確認する
だけでなく、意図的にエラーを発生させてみるなどして、エラーが起きて
然るべきところではきちんとエラーが発生するかも確認しています。
しかし、環境によって帰ってくるエラーメッセージが異なる場合が有り、
その場合に、failとなってしまいます。

もし不幸にもfailが出てしまった場合には、自分の目で
postgresql-6.4.2/src/test/regress/expected/と
postgresql-6.4.2/src/test/regress/result/の違いを目視確認するしか
ないようです。

# postgresql-6.4.2/src/test/regress/expected/と
# postgresql-6.4.2/src/test/regress/result/の差分を抜き出した
# regress.diffsが
# postgresql-6.4.2/src/test/regress/にありますので、こちらを
# チェックされても良いかと思います。
