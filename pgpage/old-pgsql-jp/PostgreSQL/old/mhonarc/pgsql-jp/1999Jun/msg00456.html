[pgsql-jp 9022] TurboLinuxV3.0でmakeが上手く行かない

  • From: Masayuki Tadano <tadano@d-product.co.jp>
  • Date: Wed, 23 Jun 1999 22:31:04 +0900
あまりにも初歩的な質問なので過去のMLも検索してみたのですが、
makeをすると、エラーが出てしまうのです。

TurboLinix v3.0(2.0.36)でそれに付いているPostgreSQL(多分6.3)を
インストールしてあります。

ローカス社のLinix/FreeBSD SQLデータベース構築入門 PostgreSQL6.4.2+PHP3
に付いているCDを用いてその本に書いてあるように作業をしたのですが、
やった作業は

1. /usr/local/srcにpostgres-6.4.2-patched.tar.gz をコピーし、展開
2. /usr/local/src/postgres-6.4.2/srcディレクトリで
   ./configure --with-mb=EUC_JP
 を実行
3. make allを実行

としたところ、
make lexverify
make -C lextest all
までは終わったようですが
make -C utils all で、
In file included from /usr/include/errno.h:36,
                 from version.c:28:
/usr/include/errnos.h:24: linux/errno.h: No such file or directory
と出てしまいます。
errnos.h には #include <linux/errno.h> と書かれており、確かに
こんなディレクトリは存在しません。

本来は出版社の方に問い合わせるべきだと思いますが、こちらの方が
詳しい方々が多いかと思い質問してみました。
