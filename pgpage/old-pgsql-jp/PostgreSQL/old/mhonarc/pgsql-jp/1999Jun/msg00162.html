ただいま、OracleとPostgreSQLの使い比べております。

さっそく質問なのですが、

　VBで作成したプログラムから PostgreSQLに接続し、
タイムアウト値を設定し大量データ検索を行なったところ、
応答が帰ってきません。
　ちなみに、ODBCのバージョンは、PostgreSQL ODBC Ver 6.4です。

　具体的には、以下の様なプログラムコードです。
 ODBCダイレクトワークスペースを作成する。
    Set wp = CreateWorkspace("", "", "", dbUseODBC)
 DBをオープンする。
    Set db = wp.OpenDatabase("", dbDriverPrompt, False, "")
 DBオブジェクトのQueryTimeoutに値を設定
    db.QueryTimeout = 30
 レコードセットを作成する。
    Set rst = db.OpenRecordset(SQL, dbOpenSnapshot)

　同じテストをOracleのODBCを使ってやってみたところ、
タイムアウト値を経過するとすぐにエラーとなってアプリケー
ションに制御が戻ってきます。

　この違いは、Postgres ODBCではタイムアウトを設定できないのでしょうか？
それとも、Postgres ODBCに何か設定が必要なのでしょうか？

皆さんよろしくお願いします。
