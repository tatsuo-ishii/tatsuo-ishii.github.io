PostgreSQL6.1.1 を HP-UX 10.20B で実行しているのですが，同様のエラー
が出ます．時間が無くてきちんと原因はおりませんが，兆候も全く同じで，

> |admin: {1} createuser priest
> |Enter user's postgres ID or RETURN to use unix user ID: 1001 -> 
> |Is user "priest" allowed to create databases (y/n) y
> |Is user "priest" allowed to add users? (y/n) n
> |createuser: priest was successfully added
> |admin: {2} su priest

というような利用者設定では permission error で発生してして create view
が出来ません．PostgreSQL6.2 でどうかまではわかりませんが，PostgreSQL の
bug の可能性があるような気がします．

結局 create database と add users の双方の権限を利用者に与える
という解決策 (ではないですね :-) を使っています．...
