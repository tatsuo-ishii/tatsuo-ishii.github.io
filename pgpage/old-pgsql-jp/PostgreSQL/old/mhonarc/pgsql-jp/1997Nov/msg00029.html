ちょっと壁にぶつかったので質問させてください。

この ML で PostODBC のことが話題になっているので
触発されて MS-Access , PostODBC をインストールしました。

動作は問題なくできました。

しかし、困ったことにユーザー認証ができないのです。
pg_hba.conf に passwd で認証するようにしたのですが
MS-Access から接続できない。

そこで PostODBC のソースを追っかけていくと
PostgreSQL に connect するところで、option をすべて 0　にしていて
authtype などが設定できないように思えました。

ODBC とは、そのようなものなのでしょうか。

DAO とか RDO は何かまだ理解できていないのですが
RDO には MSDN で調べると passwd があるように見えるのですが
はっきりしません。

microsoft のシステム回りは疑問点が多いのですが
とにかく、なんらかの方法でユーザー認証できればありがたいのですが
なにか、うまい方法はないでしょうか。

目的は、不用意に、間違ってデータを書き換えられるのを防止したいのです。
データの作成者のみ、データを変更でき、他の人は読み込み検索のみにしたい。
今のままでは、他の人が ODBC の user name のところに、データの書き換え
権限のある人の名前を打ち込めば、変更できてしまいます。

IP アドレスで制限するのは、あまり効果ないですし。
もしできても、pg_hba.conf の書き方で
trust された host 以外は、権限のある user name でも read only とはできない
ようですけど。

Windows95 で動作する ident なんてあるのかなあ?
それがあれば、良いのだろうか?
