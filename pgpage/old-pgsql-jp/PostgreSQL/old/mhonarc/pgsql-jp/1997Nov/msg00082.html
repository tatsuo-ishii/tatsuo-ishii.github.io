->石井さん、アドバイスありがとうございました。
->おかげさまで何とかアプレットでうまく動くようになりました。
->このよう些細な質問にも丁寧にお答え下さってありがとうございました。

	学生さんに質問して貰ったのですが、報告が言葉足らずなので
追加します。
	PostgreSQL のあるマシンで JAVA applet のデータベースアプリ
ケーションを動かそうとすると

	jdbc:postgresql://machine/database?user=username

だとコネクトできないようです。その代わり、

	jdbc:postgres:database?user=username

でコネクトできます。
	今度は、http サーバーと PostgreSQL マシンを同一にして、
そのマシンに applet を置いておいて、リモートマシンから

	appletviewer http://machine/DB_applet.html

のように applet を呼び出してデータベースコネクションを張る時には、

	jdbc:postgresql://machine/database?user=username

でコネクト出来ることが分かりました。
	加えて、appletviewer だけではなく、Netscape Communicator 4.04
に applet を呼び出しても、データベースコネクションが張れることが分かり
ました。
	当方としましては、これはデータベースにとって、とても良い
状況が出来たと思っております。いちいち、データベースアプリケーション
をプラットフォームに合わせなくても良いし、ftp でアプリを取ってくる
必要も無いわけです。インターネットの代名詞でもある、ブラウザから
applet を呼び出して、直接、データベース操作できるのですから、とても
便利になると思います。
							以上です。
