> > ＲＤＢＭＳとして見たとき、ＯＲＡＣＬＥの最上位版との違いって
> > なんなのでしょうか？
> > 一口にＯＲＡＣＬＥといってもいくつか種類があるんですよね？

　Oracle8i までだと、普及版である Oracle8i Workgroup Server 
があり、上位版として Oracle8i Enterprise Edition がありました。
　Oracle9i からは、普及版が Oracle9i Standard Edition と名称が
変更されています。上位版は Oracle9i Enterprise Edition で同じ
名称のままです。

　普及版と上位版の違いは、主に新機能の実装量の違いです。過去の
状況から言うと、１、２世代前の上位版の機能も、しばらくすると、
普及版に実装されてきます。最新機能を使いたい場合には、上位版を
使えという考え方ですね。

　これとは別に Oracle9i Lite という配布用の小型 DBMS もあります
が用途はちょっと別物ですね、複数ユーザ向けではありませんから。

> Oracleの最上位版というと、Paralell Serverですね。こいつは複数台のマシン

　より正確に言うと、上位版の（有償の）オプション機能です。また、
現在では、さらに機能向上して Oracle9i Real Application Clusters
（通称 9i RAC）と呼ばれています。Oracle Paralell Server と言う
名称は Oracle8i までです。

# Oracle ネタを知りたいのであれば、O4L-ML や Oracle-Wakaba-ML など
# が良いかと。

> をクラスタリングして巨大データベースを運用できると言うもので、これに比べ
> ればPostgreSQLは確かにたいした物じゃないです。

　Usogres に機能追加をすれば。。
