# おおぼけしていたことにふと気が付きました。

On Tue, 30 Apr 2002 Linux@Lupin.net wrote:

> 
> 札幌のかずです。
> 
> 皆さまのご指摘通り、
> プロバイダーのゲートウェイで止められていました。
> 無事、オフラインサーバーからの、アクセスが可能になりました。
> 
> どうもありがとうございました。
> 
> 
> ネット接続がうまく行かない場合の確認方法をまとめてみました。
> 
> ◆データベースへのアクセスを許可する
>   pb_hba.conf
>   host  DBNAME  IP_ADDRESS  ADDRESS_MASK  AUTHTYPE  [AUTH_ARGUMENT]
>   host アクセスＤＢ名　
> 
> ◆ネットアクセスを許可する
> 　postgres.conf　に以下を設定するか、
> 　　tcpip_socket = true
>   pg_ctl -o "-i"　で起動
> 
> ◆テルネットでポートを確認
>   telenet xxx.xxx.xxx.xxx 5432
> 
> ◆トレースルートで、途中のルータを確認
>   traceroute xxx.xxx.xxx.xxx
> 
> ◆ポートを変えてアクセスの試験を行う
> 　5432〜80や25など
> 
> 　※私の環境では、8080や5433などの変更はうまくいきましたが
> 　　80や、25の変更では、Postgres がうまく起動しませんでした。
> 　　見かけ上は立ち上がるのですが、restart をかけますと
> 　　立ち上がっていなかったメッセージが出てしまい
> 　　結局、80、25の試験は出来ませんでした。
> 　　（Apache や　メールサーバを止めて試験）
> 

  PostgreSQL の postmaster は root 以外で走らせなくては
なりませんでした。ところが、IP の 80や23というポートは
特権ポートで root でなくてはポートを開いて待ち受けでき
ないのでした。

どうもすみませんでした。mOm
