池田＠Oracle Master は持ってません（苦笑）。

> 会社にいる若いシステム屋さんに私が構築したPostgresの環境を貸してあげて
> いるのですが、彼曰く「Postgresは所詮Ｏｒａｃｌｅにかなわない」と事ある
> ごとに吹聴しています。

　まぁ、部分的には同意。部分的には非同意ですな（笑）

　きちんと機能比較すると、PostgreSQL が上回っている部分さえ
ありますからね。ただ、機能数で勝負したら、今のところは商用
のデータベース製品（Oracle や UDB/DB2）が勝つかも。。

> 彼は一応オラクルマスターのシルバーを持っているので、

　周囲には、Platinum や Gold がゴロゴロしてますが、、だから
と言っても、どんな場合でも正しいとは限りませんよ。

> 何らか根拠があって言っているのだとは思うのですが、
> 直接聞く気にもなれません。(^^;

　ちゃんと論理的に聞いてみたらどうでしょうか？

　技術職でも、マーケティング職の私よりも知識の無い輩なんて
業界には腐るほど居る感がありますので。。

> ただ、ＳＲＡさんのページにあるＯＲＡＣＬＥとの限界性能比較みたいなの
> を見ても現在のPostgres７．２がそれほど劣っているとも思えません。

　『Oracle との限界性能比較』なんて出せませんよ（苦笑）。

　オラクル社に許可無くベンチマーク結果を公開した時点でライセンス
違反に問われますからね（本当）。
　MS SQL Server や Sybase なども含めて商用データベース製品では、
ベンチマークテスト結果の公開を「使用権許諾書」で禁止しています。
　これは理由があって、比較する場合に勝たせたい側に優位なテストの
結果を作るのなんて、実は簡単だからです。で、人間は愚かだから結果
の数字だけを真に受けてしまうので。。

# 個人的には、幾つかの大量データ、大量ユーザにおけるテスト結果を
# 知っているので、限界性能は Oracle が上だと捉えていますが。。
 
> そこで知りたいのですが、サポートとか値段とかを一切除いて、純粋に
> ＲＤＢＭＳとして見たとき、ＯＲＡＣＬＥの最上位版との違いって
> なんなのでしょうか？
> 一口にＯＲＡＣＬＥといってもいくつか種類があるんですよね？

　別メールで説明しておきましたので読んでみてください。
　後は、Oracle のホームページでの説明や OTN-J を読むと良いかと。

ではでは。
