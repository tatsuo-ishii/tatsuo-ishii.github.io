> 彼は一応オラクルマスターのシルバーを持っているので、

他の方もおっしゃっていますが、オラクルマスターのシルバーなんて
入門編に過ぎないので、これを持っているからどうのこうのという
ほどの事はありません。
オラクルマスターは、せめてゴールド、やはりプラチナ。でしょう。

もっとも、Oracle について非常に詳しくて実務経験も豊富だけど、オラクル
マスターの資格は何も持っていないという人もまた多いですから、資格を
持っていないからダメだめだとも言えません。

逆に、ここ数年のベンダー資格試験ブーム(?)のせいか、新入社員から
ベンダー資格を取らせよう。という動きもあって、ろくな実務経験も
ないし、その対象をろくに理解しているわけでもないけど、とりあえず
試験は通った。という人が増殖しています。

ベンダー資格試験は実務能力とリンクしてるようには見えない(思えない)
ものも多いので、ベンダー資格を持っているという事は、「その試験に
通った」以上の意味があるとは思えないのですが…

偏見をまじえて言えば、若い人の場合はその人がベンダー資格を持って
いるからといっても、あまり信用できないと私は考えています。

ここでいう「若い人」は、ベンダー資格試験ブーム後に入社してきた
世代を指します。

> 一口にＯＲＡＣＬＥといってもいくつか種類があるんですよね？

ちょっと昔の表現になりますが、基本的には Workgroup Server と
Enterprise Server のふたつに分かれます。
Enterprise はレプリケーションその他、多くのオプションを組み込む
事ができます。どのオプションを組み込むかで構成が変わりますので、
その意味では「何種類もある」とも言えるかも。

PostgreSQL との比較では、Workgroup Server がそれに相当すると
言えると思いますが、これはあくまでも目安程度のものです。
