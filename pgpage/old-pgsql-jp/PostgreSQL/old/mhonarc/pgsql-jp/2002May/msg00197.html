こんにちは．

> インターネットに曝しているマシンでこの設定は致命的に危険です。
> zootのユーザの方なので、常時サーバを接続しているのかと思い指摘しました。
> 
> # trustはコンパイル時に明示的に指定しない限りは無効にすべきだと思う。
> --

件のRPMパッケージは，最初に

/etc/rc.d/init.d/postgresql start

が実行された時に，指定された場所(デフォルトでは/lib/pgsql/data)にデータベース
クラスタの有無を確認してinitdbを行います．その時にpg_hba.confも作成されるわけ
ですが，デフォルトのエントリは，

local        all                                           trust
host         all         127.0.0.1     255.255.255.255     trust

です．特にパッケージが，

host         all         0.0.0.0       0.0.0.0             trust

というエントリを追加するように作られているわけではありません．

ご参考まで．
