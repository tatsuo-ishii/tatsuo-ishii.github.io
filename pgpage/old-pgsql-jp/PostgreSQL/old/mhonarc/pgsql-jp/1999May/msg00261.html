表題が少しへんかもしれませんが^^!

先日から会津大学のたはらさんに、debian の postgres
パッケージにjumbo patchをあて再ビルドを行って
patchをあてたdebian packageの構築を御指導してもらいました。
それを、まとめたので、Upさせてもらいます。
知っている方は無視してください。

Linux distribution : debian 2.1 (slink)
version of Postgres : postgresq_6.3.2-15
マシン:自作マシン、Libretto100

1)  ftp://ftp.dti.ad.jp/pub/Linux/debian/dists/slink/main/source/misc/
から以下をとって来る
postgresql_6.3.2-15.diff.gz
postgresql_6.3.2-15.dsc
postgresql_6.3.2.orig.tar.gz

2) ソースを展開する
ftp でとってきたファイルを置いたディレクトリで実行します．
% dpkg-source -x postgresql_6.3.2-15.dsc

3) パッケージが展開されたディレクトリに移動．
% cd [package-version]

4)[package-version]/debian/rulesのconfigreで以下を追加
--with-mb

4) patch(jumbo.patch)をあてる。
gunzip -c jumbo.patch.gz|patch -p1

5) ビルドする
ソースツリーのトップで以下のコマンドを実行します．
% dpkg-buildpackage -rfakeroot -us -uc -b
ここでmake error no such fileで怒られたらContents-i386.gzからpostgresに関する
packageをgrepで調べてそのpackageを入れる。

私は5)で苦労しました。たはらさんのおかげで、ここまで
たどりつきました。次は6.4を試すつもりです。
