はじめまして。盛川 徹と申します。

現在、ソフトウェア開発の仕事に従事いたしております。
趣味は釣りで関連するHPを作成しており、このページを
充実させるため、今回始めて PsotgreSQL 6.4.2 を導入
いたしました。動作させているマシンは FMV Diskpower
でOSは FreeBSD 2.2.8 でWebServerには Apache 1.3.3
を利用しております。

やりたい事は、Databaseを利用してC言語にてCGIを作成
して、それをグラフ化して表示する機能です。

現在 テキストベースのファイルを読み込み表示している
ところをDBからの読み込みに移植したいと思っております。

さて質問なのですが、test.cgi を作成してこれをWebBrowser
から実行すると以下のようなエラーとなってしまいます。

Internal Server Error

The server encountered an internal error or misconfiguration and was
unable to complete your request.

Please contact the server administrator, usrbin@carrot.co.jp and
inform them of the time the error occurred, and anything you
might have done that may have caused the error.

Premature end of script headers: /home/member/morikawa/public_html/test1.cgi

Apache/1.3.3 Server at bsd.carrot.co.jp Port 80


apache は、最初 nobody権限で実行していたのですが、postgreSQL
をインストールした権限に変更しても正常に動作しません。

test.cgi では PQsetdb しか発行していません。ちなみにこの部分
をコメントにして動作させると正常に動作します。

conn = PQsetdb(NULL,NULL,NULL,NULL, "testdb");

また、nobody は createuser にて Grant にて testdb にあるテー
ブルに対して select 許可にしております。

apacheの動作にかなり依存しているため、こちらのMLに投稿して
よろしいものかと思いましたが、ご享受頂ければ幸いです。

よろしくお願いいたします。
