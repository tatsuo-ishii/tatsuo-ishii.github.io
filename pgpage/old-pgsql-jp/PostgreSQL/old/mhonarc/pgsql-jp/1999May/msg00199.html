会社のWWW-RDBサーバーをMacからLinux（RedHat5.2J）に移行しておりますが
WU-FTPに接続ができなくなり困っております。

やったこと
Postgresql6.3.2→6.4.2にヴァージョンアップ（6.3.2は削除）
Apache1.3.3→1.3.6に（1.3.3は削除せず）
PHP3.0.7を日本語パッチを当ててインストール
wu-ftpのftpaccessファイルにuploadを追加

ここまではwu-ftpにアクセスできました。

ブート時にPostgres、Apacheを起動させるために
次の設定を変更

/etc/r.d/rc.localに下記を追加

POSTGRESDIR=/usr/local/pgsql
if [ -x POSTGRESDIR/bin/postmaster -a -d $POSTGRESDIR/data ];then
	rm -f /tmp/s.PGSQL.5432
	su - postgres -c "postmaster -S -i!
	echo -n 'postmaster '
fi

次に

# cp /usr/local/apache/bin/apachectl /etc/rc.d/init.d/httpd
# cd /etc/rc.d/rc3.d
# ln -s ../init.d. S98httpd

を実行し、rebootしwu-ftpにfetchでアクセスしたところ
「エラー：サーバーから切られてしまいました。たぶんサーバーがビジーのため
です」が表示され、

Linuxからftp localhostとしたところ
「421 Service not available, remote server has closed connection」と表示
します。

プロセス(#ps ax)を確認すると264 inetd SとなっているのでDemonは動作してい
ると思うのですが、解決方法がまったく分かりません。

どうか、解決方法をご教授ください。（Linuxは始めてなのでちょっと詳しめでお
願いしたいのですが ^^;）

よろしくお願いします。
