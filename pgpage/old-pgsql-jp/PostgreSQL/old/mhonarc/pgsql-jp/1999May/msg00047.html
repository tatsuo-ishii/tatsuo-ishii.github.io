PostgreSQL には large object というものがあって、それを使えば画像デー
タに限らずバイナリデータを扱うことができます。今月の中旬に発売予定の
「ソフトウェアデザイン」6月号に、ちょうどPostgreSQL+PHP+Apacheで large
objectを使って画像データを管理するツールの記事を書きました。良かったら
参考にして下さい(一応Cobalt用の記事ですが、Cobaltでなくてもツールは動
きます)。

ただし、large object には制約もあります。

o pg_dumpでバックアップできない
o 非常にたくさんのlarge objectを登録すると性能が悪くなる

というわけで、WWWなら普通にファイルで画像を管理する手もあると思うので、
比較検討してみて下さい。
