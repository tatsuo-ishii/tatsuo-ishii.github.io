PostgreSQL は、日本語コードには EUC か、mule の内部コードしかなかった
はずです。したがって、SJIS の code を PostgreSQL に読ませると、意図し
ない文字列として処理されてしまう可能性があります。
#種々の日本語コードは、 CGI を作る時の泣き所、と私は思ってます。

たしか、PHP + 日本語 patch では、色々な日本語コードを EUC に変換するよ
うな function があったはずです。それを使うと、PHP より PostgreSQL 側で
は、日本語コードを EUC-JP に統一出来るはずです。
