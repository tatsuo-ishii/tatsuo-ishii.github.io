早速おききしたいことがあります。

platform : Linux 2.0.35(Slackware3.5)
JServ    : ver1.0b4
JDK      : ver1.1.6-v5
JSDK     : ver2.0
PostgreSQL: 6.4.2 (1999/05/07 patch済)

以上の環境で、いろいろな資料を参考にservletを作成しております。

WEBからユーザーにデータを入力してもらい、確認の後にDatabaseに
登録するという処理でつまづいてしまいました。
日本語がうまく登録されないのです。
databaseのエンコーディングは EUC_JP を指定してあり、手入力で
入れた日本語データをservletで読み出して表示するのはうまくでき
ています。読み出し部分は次のようにするとうまくいくことが、
いろいろテストしてわかりました。

	ResultSetを rs、PostgreSQL内での項目名を item とすると

	String DB_item = rs.getString("item");
	String j_item  = new String (DB_item.getBytes("EUC_JP"));

	これでブラウザに表示すると正しく表示されます。

問題は書き込みの方で、いろいろやってみたのですがデータベースへ
登録される日本語をどうエンコーディング処理してやれば良いのかが
わからないのです。

どうかよろしくお願いいたします。
