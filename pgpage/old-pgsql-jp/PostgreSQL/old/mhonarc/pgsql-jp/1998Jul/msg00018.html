詳細説明は詳しい方に譲るとして、要求から考えるにbetterと思われる
実現方式のアイデアを提案します。

At 1998/07/02(Thu) 10:48:02, Hitoshi OIKAWA wrote:
>なぜかというと、データベースの抽出結果を、まだグラフ出力
>するとこが出来上がってないので、それまでの間、抽出結果を
>CSV ファイルに落として Excel でグラフ化させようかとおもい
>まして、

   PC-UNIX         Windows95/NT
+------------+  +---------------+
| PostgreSQL |  | Excel         |
+------------+  +---------------+
|      ?     |  | MS-Query      |
+------+-----+  +---------------+
       |        | ODBC manager  |
       |        +---------------+
       |        | PostgersSQL   |
       |        |   ODBC Driver |
+------+-----+  +---------------+
|  TCP/IP    |  | TCP/IP        |
+------+-----+  +-------+-------+
       |                |
+------+----------------+-------+ network

（MS-Query,ODBC managerはExcelのインストール時にカスタムインストール
するか、後からコンポーネントの追加でインストールします。）

CSVファイルが別の目的にも必要であるならば使えませんが、
そうでなければこの構成の方が良いと思われます。

利点
1.操作がとても簡単
　DBの最新情報を入手するのにWindows上での操作のみで完了する。(手順失念)
　次いで、Excelからコマンド一発でグラフが更新できる。

欠点
1.PostgresSQL ODBC DriverやODBC managerの完成度が今一つ。
あ、決して開発者を非難中傷するつもりは全くありません。
