>>>>> On Sat, 18 Jul 1998 14:56:25 +0900, Mitsushi SUGIMOTO <m-sugimt@sra.co.jp> said:

> Jul 17,1998 Infoworld(http://www.infoworld.com)によると、Oracle社は、
> Oracle8のLinuxへの移植を行うそうです。
> また、Infomixも移植を行うようですね。

今月の Software Design の岩谷宏氏の記事の中に、「Linux Journal 6月号
Letters 欄に 'Linux 向け Sybase の開発と提供は、中断していたプロジェクト
が最近また再開した' とある」とあります。

# Linux Journal は 'Japan' のつかないアチラ版です。念の為。

この Sybase に関する情報自身の信憑性は保証できませんが、^^; もし本当だと
すれば、そのうちには Oracle, Sybase, Informix の三大 DBMS が Linux で
動くようになるという事になります。

Oracle で言えば、Enterprise Server とか、Workgroup Server が Linux 上で
動くとなれば、導入を真剣に検討するサイトもあるんじゃないでしょうか。
ソフト的な価格は Sun や HP などのメーカマシン用と変わらなくとも
ユーザから見れば十分魅力がある事になります。

本当にクリティカルな業務でハードウェアの信頼性などが要求されるところ
以外ではハードウェアが PC でいいというのは嬉しい人も多いでしょう。

実際、私の勤務先でもゲートウェイや内部 Mail/News サーバなどは SunOS や
Solaris を搭載した Sun のマシンから PC + FreeBSD という構成に移行しつつ
あるのですが、DBMS マシンだけは FreeBSD などに対応した製品がないので
Sun などのマシンを使わざるを得ません。

個人的には Linux 版よりも FreeBSD 版が出て来て欲しいんですが…
色々なディストリビューションがある Linux よりも、事実上ひとつしかない
FreeBSD の方が製品を出しやすいと思うんだけど… 海の向こうでは FreeBSD
がマイナーだという話しはこういうところからも推し量れますね。:-(

Oracle8 にしろ Informix にしろ、日本語版が出るのがいつかというのも
関心があります。ひょっとして出なかったりして… ^^;
