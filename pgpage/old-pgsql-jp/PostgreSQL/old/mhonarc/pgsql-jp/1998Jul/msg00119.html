Invalid packet length

このメッセージ自体は、バージョンの違う PostgreSQL を混ぜて使ったときに
出て来ます。

まず確認ですが、手順はこれであってますね？

setenv POSTGRES_HOME /usr/local/pgsql
perl Makefile.PL
make
make test
→ここでエラー

だとすると、考えられるのは perl を作る＊前に＊ PostgreSQL 6.3.2 が正し
くインストールされていないことです。特に、$POSTGRES_HOME/lib/libpq.a 
などが 6.3.2 になっているか確認して下さい。また、走っているサーバが
6.3.2 であることもチェックして下さい。

P.S. 業務の都合で 6.1.1 を止めたくない場合は、6.3.2 を別ディレクトリに
インストールした上で別ポートで走らせる方法もあります。必要ならばご質問
下さい。
