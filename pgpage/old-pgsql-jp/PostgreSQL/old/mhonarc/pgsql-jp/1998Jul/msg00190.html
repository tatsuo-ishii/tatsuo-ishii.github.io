再び田端＠印刷局研究所です。度々申し訳ありません。

実は6.3.2に上げると、pgpostが動かなくなってしまいました。
index.phtmlで「検索開始」ボタンをクリックすると、

select.phtml: Could not connect to database (Invalid packet length) 
on line 87
$con = pg_Connect($hostname,"","","",$dbname); 
localhost のデータベース postal に接続出来ませんでした。

あるいは

select.phtml: Could not connect to database (PQexec() -- Request was 
sent to backend, but backend closed the channel before responding. 
This probably means the backend terminated abnormally before or while
processing the request. ) on line 87
$con = pg_Connect($hostname,"","","",$dbname); 
localhost のデータベース postal に接続出来ませんでした。

というエラーが出てしまいます。（この二つのエラーの再現性もまだよくわ
かっていません）

新たにインストールしたのはPostgreSQL6.3.2とpgpost-1.1の二つで、apache
1.2.5とPHP/FI2.0b12は以前のまま変更していません。

立ち上げは postmaster -D/usr/local/pgsql/data -S -i としています。

pg_hba.conf も変更して、host all 0.0.0.0 0.0.0.0 trust のコメントを外
しています。

PostgreSQLをインストールした時のユーザー postgres でログインして、コマ
ンドラインから psql postal を実行すると問題なく検索はできます。また、
ここで

grant select on postal to web;

としてみましたが、やはり駄目でした。

この時点で私にはお手上げ状態になってしまいました。
もしよろしければ何かお知恵をお貸しいただければ幸いです。

なにぶん質問をするのが初めてなので、何か不手際がありましたらお許し下さ
い。なお、実際の検索画面は、

http://www.res-pbmof.go.jp/~tabata/postal/index.phtml

に置いてあります。

それでは失礼いたします。
