その中でわからない事が有ります.教えてください.

perlのバッククォートを使ってpsqlの出力値をリターンする以下のような関数を作ってい
ます.
この関数を使ったmain.cgiの関数を実行したところ，webのurlで直接http://***.co.jp/*
**/main.cgiという形で動作させた場合とコマンドラインからperl main.cgiで動作させた
場合で出力が異なります.

具体的には
perl main.cgiで動作させると$retの戻り値をprintするようなスクリプトで標準出力に表
示されます．

しかし，webのurlで直接http://***.co.jp/***/main.cgiという形で動作させた場合は
$retを上記と同じ方法で出力しているにも関わらず，ブラウザには何も出てきません．

同様にpg_sql()の中に直接print文を入れてみてもやはり，$retの中身はからです.

このような場合どのようにしてpsqlの出力を取り込めば良いのでしょうか？バッククォー
ト以外にもfork等を使う場合もあるのですが，どういう関数にすればperlでpsqlの出力を
取り込む事ができますか？

ご回答よろしくお願いします.


# FUNCTION : Pg_Sql()
# INPUT    : ＳＱＬ文字列
# OUTPUT   : ＳＱＬの実行結果が格納された変数
sub Pg_Sql{
    local($db,$qstr) = @_;   #パラメータを各変数（$db:DB名,$qstr:SQL文）に格納
    $ret = `psql -d $db -c '$qstr'`;  #ＳＱＬの実行(psql) ,実行結果を$retに格納
    return $ret;
}
