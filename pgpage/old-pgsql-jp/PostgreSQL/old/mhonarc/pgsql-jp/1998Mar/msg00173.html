石井さん、いつもお世話になっております。

実は、ＩＢＭのメインフレーム上に定義されているテーブルをそのままの形で
Ｐｏｓｔｇｒｅｓにデータをインポートして、ｗｅｂ上から検索できるように
して、仕事に活用しているのですが、そのテーブルがあるフィールド以降がｃｈａｒ（１）で
１００フィールドほど、定義されていまして。

それで、ある条件で、そのなかのＮフィールド目からＭフィールド目のデータをｓｅｌｅｃｔ
してくるという条件で検索しています。

                      ←−− 100 フィルド程度繰り返し −−−→
  +--------+---------+-----------+-----------+----    --------+
  | a int  |  b int  | c char(1) | d char(1) | ........char(1)|
  +--------+---------+-----------+-----------+----    --------+
 
    create table tbla )
       a int,
       b int,
       c char2 [];
    }

select tbla.c[n * 2] as bbb tbla.c[n * 2 - 1] as ccc from tbla 
where b = 1000  ; 

のような感じです。これは例ですので、実際とは違いますが。


substrを利用する方法も考えたのですが、手っ取り早く配列で定義しました。
