<構成>
FreeBSD2.2.1
PostgreSQL 6.3＋6.3mbPL1.patch
PostODBC version 0.21（1998/03/03版）  

で、ＶＣ＋＋で作成したクライアントプログラムで認証タイプを
ホスト認証にした場合、正常に接続できるのですが、
パスワード認証にしたとき、
OBDBCINFO
Could not establish a communications link to the server.
と表示され接続できません。
パケットモニターで見ると、EInvalid packet lengthという文字列
がクライアントに返され切断されます。
以下の設定をしているのですが、どうしてもつながりません
設定上どこが悪いのでしょうか？

postmsterのオプションスイッチ
/usr/local/pgsql/bin/postmaster -D/usr/local/pgsql/data -i -S

<pg_hba.conf>
local        all                                         trust
host         all         127.0.0.1     255.255.255.255   trust
host         all         xxx.xxx.xx.0     255.255.240.0 password passwd
passwdはpg_passwdで作成。PGDATAは/usr/local/pgsql/dataにセット済み。
passwordをtrustにすると、ホスト認証で接続はできるのですが。。

identモードでも実行したのですが、結果は同じでした。

postmap tomeba7 postgres
postmap postgres postgres
