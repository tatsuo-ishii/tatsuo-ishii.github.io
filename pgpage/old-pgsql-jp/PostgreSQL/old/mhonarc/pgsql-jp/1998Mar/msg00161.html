[pgsql-jp 3593] access control with mod_auth_pg95

始めまして植田@物性研です。

最近、ふとした機会から図書のデータベースを作成を手伝うことになりました。
そのシステムに、UnixMagazine とかで紹介されていた PostgreSQL を使って
行ない始めた関係から ML の方に参加させていただきました。

で、早速質問で恐縮なんですが…。

図書のデータベースという性格上、更新と新規データの追加が必要になります。
しかし、図書の事務を行なっている方々に、「ここにログインして、psql と
いうコマンドを実行して云々」などというような呪文は通じないことが予想で
きるので、出来れば web browser あたりから操作できるような形態に出来れ
ば良いなと思っています。

UnixMagazine 97 年 12 月号には、mod_auth_pg95 を組み込んだ apache を使
用して、新規追加が出来るユーザーを認証する方法が掲載されていたのですが、
この方法では httpd のプロセスの実行ユーザー(例えば nobody)には insert 
や update を行なう権限を与えておかなければなりません。

折角 PostgreSQL 側でユーザー毎にアクセス権限の制御が出来るのだから、

1.nobody には select だけを許可。その上で operator といったユーザーを
追加し、そのユーザーに insert や update などを許可するようにする。
2.このユーザーの認証に mod_auth_pg95 あたりを使用したい

などが出来ないかな？と考えています。一応 .htaccess の Auth_PGpwd_table 
のところに pg_user を書いてみるなんて方法で通らないかな？なんて思い、
試してみたのですが、ものの見事に蹴られました ;-)

さらに以前の ML の議論なども検索してみたのですが、その時の議論としては

1.nobody で運用
2.別口の httpd を postgres で起動しておく

というところで議論が収束していたようで、たしかに 2 はかなり現実的な方
法だとは思ったのですが、それほどリソースの無い機械ではちょっとつらいの
で、手が出しかねています。で、結局 1 の方なのかな…、なんて思っていま
す。

みなさん、何か良い知恵を御教示していただけませんでしょうか？

ちなみに PostgreSQL の verion は 6.3,apache の version は 1.2.5
,mod_auth_pg95 は mod_auth_pg95-0.4.tar.gz に入っていたものを用いてい
ます。御参考までにしていただけると幸いです。
