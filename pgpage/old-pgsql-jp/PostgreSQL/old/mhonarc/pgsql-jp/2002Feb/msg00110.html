そうですね。前にも書きましたが、PostgreSQL の SERIAL
型は、その表に UNIQ な 連番を生成するものではなく、
単に決められた値からの連番を生成するだけなので、外部
からデーターを import したり、SERIAL 型の項目に直接
値を設定して INSERT した場合など SEQUENCE で採番され
る連番と重複エラーを生じます。

SERIAL 型を持つ表について、次のような SQL を用意して
おいて、外部データーインポートの都度実行すると幸せに
なります。

SELECT setval('"jum_juid_seq"', (select max(juid) from jum));
-- 表 jum SERIAL型列 jumid 
-- 既存データ-の最大値にＳＥＱ再設定
