f-msr>   一応、教えていただいたWebマニュアルの方で色々かいてあったんですが。
f-msr>   自分的な感触としては、
f-msr> ・NEXTVALで値を取得する
f-msr> ・NULLを使ったデフォルト設定ではなく、値を明示的にINSERTする
f-msr>   というのが一番確実な方法かなぁ、という感じでした。

pg_dumpして見てみるとわかると思うんですが、

CREATE TABLE a (
       column_a serial
);

として定義したテーブルはPostgreSQLでは

CREATE TABLE "a" (
       "column_a" integer DEFAULT nextval('a_column_a_seq'::TEXT) NOT NULL
);
CREATE SEQUENCE "a_column_a_seq" start 1 increment 1 maxvalue 2147483647 minvalue 1 cache 1;

というように展開されてます。
つまり、nextval()を使用しているという事です。
