JDBCの質問をさせていただいております大出です。

皆さんのアドバイスをもとに作業を行った結果、JDBCによる接続が成功しました
のでご報告し、お礼を申し上げます。ありがとうございました。

改善点を以下に書かせていただきます
。
プログラムjtest.javaをコンパイル後実行できない点について

c:\jbuilder\jdk1.3\jre\lib内にextフォルダを作成し、jdbc7.1-1.2.jarを保存。
こうすることで-classpathオプションを追加しなくてもjtest.javaをコンパイル、実行する
ことができました。

しかし、プログラムを実行した際に、以下のエラーが発生しました。
Exception!
Connection refused. Check that the hostname and port is correct, and that the po
stmaster is running with the -i flag, which enables TCP/IP networking.

この問題の原因としては
１　postmaster起動時に-iオプションをつけていない
２　pg_hba.confの設定がおかしい
３　javaプログラム内での接続先IPアドレス、ポートが違っている（以下の記述)
     String     url = "jdbc:postgresql://192.168.1.1:1025/yasuto";

が考えられると思ったのですが、
３番のurlの記述を以下のように変更（ポート番号の指定を削除)したところ接続
に成功してしましました。
     String url = "jdbc:postgresql://192.168.1.1/yasuto";

２台のマシン間でJDBCを使うときはポート番号を指定する、とさまざまなところ
に書いてあったのですが、なぜか接続されています。。

どなたか心当たりのある方は助言をよろしくお願いします。


結果的には接続できました。ご協力ありがとうございました。
