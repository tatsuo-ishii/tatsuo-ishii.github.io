AccessのテーブルのデータをPostgreSQLに移行するにあたり、

１．Accessのテーブルをテキスト（タブ区切り）で保存
２．文字コードを、EUC、改行をLFにした

上記２工程を行なって、\copyコマンドでテーブルにインポート
しようとしていますが、ERRORがでて、インポートが完了しません。

TELフィールド（もしくは連絡先電話番号）でエラーになっているようです。
電話番号のフォーマットは「0000-00-0000」です。
0000-00-0000がdateの形式でもあるので、その辺で失敗しているとは
考えられないでしょうか。

---- 問題のテーブル構成 -----------
create table WORKER_T (
  ID varchar(20),
  フリガナ varchar(30),
  生年月日 date,
  氏名 varchar(30),
  性別 varchar(10),
  郵便番号 varchar(30),
  都道府県 varchar(10),
  住所1 text,
  住所2 text,
  TEL text,
  FAX text,
  EMail varchar(40),
  連絡先名 varchar(40),
  連絡先郵便番号 varchar(30),
  連絡先住所 text,
  連絡先電話番号 text,
  連絡先FAX text,
  連絡先EMail varchar(40),
  業務内容 text
);
------------------------------------

---- エラー ------------------------------
XXXXXX=# \copy worker_t from telework.txt
\.
ERROR:  copy: line 2, Bad date external representation '0000-00-0000'
PQendcopy: resetting connection
