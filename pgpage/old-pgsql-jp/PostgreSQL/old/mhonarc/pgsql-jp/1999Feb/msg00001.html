最近別の担当者が、おなじ PostgreSQL のサーバに Windows98
からアクセスすることが増え、いくつか思い通りに動かなく
なっています。おかしな質問だったらごめんなさい。

事象
Access から漢字項目を含むレコードを Insert しようとすると
1件あたり、２分ぐらい返ってこない

更新後、ODBC 接続で SELECT すると、無応答となり、DBMS も
死んでる。

DBMSを再起動しようと Windows98 の telnet で接続しようとしても
応答しない。(たぶん、バックグランド処理がCPUをほとんど使ってる)

UNIX 上から psql -f xxxxx  db-name として、Insert を行う SQL を実行して
も、
また、インタラクティブに Insert してもうまくできる。



私自身、Windows系に弱いため、さっぱりわかりませんが、もしかしたら
漢字コードが SHIFT JIS になっているのではないかと思われます。
このような事象の解決方法が論議されたことがあったら、どのへんの
記事なのか教えてください。
