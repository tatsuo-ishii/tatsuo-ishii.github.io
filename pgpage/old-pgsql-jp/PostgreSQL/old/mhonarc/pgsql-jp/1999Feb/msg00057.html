> 別に Solaris も Linux も使用したことがないので

 すみません、私も Linux を入れて、regression test をしてからに
しようと思っていたのですが、なかなか都合がつかないので、ここで
書かせて頂きます。

 # ただいま Sybase と Oracle できゅうきゅうしてます (;_;)


> この差で大きいのは Linux が async でファイルに書き込みするからでは
> ないですか？

 そのほかにも Solaris 7 から PCI Bus Master IDE をサポートする
様になったのですが、これは対応しているチップセットに限定があり
ます。これが使えないと、かなり苦戦するでしょう。


> また Linux は瞬発力はありますが
> 負荷をかけると耐えられなくなる傾向があるようです。
> ちょっとこれを説明した URL は今はわかりません。

 ええ、一つのプロセスに CPU 時間をより与える傾向にあると思い
ます。だからベンチマークなどを実行すると良い結果が得られます。

 SunOS 5 も、ページングのポリシーを変更する事が出来るように
なったので、これを設定すると性能が変化するかもしれません。

 http://www.sun.com/sun-on-net/performance/priority_paging.html

 # これを設定してテストをした場合に、どれくらい効果があるのか
 # 試してみたかったんですが、果たせずにいます。


> このような事があるので
> ベンチマークって個人的にはあまり信用しないようにしているんですが。
> やっぱり自分が普段使用する状態で自分で判断しないと。

 そうですね、ある一面だけを切り出して、それが全てであるかの
ように言うのはよろしくないですね。
 そんなわけで、Linux を常用していない私はあまり比較ができま
せんので、両方を使い込んでいる方に出てきて頂けるとありがたい
です。


 それから、話題は変わりますが、[pgsql-jp 6844] で

> 同じ SysV系でも Solaris の場合は /etc/rc3.d , /etc/rc0.d , /etc/rc6.d
> になると思います。

 とありますが、 Linux は別に System V 系というものでもないで
しょうし、/etc/rc.d/rc?.d/ というのは全然普通でなく、Solaris
のものが普通のはずです。

 # HP-UX 10.20 では /sbin/rc?.d でしたが、11 からは /etc/rc?.d
 # らしいです。
