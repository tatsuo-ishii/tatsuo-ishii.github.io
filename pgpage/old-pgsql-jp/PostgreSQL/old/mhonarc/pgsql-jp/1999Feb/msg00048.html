> ERROR:  parser: parse error at or near "constraint"

> また、簡単に下記のSQLでも同様の結果でした。

PostgreSQLの構文については詳しくないのですが、一般的に制約の指定は（）内に
書くのが正しいと思います。

create table emppay(
    name text not null,
    wage float4 default 10.00,
    constraint empcon check (wage > 5.30 and wage <= 30.00)
);

ちなみに、これが PostgreSQLで通るかどうかは確認していません。
