	PSQLODBC.DLL  98/12/07版 を使用しています。オプションは

	[Driver]
	■Disabre Genetic Optimizer		□CommLog
	■KSQO							□ReadOnly
	□Recognize Unique				□Parse Statements
	□Use Declare/Fetch
	●Maximum
	■Text as LongVarChar
	Chache 100 / Varchar 254 / LongVarChar 4094
	SysTablePrefixes dd_;

	[DataSource]
	□ReadOnly		□Rou Versioning
	□Show System Tables
	●6.4

	となっています。

	テーブルを構成しているカラムは
		num int4 default nextval('id') primary key,
		chk bool default 'no',
		name varchar(256),
		indate date
	といった感じです。

	特定のカラムだけデータ修正できないというわけではなく、全てのカラムで
	「他のユーザーの変更を反映 or クリップボードにコピー」のダイアログが
	出現します。そしてレコードの削除の場合は「他のユーザーが変更してます」
	が出現します。


	最終的には DELETE句でデータを削除できたので現在は問題ないのですが、
	今後こういったことがあった場合のために、是非とも原因と対処法を知って
	おきたいので、どうかよろしくお願いします。
