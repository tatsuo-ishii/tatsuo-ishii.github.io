Mesages-Id: ヘッダにどのような内容のものを付けるかは、MTA (メイル
配信エージェント)の実装および MTA に対する構成定義によります。

ただし Message-Id: は、そのメッセージが世界でただ１つのもので
ある事を保証し、一意的に識別できるものでなければなりません。
この事さえ保証できればそれでよいはずです。

実際には、「世界でただひとつ」を保証するためには、FQDN ないしは
それに準じたホスト名と、日付、MTA 内で処理した連番などを組み
合わせたものが確かだと考えられていて、そのような方法が採られる
事が多いようです。

詳しい事が知りたければ、該当する RFC などを参照した方がよいでしょう。
この件だけなら、RFC822 と RFC1123 くらいで用が足りるんじゃないでしょうか。
あとは、sendmail や qmail のドキュメントとか…

# たまに変な Message-Id: のついたものを流して顰蹙をかう人がいますね。
# Microsoft のとか Lotus など、「独自の世界のグループウェア」みたいな
# ツールの Internet mail 連携機能が変なものをつけて顰蹙をかいまくって
# た事もありました。
