> 「多言語を同時に扱えないOS」の意味がわかりませんが・・・普通は同時に扱えません

 「多言語を同時には扱えない "OS 環境等"」で区切って下さい。修飾は、
念押しで書いてます。

> ディフォルトのエンコーディングとは，環境変数の内容で判断するということですか？

 OS に依ります。UNIX では、LANG 環境変数でロカール情報が設定されるように
なっているのが大勢です。

> これが，後で出てくる，正しくJAVA環境に伝わってないという事ですか？

 Java VM は、C 言語のインタフェースでロカール情報を取得するようです。
C のロカール関数のインタフェースについては、setlocale(3) です。

>>   1. DB とのエンコーディングは EUC_JP で正しい。
> 
> パッチを当てていないと，EUC_JPになるということですね

 パッチなしでは、DB とのやり取りにも

     byte[] eucbytes = "漢字".getBytes();
     String javaString = new String(eucbytes);

の形で扱われるので、-Dfile.encoding=EUC_JP とすると、
暗黙のエンコーディングが EUC_JP になる、ということです。
 それが化けなかったのだから、逆に推測して、"EUC_JP で正しい" と
想像されます。

>>   3. Java は OS 環境から EUC_JP の指示を受け取れていない。
>>
> 
> では，これが，正常に受け取れていれば，-Dfile.encoding=EUC_JPも不用ということで
> すか？

 正常に動作すると思います。が、Java では、JAR ファイルがクライアント
ローカルにインストールされているとは限らないので、クライアントの種類に
よって違う postgresql.jar を取って来ないといけない、ということでは
まずいと思うんですよね。
＃ じゃあパッチの有無でも名前変えろよ、って話もあるけど。
