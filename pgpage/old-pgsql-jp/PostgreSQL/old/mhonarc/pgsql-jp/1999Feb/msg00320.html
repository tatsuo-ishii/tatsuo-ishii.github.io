postgres6.3.2を利用しておりますが、
あさのさんとおなじような現象に苦しんでいます。

便乗して質問させていただきたいと思います。

> 二つの接続が同じＤＢに対してあるとき
> １つの接続から

begin　transaction
update　tableA　set　…

そこに他方の接続から

select　*　from　tableA

とすると
このselectは保留されている
＝transactionの終了を待ちつづけているのです。

同じようにまた他方の接続から
update　tableA　set　…

とした場合にも
このupdateは保留されている
＝transactionの終了を待ちつづけているのです。

このようなひたすらtransactionの終了を待つ接続を
deadlockのときのように一定時間でTIMEOUTにしたいのですが
その方法をご存知の方がおられましたら教えてください。
