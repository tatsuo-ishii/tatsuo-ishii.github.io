すみません。

PostgreSQL 6.4.2 を使っているのですが、
テーブル・ロックでちょっと良く分からないことが
あるので教えてくれると助かります。

ＦＡＱだったらごめんなさい。

二つの接続が同じＤＢに対してあるとき

１つの接続から

begin;
lock a;
...

という具わいにテーブルＡをロックしました。

そのロック中に他方の接続から、

select * from a;

とすると、
この select が止まっている（保留されるの）のですが、
これは正しいのでしょうか？

SQL言語にあまり詳しくないのですが、
手元に ORACLE のSQL言語のマニュアルがあったので 
LOCK TABLE を読んでみると、構文に差があるようですが、
問い合わせはロックしていても可能とかいてあるのですが、
PostgreSQL では仕様が違うのでしょうか？

PostgreSQL の マニュアルの LOCK の説明を読んでも
select が保留されるとは書いてないように思うのですが、
違いますでしょうか？
