いろいろ調べてみましたが、create table で、
フィールドに対し NOT NULL を指定した場合、
 primary key と unique index は、同様の扱いのようです。
また、foreign key はサポートされていないみたいですね。
（PostgreSQL6.4.2 の場合）

create table test ( id int2 NOT NULL primary key );
	||
create table test ( id int2 );
create unique index test_pkey on test (id);

したい場合など、foreign key があると便利なんですけどね...
