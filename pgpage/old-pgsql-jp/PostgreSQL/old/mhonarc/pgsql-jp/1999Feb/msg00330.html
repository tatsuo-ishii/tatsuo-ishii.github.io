　わからないです。(^_^;;

　これだけではなんなので、調べてみました。

postgresPerl-1.3.tar.gz　をキーワードにinfoseek.com で
検索してみると、3件ほど出てきました。しかし、リンク先に
行ってもpostgresPerl-1.3.tar.gzはありませんでした。

http://sunsite.doc.ic.ac.uk/packages/CPAN/authors/id/VKHERA/

というディレクトリに、Postgres-1.4.readme というファイルが
あり、見てみると、

--------------------------------------------------------------------
(略)
NAME
    Postgres - Perl interface to the Postgres95 SQL database engine

SYNOPSIS
     use Postgres;

     $conn = db_connect($database,$host,$port)
       or die "could not connect -- $Postgres::error";

     The parameters $host and $port are optional (or may be undef).
     This will use the default Postgres95 values for them.

(略)
--------------------------------------------------------------------

なんてことが書いてありました。なんかこれって、Pg の古いやつの
ことですかね。そのディレクトリにPostgres-1.4.tar.gz というもの
があって、それがお探しのもののバージョン違いのもののようです。
1.3 は、これの一つ前のバージョンのようですが、それはありません
でした。
Postgres95 用となっていて、なんでまたこれが欲しいのかがわから
ないですが、6.3.2 を使っているのでしたら、
postgresql-6.3.2/src/interfaces/ の下にあるPgを使えばいいような
気がします。
まあ、Pg のほうは、Pg - Perl5 extension for PostgreSQL という
ことで、文法もちょっと違うみたいですが、なんか互換性とかのために
postgresPerl-1.3.tar.gz のほうが必要なのでしょうか。
こういった憶測をこちらがするのもなんなので、これが必要な
背景とかも説明されるといいと思います。

　というわけで、まとめですが、
・6.3.2 ならpostgresql-6.3.2/src/interfaces/ にあるPgを使えば
　よいのでは？
・なぜPostgres95 用のものが必要なのかがわかりません。
・1.3 ではなく、1.4ならあります。

　あと、他の皆様にお願いですが、「自己紹介とご質問」などのような
サブジェクトのまま話を続けると、重要なことが書いてあっても
題名から内容を検索できなくなってしまうので、適当にサブジェクトを
変えて話を続けていただけないでしょうか。
