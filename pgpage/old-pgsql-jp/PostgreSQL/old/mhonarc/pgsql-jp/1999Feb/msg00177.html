「[pgsql-jp 7050] Re: NetBSD/macppc 」なるメールで仰ることに:

> Makefile.shlib
> とりあえずELFの定義を入れてみたんですが、これだとpsql等がstatic linkされ
> てしまいます。
> # FreeBSD 3.Xでも同様のようですが。
> ld がlibpq.so.2.0 を見ていないようです。(libpq.soしか見ていない?)
> そこで ln -s libpq.so.2.0 libpq.so を作っておけば
> dynamic linkされるので、手を入れるとしたら、
> $(shlib)ターゲットにinstall-shlibと同様にsymlinkを追加すればいい?

ELF を採用している system 全般に当てはまるかどうかは分かりませんが、
FreeBSD 3.0 では 〜.so.x.y 形式の shared library 名は認識されません。
（〜.so.x か、〜.so でなければならない）

FreeBSD の場合、もしかすると port の方でこの問題に対応しているかもしれ
ません（ちなみに、port の Makefile ないでは変数を判別することによって
make の動作を切り替えることが可能です）。

＃ 少なくとも、他の software では port 内で target の filename を切り
＃ 替えているものがあります。
