[pgsql-jp 7065] Binary ファイルの判別

  • From: uchida@yrd.suzuki.co.jp (Uchida Hitoshi)
  • Date: Sat, 13 Feb 1999 17:47:35 +0900

内田です。

どこかに、PHP 本体の Copyright とバッティングしなさそうな、出力バッファ
内の文字列が、Plain Text か、それとも Binary File であるか否かの判別ルー
チンで、良さそうなものはありませんでしょうか？

先日「うまくいかない」と報告した PHP 3.0.6 + 漢字パッチの問題を、暇を見
つけて直せないかと考えています。

現在のパッチでは php.h 内で定義されている PHPWRITE マクロが置き換えられ
てしまうため、PHPWRITE や php3_write() を使った出力が、全てEUC 変換をす
る php3_kanji_write() を通過するため、Binary の出力がおかしくなる問題が
あります。

http://sidecar.ics.es.osaka-u.ac.jp/php-jp/archive/msg00552.html では、
gd.c 内部で php3_write() を rwrite() と置き換えて解決していますが、この
方法で解決するのは当然 gd の出力だけです。

解決法としては、以下のようなものが考えられますが、とりあえず (3) をやっ
てみようと思っています。

  (1) GD の場合と同じように都合の悪い出力部分を rwrite() に置き換える
  (2) 変換ルーチンを通るかどうか、ユーザが指定できるようにする
  (3) php3_kanji_write() 内に出力がバイナリかどうかの判別ルーチンを入れる

いまのところ、qkc に使われている判別ルーチンで実験してみて、うまくいきそ
うなのですが、どうも qkc の Copyright はコードの改編 etc を許していない
ようなので、PHP へのパッチには使いにくそうです。

まぁ、自分でコードを書けばいいんですが (^^; この手のルーチンは、先達の手
による出来のいいのがどっかにありそうな気がしますので、まずは聞いてみよう
かと思った次第です。
