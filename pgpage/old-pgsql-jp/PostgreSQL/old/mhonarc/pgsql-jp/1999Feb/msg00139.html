 Turbo Linux には触ったこともないので余計なお世話かも知れませんが...

>> Ｔｕｒｂｏ　Ｌｉｎｕｘ　にはもともとpostgresql が入って居るらしく　/usr/bin
>> にある以前のバージョンのinitdbが動いていたようです。

 OS に付属して来た古いバイナリがパッケージ操作ツールでもきれいに消えて
くれないということであれば、オプションパッケージを /usr/bin や /usr/lib の
直下に置く、というのはそれはそれで 1 つの（Turbo Linux の）ポリシーであると
思うので、

  --prefix=/usr でコンパイルしてしまう

というのも手なのではないでしょうか。GNU ものでは、--prefix= で指定された
位置にある旧いバイナリ等は make install 時にきれいにするのが作法の
ようですし。

＃ ちなみに私は NetBSD で --prefix=/usr/local/postgresql-6.4.2 で
＃ つくりたい人なので、上記のようなポリシーはイヤです。あしからす。
