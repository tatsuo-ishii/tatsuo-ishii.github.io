# 厳密には DECIMAL では DECIMAL(p,s)で与えられた精度を上回る実
# 装をしても構わないことになっているので、必ずしも小数点以下の桁数が s 
# になるとは限らないのですが、PostgreSQLではおそらく NUMERIC == DECIMAL 
# なので関係ないでしょう。

>少し以前に 1.23 が単精度なのか倍精度なのかという話題がありましたが
>6.5からは 固定小数点ですといえばよいのでしょうかね ？

ん？ float4 vs. float8 の議論とは無関係ですよ。もともとSQLでもNUMERIC
とかDECIMALは固定小数点データですから。

単純に 1.23 を評価すると 6.5 でも '1.23'::float8 になるはずです。

numeric として 1.23 を評価するためには、

'1.23'
'1.23'::numeric

とかする必要があるはずです。ただ、Thomas さんが今このあたりに関係する
パーサ部分をいじっている最中なので、最終的に 6.5 でどうなるかまだ良く
分かりませんが。
