市川＠お茶大です．

  ML の話題が 6.5βに移りつつあるところで，大変恐縮なのですが，
jade について教えて下さい．マニュアルの翻訳をされている皆さんは
SGML ソースを書換えていらっしゃると思いますが，どのような環境を
使っていらっしゃいますでしょうか．

  この ML に貢献するような話でもないのですが，一応どういう動機か
ということと，こちらの環境を以下で説明しますので，もしも何か
思い当たる点などあればご教示頂ければと思います．


-*-*-*-*-*-*-*-*-*-

  別な作業で SGML パーザの SP をインストールし直す必要が
あったので，ついでに jade-1.2.1 の設定をし始めました．
本来の作業は SGMLSpm + LaTeX で，DSSSL の機能は今のところ
使っていないのですが，いずれは style sheet 方式に移行する
必要もあるだろうと考えて，PostgreSQL マニュアルの
SGML ソースからの生成を試しております．

  簡単なテストはうまくいったのですが，いざマニュアルの処理を
させてみると，DSSSL 云々以前に，SGML パーザの nsgmls のレベルで，
構文エラーがざくざく出て進まない状態にはまっております．

  こちらの実行環境は

	OS:	Solaris2.6 (Sparc)
	GCC:	2.8.1
	JADE:	1.2.1 (sp-1.3.3)
	DOCBOOK: 3.0	-- 古い方もインストールしました．

で，テストとしてはごく簡単な以下のようなものを試しております:n

    1. Jade-1.3.1 の dsssl/demo.sgm は OK
    2. N. Walsh さんのページにある，jtest.sgm のテストも OK．
       (jtest.dtd, jtest.dsl で RTF にするものです．)
    3. OASIS のページにある at1.sgm も OK．
       (DocBook 3.0 をつかったもので，nsgmls で ESIS 形式に
       するところだけやっています．)
