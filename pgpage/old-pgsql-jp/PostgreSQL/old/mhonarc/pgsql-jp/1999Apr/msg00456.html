# なんでこんな簡単なのができないの？ というのは無しよ。(^^)

--難しかった点--

Debian という特定の distribution のため設定が少しことなっておりました。

(1)apache
   debian の apache は、「www-data」 という account で起動することに
   なっているのですが、 createuser www-data は 「-」が入っている
   ためユーザ登録できないという結果になってしまいました。
   vipw にて 「wwwdata」という account を新規作成しました。
   
   wwwdata:x:33:33:wwwdata:/var/www:/bin/sh
   www-data:x:33:33:www-data:/var/www:/bin/sh

(2)eijiro.tar.gz
   install.sh は、以下の行を修正しました。

   1行目:#!/bin/perl
   7行目:username=wwwdata(これに随分悩まされました(^^)
   25行目:txtconv.pl -> /var/lib/postgres/public_html/eujiro/txtconv.pl

   尚、keyword.php3 の count(er) 関連の行を割愛しました(^^;)

(3)php3
   debian の php3 を使うときは、script の中に以下を埋め込まねば
   なりませんでした。

   dl("pgsql.so");

   user_funcs.php3 の 77行目付近、即ち postgres と接続する付近で
   error がでましたので、上記をそこに挿入したところ、
   接続出来るようになりました。

--最後に(感想とお礼)--

communicator 越しの access はかなり遅いですが、lynx 等の term ベースの
ものを使うと、これはもう完璧な "my dictionary" です。

ありがとうございました。とりあえず御報告まで。
