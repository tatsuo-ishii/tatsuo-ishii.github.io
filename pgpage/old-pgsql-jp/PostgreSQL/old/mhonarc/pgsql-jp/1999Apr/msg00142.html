[pgsql-jp 7951] Q) SQLのパフォーマンスについて

  • From: Takeshi Yamada <ken@tydfam.machida.tokyo.jp>
  • Date: Sun, 11 Apr 1999 14:31:45 +0900
  山田＠町田市です。

  また、初歩的な質問で恐縮ですが、下のように今年と去年を比べる為に、同じ
テーブルを嘗めるような SQL を書くと explain 上は遅い様には思えないのに
異常に遅く思われるのですが、temp ファイルを作って実行すべき物なのでしょうか？
  FreeBSD-current だと、ちょっと大きいテーブルでこれを実行すると、backend
が落ちてしまうので、メモリを食っているのは理解できますが、なにか早く実行させる
良い方法があれば教えてください。

例)

create table sales (
  item_code   char(16),
  day         date,
  value       float8,
  volume      float8,
  unit_price  float8
);

select sum(t0.value), sum(t1.value), 
sum(t1.value)*100/sum(t0.value)
from sales t0, sales t1
where date_part('year',t0.year) = 1997 
and date_part('year',t1.year) = 1998;

(or)

select sum(t0.value), sum(t1.value), 
sum(t1.value)*100/sum(t0.value)
from sales t0, sales t1
where date_part('year',t0.year) = 1997 
and date_part('year',t1.year) = 1998
and item_code = 'xxxx';
