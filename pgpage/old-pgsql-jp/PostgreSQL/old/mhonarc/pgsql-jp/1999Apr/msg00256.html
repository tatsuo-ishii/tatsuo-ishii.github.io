> 話は、ちがいますが、先週から、イメージ的に、ＪＤＢＣインターフェース

> 次に、これをアプレット化し最初はＤＢと同じマシンのWWWサーバに格納しました。
> それを、ブラウザ（NC4.5)で見ると、なぜかうまく行きません。
> security.checkpropaccess.key エラーになります。appletviewerでも同じです。

 たぶん、JDBC マルチバイトパッチのバグです。Connection.java で、

  String encoding = System.getProperty(dbEncodingPropName);

とやっているところがあるのですが、アプレットからは Java のプロパティの
いくつかしか参照できないので、例外が発生します。この部分を try〜catch で
くくって、例外をつかまえてやれば、動作するようになります。
＃ アプレットではプロパティ経由のエンコーディング指定はできない、
＃ ということになります。

 ずっと昔にバグレポートもらってるんですが、アプレット動作させる
環境を用意するヒマがなくて、放置してます。ごめんなさい。

> これは、どうも、Linuxでコンパイルして作成したpostgresql.jarをそのまま
> 気楽にもってきたことに問題があるのでないかと思っています。

 Java なんですから、そんなことはないはずです。
