> 2.12)	When I try to start the postmaster, I get IpcMemoryCreate errors.
> 
> 	You either do not have shared memory configured properly in kernel
> 	or you need to enlarge the shared memory available in the kernel.
> 	The exact amount you need depends on your architecture and how
> 	many buffers you configure postmaster to run with.   For most
> 	systems, with default buffer sizes, you need a minimum of ~760K.  
> 
2.12)	Postmaster を走らせようとすると、IpcMemoryCreate エラーを受け取ります。

	あらかじめカーネルに共有メモリーを持つ設定にしていなかったか、
	あるいは、カーネルに対して使える共有メモリの大きさを大きくする
	必要があります。具体的な大きさは、使っているアーキテクチャと
	postmaster を走らせるときに設定するバッファの数に依存します。
	ほとんどのシステムでは、デフォルトのバッファサイズとして、少な
	くても約760Kが必要です。

> 3.20) I experienced a server crash during a vacuum.  How do I remove
> the lock file?  
> 
> If the server crashes during a vacuum command, chances are it will
> leave a lock file hangin around.  Attempts to re-run the vacuum
> command result in 
> 	WARN:can't create lock file -- another vacuum cleaner running?
> 
> If you are sure that no vacuum is actually running, you can remove the
> file caled "pg_vlock" in your database directory (which is
> $PGDATA/base/<dbName>) 
> 
3.20) バキュームを行なっている時にサーバーがクラッシュする経験をしました。
ロックファイルはどのようにして消せばよいのですか？

	バキューム・コマンドの途中でサーバがクラッシュすると、ロック・
	ファイルを残したままにしてしまうでしょう。再びバキューム・コマ
	ンドを走らせようとすると次のような結果で終ってしまいます。
		WARN:can't create lock file -- another vacuum cleaner running?

	もし、バキュームが実際に走っていないことが確認できれば、あなたの
	データベース・ディレクトリ($PGDATA/base/<dbName>ディレクトリ)の
	 "pg_vlock" というファイルを消すことが出来ます。

> 3.21) What is the difference between varchar, bpchar, bytea, and text?
> 
> varchar correspond to the VARCHAR(n) SQL type and bpchar correspond to the
> CHAR(n) SQL type. bpchar is blank-padded to the specified number of characters
> (n) while varchar is not. Both of them disallow strings longer than the
> specified number of characters.
> 
> bytea is an variable-length array of bytes.   text is a
> variable-length ASCII string. all of the four types above are
> "varlena" types (i.e. the first four bytes is the length, followed by
> the data). 
> 
3.21) varchar、bpchar、bytea、それから、text の違いは何ですか？

	varchar は SQL の VARCHAR(n) 型に対応し、bpchar は SQL の
	CHAR(n) 型に対応します。bpchar は (n) で指定した文字数になるよ
	うにブランク詰めがされますが、varchar ではされません。両方とも
	指定した数より長い文字列は許しません。

	bytea はバイトの可変長配列です。text は可変長 ASCII 文字列です。
	上記の４つの型すべては "varlena" 型です(すなわち、最初の４バイ
	トが長さで、データがそれに続きます)。

> 3.22)  In a query, how do I detect if a field is NULL?
> 
> Postgres95 has two builtin keywords, "isnull" and "notnull" (note no
> spaces).  For example:
> 	select * from tbl where field isnull
> will return rows from tbl where the field is null valued.

3.22)  問い合わせの中で、どのようにして NULL のフィールドを捜しますか？

	Postgres95 は "isnull" と "notnull" の２つのビルトイン・キーワー
	ドを持ちます(注意、どちらもスペースを入れないで)。
	たとえば：
		select * from tbl where field isnull
	は tbl から fieled がヌル値の行を返すはずです。
