この件ですが、どうも current のソースを見るとデフォルトで serializable 
になっているようです。show の表示もそれを裏付けているようです。

test=> show transaction isolation level;
NOTICE:  TRANSACTION ISOLATION LEVEL is SERIALIZABLE
SHOW VARIABLE

よく確かめもしないで嘘言って済みません。

> >実は、autocommitをOFFにする（デフォルトで）という話も同時にでていたので
> >一緒に確認していただけるとありがたいです。（こちらの方がずっと影響が
> >大きくて問題なんだけど）

それと、これも確認したいのですが、

o autocommit on : 単独の SQL 文がそれぞれトランザクションになっている
		  (6.4と同じ動作)

o autocommit off: 明示的に commit を発行しないと更新がDBに反映されない

という理解でよろしいですか？であれば、current のソースではデフォルト 
autocommt on になっています。

というわけで、特に本家に問い質す必要はなさそうです...
