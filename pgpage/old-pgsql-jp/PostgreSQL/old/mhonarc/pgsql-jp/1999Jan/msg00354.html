私の経験では、PostgreSQL 6.4.2 に対して PostgreSQL ODBC ドライバ 6.40.0001 JP で
接続して（プロトコルは 6.4）MS-Access97 のテーブルをエキスポートすると、
下記のような不具合があるように見受けられました。

現象：	エキスポートは正常終了したように見え、エキスポートしたテーブルに対し
	リンクテーブルを張ることは可能であるが、エキスポート元の
	テーブル名/フィールド名に大文字アルファベットが含まれる場合、
	データそのものが見ない。
	また、この場合psqlによるエキスポート先テーブルへのアクセスが出来ない。
	psql では、\d の出力はなされるが、drop, select 等の sql は失敗する。

対処：　PostgreSQL では、テーブルおよびカラム（フィールド）名は全て小文字で
	管理されるため（間違っているかもしれませんが少なくとも区別されない
	とは言いきれません、sql 文で記述する場合は変換されるようです）
	Access 側でテーブルおよびカラム名を全て小文字にした後に再度エキスポーとする。
	私は、間違ってエキスポートしたテーブルを消すために、データベースごと
	消しました。（ODBC 経由では drop table が有効であるかもしれません）
