Miyasaka> Apache_1.3.3 + PHP-3.0.6 を試そうと思い、Solaris 2.6 上でメー

最近 (PostgreSQL-6.4.2 +) Apache-1.3.4 + PHP-3.0.6 を Solaris2.5.1 に
インストールしましたが、特に問題はありませんでした。使いはじめてまだ日
が浅く、あまり使い込んでいませんが、簡単にテストした範囲では、正常に動
いているようです。コンパイル方法は、ほとんど

  http://www.sra.co.jp/people/t-ishii/PostgreSQL/6.4/apache_php.html

に書いてあるままで、変えたのは以下のような Apache のディレクトリ, ファ
イル構成の違いの部分だけだったように思います。

  /usr/local/apache/sbin -> /usr/local/apache/bin
  /usr/local/apache/etc -> /usr/local/apache/conf
  srm.conf -> httpd.conf

Miyasaka> GNU の gdbm-1.7.3 を展開し、すべてのコンパイルが完了したとこ
Miyasaka> ろで
Miyasaka> ld -G -o libgdbm.so *.o
Miyasaka> を実行して強引に libgdbm.soを作成したんですが、bucket.o がな
Miyasaka> いと怒られます。

こちらは、libgdbm.so ではなく、gdbm-1.7.3 から普通にインストールされる 
libgdbm.a を使いました。

Miyasaka> /usr/local/apache/libexec/libphp3.so: シンボル res_search: 
Miyasaka> 参照されたシンボルがありません

res_search は、

  /usr/lib/libresolv.so.2

の関数 (nm /usr/lib/libresolv.so.2 より) で、libresolv.so.2 は、

  /usr/local/apache/libexec/libphp3.so

が必要とするファイル (ldd /usr/local/apache/libexec/libphp3.so より) 
ですが、ldd の結果に /usr/lib/libresolv.so.2 は現れていますか？
