いつもお世話になっています。石川＠豊田町と申します。

PostgreSQL 上のテーブルをACCESS2000でリンクし操作を行っています。

以下のように　SELECT文で　DISTINCT　と　固定値が　含まれるクエリを実行するとODBC呼び出しに失敗します。

SELECT DISTINCT data.rno, data.crno, data.shikenno, data.koumokuno, data.joukenno,'TEST'
FROM data
WHERE (((data.rno)=2546) And ((data.crno)=1) And ((data.shikenno)=1));

もちろんPostgreSQL側では、上のSQL文が動作することを確認しておりますし、ACCESS2000どうしのODBCリンクでは
問題なく動作しております。最終的には下のようにinsertに渡したいのですが、動作しないので困っています。

INSERT INTO datatemp ( rno, crno, shikenno, koumokuno, joukenno, 工程 )
SELECT DISTINCT -;

何かよい解決策はないものでしょうか？　ご教授ください。よろしくお願いします。

環境 PostgreSQL6.5.3(サーバーlinux plamo1.4 )+ODBC(片岡さんの2000/6/17版)+ACCESS2000SR-1(Jet4.0SP4)

テーブルの構造

create sequence data_id2_seq 62362;
create table data
(rno int2,
 crno int2,
 koumokuno int2,
 joukenno int2,
 工程数 int2,
 工程 varchar(30),
 試験数 int2,
 不良数 int2,
 不良内容 varchar(30),
 shikenno int2,
 id2 int4 default nextval('data_id2_seq') primary key
);

create table datatemp
(rno int2,
 crno int2,
 shikenno int2,
 koumokuno int2,
 joukenno int2,
 工程 varchar(30)
);
