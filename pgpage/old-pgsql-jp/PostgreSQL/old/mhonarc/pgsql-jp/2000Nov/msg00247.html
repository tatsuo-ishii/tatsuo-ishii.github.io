こんにちは

>新しくテーブル定義を考えているのですが
>以前どこかで PostgreSQLの text と varchar は text の方が
>効率が良いとかパフォーマンスが良いとか読んだ覚えがあります。
>私には char , text , varchar の使い分けが分かりません。
>char は、指定したバイト数が必ず使用されるのでしょうか？
>そうすると「名前」等の項目は、char(32) とした方が検索
>しやすいのかな？（内部的には長さが決まっている方が楽なはず）
>
>とりあえず標準ＳＱＬの事を考えないのならば text オンリーで
>やった方が良いのでしょうか。
>それとも text 型とは「伝言文章」や「説明文章」のように
>長さが比較的不確定のものに使うべきでしょうか？
>
>みなさんはどのように使い分けされていますか？
>＃PostgreSQL オンリーで使うという前提で。

PostgreSQLの DOC の Chapter 8. データ型の
Table 8-5. Postgres文字型で char,char(n),text,varchar(n) の説明で
text型が『Best choics』と推奨（?）されていますね。

で 私は、index を張らない文字型は text型 に 統一しています。
文字数を考えなくてよいので、便利ですね。

但し OracleとPostgreSQLとの間で，データベース複製が可能な「FCR CrossDB」
http://linux.nikkeibp.co.jp/news2/410.org.shtml

の話を聞いて、これを使えば、外向けには PostgreSQL 内向けには Oracle と
棲み分けつつ連携がはかれるなと考えているのですが、そうするとこのtext型
 は どうすれば よいのかな と悩んでいるところです。
