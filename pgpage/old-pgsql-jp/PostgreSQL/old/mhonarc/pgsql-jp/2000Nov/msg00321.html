Note: To ensure compatibility to earlier versions of Postgres we also continue to
      provide datetime (equivalent to timestamp) and timespan (equivalent to interval),
      however support for these is now restricted to having an implicit translation to
      timestamp and interval. The types abstime and reltime are lower precision types which
      are used internally. You are discouraged from using any of these types in new
      applications and are encouraged to move any old ones over when appropriate. Any or
      all of these internal types might disappear in a future release. 
------------
注：  以前のバージョンとの互換性をとるため、datetime型（timestamp型と等価）とtimespan型
     （interval型と等価）は継続して提供しています。しかし、これらの型は内部的にtimestamp型、
      interval型に変換されるだけのサポートしかしていません。
      abstime型およびreltime型は精度の低い型であり、これも内部的に使われているものです。
      これらの型の新規アプリケーションでの採用は控え、適当な時期に古いものを淘汰する事を
      お勧めします。
      これらの型は、将来削除される可能性があります。
