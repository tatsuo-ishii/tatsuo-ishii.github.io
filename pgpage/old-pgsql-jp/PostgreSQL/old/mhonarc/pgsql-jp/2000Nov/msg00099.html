> 私の環境ではDBサーバーとそれをアクセスするCGIのサーバー
> は別になるので、DBサーバーにPostgreSQLをインストールしたあと、
> CGIサーバーにPerlインターフェースだけインストールを試みました。
> が、この場合だとmakeを実行した時に以下のエラーになってしまいます。
> 
> Pg.xs:16:libpq-fe.h:No such file or directory

UNIX だどうのという話は何も関係ありません。これはどの OS … Windows 系
であろうが、メイフレームであろうが同じです。

Pg.pm が一体どのようにして PostgreSQL サーバと通信を行うのかと
いうのを理解していればわかる話です。

Pg.pm は、PostgreSQL が提供するライブラリを利用して PostgreSQL サーバと
通信を行うので、少なくともそのマシンに PostgreSQL クライアント開発環境…
すなわち、ヘッダファイルとライブラリが無ければ make できません。

スタティックリンクした Pg.pm を作るのでなければ、実行環境としても
PostgreSQL の共有ライブラリが必要になります。

ただ、それだけのことです。

> この結果からは、Perlインターフェースをインストールするには
> Postgres本体のインストールも必要と言うことになります？
> postmasterを立ち上げる必要の無いDBクライアント側に本体を
> インストールするのはかなり無駄が大きいのではないかと思ったのですが、

postmaster を動かす必要も無ければ、PostgreSQL のすべての環境が
必要なわけではありません。

ただ、PostgreSQL クライアントプログラム開発環境があればいいのです。
