[pgsql-jp 18695] PostgreSQL （psql）でカラムリストがとれない

  • From: Shoji Kawano <shoji@Macos.com>
  • Date: Wed, 15 Nov 2000 11:22:15 +0900
河野＠MacOS.comです。

VineLinux ppc上でインストールしたPostgreSQL7.0.2において、不具合と思われる現
象がありましたので、同環境のみなさんの意見をお聞かせください。

環境：

　本体    ： PowerMacintosh G3/233
　Version ： VineLinux ppc 2.1（および2.0）

現象：

　psqlおよびpgaccessにおいてテーブルのカラムリストを取得できない。
　（ERROR:  getattproperties: no attribute tuple 1259 -2）

インストールしたのはVineLinux ppc 2.1と共に配付されているRPM版のPostgreSQL
7.0.2です。通常通り、ユーザとデータベースを作成し、psqlによってテーブルを作
成しました。

作成したテーブルはテスト用で以下の通りです。

create table test (
    test text
);

ここで、\d test としてカラムリストの取得を試みると

ERROR:  getattproperties: no attribute tuple 1259 -2

というエラーが出てカラムリストを得る事ができません。

psqlの問題かを確かめるために、pgaccessまたは他のディストリビューションで構築
されたpsqlからアクセスしたのですが、同様の問題が起きましたので、PostgreSQLの
構築時の問題ではないかと思います。

VineLinux 2.0へのインストールでも同様のエラーが表示されます。また、SRPMから
の再構築も行いましたが、こちらも同様のエラーが表示されます。

みなさんの環境ではうまく動いていますでしょうか。また、何がうまくないのかをお
教えいただけたら幸いです。

＃ i386の方は問題なく動作しています
