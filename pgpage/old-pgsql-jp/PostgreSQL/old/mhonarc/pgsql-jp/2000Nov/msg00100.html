みなさん　こんにちは　山口と申します。

Postgresが私の会社で次第に認識され定期メンテナンスが必要となりました。
そこで毎晩　pg_dumpでバックアップをしたいのですが、以下のスクリプト
を実行するとPATHが切れていないようなエラーメッセージが表示され
うまくいきません。ちなみに同じ作業を手動でやるとうまくいきます。
うまくいけばcronにセットして自動化したいのですが・・・

過去のログを探したのですが、どうにも解決しないのでご存知の
方がいらっしゃれば教えてください。

スクリプト内容（ユーザーはrootで実行）
#!/bin/sh
su - postgres
/usr/local/pgsql/bin/pg_dump kosyo >/home/postgres/kosyo.dat

エラーメッセージ
/usr/local/pgsql/bin/pg_dump: error in loading shared libraries
libpq.so.2.0: cannot open shared object file: No such file or directory
