bashの組込みコマンドをリネームするのはつらい選択でしたが、
直接SQLの便利さを優先するために、いたしかたないと思いまし
た。pgbashでは、次３つのコマンドをリネームしています。

　bashのdeclareコマンド ==> declares
  bashのsetコマンド     ==> sets
  bashのselectコマンド  ==> selects

> set を独自に実装するのを止めて、Dynamic Variable Extension
> (bash-2.03/variable.c の 870行前後を参照)を利用して、
> 
>   EXEC_SQL_OPTION="-d db3";
>   select * from test3;
> 
> とするのはどうでしょうか？
> 
> 変数に値をセットした時点で引数解析して、内部に値を持っとく。

なるほど、"引数解析"の方法は何かに利用できるかもしれません。

ただ、set に関しましては、set connection、set tranzaction 
のようなSQL構文もありますので、"引数解析"の方法では解決で
きません。
