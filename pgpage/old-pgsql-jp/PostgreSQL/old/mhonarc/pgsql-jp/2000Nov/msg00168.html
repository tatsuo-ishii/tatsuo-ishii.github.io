| pgbash では、予約語や組み込み関数の名前の変更をソースへのパッチで行なっ
| ており、また、直接 SQL から exec_sql への変更も固定の文字列に対して行
| なっていますが、それらを動的に変更させる C 言語の API を用意して、それ
| らの API を呼ばなければ bash オリジナルとまったく同じ動作するようにさ

今回のパッチは、再利用可能にする前の準備段階です。API の定義はこれから。
作業は土日に限定するつもり。

ところで、直接 SQL でオプションをつけられるようにするとうれしいことあ
りますか？

直接 SQL で、
    select -d db3 * from test3
とすると、
    exec_sql -d db3 "select * from test3"
を実行したのと等値にするとか。

select, insert, update, delete の次に - が来るような SQL を発行するこ
とはないと思うので、構文的には大丈夫だと思う。実装的には、パッチ後の
parse.y の exec_sql_get_argument() を拡張すれば大丈夫でしょう。
