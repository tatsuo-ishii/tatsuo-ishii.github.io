ドライバが実行している環境の文字コードしかサポートしないのなら
Remote Method Invocation (RMI) を使ってバイト配列でサーバーに渡して
サーバー側のプログラムでSQL文を発行すればいいというわけですか。
誰か試したことのある方はいらっしゃいますか？
感想をお聞かせ願えれば・・・うれしいです。

> バイト配列で扱えば問題は発生しないと考え PreparedStatement で
>     PreparedStatement ps = db.prepareStatement("insert into basic values (?,?)");
>     ps.setBytes(1,byte [] b);
>     ps.setBytes(2,byte [] b);
>     ps.executeUpdate();
>     #カラムのタイプはともにvarchar()
> を試しましたが、これでもエラーになってしまいます。
> あと、このように書くとlargeobjectを呼ぼうとするらしいのですが、
>     import postgres.largeobject;
> としておく必要があるのでしょうか。
エラーの内容は
FastPath call returned ERROR: Io_write: invalid large obj descriptor (0)
で FastPath と聞くと連想するのが ドライバの JDBC API の拡張なんですが、
マニュアルの「Chapter 58. JDBC Interface」の「Postgres Extensions
 to the JDBC API」を読んでも PreparedStatement に関する記述は
ありませんでした。なぜ FastPath がエラーだといっているのでしょう。
