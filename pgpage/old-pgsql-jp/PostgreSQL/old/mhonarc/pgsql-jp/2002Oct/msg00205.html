> ご指摘等いただきましたら幸甚に存じます。

   FATAL 1:  could not read configuration file `postgresql.conf':
        Invalid argument

なのですから、postgresql.conf が読めないのでは？

私も Windows 版なんかは使ったことがないので、どこでどう指定するの
かは知りませんが、コンフィグレーションファイルにしろ、DB ファイル
にしろ、どこにあるのかを教えなければ postmaster がわかるはずも
ありません。

Windows 版依存部分のドキュメントをよく読めばたぶんわかるでしょう。
っていうか、ドキュメントにはわかるように書いてあるだろうと思うけど。

情報を探す時は、二次情報以降のものでなく、一次情報を探すように
した方がよいです。この場合の一次情報は、パッケージに同梱されて
いるドキュメントになると考えます。

# 私見ですけど、この場所(ML)では、Windows 版なんて "ド" マイナー
