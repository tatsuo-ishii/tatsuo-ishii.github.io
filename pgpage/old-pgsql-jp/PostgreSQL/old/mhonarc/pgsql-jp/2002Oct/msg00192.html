  Mac OS X 10.1.5 では、PostgreSQL を動かしたままで、再起動とシステム終了した
場合に、Mac OS X のカーネルのバグのために PostgreSQL の終了処理が行われません。
このため場合によっては、データベースの整合性が崩れることがあり得ます。再起動と
システム終了の前に、PostgreSQL を終了させれば、問題なく PostgreSQL の終了処理
が行われるので大丈夫です。

  Mac OS 10.2 で、このバグは修正されました。自宅のマシンでは、マシンの起動時に
PostgreSQL が立ち上がるように設定して、何十日の間、ちゃんと動いています。

;;; あえてUnixを避けていたのですが、そうも言ってられない状況になってきました。
;;; Mac OS Xなりの流儀があるようですので情報を集めつつ、こちらも勉強していく予定で
;;; す。

# Mac OS X なりというならば、WebObjects/Java + PostgreSQL (あるい他のデータベー
# ス) かなと思いました。

  Mac OS X も UNIX なので、Mac OS 9 に比べるととても見通しがよくて、とてもやり
やすいです。今は、PostgreSQL と PHP も特に問題なく動かせます。
