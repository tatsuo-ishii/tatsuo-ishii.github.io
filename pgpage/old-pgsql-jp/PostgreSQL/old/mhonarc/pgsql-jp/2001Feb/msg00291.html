[pgsql-jp 20048] 配列をWHERE句に渡すには？

  • From: "Shohei Sato" <shohei@shohei.ota.tokyo.jp>
  • Date: Wed, 21 Feb 2001 03:06:55 +0900
初めて投稿させていただきます。PostgreSQL初心者の 佐藤＠でぶちん と申します。

今回配列について分からない点が発生したため、手段がありましたら、力添えお願
いしたく思います。
配列に関してなのですが、

例えば
一つ目のテーブル
create table test1
    key1 varchar(8),
    key2 varchar(8)
);
insert into test1(key1,key2) values('00000001','A');
insert into test1(key1,key2) values('00000002','B');
insert into test1(key1,key2) values('00000003','C');
insert into test1(key1,key2) values('00000004','D');
二つ目のテーブル
create table test2(
    key1 varchar(8),
    key2 varchar(8)[]
);
insert into test2(key1,key2) values('X','{"00000001","00000002"}');
insert into test2(key1,key2) values('Y','{"00000003","00000004"}');
insert into test2(key1,key2) values('Z','{"00000001","00000004"}');

と二つのテーブルが有った場合に、test2の'X'を検索条件にして、配列
{"00000001","00000002"}を抽出して、この結果を検索条件にして、test1から
'A','B'の二行を抽出するようなことはできませんでしょうか？

とりあえず、以下のようにSQLを書いてみたのですが、エラーになってしま
いました。
SELECT key2 FROM test1 WHERE key1 IN (SELECT key2 FROM test2 WHERE
key1='X');
エラーメッセージは  Unable to identify an operator '=' for types
'varchar' and '_varchar' .... です。

上記のように配列の出力を WHERE句に渡す方法がありましたらお教え願いたく
思います。
