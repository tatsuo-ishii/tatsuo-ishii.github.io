SJIS <-> UNICODE　でIBM拡張漢字に対応させました。
http://www.sankyo-unyu.co.jp/Pool/PostgreSQL.htm
に差し替えファイルがあります。
UNICODEのサイトにあるSHIFTJIS.TXTはUNICODEバージョン1以前のものでIBM拡張漢字
に全て対応していません。そこで、同サイトの比較的新しいテーブルCP932.TXTをも
とに作成しました。
これでUNICODEベースのDBの構築も可能です。

「テーブルは変更しない」つもりではいたのですが、明らかに対応できていないので
やも得ず別のテーブルを使ってしまいました。(石井さんゴメンナサイ・・・・、で
もUNICODEのサイトにあるものを使っているので、許して!!)

しかし、IBM拡張漢字は新しい第3・第4水準JIS漢字(JIS X 0213)にコード領域を奪わ
れ、SJISやEUCのコードから外されます。
既にktermなどもJIS X 0213を採用されています。
まだJIS X 0213とUNICODEの対応表がUNICODEのサイトにはないようなので、今回IBM
拡張漢字で対応させてみました。

と、言うことでJIS X 0213 <-> UNICODE対応表が公式にUNICODEサイトで公開されれ
ば、対応を考えたいと思います。
その日の為に独自に調べていますが、JIS X 0213 に無いIBM拡張漢字はフォントを見
比べただけでも約90文字ほどあります。
またUNICODEに無いJIS X 0213もかなりの数がありそうです。
