長さゼロの文字列などの場合ですか。ふむ、そういう動きになってたかなぁ。
8.0.5 で試しましたが、長さゼロの文字列を insert するとそういう動きに
なりますね。

しかし、「長さゼロの文字列が NULL になる」というのは本来的にはやはり
おかしいと考えます。
長さゼロのデータが存在する事とデータが存在しない状態は同一ではないからです。

Oracle の場合、「長さゼロのデータはすべて NULL」と判断してるのかなぁ。

私はあるカラムを NULL にしたい時などに長さゼロのデータを挿入しようとした
ことはないし、そもそも長さゼロのデータを自分で積極的に入れようとした事は
ないため、Oracle を使っていても気づかなかったな。

ただ、私が前の記事を書いた時には長さゼロの文字列などの事は頭になくて、
あくまでも空白と NULL の話だと思ってました。
