私も RDB で空白と NULL を同一視する製品があるのか知りたいです。
近藤さんがおっしゃるように、それは RDB としてはおかしいと
考えるからです。

ただ、RDB 以外の DBMS であれば、空白と NULL を同一視するものは
存在するでしょう。
元記事の人は、それを混同なさったのではないかとも思いますが…

> ただし、RDB の NULL が、わかりにくいのは確かです。
> 
> 私は、通常のＤＢ設計では、できるだけ NOT NULL 制約
> と DEFAULT をつけて、0 や 空白文字列 を、設定値未定
> の意味で使えるようにしています。この方がプログラムも
> 組みやすいことが多いです。

「空白というデータ」と、「データが存在していないという意味の NULL」を
区別したい時もあります。

私は個人的には RDB の NULL がわかりにくいとは思いませんが、これは
人それぞれなのかも知れません。
