> いつもお世話になります。

> 
> Postgresの大文字小文字がうまく処理できなくて
> こまっています。
> たとえば、
> [postgres@ns1 postgres]$ dropuser PC0451041
> DROP USER
> [postgres@ns1 postgres]$ createuser PC0451041
> CREATE USER
> [postgres@ns1 postgres]$ psql remon
> remon=# alter user PC0451041 with password 'AA12345';
> ERROR:  ALTER USER: user "pc0451041" does not exist
> となり
> 大文字小文字がうまく処理できないようです。
> ほかに、大文字のテーブル、フィールドも作成できません。
> 大文字のデータベスだけは作成できます。
> 設定の仕方等よいアドバイスがあればおしえてください。
> よろしくおねがいいたします。

識別子は自動的に小文字にされてしまうので，大文字と小文字を混在させるで
あれば，"" でくくって "PC004" などとすれば pc004 と区別してくれます．
(delimited identifier と呼ばれます．)

なお，ユーザ名について同様だと思います． # ちょっと自信がないですが．
