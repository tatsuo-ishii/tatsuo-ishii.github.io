こんにちは、徳家です。

 ftp.postgresql.org/pub/odbc/versions/psqlodbc-07_01_0003.zip で
ダウンロードしたpsqlODBC(当然英語版)なのですが、SERIAL型を
持つテーブルに対して若干の問題があるようです。

たとえば、7.1beta5のノンマルチバイトにてmakeされたもので、

CREATE TABLE test ( id SERIAL PRIMARY KEY, txt TEXT);

と言う簡単なテーブルを作ったとします。
これをpsqlODBC7.1.0.3 で Access2000 にリンクテーブルとした場合。

このテーブルを開いてtxtのフィールドに"ABC"を入力してエンターを押すと、

id も txt も #Deleted になります。
(一度テーブルを閉じてまた開くと値は正常に入ってます。)

ちなみに同ドライバで7.0.3に同じテーブルを作ってテストしても
この様な問題は起きません。

何か新しい設定でもあるのでしょうか？
