> ただ、Access にアタッチした MS-SQL Server の NOT NULL 制約のカラムへ、長
> さゼロの文字列を入れようとすると、エラーになったような記憶があります。
> (まぁこれは、SQL Server ODBC の仕様だったのかも)

　クライアントのモジュールやミドルソフトウェアの仕様によって、
本来の RDBMS エンジンの特性と違う動きになる場合があるの
で、そこは注意しておいた方が良いと思います。

　Oracle でも RDBMS エンジン内部では桁数を厳密に保持する
傾向がありますが、ミドルソフトウェア（DDE Manager は駄目で
あった記憶が）によっては桁数を勝手に丸めたりします。
　同様に char や varchar、NULL への設定制御もミドルソフト
ウェアが勝手に変換している可能性もあるので要注意です。

　そういう意味では、純正（その企業提供）ミドルだろうと信用を
しすぎるのは危険です。

　PostgreSQL も注目され、ミドルソフトウェアが揃ってくるなら
このような点でも注意が必要ですね。
