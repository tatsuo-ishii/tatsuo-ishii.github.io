余談なのですが・・。

> しかし、「長さゼロの文字列が NULL になる」というのは本来的にはやはり
> おかしいと考えます。
> 長さゼロのデータが存在する事とデータが存在しない状態は同一ではないからです。
> 
> Oracle の場合、「長さゼロのデータはすべて NULL」と判断してるのかなぁ。

昔、MS-SQL Server (バージョン忘れ 6 か 6.5 あるいは Access)のオンラインマ
ニュアルを見ていたら、「MS-SQL Server(Accessかも)では、長さゼロの文字列と
NULL は『完全に別の物として扱います』」(『』は斜体)というように、やたら強
調していましたが、Oracle を意識してのことだったのかな？と思ってしまいまし
た。
ただ、Access にアタッチした MS-SQL Server の NOT NULL 制約のカラムへ、長
さゼロの文字列を入れようとすると、エラーになったような記憶があります。
(まぁこれは、SQL Server ODBC の仕様だったのかも)

完全に余談です。聞き流してください。汗
