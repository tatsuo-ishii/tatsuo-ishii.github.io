便乗でお聞きします（＾＾;

> > ｄａｔｅ型のカラムに空欄を入力することは出来るのでしょうか？
> 
> date 型のカラムに、「空白文字」を登録することはできないようですが、
> 代わりに null を使用されてみてはどうでしょうか。

if( データが空なら )	null を代入
あるいは
if( データが空なら )	insert, update でカラムを対象にしない

というのは sql 前のスクリプト処理でなんとかできるんですが、困ったのが 
copy コマンド。

これも元データの date 部分が空の時はエラーになってしまいます。
スクリプトでせっせと対応部分を null にするように書けば良いんでしょうが、
テーブル毎に書く手間がバカになりません。
他からインポートするときに困ってしまいます。（結構、歯抜けのが多い）

こうしなければ問題がある、ということがあるんでしょうか？
ブランクを許すカラムでは空白を受け入れても良いように思うんですが。
'' とブランクデータに引用符付けると perse error になるのも不便です。

この辺、許してくれると随分楽になるとは思いますが、そうするとまずいこと
になる、というのはどういう場合なんでしょうか？
どうも認識不足で思いつきません。ご教示頂ければ幸いです。
