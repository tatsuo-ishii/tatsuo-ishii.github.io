　個人的な意見ですが、、SQL92 のフル実装や SQL99 を実装
するのは、当面無理なんじゃないかと捉えてます。
　現実にある RDBMS を見る限りでは SQL92 Entry を実装する
のにも精一杯な感がありますので。。

　でも、多くの RDBMS がSQL99 Core は実装して欲しいとは思い
ますけども。。

　で、話は変りますが、趣味（笑）を兼ねて、Oracle と PostgreSQL
の比較を作ってみてます。これって他の方にも参考になりますか？

　・decode は case に書き換える、case の方が判り易い。
　・NUMBER 型は存在しないので適切な数値項目にする
　・MVCCの考え方は Oracle と同様だがさらに rbs の大きさに
　　依存しないので便利
　・件数指定の検索は PostgreSQL は非常に簡単。 Oracle は
　　不便。rounum と副問い合わせを駆使しないといけない。

と言うような事を元オラクル使いとしてメモしてます。
