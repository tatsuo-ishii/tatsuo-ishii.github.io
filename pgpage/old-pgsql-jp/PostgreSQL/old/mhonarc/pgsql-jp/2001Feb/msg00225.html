すると、INSERT INTO hoge(foo) VALUES('  '); と、
空欄('') でなく、空白文字列('  ')を挿入したらどう
なりますか？。

> > 同じ行が選択されるとは信じられない。本当に Oracle って
> > そんなへんなことをするんでしょうか？。
> ちょっとは変かも。

そうですね。
ANSI SQL の CHAR(x) の挿入/比較は、x で指定した長さより
短い場合は、長さが x になるように末尾に空白が補われ
なければならないです。COBOL の文字列の扱とおなじ。

ANSI SQL が手元にないので、古いですが、JIS X 3005
(1987年)だと、8.7<INSERT>文の 8.7.4 の(7) の (a),(b)
に上記の主旨のことが規定してあります。

まあ、ANSI SQL なんて、既存の各社の RDB 問合わせ言語
をごたまぜにしてでっちあげたのが発端みたいなもんですが、
規格も年々整備拡張されているし、仮にも、国家の定めた
規格であり、かつ事実上の標準となっているものですから、
だまって、SQL といえば、ANSI/ISO/JIS SQL のことと解釈
すべきだと思いますが如何でしょう。

無論、規格の類を尊重するか、無視するか、厳格に守る
かは、各自/各社の自由ですが。

たいていのＤＢ製品には、ANSI SQL xxx 準拠とか、水準xx
に準拠、あるいは、サブセットに独自拡張などと記述して
あるものですが、Oracle には、なんて書いてあるんでしょう。
 
# 昔、情報処理技術者試験の COBOL は、JIS 規格で採点
