(snip)

> > ブランク('' 等) と NULL とは違います。
> RDBにもいろいろありますが、''とNULLを同一視するものもあります

RDB で、 '' と NULL を同一視している製品があるので
すか？。是非、お教えください。

''  と NULL を同一としたのでは、Codd の RDB の定義に
反するのでは。ANSI SQL にも違反すると思いますが。

> > SQL の集合関数では、NULL 型は無視するようになって
> > います。
> といった一般化がされていると乗り換え組や、場合に応じて使い分けたい組
> からすると多少の抵抗は感じます。

暗黙の了解として、SQL と言えば、ANSI SQL のことであり、
個々の製品で、ANSI SQL に違反して実装している場合は、
製品名を明記して、『 hogehoge の SQL では』という表現
にしてもらわないと、混乱するだけでしょう。

ただし、RDB の NULL が、わかりにくいのは確かです。

私は、通常のＤＢ設計では、できるだけ NOT NULL 制約
と DEFAULT をつけて、0 や 空白文字列 を、設定値未定
の意味で使えるようにしています。この方がプログラムも
組みやすいことが多いです。

また、SQL レベルでなく、言語レベルで、NULL 値を、
ブランクや 0 として便宜的に扱うのも、便利です。但し、
IS_NULL のような関数などで、NULL 値かどうかのチェック
も可能になっていることが条件ですが。
