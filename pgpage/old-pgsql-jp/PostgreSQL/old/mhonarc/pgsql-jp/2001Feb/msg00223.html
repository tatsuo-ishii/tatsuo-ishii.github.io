実は現在、次のような少し複雑な手順で、印刷をしています。
まずデータベースにあるデータを php を用いて html ファイルにします。
それをダウンロードして、その html ファイルを自作の C プログラムで
TeX ファイルに変換します。
最後に、その TeX ファイルを印刷します。

この方法ですと、TeX のDTPの能力がフルに
(自分の知識の及ぶ限り、という意味ですが))利用できるので、
印刷結果には満足できる質のものが得られます。

しかし問題は、この方法は TeX を知らない人が、使えない事です。
と言うのは、自作の html2tex  変換プログラムのできが
(素人が作成したものなので)まだまだなレベルであることと、
もっと根本的には、データベースに蓄積された
文字列データにときどき変なモノが入っている
(改行マークが無意味にたくさん入っているとか、TeX のエスケープコード
が入っているとか、等)
ことにより、最終的に印刷する前に、変換プログラムが出力した TeX の
ソースファイルを一部分、手作業で修正することが必要になるからです。

やはり TeX ではなく他の方法を、
と考えて今回、質問をさせていただきました。
PDF ファイルに変換するやり方は、とても良さそうに思えますので、
これから勉強してみます。
大変ありがとうございました。
