島村です。

いま、PostgreSQL 7.2.1 と PHP 4.2.2 の環境で作業をしているのですが、スケ
ジュール帳の設計で考え込んでいます。

create table my_calendar (
  yy smallint,
  mm smallint,
  dd01 smallint,
       :
  dd31 smallint,

  primary key (yy, mm)
)

と、オーソドックスなスタイルを考えています。（日付の部分には該当する処理
コード番号を入れる予定なので smallint のみです）

これでやっていくと、日付の展開が非常に面倒くさいことになってしまいます。
「配列」の発想で PHP とうまく連携が取れるような形にするには、どのように
テーブルを設計するのが良いでしょうか？
