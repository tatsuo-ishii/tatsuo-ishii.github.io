>■ 一意性制約(UNIQUE):これは、表のどのような列または別の列の組合せに対し
>ても指定できる。そのような列や列の組合せ内のすべての列は、NOT NULLも指定
>しなければならない。

今手元にないので記憶で言っているのですが、SQL92完全ガイド(トッパン)に
よれば、UNIQUE 制約に NOT NULL が必須だったのは SQL89の話で、SQL92では 
NULL を含むカラムに対する UNIQUE が許されるようになったとあったような。

で、富田さんの件ですが、

>SQL では、NULL = NULL は成り立たないので、NULL が複数行にあっても
>UNIQUE の影響は受けないはず… と私は思ってました。

上の本に「UNIQUE カラムでは NULL が一つ以上あってはいけない」と書いて
あった記憶があります。

うーむ、Standard ではどう定義しているのだろう...
