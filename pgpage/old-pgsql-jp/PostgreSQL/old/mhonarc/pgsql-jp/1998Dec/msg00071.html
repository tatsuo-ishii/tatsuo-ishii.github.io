GCC の distribution においては、/usr/local/bin/gcc は

  GNU C Compiler（狭義の gcc）
  GNU C++ Compiler（いわゆる G++）
  Objective-C Compiler（NEXT からの contribution）
  C Pre-processor

の frontend に過ぎません。/usr/local/bin/g++ は、（乱暴な言い方をすれ
ば）source code の default language が C++ になっている程度で、実態と
しては /usr/local/bin/gcc と殆んど変わりません（同様のことは 
/usr/local/bin/c++ にもいえます）。

＃ /usr/local/bin/gcc が呼び出す、pre-processor、各言語用の frontend 
＃ および backend（共通）、専用 library などの実体は、特に指定しない場
＃ 合 /usr/local/lib/gcc-lib/i386-unknown-freebsd2.2.1/2.8.1（ 2.8.1 
＃ を FreeBSD 2.2.1-RELEASE に install した場合の default）に格納され
＃ ます。

逆に注意しなければならないのは、GCC を build する際に lang に指定する 
frontend から意図的に外した言語は compile できなくなることです（何も指
定しない場合はすべての言語を指定したものと見做されます）。例えば、lang 
に C のみを指定すると、/usr/local/bin/gcc で C++ の source を compile 
することはできません（同時に /usr/local/bin/g++ も作成されなくなるはず
です）。ですから、/usr/local/bin/gcc で C++ の compile ができるとは限
らないことになります（build 時の指定に依存します）。


ちなみに、FreeBSD に標準で附属する /usr/bin/cc や /usr/bin/c++ は、そ
れぞれ /usr/bin/gcc、/usr/bin/g++ と全く同じもの（hard link）です。

＃ ただし、これらは通常とは異なる方法で build されたもので、GCC を普通
＃ に install したものとは質的に違います。
