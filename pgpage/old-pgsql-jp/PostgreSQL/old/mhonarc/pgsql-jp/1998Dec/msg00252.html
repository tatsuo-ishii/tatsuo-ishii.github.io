At 10:32 AM +0900 98.12.18, kuma wrote:
> 西井です
> 　Hello World と表示するだけのCGIではうまく行っていたので問題
> は起きないかと思っていたのですが、libpq-fe.hをインクルードし、
> コンパイルを
> cc -I/usr/local/pgsql/include/ test.c -o test.cgi \
> -L /usr/local/pgsql/lib -lpq -lcrypt
> としてやってみると、Internal Server Errorが生じるようです。
> これはコンパイラのオプションの問題なのでしょうか？

　FreeBSD2.2.7Rであれば、/etc/rcの中の下記のような部分にpgsqlの記述を
足してあげるときちんと動くようになります（下から4行目）。修正が
終わったらサーバーをリスタートさせてください。

if [ -x /sbin/ldconfig ]; then
        _LDC=/usr/lib
        if [ -d /usr/lib/compat ]; then _LDC="${_LDC} /usr/lib/compat" ; fi
        if [ -d /usr/X11R6/lib ]; then _LDC="${_LDC} /usr/X11R6/lib" ; fi
        if [ -d /usr/X386/lib ]; then _LDC="${_LDC} /usr/X386/lib" ; fi
        if [ -d /usr/local/lib ]; then _LDC="${_LDC} /usr/local/lib" ; fi
        if [ -d /usr/local/pgsql/lib ]; then _LDC="${_LDC} /usr/local/pgsql/lib\
" ; fi
        echo 'setting ldconfig path:' ${_LDC}
        ldconfig ${_LDC}
fi

　これは僕も全く同じようなことでしばらく悩んでいたのですが、
どこかのホームページのFAQかインストール方法の説明にこのことが
書いてあり、おかげで動くようになりました。
