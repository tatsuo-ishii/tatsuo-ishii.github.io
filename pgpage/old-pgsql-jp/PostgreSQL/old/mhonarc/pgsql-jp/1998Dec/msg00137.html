まず、第一段階の状況報告です。

(1) gccは、2.8.1ではありませんでした。申し訳ない。

gcc version 2.7.2.2+myc1

(2) コンパイル時には、yaccがエラーになりました。これは、PC上で既に
    yaccで処理済のファイルを持ってきて対応しました。本来は、bison
    が必要だったのでしょう。

gmake[2]: Entering directory `/home/muc/postgresql-6.3.2/src/backend/parser'
/usr/bin/yacc -d gram.y
/usr/bin/yacc: f - maxmum table size exceeded

(3) initdbでエラーになります。

initdb

Running: posrgres -boot -C -F -D/usr/local/pgsql/data -Q template1
ERROR:  fmgr_info: function 0: cache lookup failed
longjump botch.
Illegal instruction
initdb: could not create template database

(4) コンパイルにとっても時間がかかります。

m68k 25MHzマシンでPostgreSQLをmakeするのに、3時間位かかっている
と思われます。というのも、3時位まで粘って寝てしまったので、良く
押えていないんです。ですから、ちょっと試すといっても結構大変
ですね。ちなみに、どの程度遅いのかと思い、PostgreSQL6.4をP2-400
MHzでコンパイルすると、こちらは約4分で完了します。8年間の違いが、
この数十倍なって現れるのですね。
