[pgsql-jp 6222] Bad float8 input format について

  • From: "Kiyoshi Yatsu" <clshiro@seagreen.ocn.ne.jp>
  • Date: Fri, 18 Dec 1998 09:56:53 +0900
最近、PostgreSQLについて勉強し始めた初心者ですが
教えて頂きたい事があります。

psql で　次のようなエラーになります。
(一例）
yatsu=> SELECT dsqrt('64'::float8) AS eight;
ERROR:  Bad float8 input format -- overflow
等で,float関係の数字を扱うものが同様にエラー
になります。

環境は
NEC PC9801DA
CPU: Cyrix 486SLC/DLC (486-class CPU)
FreeBSD(98) 2.2.7-RELEASE
gcc version 2.8.1
GNU Make version 3.77

PostgreSQL-v6.4　をportsを使ってインストール
その後、regression で　int4、int8，float4,float8　ほか多数
失敗なので、portsのsrcに移って
gmake clean
./configure --with-mb=EUC_JP
gmake all
gmake install
で再インストールしました。

Unixも初心者なので、もしかして、単純なことで、解決できる
事かもしれませんが、どなたか、アドバイスお願いします。
