>GB は 8bit code なので、予想通り動きました。もう一つの 8bit code
>"Big5"(こちらは香港と台湾で使われているそうです)でもきっと大丈夫でしょ
>う。

GB は EUC-JP と encoding が同じ(character set が違うだけ)なので、
EUC-JP が使えれば当然 GB も使えるでしょうけど、big5 は2バイト目に
|\{}[] がくる可能性があります。
＃big5 の三字経のテキストを16進DUMPして確認しました。

データそのものに big5 は使えるでしょうが、正規表現は無理じゃないかな。

>中国語にはさらに "HZ" というコードがあるそうですが、これは 7bit code 
>なので多分ダメだと思います。

HZ と big5 の関係は、ものすごく大雑把に言って、iso-2022-jp(いわゆるJIS
コード) と SJIS/EUC の関係みたいなものなので、メイルやニュース以外では、
HZ を用いる機会はないのでないかと....。
＃まちがってたら誰かつっこんで！
