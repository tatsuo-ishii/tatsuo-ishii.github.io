	現在、informix-SE と HTTPd を用いて、web 上の蔵書 *検索*
システムのプロトタイプを作成しています。しかし、性能が悪くて
使い物になりません。テーブルは、蔵書 *管理* システムのテーブル
を借りてきています。

create table 蔵書
       (
        キー書籍        integer,
        登録番号        integer,
        書名            char(120),
        書名_読み       char(160),
        双書名          char(60),
        双書名_読み     char(30),
        著者１          char(30),
        著者１_読み     char(30),
        著者２          char(30),
        著者２_読み     char(30),
        巻号            integer,
        版              integer,
        出版年          integer,
        出版社          integer,
        ページ数        integer,
        大きさ          integer,
        和_洋           char(2),
        備品区分        char(2),
        分類番号        integer,
        著者記号        char(8),
        巻冊複本数      char(10),
        禁貸区分        char(2),
        キーワード１    char(20),
        キーワード２    char(20),
        キーワード３    char(20),
        設置場所        integer,
        定価            integer,
        登録日          date,
        貸与日          date
)

このテーブルに 10 万件のデータを入れて、キーワードが'*データ
ベース*' の検索を行うと 22 件ヒットするのですが、70 秒もかかっ
てしまいます。
	sra さんでお使いの xtosho では、どの程度のテーブルに
何件入れて、どのくらいのスピードで検索できるのでしょうか?
場合によっては検索エンジンを PostgreSQL に変えることも考えて
います。
	以上、勝手なお願いで恐縮ですが、検索性能をお教え頂け
れば幸いです。
