前から試したかったのですが、jp patch を当てた PostgreSQL (6.1.1)で中国
語が扱えるかどうか試してみました。テストデータは中国出身の人に作っても
らいました(実際には中国語の "GB" というコードで encoding されてます)。

create table 中文 (文字 text);
insert into 中文 values ('計算机');
insert into 中文 values ('計算尺');
insert into 中文 values ('計算中心');
select * from 中文 where 文字 ~ '計算[机尺]';

GB は 8bit code なので、予想通り動きました。もう一つの 8bit code
"Big5"(こちらは香港と台湾で使われているそうです)でもきっと大丈夫でしょ
う。

中国語にはさらに "HZ" というコードがあるそうですが、これは 7bit code 
なので多分ダメだと思います。
