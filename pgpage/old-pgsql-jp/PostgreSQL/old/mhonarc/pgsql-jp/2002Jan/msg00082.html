三村です

fujio> Apacheのhttpd.confに
fujio> User nobody
fujio> を追加後サービスを再起動するのですが、同じエラーです。
fujio> そこで、サーバ側で、
fujio> Createuser SYSTEM
fujio> とし、データベースに
fujio> grant all on テーブル名 to SYSTEM;
fujio> としようとしたのですが、
fujio> ERROR:  aclparse: non-existent user "system"
fujio> というエラーになります。systemというユーザは使えないのかな、と思います。
fujio> やはりクライアント側をnobodyで動作させたいのですが、方法はありますでしょうか？
fujio> もしくはPostgreSQLでの解決法があれば、ご教授ください。

systemというユーザは作成できます。
ここで問題になっているのは大文字、小文字です。
grant all on table to 'SYSTEM';
とやってみて下さい。こうするとgrantできるはずです。
それから、PHPで作成しているのなら、
pg_connect("host=hostname port=port_no dbname=db_name user=user_name password=password");

という形でユーザも指定してあげれば接続できるはずです。
