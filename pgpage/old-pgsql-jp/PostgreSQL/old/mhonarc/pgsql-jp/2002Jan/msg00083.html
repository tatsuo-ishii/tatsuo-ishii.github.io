年を越しましたがやっと解決です。

【原因】
 Tomcatの起動スクリプト内で、CLASSPATH が上書きされていました。

【処置】
1) /usr/local/tomcat/bin/catalina.sh の中で postgresql.jar の PATH を
  追加
# ----- Set Up The System Classpath ---------------------------

CP="$CATALINA_HOME/bin/bootstrap.jar"
CP=$CP:"/usr/local/pgsql/share/java/postgresql.jar"
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
あと直接の問題解決ではありませんが、
2) Tomcatのstartup.sh を www ユーザ にて実行

で無事 エラーは解消され、
RedHat6.2/Tomcat3.1/PostgreSQL7.0.2 で稼動しているシステムを
別マシンの
RedHat7.2/Tomcat4.0.1/PostgreSQL7.1.3 へ更新のは進みました。

＃やっぱり CLASSPATH の問題でした。いろいろTomcat4.0の勉強が進み
＃ましたです。はい(^_^;;
