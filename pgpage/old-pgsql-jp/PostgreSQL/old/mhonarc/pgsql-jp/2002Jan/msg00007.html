> 目的などにもよりますが，
> select count(*) from pg_class where relname = 'テーブル名'
> てのが一番簡単だと思います．あれば 1 なければ 0 になります．

話からちょっとそれてしまいますが・・・。
僕は、基本的にデータの有る無しでCount(*)は使わないようにしていま
す。
Oracleの講師からは、
begin
  select relname from pg_class where relname = 'テーブル名'
  有る。
exception
  when NO_DATA_FOUND then
　　　データ無し
  when others then
      なんかのエラー
end ;
のほうが、Indexが聞くし全件なめるわけではないので早い、と言われ
ました。

Postgresにこんな記述ができるかは定かではありませんが・・・。
