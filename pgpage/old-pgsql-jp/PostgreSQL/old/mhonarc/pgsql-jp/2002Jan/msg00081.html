[pgsql-jp 24418] Windows ではユーザがSYSTEM に

  • From: Hidekatsu Fujio <fujio@p-brain.co.jp>
  • Date: Sat, 12 Jan 2002 17:08:14 +0900
はじめまして、藤尾です。
問題はPostgreSQLなのですが、Apacheでの解決法が正しいのかもしれません。
私には判断ができませんでしたので、申し訳ないのですが当MLで質問させてください。

クライアントの環境は、
WindowsXP + Apache1.3.20 + PHP 4.06 + MySQL3.23.32
サーバの環境は、
VineLinux2.0 + Apache1.3.14 + PHP-3.0.18-i18n-ja-2 + PostgreSQL 7.0.2

上記の環境で、クライアントのMySQLに入っているデータを加工し、サーバのPostgreSQLに
登録したいと考えています。インターフェイスはIE6を使用します。
そこで、クライアントのPHPプログラムの中でMySQLのデータを取得し、
サーバのPostgreSQLにアクセスしようとするのですが、pg_connect関数で、
Warning: Unable to connect to PostgreSQL server: FATAL 1: SetUserId: user 'SYSTEM'
is not in 'pg_shadow' 
となります。これは、ユーザ名がSYSTEMでconnectしようとしていて、そういうユーザは
いないというエラーだと判断しました。
サーバ側のApacheはUser nobodyで動かしていますので、クライアントの
Apacheのhttpd.confに
User nobody
を追加後サービスを再起動するのですが、同じエラーです。
そこで、サーバ側で、
Createuser SYSTEM
とし、データベースに
grant all on テーブル名 to SYSTEM;
としようとしたのですが、
ERROR:  aclparse: non-existent user "system"
というエラーになります。systemというユーザは使えないのかな、と思います。
やはりクライアント側をnobodyで動作させたいのですが、方法はありますでしょうか？
もしくはPostgreSQLでの解決法があれば、ご教授ください。
