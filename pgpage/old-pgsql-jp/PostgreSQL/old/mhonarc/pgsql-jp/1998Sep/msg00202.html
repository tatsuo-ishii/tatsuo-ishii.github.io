[pgsql-jp 5409] Web 連携システムでの印刷

  • From: Kazumasa Gotoh <kgotoh@cic-kk.co.jp>
  • Date: Fri, 18 Sep 1998 12:22:30 +0900

PostgreSQL に直接関係する話しではないんですが、^^; 皆様のお知恵を
貸して下さい。_o_

Web + DBMS というようなシステムが今は流行してきてますけど、このような
Web 連携システムでの印刷処理はみなさんどうしてらっしゃいますか？

問い合わせ画面を作るのは問題ないですし、検索結果の表示も同じく問題ない。
しかし、その結果の印刷はどうしようか… と、考え込んでいるのです。^^;

検索結果表示をブラウザの印刷機能を用いて出すのは簡単ですけど、
これだとページの概念がないとか、紙に出す事を前提に考えると色々と
不都合があります。紙を出す必要のある業務システムへの応用はどうしたら
いいんでしょうか。

Java 関係はほとんど無恥なんですけど、^^; ひょっとして Java でプログラム
書けば何とかなるんでしょうか？
紙はブラウザを動かしているクライアントマシン上で出す事を想定して
います。

「ORACLE Web Appication Server ハンドブック」という本を買ってきて
読んでるんですけど、これには ORACLE Reports その他のプロダクトを
買っての話しにはなりますが、PDF ファイルを作成してブラウザ側に
送り付ける事ができるとかいう方法が書いてあります。

ブラウザ側の Acrobat Reader プラグインを組み込んだり、またはそのような
データが送られてきた時には Acrobat Reader を起動するようにしておけば
印刷問題から開放されそうなんで、これはうまい方法かな… と思いました。

CGI とか PHP でガシガシ書けば同様な事はできるんでしょうけど、ちょっと
大変そうな… ^^; それとも実はそうでもないとか？
