持田＠TPJ です。

> hash table out of memory と出て、postmasterで-B つけてバッファを

 postmaster の manpage によると -B の単位は 8K bytes とあるので、
8K x 512 でもせいぜい 4M です。ヒープが足りない、ということなら、
多分 ulimit だと思います（csh の limit, sh の ulimit 参照）。
シェアードメモリが足りてない、ということなら、カーネルパラメータでしょう。
NetBSD だと、options(4) の options SHMPAXPGS=value です。

> どれくらいメモリを搭載しなければならないのでしょうか？

 実メモリに固定される（= swap out されない）メモリが足りない場合は
メモリを増やさないとダメですが、その確率は低いと思います。
