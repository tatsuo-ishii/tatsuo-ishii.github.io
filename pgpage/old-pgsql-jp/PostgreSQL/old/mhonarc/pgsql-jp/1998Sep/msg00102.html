>長尾です。
>
>パスワードをかける方法もありますけど、
>phtml を書いた権限でしかアクセスできないという方法はないでしょうか？
>
>通常は、nobody に権限を与えるわけですけど、それでは、phtml かけば
>誰でもやりたい放題の権限を得てしまうので、まずいと思っています。

もしかして、一般ユーザが自由にweb serverにログインできる環境ですか？そ
いう環境ではセキュリティを保つのは非常に難しいと思いますが。

>パスワードで制限だとめんどくさいでし、不特定多数の人に公開する場合に
>うまくいきません。
>
>一般公開するが特定のphtml を経由した変更しか認めない方法が
>あるとうれしいのですが、

nobody 以外のユーザで web server をもう1本上げ、ドキュメントルートを一
般ユーザが入れない場所に設定しておくのが簡単なのでは。

PostgreSQL では そのユーザに対して crypt か password 認証を掛けておき
ます。で、パスワードはファイルに書いておき、そのユーザ以外の read
permission を落しておく。PHP でそのファイルからパスワードを読むように
しておけば、人間がパスワードを入力する必要がなくなります。
