# ちょっと忙しくて ROM もさぼってました。

> そういえば、Kerberos って PHP+PostgreSQL で使えますか？ > 桑村さん

  PHP3 はまだ触ってないのですが、PHP/FI-2 には Kerberos の実装はなかった
と思います。もっとも、PHP の場合 WEB サーバ上で実行されますので、クライア
ントからパスワードを渡すときそのまま平文がネットを流れて意味がありません。
PHP でブラウザ側のホストにあるチケットファイルを見る事もできないでしょう
し、ちょっと無理だと思います。

  Kerberos 認証についてはいろいろ考えてみたのですが、実装が面倒なこともあ
って、ブラウザに実装するのが難しいでしょう(Netscapeなどの場合 Java Applet
を使えばできるのかな？)。それにKerberosの認証局の設置も必要ですし。それと、
今、PostgreSQLの認証がどのくらい柔軟になったかをしらないのですが、以前は、
Kerberosの認証とその他の認証とは排他関係にあったと思います。すでにKerberos
を運用している大きなサイトであれば意味があるのでしょうが、僕はメンテに使え
ないかと思ったのですが、Kerberos以外の認証も必要だと思い、そのときはあき
らめました。
