桑村です。

  ソース付属の文書をOpenJadeでコンパイルできないでしょ
うか？ http://osb.sra.co.jp/PostgreSQL/Manual/ から
SGMLのアーカイブをダウンロードしてmanを作りたいと思っ
てるのですが、ソース付属のオリジナル文書自体でうまく
ゆかず、お手上げです。

  Plamo Linux 2.2.1 を使ってますが、わけもわからず、

libxml2-2.4.12 
libxslt-1.0.9  
OpenJade-1.3   
docbook2X-0.7.0  
SGMLSpm

をフォルトコンフィグレーションでインストールし、
DocBook(db162.zip)を展開しました。

postgresql-7.1.3/doc/src/sgml/Makefile を
--
# db162.zipを/usr/local/share/sgml/ 下に展開
DOCBOOKSTYLE =  /usr/local/share/sgml/docbook

# docbook2man-spec.plのある場所
D2MDIR= /usr/local/share/docbook2X
--

とし、postgresql-7.1.3/doc/src で `make man.tar`
を実行したのですが、

--
> make man.tar
make -C sgml man
make[1]: 入ります ディレクトリ
`/opt/src/dbms/postgresql-7.1.3/doc/src/sgml' onsgmls  ./book-decl.sgml ./reference.sgml \
  | sgmlspl /usr/local/share/docbook2X/docbook2man-spec.pl --lowercase --section l
onsgmls:./book-decl.sgml:1:55:W: cannot generate system identifier for public text "-//OASIS//DTD DocBook V3.1//EN"
onsgmls:./book-decl.sgml:11:0:E: reference to entity "BOOK" for which no system identifier could be generated
onsgmls:./book-decl.sgml:1:0: entity was defined here
onsgmls:./book-decl.sgml:11:0:E: DTD did not contain element declaration for document type name
onsgmls:./reference.sgml:7:9:E: there is no attribute "ID"
onsgmls:./reference.sgml:7:20:E: element "BOOK" undefined
.....
-- 

てな具合いです。どなたかコンパイルしたの持ってませんか？
やはりきちんと勉強してからでなくてはだめかな＞SGML
