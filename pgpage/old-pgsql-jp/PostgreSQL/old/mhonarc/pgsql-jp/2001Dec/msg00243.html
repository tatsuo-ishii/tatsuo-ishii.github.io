DATETIME は PostgreSQL 独自の型ですね。

PostgreSQL は日付関係では DATE, TIME, TIMESTAMP, DATETIME という型を
サポートしてきましたが、過去の経緯で言うと TIIMESTAMP の実装の問題
から精度があまりよくなかったので、DATETIME が実装されたようです。

この辺は石井さんがお詳しいだろうと思いますが、Oracle の VARCHAR と
VARCHAR2 の関係に近いかも知れません。

しかし、その後の議論でこれは望ましくないとなったのでしょう。
7.0 からは DATETIME は TIMESTAMP に置き換えろとなっており、
pg_dump を使って 6.5.x からデータベースの移行を行うと、DATETIME を
使っているカラムは TIMESTAMP を使ったカラムに作り変えられます。

で、このスレッドの元の話の「SQL 標準でない DATETIME は 7.2 では
もうサポートされないのね」というものにつながるわけです。

初期の SQL 標準に TIMESTAMP 型があったかどうかは私も知りません。 (^^;
記憶がかなりあやふやですが、昔の UNIFY などにはこの型はなかった
ような気がするので、初期の SQL 標準では DATE しかなかったのかも
知れません…
