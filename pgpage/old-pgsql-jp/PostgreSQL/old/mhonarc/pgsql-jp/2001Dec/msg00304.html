丸矢＠ＳＲＡともうします。お世話になります。

PostgreSQLをUNICODEで利用したいのですが、psqlからSQL文を発行して
データベースに登録したときにUNICODEに変換されず、EUC_JPのまま登録されて
しまいます。

このデータをPostgreSQL JDBCドライバから利用したときに長さ０のStringが
帰ってきてしまって困っています。
逆に、JDBC経由でJavaアプリケーションから発行したSQLについては正常に
UNICODEで格納されています（当然といえば当然ですが）

psqlから、EUC_JPで書いた文字をUNICODEに変換してくれるような
設定でなにか試してみる方法がありましたら、教えていただけないでしょうか。
現在、PostgreSQLに行った設定などはこのようになっています。

データベースの情報
 List of databases
 Database  |  Owner   | Encoding
-----------+----------+----------
 postgres  | postgres | UNICODE

psqlの情報

postgres=> show server_encoding;
NOTICE:  Current server encoding is UNICODE
SHOW VARIABLE
postgres=> show client_encoding;
NOTICE:  Current client encoding is EUC_JP
SHOW VARIABLE


コンパイル時のConfigure情報

$ pg_config --configure --prefix=/usr/local/pgsql
--enable-multibyte=UNICODE
--enable-unicode-conversion --with-tcl --with-perl
--enable-syslog --enable-debug --with-CXX
--with-java
