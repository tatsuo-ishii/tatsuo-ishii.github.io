> 何故「Date型」には年月日しか入らないのでしょうか。
> そのほうが高速？
> Oracleの考え方が変？

SQL92 の規格によると、

 ・ DATE      : 0001-01-01 から 9999-12-31 までの有効な日付
 ・ TIMESTAMP : 0001-01-01 00:00:00... から 9999-12-31 23:59:61.999
                までの有効なタイムスタンプ

となっています。0001, 9999 は西暦です。

つまり、PostgreSQL は SQL92 の規格に従っているだけだという
ことになります。

では Oracle がおかしいのか？ というと、SQL92 の規格に従っていない
のは確かですが、それだけをもって「おかしい」というのも言い過ぎなの
ではないかと思います。

Oracle は Outer Join などの書き方も SQL92 の規格に従ってはいませんが、
これは歴史的経緯を持った事情があり、今さら標準 SQL に合わせる事は
難しいのでしょう。
PostgreSQL はできるだけ標準 SQL に合わせようとしているようです。

まぁ Oracle の SQL は色々と方言めいた点があるのは確かなので、
「Oracle = 標準 SQL」だと考えていたとしたら、それは単なる思い込みです。

DBMS の移行を行うときなどは、移行元と移行先の特性はよく調べる必要が
ありますし、移行などがなくとも自分の使っている DBMS が標準 SQL に
どの程度準拠しているか、何が違うのかを把握しておくことは長い目で
見れば無駄にはならないでしょう。
