Oracle のドキュメントを読み直していないのでうろ覚えですが、(^^;
INTEGER はともかく VARCHAR に関しては Oracle の VARCHAR の実装(?)
には問題があって推奨できない。 VARCHAR2 はそれを改善しているから
VARCHAR2 を使うべきだ。 のような意味の文章を読んだことがあります。

それなら VARCHAR の方を直せば(仕様変更すれば)よさそうなものですが、
VARCHAR は VARCHAR のままにしておいて、VARCHAR2 というのを新たに
作るという方針を選んだのでしょう。
確かに影響範囲を特定するためにはこうした方が安全ですが…

数字にしても INTEGER や DECIMAL などを使い分けるのが SQL 標準としては
正しいわけですが、Oracle としては「なんでも NUMBER でいけまっせ」と
アナウンスしてた事を考えると今さら… なのでしょう。

> 過日ストールマンが「フリーソフトウェアの目的は多くの人に使われることで
> はない。使う人を自由にすることだ」と述べたそうですが，これは非常に含蓄
> のある言葉だと思います．

そうですね。

ところで、なかやまさんが書いていた「ユーザは標準がどうあれ、買った
ものがマニュアルの通りに動いてくれれば文句はない」という主張も理解
できます。
私も自分の用が足りるとか、顧客の要望が満たせてお金を支払ってもらえ
れば、それで構わん。 という部分はあります。

ただ、それをあまり声高に宣言してしまうと標準規格というものの存在意義を
否定することになってしまいます。なかやまさんも別にそういう主張を
なさりたいわけではないでしょうけれども。

この辺は、その時にどのような立場で話をしているのかによっても多少は
話が違ってくる部分でしょう。
