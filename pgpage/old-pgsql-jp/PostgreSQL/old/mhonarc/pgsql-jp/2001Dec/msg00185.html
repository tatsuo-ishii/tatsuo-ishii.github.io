先ほど投稿した中谷誠和なのですが、すみません。
どうもcreatelang -L /usr/local/pgsql/lib plpgsqlの指定だと
シーラカンス本のP184の例文を作成し、
select odd_even(2);と入力すると、

error :Load of file /usr/local/lib/plpgsql.so failed: /usr/local/pgsql/
lib/plpgsql.so :undefined symbol: SPI_lastoid
と出て動いてくれません。

ただし、一度droplang plpgsqlで削除し、createlang -L /usr/lib/pgsql で指
定しなおすと、シーラカンス本P184の例文は無事に動きました。
ですので、一応問題はないのですが一応報告をしておこうと思いました。

ただ、私はC言語が読めないので、ちょっとどこが悪いのかまでは
わかりませんでしたが、、、

私の環境は先ほどのメールと同じで、vine-linux2.1.5+postgres7.1.3でして、
一度、デフォルトで入っていたpostgres7.0.3からアップグレードを
行っています。
また、plpgsql.soは、
/usr/lib/pgsql/plpgsql.so
/usr/local/pgsql/lib/plpgsql/plpgsql.so
の二箇所に入っています。

シーラカンス本の例文は私が入力したのは
create function odd_even(integer)returns text as '
	declare
		num alias for $1;
	begin
		if num % 2 = 0 then
			return ''guusuu'';
		else
			return ''kisuu'';
		end if;
	end;
'language 'plpgsql';
です。
