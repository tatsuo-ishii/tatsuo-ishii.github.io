いつも参考にさせていただいております。
現在下記のようなSELECT文を書いてシーケンス番号に
基づいたデータ（件数は数百〜数万)の一覧を作成しようと
しているのですが、特定のSELECT文の書き方をすると
インデックスが使われずに困っています。

具体的には
　WHERE句に'>' もしくは '>='　の場合にはインデックスが使われず、
　'<' , '<=', '='の場合にはインデックスが利用されます。

これはフィールドがSERIALの場合だけでなく、単純にint4のフィールド
を作成し、それにインデックスを張っても同じ現象が発生しました。
（CREATE INDEXで作成した複合インデックスや単独インデックス
両方でも確認しています。）

もしかすると、とても初歩的なことかもしれませんが、
解決方法もしくは回避方法をご教授願えませんでしょうか？


実行SELECT文
  SELECT * FROM datatbl WHERE seqid > 1000;

検索するテーブル
  CREATE TABLE datatbl (
                seqid   SERIAL PRIMARY KEY,
                strdata  TEXT
  );
