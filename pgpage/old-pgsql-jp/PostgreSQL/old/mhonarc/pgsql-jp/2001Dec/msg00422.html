[pgsql-jp 24314] pg_ctl を使うとpostgresql.conf を参照しないことがある？

  • From: Takeuchi Akira <takeuchi@sec.co.jp>
  • Date: Thu, 27 Dec 2001 11:45:08 +0900
セックの竹内です。

pg_ctlを使った場合のpostmaster起動動作で疑問が出てきたので
お教えいただけますでしょうか。

環境は、
  PostgreSQL 7.1.2
  Red Hat Linux 7.0.1
です。

7.1からpostgresql.confを参照してpostmasterが起動するように
なっていますが、pg_ctlを使ってpostmasterを起動すると
以前の設定ファイルであるpostmaster.opts.defaultを
参照することがあるようです。
これは仕様でしょうか？

postgresql.confのあるディレクトリに、以前の設定内容を
確認するため postmaster.opts.defaultを置いてあります。
postgresql.confは
  tcpip_socket = on
  max_connections = 50
  silent_mode = on
         :
postmaster.opts.defaultは
  -S -i -N 25
となってます。

pg_ctl -w stop して pg_ctl -w start した後のプロセスを
見てみると、

$ ps -eo user,cmd | grep postmaster
postgres /usr/local/pgsql/bin/postmaster -D /usr/local/pgsql/data -S -i -N 25

となるので以前の設定値を見ているようです。
今まではあまりpg_ctlを使っていなかった（シーラカンス本にある
「postmasterの自動起動」設定で事足りていた）ので、
気がつきませんでした。

postmaster.opts.defaultファイルを置かなければいいのでしょうが
ちょっと気になったもので。
