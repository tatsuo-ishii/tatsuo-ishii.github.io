池田です。

> Oracle からしてみれば SQL92 に準拠してしまうと
> お客様が簡単に他の DB へ移行できてしまい
> 「囲い込み」ができないので独自路線を歩んでいるのでしょう。

　「Client/Server形態」以前のシステムでは、「プログラムAPI」にて十分に
囲い込みができていたようにも思えますね、SQL*Net や Net8、もしくは OCI
で対応したプログラムで作ると他に展開できない。
　と言っても日本国内で、Oracle がシェアが高くなったのはこの５年ぐらい
ですけどね。私が入社した頃なんかだと、Informix は知っていても Oracle
なんて知らない企業が多かったです、、栄枯盛衰。

　で、現状では、ハードウェア性能が上がり、RDBMS とのインタフェースも
ほぼ同じ（JDBCとか）になっているので、囲い込みってやり難いかと。

> # free で色々な事に利用できる postgres に移行されて
> # 「Oracle じゃなくてもできるじゃん！」なんて事になったら
> # 困るから SQL92 は無視してるのかも :p

　顧客の要望がそっちじゃない、、ようですよ。大規模、信頼性を狙っている
ので、PostgreSQL とは衝突する部分は少ないでしょう。

　実際、PostgreSQL と Oracle の採用って住み分けてる感があります。


> 下位互換を残しながら Oracle SQL を拡張(標準にあわせる)事が、
> 不可能だとは思えません。
> ぜひ、準拠していただきたいですね。

　「不可能」だとは思いませんが、、標準に合わせるのがビジネス上での
最優先とも思えないので、優先度は低いでしょうね。同等な機能が別の
書き方で対処できるのであれば、無理に実装する暇があるならば、他の
顧客が望む新機能や、バグ修正してくれた方が良いかも（爆）。

> 私は Oracle ユーザでもなんでもないので、
> 文句を言うことも出来ませんが、
> ユーザの方は Oracle に SQL92 に準拠するように
> リクエストしてみてはいかがですか？

　単なる元社員なので、、今後どうなるかは保証できませぬが。。
　ユーザはバグ修正とかを強く言いたい、と思う（苦笑）。

ではでは。
