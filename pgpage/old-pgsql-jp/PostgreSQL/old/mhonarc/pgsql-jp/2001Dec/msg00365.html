MSDE って、MS-SQL Server 7.0 と違わないんじゃぁ。違いは、同時接続数の制限
だけだったような・・・。うろ覚えですが。
Access と MSDE って、完全に別ですよね。JET じゃぁ、DBMS になってないと思
うので、そう(勝手に)捉えてるのですが。

> > Accessでパフォーマンスが頭打ちになってきた時、Webシステムに
> > 移行したくなったとき、規模を広げたいが大きなバジェットが組めないとき、
> > PostgreSQLは、その有力な候補になると思います。

私もこの意見に賛成です。

> 本格的 Database として　PostgreSQL の優位は動かないとしても、
> 書店などでパラパラといろんな本をめくってみると、どうも、
> MySQL が急速に使用場面を増やしているように見えました。
> 取敢えず、今回の私には関係ないのですが、、、

特に ODBC + Ms-Office系アプリ では、PostgreSQL より MSDE のが全然良いです。
さすがに MS製品で固めると、速度的にもメリットがあります。
もちろん、構築したいシステムに依存しますが、「データを中央に集中して置い
ておきたいだけ」くらいのシステムであれば、MSDE で十分です。
また、Webシステムに移行する際も、NT + IIS を使うのであれば、MSDE か、
MS-SQL Server のが良いと思います。

金額的なメリットが無いという話でしたが、MS-Office なら、2年ごとにバージョ
ンアップが発生し、ライセンスの更新、(そしてほとんどの場合)ソフトウェアの
書換が必要になります。もちろん、バージョンアップしなければ良いのですが、
そのあたりは MS の上手なところで、バージョンアップせざるを得なくなるんで
すよね・・・。それに伴い、マシンスペックも上げなければならなかったり・・
・。
PostgreSQL を使っても、ODBC + MS-Office なら同じことなのですが(DBサーバー
+ライセンス分だけ金がかからない)。

とりあえず、移行を考慮しつつ MSDE で設計していけば良いのでは。
(こうすると、PostgreSQL に移行したときにそのすごさが実感できると思う)
