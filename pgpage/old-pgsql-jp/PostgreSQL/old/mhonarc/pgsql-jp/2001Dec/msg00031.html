はじめまして、初心者の岩永といいます。

この間TURBOLINUXを導入し、その中にあるPostgreSQLを起動させようと
したのですが、上手くいきませんでした。

環境動作
　TURBOLINUX 6.1
　PostgreSQL-6.5.3

TURBOLINUXをインストールしたときにPostgresユーザーが作成されていたので
とりあえずPostgresユーザーになり、

# su postgres

Postgresのホーム(/var/bin/pgsql)に下記の環境変数を追加しました。

$ vi .bash_profile

	export USERNAME ENV PATH

	export POSTGRES_HOME=/var/lib/pgsql
	export PGLIB=/var/lib/pgsql
	export PGDATA=$POSTGRES_HOME
	export LD_LIBRARY_PATH="$LD_LIBRARY_PATH":"$PGLIB"


そしてデータベースを初期化しようと思い

$ /usr/bin/initdb -D /var/lib/pgsql/data

をしたのですが、

initdb does not know where to find the files that make up
Postgres (the PGLIB directory).  You must identify the PGLIB
directory either with a --pglib invocation option, or by
setting the PGLIB environment variable, or by having a program
called 'postconfig' in your search path that outputs an asignment
for PGLIB.


というエラーが出ました。

PGLIBがどこにあるか解らないということでしょうか。
ということは環境変数の追加の仕方が悪かったのでしょうか？
そもそもpostgresユーザーのなかに.bash_profileという
ファイルはなかったので、追加ではなく作成したのですが・・・


初心者で何も解りません。
どなたかご存知でしたら、教えてください。
