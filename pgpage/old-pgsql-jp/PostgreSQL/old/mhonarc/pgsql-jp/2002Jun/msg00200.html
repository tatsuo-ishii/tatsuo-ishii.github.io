On Tue, 18 Jun 2002 13:59:20 +0900
Atsushi Shibata <shibata@bitark.com> wrote:

> こんにちは。柴田と申します。
> Flashには，"〜?name=foo&email=..."のように引数が渡せます。この部分をPHP
> なり他のアプリケーションサーバーで動的に生成すれば，FlashとRDBの連携が可
> 能です。
> 他にも，手軽な連携の方法はいくつかあるようです。
FlashはVer5からXMLも扱えます
//----------Flash Action Script----------
function loadXML(url){
	myXML=new XML();
	myXML.onLoad=this.loadEnd;
	myXML.load(url);
}
//----------Flash Action Script----------
などとしておき、Webサーバに対してURLを渡し、結果XMLがかえってくればFlash
はそれを扱えます。


> Flash + PHP + PostgreSQLで動いているオンラインゲームが実際にあります。
> PostgreSQLにデータを保存して，Flashとの橋渡しにPHPを使っています。
XML読み込みのためにFlashからWebサーバへ渡す値はurlなので、FlashからWebサー
バへはGetクエリ値等を含めておけば、かなり使えるいいやつです。
注意として、はそのXMLは改行コードやタブなどを含めてはいけないそうです。
さらにFlash Player Ver5はShift-JISで、Ver6はユニコードでなければだめみた
いです。

Flashのよいところは、pgsql-jp: 26414で小川さんが一部指摘しているように、
なんと言ってもクロスプラットフォームなところです。
Javascriptで構築する場合と比べ、Win,Mac,IE,NN,どの環境でもほぼ同じように
表示されます。制作サイドとしてこれほどありがたいことはありません。
