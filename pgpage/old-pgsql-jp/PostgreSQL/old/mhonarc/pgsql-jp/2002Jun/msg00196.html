# 私はやるつもりはありませんが。

> ＦＬＡＳＨのメリットですが、近藤さんが作った郵便番号のフォーム
> であるとか、書籍の検索であるとかを一つずつムービークリップで
> 作って読みこましていくと、いくつでも１つのＦＬＡＳＨの画面の中で
> 表示できるということです。

実際に DB にアクセスして動いている頁を見たいですね。
 
> 例えば郵便番号のフォームが表示されていて上に書籍のページが来て
> 郵便番号のフォームが、見えなくなってもレイヤーの重なり順を変えてやれば、
> 書籍のページを保持したまま配達先住所がフォームにうてます。
...略...

このあたりは、IE 5.5 や Mozilla 1.0 の JavaScript でも、
position:absolute の CSS と、innerHTML を使って、レイヤー
の重なりや表示/非表示、移動などのコントロールができます。

どちらが使いやすいのか、プログラム保守性がいいのかはわか
りませんが。

あまり細かい話になると、興味のない方もいらっしゃるでしょう
から、業務アプリ分科会の ML で話しましょうか。

中は勿論、データー型対応の判定など殆んど全部を JavaScript
で処理させています。
