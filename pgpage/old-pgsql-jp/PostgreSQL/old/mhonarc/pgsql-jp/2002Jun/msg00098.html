以下の構成で Postgres の二重化を行おうと
試みております。

[Host A]
・usogres-0.8.1 を起動

[Host B]
・postmaster -S -i -p 5434 -D /usr/local/pgsql/data
・postmaster -S -i -p 5436 -D /usr/local/pgsql/data2

Host A 上のUsogres を用いて、Host B 上の
２つの postmaster を二重化しようとしていますが、

Host A 上で psql ツールを引数無しで動かすと

psql: pqReadData() -- backend closed the channel unexpectedly.
        This probably means the backend terminated abnormally
        before or while processing the request.

のエラーが出力されます。

Host A から
psql -h hostname -p portno では正常にＤＢを
参照できるため、Host B のＤＢは正常に動作、
かつ、Host A →B 間の通信は確立されていると
判断しております。

.usogresrc には以下の記述をしております。

host = "127.0.0.1" ← Host A
port = 5432
hostorg = "192.168.10.1" ← Host B
portorg = 5434
hostcopy = "192.168.10.1"
portcopy = 5436

Host A の環境変数には以下を設定しております。

export PGHOST="127.0.0.1"
export PGPORT="5432"

以前、Usogres, Postgres を同一ホストに配置して
実験した際には、正常に二重化されました。
リモートで行う際の注意点などありましたら
ご教示願います。

環境は
Red Hat Linux
PostgreSQL 7.1.3
Usogres 0.8.1

です、よろしくお願いいたします。
