Excel に表定義を書いて、マクロ(VBA)で SQL を吐き出させて使っています。
開発が終わった時点で、それらの SQL を一つにまとめて、エラーチェック入れて、
バッチにします。


そもそも、最初から PostgreSQL を使う目的でいるのにもかかわらず、MS-Access
で作ったデータベースを PostgreSQLに移植したいという考え方が分かりません。
データベースシステムそれぞれに特性がありますから・・・。

テーブルの継承を使うと楽になる、外部参照整合性はマトモなものを使いたい・
・・ならば、PostgreSQL を使おう・・となるのでは。
社内ネットワークで NTドメインを利用していて、データベースに対するセキュリ
ティは NTドメインユーザー(グループ)単位で行いたい・・とか、今回のデータは
かなり重要なものだから、トランザクションログを利用してデータベースの復旧
が可能なモノを使わなければならない・・とかならば、迷わず MSDE にしてしま
えば良いのでは。(おっと、MSDE(MS-SQL7.0)ではトランザクションログからの復
旧はバグってて出来ないんでしたっけ)
まず、これらの考えがあった上でスタートするので、移植なんてしないけど・・
・。(クライアントに「大体こんな感じです」的なものを見せるのに、Access で
チロっとつくったりすることはありますけど、それはそれ、システムにする時は
書き直しですね)
