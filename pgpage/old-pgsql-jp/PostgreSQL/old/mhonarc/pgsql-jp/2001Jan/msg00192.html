＃分岐は少ない方が書くのも処理も楽だけど
＃それでも必要な物は書かなきゃいかんのでは？

Mamada wrote;

>select ... from ... where ... and ...
>というSQL文を使ってすべての行を検索させる方法はないでしょうか。
>select ... from ...
>で条件をつけなければ検索できるのはわかりますが、その場合、SQLを2個作るか、
>where以降だけ別に作成するなどして、条件で置きかえる必要があると思います。


なにも、初めから別に作成して、条件で置き換えないで
条件によってwhere以下をつなぐかつながないか判断して、
最終的に欲しいsql文をphpで作っていけばいいんじゃないの？

>たとえば、PHPで、50以上、50未満、全てという項目がある場合、「50以上」と「50
>未満」は、最初のSQLで検索可能ですが、
>「全て」の場合、最初のSQLで検索する方法がわかりません。

初めに「条件検索」が頭にあって、次に「全検索」と考えるから
全検索でwhereが使えないかとか考えるのかなぁ

頭に書いてるように全検索の時はwhere以下が無ければOKで
「全て」を検索するケースをデフォルトのsql文として
条件によってwhere以下を追加すれば、

PHPの流れ的には

if(検索条件が無い時){
	$sql=$sql;
}
elseif(50以上の場合){
	$sql=$sql . "where i>=50"
}
eles{
	$sql=$sql . "where i<50"
}

ってな感じで検索条件によってwhere以下を追加して
条件に合ったsql文を動的に作ってやったら
$result = pg_exec($sql);

で、「全て」場合だろうと条件つこうと、最終的な$sqlに
間違いが無ければ、検索できると思いますけど。
sql文を2個作ったわけでもないし、追加はしてるけど
初めから準備してた物に置き換えてるわけでもないし

>このような場合に、SQLを1つで全ての行を検索する方法があったら教えてください。

実際に使うデータの中身が正数とか分かってれば
「ゼロ以上」を条件にすれば全検索はできると思うけど。
メールに書いて分かってる様に全検索の時は
where以下をつけなきゃいいだけでは？
