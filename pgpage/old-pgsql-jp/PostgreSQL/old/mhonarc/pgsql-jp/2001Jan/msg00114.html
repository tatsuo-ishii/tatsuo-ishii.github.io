でも、結局、Release時点のportsはその時のものなので
ports使うなら、/usr/portsの下をcvsupで更新かけておかないと
少し経つとバージョンは当然古い物も出てくるんですよ。

ぼくはApache+PHP+PostgreSQLオプションいじることが多いので
ports使わないようになってしまいました。
ソースが出たら、portsの更新待たずに自分でmakeできるし


で、素直にportsからインストールすると

pgsql:*:70:70:PostgreSQL pseudo-user:/usr/local/pgsql:/bin/sh
と/etc/passwdに登録されて(pkg_deleteしても消えないと思う)

pgsql:*:70:
と/etc/groupに登録されて(pkg_deleteしても消えないと思う)

/usr/local/etc/rc.d/pgsql.sh
が自動で出来て

>> /etc/passwd に登録された User postgres の Shell が sh ではなく、csh に
>> なっているに 50 点 :-)
>>
>>   #csh の場合は 2>&1 > /usr... じゃなく >& /usr... ね
>>
>　まさに仰せの通りでした。通常 tcsh を使用しておりますもので、つい postgres
>にも tcsh を使用させておりました。
>
>　試行錯誤の結果、シェルを sh にしたら、ずばり通るようになりました。
>　皆様には、御迷惑をおかけしました。

/usr/local/pgsql/.profileをちゃんと書くとかはあるけど
ここまでportsから済んでれば、rootの状態でpgsqlになって
initdb以下の作業をしてから　起動スクリプト　startで起動させて

って感じで進められるはずで、portsから入れると

「postgresというユーザーを新たに作らなくても作業が出来る」
　(rootからpgsqlになるので、違いは無いけど)

ので、最初のメールのようなエラーは実は出ないんですよね。
インストール前にpostgresってユーザーを手で登録したんでしょうかね？
