北村＠アシェット(逃避モード)です。

> そこで、 psql -eを調べてみました。
> manでは、
> Echo the query sent to the backend
> という記述がありますが、この「backend」とはどういう意味でしょうか。

場合場合によって意味が変わりますが、この場合、DBMS 本体( postmaster )
です (と思う)。

私も正確に文献などで知ったわけではないのですが。
PC(クライアントとなる端末)に向き合った時に、一番手前(フロント)のアプリ
ケーションが フロントエンド、一番奥(バック)のアプリケーションが バック
エンド となるのでは。
psql をつかって PostgreSQL データベースの操作をする場合、フロントエン
ドが psql、バックエンドが postmaster ですね。
PC 上から Web ブラウザを使って、Webサーバー経由で PostgreSQL のデータ
ベースの操作をする場合、フロントエンドは Web ブラウザ、バックエンドは
postmaster ですね。この場合、経由される Web サーバや、Web サーバと
PostgreSQL の連携を行なうアプリケーション(PHP だったり Perl だったり自
作 C プログラムだったり)を、ミドルウェアと呼んだりします。
Web ブラウザから、プレーンな HTML ドキュメントを呼び出す場合、フロント
エンドは Web ブラウザ、バックエンドは Web サーバー になります。
PostgreSQL を使ってアレコレ・・という場合、バックエンドは PostgreSQL
( postmaster )に決まりではないでしょうか。

では。
