[pgsql-jp 19332] createdb の-Eオプション

  • From: "Shigeyuki Kuwaki" <kobe@grn.mmtr.or.jp>
  • Date: Fri, 5 Jan 2001 00:42:35 +0900
  • Importance: Normal
私は、海外のサーバー（ホスティング）でPostgreSQLを始めようとしている初心者で
す。私はPostgreSQLのスーパユーザ権限を持っていないので、ホスティング会社に依
頼して自分のデータベースを作ってもらいます。その時、EUC_JPの文字コードを指定
していただくように依頼しました。しかし、ホスティング会社も上手く出来ないよう
で、「PostgreSQLスーパーユーザ権限を、一時的にあなたに渡しますので自分でセッ
トしてください。」と言うメールを頂き、自分でセットすることになりました。

石井達夫先生の本を読むと、データベース作成時の文字コード（-Eオプション)指定
は以下の何れかだと解釈しましたが、上手く文字コードがセット出来ません。

$ create database dbname --pgencoding=EUC_JP
$ create database dbname --enable-multibyte=EUC_JP
$ create database dbname --with-mb=EUC_JP
$ create database dbname -E EUC_JP
$ create database dbname -E --pgencoding=EUC_JP
$ create database dbname -E --enable-multibyte=EUC_JP

createdbでは文法エラーになり、create databaseで作成できます。「なぜEUC_JPの
文字コードが指定出来ないのか？」分る方教えてください。

-Eオプション指定のエラーメッセージでは、-　のところに問題があると表示されま
すので -E のオプション指定が上手く出来ないものと思います。最初の3個のコマン
ドではデータベースは作成可能ですが、encoding=0で文字コードがEUC_JPにセットで
きません。


[PostgreSQL 6.5.2 on i686-pc-linux-gnu, compiled by gcc egcs-2.91.66]
