永井です。

> 2) 「■ＳＱＬ会話プログラムにおいて使用する文字コードを指定」
>
>    psqlでは，"\encoding 文字コード"です．
>
>    SQL使うなら"SET NAMES"も使えますが，文字コードがSJISの時にpsqlでは
>    不都合が起きるでしょう．

の報告ですが、私の環境では、SQL では設定できないようです（メタな
コマンドの方でちゃんと設定できるので問題ありませんが）。

　nagai=# select version();
　                               version
　---------------------------------------------------------------------
　 PostgreSQL 7.0.2 on i686-pc-linux-gnu, compiled by gcc egcs-2.91.66
　(1 row)

　nagai=# SET NAMES SQL_ASCII;
　ERROR:  SET NAMES is not supported.

今度、違うバージョンで実験してみます。

蛇足ですが、他のRDBMS では、

　Sybase:
　1> set names eucjis
　2> go
　Msg 195, Level 15, State 7:
　Line 1:
　'names' は、SET コマンド として認識できません。

　Oracle:
　SQL> set names EUCJ_0208; /* EUCJ_0208は適当ですが */
　SP2-0158: unknown SET option "names"

となるようです。

この「SET NAMES」ですが、これ「標準ＳＱＬ」なのですね。
標準には「セッション文」というのがあるそうです。知りませんでした。

実装しているのは、InterBaseと（私は確認できていませんが）PostgreSQL
だけのようですね。

では、失礼します。
