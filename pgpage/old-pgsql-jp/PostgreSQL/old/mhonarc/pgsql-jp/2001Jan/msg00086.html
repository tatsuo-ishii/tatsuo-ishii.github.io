初めまして　奥　田と申します

turbolinux　6.1　と　postgre　6.53　の　組み合わせで

INITDB　”データベースを　任意のフォルダーに作成する”　が　出来ません
$ initlocation /home/data
$ export PGDATA2=/home/data
$ postmqster -s -i
$ createdb -D PGDATA2 test
で　１行目で　フォルダーは作成されます。　が　
３行目で　
/usr/bin/postmaster は　データベースシステムの　データが　見つけられないと　
云われ
４行目で
PGDATA2 testの　パスが分からない　サーバーに不整合がある　データベースは作成
できない
となります

理由は、
ターボリナックス６．１　ＦＴＰ版を　インストールしたので　ポストグレは特別イ
ンストールしなくても、６．５．３　が　入っていた
しかし　フォルダーが　
/var/lib/pgsql  と　/usr/lib/pgsql  と　二つ在って、環境変数の設定が　出来ない
（良く分からない　どのフォルダーに　.bashrc　を　作るのか、また　どれにパス
を通すのか）

かつ　初期設定も　ＰＧＬＩＢ　がない　　の　エラーです
よって、任意のフォルダーに　データベースのインスタンスを　作成出来ない

でも
su postgres が　ＯＫで　createdb で　データベースは　/var/lib/pgsql/base/の
　下に　作成でき
PGACCESS　が　動き　エクセルのデータまで　取り込めました。また　ＷＩＮ環境の
　ＯＤＢＣも　ＯＫで
アクセスからデータが見ることもできます。

結局　やれないのが　”任意のフォルダーに　データベースのインスタンスを　作成
出来ない”
こと　のみです。

で、質問ですが

１　このまま、手順を進めて行って、ネットで在庫確認が可能な　ＷＥＢページが作
成可能か
        （データベースフォルダとＷＥＢページに　　リンクを張って、別の場所に
あっても構わないので        しょうか？）

２　現状の　6.5.3-7　を　削除して　７．２　を再インストールする　べきか

        ところが　rpm -e postgresql　と　やっても　エラーで　削除出来ない

３　このまま　７．２　を　かぶせて　インストールしても　いいのか
        教科書と　現状　上記のように　フォルダーが　異なるので　かぶさるかど
うかも　不安ですが
