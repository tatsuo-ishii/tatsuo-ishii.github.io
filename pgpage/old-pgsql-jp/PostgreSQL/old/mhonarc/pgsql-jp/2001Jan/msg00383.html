> PostgresSQLに一度に何接続可能なのかということを調べる場合はどこを見ればいい
> のでしょうか。


6.5.3での話しですが、「make install-man」 でインストールされる
MANファイル(src/man)を閲覧できるように適切に設定すれば、

	% man postmaster

でオンラインマニュアルを見る事ができます。
また、HTMLファイルのドキュメントファイルが  doc/*.tar.gz の中にあります。
まずこれらの中の 「postmaster」の記述をみれば、

| NAME
|        postmaster - run the Postgres postmaster
|
| SYNOPSIS
|        postmaster [-B n_buffers] [-D data_dir] [-N n_backends]
|             [-S]    [-a   system]   [-b   backend_pathname]   [-d
|             [debug_level]] [-i]  [-n]  [-o  backend_options]  [-p
|             port] [-s]
|
| DESCRIPTION
|       The  postmaster manages the communication between frontend
|
|       ...
|
|       -N n_backends
|             n_backends is the maximum number  of  backend  server
|             processes  that  this postmaster is allowed to start.
|             ...
|             ...

という記述がみつかります。doc/*.tar.gzを展開して御一読下さい。


> また、排他的制御は自動で行われているのでしょうか。

排他的制御とは、たぶん、トランザクションやロックの事をおっしゃっているん
だと思いますが、データベースの一貫性を保つ為の機能はきちんと存在しています。
begin , end , commit, rollback, abort, lock などのSQLキーワードを
を調べたり、SQLの教科書をみてください。
別T端末画面でpsqlを2つ起動してて、更新トランザクションをステップbyステップ
で実行すると、動作が良く理解できるのではないでしょうか?
