北村さん、松嶋 さん、コメントありがとうございました。

松島さん wote

> 上記 SQL を見ると、「'」がエスケープされています。
> おそらく、php3.ini の magic_quotes_gpc の設定が
>
> magic_quotes_gpc = On   ; magic quotes for incoming GET/POST/Cookie data
>
> となっているのでしょう。
>
> 上記を Off にして httpd を再起動するか、北村＠アシェット様のおっしゃるとおり、
> stripslashes 関数を使うか、というところが解決方法かと。

sql.php3 の echo $sql; の後に

$sql = stripslashes( $sql );

を挿入したら select * form table_1 where name like '%John%'; が動作しました。ま
た、
root で /usr/local/lib/php3.ini の magic_quotes_gqc = Off にして、
/usr/local/apache/bin/apachectl restart を実行したら、上の1行がなくてもきちんと
動くことを確認しました。

北村さん wrote

> マニュアル読みましょうね。
> http://snaps.php.net/manual/ja/html/function.stripslashes.html

さっそく、
$ cd ~/index/php_manual
$ mknmz /home/myaccount/src/php-3.0.12jp-beta/doc-jp/
として php マニュアルの namazu インデックスを作って次のスクリプトで、
phpnmz <keyword> で全文検索できるようにしました。

#!/bin/sh
namazu -h $@ /home/myaccount/index/php_manual/ | w3m -T text/html
# phpnmz

もちろん、これとは別に頭からしっぽまで真面目に読むつもりです。

ありがとうございました。
