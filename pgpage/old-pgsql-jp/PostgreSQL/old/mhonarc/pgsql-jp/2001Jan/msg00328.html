kamitaです。
久しぶりの発言です。


今，PostgresとPHPの組み合わせで
テーブルの定義書から
いかに効率的にテーブルを作成し
テストなどを行えるように
試行錯誤しています。

WindowsのAccess2000でテーブルを作成し
ODBC経由で、Linux上のPOSTGRESに
エクスポートするのが、かなりいいなあと
思ってやっているのですが
(テーブルも簡単に作れますし、ドキュメントもできますので)

テーブルの定義で
id serial primary key
とPostgres上で定義したいのですが

Accessで、
id オートナンバー型とやってエクスポートしても
id integer とだけなってしまいます。

ALTER文などでこれをあとから
id serial primary keyに修正することは
できないでしょうか。

psql上で\h alterを見てみたのですが
いろいろ書いてみてもうまくいきません。

もしくは、みなさんテーブルの定義から
実際の作成、テスト、SQL文の作成までなど
どのような手法、ツールでやっているか
教えていただけませんか。
サンプルデータの入力などについても。

私は，まずExcelでテーブルの定義を書いてから
それを見ながら、create文をエディタで書いて
psqlもしくは、phpなどを利用してテーブルまで
作成しています。
