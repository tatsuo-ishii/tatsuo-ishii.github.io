昔から何度も同じような質問が出ているので、こういうのは過去ログ検索
ページからでも捜してもらいたいところです。
PostgreSQL でも MySQL でも Oracle, Sybase などでも、みんな話は同じです。

DBMS を使いにいくクライアントプログラムをコンパイルしたり実行したり
するためにはヘッダファイルとライブラリが必要です。

DBMS そのものをインストールしてもいいでしょうが(動かす必要はない)、
一般にはクライアント環境のみをインストールすればよいのです。

PostgreSQL はライセンスフリーですから、DB サーバマシンと Web サーバマシン
の稼動 OS が同じなのであれば、DB サーバから持ってくればいいのです。

--with-pgsql は、ようするにヘッダファイルとライブラリがどこにあるのか
指示しているのですから、あなたの好きなところに置いて、そこを見に行く
ように指定すればいいだけです。

こういう話は C でプログラムを書く場合のイロハです。
最近は C を書いたことのない人が色々やろうとするので、前提知識の多くを
知らずに悩んでるみたいですね。
