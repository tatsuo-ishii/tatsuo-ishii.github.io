ところで，「Table 5-2. ソースは綺麗にするのだ」のところで，「何かツー
ルを作ってみんなで使ってるんでしょうか?」という疑問がありましたが，そ
ういうツールが実際あります．src/tools/pgindent がそうです．一通り開発
が進み，ベータ期間が終了に近づいてくると，Bruceさんがこれを使って一気
に全ソースをフォーマットします．だから，他の開発者は結構適当にコーディ
ングしていても大丈夫です．最初のころは私のCのコーディングスタイルは他
の人と全然違っていたので，pgindentされた後のソースはまるきり違ったもの
になっていて，まるで人のソースをみているような気がしたものです．

それと，私はemacs派ですが，ソースをきれいに見るためのelispコードが本家
で配布されていて，これを使うようになってからemacsでもソースがみやすく
なりました．
# でも，これを使うとCのソース以外でもタブ幅が変わってしまうので，ちょっ
# と困ります:-)

;; check for files with a path containing "postgres" or "pgsql"
(setq auto-mode-alist (cons '("\\(postgres\\|pgsql\\).*\\.[ch]\\'" . pgsql-c-mode) auto-mode-alist))
(setq auto-mode-alist (cons '("\\(postgres\\|pgsql\\).*\\.cc\\'" . pgsql-c-mode) auto-mode-alist))

(defun pgsql-c-mode ()
  ;; sets up formatting for Postgres C code
  (interactive)
  (c-mode)
  (setq-default tab-width 4)
  (c-set-style "bsd")             ; set c-basic-offset to 4, plus other stuff
  (c-set-offset 'case-label '+)   ; tweak case indent to match PG custom
  (setq indent-tabs-mode t))      ; make sure we keep tabs when indenting
