> ところで、中身を知りたいのですが、
> havingでカウントしているのには、どのような意味があるのでしょうか？

こんなに短いSQLでも説明が必要ですか？そのまま数えているだけですよ！
例に上げたSQLを分解すると こんな風になります。

A select
A  a.fa01,
A  a.fa02
C  ,count(a.fa01) <== 見やすいように追加したが "," の位置が無理矢理
A from a, b
B where (a.fa01 = b.fb02)
B   and (b.fb03 in (1,6))
C group by
C  a.fa01,
C  a.fa02
D having
D  (count(a.fa01)=2);

実行する時には余分な文字を除き、Aのみ、A+B,A+B+C,A+B+C+Dの各段階で
どんな結果が帰ってくるか確認して下さい。
それでもわからないようでしたら、Accessのヘルプや入門者用のSQL解説本
を購入されて基本的な部分を学習する事が先です。

注:Aのみの実行結果は入門者だと訳ワカランかも？

> 変なテーブルで混乱させて申し訳ありません。
> サンプルでこういう形にはなっていますが、本番ではこれにまだテーブルが追加
> されたりしています。

テーブルが変というより必要な条件が書かれていないから、他の方も想像で
答えられているのです。
テーブルを抽出する条件によって必要なSQLも変ります。
私のSQLの場合、FB02/FB03の組み合わせでユニークという前提が無いと
予想外の結果になりますよ。
