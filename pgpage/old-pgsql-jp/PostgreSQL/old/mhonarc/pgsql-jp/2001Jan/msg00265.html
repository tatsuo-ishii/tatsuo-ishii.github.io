In message "[pgsql-jp 19571] Re: 日付の default 入力について",Jun Kitamura wrote...

>ですが、この場合 entry_day の default 'today' には「create table を実行し
>た日付」がデフォルト値として入ることになる（と思います）。
>つまり、いつ何をインサートしても、デフォルトが 2000年1月17日 なのですよ。

ああ、私の読み違え(^^;)
テーブル作った時にその日決め打ちで
データベースに突っ込んでるのか。


>実際には log にあるように、「insert 文を実行した日付」が入って欲しいのに
>・・・って事だと思ったのですが。

そう読めばそれが正しそうですね
そうすると、元の方の

>    入力画面作成して当日入力は正確な年
>   月日（01-17-2001)が出力されるが翌日も
>   today->01-17-2001と変わりません。

って、初めっから値が入ってるんだから入力(insert)できるわけなくて
更新(update)しなきゃいけないわけなんだけど
ホントに入力されてるんだろうか？という疑問が
やってることがまったく意味不明だなぁ。

なのでテーブル作る時点で「today」で日付を突っ込んでるのが敗因で
insertする時に「now」で登録
updateする時も「now」で更新
ってしたいんでしょうかね？　もしかして

仮にテーブル作る時点で「today」で日付を突っ込むなら
updateする時に「now」で更新かけないと一生変わらんわなぁ

ぼくは
後々初回登録日を忘れさせたくないこともあるかなと思う時は
初回登録日と最終更新日を別にしておいて
1回目だけはcreate_dayに「now」、update_dayに「now」
更新かけたときはupdate_dayだけ「now」とかよくやります。


>まぁ、われわれ部外者があーだこーだ言ってても、本人が居なきゃしょうがない
>ですね。笑

確かに、一体何をどうしたいのか説明欲しいかも
