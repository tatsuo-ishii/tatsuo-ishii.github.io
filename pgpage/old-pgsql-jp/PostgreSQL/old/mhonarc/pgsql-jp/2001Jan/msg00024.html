Keiji Watanabe さん曰く:

> なんとなく、JST/GMTあたりが気になるのですが。
> INSERTステートメントはJSTで行われてて、date_part関数はGMTしか
> 判断しないとか。

御教授して頂いたことを参考に，とりあえず，明示的にタイムゾーンを
指定しました．
# 対処方法を勘違いしていましたら，申し訳ありません．

FreeBSD，Kondara Linux共に，タイムゾーンがJSTになっておりますので，

# SET TIME ZONE = 'JST';

でタイムゾーンをJSTにしました．念のため，テーブルをDROPしてから，
新たに下記のようにしてみました．

# CREATE TABLE test(test_date date, txt text);
CREATE
# INSERT INTO test VALUES ('2001-01-05','hoge');
INSERT 139490 1
# SELECT date_part('day',tdate) from test;
date_part 
-----------
         4
  (1 row)
	  
残念ながら，結果は変わりませんでした．
# SET TIME ZONE = 'GMT';
してから，上記の操作をしてもみましたが，やはり，結果は同じでした．

その他に対処方法はありますでしょうか．
