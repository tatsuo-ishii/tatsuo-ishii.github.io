snip...

> そうなんですか? 7.0.3でmakeすると，
>
> ld -Bdynamic -shared -soname libpq.so.2.1 -o libpq.so.2.1 fe-auth.o fe-connect.o fe-exec.o
fe-misc.o fe-print.o fe-lobj.o pqexpbuffer.o dllist.o pqsignal.o   common.o wchar.o conv.o

> ってなって，-sonameで(C)を指定しているように見えます．

>
> $ objdump -p libpq.so.2.1 |grep SONAME
>   SONAME      libpq.so.2.1
>
> ってなってますが... もしかして，7.0.3 のRPM固有の問題?

私のところでもありました。
PHP の rpm を PostgreSQL 7.0.3 の rpm を利用して作成し、その PHP の rpm を
別のマシン(PostgreSQL 7.0.2 の rpm が入っている)にいれると require libpq.so
でインストールできなかったことがあります。
その時は、PostgreSQL を 7.0.3 に Update しましたけど。
ちょっと時間ができたら調べてみます。

ついでに、VinePlus 用に php3-3.0.18_i18n と php-4.0.4 の RPMS を作っています。
--enable-versioning を指定していますので同居可能になっています。
必要でしたら、お近くの VinePlus のミラーで取得してください。
