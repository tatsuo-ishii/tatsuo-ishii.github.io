篠木といいます
いつも当メーリングリストではお世話になっています
PostgreSQLを使いだしてちょうど１年目ですが思わぬトラブルで悩んでいます

快調に稼動していたPostgresですが先日、都合によりPostmasterを停止しない
でLinuxをShutdownさせました
後日、Postmaster -S -iを実行したところ（いつも手動で起動しています）

FATAL: StreamServerPort: bind() failed: 許可がありません
        Is another postmaster already running on that port?
        If not, remove socket node (/tmp/.s.PGSQL.5432) and retry.
/usr/local/pgsql/bin/postmaster: cannot create UNIX stream port

と出てきました。/tmpをls -aで調べましたが.s.PGSQL.5432ファイルは、あり
ませんでした。そこでtelnet localhost 5432 を実行したところ
telnet: Unable to connect to remote host: 接続を拒否されました
と言われましたので /etc/services を調べたところ
postgres        5432/tcp                        # POSTGRES
postgres        5432/udp                        # POSTGRES
とちゃんとあります。
次に、postgres$ postgres 5432/tcp を実行したところ次のメッセージでした
FATAL 1:  Database 5432/tcp does not exist in pg_database
FATAL 1:  Database 5432/tcp does not exist in pg_database

OSは Vine Linux2.0 PostgreSQLは Ver6.5.2です
メッセージからすればTCP/IPの問題で、Port5432に接続できないのが主原因だと
思うのですがここから先、どうすれば接続できるのかわかりません
是非、皆様のお知恵で急場を乗り切りたくよろしくご教授ください。
