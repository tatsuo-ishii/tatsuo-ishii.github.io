いえ、そんな事はないですよ。

フォーム入力から改行コード込みのテキストを受け取り、それをそのまま
JDBC 経由で insert などする Servlet プログラムを書きましたが、
エラーにならずにちゃんと動きます。

このレベルの SQL 文のパースは psql や PHP が行っているのではなく、
DBMS 側… 今回のケースであれば PostgreSQL 本体が行うものです。

JDBC の場合でも、JDBC ドライバ側が SQL 文のパースを行っているの
ではなくて、DBMS 側に投げてその結果を取得する形になってるはずです。
ソースは調べてませんが、(^^; そのはずです。

そうじゃないと、話が面倒になってしょうがないですから。
