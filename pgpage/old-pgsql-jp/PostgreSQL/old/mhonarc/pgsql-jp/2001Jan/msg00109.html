On Wed, 10 Jan 2001 09:07:01 +0900 (JST)
Nishiguchi Hidetoshi <h_nishiguchi@yahoo.co.jp> wrote:

NH> 今、psqlではupdateできるのに、cgi経由でperlからは
NH> updateできないという現象が起こっていて悩んでいます。

　チェックしなければならないことを以下に挙げときます。

・connectは行われているか?
・selectはできるのか?
・updateのときにどのようなエラーが返ってきているか?
・トランザクション開始をcommitせずにabortしていないか?

　実際にやるべきことは、小菅さんが挙げられたとおりです。


NH> updateのSQLは、以下のとおりです。
NH> update A set B = 3 where C = 'AA';

　SQLを疑う前にすることがたくさんあります。


NH> 権限のほうも、今は、all、publicで与えてあるのですが、
NH> うまくいきません。

　気を抜くと、pg_hba.conf 問題にすぐぶちあたってしまいます。わたしはよく、
うっかりpg_hba.confを緩め忘れてしまいます。サーバ起こすとき一度しかハマ
らないのでなかなか覚え切れません。って自分のことは置いといて。


NH> インストールしているソフトは、以下のとおりです。
NH> postgres 6.5.3
NH> perl5 + pgsql
NH> apache1.3.13

　それぞれ、どのようにインストールしましたか?

　2行目の、 pgsql というものに関してはわたしは存じ上げません。わたしはモ
ジュールPgしか使用したことがありません。これはpostgresqlのソースからイン
ストールしました。また、Cのライブラリ名ということでしたら、libpqが正しい
名称です。


NH> アドバイスお願いします。

　プロに必要なのは、コーディング能力・設計能力もさることながら、エラー対
処・テスト手順、なのだと思います。「何が悪いのか」をご自身で突き止められ
るように、あれこれ仕掛けを入れるように心がけましょう。Pgなどには、バック
エンドから返されたエラーメッセージを取り出すメソッドなどが用意されていた
はずです。
