[pgsql-jp 19737] Win98 クライアントからODBC 接続できない

  • From: ando_t@denpa.ac.jp (情報システム課　安藤　巧（Taku Ando）)
  • Date: Tue, 30 Jan 2001 15:23:40 +0900
皆さん始めまして、初めて投稿させていただきます。

RedhatLinux6.2Jインストール時にPGSQLも入るよう指定して
インストールを行ったLinuxマシンがあります。
このマシンは現在Apacheが立ち上がっていて、
PGSQLデータベース内に作ったテーブルのデータをWebからPHPで
動的に表示できます。
また、Win98クライアントからブラウザを通してLinuxサーバ上のWebページが
表示できますし、TelnetやFTPで接続も出来ます。

Win98からAccess97でPGSQL内のテーブルをリンクして使いたいと
考えているのですが、ODBCがうまく繋がりません。

Win98にはPostgreSQL ODBC Driver 6.40.0004英語版と、
PostgreSQL ODBC Driver 日本語版 2000/6/17版をインストールしました。
この状態でODBCデータソースを作成し、Access97からリンクテーブルを
作ろうとしたとき、データソースを選んだところで
Accessのエラーメッセージで

ODBC--呼び出しが失敗しました。
The database does not exist on the server
or user authentication failed.(#105)[Microsoft][ODBC Driver Manager]
接続は開いていません。(#0)

と出てしまいます。
クライアント側ではなく、サーバ側に何か設定が必要なんじゃないかと
思っているのですが、何をすればいいのでしょうか？
もし心当たりがあれば教えていただけないでしょうか？
よろしくお願いします。

以下、サーバの環境やら私がいじくったところを思いつく限り書いてみます。
///////////////////////////////////////////////////////////////////////////
１、RedHatLinux 6.2Jをカスタムインストール
　　選択したインストールパッケージは
　　・Networked Workstation
　　・SMB(Samba) Server
　　・Anonymous FTP Server
　　・Web Server
　　・Postgres(SQL) Server
　　・Development
　　・Kernel Development
２、ネットワークの設定をし、Win98クライアントからTELNET、FTPが
　ちゃんとできることを確認。
３、Apacheの設定をし、Win98クライアントからApacheのページが見れた
４、HTMLに<?phpinfo()?>を埋め込み、クライアントから結果が見れた
５、PGSQLの設定をし、postmaster -S -iで自動起動するよう設定した
６、サーバからpostgresユーザでinitdb、createdbを実行し、psqlでテーブルを作成した。
７、PGSQLユーザにnobodyを追加した（現在PGSQLのユーザはpostgresとnobodyの２つのみ）
８、PHPでデータベース内のテーブルの中身を表示するサンプルをつくり、
　クライアントからブラウザでちゃんと出ることを確認
９、クライアントにODBCドライバをインストール
１０、クライアントのデータソースを作成
１１、Accessからリンクテーブルの作成でデータソースを選択・・・エラー
///////////////////////////////////////////////////////////////////////////
