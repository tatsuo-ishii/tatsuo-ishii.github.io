＃本日持ち帰ってきました（が、今度は職場で読めない...f^^;）。

教えて頂いた、

> 4) 「■現在の時刻の取得方法」
>
>    SELECT CURRENT_TIMESTAMP;
>
>    っていうか，これがSQL標準です．

を今日実験してみました。PostgreSQL は、

　nagai=# SELECT CURRENT_DATE;
　    date
　------------
　 2001-01-15
　(1 row)

　nagai=# SELECT CURRENT_TIME;
　   time
　----------
　 19:59:40
　(1 row)

　nagai=# SELECT CURRENT_TIMESTAMP;
　       timestamp
　------------------------
　 2001-01-15 19:58:42+09
　(1 row)

と優秀です。precision は、

　nagai=# SELECT CURRENT_TIME(1);
　NOTICE:  CURRENT_TIME(1) precision not implemented; zero used instead

となり、パーサは、大変よく標準を分かっていらっしゃるようです。;-)

逆に、他のRDBMS もためしてみましたが、

　I:
　SQL> SELECT CURRENT_TIMESTAMP;
　Statement failed, SQLCODE = -104

　Dynamic SQL Error
　-SQL error code = -104
　-Unexpected end of command

　S:
　1> SELECT CURRENT_TIMESTAM
　2> go
　Msg 207, Level 16, State 2:
　Line 1:
　カラム名 'CURRENT_TIMESTAM' が正しくありません。

　O:
　SQL> SELECT CURRENT_TIME;
　SELECT CURRENT_TIME
　                  *
　ERROR at line 1:
　ORA-00923: FROM keyword not found where expected

となり全滅でした。(^^;

では、失礼します。
