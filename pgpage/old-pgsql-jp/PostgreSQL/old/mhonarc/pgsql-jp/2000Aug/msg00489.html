[pgsql-jp 16875] IEでファイルダウンロード時にファイル名を渡す方法

  • From: Kuritani@ns.icc.ogis-ri.co.jp (栗谷 泰輔)
  • Date: Wed, 23 Aug 2000 11:30:22 +0900

＠はじめまして 栗谷＠NJKです。

  現在 PHP TurboLinux Apacje Postgres でファイルのダウンロードアプリを開発しています。
  内容は バイナリのファイルを読み込んでそれをバイナリリードで出力する方法をしています。
  なぜ 直接ファイルにリンクしないかというと、アップロード機能で、ファイルをサーバにアップロード
  するときに 資源を節約するために サーバ上では圧縮しています。名前もこちらで決めた一意な名前をつけ
  ています。
  そこで ダウンロードする時に 解凍して本来の名前を付けてダウンロードするのですが なにぶんうまくいきません。
  過去のMLをみて 参考にしようと思ったのですが
  どれも うまくいきませんでした。
  
  リンクでファイルをダウンロードする場合なのですが、
  URL引数に http://hoge/cgi-bin/download.cgi/file_name?param1=xxxxxxx&param2=yyyyyyy&param3=zzzzzzzz
  とを追加すると NetScapeでは うまくファイル名が表示されるのですが、
  IEだと EXEの拡張子のものが 拡張子が削除されてしまいます。

  そこで
  $str1 = "Content-disposition: attachment; filename=$dl_filenm";
  $str2 = "Content-Type: application/octet-stream";
  $str3 = "Content-Length: $filesize2";
  $header($str1);
  $header($str2);
  $header($str3);

  /* ファイルのダウンロード */
  $fd = fopen ($filename, "r");
   $contents = fread ($fd, filesize($filename));
  fclose ($fd);
  echo $contents;

  としたところ EXEの拡張子は付いたのですが 名前がEUCコードで（文字化けして）保存ダイアログに表示されています。
  この回避の仕方をご存知の方がおられれば 教えてください。
