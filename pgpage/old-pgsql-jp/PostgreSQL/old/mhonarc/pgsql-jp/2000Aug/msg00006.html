石井です。

> PostgreSQL6.5.3を利用していたときに int(timestamp)として、
> 得られた数値をグラフを作成する際の時間軸の座標に利用して
> いたのですが、
> PostgreSQL7.0で同様にint(timestamp)とすると、
> 
> ERROR:  Function 'int(timestamp)' does not exist
>         Unable to identify a function that satisfies the given argument types
>         You may need to add explicit typecasts
> 
> 
> となってしまいます。

7.0ではtimestampは6.5のdatetimeと同等なので、まあしょうがないですね。

> PostgreSQL7.0でint(timestamp 'now')と同等の出力を得るには
> どうすればよいのでしょうか？

同等かどうかわかりませんが、

select date_part('epoch',timestamp 'now');

で 1970/1/1からの経過秒が出てきます。

test=# select date_part('epoch',timestamp 'now');
 date_part 
-----------
 965086757
(1 row)

もしかしたら6.5のint(timestamp 'now')は1900/1/1からの経過秒かも知れま
せんが、その時は適当に調整して下さい。
