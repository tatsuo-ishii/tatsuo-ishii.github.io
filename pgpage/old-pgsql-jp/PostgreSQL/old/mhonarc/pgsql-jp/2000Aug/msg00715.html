     AS演算子で別名をつける手法はごく一般的ですが、
     文字をASで別名をつけるとorder byが効かなくなります。
     
     仮に、
     create table table_a(emp_no integer,name text);
     create table table_b(emp_no integer,name text);
     として、
     
     ******************************************
     select name, '1' as ID from table_a
     union all
     select name, '2' as ID from table_b
     order by ID;
     ******************************************
     
     エラーメッセージは、
     「Unable to identify a binary operator '<' for types
          unknown and unknown」と出ます。
     
     もともと、PSQLではできないものなのでしょうか？
     代替できるものはありますでしょうか？
     
     どなたかご存知でしたら教えて下さいませんでしょうか？
