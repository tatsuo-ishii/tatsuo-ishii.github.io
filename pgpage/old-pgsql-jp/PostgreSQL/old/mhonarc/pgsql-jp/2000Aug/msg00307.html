  早速質問なのですが，7.0 と 7.0.2 では
  正規表現の解釈に違いがあるのでしょうか？

  実は自分の所のサーバで，

      where name ~ '^[$ini].*'

  と言う風に検索をして，正しい結果が返ってくるのですが，
  納品した客先では登録されているすべてのデータが返ってくるというのです。
  私の使っている PostgreSQL は 7.0，客先は 7.0.2 です。

  like を使えばいいハナシなのですが，
  これは頭文字検索でして，例えば頭文字「は」を検索したい場合，
  濁音と半濁音も一緒に検索したいのです。

      where name ~ '^[は|ば|ぱ].*'

  like だと煩雑になってしまうのでこのような方法を採っています。
  手元では正しい結果が返るのに，なぜこのような事が起こるのでしょう。
  PHP だけではなく，psql からでも同じように動作しているようです。

  明らかに向こうがおかしいのに，
  こっちが折れて like を使うように書き換えるのはクヤシイです（笑）。
  どなたか解決方法をご存じでしたらご教示願います。
