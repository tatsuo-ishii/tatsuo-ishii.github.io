なんか怪しい気がします。例えばこんなかんじ。

s-kosuge@bash-2.02$ cat a.pl
# !/usr/local/bin/perl

print "hoge\n";

s-kosuge@bash-2.02$ perl -cw a.pl
a.pl syntax OK
s-kosuge@bash-2.02$ perl a.pl
hoge
s-kosuge@bash-2.02$ ./a.pl
./a.pl: print: command not found
s-kosuge@bash-2.02$

「それは大丈夫でした」ではなくて、実際に書いてあるものを
カット＆ペーストしてくれたほうがいいです。パスについても、実際に
ls コマンドを実行した結果をつけてくれるほうが明白です。
もしも、
#!/usr/local/bin/perl

でなくて

# !/usr/local/bin/perl

と記述していたものを自分の判断で「それは大丈夫でした。」と回答して
いたとしたら、全くの無駄骨になってしまいます。> 元記事の方
