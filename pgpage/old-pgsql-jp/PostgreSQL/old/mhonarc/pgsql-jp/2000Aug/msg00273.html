いつもお世話になっています。
稲田＠京都です。
まだまだ初心者なもので下記についてお尋ねします。

pg_dumpでデータベースのバックアップをいつもしているのですが、cronでこれを自 
動処理しようと、
su - postgres で　crontab -e を実行して、

  0 4 * * * /usr/local/pgsql/bin/pg_dump -f /tmp/hanbai.dump hanbai

を保存して、実行を待っていたのですが、一向に/tmpディレクトリーには当該ファイ 
ルは作成されず、変わりに下記のようなエラーメッセージが帰ってきました。

Subject: Cron <postgres@server> /usr/local/pgsql/bin/pg_dump -f 
/tmp/hanbai.dump hanbai
X-Cron-Env: <SHELL=/bin/sh>
X-Cron-Env: <HOME=/home/postgres>
X-Cron-Env: <PATH=/usr/bin:/bin>
X-Cron-Env: <LOGNAME=postgres>
X-UIDL: DZ/!!T*'!!HCN!!p26"!

/usr/local/pgsql/bin/pg_dump: error in loading shared libraries: 
libpq.so.2.1: cannot open shared object file: No such file or directory

これを見る限り libpq.so.2.1 
を開けないようなのですが、手作業でdumpするときにはまったく問題が無いし、？っ 
と、考えているのですが・・・・？

また、この libpq.so ってapache の httpd.conf 
の設定時に（PHPとの関係に於いて）意識したことが無いのですがそもそもどのよう 
に重要で何をする物なのでしょうか？（勉強不足で済みません！）

とにかくシェルスクリプトの勉強も始めたばかりで、勉強不足を痛感しながらいろん 
な本を読み漁っています。しかしながら当面この crontab 
問題を解決しなければならず、正しい表記方法等の解決策をご教授願えれば幸いです。

よろしくお願いいたします。

なお環境は
LASER5 LINUX 6.0
PostgreSQL 7.0.2
PHP-3.0.15-i18n
です。
