　通常自分ではpostgresのためにRPMを使うことはないので、経験
談ではありませんが、....
　手元のRedhat 6.2[日本語版(ftp)]のCD内のpostgres関係
のrpmファイルを確認してみました。


まず、postgresのサーバ本体のプログラムは

 postgresql-server-6.5.3_jp-1.i386

なのでここから入れるのが筋というものでしょう。

ただし、同rpm内のDESCRIPTIONにも下記のような記述があります
が、
---------------------------------------------------
 You should install postgresql-server if you want to create and
 maintain your own PostgreSQL databases and/or your own
PostgreSQL server. You also need to install the postgresql and
postgresql-devel packages.
-----------------------------------------------以上
postgresql-server-6.5.3_jp-1.i386内のINFO/DESCRIPTIONより

　みにすとりいさんのダウンロードしたファイル以外にもpostgres用
のrpmが２種類あるはずかと思います。
　そのうちの postgresql-6.5.3_jp-1.i386はpostgresのclientが入っ
ているrpmです。これにはpostgresのサーバと対話するpsqlがふくま
れているので、「これからpostgresを始めて」という用途には必須だと思
います。もう一度vineサイトを確認してみたらどうでしょう。

  後はみにすとりいさん自身がpostgresを使って何をしたいかという
ことによって、どれを入れるかが決まってくると思います。

  例えば石井さんのシーラカンス本でも取り上げられているpgaccess
を使いたいならtclが必要ですし、同じくシーラカンス本でも記述のある
regression test（install後に自分の環境でpostgresが正しく動作する
かどうかのチェックテスト)をするのにはtestが必要です。
（この本には大変お世話になりました。石井先生ありがとうございます。
この場を借りてお礼を言わせて頂きます。）
　javaのapplicateion,applet,Servelet,JSP等からアクセスをする
プログラムを書きたければjdbc、M$なクライアントをフロントエンドに
使いたければodbc、.....etcと必要に応じてインストールなさればいいと
思います。

　まあ、ストレージに余裕があって、これから、postgresのことを一通り
研究したいということであれば全部入れてしまうというのも良いのでは
ないでしょうか。

　それから蛇足ながら、MLへの投稿のときにはHTMLメールは避けた
方がいいですよ。様々なワークステーション（またはモバイルまたは？）
環境の方が参加していますので、誰もがHTMLメールが読めるとは限り
ませんので。。
