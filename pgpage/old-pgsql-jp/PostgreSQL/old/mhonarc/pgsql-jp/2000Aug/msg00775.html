Linuxをさわり初めて５年ほどですがふと書店に並べてあった
シーラカランス本の中にPHPを使えば誰でも簡単にシステムが作れます
という内容に惹かれて無謀にも在庫管理システムを作って試運転中です。
RDMSについては初心者で廻りに詳しい人もいませんのでもっぱらマンモス、
シーラカランス本頼りですがPHP,SQLはスクリプトのためデバッグが楽で
プログラム初心者には取っつきやすくそしてたいへんおもしろいですね。
またLinuxがwindowz9xなどと違いNET OSであるということを再認識した次第です。

早速質問で恐縮ですがnull値を含むテーブルのexceptが正しいが得られ
ないという問題がどうしてもわかりません。

CREATE TABLE test1 (hinmei text);
INSERT INTO test1 values ('BANANA');
INSERT INTO test1 values ('LEMON');
INSERT INTO test1 values ('ORANGE');
INSERT INTO test1 values ('APPLE');
INSERT INTO test1 values ('MANGO');
INSERT INTO test1 values ('KIWI');

CREATE TABLE test2 (hinmei text);
INSERT INTO test2 values ('ONION');
INSERT INTO test2 values ('ORANGE');
INSERT INTO test2 values (null);
INSERT INTO test2 values ('POTATO');
INSERT INTO test2 values ('KIWI');

上記2つのテーブルに対してexceptを行うと結果が得られません。

select * from test1 except select * from test2 ; 
 hinmei 
--------
(0 rows)

なお不思議なことに逆はOKで
select * from test2 except select * from test1 ; 
 hinmei 
--------
 ONION
 POTATO
(2 rows)

またnull値のカラムを削除すると正しい答えが得られますし
null値を空文字列と入れ替えても正しい答えが得られます。
これは仕様なのでしょうか、たぶん初歩的なミスをおかしていると
思いますがご指摘いただけると幸いです。

環境：Slackware7.0 PostgreSQL-7.0 Apache-1.3.12 PHP-3.0.16
