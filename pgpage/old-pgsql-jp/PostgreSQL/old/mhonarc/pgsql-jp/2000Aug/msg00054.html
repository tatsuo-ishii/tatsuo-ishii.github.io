増山です。　お世話になります。

> こちらの--enable-multibyte 指定のほうに進んで、initdb の
> 失敗原因を突き止める方が正道なのではないかと。

失礼しました。　configure --enable-multibyte=ECU_JPで
インストールすると、initdbで以下のようなエラーが出ます。

------------------------------------------------------
initdb: pg_encoding failed

Perhaps you did not configure PostgreSQL for multibyte support or
the program was not successfully intalled.
------------------------------------------------------

別の議題「initdbのエラーについて」にあるように環境変数は設定してます。
shellがshなのでちょっと書き方は違うのですが、
echo $PGLIB   echo $PGDATA でそれぞれちゃんと設定されているのを
確認出来ました。

> --enable-mb というオプションはありません。どういうオプションが
> あるかは、postgresql-7.0.2/INSTALL や postgresql-7.0.2/src/configure
> そのものを見るとか、./configure --help を実行するとわかります。
> 、、、のはずですか、--enable-mb なんてどこかに書いてありました?

マニュアルを見たわけではないのですが、ログの中にそのような指定方法で
書いてありました。　これだとinitdbはうまくいくのですが・・・。

> pg_ctl にどんなオプションをつけたのかわからないとか、
> 「．強制終了」という文字列以外に何も出ていないのか、とかが
> わからないので、入出力した文字列をそのままカット＆ペースト
> してくれたほうがいいです。同じメッセージは(略) とかで
> はしょってもいいと思います。

申し訳ありません。　以下のようなエラーが出ます。

$pg_ctl -w start
.強制終了
.強制終了
   （略）
.強制終了
.postmaster does not start up.

やはり configureのオプションは--enable-multibyteが正道なのでしょうか。
FAQ等いろいろ見てやってみたのですが、どうもうまくいきません。
よろしくお願いします。
