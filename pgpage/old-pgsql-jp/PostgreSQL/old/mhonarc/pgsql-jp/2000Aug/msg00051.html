徳家です。

私も極力日本語のテーブル名やカラム名を使って構築したい者です。(^^)
だけどPostgreSQLでは基本的にカラム名やオブジェクト名の長さは32バイト(有効31
バイト)に決められてます。
あまりに長い名前は不便でもありますし、残念ですが、その範囲でという事になりま
す。

シーケンスは個別に作る事も可能ですので、個別に作ってみてはいかがでしょうか？

参照
User's Guide
	CREATE SEQUENCE


あと、私流のテーブル管理の仕方ですが、
「テーブル名とカラム名の単語が重複しないようにする。」
と決めています。

沖本さんの場合のテーブルを私流にアレンジすると、

create table 物件特記事項code (
id sireal);

※半角英字を大文字にする場合はテーブル名やカラム名を「"」で括ると大文字小文
字も自在です。

このようにすれば、シーケンス名も"物件特記事項code_id_seq"と31バイト内に収ま
りますし、SELECT文をAlias無しで書く場合などの時も、「物件特記事項code.id」と
いう記述で書きやすいし視覚的にも読みやすくなります。(と、思ってる・・・・。)

いろいろ好みのテザインを考えてみられてはいかがでしょうか？
