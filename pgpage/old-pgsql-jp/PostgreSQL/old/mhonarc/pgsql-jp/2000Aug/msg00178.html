はじめましてナカノタダシと申します。

Access2000からあるテーブルに限って検索、更新が
できない現象が発生しています。
ODBCのログを見てみましたが原因がわかりません。
どなたか原因に心当たりのある方はいらっしゃいませんでしょうか？

●現象
以下のテーブルにデータがある場合に、Access2000から
テーブルのリンクでデータを検索すると、
「ODBC−呼び出しが失敗しました。」と表示されて、
データが表示されるところに「#NAME?」と表示されます。
こうなると、データの入力も更新もできません。

データが無い場合には、表示も入力も問題なくできます。
その他のテーブルでは検索、更新ともできています。

create table aaa       (
        a1   int2 not null,
        a2   char(3) not null,
        a3   int2 ,
        a4   char(40),
        a5   char(40),
        a6   char(40),
        a7   char(40),
        a8   char(40),
        a9   char(12),
        a10  char(12),
        a11  char(50),
        a12  text,
        a13  text,
        oper_cd int2 not null,
        oper_dt datetime not null,
        primary key ( a1, a2 )
);

●環境
RedHat6.1日本語版
PostgreSQL 6.5.3
ODBC 6.40.00.07
Access2000 + SR1
