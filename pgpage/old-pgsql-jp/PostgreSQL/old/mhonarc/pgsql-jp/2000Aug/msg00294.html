一般的なユーザーがテーブルやクエリー画面に触れることがないように調整すれば、
Delete表示問題を表面化せずに済みます。ＯＩＤは困り者です。

以下のような単純なVBAコードもPostgreSQLに通用する事が分かりましたので、
今後は、さらに挑戦しようかと思います。（生産管理実務中心ですが。）

# しかし、いくつか手間取った部分がありましたので
# 併せて報告します。
############### サンプル ###################

Private Sub CmdInsert_Click()
    Dim MyDB As Database, MySet As Recordset, MySQL As String
    Dim Shinamono As String, Kisetsu As String

    '// データ入力
    Shinamono = InputBox("品物はなにぃ？")
    Kisetsu = InputBox("旬はいつぅ？")

    '// データベース接続 (初の接続は感動もの)
    Set MyDB = Workspaces(0).OpenDatabase("", False, False, _
    "ODBC;DATABASE=test2;UID=username;PWD=iya;DSN=PostgreSQL;")

    '// SQLステートメント    （↓↓OIDの存在あるため、明記する）
    MySQL = "INSERT INTO shun (shinamono,kisetsu) VALUES _
    ('" & Shinamono & "','" & Kisetsu & "')"

    '//SQL実行 、正常に実行
    MyDB.Execute MySQL

    '//サブフォーム更新、正常に更新
    FrmFirst.Requery

End Sub
