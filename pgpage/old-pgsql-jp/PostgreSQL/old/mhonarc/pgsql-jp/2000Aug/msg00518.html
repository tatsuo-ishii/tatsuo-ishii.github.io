> 		Formattting Functions の灌を徊救してください。

たしかにto_char(),to_number()は蝗えるようですが、
讳の雇えていた蝗い数とは警しちがうようです。
讳の棱汤稍颅だったようですみません。

呵介の毋で、
≈钳勿が11和笆惧20和笆布の客∽を浮瑚する眷圭、
select id,age from tbl_A where age>='11' and age<='20';
とすると、id,ageがvarchar房のため矢机での浮瑚となり、
11和笆惧20和笆布の客と2和の客が浮瑚されてしまいます。

そこで、to_number()を蝗って
select id,age from tbl_A
   where to_number(age,'9')>='11' and to_number(age,'9')<='20';
のような蝗い数をしてしたいと蛔っていました。

悸狠、このSQLを悸乖すると、
ERROR:  Bad numeric input format ' '
が手笛されます。

そもそも≈varchar房のデ〖タを眶猛に恃垂して浮瑚する∽というような
浮瑚はできないのでしょうか々
