> クライアントにAccess2000をお使いでしたら、ODBCを使えば簡単にPostgreSQLのラー
> ジオブシェクトが扱えるからです。
> ODBC接続ではダメですか？
> 
> 方法はODBCの解説HTMLにあったと思いますが、ここにも書いておきます。
> 
> ＰｏｓｔｇｒｅＳＱＬにlo型というものをCREATE TYPEで作ってあげます。
> 
> CREATE TYPE "lo" (
>  internallength = 4,
>  externallength = 10,
>  input = int4in,
>  output = int4out,
>  send = int4out,
>  receive = int4in,
>  default = '',passedbyvalue
> );
> 
> (このlo型がODBC接続したAccess側でOLE型として認識されます。)
> 
> で、沖本さんの作ったテーブルに当てはめてみると、
> 
> CREATE TABLE 画像 (
>  GazouID INT,
>  Picture lo);
> 
> です。
> 
> このテーブルをODBC接続でリンクすると、
> Picture は、AccessでOLE型として認識されます。

この方法は知っていて試した事があるのですが、
諸々の事情によりその方法でいくのをやめています。
理由：
１．アクセス時間が長い・・・。
２．Access2000に対しては便利だが、Web側が
苦労する。（読み書きの点で）
３．lo型オブジェクトのバキュームがサポートされていない・・・。
    （lo_manageを使えばなんとかなるが・・・）
なのでOIDを使いたいと思っています。
理由：
１．アクセス時間が短い。
２．Web側が苦労なく画像の読み書きができる。
３．vacuumloが使える。
しかしながらAccess2000でどうやって画像を表示して
あげればよいかがわからないのです・・・。
どなたかご存知でしょうか？
