こんにちわ。たかはしです。

pgsql + perlがApacheで動かすとエラーになる問題で悩んでいます。
お知恵をかしてください。

LASER5 Linux 6.2（FTP版）にPostgreSQL 7.0.2をインストールし、
キャラクタラクターベースではコマンドでpsqlを発行すると
さくさくさく〜っと、結果が返ってきます。

そこで、Apacheを経由してWeb上でCGIとして動かすとエラーがでます。
他のCGIは問題なく動いているのでApacheの設定の問題ではないと思います。

web上のエラーは
　Internal Server Error
　The server encountered an internal error or misconfiguration and was
unable to complete your request.
　Please contact the server administrator, root@localhost and inform
them of the time the error occurred, 
　and anything you might have done that may have caused the error.
　More information about this error may be available in the server error
log.
　Apache/1.3.12 Server at localhost Port 80

Apacheのエラーログは
　[Tue Aug 29 16:28:18 2000] [error] 
　　(2)No such file or directory: exec of
/home/httpd/cgi-bin/search/search.cgi failed
　[Tue Aug 29 16:28:18 2000] [error] 
　　[client 192.168.50.157] Premature end of script headers:
/home/httpd/cgi-bin/search/search.cgi

と、なっています。
No such file or directoryとなっていますが、パスが間違っているようなミス
ではありません。

そこで過去MLで同じような現象を検索してみて
[pgsql-jp 12290] の以下の部分を参考にさせていただきました。
***
　srm.confにLD_LIBRARY_PATHは設定してあるでしょうか？
　なければ、LD_LIBRARY_PATHの設定が必要だと思います。
　私の場合は、
  SetEnv LD_LIBRARY_PATH /usr/local/pgsql/lib
***
さっそく、（srm.confではなく）httpd.confに
SetEnv LD_LIBRARY_PATH /usr/local/pgsql/lib　を追加して再起動してみまし
た。
もちろんLD_LIBRARY_PATHのパスも確認しました。

ところが、再起動してもweb上のエラーもApacheのエラーログにも変化がありま
せん。

ログの添付なので非常に長くて読みにくい文章になってしまったと思います。
申し訳ないですけど、どなたか　お知恵を拝借できませんでしょうか。。。

よろしくお願いします。
