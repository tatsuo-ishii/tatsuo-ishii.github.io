rpmのqueryオプションを使ってrpmパッケージの中のアーカイブ内容
を見ることができますね。

rpm -qilp <rpm file name>　あるいは　rpm -qilpv <rpm file name>

（下記に例あり）
などのrpmコマンドの利用で、rpmのinformationやrpmの中にアーカイブ
されているファイルのdirectry構造が確認できます。

rpmアーカイブ内のdirectory構造
= インストールするマシンのどのdirectoryにどのファイルがインストール
　されるのか　

なので、該当rpm内の/usr/bin下に何があるかでどのような実行ファイル
がインストールされるのかがわかります。それから、GUI派の人には、最
近は（？どういう時間感覚の最近かわかりませんが :-))gnomeに含ま
れて配布されるようになったMidnight Commanderがお奨めですね。
  アーカイブ・ファイルの中まで透過的に見ることができるので複数のアーカ
イブ・ファイルの中身のチェックをするなどの用途使うのには便利です。(実行
ファイル名はgmc)

例えば、
手元のPgのrpmを別途作成したdirectoryに移し、
そのdirectory内で

rpm -qilp ./* > rpminfo

で全rpm内の情報がrpminfoというファイル名でできますから、これを好みの
editorで開いて読めば中身はばっちりわかりますよね。

また、
rpm -qilp ./* | grep /usr/bin/psql
とか
rpm -qilp ./* | grep psql

または上記のrpminfoを使って
cat rpminfo | grep /usr/bin/psql
などと実行してみれば特定の実行ファイルが含まれているのかどうかがわかります。
