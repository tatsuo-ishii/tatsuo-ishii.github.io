>はじめまして。
>小林と申します。ご質問がございましてメールさせていただきました。
>
>以下の環境で
>postgresql-7.0.2-patched.tar.gz
>をインストールしました。
>./configure --enable-multibyte
>gmake all; gmake install
>と無事通った(色々なファイルはきちんとインストールされているっぽい)の
>ですが、initdbをしようとしたら、
>
>> %initdb
>> initdb: pg_encoding failed

　文字コードを指定してないからではないでしょうか。
initdbのオプションで文字コードを指定してあげれば、エラーにはならないと思います。
#configureの--enable-multibyteで=EUC_JPなどと文字コードを指定してませんから。

>> %pg_encoding 
>> /usr/libexec/ld-elf.so.1: Shared object "libpq.so.2" not found
>
>としたら、こういわれます。原因はわかりますでしょうか？

　これは、環境変数LD_LIBRARY_PATHで、ライブラリのパスが設定されていないためか
と思います。
