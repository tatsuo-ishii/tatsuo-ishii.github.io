　配列は使ったことがないのですが

"miyao" <miyao@miyazaki-nw.or.jp> wrote:
> create table test_tab
> (		
> 	n0	char(2),
> 	n1	char(2),
> 	n2	char(3),
> 	n3	char(2),
> 	n4	char(2),
> 	ac01	varchar(128),
> 	ac02	varchar(120),
> 	ac03	varchar(64),
> 	ac01	int,
> 	ac09	int,
> 	ac04	int,
> 	ac06	char(128)
> );
　たぶんn0からn4までを配列にしたいというような意味なんですよね？
　でしたら

　create table test_Tab (
	n	char(2)[5],
    ......   
　);

　というような感じになると思います(char(2)はいいのかしら？)。配列
の添字は1からになるそうですからn0からn4の5項目を配列で作ろうと思っ
たらn[1]からn[5]になるのだと思います。

　とりあえず、シーラカンス本を読んでみたらいかがでしょうか。第1版
は
　http://www.sra.co.jp/people/t-ishii/PostgreSQL/
　に著者の石井さんがPDFで公開してくださっています。ちなみに配列の
解説は９６ページあたりにあります。

あと、PostgreSQLには1行のデータ量の制限があったはずです。なので
> １つのテーブルがざっと３００項目ぐらいです。×２〜３個作成予定
は難しいと思います。そうでなくても1テーブルに300項目というのは相当
無理があると思います。
