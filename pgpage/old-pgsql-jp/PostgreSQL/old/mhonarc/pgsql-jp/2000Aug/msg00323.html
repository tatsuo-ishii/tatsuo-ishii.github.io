・/usr/local/apache/bin/apachectl start とcommand not found の
　間に何か書いてなかったか。単語でもなんでもいいですけど。
　「command not foundとでてしまいます」ではなくて、次のような
　エラーになってしまいます、のように述べて、入力したコマンドと
　実行結果をカット＆ペーストしてほしいです。例えばこんなふうにです。

postgres@bash-2.02$ b
./b: c: command not found

　これは、b というファイルにc という文字一つを書いて実行した例です
が、c というコマンドがないという情報がこのエラーメッセージから
わかります。

・ls -l /usr/local/apache/bin/apachectl の実行結果は? 実行権が
　設定されてないかも(可能性低い)
・そもそもapachectl はシェルスクリプトで、中でecho などのコマンド
　を呼んでいるので、それらが実行できるようにパスを設定しておく
　必要があるかもしれません。(こちらの可能性高い)
・/usr/local/apache/bin/ にはapachectl の他に何があるでしょうか。
　特に、/usr/local/apache/bin/httpd はあるでしょうか。

とりあえずそのくらいの情報が必要なのではと思います。
