以下、一般論ですが。

商用 DB とか、商用のシステム系統のソフトは、互換性
を重視していますね。

また、旧 version との互換性を維持するための仕組み
とか、変換ツールとかを提要しています。

山のよう基幹業務の重要なアプリケーションをかかえる
商用 DB では互換性は重要問題です。一部の機能の互換性
の問題のために、やむおえず古い version をしかたなく
使っているというような例も雑誌で見た記憶があります。

PostgreSQL も、商用 DB と同様に使われるようになって
きたので、互換性問題は、十分に検討した上で、本当に
互換性を切捨てなければいけないのか、あるいは、実際
に互換性を損うようなケースはごくわずかで、無視して
もさしつかえない程度なのか、十分に事前検討をして欲
しいと思います。

Version UP のたびに、アプリを書直さなければいけない
という悪名高い某社のようにはなってほしくない。
