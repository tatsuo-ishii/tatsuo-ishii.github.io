お世話になります。
ゼロです。

私は今PostgreSQL7.1.2のECPGを使用していますが、Linuxではコンパイル
できるソースがCygwinではコンパイルできないという事象に悩まされています。
このソースは日本語でコメントを書いているのですが、どうもそれが原因の
ように思えます。

PC上で作成したソースをLinuxにFTPする時はEUCに変換していますが、Cygwin
ではS-JISのまま、コンパイルしています。

PostgreSQLのコンパイルオプションはどちらも、
[Linux]
$ ./configure --prefix=/home/pgsql --enable-locale --enable-multibyte=EUC_JP
$ make
$ make install

[Cygwin]
$ ./configure --prefix=/home/pgsql --enable-locale --enable-multibyte=EUC_JP
$ make
$ make install
としています。

書いてて思ったのですが、Cygwinの「--enable-multibyte=EUC_JP」はECPGまで
影響するのでしょうか？
Cygwinの環境変数でLANGにja_JP.SJISとか設定してもだめでした。
そもそもCygwinではviでファイルを開くと日本語の部分が化けているのですが
こちらがいけないのでしょうか？

※ECPGのパッチ所在をご存知でしたら教えて頂けないでしょうか？
