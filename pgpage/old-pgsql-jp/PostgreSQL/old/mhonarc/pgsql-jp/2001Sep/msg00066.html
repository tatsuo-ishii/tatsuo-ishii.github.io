At 午後 03:57 01/09/07 +0900, you wrote:
> > # それ以前に、どうしてサイズを測りたいのかに興味がありますが。
>
>文字数が多すぎたら警告を出して、
>範囲内に収めてくださいと、エラーを表示するためです。
>
>CREATE TABLE hoge (foo VARCHAR(10));
>INSERT INTO hoge VALUES( '12345678901' );
>SELECT * FROM hoge WHERE foo = '12345678901';
>SELECT * FROM hoge WHERE foo = '1234567890';
>
>上の様に実行すると、
>11 桁の文字列を丸め込んで foo に入れてくれます。
># エラーでません。
>そのため最初の SELECT では何も表示されません。
>
>foo というフィールドが
>passward とかだったらと思うと、
>怖くなるとおもうのですが、
>文字列を INSERT する前に、
>文字数を調べというのはあまり、
>一般的ではないでしょうか？

　もちろんINSERTする前に文字列の長さ等はチェックしておくべきですが、
僕が思うに、そういうのはSQLを生成するアプリケーション側でやる作業
ではないかと。
　さもなくば、CHECK制約を使う手かなあ。

CREATE TABLE hoge (
     foo    varchar(1024) CHECK(CHAR_LENGTH(foo) < 1024)
);
