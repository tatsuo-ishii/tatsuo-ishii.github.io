In article of <[pgsql-jp 2803] Re: How to copy>,
    t-ishii@sra.co.jp (Tatsuo Ishii) wrote: 
> >> 後、ご存知とは思いますが、日本語がデータに混じっている場合は EUC に
> >> しておく必要がありますし、改行コードも Unix (\n)でないといけません。
> >UNIX って CR だけですよね？  \n って プラットフォームによって
> >意味が変わっていたと思うのですが。
> CR って、Octal で 015, Hex で 0x0d のコードのことですよね。これを改行に
> 使っている Unix って見たことないですが。また、同じ Unix プラットフォームで
>  \n の意味が違ってくるというのも分かりません。どういうことでしょうか？
すいません。UNIX での 改行は CR ではなくて LF でした。
それで 『\n がプラットフォームによって変わる』というのは、DOSの場合
\n → CR+LF になります。 UNIX というグループで分けた場合はすべて
\n → LF    となるということです。
＃ なんか余計に混乱させたようで 申し訳ありませんです。

> >> >>copyコマンドはスーパーユーザーしか使えないのですか。
> >> >そんなことはないです。
> >> たぶんここで言っている「スーパーユーザ」というのは、PostgreSQL の「スー
> >> パーユーザ」のことだと思うのですが、確かに直接ファイルからの copy は 
> >> PostgreSQL のスーパーユーザしかできません。普通のユーザの場合は、標準
> >> 入力からの copy が使えます。
> >あれ？ ここでいう PostgreSQL の「スーパーユーザ」というのは
> >    postmaster を起動しているユーザ
> >    createuser で登録されて かつ user 登録権限を与えられているユーザ
> >のどちらでしょうか？
> 両方です。postmaster を起動できるユーザ = initdb を起動したユーザですから、
> これは自動的に PostgreSQL のスーパユーザになります。また、おっしゃるとおり
> user 登録権限を持っている PostgreSQL ユーザもまた PostgreSQL のスーパユーザ
> です。権限的にはどちらもほとんど区別がありません。
理解しました。

『PostgreSQL のスーパユーザ』しか 実行できないコマンドリストって
