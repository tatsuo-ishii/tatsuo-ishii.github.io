>そのあと自分でも組み込みの関数を作りたくて、見よう見まねで関数を
>作ってみました。＃ C についてはホントに初心者なんですが。
>
>システムのユーザーID を与えると /etc/passwdのコメント部分を抜き
>出して（pwd.h の getpwuid を使ってます） text で返すという関数な
>のですが、うまく動きません。
>存在しない UID のときには Null を返したいのですが、それがうまく
>動いていないようなんです。

Null を SQL で言う NULL であると解釈します。考えてみると、C 関数で普通
は text、何かまずい場合には SQL の NULL と解釈される値を返す、というの
が可能かどうか考えてませんでした。で、本家の ML に質問してみたのですが、
どうも現状では出来ないようです。

というわけで、今のところ、存在しない UID のときには "" とか、"Invalid
user" とか、とにかく何らかの文字列を返してあげるしかないようです。

他の方法としては、
	typedef struct {
		bool is_uid_exist;
		text *gecos;
	} Gecos;

のように structure を定義し、新しいデータ型を作る方法もありますが、た
かが NULL を扱うためにしてはちょっと面倒すぎですね。

やはり、PostgreSQL が修正されるのを期待したほうがいいかな。
