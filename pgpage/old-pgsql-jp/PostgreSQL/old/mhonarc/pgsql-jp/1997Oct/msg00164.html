>　こんにちは中村＠日大理工です。
>
>　おかげさまで、makeindexで単語の抽出までは出来ました。
>
>　肝心のpsqlで登録なのですが、wordindex.dataの方は問題なく
>完了した様なのですが、header.dataコピー時に、以下の様な
>Warningが表示されて、一個も登録されません。
>
>------------------------------------------------------------------------
>QUERY:  
>drop table header;
>QUERY:  
>create table header (hfrom text, subject text, date abstime, fname text);
>QUERY: copy header from '/somewhere/header.data';
>WARN:Bad abstime external representation ''

このように、Bad abstime external representation '' と '' が表示される
(普通は悪い日付データの文字列が表示される)ところを見ると、makeindex の
バグにぶちあたった可能性があります。今まで分かっているところでは、
Subject: にタブが含まれていると、このような現象が起きます。
http://www.sra.co.jp/people/t-ishii/sd/sd9710.html を御覧の上、最新の 
makeindex でデータを作り直し、もう一度最初から試してみてもらえませんか？
