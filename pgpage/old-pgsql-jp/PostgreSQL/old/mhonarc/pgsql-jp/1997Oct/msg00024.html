佐々木＠ＳＭＡＴです。

PosgreSQLで以下の様なテーブルがあって、不一致なレコードを抽出したいのです
が良い方法はないでしょうか？

CREATE TABLE aaa ( adat char4);
CREATE TABLE bbb ( bdat char4);

INSERT INTO aaa VALUES( '0001');
INSERT INTO aaa VALUES( '0002');
INSERT INTO aaa VALUES( '0003');
INSERT INTO aaa VALUES( '0004');
INSERT INTO bbb VALUES( '0001');
INSERT INTO bbb VALUES( '0003');

基本的にbbbにあってaaaにないデータはないという条件です。
現在以下の様な方法でやっているのですが、他にないでしょうか？

SELECT adat INTO TABLE tmp1 FROM aaa;
DELETE FROM tmp1 WHERE adat=bbb.adat;
