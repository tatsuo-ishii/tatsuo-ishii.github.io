PostgreSQL v6.2 has increased support for the SQL/92 language standard.
Included in that standard is a list of "reserved words" which are
specifically disallowed as identifiers (e.g. column names).

Check src/tools/SQL_keywords for a list of these, and _stay away_ from
them! Other RDBMSes probably disallow them, and future versions of Postgres
will likely disallow more and more keywords from this list in an effort to
be SQL/92-compliant :(

                                                                   - Tom

btw, there is one keyword which disallowed in v6.2 but which should not be:
"time" is on the SQL/92 non-reserved-word list (but "zone" _is_ reserved;
go figure). I can post patches if anyone is interested in re-enabling this
keyword, and it will be fixed in the next release.
