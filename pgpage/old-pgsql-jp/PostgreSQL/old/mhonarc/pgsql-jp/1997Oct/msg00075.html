ちょっと PostgreSQL からは離れますが SQL 標準がからんでくるので少し．

# う．仕事逃避モードに入ってしまった ... 

PostgreSQL は今のところは通さないようですが，SQL での「必殺技」はやはり
"date" やら "time" でしょう．delimited identifier というのが正式名称
のようです:

	A delimited identifer is any string of not more than 128
	characters (not necessarily SQL language characters) enclosed
	in double quotes; for example, SELECT is a not valid regular
	identifier, because it is identical to a reserved key word,
	but "SELECT" is a valid delimited identifier.  Within such
	a string, the double quote character itself is represented
	by a pair of immedicately adjacent double quotes (such a pair
	of double quotes counting, of course, as a single character
	with respect to the 128-character limit).

	(C. J. Date, {\em A Guide to The SQL Standard}, {\em 4th ed.},
	 Addision-Wesley, 1996, p. 33.)

ちなみに，某社の SQL server では 「日本語 identifier は delimited であ
れば使える」という話ではありましたが，なんかでこけたような記憶があります．
標準では許していても内部の処理がきちんと対応していなかったのでしょうか．
更に別な SQL server では locale 指定をすれば EUC identifer も OK でした．

どちらがいいかは難しいですが，漢字コードを含む場合は localization の
範疇できちんと処理してくれたほうが利用する側としては使い勝手は
いいような気がします．
