初めて投稿いたしますが、よろしくお願いします。

PC-AT互換機を2台用意して、1台目のPCには PostgreSQL だけを導入、2台目の
PC には Apache + Perl で導入して、2台目のPCから 1台目のPCの
PostgreSQLに Perlで検索を行い、結果を HTMLで表示することを考えています。

現在想定している環境は次のようになります。
1台目のPC
  FreeBSD 3.2 Release
  PostgreSQL 6.5.1

2台目のPC
  FreeBSD 2.2-RELENG
  Apache 1.2.4
  Perl 5.003

1台目の環境はすでにできあがっていて、2台目のセットアップについて次のよ
うな方法を考えています。

・PostgreSQL 6.5.1(または 6.5.2)のソースを展開
・src/interfaces/perl5 のディレクトリで下記コマンドを実行
  (このPCにはPostgreSQLがありませんので、環境変数 POSTGRES_HOMEは設定
  しません。)
  perl Makefile.PL
  make
  make test
  (test が OK なら、rootになって)
  make install

Makefile.PL の中を見てみたところ、環境変数 POSTGRES_HOMEが設定されてい
ない場合 ...
  src/interfaces/libpq
  src/include
の 2つのディレクトリの存在をチェックしているようですのが、これでだけで
いけそうな気がするのですが、どなたかご存知の方いらっしゃいましたらお教
え願えますでしょうか。

一度試してみたらいいとは思うのですが、2台目のPCがすでに稼動しているた
めに、失敗して環境をこわしたり、停止させる訳にはいかないという状況です。
