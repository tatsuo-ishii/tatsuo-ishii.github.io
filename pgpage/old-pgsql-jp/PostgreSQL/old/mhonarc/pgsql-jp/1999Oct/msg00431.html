私は上記のようなことをしなければならない状況にないので
わからないですが、そもそもpostmaster を起動した時に指定した
ディレクトリとcreatedb するディレクトリを別にしてうまく整合性が
とれるもんなんでしょうか。そういうことをしたい時は、異なる
ポート番号を使って別個のpostmaster を起動したほうがいい
のではないかとも思いますが、どうなんでしょう。 > 識者
少なくともpostmaster とcreatedb は-p オプションでポート番号を
指定できます。

postgres@bash-2.02$ man postmaster

PostgreSQL                                       POSTMASTER(UNIX)

NAME
     postmaster - run the Postgres postmaster

SYNOPSIS
     postmaster [-B n_buffers] [-D data_dir] [-N n_backends]
          [-S]   [-a   system]    [-b    backend_pathname]    [-d
          [debug_level]] [-i] [-n] [-o backend_options] [-p port]
          [-s]

postgres@bash-2.02$ man createdb

PostgreSQL                                         CREATEDB(UNIX)

NAME
     createdb - create a database

SYNOPSIS
     createdb [-a system] [-h host] [-p port] [-u] [-D  location]
     [dbname]
