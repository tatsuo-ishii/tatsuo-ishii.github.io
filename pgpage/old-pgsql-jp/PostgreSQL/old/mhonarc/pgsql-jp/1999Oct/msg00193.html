ちょっと、変なデータベースを作ることになりまして
お知恵を拝借した区思います。

以下のようなテーブル構造で高速に select を行うにはどのように
するのがいいのでしょうか？

create table main (
    f1  int2,
    f2  int2,
    f3  int2,
    ...
    f22 int2,
    dt  varchar(30)
);

create tale t1 (
    f1  int2,
    d   varchar(20)
);

create table t2 (
    f2  int2,
    d   varchar(20)
);

    ...

create table t22 (
    f22 int2,
    d   varchar(20)
);

 f?? に対するprimary key は全て定義しています。
上記テーブルで
select main.dt, t1.d, t2.d, t3.d, ... , t22.d 
     from main, t1, t2, t3, ..., t22 
     where main.f1 = t1.f1 
     and main.f2 = t2.f2 and ... and main.f22 = t2.f22;

とやったとき
main が数件でも暫く返ってきません。
MSAccess では、このようなリレーションでも一応それなりの速度で
表示してくれるのです（もちろん left join になっています）
このような場合に高速（それなりに）に表示する方法はないのでしょうか？
