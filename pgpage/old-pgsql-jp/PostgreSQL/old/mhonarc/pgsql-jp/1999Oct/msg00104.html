のぼり＠ソニーです．よろしくお願いします

こちらでapache-1.3.6、postgres-6.5.2、php-3.0.12jpでwebからデータベース
サーバーにアクセスするシステムを組んでます。
全て同じhost(solaris-2.6)で動いているのですが認証関係で悩んでいます。

・やりたいこと
1. web からアクセスする時はユーザー名とパスワードが必要。
2. psql コマンドはunixのユーザーを信用する。

・やった事
1.pg_hba.confに
local        all                                         trust
host         all         127.0.0.1     255.255.255.255   password passwd
host         all         192.16.1.0   255.255.255.0   password passwd
と書いてdomain socketはtrust、ネットワーク経由ではpassword認証をする。
passwdファイルはNISで持って来たunixパスワードファイル。
(192.16.1.0はローカルネットワークだと思ってください)

2.php3の中で
pg_connect("dbname=template1 port=5432 user=$PHP_AUTH_USER password=$PHP_AUTH_PW host=foo.baa.co.jp");

とやってapacheからはネットワーク経由でpostgresにアクセス。


これでpsqlはパスワードなしで、apacheからはパスワードを求められる、と思っ
たのですが、どうもapacheからユーザー名だけを与えるとパスワード無しでもデー
タベースが開けてしまうようです。


やりたいことを満たすような設定は一体どうすればよいのでしょうか。
よろしくお願いします。
