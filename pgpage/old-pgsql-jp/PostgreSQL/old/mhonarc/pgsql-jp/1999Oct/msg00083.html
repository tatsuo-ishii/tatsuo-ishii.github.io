> HP-UX等だと、複数ボリュームをワンパーティションと見せる論理ボリュームなんちゃら(LVM)
> だったと思いますが、あるので、あまり気にしない。
> 
> FreeBSD/Linux等ならば、下記の手順でなんとかならないでしょうか?
>  1. createdb でdatabaseを作成
>  2. mvで他の物理パーティションに移動
>  3. 2.で移動したものに対しリンクを張る
>       % ln /foo/var/database_name $PGDATA/base/database_name

base/DB名の下の一部のファイルを別スピンドルの HDD
に mv して、シンボリックリンクを張って動かしてみた
ことはあります。(pgbench で試してみました)。この時
は、動きましたが、この方法は、お勧めしません。
# 別 Partition だから、Hard Link でなく Symbolic
# Link  ln -s  ですね。

TABLE サイズが 1 GB を超えて、１表複数ファイルになる
場合や、DROP/CREATE した場合など、問題が多くて危険だ
と思います。

素直に、大容量の RAID を入れるなど、まっとうな方法で
対処された方がいいと思います。

# 今、１ドライブ 10GB 以上の安価な HDD が出てますが、
# それでも足りないというと、よほど大きなデーターな
# んでしょうか？。
