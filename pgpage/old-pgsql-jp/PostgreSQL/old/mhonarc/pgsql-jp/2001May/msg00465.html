本題からそれますし、余計なことで恐縮ですが茶々を少々。

私も日々研鑚を続けている立場です。経験を元に言わせて頂くと、Postgres 自
体の理解度も必要ですが、問題が発生するたいていの場合、OS に対する理解度
が低いために躓いたケースが多かったように思います。

Linux 等の Unix 系 OS の場合、コマンドラインによる作業も多くなりますし、何か
問
題が発生したとき、例えば Windows でのそれのように直感的な解決を行うのは（初
心者には）困難です。

まずはじっくりと OS を理解することが必要です。
具体的にはシェルスクリプトを書いてみたり、Kernel の再構築を行ってみたり、ア
プリ
ケーションをソースからインストールしてみたり。失敗しても何度も何度もチャレン
ジを
繰り返すことが大切です。
仮に Postgres のインストールに成功し、起動が上手くいっても、基本的には SQL
が
書けなければ意味がありませんし、Postgres で何をなさりたいのかは分りません
が、
恐らくそれを実現するためにはかなり広範囲にわたる知識とスキルが必要となること
でしょう。
　＃やりたいことがそのレベルでないなら、MsAccess のほうが直感的に操作できま
す

インストールひとつとってみても、シーラカンス本に書いてあることをそのまま実行
す
るだけではいつまでたっても上達しません。
　＃むろん、最初は「書いてあるとおり」に試してみる必要があります

なぜ .bashrc にたくさん書き込まなくてはならないのか。
なぜ起動オプションなるものがあるのか。
もっともっと初歩的なところでは、「 - 」と「−」はどう違うのか。
etc

ひとつずつ、その「なぜ」を解消していくのが最終的な近道だと思いますし、構造を
理解することにもつながってきます。
何のために身に付けようとなさっているのかは文面からは窺い知れませんが、どう
しても急を要するようでしたら、有償のセミナー等の受講をお勧めします。
　＃私も SRA さんの Postgres のセミナーを受講しました
