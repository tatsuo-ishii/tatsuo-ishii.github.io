こんにちは。Narushima Hironoriです。


eruby + mod_ruby から PostgreSQL を使おうとしていますが、うまく動作し
ません。eruby のスクリプトに

require 'postgres' 

と書いた eruby スクリプトにWWWブラウザからアクセスすると
$ w3m http://localhost/test_sql.rhtml

(eval): Shared object "libpq.so.2" not found
- /usr/local/lib/ruby/1.6/i386-freebsdelf3.5.1/postgres.so (LoadError)

というようなエラーが出て動作しません。


設定したことしては

[httpd.conf] (Apache)
User nobody
Group nobody

があることを確認して postgres スーパーユーザが

$ createuser nobody

と、しています。
また、postmaster が psql から

test=# grant select on cd to nobody;
CHANGE

と設定しました。

データベースにアクセスできる権限をもってるユーザがシェルから

$ eruby test_sql.rhtml

とすると、問題なくアクセスできます。
mod_ruby と eruby は正常に動作しています。

nobody の権限に関する設定が間違ってると見当をつけているのですが、どう
設定したらよいのか分かりません。

うまく動かす設定に思い当たることがあれば教えてください。
