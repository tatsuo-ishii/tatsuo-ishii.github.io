kazuto@sainet.or.jpさんは書きました。
kazuto> 私も2.2.5で同じ現象になりました。
kazuto> gcc version 2.7.2.1で駄目だったので 2.95.3にしましたが
kazuto> 状況に変化はありませんでした。
kazuto> postmaster.c: In function `PostmasterMain':
kazuto> postmaster.c:406: `optreset' undeclared (first use in this function)
kazuto> postmaster.c:406: (Each undeclared identifier is reported only once
kazuto> postmaster.c:406: for each function it appears in.)
ちょっと手元の機械を見てみたんですが, FreeBSD-2.2.8 の
/usr/include/stdlib.h には optreset の宣言がされていません. ただ,
/usr/lib/libc.a の中には optreset が定義されているようなので, 本
当にこのメッセージ通り「optreset が宣言されていない」だけであれば
ソースのどこかに
extern int optreset;
を入れればコンパイルできるかもしれません.

もっとも, gcc-2.95.3 にして
kazuto> Undefined entry symbol `xport-dynamic'
kazuto> ld: Spurious undefined symbols: # undefined symbols 1, reported 0
kazuto> collect2: ld returned 1 exit status
というのはちょっとわかりません.
