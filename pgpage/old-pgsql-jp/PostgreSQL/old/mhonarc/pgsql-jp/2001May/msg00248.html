神崎です。

下記のようなテーブルが存在します。

create table custtbl(
id int primary key,
name varchar(50),
tel varchar(20),
);

idはシーケンスで連番を振っています。

そしてtelにindexを下記のようにして設定しました。

create index cust_tel_idx on custtbl(tel);

しかしアクセス過多のためか同じデータが複数インサートされてしまいます。
もちろんidのキーは重複しないで振られるのですがその他のカラムのデータが
全く同じものが3つも4つも入る場合があるのです。
毎回入るわけではありません。
アクセスの非常に多い時にこの現象が現れます。
telが同じものが複数入るとシステム的に非常にまずいため大変困っています。
恐らく送信者が何度も送信ボタンを押したために起こっている現象だと
推測しています。
telは送信ボタンを押下された時にチェックして重複しないようにしており、
正常にチェックがかかっていることは確認しています。しかし現実は重複して
インサートされてしまいます。
そこで下記のようにuniqueオプションをつけてindexを作ってあげれば
絶対に重複しないのでしょうか？

create unique index cust_tel_idx on custtbl(tel);

もし内部的に何度も同じインサートしようとしたとき動作は
どのようになるのでしょうか？

さらにuniqueにしたら正常になるとすると、現在設定しているindexはどのよう
に削除したらよいのでしょうか？
create unique index cust_tel_idx on custtbl(tel);
を実行するとこちらが優先されて使用されるのでしょうか？
過去ログのindexの項目を見たのですがいまいち該当する
回答がありませんでした。
postgresはcobalt raq4のデフォルトのものを使っており、インターフェイスは
PHP4で開発しております。

何卒、ご教授お願いいたします。
