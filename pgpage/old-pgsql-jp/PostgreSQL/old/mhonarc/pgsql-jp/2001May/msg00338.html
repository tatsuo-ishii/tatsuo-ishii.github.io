もともと私のエスケープのコードは松尾さんのメールの中でも使われている
DBD::Pgのquoteメソッドから引っ張ってきています。

<引用>
    sub quote {
        my ($dbh, $str, $data_type) = @_;

        return "NULL" unless defined $str;

        unless ($data_type) {
            $str =~ s/'/''/g;           # ISO SQL2
            # In addition to the DBI method it doubles also the
            # backslash, because PostgreSQL treats a backslash as an
            # escape character.
            $str =~ s/\\/\\\\/g;
            return "'$str'";
        }
       （この後、$data_typeがわかっている時の処理へ）
</引用>
中のコメントにかかれている通り「PostgreSQLはバックスラッシュをエスケープ
文字として使うので」というつもりです。
