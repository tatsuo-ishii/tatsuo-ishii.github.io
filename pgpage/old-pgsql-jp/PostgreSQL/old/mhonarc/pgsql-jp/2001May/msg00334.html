重松と申します。

「プログラマのための SQL」という書籍を読んで勉強しています。そのなかの、IN
述語の最適化 (p.136) について教えてください。

ここでは切符を買う例として、

SELECT *
 FROM Personnel
WHERE city IN (SELECT city FROM SportsTeams)
 AND state IN (SELECT state FROM SprotsTeams);

を一意である組み合わせ (city,state) を利用して

SELECT *
 FROM Personnel
WHERE city || state IN (SELECT city || state FROM SportsTeams);

のように最適化する方法が紹介されています。

実際に試すと、動作しますが、なぜか Visual Studio では || を | | という風に書
き換えてしまって、二度目に実行するときに不便なのです。これは、何か間違えてい
るのでしょうか?

また、あわせて SQL-92 の例として、

SELECT *
 FROM Personnel
WHERE (city,state) IN (SELECT (city,state) FROM SportsTeams);

が示されているのですが、これはエラーになります。

しかし、もっと単純な構文、

SELECT * FORM SportsTeams
WHERE (city,state) = ('LA','CA');

だと動作します。

PostgreSQL ではどこまでサポートされているのか知りたいのですが、どのように調
べればよいですか? シーラカンス本 (1, 2 版ともに) はもっております。

よろしくお願いいたします。
