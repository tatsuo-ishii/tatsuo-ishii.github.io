石井さんのおっしゃるように、作成したqueryをVineLinux上にもっていきそこから 
psqlで送り込むことは問題なく出来ることが分かりました。Windowsのlibpqの問題と 
いうことなのでしょうか？そうなるとちょっと私には解決出来ないかなあと思ってお

取り急ぎ御礼まで。


>石井です．
>
>> libpqを使って配列データ（整数、実数）をデータベースにINSERTしたいと思ってい
>> ます。環境は
>> ＜PostgreSQL側＞
>> Vine Linux2.0 on Powerbook2400c/180
>> PostgreSQL7.1 --enable-multibyte
>>
>> ＜クライアント側＞
>> Windows98 on Dynabook
>> libpqを使ったMathLinkプログラム(Mathematica外部プログラム）からの配列送信
>>
>> ＜疑問点＞
>> res = PQexec(PGconn *conn, const char *query);
>>
>> という指定のqueryには実際には、INSERTしたい場合には文字列(char *)として送り
>> 込みたい配列データを記述していますが、この配列が大きくなっていくと
>> PQexec()の返り値はNULL(0)になってしまってます。queryが実行されていない様子で
>> す。よく分かっていないところが多いのですが、データとして受け渡すquery文字列
>> が32kを越えてしまうといけないといった、決まりがあるのでしょうか。返り値が
>> NULLの場合には、PGRES_FATAL_ERRORの扱いになるとドキュメントにはありますが
>> PQerrorMessage(res)からは何のエラーメッセージも返ってきておりません。
>> PostgreSQLのソースの中でこのqueryの上限が規定されているかどうかを探しきれま
>> せんでした。もしquery文字列長さに上限があるのならばそれを変更するためにはど
>> うすればよいのでしょうか？少なくとも数千から数万の実数データを送り込みたいと
>> 思っています。
>>
>> ちなみに送り込む配列が小さい場合には、問題なくデータが送られています。徐々に
>> 増やすうちにPQexec()の返り値がNULLになります。
>
>PostgreSQL 7.0からqueryの長さに制限がなくなりました．ですからこの現象
>はちょっと解せないのですが，もしかしてWindowsクライアントの問題という
>ことはないでしょうか?同じ問い合わせをVine Linux上でpsqlから実行すると
>動くということはありませんか？
