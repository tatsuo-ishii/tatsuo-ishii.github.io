中島さんの考えた設計とは全く違うということです。
「ToDo リストでメール機能付き」というキーワードから思いつきで書いただけです。

例えばテーブルにすると
create table todo_list
(
	do_text		text, // やることの内容
	limit_date	date, // やることの日限
	mail_flag	int,  // メールを出すかどうかのフラグ
	repeat_kind     int,  // リピートなし、日限が過ぎたら毎日、毎週、毎月、毎年などの区別
	day_of_week     int,  // 毎週の時の曜日
        day_of_month    int,  // 毎月の時の日にち（例えば毎月１５日とか）
        month_of_year   int,  // 毎年の場合の月数、day_of_month と組み合わせる
        time_of_mail	time, // メールを送信する時間
)

の様な単純（？）なテーブルを作り毎日数回チェックすればメールを出すものを抽出できると
思います。
＃200LX では、毎月第２週の水曜日というような指定も出きるが、ここでは省略。

「このメールを送るのは１０回目ですよ」とか「ずいぶん期限が過ぎています」
とかですか？このようなメッセージはメールを受け取った人が判断することであって
システム側から出すとよけいなお世話のように思います。

勘違いされては困るのですが、HP200LXにメールを出す機能まではありません。
ToDo にリピートをセットする方法があるだけです。リピートにしておくと画面上に通知が表示されます。

それでは、ご参考まで。
