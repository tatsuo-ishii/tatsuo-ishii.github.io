接続不可の場合（pingが通らない場合）の異常系処理にて
下記のようなPerlスクリプトでexitさせたいのですが、
exitする場合とexitしない場合があるのですが、
終了処理で特別なことが必要なのでしょうか？

＃正常系（ping OK, postmaster -i起動)ではちゃんと
＃接続できます。

---スクリプトここから---
#!/usr/bin/perl -w

#
#       ＤＢ接続
#
use Pg;

$conn = Pg::connectdb("dbname=testdb user=testuser host=testsvr");
if (PGRES_CONNECTION_OK ne $conn->status) {
        printf("ＤＢ接続エラー:%s",$conn->errorMessage);
        exit(1);
}
：
：
---スクリプトここまで---


実際にスクリプトを実行すると、

＞ＤＢ接続エラー:PQconnectPoll() -- connect() failed: No route to host
＞      Is the postmaster running (with -i) at 'testsvr'
＞      and accepting connections on TCP/IP port 5432?

というメッセージを表示したのち、Ctrl+Cでの強制終了以外では
すぐにはコマンドプロンプトには返ってきません。

環境は、
Linux 2.2.16-3 + PostgreSQL7.1.1
で行っています。
