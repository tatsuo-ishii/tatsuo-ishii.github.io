私の環境は Linux ですが、No route to host は
/usr/include/asm/errno.h によると、

#define EHOSTUNREACH    113 /* No route to host */

なので、Host Unreachable な ICMP メッセージが帰ってくるように
してやればいいのだと思います。で、会社から実在するプライベート
ネットワーク上の実在しないホストに ping をかけると出ました。

hotta@star:~/temp$ ping 192.168.6.187（存在しないホスト）
PING 192.168.6.187 (192.168.6.187): 56 data bytes
（だんまり）

[別 Windows の tcpdump]
root@star:~# tcpdump -n host star
Kernel filter, protocol ALL, datagram packet socket
tcpdump: listening on all devices
08:58:15.014571 eth0 > 192.168.0.220 > 192.168.6.187: 
  icmp: echo request
08:58:15.940216 eth0 < 192.168.6.248 > 192.168.0.220: 
  icmp: host 192.168.6.187 unreachable

192.168.6.248 は、相手側ネットワークの入り口にある対向ルータ
です。ご参考になりますか?
