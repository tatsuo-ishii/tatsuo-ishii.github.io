[pgsql-jp 21001] SJIS で作成したDBでpsql \dtが失敗

  • From: Moritaka Ogasawara <oga@psc.hitachi-sk.co.jp>
  • Date: Tue, 08 May 2001 10:58:45 +0900
小笠原です。

PostgreSQL 7.0.2をLinux 2.2.15で
./configure --with-template=linux_i386 --enable-multibyte
後にmakeして使用していますが、
SJISでcreatedbした場合、psqlの\dtが失敗します。
# selectなど通常のSQL実行では問題ありませんが、テーブル一覧が見れないので
# 不便してます。
この件について何か情報があればご教示戴きたくよろしくお願いします。


hotdoggin:/home/oga <1> createdb -E SJIS xxx
CREATE DATABASE
hotdoggin:/home/oga <2> psql xxx
  :
xxx=# \dt
The connection to the server was lost. Attempting reset: Failed.  <=コネクションが切れた
!#

hotdoggin:/home/oga <3> dropdb xxx

hotdoggin:/home/oga <4> createdb -E EUC_JP xxx
CREATE DATABASE
hotdoggin:/home/oga <5> psql xxx
  :
xxx=# \dt
No relations found.  <= EUC_JPでは特に問題なし。
xxx=#

hotdoggin:/home/oga <6> env |grep PG
PGLIB=/usr/local/pgsql/lib
PGDATA=/var/lib/pgsql/data
PGCLIENTENCODING=SJIS
