自己レスですが、解決方法が見つかりましたので報告いたします。

jdbc1のStatement.javaにバグがあります。

（291行目）
if (result != null) && ((org.postgresql.ResultSet)result.reallyResultSet())
（正）
if (result != null && ((org.postgresql.ResultSet) result).reallyResultSet())


やっとコンパイルは通りましたが、できあがったpostgresql.jarは
org.postgresql.core.ConnectionHook内でArrayListを使っているため、
動作環境としてはJava2必須でした。

希望のjdk1.1.8環境で動作するPostgreSQL 7.1のjdbcドライバを得る
ことができず残念ですが、一応報告まで。
