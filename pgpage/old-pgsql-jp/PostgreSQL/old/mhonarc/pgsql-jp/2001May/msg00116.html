はじめましてgackuと申します。
apache php postgreSQLでウェブアプリケーションを作ろうとしています。

やりたいのは、ToDo リストというもので、やるべきことを確認する
為に使います。
機能としては、todo を登録するとリストに登録され、ある経過時間
になるとEMAILを送ります。
其の経過時間というものが、複数あるとします。つまり、登録した
内容を何度かおくるようにしたいのです。
このときのテーブルの作り方で悩んでいます。
以下のようにふたつのテーブルにわけています。

--todoテーブル--
todo_id  serial(not null)  (主キー)
todo_name varchar(255)(not null) 登録題名
memo  varchar(500)(not null) 登録単語の意味
priority_code int (not null)   頻度のコード（外部キー)
end_time timestamp    期限
ended  int       完了フラグ
changed  timestamp    更新日時
uname  text(not null)   登録したユーザ名
-追加分-
user_id  varchar(32) (not null) ユーザID（外部キー)

--頻度 priorityテーブル--
priority_code int (not null)   頻度のコード
priority_name text     頻度の名前
priority_interval int2     頻度の値(何日後にメール送信日がくるか)

このようにしたとすると、頻度の価が１つしか登録できないことになります。

priority_interval  = now() - changed のようにして
現在と登録日との曜日の差を抽出して、それが
予めある頻度の価と一致したらメールを送信するという
ものです。
頻度の価が(1 , 5 , 10)とするとその頻度に割り当てられた
ToDoは登録した日から一日後、五日後、十日後に送信
されるようにしたいのです。

恐らく、設計思想からの問題と思われますが、なにぶん
プログラミング初心者の為に的外れな質問かと思いますが、
どうかご教授ください。
