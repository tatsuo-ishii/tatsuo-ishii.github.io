いつも読ませて頂いております。

Solaris2.6のv6.4.2をv7.1.3にバージョンアップしたのですが、
v6.4.2の時作成した、Ｃ言語のプログラム（libpqを使用）をコンパイルすると
----
	ild: (未定義のシンボル) inet_aton -- /usr/local/pgsql/lib/libpq.a(fe-connect.o)
	の text セグメントから参照
----
とリンクエラーになります。

configure時の表示では
----
	           :
	checking for inet_aton... yes
	           :
----
と表示されています。
また、レグレションテストも全て正常でした。
今回インストールしたWSには、inet_aton()は無いと思うのですが。。。

configure後の、src以下のファイルをいくつか見た結果です。

	/usr/local/src/postgresql-7.1.3/src/Makefile.global
		INET_ATON =

	/usr/local/src/postgresql-7.1.3/src/include/config.h
		/* Set to 1 if you have inet_aton() */
		#define HAVE_INET_ATON 1

解決方法をご存知の方は、教えて下さい。
よろしくお願いします。
