以前PostgreSQL6.4 を Redhat5.2J上で少しの間使っていまして、その折りは
このメーリングリストに御世話になりました。
今度はWindows2000上でPostgreSQL7.1.3です。
先日、Cygwin 1.3.2をダウンロードしてインストールして試用しています。
PostgreSQLの7.1.3が組み込まれているようなので、使ってみたいと思って
日本語ドキュメントを参考に試しているのですが、うまくいきません。

PostgreSQL 7.1.2 ドキュメント(日本語版 0.6)の
>Tip: もし Windows NT/2000 を使っているならば、Cygwin ツールキットを先に
>インストールすれば PostgreSQL の全てを"Unix 方式で"使うことができます。
>その場合はここを見て下さいChapter 1。
-------------------------------------------------------------------------
/usr/local/pgsql/bin/initdb -D /usr/local/pgsql/data
/usr/local/pgsql/bin/postmaster -D /usr/local/pgsql/data >logfile 2>&1 &
/usr/local/pgsql/bin/createdb test
/usr/local/pgsql/bin/psql test
-------------------------------------------------------------------------
ということで、これを参考にしてコマンドを実行するのですが、

-------------------------------------------------------------------------
[maxima@lavie ~] $ mkdir postgre
[maxima@lavie ~] $ ls
./  ../  .bash_history  .bashrc  d/  logdata  logfile  postgre/
[maxima@lavie ~] $ initdb -D /home/maxima/postgre
This database system will be initialized with username "maxima".
This user will own all the data files and must also own the server process.

Fixing permissions on existing directory /home/maxima/postgre
Creating directory /home/maxima/postgre/base
Creating directory /home/maxima/postgre/global
Creating directory /home/maxima/postgre/pg_xlog
Creating template1 database in /home/maxima/postgre/base/1
-------------------------------------------------------------------------
ここで、止まってしまいます。
この状態で、かなりCPUに負荷がかかっているようで、他のアプリの動作やキー
入力の反応は非常に重いです。Cygwinの窓ではカーソルは自由に動き文字も入力
可能なのですが何も反応しません。
やむなくタスクマネージャで強制終了することになります。
何が悪いのでしょうか？

ちなみに環境は
OS : Windos2000 Porfessional (SP2)
CPU: celeron433，メモリは128MBです。
