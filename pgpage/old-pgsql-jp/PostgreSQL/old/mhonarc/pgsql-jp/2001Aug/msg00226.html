> UPDATE [ ONLY ] table SET col = expression [, ...]
>     [ FROM fromlist ]
>     [ WHERE condition ]
>   
> だそうです。この FROM 句を使えばよいと思います。

そういう話ではなくて、UPDATE で更新対象テーブルを複数指定しようと
しているのがそもそもおかしいということなのでは？

FROM fromlist は WHERE 句で指定する条件に更新対象テーブル以外の
ものも指定できるという PostgreSQL の拡張になります。

元記事の人の質問に対していえば、ひとつの SQL で処理仕様とせずに
アプリケーション的な処置を行う事によって処理すればよいのでは
ないでしょうか。

トリガーとストアードファインクションを使うようにしてもいいでしょうが、
いずれにしろアプリケーション的な処置は必要になるでしょう。
