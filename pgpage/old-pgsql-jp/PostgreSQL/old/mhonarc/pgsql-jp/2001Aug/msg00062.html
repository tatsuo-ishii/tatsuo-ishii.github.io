土屋と申します。

データベースの処理速度についてテストを行っています。
更新処理のテストを行った際に
UPDATEの処理が異常に遅いのが気になりました。

＜テスト環境＞
　・サーバ：IBM PC-750(CPU：DX4 75MHz,メモリ：24MB)
　・サーバOS：Laser5 Linux 6.4
　・PostgreSQL：postgresql-7.0.2

＜テストDataBase＞
          Table "test"
 Attribute |   Type   | Modifier
-----------+----------+----------
 field1    | char(10) |
 field2    | char(20) |
 field3    | char(40) |
 field4    | smallint |
 field5    | integer  |
 field6    | bigint   |　

＜テスト結果 1000件＞
　INSERT：1件−0.24秒
　　SQL文　"insert into test values('あああ','いいい','ううう',1000,2000,3000");"

　UPDATE：1件−21秒
　※上記の1000件レコードについて
　　SQL文　"update test set field1 = 'えええ' where field5 = 2000;"

　SELECT：1件−20秒
　※上記の1000件レコードについてWHEREで検索した場合
　　SQL文　"select * from test where field5 = 2000;"

　DELETE：1000件−21秒
　　SQL文　"delete from test;"

　DataBase容量は1000件レコードで57MB

同様に10000件テストも行いましたがほぼ同等の秒数となりました。
原因は
　　→テスト機の低スペック(CPU:75MHz,メモリ:24MB)が原因？
　　→データベースの設定の問題？
　　→検索の仕方の問題？
なのでしょうか。
