> シリアル型のフィールドを持ちますテーブルへWeb上から登録
> しようとしますと以下のようなメッセージが出まして登録できません。
> Warning: PostgreSQL query failed: ERROR: jukou_kanri_no_seq.nextval:
>  you don't have permissions to set sequence jukou_kanri_no_seq in
> /usr/local/apache/htdocs/bmdata/jyukou_ins.php on line 17

ご自分でも書かれていますが、permissionがないだけです。serial型の場合、
変数名_seq と言うsequenceができます。これにも権限与えないと×です。

権限を与えちゃいましょう。でも、nobodyアクセスさせるよりも、管理者権限
の無いユーザを作って、そのユーザの持ち物でDB(とテーブルその他)を作った
方が良いかと思います。PHPの変更自体も、pg_connectでDBしか指定しないな
い部分がユーザ/パスワードを指定するだけなんですから。

実際は、

   // 大文字部分は環境にあわせて..
   $conn = pg_connect("dbname=DB user=USER"); // パスワード無し
   $conn = pg_connect("dbname=DB user=USER password=PASSWORD");

ですね。それでは〜
