>/#setenv POSTGRES_INCLUDE /usr/local/pgsql/lib/libpq.so
>/#setenv POSTGRES_LIB /usr/local/pgsql/lib/libpq.so

そもそも、環境変数の設定の書き方違ってません？

＃.bashrcあたりにかいとけばいいと思うんだけど

csh/tcshの書き方なのでbashだと違うけど

setenv PGLIB           /usr/local/pgsql/lib
setenv LD_LIBRARY_PATH /usr/local/pgsql/lib

なので、「置いてある場所」を書くのであって
「置いてあるファイル」を書くんじゃないと思うんですが・


>Pg.h:17: libpq-fe.h: No such file or directory
>make: *** [Pg.o] Error 1

「libpq-fe.h」は「/usr/local/pgsql/include」にあって
環境変数の設定が間違ってるので探せてないだけでしょう

こっちの環境だと

# PostgreSQL
set path = ( $path /usr/local/pgsql/bin )
setenv MANPATH         ${MANPATH}:/usr/local/pgsql/man
setenv PGLIB           /usr/local/pgsql/lib
setenv LD_LIBRARY_PATH /usr/local/pgsql/lib
setenv PGDATA          /usr/local/pgsql/data 
setenv POSTGRES_HOME   /usr/local/pgsql

って感じで「include」は特に指定してない状態で
特に問題起きてないから「/usr/local/pgsql/lib」が
分かってればいいのかな？
