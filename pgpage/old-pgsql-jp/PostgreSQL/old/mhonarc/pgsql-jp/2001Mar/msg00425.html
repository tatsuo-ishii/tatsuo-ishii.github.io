自己レスです。

サーバの環境を良く調べてみたところ Red Hat ディストロビューション
の Postgres が既にインストールされていました。

bash$ rpm -q postgresql
postgresql-6.5.3-6

私が行った対応は、自動起動をしている rc.local 内の記述をフルパス
にしました。（/etc/rc.d/rc.local）

POSTGRESDIR=/usr/local/pgsql
if [ -x $POSTGRESDIR/bin/postmaster -a -d $POSTGRESDIR/data ];then
    rm -f /tmp/.s.PGSQL.5432
    su - postgres -c "/usr/local/pgsql/bin/postmaster -S -i"
    echo -n 'postmaster '
fi

この状態で正常に動作させることができました。

本来は RPM によりインストールされた Postgres を削除すべきなので
しょう。

皆様大変ご迷惑をおかけしました。
