[pgsql-jp 20514] Ver 6.5.3 日本語の検索が正しくできません

  • From: Munenori Sekine <sekine@almedio.co.jp>
  • Date: Wed, 28 Mar 2001 11:10:06 +0900
関根と申します。

日本語の検索が正しくできなくて困っています。

以前の投稿に、正規表現を利用したケースが載っていたのは確認しました。
    [pgsql-jp 14648]
しかし、今回は既存プログラムの変更ということで、検索方法を変更しな
いで対応したいと考えています。

また、日本語のソートが正しくできないケースも確認しました。
    [pgsql-jp 19221]

対応方法をご存知の方がいましたら、ご教授お願い致します。

項目：
  name           varchar
  search_name    varchar

データ：
  name         search_name
  青木　太郎    あおきたろう
  鈴木　大介    すずきだいすけ

SQL：
  select name from emp
   where substring(search_name from 1 for 1) = 'あ'
     and substring(search_name from 1 for 1) = 'お' ;

望む結果：
  name
  ---------
  青木　太郎

問題の結果：
  name
  ---------
  青木　太郎
  鈴木　大介

  「鈴木　大介」までselectされてしまう。

本番環境：
  Solaris
  Postgres 6.5.0

  この本番環境で動作させるために、上記SQLを維持したい。

開発環境：
  Linux RedHat 6.2J
  Postgres 6.5.3
           configureから --enable-locale をコメントアウト
           configure --with-mb=EUC_JP
           initdb -e EUC_JP
