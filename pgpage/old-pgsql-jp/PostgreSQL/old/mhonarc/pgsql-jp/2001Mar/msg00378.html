Subject: [pgsql-jp 20508] postmaster起動について
"鈴木　規正" <n_suzu@abk.nesic.nec.co.jp> wrote:
> postmasterを起動時、pg_ctl -w start等の
> コマンドで起動すると思いますが、
> postmasterの起動＆停止を手動ではなく、シェルを使用して
> OS起動時に行いたいのですが上手くいっておりません。
> (現在　/etc/rc3.d/配下のファイルをまねて作っています
> 　Ｓ〜　ファイル）
> 
> OS起動時に、postmaterを上手く起動させる（リブート時に停止）
> 詳しい設定の仕方(どこの配下にファイルを設定、ファイルの内容
> をどのようにすればよいか、OS起動のどのタイミングで立ち上げれ
> ばよいのか等)

このあたりは導入している OS や、Linux であればさらにディストリ
ビューションにもよりますし、各ソフトウェアをどのようにしてイン
ストールしたか（ソースから、RPM などのパッケージで）でも違って
きますので、質問の際はそのあたりの情報も必要です。RPM などのパ
ッケージで入れた場合は、自動起動の設定などはそのパッケージの制
作者がよきにはからってくれているはずです。

私は Vine Linux 2.0 ですが、apache や postgres をソースから入れ
ている関係上、以下のように書いています。

hotta@star:~$ tail -6 /etc/rc.d/rc.local 
if [ -x /usr/local/pgsql/bin/pg_ctl ]; then
    su postgres -c "/usr/local/pgsql/bin/pg_ctl -D /usr/local/pgsql/data
start"
fi
if [ -x /usr/local/apache/bin/apachectl ]; then
    /usr/local/apache/bin/apachectl start
fi
