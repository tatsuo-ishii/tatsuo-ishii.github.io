「いろいろ」ではなく、正確に記述してください。読む方は情報が
少ないと、その分余計な想像力を働かせなければなりません(--;

> やはりmake allできません
> 
> $ make all
> make -C utils all
> make[1]: Entering directory `/usr/local/src/postgresql-7.0.2/src/utils'
> gcc -I../include -I../backend   -O2 -Wall -Wmissing-prototypes
> -Wmissing-declarations   -c -o version.o version.c
> In file included from /usr/include/errno.h:36,
>                  from version.c:23:
> /usr/include/bits/errno.h:25: linux/errno.h: No such file or directory
> make[1]: *** [version.o] Error 1
> make[1]: Leaving directory `/usr/local/src/postgresql-7.0.2/src/utils'
> make: *** [all] Error 2

/usr/include/bits/errno.h の 25 行目を見ると

# include <linux/errno.h>

となってまして、相対パスの場合は /usr/include から検索されること
になっておりますので、結局 /usr/include/linux/errno.h がないと
いうことになります。

ls -ld /usr/include/linux して、そのディレクトリが存在しますか?

これはカーネル関連のヘッダファイル群ですので、今度は山中さんのご
指摘通り、カーネルヘッダをインストールしなければならないのでしょ
う。こちらの環境ではこうなります。

hotta@star:~$ rpm -qf /usr/include/linux/errno.h 
kernel-headers-2.2.17-0vl2

kernel-headers パッケージが入っていなければ、

  rpm -Uvh /どこかの/kernel-headers-XXX

でインストールしましょう。
