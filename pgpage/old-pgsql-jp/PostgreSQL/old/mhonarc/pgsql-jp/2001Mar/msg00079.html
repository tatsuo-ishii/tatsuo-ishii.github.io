こんにちは。

　以下のような現象に遭遇しており、中々原因が掴めないで悩んでおります。何か情
報がないかと思い投稿いたします。

現象:
　リンクしたどのテーブルを開くにも異常に時間がかかる(レコード数に比例してい
る模様)
　※144 record のテーブルを開くのに 69 秒
　※577 record のテーブルを開くまで 112 秒、総レコード数が表示され編集可能に
なるまで更に 102 秒

環境の変更:
　WinNT + Access97 の組み合わせから Win2000 + Access2000 に変更(PostgreSQL
のテーブルを ODBC 経由でリンク)
　DB ホスト側は変更無し

環境:
　OS: ホスト: Linux 2.2.16 / クライアント: Windows2000
　ODBCドライバ: 6.50.00.00
　DB: PostgreSQL 6.5.3 on i686-pc-linux-gnu, compiled by gcc 2.95.2

判っていること:
　1.DB ホスト側、クライアント側共に CPU 負荷は殆ど無し(Access が 平均 4〜5%
程度)
　2.現象が発生している間、ディスクへの断続的なアクセス音(ときどき止む)が発生
　3.2.よりクライアント側のスワップアウトを疑うも、実メモリ 256MB のうち約
100MB が空いたまま(タスクマネージャで確認)

その他:
　ODBC の CommonLog を on にして採ったログを添付します(一部隠蔽)。
　先頭の方になにやら身に覚えの無いエラーが出まくってます^^;。msysconf なんて
テーブル、作った覚えも無ければ参照するように指定してもいない。で、肝心の開き
たいテーブルを開いたときのログは最後の四行のみ(途中略)。
　サーバ側のログは pg_options に verbose=2 と query を加えて採ったものがあり
ますが、msysconf が無いというエラー以外に大して得るものがないように思います
ので、あえて添付しません。


　なお、WinNT の環境に戻ると現実的なレスポンスで使用できるようになります。
