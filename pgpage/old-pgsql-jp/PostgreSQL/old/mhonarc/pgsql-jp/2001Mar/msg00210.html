>     一度作成された２カラム構成のテーブルにカラムを追加したい場合、Excelのように
> 一段右にずらすようなことはできるのですか？それとも一度CSVにし、編集し、テーブル
> を再度作り直すのですか（今そうしてます）。
>     たまにテーブル内のあそこは要らない、追加したいと感じることがあるのでお尋ね
> しました。
> 
>     create table tab_prev (            ←最初に作ったテーブル
>         name   varchar2(30),
>         date    timestamp,
>     );
> 
>     (insertのようなコマンド)  tab_prev   （新しいカラム,  varchar2(10)) 

属性の(自然な)順序を気にしないのであれば，

   ALTER TABLE tab_prev ADD "新カラム" int4;

とでもやれば変更できます．

oid はどうでも良いけども属性の順序が気になるといのでしたら，

   ALTER TABLE tab_prev RENAME TO tab_tmp;
   CREATE TABLE tab_prev ("新カラム" int4, name varchar2(30), date timestamp);
   INSERT INTO tab_prev(date,timestamp) SELECT date,timestamp FROM tab_new;

なんて方が便利ですね．私はこっちをつかっています．
CSV 形式に落しているのと大差は無いですので，頻度が低ければもう後は好みと
慣れの問題でしょう．
