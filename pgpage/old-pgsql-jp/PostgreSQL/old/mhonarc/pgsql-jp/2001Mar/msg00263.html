On Mon, 19 Mar 2001 15:28:36 +0900, Kaori Inaba wrote:
>> PostgreSQLは、内部でアクセス管理をユーザ名で行っていますね。
>> PostgreSQLにアクセスすると、ＯＳにログインしたユーザ名とuidが
>> PostgreSQL内部（pg_user）に登録されます。
>> では、ＯＳのユーザ名もしくは、uidを変更した場合、pg_userは変更できませんし、
>> 発生した矛盾をどのように解決すれば良いのでしょうか？
>> uidの変更であれば、矛盾のままアクセスの継続が可能でしょうが、
>> ユーザ名の変更の場合、アクセス不能となります。
>
>pg_user は変更できませんが、pg_shadow からの VIEW になっているので、
>pg_shadow を変更すれば可能です。

うまくいきました。ありがとうございます。
