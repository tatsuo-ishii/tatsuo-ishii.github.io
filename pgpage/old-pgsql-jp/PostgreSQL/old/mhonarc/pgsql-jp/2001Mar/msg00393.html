    こんにちは、皆様から数々の助言を頂きありがとうございます。
    今回はconnection to the server was lostについてです。
    \d と入力すると、このような出力メッセージが現れテーブル情報
が得られません。今まではつかえたのですが、EUCやSJIS関連を
PGCLIENTENCODINGで色々変えたりしていたらこのようになって
しまいました。ついでにlikeも使えなくなりました：


nobody=# \d
The connection to the server was lost. Attempting reset: Failed.
!#

nobody=# select numsrl from test_table where numsrl like 3;
pqReadData() -- backend closed the channel unexpectedly.
        This probably means the backend terminated abnormally
        before or while processing the request.
The connection to the server was lost. Attempting reset: Failed.
!#

    これはどうしたらSJISでもって普通どおりに使えるようになるのでしょう？
一度全部削除して、Postgres7.0.2をリメイクしました。エラー無しにインストール
できたのですが、やはり\dとlikeは使えず、select * ...やcopyはOKです。
