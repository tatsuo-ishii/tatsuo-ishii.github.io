組込みSQL関数lo_*の権限のこと、教えて下さい。
Vine Linux2.0でPostgreSQL7.0.2を使っています。

kasa=> select lo_import('temp.dat');
ERROR:  You must have Postgres superuser privilege to use server-side lo_import().
	Anyone can use the client-side lo_import() provided by libpq.

と怒られてしまいます。
client-sideのlo_import()とは何なの？という感じで、psqlを起動するとき
に`-h'オプションを付けてみたりしましたが状況は変わりません。
同様にlo_export()も一般ユーザでは使用できないようです。

psqlのスラッシュコマンドを使用すれば一般ユーザでもimportできます。

kasa=> \lo_import 'temp.dat'
lo_import 104225

そして、以下のようにlo_unlink関数はスーパーユーザ権限がなくてできて
しまいます。

kasa=> select lo_unlink(104225);
 lo_unlink 
-----------
         1
(1 row)

以上をふまえた上で質問なのですが...

1) lo_import/lo_export関数がスーパーユーザ権限がないとエラーになって
   しまうのは、どういうセキュリティポリシーによるのでしょうか？
   一般ユーザがlarge objectを作れることはさほど危険なこととは思えま
   せんし、もしディスク容量が問題ならば、client-sideが許可されている
   理由が思いつきません。
2) lo_unlinkが一般権限ユーザで実行できてしまうことは、セキュリティ
   ポリシー上一貫性がないように思えますが、これは実装上のミスでしょ
   うか？
