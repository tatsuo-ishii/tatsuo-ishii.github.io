後藤様こんにちわ．

>>
>>勘ですけど、PostgreSQL をコンパイルしようとしていた時の CLASSPATH に、
>>
>> /usr/java/jdk1.2.2/jre/lib/ext/classes12.zip
>>
>>が含まれていたのではないでしょうか。
>>
>>Tomcat の実行環境 CLASSPATH には Oracle を使うのであれば、上記の
>>ものが含まれている必要がありますが、PostgreSQL の JDBC ドライバを
>>make する時にそんなものがあるとまずいのかも知れません。
>>
>>そんな設定で make した事がないので、はずしてるかも知れませんが…
>>

そうなんです．今のPostgresql は，javaのビルドに ant(makeみたいなもの）
と言うのを，使用していて，コンパイルコマンドに， classpath を付けて
くれるみたい．
で．．．Oracle JDBC の中身を参照しようとしたみたいで，
結局，クラスが未定義のエラーになるはずが，Oracle JDBC にあるので，
エラーにならず，別の処理を，実行せずにその後の処理でエラーでした．

通常，JDBC ドライバなど，ここに置くって，聞いてたような気がしてました．

でも，このおかげで，ant を少し勉強しましたけど．
