> linuxのマシンにApache + PHP + PostgreSQLをインストールして、社内のイントラネットDBを構築しています。
> 既に構築された方々のホームページ（http://www.hizlab.net/app/index.html等）を参考に
> それぞれのアプリケーションをインストールして、単体での動作は確認しましたが、
> HTMLからphpを通してPostgreSQLに接続しようとすると、
> 
> Warning: Unable to connect to PostgreSQL server: FATAL 1: IDENT authentication failed for user "****"
> 
> というメッセージが表示されてアクセスできません。

Red Hat Linux7.3に付属のRPMでPostgreSQLをインストールすると、
デフォルトのpg_hba.confが以下のようになります。
-----
local      all    ident    sameuser
-----
これに対して、ソースからインストールしたものは
-----
local      all                                          trust
host       all         127.0.0.1     255.255.255.255    trust
-----
となります（間違っていたらフォローお願いします＞皆様）
ご覧になられたページが、ソースからのインストールを前提としていた場合は
気をつけてください。

既にHIROSHI OOTAさんから回答がありますが、
pg_hba.confの設定を確認すると良いと思います。
http://www.postgresql.jp/document/pg721doc/admin/client-authentication.html#PG-HBA-CONF

本題とはそれますが、Red Hat Linux付属のRPMに関しては、
過去にも何度か話題になっていますので、一読されることをお勧めします。
