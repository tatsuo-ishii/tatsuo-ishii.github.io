FreeBSD のことは良くわかりませんが、とりあえず core を吐いているんなら、
debugger で trace back を取ればもう少し詳しいことがわかるでしょう。

あと思い当たるところでは、postmaster で bad system call のエラーが出る
場合には、System V の shared memory 拡張が kernel に定義されているかど
うかチェックしてみよ、と FAQ にあります。
