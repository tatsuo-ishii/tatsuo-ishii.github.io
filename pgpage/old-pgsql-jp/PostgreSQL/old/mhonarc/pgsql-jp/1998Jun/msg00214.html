石井です。

次期MB パッチ & 6.4 向けの機能追加として、データベース作成時にサーバ側
のデフォルトエンコーディングを設定できる機能の追加を検討しています。
(現在はコンパイル時にエンコーディングが決ってしまうわけですが、これだ
とやはり不便ですから) SQL92 標準でこの機能に相当する構文を見つけること
ができなかったので、勝手に以下のような構文を考えました。

CREATE DATABASE WITH LOCATION = 'dbpath' ENCODING = 'encoding';

(ENCODING = ... 以下が今回追加分)

ENCODING の指定を省略すると、コンパイル時に指定したエンコーディングが
採用されます。

また、このエンコーディングに関する情報は pg_database テーブルにカラム
を追加して格納する予定です(initdb が必要)。

これに伴い、サーバ側のエンコーディングを動的に変更するコマンドも追加し
ます。

SET SERVER_ENCODING TO 'encoding';
SHOW SERVER_ENCODING;
RESET SERVER_ENCODING;

これは SQL92 標準にあるぞ、とかもっと良い方法がある、あるいはこういう
問題がありはしないか？などの御意見・ご質問をお待ちしています。
