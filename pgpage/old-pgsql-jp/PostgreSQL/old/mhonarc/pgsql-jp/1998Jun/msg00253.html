石井です。

>塩崎です。
>char(N)型の埋め草の空白関係で教えてください。
>「=」演算子ではそれらの空白はあってなきが如くに扱われますが、「~」や「~~」
>の演算子では、それらの存在が認識されているようです。
>これって、SQL92的およびPostgreSQL的にこういう仕様なんでしょうか?

"=" に関しては PostgreSQL の振舞は SQL92 に即しています。"~" などはも
ともと SQL92 に存在しないオペレータですから、即しているともいないとも
言えないと思います。

>たとえば、
>create table t (c char(8));
>insert into t values ('a');
>insert into t values ('a ');
>という表を作ったときに、
>select * from t where c = 'a';
>だと2件出てくるのはまだよいのですが、
>select * from t where c ~ 'a$';
>select * from t where c ~~ '%a';
>などだと0件になってしまうのは、検索式を書くときに表定義のchar(N)型のバイ
>ト数まで意識しなくてはならず、ちょっと妙な感じもするのですがそういうもの
>なのでしょうか?
>それとも、こういう目的だけのためにもvarcharを使うべきなんでしょうか?

逆に、'a' と 'a ' を同じものと見倣したい場合以外に char() を使いたいケー
スってありますか？
