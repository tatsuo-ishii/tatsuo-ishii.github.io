t-ishii> どういう SQL を出しているのか分からないのですが、メッセージの意味は、

「これ以上どないせぇちゅうねん！」と言うくらい素直な SQL文です。
(と、本人は思い込んでいる)

例として、

    create table test (
        id char(3),
        data datetime,
        upflag char);

と言うクラスに、

    id |data   |upflag
    ---+-------+------
    001|ABCDEFG|N
    001|HIJKLMN|N
    001|OPQRSTU|N
    001|VWXYZAB|N

と言うデータが入っており、

    update test set upflag='Y' where id='001'

とする分けです。

これに begin や lock と 絡めて psql から実行すると、
本当に*まれ*に 

    NOTICE:  Non-functional update, only first update is performed

と怒られる時があります。
そして、怒られた時の結果は、

    id |rdate                       |upflag
    ---+----------------------------+------
    001|Wed Jun 03 16:33:01 1998 JST|Y
    001|Wed Jun 03 16:33:10 1998 JST|N
    001|Wed Jun 03 16:33:06 1998 JST|N
    001|Wed Jun 03 16:33:21 1998 JST|N

になっています。

バグかどうかの判断の役に立てば幸いです。
