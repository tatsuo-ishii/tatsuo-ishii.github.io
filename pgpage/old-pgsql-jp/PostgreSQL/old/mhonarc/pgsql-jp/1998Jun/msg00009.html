PostgreSQL では、"major version up" (6.1 -> 6.2 のように、バージョン番
号の少数点以下1桁目が変わる)の際には、pg_dump 一発で簡単にデータベース
を移行するというわけにはいきません。詳しくは、migration の下のドキュメ
ントを読んで下さい。

移行するデータ量がどの位か知りませんが、一番確実なのは、

(1) 6.1 のシステムを動かしておいて、そこで copy を使って ASCII file の
    形でデータをセーブする

(2) 6.3.2 を起動し、テーブルを作成する

(3) (1)でセーブしたデータを copy で戻す

(4) 必要ならば、ユーザ定義関数の登録、インデックスを設定し直しなどを行
    なう

という手順です。
