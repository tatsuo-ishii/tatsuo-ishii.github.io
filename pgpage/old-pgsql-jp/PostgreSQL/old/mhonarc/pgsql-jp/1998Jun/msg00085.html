Linuxを勉強し始めて約１ケ月半の初心者で 関 といいます。
1年間Web作成(とは言っても素人の簡単なHomePageの作成）をやってきました。
今年はWebServerの勉強です。
LinuxTurboProを購入し市販本、メーリングリスト等によりLAN環境ができました。
日常のDB業務にLinuxを応用したいと思い、Linux Usersメーリングリストを
通じて、このPostgreSQLサーバを教えて頂きました。
全くのSQLの素人のため、,先ず情報量の多さに驚き、どこから取り掛かろうかと迷
います。
全体像
の把握するため雑誌SoftDesign掲載記事の読解から始めましたが、システム環境を
作らなければ理解できないと分かり、
今「Apache 1.2.6+PHP/FI 2.0.1+PostgreSQL
6.3.2のインストール方法」に取り掛かっています。
早速、「インストールの前に」で多立ち止まってしまいました。
初歩的な質問で申し訳ないですがご教示願います。


１．先ず、インストールのやり方として
Plagiaware 用の
○"石井@SRA さんによるマルチバイト拡張オプション"の
「Apache 1.2.6+PHP/FI 2.0.1+PostgreSQL 6.3.2のインストール方法」と
○"こじまさんのサイト
http://www.st.rim.or.jp/~isle/Plagia/のインストール方法"
が掲載されていますが、LinuxTurboを前提とした場合相違がありますか。

２．"石井@SRA さんによるマルチバイト拡張オプション"の
「Apache 1.2.6+PHP/FI 2.0.1+PostgreSQL 6.3.2のインストール方法」PostgreSQL
6.3で確認させて下さい。

>PostgreSQL をインストールするためには、少なくとも 2.7.2 以上の gcc、3.75
以上の
> GNU make (gmake) が必要です。 
今ToshibaのNotePCにTurboLinuxPro1.0をインストールしています。

HDDが少ない容量のため1.4verのCD-ROMは購入済みですが未使用です。
この gcc、GNU make は今のマシンに手当てを必要なのでしょうか。

>また、yacc はトラブルが起きる可能性があるのでお勧めしません。bison
をお使い下さい。 
>lex ではなくて flex を使う場合は、flex-2.5.2 or flex-2.5.4
を用意して下さい。
>(flex-2.5.3 は使えません) 

ここも同様今のマシンに変更が必要でしょうか。

>必要なディスク容量は、約30MB〜50MB です。
>これに更に データベース自体の容量を追加する必要があります。
>ただし、データ ベース領域は
PostgreSQLをインストールするファイルシステムと別にするこ とも可能です。 

今のマシンのHDDの容量の730M中アミューズメント、ゲームソフトを外して590Mに減
らし100M程度の
空きを作りましたが、Apache 1.2.6+PHP/FI
2.0.1もインストールするとしたら、容量不足になるでしょうか。
もし、そうなら、大きなHDDを入れ替えねばなりません。

> 可能ならば PostgreSQL 専用のアカウントを作ってコンパイル、イン
ストールした方が良いでしょう。
> root (uid 0) のままでは、コンパイル、インストール(make install)
まではできても、その後の
>データベースのインストール (initdb) ができませ んので、ご注意下さい。
 
これは、ROOTで以外の別途登録ユーザーでLOG−INするという意味でしようか。

３．Linux Usersメーリングリストの中で
「PostgreSQLやApacheをJRPMのものと入れ替えたほうがよいです。」
とアドバイスをいただいたのですが、Linux Japanese RPM
Projectホームページには  

****************************************************************************
******
JRPM のパッケージを利用される際には rpm
コマンドのバージョンアップをお勧め致します。
   failed dependencies:
        /bin/sh is needed by ○○
となり、パッケージがインストールできないという場合、rpm-2.4.X をインス
トールすれば解決します。
 rpm コマンドの最新版は
       ftp://ftp.redhat.com/pub/redhat/code/rpm/
にあります。
====================
====================
 glibc 用のディレクトリが作成されました。
       ftp://ftp.linux.or.jp/pub/RPM/glibc/
 RedHat 5.0 の方は、この XFree86 代替ライブラリを御利用下さい。
 まず README をお読み下さい。
 また glibc 用に incoming はこちらです。
       ftp://ftp.linux.or.jp/pub/RPM/glibc/incomin
****************************************************************************
*****

とあり、一方、

”プロジェクトで作成されたパッケージの1部がパシフィック・ハイテック
の「TurboLinux」に収録して頂いております。
またメディアラボ株式会社にも 「LinuxMLD II」等で利用して頂いております。
さらに蕗出版の『新版 Linux でインターネット』, SOFTBANKの『UNIX
USER』1998年6月号 の CD-ROM にも収録して頂いております。”

とあります。

LinuxTurboProのインストール済みの現行マシンで手当てが必要でしょうか。
