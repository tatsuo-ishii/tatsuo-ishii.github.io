At 12:33 +0900 2000.07.20, Kazumasa Gotoh wrote:
> > TurboLinux Server 6.0J + PostgreSQL7.0.2 を使ってます。
> > 
> > kterm で psql を起動すると、どうやらみなさんのお話を聞く限りでは、
> > Ctrl-p で、前に実行したコマンドが出るのが本来の姿のような気がする
> > のですが、私のところでは出ません。
>
>別に Ctrl-P などが使えるのが「本来の姿」というわけではありません。
>例えば素の Solaris 環境などで make すればそうなります。
>
>Ctrl-P などは readline ライブラリにより実現されている機能ですが、
>Configure 時に readline ライブラリが無いと判定されればそのような
>動きになります。無いものは組み込めないですからね。

　ちょっと[pgsql-jp 16231] のヒントになりました。
変な文字列を入力することしか頭になかったのですが、psql -n で
起動すると、たしかにctrl-p などが効かなくなり、^N, ^P などが
表示されます。

postgres@bash-2.02$ psql -n tmp
Welcome to the POSTGRESQL interactive sql monitor:
  Please read the file COPYRIGHT for copyright terms of POSTGRESQL
[PostgreSQL 6.5.1 on sparc-sun-solaris2.6, compiled by gcc 2.7.2.1]

   type \? for help on slash commands
   type \q to quit
   type \g or terminate with semicolon to execute query
 You are currently connected to the database: tmp

tmp=> ^N^N^N^P^P


オンラインマニュアルを見ると、-n オプションの説明が
このようになってました。

postgres@bash-2.02$ man psql
(略)
     -n   Do not use the readline library for input line  editing
          and command history.


そうすると、-n オプションをつけないで起動した時はちゃんとctrl-p
などが効くので、うちの環境にreadline ライブラリは入っているという
ことなんでしょうね(いつのまに?)。ただし日本語入力できません。
ということは、日本語未対応のreadline ライブラリを使うと現象が
起きるということなのかな。 > [pgsql-jp 16231]

([pgsql-jp 16231] の続きはまた時間が取れてから。)
