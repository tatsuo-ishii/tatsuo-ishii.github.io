一般に、新しい version の binary を install する場合は

1.前の行なった変更点は全て以前の状態に戻す
2.document にしたがって、新しい version の binary を make する
#1 と 2 は順番を入れ替えても構いません
3.su して root になって、binary を所定の方法で install する
4.さらに所定の方法で、所定の設定をする。
5.起動する

という手順で行ないます。program によっては、uninstall 作業を 
"make uninstall" の一言でよしなに取り計らってくれるものもあります。し
かし、一般的にはそういうものはありません。つまり、install の時の手順を
逆に踏んでいくだけです。

#Unix の場合、Windows のような "dll 差し替え" という自滅的なことは滅多
#に行なわれませんから、install を逆 trace することはそれほど難しくあり
#ません

なにはともあれ、以前の install の際にはどういうことをしているのか？、
そして今回の install ではどういうことをする必要があるのか？という点を 
document をあたって調べてみることが肝要かと思います。

なお、Linux の rpm とか FreeBSD の packages なら、それぞれの uninstall 
用の command なりがあるはずです。それを使って uninstall するのが、"正
しい uninstall 方法" です。
