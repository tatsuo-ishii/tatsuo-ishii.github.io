まだ、Postgresを触り始めて１ヵ月といったとこです。早速ですが質問させてください。

PostgreSQL6.5.3を使用しています。
以下のようにテーブルを作成しました。

create table mailadmin 
(
   COMPANY		text, -- 企業コード
   DEPARTMENT	text, -- 本支店・部署コード  
   COUNT		text, -- カウンタ 
   MAILNO		int4, -- メール番号 
   MAILGROUP	text, -- 所属グループ番号 
   EMAIL		text, -- メールアドレス 
   SHIPDATE	date, -- 発送開始予定日 
   ERRORCODE	int2, -- エラーコード  
   primary key (COMPANY, DEPARTMENT, COUNT, MAILNO, EMAIL, SHIPDATE)
);

このテーブル(mailadmin)は、挿入と削除を頻繁に行うので、40MBを超えたあたり
でにvacuumを実行したところ、8Kまで圧縮されたのですが、プライマリーキーに
より作成されたインデックステーブル(mailadmin_pkey)は25MBと大きいサイズのま
ま残っています。

インデックステーブルを圧縮する方法を教えてください。
