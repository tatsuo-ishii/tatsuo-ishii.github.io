どの分野でも専門書の類は一冊数千円するものは珍しくない… というか、
それがフツーでしょう。
ま、気持ちはわかりますが、「本を買う事ができるだけマシ」と考える
べきでしょう。売ってないものは買う事ができないのですから。

尋常の手段では関連資料が入手できなくて切歯扼腕するというのは、
昔でも今でも分野を問わずよくある事です。

コンピュータ関連分野の場合は、資料の入手性が非常によく、しかも翻訳
された書籍類も入手できるのですから、その事を感謝するべきだと考えます。

> インターネットで探したいのは、料金の問題もありますが、何だかんだ言っても
> やっぱり情報が新しいからです。前に技術本(DB関係じゃないですが)を購入した
> 時に、ろくすっぽ初版日付を見ずに買ってしまって(外装は当然新しいです)、後
> 悔したことがあったので・・。
> インターネットでももうちょっとキーワード絞って探してみます。

北村さんの言っているのは、たぶん二次情報の方ですよね？
でしたら、いわゆる「インターネット上の情報」が新しいと過信するのは
危険です。
# 内容的にも頭から信じてしまうと危険な場合があります。

「インターネット上の情報」にはかなり古いものや、間違っているものが
含まれている事も少なくありません。

例えば ISC 版 DHCP に関する情報を国内サイトで検索すると、バージョン
1.0 のものが多く出てきて、現在の主流である 2.0 とか、β版が出ている
3.0 の情報はほとんど出てきません。
# 私の捜し方が悪いのかも知れないけど… (^^;

また、書籍類を買う時に内容をよく吟味せずに買うのは、昔から
「勇気ある行為」だとされています。:-)
# 金銭的な問題であれば、図書館を利用するという方法もあります。

SQL の様な標準技術に関して「インターネット上で検索」する場合は、
やはり一次情報源… 標準化機関が公開している情報を検索するのが
最も間違いが少ないと私は思います。
本家に行けば情報がある事がわかっているのに、他の何かないかと
捜すのは無駄な行為ではないでしょうか？

ANSI や ISO の SQL 関係の URL が何かは私は知りませんが… (^^;
どなたかご存知でしたら教えてください。_o_

とは言え、この手のものは専門性がより高い場合が多いし、英語の海に
溺れてしまう事もあるので、(^^; 翻訳書籍類を使う方が私は多いです。

出版社や訳者、監修者、著者などをチェックして、内容も見てから
買うのは基本。(^_^)
