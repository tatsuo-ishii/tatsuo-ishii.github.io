「重い sql 実行」とのことですが、
  * メモリ不良
  * CPU の熱による暴走
あたりが怪しい可能性があります。

手元のマシンでメモリを 512MB にしてある AT compatible マシンが複数台、
PostgreSQL 6.5.3 / 7.0.2 で動かしていますが、全く同一の構成のマシンを
用意しているのに、どうにも片方だけ pg_dump したデータの読み込みに失敗
したりでかい sort の入った SQL 処理で segmant fault で落ちてしまうという
症状があり、悩んでいました。

reboot するとデータの読み込みが出来てしまったりするなど、どうにも不安定
な動きで困っていたのですが、BIOS で fast startup を disable にしてみて
メモリチェックを厳密にしてみたところ、test failure したりしなかったり、
という症状があることが分かり、結局メモリ交換したらバッチリ直ってしまいま
した。

たまたま、こういうことが続けてあったので（製造ロットは違ったはずなんだけ
ど…）

> このような状態の場合、何を疑えば良いでしょうか？

見たところ、OS は (kernel version は異なりますが) Linux で同じですし、
マシンの差異もスペック的なもののようですので、ハードウェア側の不具合
が気になります。
