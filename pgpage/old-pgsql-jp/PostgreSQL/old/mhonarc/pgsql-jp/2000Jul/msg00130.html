Yoshio Funaki wrote:

> copy hoge from $HYORO/nyoro.txt；
> としておいて$HYOROの値を変えることで対応したいわけです。

B Shell の here document という機能を知っていると便利です。
上の場合、次のようなスクリプトを書いて、実行指せます。

#!/bin/sh

psql ＤＢ名  <<  //eof//

   ここに、ＳＱＬ文を記述;

//eof//

echo 文と異なり、" をエスケープしないで済みます
ので、楽です。
//eof// は、任意の文字列です。詳しくは、B shell
またば  bash の解説書を見て下さい。


> --（マイナス２個）でコメント文が書けることを知ったのですが
> これって、超初心者的疑問だと思うのですが、ふれている書籍は
> ほとんどなかったです。

まともな SQL の解説書なら必ず書いてあるはずですが。

商用ＤＢの昔の version で、 -- の後に漢字を書いたり、
長すぎたりすると、エラーになって悩んだことがありま
したが、PostgreSQL 6.5.3 では、今のところ、-- 注釈
でトラブルを経験したことはありません。
