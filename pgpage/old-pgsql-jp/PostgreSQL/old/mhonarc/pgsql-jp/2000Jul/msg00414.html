うちでは、以下のようになりました。

Vine Linux2.0CR   PostgreSQL6.5.3を入れたThinkPad240の
X-window上のgnome-terminalで実行した結果です。

----------
[postgres121@thinkpad postgres121]$ psql postgres121
Welcome to the POSTGRESQL interactive sql monitor:
  Please read the file COPYRIGHT for copyright terms of POSTGRESQL
[PostgreSQL 6.5.3 on i686-pc-linux-gnu, compiled by gcc egcs-2.91.66]

   type \? for help on slash commands
   type \q to quit
   type \g or terminate with semicolon to execute query
 You are currently connected to the database: postgres121

postgres121=> 小菅
postgres121-> 
postgres121-> ; 
ERROR:  parser: parse error at or near "小菅"
postgres121=> 何
postgres121-> 
postgres121-> ;
ERROR:  parser: parse error at or near "小菅"
postgres121=> 
----------

lsの動作はしないのですが、
エラーメッセージが、「何」になっていないのはなぜでしょう。

同様に、「何」を先に実行しようとして、次に「小菅」を実行
しようとすると、

ERROR:  parser: parse error at or near "何"

になってしまいます。
