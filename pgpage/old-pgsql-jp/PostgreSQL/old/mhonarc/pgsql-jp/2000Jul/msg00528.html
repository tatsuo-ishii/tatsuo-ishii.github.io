> 上記の条件で日本語を使おうとすると文字化けするのですが、
> どのように対処すればいいのでしょうか。JDBCドライバーの
> パッチなどがありましたら、教えていただきたいと思います。

何が「上記の条件」なのでしょうか？

PostgreSQL を動かしているプラットフォームが何なのか、Java を
動かしているプラットフォームが何なのか、JDK のバージョンは？
日本語処理の話をするのであれば、PostgreSQL 側の文字エンコーディング
に何を使用しているの？ また、Java 側の文字エンコーディングは？

これらをすべて書いてはじめて、「あなたが使用している条件」を
示した事になります。
PostgreSQL にしろ、Java にしろマルチプラットフォーム環境で動く
ものですし、文字エンコーディングも一種類ではありません。
誰もがあなたと同じ環境で動かしているのではないのをお忘れ無く。

で、この話はこの ML 上で何回も出ているものです。過去記事の検索を
行なえば話の流れなどがわかるかと思います。

要は、文字エンコーディングの処理の問題で、PostgreSQL 6.5 以降で
あれば、PostgreSQL 自身の文字エンコーディングと Java 環境の文字
エンコーディングが一致している場合には、ほぼ問題無く文字を扱う
事ができる。
しかし、両者が一致していない… PostgreSQL が EUC_JP で、Java 側が
SJIS のような場合にはうまくない。という事です。

そういや、7.0.x で私自身がこの点を追試したかなぁ… まだメイン環境を
7.0.x に移行していないんで、記憶がちょっとあやふやですが、確か
同じようになったかと思います。(^^;

PostgreSQL 6.5.x であれば、持田さんのパッチでこの問題を解決する
事ができます。

 http://www.netside.co.jp/~mochid/comp/postgresql-jdbc/index.html

PostgreSQL 7.0 用のパッチがあるのかどうか私は知りませんが、上記 URL
にあるパッチを調べて、7.0.2 に適合するように作り直せば対処できるの
ではないかと思います。
