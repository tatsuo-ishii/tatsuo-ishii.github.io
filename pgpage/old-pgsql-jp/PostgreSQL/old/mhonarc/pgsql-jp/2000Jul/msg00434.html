[pgsql-jp 16201] CREATE RULE と OID について

  • From: "Eiji Tokuya" <e-tokuya@Mail.Sankyo-Unyu.co.jp>
  • Date: Tue, 18 Jul 2000 11:44:10 +0900
  • Importance: Normal
こんにちは、徳家です。
OID が2147483647を越えた場合の不具合を見つけたので報告しておきます。
新規にRULEが作成できず

ERROR:  pg_atoi: error reading "2147485552": Numerical result out of range

と返ってきました。
"2147485552"とは明らかにOIDの値ですので、CREATE RULEはOIDが2147483647以上に
は対応してないようですね。
他にもオブジェクト作成時にOIDにatoi()を使っているものは同じエラーを返すと思
います。
# まあ、これほどOIDのカウントが進んでから新たにオブジェクトを作成することは
まずないでしょうが・・・・。(^^;;;;

CREATE TABLE や CREATE SEQUENCE や CREATE INDEX は問題ありません。
