福井と申します。

> Access2000からODBC経由でPostgresqlにリンクをしているテーブルのデーターを変更
> しようとすると、Accessから”このレコードはほかのユーザーによって変更されって
> います。”というようなメッセージが出ってきました。変更できませんでした。
当方でもこの症状は出ました。
環境 VineLinux 2.0
     Postgres7.0.2
     ODBC Driver 6.50.0000 + 2000/6/17の日本語版
     クライアント Access2000(Win98)

確認したところ、可変長文字(character varying())の値が '' （長さ０の文字）
の場合に、そのレコードを Access から変更しようとすると必ずこのエラーが
発生するようです。

> 現在の対応は、テーブルを一旦Access2000にインプットし、テーブルを一度削除して
> から、新しい作ったテーブルにデーターを流し（Accessを使い）、その後リンクをつ
> けて、変更不可のレコードも変更可能になります。
根本的な解決にはなってませんが、'' を NULL 値に置き換えると
「このレコードはほかのユーザーによって・・・」のエラーは発生しないので、
下記のような SQL をそれぞれ実行して対応しました。

update テーブル名 set 項目 = NULL WHERE 項目 = '';
