From: Ran Hosoya <ran@ptl.sony.co.jp>
Date: Wed, 19 Jul 2000 13:42:30 +0900 (JST)

> TurboLinux Server 6.0J + PostgreSQL7.0.2 を使ってます。
> 
> kterm で psql を起動すると、どうやらみなさんのお話を聞く限りでは、
> Ctrl-p で、前に実行したコマンドが出るのが本来の姿のような気がする
> のですが、私のところでは出ません。

別に Ctrl-P などが使えるのが「本来の姿」というわけではありません。
例えば素の Solaris 環境などで make すればそうなります。

Ctrl-P などは readline ライブラリにより実現されている機能ですが、
Configure 時に readline ライブラリが無いと判定されればそのような
動きになります。無いものは組み込めないですからね。

まぁ、bash では同じようにできるのにおかしいな。と思うかも知れませんが…

私は FreeBSD 派なんで、Linux の各ディストリビューションの構成には
あまり詳しくないのですが、TurboLinux Server 6.0 は、開発のための
rpm パッケージをあれこれ組み込まなければ、あまりプログラム開発用
として適したものにはならないようです。

何も考えずにリターンキーだと押したりマウスをクリックしたりして
デフォルトでインストールすると、readline の実行時ライブラリは
組み込まれるものの、readline ライブラリを使用するプログラムを
作るためのヘッダフィルが組み込まれないのです。

必要なパッケージをインストールしてから、Configure し直しましょう。

私は TurboLinux Server のこの方針が特に間違っているとは思いません。
「本番 Server には開発環境など不要」という考え方は理解できるからです。

Server は Server としだけ動けばいいのであって余分なものは不要。
開発やテストは他の Workstation やテスト用 Server で行なって、
開発したものを本番 Server に移して運用する。というのはしごく
まっとうな考え方です。

問題があるとすれば、マニュアルにそのようなコンセプトの説明とか、
開発用に使いたければこうしろ。と書いてあるか、またはユーザが
それを理解しているかどうかという部分だと思います。

マニュアルにはちゃんとそのような事が書かれているのでしょうか？
評価版しか使った事がないので、製品ではどうなっているのか知らない…

> Ctrl-h も同様に「^H」と表示されるだけで、バックスペースの動作はし
> ません。
> # Delete キーがバックスペースの動作をしています。

そう動くのが UNIX の世界ではフツーというか、当然です。
TeraTerm のキーボード設定で BackSpace キーを押した場合に Delete
キーのコードを送るというのがあるのは、その辺りの事情によるものです。

確かにデフォルトがそうではない UNIX も確かに存在しますけどね。
それでは気に入らないので変えたいというのであれば、stty コマンドで
tty の設定を確認しましょう。
