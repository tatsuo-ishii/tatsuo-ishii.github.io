PostgreSQLとのやりとりをする度に接続していたりすれば、その度に backend が
fork されてしまうので、それだけでサーバの CPU やメモリリソースが大量に
消費されてしまいますね。もちろん CGI のプログラム起動もそうでしょう。

PostgreSQL との接続を保存しておいて使い回したり、Apache の mod_perl を
利用したりするのがいいのではないでしょうか? (この辺ちょっとあやふやな
知識で書いてますが)

けっして Perl の実行速度が低いだけが原因ではない思います。CGI を C言語
で書き直しても、思った程の性能向上が見られないかもしれませんよ。
