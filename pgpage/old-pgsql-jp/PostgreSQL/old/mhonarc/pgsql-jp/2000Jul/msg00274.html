> あと、余談なんですが便乗質問。
> ANSI SQL = SQL2 = SQL92 なんですか？(インターネットでSQL92を調べても、
> 特に解説してくれているところが無かったので・・・）

SQL2 = SQL92 と捉えても、あながち間違いではないとは思います。

ただ、SQL2 は標準化作業のためのコードネームで、その作業が完結して
SQL92 となったと言う事らしいので、ある時点で「SQL2」として語られて
いる内容が、最終的に完結した SQL92 とは異なる部分があるというケースも
あるかも知れません。

また、ANSI SQL = SQL92 はちょっと正しいとは言えないでしょうね。
標準 SQL には、SQL86 や SQL89 ってのもあります。

ANSI SQL ってのは総称名ですから、敢えて言えば ANSI SQL > SQL92 かな？
それもちょっと違うと思うけど。

だいたい、ANSI ってのはアメリカの国家規格ですしね。国際規格としては
ISO の方だし。限りなく ANSI = ISO に近いものも多いような気もしますが…

また、コードネーム SQL3 として標準化作業が進められている(ひょっとして
もう終わったのかな？ 事情に疎いんでよくわかりません)ものが完結すれば
SQL?? という名前が与えられるでしょうけど、これもまた 標準 SQL だと
いう事になります。

ところで、こういう事情は「標準 SQL」とか、「SQL92 完全ガイド」などにも
書かれています。これらの本の出版社や ISBN などは、石井さんの Web ページ
にも載ってますよ。

また、どうしても「インターネットで」 :-) これらの SQL に関する情報を
捜したいのであれば、ANSI や ISO の機関自身が標準化作業の流れや結果を
Web ページ上で公開しているはずなので、そっちの公開情報の中から捜せば
出てくるはずです。

そっちが本家なんですしね。(^_^)
