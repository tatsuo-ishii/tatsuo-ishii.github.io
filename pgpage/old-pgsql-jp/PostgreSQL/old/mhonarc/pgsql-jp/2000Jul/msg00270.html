やはりこういう症状に出会った人は私以外もいらっしゃいますね。

> 確認したところ、可変長文字(character varying())の値が '' （長さ０の文字）
> の場合に、そのレコードを Access から変更しようとすると必ずこのエラーが
> 発生するようです。

私も、色々とレコードを比べって見ましたが、
内一つのテーブルのすべてフィールドデーターの長さ０と言うレコードはありません
が、このテーブルの中に変更できないレコードもあります。
変更できないレコードの共通点は漢字たくさん入力されたデーターが変更できませ
ん。
中を探って見ましたが、入力されたレコードの中、改行記号あるのレコードはすべて
変更不可です。

もう一つのテーブルはすべてのレコードは長さ０の値を持つのフィールドはいくつか
存在しています。このテーブルはすべてのレコードは変更不可です。そちらと同じ症
状です。

> 根本的な解決にはなってませんが、'' を NULL 値に置き換えると
> 「このレコードはほかのユーザーによって・・・」のエラーは発生しないので、
> 下記のような SQL をそれぞれ実行して対応しました。
>
> update テーブル名 set 項目 = NULL WHERE 項目 = '';
後でやって見ます。

症状を合わせて見ると：
１．varchar()　は長さ０の値はだめです。（null に置き換える必要）（Access2000
の場合）
２．改行記号を入力されるとだめです。（Access97、Access2000）
ちょっと見えて来ましたね、やはりドライバの問題かな？
