そういうことだと思います。

マニュアルからは、ALTER TABLEの頁に、
(PostgreSQL 6.5.2の付属マニュアルより)
>ALTER TABLE table
>    DROP [ COLUMN ] column { RESTRICT | CASCADE }
>
>Removes a column from a table. If RESTRICT is specified
>only a column with no dependent objects can be destroyed.
> If CASCADE is specified, all objects that are dependent
> on this column are also dropped.
>
>Currently, to remove an existing column the table must
>be recreated and reloaded:
>
>CREATE TABLE temp AS SELECT did, city FROM distributors;
>DROP TABLE distributors;
>CREATE TABLE distributors (
>    did      DECIMAL(3)  DEFAULT 1,
>    name     VARCHAR(40) NOT NULL,
>);
>INSERT INTO distributors SELECT * FROM temp;
>DROP TABLE temp;
とあります。私も英語は読めないんですが、例題を見ると、テーブル
distributorsの必要なカラム(did,name)から一時的なテーブル、
tempを生成し、その後、distributorsを削除、必要なカラムだけで
作成、そしてtempからdistributotorsへ戻す、ということをやって
いるな・・・と。
すぐ上の英文を見ると、「現在、存在しているカラムを削除するには、
テーブルを再作成して再読込しろ」と書いてありますね。

では。
