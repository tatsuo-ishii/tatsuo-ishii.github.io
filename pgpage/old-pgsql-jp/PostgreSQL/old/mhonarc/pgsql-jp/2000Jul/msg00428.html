落ちる直接の原因を調べるべきだと思いますので、postgres プロセス
に core を吐かせるのが先決でしょう。

釈迦に説法かもしれませんが、postgres に core を吐かせるには、
まず -g オプションを付けて PostgreSQL を make し直しておきます。
その後 postmaster を起動しているシェルスクリプトで 
  ulimit -c unlimited 
相当のことを行い、無事に？ core ができたら
  gdb /path/to/postgres core 
とやればコケたところが表示されます。後はそのログをここに流して
いただければ、どうにかなるのではないかと思います。その際、A,B 
両者のコンパイラのバージョンも明記した方がよいかもしれません。
