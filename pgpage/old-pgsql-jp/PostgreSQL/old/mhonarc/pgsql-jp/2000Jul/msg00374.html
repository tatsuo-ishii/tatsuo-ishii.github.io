[pgsql-jp 16141] psql に変な文字を食わせた時の動作

  • From: Shoichi Kosuge <s-kosuge@str.hitachi.co.jp>
  • Date: Sat, 15 Jul 2000 00:24:45 +0900
こすげです。

　最近気付いたのですが、psql を-n オプション無しで起動して、
「小菅」という文字列を入力すると、カレントディレクトリの
ファイル名一覧(要するに、ls の実行結果)を返してきます。
　
postgres@bash-2.02$ psql tmp
Welcome to the POSTGRESQL interactive sql monitor:
  Please read the file COPYRIGHT for copyright terms of POSTGRESQL
[PostgreSQL 6.5.1 on sparc-sun-solaris2.6, compiled by gcc 2.7.2.1]

   type \? for help on slash commands
   type \q to quit
   type \g or terminate with semicolon to execute query
 You are currently connected to the database: tmp

tmp=>  <- ここで「小菅」と入力する
Makefile    rmgr.o      transsup.c  varsup.o    xid.c       xlog.o
SUBSYS.o    transam.c   transsup.o  xact.c      xid.o
rmgr.c      transam.o   varsup.c    xact.o      xlog.c
tmp=>

　で、CTRL-P で前に実行したコマンドを表示させようとしても
何も表示されません。コマンドの結果と言うわけではないようです。
　単なる現象報告だけですが、御参考ということで。
　他にもあるかと思ってここに書いた文字列をカット＆ペーストして
みたところ、「何」でも同様にファイル名一覧が返ってきました。
けっこう便利かも(^_^) でもなぜ?

環境は、Solaris7 + PostgreSQL 6.5.1 で、メイクはSolaris 2.6
で行なったものです。Mac のNCSA Telnet 2.7J6 で接続してpsql を
起動しています。

過去記事検索は、いわきりさんページで「ls」をキーワードに
107件中60件程題名と最初の数行が表示されるのを読みましたが、
上記のような話は見つけられませんでした。
