お世話になります。

うぃーずの及川と申します。

FreeBSD 4.4-RELEASE 上で、
apache-1.3.20
mod_php4-4.0.6_4
postgresql-7.1.3
の構成でサーバを運用しておりますが、DB TABLE のフィールドがオーバー
フローする現象が発生しております。
原因は、半角のカンマが原因でレコード時にカンマの前にバックスラッシュ
を挿入されます。レコードが更新される度にバックスラッシュの量は倍に
なり最後には、

ERROR:  copy: line 1906, Tuple is too big: size 64916, max size
8140 PQendcopy: resetting connection

というようなエラーになります。

根本の解決方法と、
あと、増えすぎたバックスラッシュの一括削除方法等教えてくださると助
かります。

どうぞ宜しくお願い致します。
