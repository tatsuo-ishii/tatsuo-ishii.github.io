ポイントはエラーメッセージにある'/Pg.pm'という言葉でしょう。
でMakefile.PLを追いかけてみると、

> my $srcdir=$ENV{SRCDIR};

ってなところがあるので直接的な原因はこれかなと。
で、同梱されているREADMEを見ると

<引用>
INSTALLATION:
-------------

Since the perl5 interface is always contained in the source tree of
PostgreSQL,
it is usually build together with PostgreSQL itself. This can be obtained by
adding the option '--with-perl' to the configure command.
</引用>
といったことなので、そのディレクトリ（src/interfaces/perl5)で
  make clean
したあと、ソースのルートディレクトリに戻って、
  ./configure --with-perl
としてみては、いかがですか？
それでもダメなら、CPANからPgのディストリビューションを取ってきた
ほうがよいかも。

pgsql_perl5
http://search.cpan.org/search?module=Pg
