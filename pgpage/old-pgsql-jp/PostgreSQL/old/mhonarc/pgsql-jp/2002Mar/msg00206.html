Tsunehisa Kazawa wrote:
> 徳家さんのご意見を拝見していると、PostgreSQL が作る Shift JIS のス
> トリームを、よりもれなく Unicode 変換出来るのは MS932 コンバータの方
> である、ということなのでしょうか？

README.mb.jp に、Unicode との変換テーブルとして Shift JIS 系
は (JIS0208.TXT ではなく) CP932.TXT を使った、と書いてありまし
た。申し訳ありません。RTFM を肝に銘じます。

つまり Java の MS932 コンバータが同様の変換表を使っているのであ
れば (その可能性大でしょう)、そちらの方がよりふさわしいと。

谷田さんの patch は非常に正しいということになるわけですね。
