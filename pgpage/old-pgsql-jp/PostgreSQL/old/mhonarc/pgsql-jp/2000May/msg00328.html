> ODBCの大文字小文字の区別が正しいよう

createdb のスクリプトなどを見るとわかるのですが、PostgreSQL の場合は、
最終的に psql が起動されて、SQL の create database コマンドでデータ
ベースが作成されます。

確か SQL コマンドの世界では、大文字小文字の区別はなかったはずですので、
PostgreSQL のデータベース名は Case Insensitive だという事になると
思います。
つまり、テーブル名やカラム名と同じ扱いだという事です。

ODBC で大文字小文字が区別されるのは、ODBC という仕掛けのせいなのか、
PostgreSQL の ODBC ドライバ側の問題なのかは知りませんが…
