[pgsql-jp 14359] PostgreSQL 6.5.3 のJDBCドライバとJIT

  • From: Kazumasa Gotoh <kgotoh@cic-kk.co.jp>
  • Date: Mon, 08 May 2000 16:27:02 +0900

JavaHouse-Brewers に投げても反応がなかったので、今度はこっちに… (^^;

FreeBSD 3.3 + JDK 1.1.8 で、JIT を試してみたのですが、PostgreSQL 6.5.3
の JDBC ドライバを使うとうまく動かない組み合わせが、当方では発生しました。

 1. shuJIT 0.5.1 -> 問題なく実行できる。
 2. TYA 1.7      -> DriverManager.getConnection() で例外が発生して、
                    "No suitable driver" だと言われる。

もちろん両者は同じクラスファイルを用いての結果で、-Djava.complier= で、
shujit を指定するか、tya を指定するかだけの差です。
JIT 指定無しの場合は、もちろん問題なく動きます。

他の JDBC ドライバでは？ と思っても Oracle8 8.0.5 の thin タイプしか
手元にないのですが、これは shuJIT でも TYA でも問題なく実行できます。

他のプラットフォームではどうでしょうか？ 例えば、Linux では TYA 1.7 を
使っても問題ないのでしょうか？

何か情報をお持ちの方がいらっしゃいましたら、よろしくお願いいたします。

# JIT として TYA をどうしても使わなければならないというわけでは
# ないのですが、PostgreSQL のだけ動かないというのは、ちょっと
# 気持ちが悪いです…
