　PHP は使ったことがありませんので確実ではありませんが。。。
　例えば html を表示しようとするときにあるテーブルからレコードを取得して
それを多分 FORM の INPUT を使用してそれぞれ表示するといったことを行うんで
すよね？その後人の手で変更を加えた後にその変更を DB に反映させるといった
手順ですか？
DB Query -> html 表示 -> 変更（人手を使って） -> DB 更新
といった流れでしょうか。
　この「DB 更新」作業の時点で複数ある INPUT フォームとその値を引数にした
いと言うんですよね？

　軽く php のマニュアルを読んでみましたが INPUT フォームとそこに入ってい
る値を

$fruits = array(
    "INPUT1"=>"orange","INPUT2"=>"banana","INPUT3"=>"apple"
);

こんな感じで配列に代入して、それを引数で受けて関数内で展開して DB を更新
すると言った処理ではダメでしょうか？最初から INPUT の数を特定できなければ
array_push 関数で無くなるまで入れていくという感じになると思いますが。
　といっても php を使ったことがないので確信は持てませんが(汗
