> 現在oracle→postgresqlへの移行のテスト中です。
> Postgresql6.5.3で
> 
> create table sample(
>   fld numeric(3.0),
>   primary key(fld)
> );
> を実行したところ
> ERROR: Can't find a default operator class for type 1700.
> が出てしまいます。

PostgreSQL では、numeric 項目に対して index が作成できないからでは
ないでしょうか。int4 などの場合は可能です。

PostgreSQL 7.0 だと、

  kgotoh=> create table sample (
  kgotoh(>   fld numeric(3,0),
  kgotoh(>   primary key(fld)
  kgotoh(> );
  NOTICE:  CREATE TABLE/PRIMARY KEY will create implicit index 'sample_pkey' for table 'sample'
  CREATE
  kgotoh=> \d sample
             Table "sample"
   Attribute |     Type     | Modifier 
  -----------+--------------+----------
   fld       | numeric(3,0) | not null
  Index: sample_pkey

というように可能です。

今から新しいものを作るのであれば、7.0 にしておいた方がよいのでは
ないかとも思いますが、これは各サイトの都合次第でしょうね。

> またoracle→postgresqlの対応(varchar2→varchar)等の情報or情報源を
> ご存知であれば教えてください。よろしくお願いします。

整理された情報ってあるのでしょうか？ あるのなら私も見てみたいです。
双方のマニュアルを参照して比べる… というのが基本ではないかと思います。
