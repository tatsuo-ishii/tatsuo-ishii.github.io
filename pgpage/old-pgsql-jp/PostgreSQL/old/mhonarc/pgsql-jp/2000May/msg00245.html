   http://www.psn.co.jp/PostgreSQL/pgbash/

◎ pgbashのバージョンアップ内容

 1. PostgreSQL 6.5および 7.0 の両方に対応 
 2. シグナル処理の変更
 3. SQLの判定を英小文字だけにしていたバグを修正 
 4. COPY TABLE()文が指定されるとpgbash独自のコピーを使用するように
　　変更。
 5. pgbash独自のCOPY文でエラーが発生した場合、エラーコードを参照可
    能に変更
 6. データベースの接続ごとに client_encodingを保持できるように改良
 7. PSprint()におけるclient_encodingの処理方法を 7.0に合わせて改良
 8. pgbashrcの処理定義の中の "| more"を削除、プロンプト表示を変更
 9.ホスト名にIPアドレスを指定できるように変更 
10.データベース名一覧表示を 7.0のpsqlと同様に変更 
11.ヘルプメニューに "u" (ユーザ名一覧)　を追加 
12.ヘルプメニューの "v" はPostgreSQLとPgbashのバージョン表示に変更
13.その他、細かいミスを修正

◎ 6.5/7.0の互換性について

  pgbash-2.1は 6.5および7.0のバックエンドと接続できます。

  pgbash-2.1(+ libpq-6.5) <====> 6.5/7.0 backend
  pgbash-2.1(+ libpq-7.0) <====> 6.5/7.0 backend

　また、pgbash-2.1は non-MULTIBYTE(non-MB) および MULTIBYTE
  (MB)のバックエンドと接続できます。

　pgbash-2.1(+ non-MB-libpq) <==> non-MB/MB backend
  pgbash-2.1(+ MB-libpq)     <==> non-MB/MB backend

  但し、pgbash-2.1(+ MB-libpq) <==> non-MB-6.5 backend の交信
  はpgbash-2.1を使用する側で export PGCLIENTENCODING='SQL_ASCII'
  のように PGCLIENTENCODING を必ず指定しなければなりません。

◎ コピー機能について

  COPY table {TO|FROM} {'filename'|STDIN|STDOUT}..
  の形式は、PostgreSQL標準機能のコピーを使用し、

  COPY table( [ col1,.. ] ) {TO|FROM} {'filename'|STDIN|STDOUT}..
  の形式は、pgbash独自のコピーを使用するように変更しました。
