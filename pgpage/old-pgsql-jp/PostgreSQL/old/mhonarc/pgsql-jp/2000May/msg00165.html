  単純に file descriptor が足りないだけのように思えますが、OS には何を

  CPU、メモリ等の他のリソースにまだ余裕があるならば 45回/分というアク
セスは PostgreSQL の限界にはまだまだ程遠いと思います。とりあえず
kernel の再構築を行って、使用できる file descriptor の上限を上げてみる
のが先決だと思います。
