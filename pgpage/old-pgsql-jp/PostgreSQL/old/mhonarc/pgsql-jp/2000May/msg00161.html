識者の皆さんに是非ともアドバイスをお願いしたく思っております。

＜問題＞
apacheのaccess_logをapache_loggingでPostgreSQL7.0に取り込もうとしていますが

/usr/local/pgsql/bin/psql : error in loading shared libraries:
libpq.so.2.1: cannot open shared object file :Mo such file or directory

とapacheのerror_logに出てうまくいきません。

＜環境＞
RedHat6.1J
Postgresql7.0
Aｐache1.3.12
PHP4.0RC2
pdflib-3.0

＜環境の設定＞
Postgresql7.0のmake、make install、apache1.3.12をDSO（
./configure --enable-modules=so)で作り、 php4.0RC2(./config
--with-pgsql=/usr/local/pgsql　--with-apxs=/usr/local/apache/bin/apxs等)
を入れました。
その後、
httpd.confにLoadFile　/usr/local/pgsql/lib/libpq.soを加え、
/etc/ld.so.confに
/usr/local/pgsql/lib
を追加し、#ldconfig
を実行しています。

apache、Postgresとも起動し動いているように見えます。
PostgreSQL7.0でもCreateDBもでき、psqlでselectなども構築した環境ではできま
す。Apacheも起動しブラウザからもindex.phpは見れます。
ところが上記のようにapache_logiingでapache.sql、httpdconf.txtの内容をコピー
してapacheをリスタートした所、エラー状態となり、ソースを見てもどこが関係
しているのかわからず、うーんと言ったきりで解決の糸口が見えず困っておりま
す。

解決策の糸口やヒント等頂戴できれば幸いです。どうか、よろしくお願いします。
