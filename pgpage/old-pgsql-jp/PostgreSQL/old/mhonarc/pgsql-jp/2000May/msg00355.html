[pgsql-jp 14644] PostgreSQL7.0におけるinitdbのエラーについて

  • From: Hideo Inada <inada@formax.gr.jp>
  • Date: Sat, 20 May 2000 17:16:02 +0900
永らく購読していつも勉強させて頂いています
INADA@京都と申します。
社内の販売管理システムを
Laser5RedHat6.0 & PHP3.0.15-i18-ja & Apache1.3.12 & PostgreSQL6.5.3 
の環境で開発中です。
昨今PostgreSQL7.0がリリースされたことからPostgresqlのアップグレードをしよう 
とPostgreSQL7.0をインストールしましたところ、initdbでエラーが出ます。
PostgreSQL 6.5.3 のプロセスはkillしてから通常どおり postgres から

postgresql-7.0/src から
$ ./configure --enable-multibyte=EUC_JP
$ make
$ make install
$ cd ../doc
$ make install
$ initdb
としたところで　下記のようなエラーが出ます。
initdb does not find the file '/usr/local/pgsql/local1_template1.bki.source'.
This means you have a corrupted installation or identified the
wrong directory with the --pglib invocation option.
そこで前のデータはpg_dumpで退避しておいてから
/usr/local/pgsql を リムーブしたり /usr/local/src/ 
に残っている旧バージョンのpostgresql6.5.3ディレクトリ以下を削除したりしてから
再度
$ ./configure --enable-multibyte=EUC_JP
$ make
$ make install
$ cd ../doc
$ make install
$ initdb
したりと再三インストールを試みましたが、結果同様なエラーメッセージが出力され 
て進めません。
過去メールリストやFAQを確認しましたが、リリースされて間もないことから旧バー 
ジョンについてはありましたのでそれを参考に上記対応をした上でこのメーリングリ 
ストに投稿させていただきました。　6.5.3の時にはこのような事は無かったのに思 
いながらあせっています。
よろしくご指導のほどお願いいたします。
