[pgsql-jp 14838] PostgresSQL7.0インストールでのApache不稼動につて

  • From: "Masashi Ikehata" <ikehata@fujimic.fujisankei-g.co.jp>
  • Date: Sun, 28 May 2000 14:20:33 +0900
はじめまして、池畑と申します。

標記の件でご質問させて下さい。

現在：Linux/Apache1.3.12/Postgresql6.5.3/php-4.0RC1
で正常稼動しております。

そこで、Postgresqlをpostgresql-7.0RC1にバージョンアップさせ
（postgresqlは正常稼動）たところApacheが以下メッセージを出し
起動しません。

+---メッセージ---
Syntax error on line 207 of /usr/local/apache/conf/httpd.conf:
Cannot load /usr/local/apache/libexec/libphp4.so into server: libpq.so.2.0:
cann
ot open shared object file: No such file or directory
/usr/local/apache/bin/apachectl start: httpd could not be started
------+

実際、libpq.so.2.0はpostgresql-7.0RC1の/libには”libpq.so.2.0”はなく
（以前のバージョンにはあったが...）”libpq.so.2.1”が存在してます。

Apache/phpとも再度make installしなおしてはいるのですが、
なぜ、”libphp4.so”は”libpq.so.2.1”ではなく”libpq.so.2.0”を見に
行くのか、このあたり御力添えいただければ幸いです。
