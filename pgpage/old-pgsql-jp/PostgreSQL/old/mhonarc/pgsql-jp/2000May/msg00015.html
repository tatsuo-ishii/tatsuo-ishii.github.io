いまいち質問の意味が理解できないんですが、もし、「pg_connect() や

pg_exec() といった php の関数は、戻り値ないし、なんらかのステータスを出力

するのでしょうか？」という意味のことでしたら、以下のようにフォローします。

まず pg_connect() や pg_exec() などの PostgreSQL 用関数に限らず、あらゆる

php 関数は、戻り値を返すものもあるし、ないものもあります。これらの戻り値に

よって、分岐処理していくことが可能です。これら関数のエラーだけでなく、文法

上のエラーも php は出力してくれます。php.ini ファイルによってこれらを無効

にしたり有効にしたりすることもできます。ちなみに僕が

「[pgsql-jp 14292] [Q]データベース障害に備えて」で例示した php スクリプト

では、pg_connect() に、@ をつけて @pg_connect() としていますが、これは

php.ini ファイルの設定で php のエラー出力を有効にした場合、本来なら

pg_connect() が 接続に失敗して戻り値 FALSE を返した時、エラー出力してくれ

る(Unable to connect to Database のような)のですが、@ をつけることによって

この出力を抑制させることができます。なお、これらのことは、石井氏の著書にも

書いておられるし、php のマニュアルにも記載されています。
