確かに DBMS としての基本機能としては同じであるように見えます。
PostgreSQL 開発陣の本音は知りませんが、見掛け上は Oracle をひとつの
目標というか、対比するものとして扱っている気がしないでもありませんし。

伝聞なので、私もどこまで正確なのかは確信がありませんが、(^^; US Oracle
とかヨーロッパでは、DBMS の単体売り… 彼らの言い方だとライセンス商売…
になるんですかね？ から、Oracle Applications によるシステム構築ビジネス
やサポート関係のビジネスに完全にシフトしてるんだそうです。

日本オラクルは、国内でもそうしていきたいと考えているようです。
彼らの行なうセミナーでも、しきりにそういった話が出て来ますしね。

Oracle Applications ってのは、いわゆる ERP 製品ですな。使った事が
ないので名前しか知らないけど。(^^;
ERP では高いシェアを確保しているという SAP も DB エンジンとしては
Oracle を使うケースも多いようです。

そうした、トータルビジネスソリューション(横文字ばかり並ぶと意味不明に
なってくるな ^^;)の提供が彼らのビジネスモデルであって、DB エンジンは
それらを構成するひとつのコンポーネントだという事になるんでしょう。

Miracle Linux を使うシステムでこれをやれば、OS もミドルウェアも DB も
丸抱えで提供する事になるな。MiracleLinux は別会社だけど、ま、似たような
もんだ。と、外部からは見える。:-)
製品を売るだけではなく、各種の支援を行なう体制を用意して、そこでビジネス
… 要するに儲けていく。という事なのではないかと思います。

# Miracle の商売がうまく展開するかどうかは、これからでしょうけど。

そのあたりが池田さんがいう「ビジネスモデルが違うから」って事なの
でしょう。

それでも DB エンジンを自分で押えているというのが、彼らの強みである
事には違いないでしょうし、PostgreSQL をかついで上記のようなビジネスを
展開するところが出てくれば、ビジネスモデルとしては競合する事になります。

そういう意味では、「本当に競合しないのか？」という、おごちゃんの
意見も理解できます。
特におごちゃんは、そういう部分で仕事をなさっているようですので、
そのような感じ方が私などより強いのかも知れません。
