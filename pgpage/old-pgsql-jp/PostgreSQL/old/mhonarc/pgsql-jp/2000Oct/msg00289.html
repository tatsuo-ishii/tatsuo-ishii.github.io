PostgreSQL 7.0 から timestamp と datetime の名前が入れ替わっています。
これは、PostgreSQL 6.5.3 までの datetime が SQL92 の timestamp と
ほぼ同じなので、ユーザの混乱を避けるためにこうしたのでしょう。

これは PostgreSQL 7.0 以降ののリリースノートや HISTORY にも書いて
ある事ですし、石井さんの Web ページにも書かれています。

雑誌、書籍、Web ページなどが前提にしているバージョンと、自分の
使っているバージョンが少しでも異なれば、そこにかかれている事が
そのまま適用できるとは限りません。

こういうのは PostgreSQL に限らず、何だって同じです。
