Hosokawa Tetsuichi さん wrote:
> 
> create table ABC(a serial....)
> 
> としたときに、明示的ではありませんが、abc_a_seq という
> sequence が同時に作成されています。
> これの currval あたりをとってあげると良いかと。
> 
> とはいえ、これは複数実行時に問題おおありなので、そのあたり
> は考えてちょうだいね。

具体的にどういう問題点があるのでしょうか？
currval の値はセッション単位で保持されているようなので、
別セッションで insert を同時に発生させても、currval の
値は自分が insert したレコードの値が返ると思うのですが。
試しに psql を２つ起動した状態で、

    1. psql(1) で insert
    2. psql(1) で currval 表示
    3. psql(2) で insert
    4. psql(2) で currval 表示
    5. psql(1) で currval 表示

を実行してみたところ、2 と 5 では同じ値が表示されました。
（FreeBSD 3.1R + PostgreSQL 6.5.3）

このことではなく、もっと別の点で問題があるのでしょうか？
