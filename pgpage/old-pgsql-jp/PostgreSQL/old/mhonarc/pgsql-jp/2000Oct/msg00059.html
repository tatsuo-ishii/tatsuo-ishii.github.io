>   最近、ちょっと話題になっているのですが、"PostgreSQLはライバル
> にならないか" とういう見出しのミラクル・リナックスへのインタビュー
> 記事が日刊アスキーLinuxに出てました。

あれに書いてあるのは池田さんがいつも言っている事であって、特に(彼や、
彼の話を聞いた事のある者にとっては)目新しい主張ではありません。
それに、概ね正論ですしね。

現在の PostgreSQL は様々な意味において、Oracle のライバル… 対抗馬に
なるものではないというのが正直なところでしょう。

ユーザの現場にしろ、SI などを行なう会社にしろ、現時点で Oracle を
使うか PostgreSQL を使うか比較した結果、PostgreSQL を選ぶというのは、
ようするに poor man's choice をしたという事でしょう。

# 用途によっては、ライセンス料金が不要という点が決定的になるかも
# しれませんが。何かに組み込んでの販売を考える時は、結構決定的かも。
#
# でも、数千万円や億の値段をつけても売れるものなら、Oracle のライセンス
# 料金も問題にならないケースもあるだろうから、所詮は相対的な問題かな。

もちろん、それ以外の要素を重視して採用する人もいるでしょうが、世間
相場的にはそんなもんじゃないですかね。

PostgreSQL 開発陣が今後どのような方向を目指すかには興味がありますが、
何となく重装備化を指向しているような気がしないでもありません。

それはそれでいいんですが、機能的な実装の充実度が増していったとしても、
Oracle でいう Workgroup Server と対抗できるかどうかというところでは
ないかと私は感じています。
        
Oracle の商売の主体と主眼は Enterprise Server の方にあるようですから、
「PostgreSQL はライバルではない」というのは、彼らの正直な見解であると
私は思います。

つまり、言い方を変えれば、poor man's choice を行なう/行なえる人は
最初から彼らの(メインの)顧客層ではないのではないでしょうか？ (^_^)

遠い将来はわかりませんが、近い将来ではそんなものではないかと。

>   Oracle DBに対して PostgreSQLはライバルではないとしてますが、
> LinuxのカーネルまでハックしてLinuxビジネスを引っ張ってゆこうとし
> ているミラクルにとって、PostgreSQLのサポートってそんなに大変なも
> のなのかなと思ってしまいました。

ここは、くわむらさんが何を主張しようとしているのかよくわからないん
ですが、(^^; MiracleLinux 社は Linux 版 Oracle を売るための会社
なんですから、そこで PostgreSQL のサポートを行なう意味はないと
思うんですが…
