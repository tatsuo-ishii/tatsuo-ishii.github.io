さすれば、P1 で INSERT 後、P2 で INSERT し、(再び) P1 で SELECT curval()
しても、 P2 で INSERT した値でなく、P1 で INSERT した時の値を保持している
ので、証明になりますね。
ここで、P1 で 2度目の SELECT curval() が、 P2 で実行した INSERT の値を表
示してしまっていては、懸念されている「INSERT と SELECT の間に別の INSERT
が発生した時に・・・」が現実化してしまうわけですね。

テーブルロックしたとしたら、P2 で INSERT が出来ないですから、それは無いで
しょう。
