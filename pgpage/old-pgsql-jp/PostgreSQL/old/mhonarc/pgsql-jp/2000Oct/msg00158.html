At Fri, 13 Oct 2000 12:29:21 +0900,
SaitoMasaru <daisaito@lares.dti.ne.jp> wrote:
> create tableでシリアルを作成すると、
> [テーブルの名_シリアルのカラム名_SEQ]でsequenceが作成されますけど、
> このsequenceの名前がNAMEDATALEN(32文字)を越えてしまっていた場合、
> このsequenceの値を再設定したりということは不可能でしょうか？
> 
> そもそも、NAMEDATALEN(32文字)を越えたsequenceを作ってもエラーが発生せず、
> その後、SQLでアクセスが不可という事で困っているのですが。

  32文字以上の識別子で create sequence してみましたが、

koyama=> create sequnece aaaaabbbbbcccccdddddeeeeefffffggggghhhhh;
NOTICE:  identifier "aaaaabbbbbcccccdddddeeeeefffffggggghh
hhh" will be truncated to "aaaaabbbbbcccccdddddeeeeefffffg"
CREATE

きちんとケツが切れた名前で create されましたし、

koyama=> drop sequnece aaaaabbbbbcccccdddddeeeeefffffggggghhhhh;
NOTICE:  identifier "aaaaabbbbbcccccdddddeeeeefffffggggghhhh
h" will be truncated to "aaaaabbbbbcccccdddddeeeeefffffg"
DROP

drop も同様に可能でした。このケツが切れた名前に対して setval もちゃん
と動作しましたが、「アクセスが不可」とは具体的にどういう状況になるので
しょうか?

  こちらは PosgreSQL 7.0.2 でテストしています。
