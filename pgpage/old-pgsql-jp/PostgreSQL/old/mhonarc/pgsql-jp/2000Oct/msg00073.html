このスレッドも結構続きましたが、あらためて元記事に Reply します。

まぁ何と言うか、「皆さん、Oracle に PostgreSQL をライバル視して
欲しいですか？」ってな見方もあるんじゃないかと。

US Oracle の本音はよく知りませんが、日本オラクルの方(元… の方も含む)と
話をしていると、一言で言えば「商売の対象や方法論が違うからね」という点に
話が集約されます。
日本オラクルの方達によると、US Oracle も同様の考えのようです。

って、いうよりも US Oracle の方がより鮮明なのかも知れません。
日本オラクルの人達と話をしていると、「Workgroup Server クラスの製品が
売れている(数が出ている)のは日本だけだ」という話がよく出ます。

これは、DBMS だけでなく、Application Server などでも同様なのだ
そうです。

欧米と日本とでなぜ違う傾向が出るかってのは、分析して見ると色々と面白い
のではないかと思うんですが、個人的な独断で言えば、日本人の悪い意味での
ブランド信仰が出ているんじゃないかと思います。
または、変なお上依存意識ってヤツでしょうか。

DBMS にしろ、Application Server にしろ、Workgroup Server クラスになると
本来は選択肢が色々あるはずなのです。

Informix SE でもいいし、PostgreSQL や MySQL でもよいかもしれない。
Application Server 的なものが欲しければ、実は Free またはそれに近い
ものも色々あるし、JServ, Tomcat でじゅうぶんなのかも知れない。

ひょっとしたら、欧米人の方がこの辺りの割り切りや棲み分けを意識して
いるのに、我々は「とりあえず Oracle だよね」でお茶を濁していて、
誰かに「PostgreSQL は Oracle と置き換えできるね」と言って欲しい
だけなんじゃないの？ とかね。:-)
「PostgreSQL はライバルにならない」ってのがキーワードとして広まると、
それに自分で反論できないんで過剰反応してるだけとか。;-)

まぁ、私も Oralce Workgroup Server 使ってるんで、あまり偉そうな事言うと
天に唾する行為なんですが。(^^;

まぁ、考えようによっては、どっかのバカ会社みたいに PDA でも、ノート PC
でも、デスクトップでも、部門サーバでも、企業サーバでも、何でもかんでも
ボクんとこの製品使ってくれなきゃやだい！
みたいなのよりは、まだマシなんじゃないかと。;-p

気楽に考えりゃいいじゃん。っていうふうには思えませんか？

せっかく向こう様が「競合しない」と言ってるんですから、それに目を
三角にする理由は何もないと思います。
彼は彼、我には我の道がある。ってとこでいいんじゃないでしょうか。
