> postgres7.0.2にはphp3.0.15ではアクセス出来ないのでしょうか？

問題無く動いています。

OS が何かを書いてませんけど、rpm 云々というところを見ると、
Linux のいずれかのディストリビューションなんでしょうね。

PostgreSQL をどうインストールしたのかもわからないけど、rpm
によるバイナリインストールは、rpm パッケージ同士の依存性を
解決しないとインストールできません。

あなたが *どうしても* rpm パッケージでインストールしたいので
あれば、必要な rpm パッケージ群を手元に集めて、順序よくインストール
しなければなりません。

それから、ここ pgsql-jp や PHP-jp などで「rpm でうまくインストール
できません」というのをかなり頻繁に見掛けるようになってきていますが、
rpm 云々という話しは PostgreSQL にも PHP にも関係がありません。

そういうのは、当該 Linux ディストリビューションを出している
ディストリビュータに聞いて下さい。
何でディストリビュータとか、ディストリビューションに関連する掲示版や
 ML などで聞かずに、こういうところで聞こうとするんだろ？ 不思議だ。

PostgreSQL にしろ、PHP にしろ、すべてをソースから自分で build すれば
何も問題無く作る事ができます。

rpm パッケージのようなバイナリインストールは、便利に使える事も
あるのですが、個人的には PostgreSQL や PHP のように build 時の
オプションが色々あったり組み込むものが色々あるものは、ソースから
build した方が話が簡単だと思ってます。
