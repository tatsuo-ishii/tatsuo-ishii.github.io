大場です。

Akio Umiuchi wrote;

>pgaccess ではテーブルの作成、フィールドの追加変更等はできるのでしようか？
>データの入出力はＯＤＢＣ経由でＷｉｎｄｏｗｓで行えば出来るのですが、簡易的に
>データベースのテーブル内のフィールドを追加修正削除が出来ると楽なんですが・・
>・・

テーブル名の変更、カラムの追加、カラム名の変更は
おそらくalter tableでできると思うのですが。
「(テーブル名の変更に注が書いてあって)
　バグがあるので使わないほうがいい」と
シーラカンス本にありますけど。

ただ、カラム削除、制約、デフォルト値の変更は
PostgreSQLではサポートされてないとあるので

「できないこととできることが
　あってできることでもバグがある」

ですね。テーブルを直接いじる場合はデータをセーブしてから
drop table -> create table ->copy
としなさいとあります。

自由に変更ができると便利だとは思うけど

＃「うわーカラム追加だよT_T」って経験もあるし(^^;)

行き当たりばったりで追加とかすると破綻するのは
目に見えてるので、初めのテーブル設計の段階で
丁寧に考えて必要なものを洗い出してきっちりした方が
後々いいってことですね。
多分、その方が良い仕事できると思う。

＃人のシグネチャまで含んだ無駄な引用は止めて欲しい
