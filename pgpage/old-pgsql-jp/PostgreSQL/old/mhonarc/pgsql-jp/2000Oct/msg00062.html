おごちゃんです。

> ユーザの現場にしろ、SI などを行なう会社にしろ、現時点で Oracle を
> 使うか PostgreSQL を使うか比較した結果、PostgreSQL を選ぶというのは、
> ようするに poor man's choice をしたという事でしょう。

  今やってるシステムがほぼPostgresを採用することにしています(もしかした
ら、InterBase 6を使うかも知れない)。規模は非常に大きいです。万のオーダの
ホストがつながります。

  で、これくらいになるとやはりDBMSのライセンス料は、金があるのないのとい
う問題を超えて重くなって来ます。つまりは

> # 用途によっては、ライセンス料金が不要という点が決定的になるかも
> # しれませんが。何かに組み込んでの販売を考える時は、結構決定的かも。

  とゆーことです。

  これくらい大きなSIなると、Oracleを買う金でPostgresを改良するチームを組
むことも不可能ではなくなります。となると、

> Oracle の商売の主体と主眼は Enterprise Server の方にあるようですから、
> 「PostgreSQL はライバルではない」というのは、彼らの正直な見解であると
> 私は思います。

  うんと大きなシステムの場合は、やはりOracleはPostgresの敵ではなくなるわ
けです。まぁその「中間」ってのが非常にデカいわけなので、当面Oracleは安泰
なんでしょうが。

# Postgresのstorage部はかなりタコいと思うのだけど、これの改良予定はない
# のかな？
