そうでしょうか。RPMの依存関係も解消できないような人は、makeで
エラーが出たらもっとお手上げですよ。
多分、Configureに何らかのオプションを与えることすらできなくて、
きっとこのMLで聞きます:p

昔からUNIXを使っている人は、すべて自分でmakeしない気がすまない
原理主義者が多いですけれどね。そういう人は別として、RPMがある
ものは使ってほしいというのが私の考えです。
後藤さんがRPMの話題でうんざりするように、私も、新しいバージョン
が出る度に繰り返される、makeできないとか、レグレッションテスト
でエラーが出るのだけれど、という話題にうんざりなんですよ。

# makeすら自分でできない人は、RPMが出るまで待てと。
# だってそんな人がベータ版とかに飛び付いたってしようがない
# ですもん。

> rpm パッケージのようなバイナリインストールは、便利に使える事も
> あるのですが、個人的には PostgreSQL や PHP のように build 時の
> オプションが色々あったり組み込むものが色々あるものは、ソースから
> build した方が話が簡単だと思ってます。
> 
RPMのインストールができない人に、build時に適切なオプションが
指定できるかしらん。
（そういう人は、自分で情報を収集するという能力が欠如しています
  から）

少なくとも最近は、PostgreSQLやらApacheやらは、RPMそのままで何の
問題もないですよ。PHPは使ったことないですけれど。
build時になんやかんや指定しなくてはいけないようなアプリケーション
はアーキテクチャが時代遅れですよね。後からパラメタを設定ファイルに
定義できるようになっていればほとんどの場合、O.Kのはずです。
SAMBAのSWATみたいなのがあればもっと良いし。
