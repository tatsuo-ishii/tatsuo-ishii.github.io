「どの program が error を出力しているのか？」ということが分からないと、
なんとも言えません。"Parse Error" という error message は、PHP,
PostgreSQL の両方が出力しうるからです。したがって、他の方からすると、
正直なところ "Error が出ているらしい" ということしか把握できません。

個人的には、まず error message を再度御検討されてみることをお勧めしま
す。その上で、なお解決が出来ないようでしたら、行ないことと、error
message そのままを *正確に* に記述してみると、良いかと思います。

> かけても返ってきます。ＰＨＰ３のソースではコネクトまでは
> 正常ですがＰＧ＿ＥＸＥＣ（）；で上記エラーがでます。
> ＊中に設定するＳＱＬは下記のとおりです＊
> ”ＳＥＬＥＣＴ　＊　ＦＲＯＭ　表名”これだけです。

ただ、状況を考えると、おそらく PHP の source のどこか、それも 
PostgreSQL と connect した後の部分に、error があるのでしょう。たとえば、
";" が抜けている、')' が足りない、色々可能性を考えることができます。ま
ずは、source を御検証されてみるのが良いかと…。
