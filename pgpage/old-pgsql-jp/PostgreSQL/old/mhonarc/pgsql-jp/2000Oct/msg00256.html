[pgsql-jp 18212] 日本語フィールドのorder by でのトラブル

  • From: Shinichi Takase <takase@tsoffice.co.jp>
  • Date: Wed, 18 Oct 2000 11:46:01 +0900
はじめまして、高瀬と申します。

Redhat6.2 + postgres7.02で、テーブルを作成しpsqlで
select文を発行したのですが、日本語フィールド対して
order by句を付けると正しくソートされません。

具体的には、各レコードにア、イ、ウ、エ、オ、タ、チ、
ツ、テ、トと入力しselect文を発行すると、タ、エ、カ、
イ、ア、ウ、オ、チ、ツ、テ、トと並び替えられます。
内部コードはEUC_JPです。

同じフィールドに対してLIKE検索を実施すると正しい返
るし、フィールド名に日本語を付けても正しく動くので
マルチバイト処理は正しいように思います。

実行したSQL文は次のとおりです。

[テーブルの作成]
CREATE TABLE dbcs (カナ char(50));

[データの登録]
INSERT INTO dbcs (カナ) values('ア');
INSERT INTO dbcs (カナ) values('イ');
　　　　：
INSERT INTO dbcs (カナ) values('テ');
INSERT INTO dbcs (カナ) values('ト');

[データの選択]
SELECT カナ FROM dbcs ORDER BY カナ;

何が原因なのか、おわかりになる方がいらしたらお助け
ください。
