市川＠お茶大です．

i386_solaris, sparc_solaris, hpux と3つで PostgreSQL-6.1 (jp)
のインストールをしています．

hpux でサーバが走らないのはともかくとしても，i386_solaris と
sparc_solaris の間でのやりとりがうまくいっていないようです．

まず兆候ですが，とりあえず接続はできます．また，
create table, insert などの SQL 文は実行できます．
ところが，select 文の実行がうまくいきません．（当然 \d
も駄目です．）

現在試した範囲では，

	client		server		status
	-----------------------------------
	i386_solaris	sparc_solaris	fail
	hpux		sparc_solaris	ok
	sparc_solaris	i386_solaris	fail
	hpux		i386_solaris	fail

となって，どうも，i386_solaris とその他との通信がおかしく
なっているようです．

なお，endian.patch と当たっております．i386_solaris については
register を disable するためのpatch が当たっています．

何か情報をお持ちのかたございましたら御教示願います．
