大手のホスティングサービスの DB のほとんどは MySQL ですね。
自分が今使っているところ（Verio/Rapidsite）も MySQL です。

SQL をデータストアとして手軽に使いたい、ただしトランザクションや
行単位ロックを必要とするような本格的なシステムではない、という
システムで使われるのであろうと想像しています。

PHPLIB や phphoo (PHP で Yahoo! 風のページを作る) や Midgard も
デフォルトのターゲットは MySQL ですし、O'Reilly から本も
出ているし（"mSQL & MySQL"）、海外では意外なほど普及している
ように見えます。
# 逆に日本ではあまり雑誌記事や書籍を見ませんね。皆無では
　ありませんが。

もっとも普及しているからといって、PostgreSQL の商用 RDBMS に
匹敵する高度な機能の価値がなくなるわけではありません。
トランザクション、行単位ロック、PL/PgSQL、正規表現のサポート、
多様なデータ型、豊富な関数などは欠かせないというケースも多い
ですし、自分自身かなり便利に利用させていただいています。

ただ、MySQL の方が便利だな、と思ったのは、Windows 上で動作する
管理ツールがあることですね。これを使ってバックアップ・リストア、
システムカタログの操作（テーブル、ビューなどの追加・変更・削除）
ができます。
