> Makefile の中に、次のような記述があります。
> 
> # If you have problems with this rule, replace the $( ) with ` ` as some
> # shells (mainly sh under Solaris) doesn't recognise $( )
> 
> 過去に別件で似たような問題に出くわしてML に出した記憶があるのですが、
> どこでどうやって解決したか、出てきませんでした。
> ここにある通り、$(なんたら) のような箇所を`なんたら` に
> 書き直すんだったと思います。

どうも、なんでもかんでも$(なんたら)を`なんたら` に直すのでは
なさそうで、とりあえず以下の部分は必要そうです。

postgres@bash-2.02$ diff Makefile Makefile.org
30c30
<       make `$(JAVA) makeVersion`
---
>       make $$($(JAVA) makeVersion)
146c146
<       $(JAR) -c0f $@ `$(FIND) postgresql -name "*.class" -print` \
---
>       $(JAR) -c0f $@ $$($(FIND) postgresql -name "*.class" -print) \


これでmake は通るようですが、make corba が通りません。
idltojava というコマンドがないというエラーが出ます。

postgres@bash-2.02$ make corba
rm -rf stock
idltojava -fno-cpp -fno-tie example/corba/stock.idl
make: idltojava: Command not found
make: *** [example/corba/stock.built] Error 127
postgres@bash-2.02$

このidltojava が何かが不明ですが、自分の/usr/jdk1.2/bin をはじめとする
ディレクトリにはなさそうです。java-house ML の過去記事をちょっと
見たところ、「idltojavaコンパイラ」というものらしいですが、
どこにあるのでしょうか(インストールされているディレクトリ、
入手できるURLの両方とも今のところわかりません)。

環境
Solaris 2.6
PostgreSQL 6.5.2 (コンパイルを試しているバージョン)
Java は、以下の通りです。
postgres@bash-2.02$ java -version
java version "1.2.1"
Solaris VM (build Solaris_JDK_1.2.1_04, native threads, sunwjit)
