[pgsql-jp 13580] backend closed the channel unexpectedly が出ます

  • From: "Y. Fujimoto" <fujimoto@mxc.mesh.ne.jp>
  • Date: Tue, 21 Mar 2000 21:16:48 +0900
PostgreSQL-ML各位

　こんにちは。
　はじめまして。藤本と申します。

　LASER5 Linux 6.0 + PostgreSQL 6.5.3 + PHP 3.0.13 にてブラ
ウザから利用するＤＢシステムを作成しております。

　商品テーブルに対して、キーワードやリスト選択の条件から検索
を行う処理の部分で、「and」や「or」の多い検索条件を送ると、
表題の

「backend closed the channel unexpectedly.」

を出して、処理が停まってしまいます。

　ブラウザから実行しても、telnet端末から直接ＳＱＬを実行して
も結果は同様です。

　様々な検索条件を与えたい場面なので、select文のandやorの個
数はたびたび変わるのですが、一例としまして、この

kekeke=> select * from goods1_tb where 
kekeke-> (gds_serial like 'ERPOMS______') or 
kekeke-> (gds_serial like 'ERPOS4______') or 
kekeke-> (gds_serial like 'ERPOAM______') or 
kekeke-> (gds_serial like 'ERPOMU______') or 
kekeke-> (gds_serial like 'ERPO4M______') or 
kekeke-> (gds_serial like 'ERPO3M______') or 
kekeke-> (gds_serial like 'ERPOST______') or 
kekeke-> (gds_serial like 'ERPOCM______') or 
kekeke-> (gds_serial like 'ERPOP8______') or 
kekeke-> (gds_serial like 'ERPODM______');
pqReadData() -- backend closed the channel unexpectedly.
        This probably means the backend terminated abnormally
        before or while processing the request.
We have lost the connection to the backend, so further
processing is impossible.  Terminating.
[postgres@xxx postgres]$

の様に検索を行うと、停まってしまいます。

===>例の場合は「'ERPO...」の部分は皆同じですが、時には異なる
　　条件も入る場合があるもので、入れてあります。


　過去ログのNo9311以下のツリーを参考にさせていただいて、

/usr/local/pgsql/data/pg_geqo

を作成し、chmodして、

Generations   20

としてみたのですが、同様の結果です。

　稼働マシンは、Pentium120-48MBのノートマシンで、ウィンドウ
システムは使用していません。

　orやandの個数が多過ぎるのでしょうか？。

　どなたかご存知の方、ご教授お願いいたします。
