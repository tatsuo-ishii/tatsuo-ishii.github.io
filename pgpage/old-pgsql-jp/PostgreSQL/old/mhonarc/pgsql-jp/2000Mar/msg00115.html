自分も佐伯様とおなじ所でひっかかって、散々過去ログを
検索した（つもり）なのですが、使っているOSのせいなのか？
類似ケースはヒットするものの、ズバリ解決に結びつくログが見つからず、
なかなかコンパイルが通らず、困りました。結局、
神谷様の投稿に救われ、やっと昨日コンパイルまで終わりました。
もしかしたら、今後なにかの役に立つかとも思い、
自分のケースを投稿させて頂きます。

ちなみに自分は、
NetBSD1.4 on Macintosh Centris660AVの環境に、
Postgres-6.5.2をインストールしています。

引用が少し前後しますが、お許し下さい。

>>TurboLinux4.5にPostgres-6.5.2をインストールしています。
>>インストール中にエラー（？）が発生し先に進めなくなってしまいました。
>>過去ログを見たのですが同じような内容はなかったです。
>>ご存知の方いらっしゃいましたらよろしくお願いします。
>>
>>◆./configure --with-mb=EUC_JPを実行。
>
>configureスクリプトが正常に終了していないのではないでしょうか？
>GNUmakefileが、src/に有りますか？

このご回答は過去ログにもなんども出てきているようです。
で、自分の場合は、きちんとGNUmakefileができてるんですが、

>>◆make を実行すると
>>You must use GNU make to use Postgres.  It may be installed
>>on your system with the name 'gmake'.
>>
>>NOTE:  If you are sure that you are using GNU make and you are
>>       still getting this message, you may simply need to run
>>       the configure program.
>>となってしまい、gmakeを実行しても同じです。

↑と同じ結果が返ってきました。で、どうやって解決したかというと、

http://www.gnu.org

からたどって、makeの新しいバージョンをとってきて、そちらで
$ make all
すると、きちんとmakeできました。終わってみればとても単純な事ですが。
そこで、次に、
$ make install
すると、なんと6時間くらい時間を食った挙げ句エラーが２つ返り、
インストールできなかったんです（涙）。

そこで、もしや、と思い、src/以下すべて一度削除し、ソースの展開
からやりなおして、configureを、

>FAQですが、
>$ ./configure --with-mb=EUC_JP --without-CXX
>ではどうでしょうか？

こちらのほうで試したあと、再度
$ make all
$ make install
するとコンパイルがなんとかできました。


自分はまだまだ未熟ですので、もしかするとmakeとかconfigure以前に、
BSD自体の設定がうまくいってなくて、それが原因かもしれませんが、
一応、事例の報告という事で・・・。

以上、長文失礼いたしました。
