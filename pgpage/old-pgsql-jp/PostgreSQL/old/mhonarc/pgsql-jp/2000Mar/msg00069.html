[pgsql-jp 13358] pgbash script が、CTRL-C で実行中止できない。

  • From: Naofumi Kondoh <nkon@shonan.ne.jp>
  • Date: Fri, 03 Mar 2000 18:47:43 +0900
ソフト工房の近藤です。

デバッグや臨時処理に、pgbash を使って大変重宝しています。
多謝  ＞＞ 境田 さん。

ただ、SQL を実行していないスクリプト行で、CTRL+C ( ^C )
押下で、スクリプトを中断しようとしても、次のメッセ−ジ
がでるだけで、スクリプトの実行をとめられません。

>>> CANCEL request sent >>>

これは、仕様なのかもしれませんが、臨時処理でちょこっと
使うときには、不便なので、何か対策がありましたらご教授
ください。

今は、しかたないので、kterm ごと強制終了させてます。
本処理で使うときは、^C 強制中断なんてしないし、必要
なら、ちゃんと、YES/NO などで、文字判定するから問題
ないのですが、pgbash は、デバッグなど臨時処理にも重
宝なので残念。

# SQL の実行を ^C で、cancel できるのは、便利なんですが。

PGBASH version 2.0.1 (bash-2.03 + exec_sql-2.0.1 です。


簡単なテスト例

#!/usr/local/bin/pgbash
connect to xxdb;
for tab in test1 test2 test3 test4 
do
    select * from $tab ;         
    echo -n ".... continue ? "; read dummy
    ## 。。。。いろいろな処理。。。
done
