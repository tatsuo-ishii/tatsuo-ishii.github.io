、、、で、行ってみたところ、ダウンロードにはユーザ名とパスワード
の登録が必要とのこと。あれ、前に登録したのは別のページだったか、
と思いつつ、また登録してダウンロードのページへ。
　なんかftp かhttp かの選択があり。先日、jdk1_2_2-001 ではずい
ぶんひどい目に合わせてもらいました。18.7MB もあるjdk1_2_2-001-solsparc.sh
をftp でひっぱってきたまではよかったのですが、sh に食わせてもなんか
エラー(よく覚えていない) 出まくりで、ファイルの中を見ると、各行末に
^M なんてついてます。どうやって取り除いたらいいかもわからず。さんざん
試行錯誤したあげく、ファイルを今度はhttp で取ってくると、すんなり
いきました。
　で、今回はファイル名がidltojava-ea-solaris.bin なんていう名前で、
最初、これがidltojava そのものかと思ってやってみたらエラーになり、
よくよく調べたら、これは自己解凍形式のようで、実行して
みたところ、idltojava と2つのファイルが出てきました。これをパスの
通っているディレクトリにコピーして、make corba が通りました。

postgres@bash-2.02$ make corba
rm -rf stock
idltojava -fno-cpp -fno-tie example/corba/stock.idl
javac stock/*.java
Note: 2 files use or override a deprecated API.  Recompile with "-deprecation" f
or details.
1 warning
touch example/corba/stock.built
javac example/corba/StockClient.java
javac example/corba/StockDB.java
javac example/corba/StockDispenserImpl.java
javac example/corba/StockServer.java
-------------------------------------------------------
The corba example has been built. Before running, you
will need to read the example/corba/readme file on how
to run the example.
