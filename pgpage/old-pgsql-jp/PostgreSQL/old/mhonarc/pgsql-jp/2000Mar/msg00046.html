> SQL や DBMS 設定の説明などではなく、対象に即した DB 設計を解説した本を

> 
> C++: g++          <-> デザインパターンの本
> TeX: pTeX         <-> 論文の書き方解説書
> Cisco IOS         <-> ネットワーク設計の本
> SQL: PostgreSQL   <-> ???

「この技術をどう用いるか、どういう使う方をするとどういう成果が
得られるか」という内容の How to 本がほしいということでしょうか。

通称シーラカンス本と呼ばれる「PostgreSQL 完全攻略ガイド」はもう
お読みになられたのでしょうか。SQL とは何か、どう使うか、
PostgreSQL における SQL はどうなっているか等が例を交えて詳細に
解説されています。これ1冊でほとんどまかなえるのではないかと
思います。

あと、日本における PostgreSQL 総本山のサイトにも参考文献への
リファレンスが掲載されています。
http://www.sra.co.jp/people/t-ishii/postgres95/doc-jp/index.html
カテゴリとして "SQL" であればよくて、PostgreSQL に限定しないと
いうことであれば、「プログラマのための SQL」（セルコ著、
トッパン）が個人的にはかなりお勧めです。

PostgreSQL に特化した設計方法論（例えば、問題ドメインを分析する
ときに PostgreSQL の特徴を生かすような分析結果と設計の成果をあげる
ような設計方法論）の解説書、ということであれば、そういうものは
ちょっと見当たりません。あえて言えば Postgress の設計者の 
Stonebreaker 教授の書かれた「オブジェクトリレーショナル DBMS」
（和訳されています、出版社は忘れました）が一番近いのでしょうか？
でもこれはアーキテクチャの解説だからちょっと違うような気もします。

オブジェクト指向分析の結果をデータベースモデリングに生かしたいと
いうことであれば、ちょっと毛色の変わった本ですが、
『Oracle8 UML オブジェクトモデリング』翔泳社
というのが役に立つかもしれません。UML や E-R 図を用いて分析し、
モデル化した結果を、Oracle8{i} + Object Option を用いたシステムに
どう反映させるかがテーマです。ただ、それにしても PostgreSQL の
実現している "Object-Relational" DBMS と Oracle の実現している
"Object-Relational" DBMS の仕組みの間には相当のインピーダンス
ギャップがあるので、あくまで「参考になるかもしれない」という
程度ですが。

あまり内容のない返事ですみません。
