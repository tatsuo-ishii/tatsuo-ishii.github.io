列数は、予め 100 列位用意しておくとか、足りなくなっ
たら、列数追加ボタンを SUBMIT とすると、指定列数分
(規定値 50位)増加させた HTML を再送するとかで如何で
しょう。
＃Vine1.1 で Netscape の INPUT TAG の VALUE= が消え
＃るという問題がなくなったので、Linux でも安心。

問題は、VIEW や REFERENCES で、既存の表の列名を参照
したい場合、JavaScrip併用で別ウィンドーを開いても、
レスポンスが悪くて使いにくいというとこです。

最初に関連表名を CHECKBOX で選んでおいて、全列名を
別窓に表示する方法もありますが、初心者向けだと、最
初に関連する表名を特定しにくいんではないかと。

まあ、ブラウザベースのツールなんでよくばりすぎても
しょうがないから、pg_dump ＜DB名＞ -s を見やすくし
たデーターを別のブラウザに表示して、列名を手入力す
るというあたりで妥協しようと思うのですがどうでしょう。
