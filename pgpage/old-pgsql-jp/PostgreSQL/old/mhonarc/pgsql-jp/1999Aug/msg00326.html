はい。確か2ヶ月ほど前、「私はperlから直接書いてますよ」って言った
田中です。

Pg.plという、Perlインターフェースを使います。

postgresql-6.x.x/src/interfaces/perl5/README

へ、Pg.plのインストール方法が明記してあります。

# PostgreSQLのインストール前なら、./configureの時に、
# --with-perlというオプションを付けてからmakeするのが
# 手っ取り早いらしい。


インストール後、実際にPerlにてプログラムを組む方法は、

postgresql-6.4.2/src/interfaces/perl5/blib/man3/Pg.3

にマニュアルがありますので、こちらを参照下さい。

# これって、本当はどこかにコピーされるべきファイルなのかな？
# 不精なもので、これを直接見ていたりします。



ところで、半角カナをテーブルへ書き込むのは感心しません。
半角カナを全角カナへ置き換えるロジックまでコーディング
して、処理してから、テーブルへの書き込みを行う方が賢明かと
思います。
不精な私は、データ登録アプリ側で既に、半角カナ入力があると
「半角カナはだめ」ってエラー出してます。変換面倒だから。


# 半角カナを書き込もうとしていると言うことは、フロントエンド
# 側を、SJISで処理しようとしているのでしょうか?
