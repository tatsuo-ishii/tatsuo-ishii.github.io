そういうことは最初に言って下さい。余計に皆さんの時間を浪費してしまいます。

>>         set client_encoding to 'SJIS';
>
>　を試してみたところ、
>
>≫testbase=> set client_encoding to 'SJIS';
>≫NOTICE:  Unrecognized variable client_encoding
>≫SET VARIABLE
>
>≫通知:承認されていない　変化
>
>　と表示されますので、このエンコードは承認されてないという事のよう

違います。"client_encoding" という機能自体を認識してません。つまり、お
使いのPostgreSQLは日本語(マルチバイト)対応されていない英語版だったわけ
です。

>　案の定、COPYでテキストを読み込んで、Accessにリンクしてみたら、カ
>ナ部分は意味不明の漢字の羅列に変換されてしまっていました。
>
>　半角カナがUNIX環境では嫌われるのは承知ですので、もしクライアント
>エンコーディングの変更が上手くいかないときは、半角→全角の変換も考
>慮しなければならないでしょうが、perlも始めたばかりの私にそんな大そ
>れたプログラムが書けるかどうか‥‥(--;)

結局COPYとか半角カナとかそれ以前の問題で、日本語対応ではないPostgreSQL
を使っているのが全ての原因と思われます。正しい解決法は、ソースからちゃ
んとマルチバイトオプションを付けてコンパイルし直すことです。
