石川です

->> "[pgsql-jp 9942] Re: PostgrSQLとの接続方法"で
->> 99/08/03 に Kazumasa Gotoh さんは書きました：

 >>しかし、「ブラウザを使う」ってのと JDBC が結びつかないんですが… ^^;
 >>Applet の場合はクライアント側にダウンロードして使う、つまり DB との
 >>通信はクライアントマシンと DB サーバマシンとの間でのやりとりになるので
 >>Web サーバは関係がなくなります。

  ブラウザを使うなら Applet を使うのだろうと思ったので JDBC をブラウザ
にロードして、それから使うんだろうと思ったのです。これをやると、Javaの
セキュリティのために、Applet のダウンロード元(つまり Webサーバ)と同じ
マシンにしかソケットで接続できなくなるので、WebサーバとDBサーバが同じ
でないと困るだろうね、ということです。

 >>それとも、Servlet でアプリケーションを作成するのを念頭に置いて
 >>らっしゃるのかな？ この場合でも先に書いた様に Servlet が動くマシンと
 >>DB サーバは別のマシンであっても構いません。

  そーですね。Servlet なら、これが動く(クライアントじゃない)マシンとDB
サーバは別のマシンでもOKだと思います。クライアントも Applet を読み込ん
だり Java を起動させたりする手間が省けてサクサクと動くかもしれません。

  接続と言っても、直接サーバに接続するだけじゃなくて、いろいろあるもん
ですね。
