真鍋です。

> 本題から離れて申しわけありませんが、 VIEW とか、
> CREATE TABLE 等々を ブラウザからできるツールを
> 開発中です。勿論フリーソフト。
> # 検索と集計の主要機能は作成済み公開中です。
>
> つきましては、ブラウザでどういう画面構成にしたら
> 作りやすいか、何かご意見いただけると嬉しいです。
>
> # C/S なら、対話的にすすめられて楽なんですが、
> # ブラウザにすると、１項目入力のつど DB 検索する
> # するのも問題で、指定した表の列名をどうやって
> # 表示させるかなどデザインが難しくて。

完成を楽しみにしています。
入力画面はテキストベースにして、あらかじめテンプレートを用意しておいたらどう
でしょうか。そうすれば、多少操作が難しくてもいいような気がします。それに、テ
ンプレートを動かしている間に操作を覚えられるし。
話をそらすようで申し訳ありませんが、VB6からもテーブルの操作ができました。一
応、新規追加や削除、更新ができます。テーブルの削除もできたので、開発環境に
WindowsのGUIを使うという逃げ道もあるような気がするのですが。個人的には、全て
Linuxでできたほうがうれしいです。
