DBD::PgPP v0.04をCPANにアップロードしました。DBD::PgPPは
PostgreSQLのFrontend/Backend間で交わされる通信プロトコルを
Perlだけで実装したDBIのドライバです。

  file: $CPAN/authors/id/O/OY/OYAMA/DBD-PgPP-0.04.tar.gz
  size: 13016 bytes
   md5: dc018542c194608f2ac92ce76c3fa192


DBD::PgPPはPostgreSQLのlibpgライブラリ等が無い環境でも、
PerlとIO::Socketモジュールが有れば基本的に(いや多分)何処で
も動作します。ちなみに以下の環境で動作を確認してあります。

    * Mac OS 9
        with MacPerl5.6.1r1 built for PowerPC

    * Mac OS X
        with perl v5.6.0 built for darwin

    * Windows2000
        with ActivePerl5.6.1 build631.

    * FreeBSD 4.6
        with perl v5.6.1 built for i386-freebsd

    * FreeBSD 3.4
        with perl v5.6.1 built for i386-freebsd
        with perl v5.005_03 built for i386-freebsd

    * Linux
        with perl v5.005_03 built for ppc-linux

    * Solaris 2.6 (SPARC)
        with perl5.6.1 built for sun4-solaris.
        with perl5.004_04 built for sun4-solaris.


このモジュールはまだまだ開発中で、あからさまな制約事項として

	* MD5ベースのcrypt()を使用する一部のFreeBSDでは、
	  PostgreSQLの'crypt'認証が使用できない。

	* PostgreSQLの'Kerberos v4'及び'v5'認証が使用できない。

	* SSL接続が使用できない。

	* BLOBフィールドが操作できない。

等が含まれます。またDBDとして要求されるメソッドは必要最小
限の物しか実装していません。
# 結局Net::PostgreSQLはDBD::PgPPに統合することにしました。
