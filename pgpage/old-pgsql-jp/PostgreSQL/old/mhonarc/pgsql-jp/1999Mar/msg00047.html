石井です。

PostgreSQL の性能を上げるために共有バッファを増やしたり、同時接続ユー
ザを増やそうとすると sharem memory や semaphore などの設定値をいじるこ
とになります。今まで断片的に出ていた議論をドキュメント化しようと思い立
ち、現在製作中です(作りかけのページは 
http://www.sra.co.jp/people/t-ishii/PostgreSQL/doc-jp/shmem.html にあ
ります)。

そこでいくつか疑問にあたったのですが、ご存知の方教えて下さい。

o FreeBSD において、shared memory のサイズを増やすには、options
SHMMAXPGS をいじるという話があったのですが、これのデフォルト値が定義さ
れている /usr/include/machine/vmparam.h を見に行くと、

#define SHMMAXPGS       1024            /* XXX until we have more kmap space */

とあります。これは、実際には 1024 より大きくしないほうが良い(できない)
ということでしょうか？

o Linux において、SHMMAX を大きくするときの制限としては物理メモリ以外
の制約があるでしょうか？また、FreeBSD の ipcs -M や Solaris の sysdef 
のように、shared memory のサイズを教えてくれるコマンドはあるでしょうか？

以上、よろしくお願いします。
