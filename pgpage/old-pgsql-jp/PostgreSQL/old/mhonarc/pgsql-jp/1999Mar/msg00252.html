  • From: Tatsuo Ishii <t-ishii@sra.co.jp>

石井です。

> 久しぶりに本家の Web ページを見に行ったら、6.5 での追加機能に、
> 
> 'A true NUMERIC type with user-specified precision is also planned for 6.5.'
> 
> ってのが書いてありました。
> この 'true NUMERIC type' って何の事でしょう？
> 
> ひょっとして、15桁とか 31桁の BCD がサポートされるんでしょうか？
> だったら嬉しいんですけどねぇ。

snapshot のソースを見ると、numric というデータ型が追加され、

	create table t1 (n numeric(31));

とかできるようになるはずですが...snapshot のパーサがまだ対応していない
ようで、テーブルは作れるものの、insert で相変わらず大きな数を受け付け
てくれません。ちなみに、ちゃんと動くようになれば、

	#define NUMERIC_MAX_PRECISION		1000

とあるので、1000 桁まで大丈夫になると思われます(デフォルトは 30)。
