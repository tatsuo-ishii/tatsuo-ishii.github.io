石井です。

# LinuxWorld 会場ではあまりの混雑でゆっくりお話もできませんで、もうし
# わけありませんでした。

>内田です。
>
>Linux 上の Apache + PHP + PostgreSQL が、 FileTable Full で落ちてしまう
>件に関して、LinuxWorld Expo 会場で :-) 石井さんに教えていただいた方法で
>なんとかなったようです。
>
>postmaster 起動時に bash の ulimit -n 20 を行っても、まだ足りないという
>状況だったのですが、OS 起動時に以下のようなファイルをつくって起動するこ
>とで対応しました。
>
>
>/etc/rc.d/rc3.d/S00kernel :
>
>     #!/bin/sh
>     echo 2048 > /proc/sys/kernel/file-max
>
># Default では cat /proc/sys/kernel/file-max すると、 1024 でした。

解決して良かったです。実は連休を利用して自宅でもいろいろやってみたので
すが、今一つ現象が再現できませんでした。file table の拡大によって対応
できることが確認できただけでも収穫ですね。

# 本当はこのような場合にも postmaster がおかしくならないようにすべきなの
# でしょうが。

ちなみに、6.5 では postmaster に "-N" オプションと言うのが追加され、実
行時にバックエンド数の max が指定できるようになっています。これは 
"soft limit" (デフォルト 32)で、コンパイル時に決まる "hard limit" は確
か 1024 位になっているはずです。
