> From: Go Tamura <t-go@geocities.co.jp>
> Date: Thu, 29 Jun 2000 13:18:09 +0900
> 
> > 一度作ったテーブルのデータ型を変更したいと思っているのですが、本を探して
> > も見つかることが出来ません。
> 
> PostgreSQL に限らず、一般にどの DBMS でもこういう変更はできないのでは？

少なくとも、Informix では可能です。
勿論、変換後の型とデ−タ−が合わなければエラ−になりますが。
# Informix の場合： ALTER TABLE 表名 MODIFY 列名 新しい型名 制約条件。
# 質問の趣旨とは関係ない話で失礼。

> 色々な方法はあるでしょうが、私だったら pg_dump で当該テーブルの
> バックアップを行い、pg_dump の出力ファイルをエディタで編集して
> 作り直す事にするでしょう。

どの方法でもいいわけですが、私だったら、デ−タ−の変換が必要なけ
れば pg_dump してから、psql で デ−タ−を LOAD する。 7.0 だったら、
非同期 COPY ができるから、COPY の方がいいですね。但し、COPY を
使うなら pg_dump -s で、変換前の DB 定義は保存しておく。

デ−タ−変換やチェックが必要なら、変換に使い易い言語に合った方を
選ぶ。 psql や  PHP で変換するなら、DB 表から DB 表へ。
awk や perl、UNIX のフィルタ−類を使うなら、COPY で TEXT FILE に
してから変換して DB に COPY。

DB 領域を圧迫する程大量のデ−タ−なら、迷わず pg_dump か COPY。
