自己レスです。

UnixWare 7.1.1 + PostgreSQL 7.0.2 で、Unix ドメインソケットで
接続できない旨投稿させていただきましたが、本家の ML でも同様の
投稿がありました。

これによりますと、UnixWare の accept(3) が、Unix ドメインの接
続時にアドレスファミリーとして AF_UNIX (= 1) ではなく AF_UNSPEC
(= 0) を返すためのようです。（本家の ML の投稿では、「UnixWare
のバグか、posix のスペックのバグか？」となっています。）

本家の ML は

http://www.postgresql.org/mhonarc/pgsql-ports/2000-05/msg00011.html

で見ることができます。
