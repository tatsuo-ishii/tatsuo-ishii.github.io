＜結合演算子について＞
以下のデータをLeftJoinの形でSQLを記述したいのですが
PostgreSQLではサポートされていないようで
他に方法が見つかりません。
どうすれば、よいのですか。よろしくお願いします。

DAT
 +-------------------+
 |      Code         |
 +-------------------+
 |      000001       |
 +-------------------+
 |      000002       |
 +-------------------+

MST
 +-------------------+--------------------+
 |      Code         |        NAME        |
 +-------------------+--------------------+
 |      000001       |        name1       |
 +-------------------+--------------------+

{SQL 1...}
SELECT DAT.Code MST.Name 
>FROM   DAT, MST
WHERE  DAT.CODE = MST.CODE
;
{SQL 1...RESULT}
 +-------------------+--------------------+
 |      Code         |        NAME        |
 +-------------------+--------------------+
 |      000001       |        name1       |
 +-------------------+--------------------+
この様な、結果になってしまい。DATのコード000002が抽出されません。
PostgreSQLで、DATのコード000002を抽出し、NameをNULLで
結果を取得する事は不可能なのでしょうか？

以上です。

−−−−−−−−−−−−−−−−−−−−−−−−−−−−−−−−−−−
ORACLE等でサポートされている”（＋）”のような演算子を使用すると
もちろんエラーになってしまいました。↓
{SQL 2...}
SELECT DAT.Code MST.Name 
>FROM   DAT, MST
WHERE  DAT.CODE = MST.CODE (+)
;
