確かにこれも重要な点のひとつですね。

むかし汎用機に、あるパッケージソフトをインストールして環境構築する
仕事をしたのですが、そのパッケージソフトがコアで使っている DBMS が
既にディスコンであったために非常に苦労した事があります。(^^;

某 H 製作所の直系 SE 会社で仕事をしていて、その DBMS も H 製だったの
ですが、ある業務を行うにはそのパッケージソフトを使うしかなくて新規
導入したのですが、ディスコン品ですから普通では手配できません。

直系会社の強みで、特別な手配で製品のテープは入手したのですが、
マニュアルがどこにもありません(古すぎて資料室からも廃棄されてました)。

古老(笑)をたずねたりしたのですが、古老も「ちょっと前なら覚えちゃいるが、
10年前だとちとわからねぇなぁ」状態で、ソフト工場に電話すると「そんな
製品はもうサポートしていない」と、けんもほろろな対応。
もう、泣きそうになりました。(^^; 最終的にはなんとかなりましたが…

Windows の世界でも、全国に散らばってる出張所に順次システムを入れて
いったら、途中で VB とか Access のバージョンが勝手に上がって手配
不能になり、しかも新バージョンではそのシステムがちゃんと動かない…
という羽目になった事もあります。
Windows では、同じような経験をした人はたくさんいるはずです。
