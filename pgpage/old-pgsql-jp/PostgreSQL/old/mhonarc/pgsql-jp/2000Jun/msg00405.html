  山田＠町田市です。

  MS や Windoz の話で賑やかでしたが、「性能が良い、とかプログラマにとって
使いやすい」とかという問題より、企業の場合、存在し選択の余地は無く、受け入れる
しかないのが現実だと思いますし、その上で善し悪しでは無しに、使いこなしてゆくの
が仕事だと考えています。(たまには腹も立ちますが、、、。)
  まー、見てくれの良さにはかなわないし、今の一般ユーザは「見てくれ最優先」なの
だから仕方が無いですよね。

  さて、 6.4.3 の時代に Excel からのインターフェースを VB でいくつか
作ったのですが、この一部が 7.0.2 にバージョンアップしたら動かないのですが、
皆様の所ではそのような事はないのでしょうか？  ひょっとしたら私の設定の問題
かと悩んでします。

  症状は：
    漢字の単語を条件に引っ張ってこようとすると、マッチングできない。
    表示の方は、ちゃんと問題無く日本語となる。
  環境は：
    postgreSQL 7.0.2 EUC_JP on FreeBSD-current
    Windows 2000 + Excel 2000 + 片岡さんの psqlodbc.dll(20000617)
   
  具体的には：
     create table company_list (
          id_company  int4,
          company_name text,    // 日本語
          company_name_eg text, // ローマ字
          address         text,
             :        :
     );
    というテーブルがあり、

    VB で
     sql = "select id_company,company_name from company_list"
     Set dao_rst = dao_con.OpenRecordset(sql)
     Do Until dao_rst.EOF
       Debug.Print Str(dao_rst!id_company) & " : " & dao_rst!company_name
       dao_rst.MoveNext
     Loop
     は問題無く動くのですが、

     sql = "select id_company from company_list where company_name like '三共%'"
     Set dao_rst = dao_con.OpenRecordset(sql)

     はマッチする物が無いとして何も返って来ません。
     ちなみに、ローマ字の名前で同様の事をさせると、ちゃんと返って来ます。

    ( 
     sql = "select id_company from company_list where company_name_eg like 'San%'"
     Set dao_rst = dao_con.OpenRecordset(sql)   
    )

   6.4.3 では問題無く動いていたのですが、7.0(7.0.2) になってうまくいっていない
   様です。  
   また、Excel の MSQuery を使って漢字を条件の抽出にも同様に失敗しています。

   推測するに、何か漢字コードの変換でトラブッているようなのですが、、、。
