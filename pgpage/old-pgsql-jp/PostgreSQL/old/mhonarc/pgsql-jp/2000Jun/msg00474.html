高橋と申します。

HP-UX 10.20 で pgsql_perl5 を単独でインストールする作業に
苦慮しています。開発環境のない計算機にインストールする必要があり、
開発環境のある計算機上で pgsql_perl5 をコンパイルした後に tar で
固めて、開発環境のない計算機で展開しています。

開発環境のある計算機上で make test により正常動作を確かめた後、
開発環境のない計算機上で以下の sample.pl を実行した結果を示します。

[sample.pl]
#! /opt/jperl5/bin/perl -Leuc
use Pg; 

[実行結果]
Can't load '/opt/jperl5/lib/site_perl/5.005/PA-RISC2.0/auto/Pg/Pg.sl'  for
module Pg: No such file or directory at
/opt/jperl5/lib/5.00503/PA-RISC2.0/DynaLoader.pm line 169.

----------------
ここで、ファイルがないと怒られる Pg.sl はありますし、Pg.sl に行き着くまでの
permission もあります。

次に、エラーが出ている箇所（DynaLoader.pm の 169 行目）の前後を見ると、

   # Many dynamic extension loading problems will appear to come from
    # this section of code: XYZ failed at line 123 of DynaLoader.pm.
    # Often these errors are actually occurring in the initialisation
    # C code of the extension XS file. Perl reports the error as being
    # in this perl code simply because this was the last perl code
    # it executed.

    my $libref = dl_load_file($file, $module->dl_load_flags) or
        croak("Can't load '$file' for module $module: ".dl_error()."\n");

とあります。
注釈を見ると、この行でエラーが出てもその原因はもっと前にあるということの
ようなのですが、エラーの原因を探る方法はないでしょうか？
また、同様のエラーが出たことのある方も何か情報を教えていただきたく思います。
