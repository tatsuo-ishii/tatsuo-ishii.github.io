 PgモジュールはlibpqをリンクしてPg.soと言うシェアードライブラリを
作成し利用しています。
これを作るためにはPostgreSQLのソース一式がPgをインストールする
マシン上に有る方が楽です。

 >２．CGIの文の中で、「pg::connectdb("dbname=$dbname");」にデータベース名を
 >入れますが、別のマシンからの接続の場合はどのように書けばよいのでしょうか？
 >本には、オプションとして書きいれるそうですが、実際にはどのように書けばよ
 >いのでしょうか？

 Perlのモジュールはそれ自身にヘルプを含んでいます。
 % perldoc Pg
 とやってみてください。

       1. Connection

       With these functions you can establish and close a
       connection to a database. In Libpq a connection is
       represented by a structure called PGconn.

       When opening a connection a given database name is always
       converted to lower-case, unless it is surrounded by double
       quotes. All unspecified parameters are replaced by
       environment variables or by hard coded defaults:

           parameter  environment variable  hard coded default
           --------------------------------------------------
           host       PGHOST                localhost
           port       PGPORT                5432
           options    PGOPTIONS             ""
           tty        PGTTY                 ""
           dbname     PGDATABASE            current userid
           user       PGUSER                current userid
           password   PGPASSWORD            ""

           $conn = Pg::connectdb("option1=value option2=value ...")

 Pgの利用方法としてはold styleとnew styleの記述が出来ますが
 今後のことを考えnew styleにしておいた方が良いです。
