この場合、make distcleanは必要ないでしょう。make distcleanを行うと、
configureの内容が全て破棄され、Makefile、オブジェクトファイル、実行
ファイル等が全て削除されます。
make distcleanは一度makeしたけど、別の設定でmakeしたいときなどに
使うといいと思います。

二度目のアーカイブの展開以降のみでいいのですが、ここで--with-tclを
つけて、configureを行ってください。出来れば、

$./configure --enable-multibyte=EUC_JP --with-tcl | &tee CONF.log

などとして、configureのログをとり、

checking setting USE_TCL... enabled

という行があるか確かめてみてはどうでしょうか？

ちなみにtclが使える場合は、
creating cache ./config.cache
checking host system type... i386-unknown-freebsdelf3.4

checking whether to support cyrillic recode... disabled

checking setting USE_PERL... enabled
checking setting USE_ODBC... enabled

...
などとなります。
