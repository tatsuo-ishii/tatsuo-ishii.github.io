> 失礼だとは思いますが、サポート体制・使用できるSQL・簡単にDB操作できるツール
> のどれをとっても
> オラクルのほうが使いやすいと思います。

それは慣れという面もあるので、評価が難しいところです。
サポート体制については、例えば SRA さんがやっているサポート契約などを
結んでからでないと比較のベースにのらないと思います。

> 対してLinux&Postgresのほうは、マニュアル通りに設定しているつもりでも動かな
> かったり、環境変数やその他の問題で正常に動作しないといったことが多々
> あります。

それは、あくまでも「つもり」であって、ちゃんと設定していないからです。
業界の古からの黄金律のひとつに、

 「プログラムはプログラマの意図に従って動くのではなく、書かれた通りに動く」

というものがあります。これはプログラム開発について述べたものですが、
環境設定などについても同様の事が言えます。
Linux であろうが、PostgreSQL であろうが、あなたが設定した通りに
動こうとしている(あるいは設定がおかしくて動かない)だけなのです。

環境設定などについては、UNIX 系 OS の方がよほど素直に行えます。
Windows 系こそ、理不尽な動きが多くて首を捻る事が多いというのが事実です。

これは別に Windows 系の使用経験が少なくて慣れていない。という事では
ありません。MS-DOS 2.11 や Windows 3.0 (3.1 ではない)の頃から使い続けて
いても、やはり理不尽なものは理不尽なのです。

これは、汎用機経験者を含めて、多くの(まともな)業界の人間が感じている
事です。

> そこで質問なのですが、LinuxとPostgreSQLの良いところってなんなのでしょうか？
> 自分としてはWindows2000が出た今、LinuxもPostgresも必要ないのではないかと
> 思ってしまっています。

そう思う人には、別に無理に PostgreSQL を使ってもらう必要はないでしょうね。
私自身は FreeBSD 派で、Linux はあまり好きではないですけど。:-p

私個人の見解としては Windows 系は OS としてはカスだし、VisualBasic も
言語としてはカスだという評価です。

> （それほどライセンス料が高いのですか？）

値段の話をすると、これは比較にならないほど高いです。一度自分で
値段を調べてみましょう。
