インターネットライセンスは、高いですね。
アクセス数とか規模に応じた販売料金の設定が難しいのかもしれない
けど、せめてサーバーの規模(Pentium II クラスシングル CPU とか、
4 CPU とか。。)に応じた料金設定があるといいんですが。


あと、商用ＤＢとの比較で重要なのは、保守の問題です。

商用ＤＢの年間サポート料金って、購入費用と同じくらいかかる場合
も多いし、サポート契約していないと、情報が入らないので、業務用
としては恐くて使えないです。
# 商用ＤＢのＭＬも、ベンダーの Web Page での情報もあるけど、
# 所詮ソースが公開されていないので、業務用で使うときは、やはり
# 正規の保守契約をしないと不安です。

PostgreSQL をはじめ、オープンソースのＤＢは、サポート内容に対
応した、各種の料金でのサポートを受けられるようになっていくと
思います。今は、サポート会社は少ないですが。

問題は、技術力の認定制度がないくらいかな。

勿論、自分であるいは自社内で保守することも可能です。

PostgreSQL でも、商用ＤＢのハイレベルの方のサポートと同クラス
の２４時間即対応の高度なサポートを求めれば、おなじような費用に
なるのかもしれません。

ただ、自社保守も含め、保守サービスの選択肢が多くなるということ
は、大きな魅力です。

また、ソースコードで問題を確認できるし、保守会社の技術力なども
ソースコードの理解度で推定できるとか、その場でコード変更して即
対応できるというのは、ソースコード非公開の商用ＤＢにない最大の
メリットでしょう。
