境田様、
本田です。こんばんわ。

UnixWareでも問題無く、pgbash-1.0-beta使えました。
Solaris同様、
- includeの変更
- signal.hのインクルード
が必要でした。

環境は、
UnixWare 7.1.0 
  gcc-2.95.1
  postgres6.5.1
  bash-2.0.3
です。

なお、上のbashを
configure、make、make install
するだけでは、
bash-2.03$ enable -f /usr/local/pgsql/lib/libpgbash.so exec_sql
bash: enable: cannot open shared object /usr/local/pgsql/lib/libpgbash.so: dynamic linker: bash: relocation error: symbol not found: list_optarg; referenced from: /usr/local/pgsql/lib/libpgbash.so
とエラーになりました。
bashのMakefileの「LDFLAGS 」に「-Xlinker -Bexport」を追加、
LDFLAGS =   $(STATIC_LD) $(LOCAL_LDFLAGS) $(PROFILE_FLAGS) $(CFLAGS) -Xlinker -Bexport
して、makeでOKでした。
