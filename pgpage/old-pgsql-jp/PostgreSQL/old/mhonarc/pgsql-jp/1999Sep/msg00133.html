http://www.psn.co.jp/PostgreSQL/pgbash/
を参考にpgbashのインストールをしているのですが、
うまくいきません。お助け下さい。

途中bash-2.02でうまくいかず、postgresはv6.4から6.5.1に変更。
更にbash-2.03に変更しています。しかし結果はまったく同じで、

/home/postgresql-6.5.1/src/interface/pgbash-1.0b2/test/
で make しても

make: *** [all] Error 1

となってしまいます。（これは全滅って事ですよね？）
そして、気になるのは
bash-2.02（2.03ではなく）をインストールした後、Makefileを

CFLAGS+= -I/usr/local/src/bash-2.02 -I$(SRCDIR)/interfaces/libpq

と修正しなければ途中でmake出来なかったという点です。
もちろんその時、
#bash -version
で確認してます。

そのときのメッセージ（修正していないMakefile）
linux:~/postgresql-6.5.1/src/interfaces/pgbash-1.0b2$ make
gcc -I../../include -I../../backend   -O2 -Wall -Wmissing-prototypes
-I./include  -I../../interfaces
/libpq -fpic   -c exec_sql.c -o exec_sql.o
In file included from include/shell.h:26,
                 from exec_sql.h:67,
                 from exec_sql.c:40:
include/general.h:189: parse error before `string_to_rlimtype'
include/general.h:189: warning: data definition has no type or storage
class
include/general.h:190: parse error before `int'

修正したMakefileでmake,make installしても上記の通りtestが通らず、
$ enable -f /usr/local/pgsql/lib/libpgbash.so exec_sql
すると
enable: unknown option: -f
と言われてしまいます。

構成はSlackware3.6.0 kernel 2.0.35 gcc-2.7.2.3
echo $POSTGRES_HOME
/usr/local/pgsql
echo $PGLIB
/usr/local/pgsql/lib
echo $PGDATA
/home/pgdata6.5
echo $PATH

          一部省略    :/usr/local/pgsql/bin:/usr/local/pgsql/lib
