> 例えば二台のWebサーバーとNFSサーバーを用意して、Postmasterの参照するディレ
> クトリをNFS上の同じディレクトリにするということは可能でしょうか?

NFS を使った場合の危険性についてはすでに指摘がありますが、PostgreSQL は
ネットワーク経由でアクセスできますので、無理にデータベースファイル自身を
ファイル共有で行なう必要はそもそもないと思うのですが…
そういう変な事をしなくてもよいようにネットワークアクセス機能があるのですし。

マシンを何台も用意できるのであれば、DB サーバマシンと Web サーバマシンを
わけるという方法もあります。2台の Web サーバマシンと 1台の DB サーバマシン
とか色々な構成が考えられます。

1台の DB サーバマシンに負荷が集中するのが嫌だというのであれば…
そういう時はパラレル DB の出番なのですが、PostgreSQL はそういう事が
できないので、^^; アプリケーションシステムが頑張って複数の DB 同士の
同期をとってゆく事になるでしょう。
