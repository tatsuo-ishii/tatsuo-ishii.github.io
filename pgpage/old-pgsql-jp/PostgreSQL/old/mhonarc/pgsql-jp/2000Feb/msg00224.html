もうすこし具体的なことをいえば、
glibc-2.1.2 で発生しています。

といっても、発生しない Linux ディストリビューションもあります。

私の調べた範囲では、TurboLinux Server 6.0 は発生しませんし、
最近出た日本語 RedHat 6.1 改定版 では発生します。

日本語 RedHat 6.1 改定版ででるといっても、
通常の postgres ユーザでは、LANG 設定されませんので
症状がでないです。
deluser postgres
useradd postgres
すると LANG 設定されるようになるようで症状が出ます。
TurboLinux Server 6.0 で症状が出ないというのも、
ロケールデータが 日本語文字で touppoer をしたときに
誤変換をする場合があるロケールデータなので、LANG の
状態にかかわらず症状はでません。
toupper の誤変換の症状がでるロケールデータは問題あり
という説明をlinux-users ML でうけましたが、詳しいことは
わかりません。

# 最終的には glibc の問題として Fix で修正される可能性がありますが、
# 最新の glibc-2.1.3 で症状の出るロケールデータでは、同様の問題が
# 出ていますので、当分修正されないとい状況が考えられます。
# 症状がでるロケールデータは多分 Debian で使われていたlocale-ja を
# ベースにしていると思います。
