こんにちは。

> S1: begin;
> S2: begin;
> S1: select * from a ...
> S2: select * from a ...
> S2: update set A ... -- ブロックされる (deadlock)
> S1: select * from a ... -- select もできない
> :
> :

なるほど、共有ロックが行われている場合は条件なしで
排他ロックに escalation できるとは知りませんでした。
とすると

FUKUDAさん wrote:
> テストバージョンの時にはLOCKを使っていたのですが、
> プログラム処理中にブラウザを切ってしまうと
> (もしくは送信中のクリック連打による2重送信)、
> LOCKがいつまでも切れない、というような状況が出てきました。
> なので恐くなって外してしまいました。

とおっしゃってますので、デッドロックして当たり前の状況に
あるということなのかな。
