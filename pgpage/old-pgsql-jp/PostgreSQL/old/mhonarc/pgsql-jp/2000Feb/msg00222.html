> バックエンドの postmaster は PostgreSQL の super user である
> postgres によって起動されますが、ユーザ postgres の環境変数
> LANG に ja_JP.ujis なとの値がセットされていると、データベースの
> unique index の値に日本語を入れた場合、文字列が違っていても
> 一意制約違反で insert できなくなることがあります。

こちらの環境は、FreeBSD 3.3R + PostgreSQL 6.5.3 で、postmaster は
su -l pgsql -c "postmaster -S -i" として起動しており、ユーザ pgsql
の LANG は ja_JP.EUC としています。

この環境で、おぎそさんと同じテストを行うと、

  kgotoh=> create table test (a varchar(10));
  CREATE
  kgotoh=> create unique index test_index on test (a);
  CREATE
  kgotoh=> insert into test values('あああ');
  INSERT 150540 1
  kgotoh=> insert into test values('ああい');
  INSERT 150541 1
  kgotoh=> 

と、なります。

glibc という事は、Linux をお使いなのでしょうか？
