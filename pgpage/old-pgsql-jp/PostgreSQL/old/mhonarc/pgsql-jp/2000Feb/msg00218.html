  • From: Tomoyoshi ASANO <asa@use-net.co.jp>

あさの です。

最初は自分の環境だけかとおもっていたのですが、いくつかの
linuxの日本のディストリビューションで同様の症状がでること
がわかってきたので質問することにしました。

# よくわかってない部分が多々あるので、ことの是非に
# 私は確信をもっていません。

postgresql-6.5.3 (patched-2000XXXX)で、

create table test (a varchar(10));
create unique index test_index on test (a);
insert into test values('あああ');
insert into test values('ああい'); <---- ここ
で、
Cannot insert a dupliacte key into a unique index
のエラーとなります。

この症状は postmaster を LANG=ja_JP.ujis など
日本語ロケールの環境変数を持つ状態で、postmaster を
起動すると症状がでます。
postgresql は --with-mb=EUC_JP で構築したものです。

いろいろなメジャーなディストリビューションで試してみましたが、
linux-users のML で過去に話題にしたことがあるのですが、
日本語文字を toupper すると、おかしな変換をするロケール
データでは、この症状はでないようです。そこで示唆された
この問題を解決するロケールデータだと上記の症状がでるようです。

で、質問なのですが、上記は glibc のバグに関連する事項だとは
思うのですが、なにが原因なのかもうすこし特定したいと考えて
いるので、postgresql のどのあたりのソースコードをみれば解析
できそうか、教授してもらえないでしょうか？

なにとぞ、よろしくお願いします。

-- あさの
