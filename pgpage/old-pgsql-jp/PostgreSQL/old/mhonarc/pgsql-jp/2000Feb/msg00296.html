＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
色鉛筆のデータベースです。
itemは商品のidと名前、在庫数を持っています
colorは共通の色リストです。
当然セットで仕入れてセットで売りますから、色毎に数量を
持つ必要も有りません。

商品データベース
create table item (
     id  int,
     name varchar(20),
     stock int
);

色リスト
create table color (
     cid int,
     color_name varchar(20)
);

データを入れます。
insert into item values(1,'sakura',10);
insert into item values(2,'momiji',5);
insert into item values(3,'bara',3);
insert into color values(1,'white');
insert into color values(2,'red');
insert into color values(3,'black');


さてこのデータベースから色鉛筆の総数を出したいとします。
（在庫スペースの計算では、セット数ではなくて、本数が
重要となってくると思います。）

select sum(item.stock) from item,color;

としたくなりませんか？

私ならきっとそうすると思います。

そのときの答えが
実際にやってみると
sum
---
 18
(1 row)
となります。

でもやっぱりほしいのは、54ではないでしょうか？

まったく架空の想定ですし、世の中こんなに簡単なケースは無いでしょうが
想定できないことも無いと思います。
