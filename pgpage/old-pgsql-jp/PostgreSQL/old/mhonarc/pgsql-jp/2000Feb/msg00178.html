Access はくわしくないんですが、処理方法としては Access 上の
MDB … かな？ 内部形式に変換した上で処理する場合があると
聞いています。

> PostgreSQLって外部結合をサポートしていないのでしょうか？

Outer Join は 6.5.3 でもサポートされていません。
外部キーが次にでる 7.0 でサポートされるらしいです。Outer Join に
ついては、ひょっとしたら 7.1 あたりで… という話もあるようですが、
早くて今年の夏頃ではないかとの事。

# と… 先日 PHP の ML で石井さんに伺いました。(^^;

ただ、Outer Join が PostgreSQL でサポートされても、その記法が Access
のそれと同じとは限らないわけでして、現状でも Oracle などとも記述方法が
異なる部分があったと思います。

ところで、Access の SQL ビューって、「SQL 生成のため」にあるもの
なのですか？
先に書いたように、私は Access を使ってないので細かいとこはアレなん
ですが、(^^; この機能は、

「自分が GUI (?) で指定したクェリーが本当に自分の意図通りになって
  いるかを SQL で確認するもの」

だと理解していました。つまり結果を見て「アレッ？」と思った時に
見たりするものではないかと… 違うのかな？ (^^;

SQL ビューで表示された SQL 文を使うと言っても、Access のプログラム自身で
使うのならばともかく(こっちが主用途なのかな？)、他の DBMS で使おうと
しても先に書いたように無意味なので、はじめからそんな事は考えませんでした。
