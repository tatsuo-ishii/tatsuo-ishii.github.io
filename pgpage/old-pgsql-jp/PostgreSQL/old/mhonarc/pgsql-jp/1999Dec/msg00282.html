  • From: mameta@justsystem.co.jp
  • Date: Thu, 16 Dec 1999 15:41:45 +0900
いつもお世話になっています。
表題の件で質問します。

CREATE TABLE aaa(
ID serial primary key,
name varchar(16)
);

このようなテーブルにデータを入れるとき、

　１．INSERT INTO aaa(name) VALUES('****_1');

としますよね。（ID=1)
次にIDを指定して追加します。

　２．INSERT INTO aaa VALUES(3,'*****_3');

次にIDの指定無しでまた追加します。

　３．INSERT INTO aaa(name) VALUES('****_2');

するとエラーが出ます。
ということは２番目でINSERTしたときユーザーはID=3で指定してますけど、
postgreSQL内部ではID=2を保持しているのですか？

しかし、そのエラー後もう一度３番とおなじSQLを実行すると今度は登録できます。
どういう理由なのでしょうか？

またSERIAL型のフィールで、値を指定したり指定しなかったりして、
追加するとフィールド値の整合は保てないのでしょうか？
そのへんの情報を何かお持ちでしたらご教授下さい。
