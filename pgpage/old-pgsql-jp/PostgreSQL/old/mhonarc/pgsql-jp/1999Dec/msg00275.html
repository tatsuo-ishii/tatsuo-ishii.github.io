o4l (Oracle for Linux) ML では今日はこの話題で燃え上がってます。(^_^)
基調は「日本オラクル許すまじ」という意見の人の声が大きいでしょうか。
ユーザとしては納得できん。という論調ですね。

私個人としても実は心穏やかではなかったり(職場には保守契約を結んでいる
Oracle8 WGS for SPARC Solaris 環境があります)もするのですが、
日本オラクルとしては for UNIX は Enterprise Server だけの世界に持って
いきたくて、Workgroup Server のユーザは NT や Linux 使ってくれよ。
という事ではないかと憶測してます。:-)

まぁ、保守契約更新する時にプラットフォームを変更(SPARC Solaris -> Linux)
できるってんなら、ある程度は納得できなくもないんですが、そんな事は
できないでしょうからねぇ…

> > PostgreSQL が次期バージョンの 7.0 (?) で、ログ機能や外部キー、外部接合、
> > そして Decimal,Numeric カラムへのインデックス作成などがサポートされれば
> 
> ちょっと進捗報告です。このうち「Decimal,Numeric カラムへのインデックス」
> はすでに開発が終ってます。外部キーの進捗も順調。外部結合はちょっと不安。
> ログは開発者の個人的都合で一時休止中。今本家のMLでは「8Kの壁」を越える
> 実装方法について盛んに議論が行われています。

外部結合がちょっと不安というのが不安ですね。(^^; ログ機能の開発が
一時休止ってのもちょっと残念です。

これらの機能はこれまで PostgreSQL の弱点として上げられて来た部分であり、
これらがないために PostgreSQL を敬遠する人達も実際にいらっしゃいます。

これらがみんな実装されれば PostgreSQL 7.0 の大きな目玉になるのは間違い
ないのでとても期待しています。

> > ビジネスチャンスかも知れませんよ。> 石井さん :-)

一番チャンスがあるのは MS SQL Server だったりして… ;-)
「やっぱり Windows2000 で SQL Server 7.0 でしょう」という事になったら
何となく嫌げ。(^^;

そういえば、話が大きくそれちゃうんですが、MS SQL Server って現在のところ
JDBC ドライバをサポートしていないのですよね？

最近は結構 Application Server ってのが世間的には話題になる事が多くて
Application Server といえば、Servlet だ EJB だという話になります。

各社の Application Server 製品は NT をサポート(近い将来はきっと 
Windows2000 も)してますが、この組合せでは JDBC ドライバをサポートして
いない DB 製品は選択肢に入りません。
まぁ、JNI でも使うというのなら少し話は違って来ますが、開発者からすれば
あまり嬉しくないでしょう。

MS はどうするんですかね？ それともすでに JDBC ドライバをサポートして
いるのでしょうか？
それとも MS が MS SQL Server 専用の Appliction Server を出してくるん
だろうか？
