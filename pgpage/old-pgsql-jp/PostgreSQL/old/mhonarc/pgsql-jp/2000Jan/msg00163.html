> 　Solaris2.7でPostgresql６．５．３をコンパイルで以下のエラーを
> 　吐きます。　何かアドバイスいただけるとありがたいのですが。
> 
> %./configure　--with-mb=EUC_JP
> -
> -
> checking whether the C compiler (gcc  ) works... yes
> checking whether the C compiler (gcc  ) is a cross-compiler... no
> checking whether we are using GNU C... yes
> checking whether gcc accepts -g... yes
> checking how to run the C preprocessor... gcc -E
> checking whether gcc needs -traditional... no
> - setting CPPFLAGS=
> - setting LDFLAGS=
> checking for c++... c++
> checking whether the C++ compiler (c++   ) works... no
> configure: error: installation or configuration problem: C++ compiler cannot create executabl

C++ が使用できない環境であるためにでるエラーです。
回避するには、

./configure --without-CXX --with-mb=EUC_JP

で良いと思います。

このオプションをつけると、 C++ 用のインターフェースである libpq++ 
が作成されませんが、それでも支障がないようでしたら、このオプションで
いけると思います。
