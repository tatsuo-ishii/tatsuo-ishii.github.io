SQL EDITOR です。SQL の文法は解析していません。
データベースへのアクセスも、SQL 文を入力することでのみできる、
psql 的なシンプルなアプリケーションです。
( マウスでどうこう、という機能はありません。作った方が、
「I am a keyboard junkie, 」 だそうで。そこも気に入っているところです。)

psql より良いところは、
1. 入力補完機能がついて、
2. 複数の SQL 編集用バッファがあって、
3. 別のペインでテーブル構造を見ながら操作できて、
4. 更に別のペインで、 select なんかの結果も見ながら操作できる。
といったところでしょうか。

文法を解釈しなので、「sel」まで打って TAB キーを押すと、「SELECT」と
補完してくれますが、「sel」から始まるテーブル名 / フィールド名があれば、
( SQL 文中のどこであろうと) TAB キーを押すと次々と
候補として表示してくれます。

候補となる SQL 予約語の定義も、結構アバウトで、
「えぇぇ、なんでこれが出てくれないの？」というもあったりするのですが、
まあ、多くなり過ぎても使いづらいでしょうから。
適当にカスタマイズ ( ソースを ) すると良いかと。

なので、 SQL の勉強よりも、 GTK と libpq の勉強に向いているかも。
