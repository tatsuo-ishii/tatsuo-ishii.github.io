[pgsql-jp 12910] Access from Applixware to PostgreSQL through PSQLODBC.

  • From: Jun Kuwamura <juk@rccm.co.jp>

桑村です。

  片岡さんのPsqlODBCの日本語パッチの差分を postgresql-6.3.2 の
ソースツリーにある src/interface/odbc に当てるたに作ってみ
ました。

  Applixware4.4.1 の新機能というので Applix Builder というのがあ
り、そのメニューバーの 「アプリケーション」-> 「データソース」で、
現れるウィンドウから [作成], [編集] をするともう一つウィンドウが
現れるので、そこのメニューバーの「検索」 -> 「サーバ選択」という
ので現れるノートブックのポップアップから 「ODBC」が使えます。
このノートブックには Informix, Oracle, Sybase のタグもあります
が残念ながら PostgreSQL はありません。

  ODBCの設定の仕方は、psqlodbc をインストールする以外に、ホーム
ディレクトリに .odbc.ini というファイルに記述をれば良いのですが、
doc/src/odbc.sgml を見てみると Applixware 用の設定についても書か
れていました。ちなみに、私が試しに記述した内容は：

--
[ODBC Data Sources]
PostgreSQL = PostgreSQL ORDBMS

[ODBC]
InstallDir=/tlj/opt/applix/axdata/axshlib

[PostgreSQL]
Driver=/usr/local/pgsql/lib/libpsqlodbc.so
Description = PostgreSQL DBMS Test
Database=postgres
Servername=localhost
Port=5432
--

です。一度試しただけですが、text 型のフィールドは BLOB  と表示さ
れ、Applix Builder で作ってみたアプリケーションではエラーになりま
した。まだ、それ以上は試していません。

  Applix Spreadsheets では、データソースの指定の仕方がまだわから
ないので、試してません。
