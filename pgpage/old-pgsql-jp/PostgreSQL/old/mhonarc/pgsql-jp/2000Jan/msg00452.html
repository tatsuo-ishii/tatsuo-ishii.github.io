私の Macintosh IIsi に NetBSD がインストールしてあるので検証したい
ところなのですが, 電池切れでブートできない状態です. あいまいな記憶
で申し訳ありませんが, /etc/rc.local には,

if [ -x /usr/local/pgsql/bin/postmaster ]; then
  rm -f /tmp/.s.PGSQL.5432
  su - postgres -c "/usr/local/pgsql/bin/postmaster -D /usr/local/pgsql/data -p 5432 &"
  echo -n 'postmaster '
fi

と書き加えてみてうまくいきませんでしょうか. 上記の /usr/local/pgsql
というディレクトリは実態に合わせて適宜変更されるとよいかと思います.

ちなみに, NetBSD のバージョンはいくつですか? NetBSD での PostgreSQL
について詳しい方がいらっしゃいましたら, 私もご教示いただきたく思い
ます.
