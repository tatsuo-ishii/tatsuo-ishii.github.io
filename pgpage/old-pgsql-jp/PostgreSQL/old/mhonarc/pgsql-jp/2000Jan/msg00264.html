はじめまして。細野と申します。

この度、社内でPostgreSQLを導入することになり、
PostgreSQL完全攻略ガイド(改訂版)で勉強中です。
Try!PCのWeb-DB連携システムの構築の記事を見て、
何とか動かしてみたレベルです。
よろしくお願い致します。

早速ですが、今、テーブルの作成でつまづいています。

create table abc (
	a numeric(8,0)[10],
	b int8[20]
);

の様に、numericやint8で配列を定義しようとすると、

ERROR: Unable to locate type name '_numeric' in catalog

の様なエラーが出て、テーブルが作成できません。

PostgreSQLは、postmaster -S -iで起動して、
登録は、上記定義文を、create.sqlの様な名前のファイルで作成して、

psql -f create.sql database_name

の形で行っています。

調べてみると、int4やcharだと、配列が作成できる様です。

基本的なミスの様な気がするのですが、
自分なりに情報を探してみましたが、分かりませんでした。
FAQでしたら情報の在処でもいいので、教えていただけないでしょうか。

環境は、

RedHat Linux 6.1J
PostgreSQL 6.5.2
(Apache 1.3.9)
(PHP3.0.12)

です。

参加してすぐにお聞きするのは非常に心苦しいのですが、
できましたらアドバイスなどお願い致します。
