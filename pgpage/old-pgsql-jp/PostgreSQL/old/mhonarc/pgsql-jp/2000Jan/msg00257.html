アドバイスありがとうございました。
tcl.h につなげられ、ようやく、postgresql-6.5.2のコンパイルができました。
早速、書籍「PostgreSQL完全攻略ガイド改訂版」に沿ってpgaccess-0.98を
インストールし,パッチを掛け、pgaccessを編集し、/usr/local/pgsql652/binに
コピーしました。
（postgresql6.5用に/usr/local/pgsqlは使用中ですので、新たにpgsql652を
作りました）
そして、pgaccessを実行してみました。

最初、load libpgtclがうまく起動できなかったので
/usr/local/pgsql/652/lib/pgaccess/main.tcｌでloadの行を変更しました。
あのPostgreSQL accessのメイン画面が表れるのですが、
Application initialization failed : invalid command name "tcl_findLibrary"
bgerror failed to handdle background error.
original error: innvalid command name "tkscreenchanged"
Error in bgerror; invalid command name "bgerror"
というメッセージがでて、残念ながらデータベースメニューの入力ができません。

そこで、書籍に従って、/usr/local/pgsql/652/lib/pgaccess/main.tcｌでloadの行
を消し、/usr/local/pgsql652/bin/pgaccessのPATH_TO_WISH
を変更しても結果は同じく
Application initialization failed : invalid command name "tcl_findLibrary"
です。
対応方法は何が考えられますでしょうか。
