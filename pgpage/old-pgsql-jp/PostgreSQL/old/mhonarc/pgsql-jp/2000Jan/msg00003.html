> INSERT 命令でVALUESの中に漢字日本語文字列を入力し、ブラウザ上で照会
> してみてもうまく漢字が表示できず、文字化けしてしまいます。
> PostgreSQLのインストール時、configureを実行する時、--with-mb=EUC_JP
> を設定しておりますので日本語環境はできているものとばかり思っていたのです
> が、どのように対応すればよいでしょうか

どうような手法で INSERT 命令用の data を入力したのか？というところが 
point になりそうな気がします。

方法として、とりあえず思いつくものは以下の 2 つです。
1)FreeBSD-2.2.8 上で動作する Netscape communicator を使って、画面上の 
form に入力する。これを PHP 経由で PostgreSQL の database に入力。
2)psql を使って table に直接入力。

まず 2) の方についてですが、これは原因として日本語 code が EUC になっ
ていない可能性が考えられます。したがって、入力する data を日本語 EUC 
に変換すれば、問題は解決するかもしれません。

次に 1) の方ですが、これはいくつかの原因が考えられます。
一つ目は、browser が、日本語を使えるように設定されていない場合です。こ
の場合、browser の処理段階で日本語文字列の data が落ちてしまい、結果 
PostgreSQL に誤った data が入力されてしまう危険性があります。これは、
日本の FreeBSD の web page から辿れる Q&A の Netscape についてのもの
(http://www.jp.freebsd.org/QandA/HTML/Netscape.html 以下にあります)が
参考になるかもしれません。

二つ目は、Netscape で入力される日本語 code が、日本語 EUC になっていな
い場合です。PHP の日本語 patch は非常に優秀で、このあたり処理を行なう
ような関数もあったような気がしますが、やっぱり最初から日本語 EUC に統
一しておくに及くは無いでしょう。
#これは、setenv LANG ja_JP.EUC とかいう感じで設定するのでしょうか…

三つめは、PHP が日本語対応になっていないことですが、これは参考文献を見
る限り心配無さそうですね。

#他に気にするなら、PHP script の日本語 code が EUC になっていなくて混
#乱しているとかいうのもあるのかもしれません。
