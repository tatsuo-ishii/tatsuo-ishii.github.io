ほとんどちゃちゃみたいなものなのですが，

> ちなみに、商用データベースの ORACLE では、トランザクションが COMMIT し
> ていなくても create table で作ったテーブルは他のトランザクションから見
> えてしまいます。また、ROLLBACK しても取り消されません。(ただし、その新
> しいテーブルに insert などでテーブル内のレコードを更新した結果は、
> COMMIT するまで他のトランザクションからは見えません。)たぶん、どんな商
> 用データベースでも create table の類はトランザクションになってないのだ
> とい思います。
> # これが SQL の標準から見て、正しい振舞なのかどうかは知りません。

Oracleの場合は，DDL (Data Definition Language) のだと，暗黙の commit 
があるとみなして，DDL を COMMIT してしまい， ROLLBACK できません．
この仕様がいいのかわるいのかは，よく知らないです．

もう一つの Oracle Rdb (かつての DEC Rdb) では DDL でもトランザクション
を構成しますので， ROLLBACK できたりします．テストのデータベースを作っ
ていろいろ実験したい時は，後者の仕様の方が便利(あとでロールバックすれ
ばいいから)ですけど，実際の運用環境では，DDL のロールバックの必要性は
うすそうですね．
