伊藤＠ＮＴＴです。

下記のような状況に困っております。

PostgreSQLとApacheのCGIを利用してサーバを構築しようとしています。

CGIはPerlを利用するため，/src/interfaces/perl5にあるPgというツールを利用
しています。

Pgを利用したPerlスクリプト自体は正常に動作するのですが，httpd経由であると
起動できません。突き詰めていくと，「use Pg;」のところでトラブルが起きてい
るようです。
＃DBにアクセスしなくても「use Pg」を書くだけでエラーになる。

httpdの権限の設定や，ライブラリの設定等，色々試みてみたのですが解決できま
せん。

同様の現象に遭遇されて解決された方がいらっしゃいましたら，是非ご教授くだ
さい。よろしくお願いいたします。


環境は，

FreeBSD 2.2.2
Apache 1.2.5
PostgreSQL 6.2.1
Perl 5.004_04

です。

試験で試したスクリプト（test.pl）は次のようなものです。

#!/usr/bin/perl
use Pg;
print "Content-type: text/plain\n\n";
print "This is test Script.\n";
