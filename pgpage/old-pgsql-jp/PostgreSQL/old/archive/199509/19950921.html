石井です。

postgres95-1.0 で日本語正規表現検索を可能にするパッチをお届けします。
SunOS4.1.3 (JLE) でしかテストしてませんが、他のプラットフォームでの使
用レポートやバグレポートなどをいただければ幸いです。

このパッチは、正規表現を使った検索で日本語 EUC コードに対応するために、
谷本孝浩さんが fjで発表された「MB extension v1.02 for GNU grep-2.0
(1/1)」(<DOHZONO.93Jul13110121@up904.sdsft.kme.mei.co.jp>)の中から
regex.[ch]、mbc.[ch] を借用しています。もともと、postgres は何もしなく
てもデータには EUC が使えたのですが(attribute などは駄目) 、これによっ
て ~ オペレータを使った text や char16 の日本語正規表現による検索が可
能となっています。

使い方：

1. 添付のパッチは gzip+uuencode されています。解凍すると、jp.patch と
いうファイルができるはずですので、postgres95-1.0 のソースを適当なディ
レクトリに展開した後、そこに行って

	% patch -p2 < jp.patch

とします。

2. 次に、src/Makefile.global の 124 行目あたりに

	# USE_REGEX = 1

という行があるので、コメントを外します。

3. 同ファイルの最後に

	#-include $(MKDIR)/../Makefile.custom

があるので、これもコメントを外します。

4. src/Makefile.custom というファイルを作り、以下の内容を入れておきま
す。

	JP = 1
	CFLAGS += -DJP

	(SunOS JLE の場合のみ）

	CFLAGS += -D_WCHAR_T

5. 以上です。後は普通にインストールするだけです。
