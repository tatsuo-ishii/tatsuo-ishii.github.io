遅巻きながら、私も PHP/FI 2.0b2 を入手して試してみました。こちらの環境
は、

	SS10/SunOS4.1.3 JLE
	postgres95 1.01J
	NCSA httpd (CGI redirect なし)

CGI redirect をやっていないので、使う時はちょっと面倒ですが、
http://host/cgi-bin/php.cgi/hogehoge.html のようにしてます。

ちょっと思ったことは、

	複雑な select 文を PHP 言語で動的に作るのはしんどいかも

ということです。桑村さんの例題のように、

>   $result = pg_Exec($conn, "select * from kisetu");

固定もしくは非常に簡単な query だったら良いのですが、ちょっと複雑をも
のを作ろうとすると大変です。

いっそ、tcl インタプリタを PHP に組み込んで、PHP から呼べるようにすれ
ば良いのでは？と思いました。

あるいは、PHP とほとんど同じことをする cgi を tcl で書くとか。

あと、当たり前ですが、regexp が日本語対応ではありませんね。これは 
pg95(j) と同じ手法で EUC が扱えるように改造できると思います。
