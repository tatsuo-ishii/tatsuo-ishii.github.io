PostgreSQL 6.0 から 6.1 への変更点

機能の点で 6.0 から変更のあったのは以下です。
  1. テーブル名やカラム名などの識別子の大文字、小文字の区別されなくなりました。
  2. GEQO (Genetic Query Optimization)と呼ばれる高度な問い合わせ最適化機能の追加。 GEQO は Genetic Algorithm (遺伝的アルゴリズム)を使って多くのテーブルを Join するときに威力を発揮するのだそうです(なんかすごそうですね:-) (see FAQ 3.42)
  3. SEQUENCE のサポート。これは、自動的に順序を生成する機能で、デー タに番号付をしたい時になどに使います。私が見た限りでは、Oracle の SEQUNECE とほぼ同じのようです。なお、この機能は psql のオンラインヘルプでは出て来ません。(see create_sequence.l)
  4. set コマンドの追加。実行時に動的にさまざまな振舞を変更できます。 たとえば、GEQO 機能のオン／オフ、日付フォーマットの変更ができます。 (see set.l)
  5. Oracle 互換関数のサポート。(see oracle_compat.3)
  6. regression テストの結果の確認が容易になった(まだバグあり。fail したからといって、必ずしもエラーではない)。
  7. "new plaintext password functions" とありますが、正体不明。psql に -u オプションを付けるとパスワードを聞いてきますが、肝心のパスワード ファイルの設定の仕方がわからない:-)また、これは tcl とか perl のインター フェイスからも使えるのだろうか？→この件に関して市川＠お茶大さんから貴 重な情報を頂きました。多少問題はあるようですが、市川さんはパッチなども 同封して下さったのでこれで解決！概略は以下です。
    • pg_hba.conf にパスワードファイルのあり場所を書く
    • psql の -u の処理にバグあり(対処パッチあり)
    • パスワードファイルを変更するためのツールが無い(試作ツールあり)
    • tcl/perl からもこの機能は使える
    なお、提供して下さったパッチは ftp://ftp.sra.co.jp/pub/cmd/postgres/6.1/pg_passwd.patch.gz にあります。

  8. 新しい組み込みデータ型 (see builtin.3)
    • money
    • circle
    • timespan
    • timestamp
  9. locale のサポート

6.1 の問題点

  1. バイトオーダーが異なるマシン間で通信ができない。例：Sun と Intel。 また、big endian のマシン(Sun など)は 6.0 とも通信ができない。 修正パッチあり。 なお、この問題は 6.1.1 では修正されています。
