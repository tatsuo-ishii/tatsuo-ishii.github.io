  • 待望のPostgreSQL 9.6.0がリリース[2016/09/29]

    前回のメジャーバージョンアップから8ヶ月、PostgreSQL 9.6.0がリリースされました。 主な新機能・改良は以下の通りです(たくさんの機能が追加されています。そこからかなり個人の趣味で抜粋しています)。

    • パラレルクエリが実装されました。今のところ、順スキャン、ジョイン、集約関数がパラレルクエリに対応押しています。
    • VACUUM FREEZE中に不要なページスキャンを避けるようになりました。
    • ストリームレプリケーションで、複数の同期スタンバイサーバを作れるようになりました。
    • 同期レプリケーションで、スタンバイにおいてデータのWALログがリプレイされてデータベースに反映されるまで待ってからコミット応答をクライアントに返すモードが追加されました。
    • postgres_fdwが、リモート側のジョイン、ソート、UPDATE、DELETEに対応するようになりました。
    • 複数のCPUソケットを持つハードウェアでの性能が向上しました。

    本家アナウンスはこちら。 日本語の情報は、SRAOSSが提供する技術情報をご覧ください。

    ダウンロードは以下から。

    • ftp://ftp.sra.co.jp/pub/cmd/postgres/9.6.0/postgresql-9.6.0.tar.gz

  • PostgreSQL 9.5.4などがリリース[2016/08/11]

    • PostgreSQL 9.5.4などのマイナーバージョンアップがありました。

    今回のリリースはバグ修正のほかに2つの脆弱性対策が含まれています。 脆弱性の一つは、CASE式の扱いに関するものです(CVE-2016-5423)。 もうひとつは、データベース名に二重引用符やバックスラッシュを含む場合に関するものです(CVE-2016-5424)。 これに関連して、Carriage Return や Line Feed を含むようなデータベース名は将来使えなくなる可能性があるとされています。

  • PostgreSQL 9.5.2などがリリース[2016/03/31]

    • PostgreSQL 9.5.2などのマイナーバージョンアップがありました。

    9.5.0/1で、Cロケール以外でデータベースを運用している場合、プラットフォームによっては文字型データ検索結果が不正になるバグが見つかっています(Cロケールで運用している場合には影響がありません)。 これは、9.5で導入された文字型データの検索の高速化機能のバグ(正確には、OSが提供している機能のバグ)によるもので、残念ながらどのプラットフォームにも適用できる修正方法が分かっていないため、この高速化機能自体が9.5.2では停止されることになりました。

    もうひとつの重要な修正として、発生確率は低いものの、データベースがクラッシュしたときにデータが失われる可能性があるバグが修正されました。

    また、行レベルセキュリティに関連する脆弱性(CVE-2016-2193) と、contrib/pageinspectに関する脆弱性CVE-2016-3065)が修正されたセキュリティフィックスリリースになっています。

    今回リリースされたのは以下のバージョンです。

    • 9.5.2
    • 9.4.7
    • 9.3.12
    • 9.2.16
    • 9.1.21

    ダウンロードは以下から。

    • ftp://ftp.sra.co.jp/pub/cmd/postgres/9.5.2/postgresql-9.5.2.tar.gz
    • ftp://ftp.sra.co.jp/pub/cmd/postgres/9.4.7/postgresql-9.4.7.tar.gz
    • ftp://ftp.sra.co.jp/pub/cmd/postgres/9.3.12/postgresql-9.3.12.tar.gz

  • 待望のPostgreSQL 9.5.0がリリース[2016/01/07]

    前回のメジャーバージョンアップから1年、PostgreSQL 9.5.0がリリースされました。 主な新機能・改良は以下の通りです(たくさんの機能が追加されています。そこからかなり個人の趣味で抜粋しています)。

    • UPDATE+INSERTを安全かつ効率的に実行する「UPSERT」が導入されました。
    • 順スキャンを効率良く行う新しいインデックス「BRINインデックス」が導入されました。
    • 行レベルのセキュリティ機能が導入されました。
    • データ分析用の新しい構文 CUBE, ROLLUP, GROUPING SETSが導入されました。
    • ロック競合の軽減により、検索処理の並列実行が更にスケールするようになりました。
    • 並列にVACUUMが実行できるようになりました。
    • ロックのかかっている行をスキップすることができるようになりました。

    本家アナウンスはこちら。 日本語の情報は、SRAOSSが提供する技術情報をご覧ください。

    ダウンロードは以下から。

    • ftp://ftp.sra.co.jp/pub/cmd/postgres/9.5.0/postgresql-9.5.0.tar.gz
